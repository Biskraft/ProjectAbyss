# 품질 체크리스트 (Quality Checklist)

## 1. 3-Tier 품질 등급

### L1: 필수 (Mandatory) - 이 조건을 충족하지 않으면 문서를 제출할 수 없음

모든 문서 공통:
- [ ] 문서 상단에 "필수 참고 자료 (Mandatory References)" 섹션 존재
- [ ] 구현 현황 테이블 존재 (기능 ID, 분류, 기능명, 우선순위, 상태, 비고)
- [ ] 기능 ID가 [접두사]-[번호]-[소문자] 형식을 준수
- [ ] 헤더 깊이 4단계 이하
- [ ] 헤더 형식: 한글 + 괄호 영문
- [ ] 스마트 따옴표 미사용 (직선 따옴표만)
- [ ] 테이블 셀 내 Bold 미사용
- [ ] 모호한 표현 없음 ("아마도", "할 수도 있다", "적절히", "대략" 등)
- [ ] 3대 기둥 중 최소 1개 정렬 명시
- [ ] 3-Space 분류 명시 (World/Item World/Hub)
- [ ] 본문에 수치 하드코딩 없음 (YAML/CSV로 분리)

### L2: 권장 (Recommended) - 좋은 문서의 기준

모든 문서 공통:
- [ ] Mermaid 다이어그램 최소 1개 포함
- [ ] 사용자 스토리(User Story) 포함
- [ ] Risk & Reward 양쪽 기술
- [ ] 관련 Reference 문서 링크 포함
- [ ] 설계 의도(Intent)가 첫 단락에 명시
- [ ] Cursed Problem 해당 여부 검토 기록
- [ ] CSV SSoT 링크 포함

### L3: 우수 (Excellent) - 모범적인 문서

모든 문서 공통:
- [ ] 수치 시뮬레이션 결과 포함 (성장곡선, 데미지 분포 등)
- [ ] 다른 GDD 문서와 상호참조 (최소 2개)
- [ ] 코칭 질문 응답 기록 포함
- [ ] Before/After 비교 또는 대안 분석
- [ ] 경쟁작 벤치마크 데이터 포함

## 2. 문서 유형별 추가 체크리스트

### SYS (System) 문서

L1 추가:
- [ ] 5단계 구조 완전: 개요(Concept), 메커닉(Mechanics), 규칙(Rules), 파라미터(Parameters), 예외처리(Edge Cases)
- [ ] 메커닉: Action -> Reaction -> Effect 형식
- [ ] 규칙: Condition -> Process -> Result 형식, 우선순위 정렬
- [ ] 파라미터: YAML 코드 블록, Snake_Case, 단위 접미사
- [ ] Edge Case 최소 3개 (네트워크 지연, 동시 입력, 자원 부족 포함)
- [ ] 검증 체크리스트 섹션 존재

L2 추가:
- [ ] flowchart TD 또는 stateDiagram-v2 다이어그램 포함
- [ ] 게이트 조건 명시 (스탯/능력/진행도)
- [ ] 다른 시스템과의 상호작용 명시

### WLD (World/Zone) 문서

L1 추가:
- [ ] 존 연결도 (graph LR) 다이어그램 포함
- [ ] 게이트 종류 및 해제 조건 명시
- [ ] 입구/출구 연결 정보

L2 추가:
- [ ] 존 테마 및 시각적 특징 서술
- [ ] 숨겨진 영역 및 비밀 경로 정보
- [ ] 재방문 시 새로운 콘텐츠 목록
- [ ] mindmap 다이어그램 포함

### IW (Item World) 문서

L1 추가:
- [ ] 층 구조 정의 (일반층/보스층/이벤트층)
- [ ] 시드 생성 규칙 명시
- [ ] 보상 테이블 (YAML)
- [ ] 재귀 진입 규칙 명시

L2 추가:
- [ ] 룸 템플릿 구성 규칙
- [ ] 난이도 스케일링 곡선
- [ ] Innocent 출현 테이블
- [ ] 탈출/구조 메카닉

### BAL (Balance) 문서

L1 추가:
- [ ] 수식을 코드 블록 내에 작성
- [ ] 입력 변수와 출력 변수 명시
- [ ] 수치 범위 (최소/최대/평균) 명시

L2 추가:
- [ ] 시뮬레이션 결과 (그래프 또는 테이블)
- [ ] 레벨별/티어별 수치 변화표
- [ ] 밸런스 조정 히스토리

### MP (Multiplayer) 문서

L1 추가:
- [ ] 동기화 방식 명시 (클라이언트/서버 권한)
- [ ] 파티 인원별 스케일링 규칙
- [ ] 접속 해제/재접속 처리

L2 추가:
- [ ] sequenceDiagram (클라이언트-서버 통신)
- [ ] 레이턴시 허용 범위
- [ ] 치트 방지 메커니즘

### CNT (Content) 문서

L1 추가:
- [ ] ID-First CSV 구조
- [ ] 카테고리/태그 분류
- [ ] Description 열 (한글)

L2 추가:
- [ ] 데이터 의존성 매핑
- [ ] 획득 경로 명시
- [ ] 희귀도/티어별 분포

### UI 문서

L1 추가:
- [ ] 화면 구성 요소 목록
- [ ] 데이터 바인딩 명세 (어떤 데이터가 어디에 표시되는지)
- [ ] 인터랙션 플로우

L2 추가:
- [ ] 와이어프레임 설명 또는 참조
- [ ] 상태별 UI 변화 (flowchart)
- [ ] 접근성 고려사항

### GDD (Generic) 문서

L1 추가:
- [ ] 목적과 범위 명시
- [ ] 관련 시스템 문서 링크

## 3. 자동 검증 가능 항목

다음 항목은 validator가 자동으로 검증합니다:

| 검증 항목 | 방법 | 심각도 |
| :--- | :--- | :--- |
| 스마트 따옴표 사용 | 정규식 검색 | Error |
| 헤더 5단계 이상 | `#####` 패턴 검색 | Error |
| 테이블 셀 Bold | 테이블 행 내 `**` 검색 | Warning |
| 수치 하드코딩 | 본문 내 naked number 패턴 | Warning |
| 모호한 표현 | 금지어 목록 매칭 | Error |
| Mermaid 블록 존재 | ` ```mermaid` 패턴 검색 | Info (SYS는 Warning) |
| 3-Space 테이블 존재 | "적용 공간" 헤더 검색 | Warning |
| Edge Case 섹션 | "예외 처리" 또는 "Edge Case" 헤더 검색 | Error (SYS만) |
| YAML 파라미터 블록 | ` ```yaml` 패턴 검색 | Warning (SYS는 Error) |

## 4. 검증 결과 보고 형식

```
=== GDD 품질 검증 보고 ===
문서: [파일명]
유형: [SYS/WLD/IW/BAL/MP/CNT/UI/GDD]
등급: [L1 미달 / L1 / L2 / L3]

[Error] (N개)
- [항목]: [설명]

[Warning] (N개)
- [항목]: [설명]

[Info] (N개)
- [항목]: [설명]

L1 충족: O/X (N/M 항목)
L2 충족: O/X (N/M 항목)
L3 충족: O/X (N/M 항목)
```
