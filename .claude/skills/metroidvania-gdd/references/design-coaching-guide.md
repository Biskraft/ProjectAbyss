# 설계 코칭 가이드 (Design Coaching Guide)

## 1. 코칭 3단계 프로세스

### Stage A: 의도 검증 (Intent Verification)
- 사용자의 설계 요청에서 "왜?"를 파악
- 3대 기둥 중 어디에 정렬되는지 확인
- 의도가 불명확하면 코칭 질문으로 명확화

### Stage B: 위험과 보상 검증 (Risk & Reward Verification)
- 이 설계가 실패하면 어떤 문제가 생기는가?
- 이 설계가 성공하면 어떤 경험을 제공하는가?
- 피크 모먼트(Peak Moment)는 무엇인가?

### Stage C: 반론 검증 (Counter-Argument Verification)
- "만약 이 기능이 없다면?" 테스트
- "최악의 경우 어떻게 되는가?" 시나리오
- 더 단순한 대안이 있는가?

## 2. 3대 기둥 (Three Design Pillars)

### 기둥 1: Metroidvania 탐험 (Exploration)
핵심 가치: 능력 게이트 + 스탯 게이트 기반 비선형 월드 탐색, 재방문 보상
검증 질문: "이 시스템이 재방문/탐험 동기를 강화하는가?"

관련 키워드: 맵, 존, 게이트, 능력, 스탯 요구치, 숨겨진 영역, 비선형, 분기, 백트래킹, 시퀀스 브레이크

설계 원칙:
- 모든 잠긴 경로는 최소 2가지 방법으로 해제 가능해야 한다 (스탯 게이트 OR 능력 게이트)
- 재방문 시 새로운 보상이 있어야 한다 (숨겨진 Innocent, 새로운 통로)
- 맵은 "지름길"을 통해 이전 구역과 연결되어야 한다

### 기둥 2: Item World 야리코미 (Yarikomi/Grinding)
핵심 가치: 장비 내부 던전, Innocent 시스템, 무한 성장 루프
검증 질문: "이 시스템이 반복 플레이의 깊이와 보상감을 제공하는가?"

관련 키워드: Item World, 층, 보스, Innocent, Wild, Tamed, 합성, 번식, 희귀도, 재귀 진입, 시드

설계 원칙:
- 반복은 "같은 것의 반복"이 아닌 "변주"여야 한다 (시드 기반 절차적 생성)
- 깊이 진행에 비례하는 보상 스케일링 (리스크=리워드)
- "한 판 더"의 동기를 만드는 중간 체크포인트 (10층마다 보스)

### 기둥 3: Online 멀티플레이 (Online Multiplayer)
핵심 가치: 실시간 협력/경쟁, 자동사냥, Hub 사교
검증 질문: "이 시스템이 혼자서도 재미있고 함께하면 더 재미있는가?"

관련 키워드: 파티, 협력, PvP, Hub, 자동사냥, 원정대, 거래, 채팅, 길드

설계 원칙:
- 솔로 플레이 기준으로 밸런싱하고, 멀티는 "보너스"로 느껴져야 한다
- 자동사냥은 직접 플레이 대비 60~70% 효율로 제한
- 사회적 기능은 Hub 공간에 집중

## 3. 저주받은 문제 (Cursed Problems) 7종

"저주받은 문제"란 어떤 해결책을 선택해도 다른 문제를 야기하는 설계 딜레마입니다. 이러한 문제를 인식하고 트레이드오프를 명시하는 것이 중요합니다.

### CP-1: 게이트 좌절 vs 탐험 자유
설명: 스탯/능력 게이트가 너무 많으면 답답하고, 적으면 비선형성이 상실됨
관련 기둥: Metroidvania 탐험
트레이드오프 축:
- 게이트 많음 -> 명확한 진행 구조, 성취감 있는 해금. BUT 답답함, 자유도 저하
- 게이트 적음 -> 자유로운 탐험, 시퀀스 브레이크 가능. BUT 방향성 상실, 밸런스 붕괴
완화 전략: 주요 경로에는 필수 게이트, 비밀 경로에는 선택적 게이트. 스탯 게이트는 장비로 우회 가능.

### CP-2: 야리코미 깊이 vs 접근성
설명: Item World 100층 구조가 코어 유저에겐 보상이지만 캐주얼 유저에겐 벽
관련 기둥: Item World 야리코미
트레이드오프 축:
- 깊이 강조 -> 코어 유저 만족, 장기 리텐션. BUT 캐주얼 이탈, 진입장벽
- 접근성 강조 -> 폭넓은 유저층, 빠른 보상. BUT 깊이 부족, 코어 유저 이탈
완화 전략: 10층 단위 세이브포인트, 처음 30층은 튜토리얼 난이도, 자동사냥으로 낮은 층 스킵 가능.

### CP-3: 자동사냥 편의 vs 플레이 가치
설명: 자동이 너무 효율적이면 직접 플레이의 의미가 소실됨
관련 기둥: Online 멀티플레이
트레이드오프 축:
- 자동 효율 높음 -> 편의성, 바쁜 유저 유지. BUT 게임 플레이 의미 퇴색
- 자동 효율 낮음 -> 직접 플레이 가치 유지. BUT 불편, 경쟁 게임 대비 불리
완화 전략: 자동사냥 효율 상한 60~70%, 직접 플레이 전용 보상(희귀 Innocent, 숨겨진 방), 자동사냥 불가 구역.

### CP-4: 솔로 밸런스 vs 파티 밸런스
설명: World(1~2p)와 Item World(1~4p)의 난이도 스케일링 딜레마
관련 기둥: Metroidvania 탐험 x Online 멀티플레이
트레이드오프 축:
- 솔로 기준 밸런싱 -> 혼자서도 클리어 가능. BUT 파티 플레이 시 너무 쉬움
- 파티 기준 밸런싱 -> 협력의 필요성. BUT 솔로 플레이어 벽
완화 전략: 동적 난이도 스케일링(파티 인원에 따라 몬스터 HP/공격력 조정), 솔로 기준 설계 후 파티 보너스 추가.

### CP-5: 시드 재현성 vs 신선함
설명: Spelunky식 시드 기반 생성에서 패턴 학습 후 반복감
관련 기둥: Metroidvania 탐험 x Item World 야리코미
트레이드오프 축:
- 시드 고정 -> 공략 공유 가능, 스피드런 문화. BUT 반복감, 예측 가능
- 완전 랜덤 -> 매번 신선. BUT 공략 불가, 불공정 느낌
완화 전략: Item World 시드는 hash(itemID + level + floor)로 결정하되, 룸 템플릿 풀을 지속 업데이트. 특수 이벤트는 별도 랜덤.

### CP-6: Innocent 복잡도 vs 이해도
설명: Wild/Tamed/합성/번식 시스템이 깊지만 진입장벽이 높음
관련 기둥: Item World 야리코미
트레이드오프 축:
- 복잡한 시스템 -> 깊은 커스터마이징, 전략성. BUT 이해 어려움, 튜토리얼 부담
- 단순한 시스템 -> 쉬운 이해, 빠른 적응. BUT 깊이 부족, 야리코미 동기 약화
완화 전략: 단계적 해금(기본 Innocent -> 합성 -> 번식), 인게임 도감으로 정보 제공, 자동 추천 시스템.

### CP-7: 순환 구조 강제 vs 자율
설명: World <-> Item World <-> Hub 순환이 의무적이면 자유도가 하락
관련 기둥: 전체
트레이드오프 축:
- 강제 순환 -> 모든 콘텐츠 경험, 설계 의도대로. BUT 자유도 저하, 답답함
- 자율 순환 -> 플레이어 선택 존중. BUT 콘텐츠 편식, 밸런스 이슈
완화 전략: 주요 퀘스트는 순환 유도하되 강제하지 않음. 각 공간에 독립적 보상이 있어 자연스러운 이동 동기 부여.

## 4. 3대 기둥 정합성 매트릭스

새로운 시스템 설계 시, 다음 매트릭스로 검증:

| 검증 항목 | 탐험 | 야리코미 | 멀티플레이 |
| :--- | :--- | :--- | :--- |
| 이 시스템이 해당 기둥을 강화하는가? | ? | ? | ? |
| 이 시스템이 해당 기둥을 약화시키는가? | ? | ? | ? |
| 이 시스템 없이 해당 기둥이 성립하는가? | ? | ? | ? |

규칙:
- 최소 1개 기둥을 강화해야 함
- 어떤 기둥도 약화시키지 않아야 함
- 2개 이상 기둥을 강화하면 우수

## 5. Risk & Reward 검증 체크리스트

### 리스크 검증
- [ ] 이 시스템이 실패하면 게임 전체에 미치는 영향은?
- [ ] 이 시스템으로 인해 발생할 수 있는 어뷰징은?
- [ ] 기술적 구현 난이도와 성능 영향은?
- [ ] 다른 시스템과 충돌 가능성은?

### 보상 검증
- [ ] 플레이어가 이 시스템에서 얻는 핵심 감정은?
- [ ] 이 시스템의 피크 모먼트(가장 짜릿한 순간)는?
- [ ] 리텐션에 기여하는 방식은?
- [ ] 수익화와의 관계는?

## 6. 코칭 질문 뱅크 (30문항)

### Intent (의도) Q1~Q5
- Q1: "이 시스템이 없으면 게임이 어떻게 달라지나요?"
- Q2: "이 시스템을 통해 플레이어에게 전달하고 싶은 핵심 감정은?"
- Q3: "이 시스템이 3대 기둥 중 어디에 기여하나요?"
- Q4: "비슷한 시스템을 가진 다른 게임에서 가장 잘 된 점과 못 된 점은?"
- Q5: "이 시스템의 MVP(최소 기능 제품)는 어떤 모습인가요?"

### Risk & Reward Q6~Q10
- Q6: "이 시스템이 완전히 실패하면 어떤 상황이 벌어지나요?"
- Q7: "플레이어가 이 시스템을 악용할 수 있는 방법이 있나요?"
- Q8: "이 시스템의 피크 모먼트(가장 짜릿한 순간)를 묘사해주세요"
- Q9: "이 시스템이 서버 부하에 미치는 영향은?"
- Q10: "이 시스템 없이 같은 경험을 제공할 더 단순한 방법은?"

### Balance Q11~Q15
- Q11: "솔로 플레이어와 파티 플레이어 모두에게 공정한가요?"
- Q12: "신규 유저와 기존 유저 간 격차를 얼마나 벌리나요?"
- Q13: "자동사냥과 직접 플레이 간 효율 차이는?"
- Q14: "이 시스템의 수치가 다른 시스템 수치와 충돌하지 않나요?"
- Q15: "100층까지 갔을 때와 1층의 보상 차이가 적절한가요?"

### UX Q16~Q20
- Q16: "처음 접하는 플레이어가 3분 안에 이해할 수 있나요?"
- Q17: "이 시스템의 피드백(시각/청각)은 충분한가요?"
- Q18: "모바일/웹 환경에서 조작이 불편하지 않나요?"
- Q19: "정보 과부하 없이 핵심 정보만 보여주고 있나요?"
- Q20: "실패했을 때 플레이어가 '왜 실패했는지' 이해할 수 있나요?"

### Pillar-Specific Q21~Q30
- Q21: (탐험) "이 시스템이 맵의 재방문 가치를 높이나요?"
- Q22: (탐험) "새로운 능력을 얻었을 때 이전 맵에서 뭘 할 수 있나요?"
- Q23: (야리코미) "100번 반복해도 의미 있는 변주가 존재하나요?"
- Q24: (야리코미) "Innocent 시스템과 어떻게 연동되나요?"
- Q25: (야리코미) "Item World 깊은 층으로 갈수록 전략이 달라지나요?"
- Q26: (멀티) "파티원이 중간에 나가면 어떻게 되나요?"
- Q27: (멀티) "혼자서도 재미있고 함께하면 더 재미있나요?"
- Q28: (멀티) "PvP와 PvE 밸런스가 서로 영향을 주지 않나요?"
- Q29: (순환) "World -> Item World -> Hub 순환이 자연스러운가요?"
- Q30: (순환) "각 공간에 돌아갈 독립적인 이유가 있나요?"
