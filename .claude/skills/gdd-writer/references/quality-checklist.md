> gdd-writer 스킬의 문서 유형별 완성도 체크리스트. L1 필수 / L2 권장 / L3 우수.

# 문서 품질 체크리스트 (Quality Checklist)

## 개요 (Overview)

GDD 문서의 완성도를 3단계로 평가한다.

| 등급 | 의미 | 기준 |
| :--- | :--- | :--- |
| L1 | 필수 | 이 항목이 없으면 문서로서 기능하지 않음 |
| L2 | 권장 | 이 항목이 있으면 구현 팀이 추가 질문 없이 작업 가능 |
| L3 | 우수 | 이 항목이 있으면 설계 의도까지 전달됨 |

---

## 1. 공통 체크리스트 (All Documents)

모든 GDD 문서에 적용되는 체크리스트.

### L1 필수 (Mandatory)

- [ ] 메타데이터 존재 (최근 업데이트 날짜, 문서 상태)
- [ ] 구현 현황 테이블 존재 (기능 ID, 분류, 기능명, 우선순위, 구현 상태)
- [ ] 필수 참고 자료 섹션 존재 (Writing Standards, Project Definition, 관련 시스템)
- [ ] 예외 처리 최소 3건 (네트워크, 동시 입력, 자원 부족)
- [ ] 폐기 용어 미사용 (Plasma, Diamond, Emerald, Plastic, Uranium, Scrap Metal)
- [ ] 헤더 형식 준수 (한글 + 괄호 영문)
- [ ] 헤더 최대 4단계 (5단계 이상 금지)
- [ ] 직선 따옴표만 사용 (스마트 따옴표 금지)
- [ ] 테이블 셀 내 Bold 금지
- [ ] 모호한 표현 미사용 ("적절히", "빠르게", "많은")

### L2 권장 (Recommended)

- [ ] Mermaid 다이어그램 최소 1개 존재
- [ ] YAML 파라미터 블록 존재 (본문 수치 하드코딩 없음)
- [ ] CSV SSoT 링크 존재 (상대 경로)
- [ ] 검증 체크리스트 존재 (문서 하단)
- [ ] 상호 참조 링크 유효 (관련 문서 링크)

### L3 우수 (Excellent)

- [ ] 설계 의도(왜?) 명시 (Intent, Reasoning)
- [ ] 코칭 질문 최소 3개 포함
- [ ] 검증 기준(Verification Checklist) 구체적 항목 5개 이상
- [ ] 대안 분석 포함 (채택하지 않은 방법과 그 이유)
- [ ] 피드백 명세 (시각/청각/촉각 피드백)

## 2. System_*.md 추가 체크리스트

System_ 접두사 문서에 추가로 적용되는 항목.

### L1 필수

- [ ] 5단계 구조 완전 (개요, 메커닉, 규칙, 파라미터, 예외처리)
- [ ] 개요에 Intent, Reasoning 존재
- [ ] 개요에 Cursed Problem Check 존재
- [ ] 개요에 Risk & Reward 명시
- [ ] 메커닉이 Verb 중심 (Action -> Reaction -> Effect)
- [ ] 규칙에 조건 -> 처리 -> 결과 형식
- [ ] 파라미터에 YAML 코드 블록 (변수명 영문 Snake_Case, 단위 주석)

### L2 권장

- [ ] flowchart TD (유저 플로우) 존재
- [ ] stateDiagram-v2 (상태 전이) 존재
- [ ] 피드백 명세 존재 (시각, 청각, 촉각 각각)
- [ ] 규칙에 우선순위 정의 (충돌 시 적용 순서)
- [ ] 다른 시스템과의 상호작용 규칙 최소 2개

### L3 우수

- [ ] graph LR (시스템 관계도) 존재
- [ ] 관련 시스템 문서 링크 3개 이상
- [ ] 3대 기둥 연결 명시
- [ ] 밸런스 지표 정의 (모니터링 대상 수치)
- [ ] 프로토타입 링크 또는 플레이테스트 결과

## 3. UI_*.md 추가 체크리스트

UI_ 접두사 문서에 추가로 적용되는 항목.

### L1 필수

- [ ] 와이어프레임 존재 (이미지 또는 ASCII 레이아웃)
- [ ] 데이터 바인딩 명시 (어떤 데이터가 어디에 표시되는지)
- [ ] 입력 방식 정의 (마우스, 키보드, 게임패드)
- [ ] UI 요소 ID 목록

### L2 권장

- [ ] stateDiagram-v2 (UI 상태 전이) 존재
- [ ] 스타일 파라미터 존재 (폰트 크기, 색상, 투명도)
- [ ] 애니메이션/전환 효과 정의 (페이드, 슬라이드 등)
- [ ] 해상도별 레이아웃 대응 규칙

### L3 우수

- [ ] 프로토타입 링크 (Prototypes/ 폴더)
- [ ] 접근성 지원 (색맹, 글자 크기 조절)
- [ ] 성능 고려사항 (UI 업데이트 빈도, 배칭)

---

## 4. Gadget_*.md 추가 체크리스트

Gadget_ 접두사 문서에 추가로 적용되는 항목.

### L1 필수

- [ ] 발동 조건 명시 (입력, 자원 비용, 쿨다운)
- [ ] 효과 범위 명시 (반경, 지속 시간, 대상 수)
- [ ] GadgetList CSV ID 연결
- [ ] 종료 조건 명시 (시간, 파괴, 수동 취소)

### L2 권장

- [ ] 시너지 관계 정의 (어떤 가젯/무기와 시너지)
- [ ] 카운터 관계 정의 (어떤 가젯/행동으로 무력화)
- [ ] 전투 시나리오 최소 2개 (공격 활용, 방어 활용)
- [ ] flowchart TD (발동 조건 분기)

### L3 우수

- [ ] sequenceDiagram (발동 -> 효과 -> 종료 타임라인)
- [ ] 팀 전술 활용 시나리오
- [ ] 밸런스 비교 (동급 가젯 대비 장단점)

---

## 5. Design_*.md 추가 체크리스트

Design_ 접두사 문서에 추가로 적용되는 항목.

### L1 필수

- [ ] 3대 기둥과의 연결 명시
- [ ] 트레이드오프 명시 (무엇을 얻고 무엇을 잃는가)
- [ ] 설계 원칙 명문화 (명확한 규칙으로 표현)

### L2 권장

- [ ] 레퍼런스 체인 (참고한 게임/자료와 차용 포인트)
- [ ] mindmap (설계 원칙 관계도)
- [ ] 적용 범위 정의 (이 원칙이 영향을 미치는 시스템 목록)

### L3 우수

- [ ] 대안 분석 (검토했으나 채택하지 않은 방향과 이유)
- [ ] 저주받은 문제 식별 및 선택 기록
- [ ] 향후 확장 가능성 (이 원칙이 확장될 수 있는 방향)

---

## 6. Content_*.md 추가 체크리스트

Content_ 접두사 문서에 추가로 적용되는 항목.

### L1 필수

- [ ] CSV 링크 존재 (SSoT 원칙)
- [ ] 에셋 목록 존재 (ID, 이름, 상태)
- [ ] 분류 체계 정의 (카테고리, 타입, 태그)

### L2 권장

- [ ] 상태 추적 테이블 (제작 필요, 진행 중, 완료)
- [ ] ID 규칙 준수 확인 (UPPER_SNAKE_CASE, 대분류_중분류_식별명)
- [ ] 수량/범위 명시 (각 카테고리별 에셋 수)

### L3 우수

- [ ] erDiagram (데이터 관계도)
- [ ] 우선순위 매트릭스 (P1/P2/P3 분류)
- [ ] 에셋 제작 가이드라인 (톤 & 매너, 기술 스펙)

## 평가 요약 템플릿 (Assessment Summary Template)

문서 검토 시 아래 형식으로 평가 결과를 요약한다.

```markdown
## 문서 품질 평가

| 구분 | L1 필수 | L2 권장 | L3 우수 |
| :--- | :-----: | :-----: | :-----: |
| 공통 | O/X개 충족 | O/X개 충족 | O/X개 충족 |
| 유형별 | O/X개 충족 | O/X개 충족 | O/X개 충족 |

### 미충족 항목
- [L1] 항목명 -- 수정 방법
- [L2] 항목명 -- 수정 방법

### 총평
- 등급: L1 / L2 / L3
- 다음 등급까지 필요 작업: ...
```
