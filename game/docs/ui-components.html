UI COMPONENT CATALOG

이 페이지는 ECHORIS 게임의 모든 UI 컴포넌트와 디자인 토큰의 단일 진실 공급원(SSoT)이다.
소스 파일: game/src/ui/ · 토큰 정의: ModalPanel.ts

사용 규칙
① 새 UI 작업은 반드시 이 페이지의 컴포넌트 목록을 참고한다.
② 목록에 없는 컴포넌트/토큰은 사용 금지. 필요하면 먼저 src/ui/ 와 이 페이지에 추가하라.
③ 색상·폰트 사이즈·여백은 모두 ModalPanel.ts 의 토큰만 사용. 하드코딩 금지.
④ 새 색/사이즈가 정말 필요하면 ModalPanel.ts 에 토큰을 추가한 뒤 이 페이지의 토큰 섹션도 동기화한다.

Selection State FOUNDATION

SSoT: ROW_SELECTED / SLOT_SELECTED tokens · 모든 리스트·메뉴 컴포넌트가 이 패턴을 따른다

플레이어가 "지금 어디에 커서가 있는가" 를 한 번에 인식하지 못하면 메뉴 UI는 실패한다. Bloodstained: Ritual of the Night 의 메뉴를 기준으로, 선택 상태는 다음 4 레이어를 동시에 적용한다. 어느 한 레이어도 단독으로는 충분하지 않다.

Key Color · Orange (#ff9933)
청록 액센트는 다크 네이비 모달 패널에서 묻힌다. 선택 상태는 프로젝트의 워밍 톤(주황 단조 팔레트)과 일치하는 오렌지 키컬러로 통일한다. 장착 상태(#ff8c00 bar)와 색상이 겹칠 때는 펄스 + 챠브론 + [E] 배지로 구분한다.

4-Layer Recipe

① Saturated Fill — 비선택과 채도 차이가 충분한 배경 (단색 아님, 좌→우 그라디언트로 깊이 부여)
② Full-Alpha Edge — 2px 액센트 보더, alpha 0.95+ (희미한 outline 금지)
③ Symmetric Chevrons — ▶ (좌) + ◀ (우) 양쪽에 액센트 색상 표시
④ Outer Halo + Pulse — 외부 box-shadow 글로우 + 1.6s 펄스 애니메이션 (정적 UI에서도 살아있는 느낌)

SELECTED — 4-layer 적용
▶ ⚔ Iron Sword ATK 12
▶ ⚔ Steel Blade ATK 28
▶ ⚔ Frost Edge ATK 34

Anti-Pattern (이전 버전, 사용 금지)

다음 예시는 실제 플레이테스트에서 "선택이 명확하지 않다" 피드백을 받은 구버전이다. 단일 outline 35% alpha + ▶ 1개로는 화면 활동량이 많을 때 묻혀버린다.

DEPRECATED — 단일 outline 35% alpha
▶ ⚔ Iron Sword ATK 12
▶ ⚔ Steel Blade ATK 28
▶ ⚔ Frost Edge ATK 34

State Combinatorics

Selected는 모든 다른 상태(Equipped, On Anvil, Locked, Cleared)와 동시에 표현 가능해야 한다. 배경/보더는 selection 이 잡고, 다른 상태는 좌측 바·우측 뱃지·텍스트 색으로 분리한다.

▶ ⚔ Selected + Equipped ATK 28 E
▶ ⚔ Selected + Dive Ready ATK 34 DIVE
▶ ⚔ Selected + Cleared ATK 41 CLR
▶ ⚔ Selected + Locked ATK 12 🔒

Normal: 어두운 배경, 흐린 텍스트. Selected: 밝은 배경 + 청록 ▶ 커서 + 흰색 아웃라인 + 이름 glow. Equipped: 주황 좌측 바 3px + [E] 뱃지. Equipped+Selected: 양쪽 모두 적용.

  ⚔    Normal — dim text, dark bg ATK 10

구분 포인트

① Normal→Selected: 배경 밝아짐 + ▶ 청록 커서 + 흰 아웃라인 2px + 이름 glow
② Normal→Equipped: 주황 좌측 바 3px + E 뱃지
③ 색상만이 아닌 형태 차이: 커서(▶), 바(좌측 3px), 뱃지(E), 아웃라인 — 색맹 안전

4-Layer 합성 사양

Layer 역할 PixiJS 구현 메모
1. Base ellipse 기존 pulse, 레어리티 컬러, 코어 하이라이트 portalGfx.ellipse 그대로 유지

  • Esc는 패널 닫힘 신호

FeedbackPanel ORGANISM

FeedbackPanel.ts · F 키로 호출, 게임 일시정지 후 표시

익명 사용자가 30초 안에 피드백 1건을 보낼 수 있는 인게임 채널. 본문은 Google Form(저장소), 카운트·카테고리·메타는 GA4(feedback_submitted). 새 탭 안 뜸.

구성 (위 → 아래)

  • 타이틀: "Feedback (F)" · FONT_TITLE · TEXT_PRIMARY · 좌측
  • 자동 컨텍스트 1줄: 예 "itemworld · 12,7 · 0:14:23" · FONT_HINT · TEXT_SECONDARY. level_id/room이 없으면 해당 토큰 생략
  • 텍스트 입력: TextInputField · 8줄 · 최대 500자
  • 카테고리 라디오 (3 가로): Bug 버그 / Idea 제안 / Other 기타 · 기본 미선택 · 1개 필수
  • 하단 힌트: "[Enter] Send [Esc] Close" · FONT_HINT · TEXT_SECONDARY

토큰 / 사이즈

  • 패널 배경: createPanelBg(480, 320) · 중앙 정렬 (centerPos)
  • 오버레이: createOverlay() (배경 dim)
  • 카테고리 선택 색: orange #FF8000 (KeyPrompt와 통일, reference_keyicon_gauge.md)
  • 카테고리 미선택 색: TEXT_SECONDARY

상태 / 가드

  • 활성 씬이 LdtkWorldScene 또는 ItemWorldScene이고, 그 씬의 isPaused가 false일 때만 F가 패널을 연다
  • 패널이 열리면 game.feedbackOpen = true 로 씬 update 차단(hitstopFrames 패턴 미러)
  • 30초 쿨다운 — 쿨다운 중 F 누름 → Toast "Please wait" 0.8s
  • 제출 시 → GA4 feedback_submitted + Google Form POST → Toast "Sent" 1.2s → 패널 닫힘
  • Esc → 즉시 닫힘 (쿨다운 없음)
  • 빈 본문 또는 카테고리 미선택 시 Enter 제출 비활성
  • 네트워크 실패 → "Sent" 동일 표시 (no-cors 응답 못 읽음). localStorage 5건 보존

데이터 흐름

  • 본문 텍스트: Google Form feedback_text 필드. 제한 없음(GA4에는 보내지 않음 — 100자 트렁케이트 + GDPR)
  • 자동 컨텍스트: auto_context_json 필드에 JSON 직렬화. 씬의 getFeedbackContext() 스냅샷 + Analytics getter(run_id, build, utm, last_3_events, playtime) + window/navigator(ua_short, screen_w/h)
  • GA4: feedback_opened(area, playtime_sec) · feedback_submitted(area, playtime_sec, category, text_length)
