PixiJS REFERENCE

ECHORIS 가 활용 가능한 PixiJS v8 기능 + 보조 라이브러리 카탈로그. 각 항목은 적용 후보·임팩트·비용·우선순위로 표기.

분류 의도: 코드 변경 비용 vs 시각/성능 임팩트로 판단. 먼저 검토: 임팩트 HIGH × 비용 LOW. 차후: 임팩트 HIGH × 비용 MID. 실험적: 임팩트 MID/LOW.

Roadmap (권장 순서)

우선순위 P1 → P2 → P3
Pri 항목 Impact Cost 이유
P1 WebGPU renderer HIGH LOW Game.ts 한 줄 변경. 즉각 GPU 가속 (지원 환경에서)
P1 GlowFilter HIGH LOW Trapdoor / Anvil / Boss / Memory Shard 시각 정체성 즉시 강화
P1 Asset Bundle MID LOW 첫 로딩 단축 + stratum 별 lazy. 출시 폴리시 필수

P2 OutlineFilter (resident) MID LOW DEC-038 청록 외곽선을 코드 그래픽 → filter 로 일관
P2 ParticleContainer MID LOW HitSpark 등 같은 texture manager 통합
P2 Tilemap 마이그 HIGH HIGH Ancient 셀 80+ 의 sprite 자체를 1/500 로 감소. cell culling 의 다음 단계
P2 @pixi/particle-emitter MID MID 25+ manager 통합 + JSON config — 디자이너 친화
P3 Custom Shader Filter HIGH HIGH 5색 기질 별 dissolve / 결정화 / 산화 — 아이덴티티 완성도
P3 Spine 2D HIGH HIGH 거대 빌더 / 보스 — 라이선스 + 학습 비용 큼
P3 CRT / OldFilm / Glitch MID LOW 일시 cutscene 효과 — 핵심 아님
