# 특수 아이템 카테고리 및 빌드 정의 시스템 리서치

> ECHORIS 설계를 위한 액션 RPG / 메트로베니아 특수 아이템 시스템 비교 분석

---

## A. 렐릭/능력 아이템 (Relic / Ability Items)

능력 게이트를 통해 탐험 범위를 확장하는 영구 획득 아이템 시스템을 비교합니다.

### 1. Hollow Knight -- Abilities

| 항목 | 내용 |
|------|------|
| 능력 수 | 7개 (+ 업그레이드 2개) |
| 핵심 능력 | Mothwing Cloak(대시), Mantis Claw(벽타기), Crystal Heart(돌진), Monarch Wings(이단점프), Isma's Tear(산성 면역), Dream Nail(꿈 진입), Shade Cloak(그림자 대시) |
| 진행 순서 | Mothwing Cloak -> Mantis Claw -> Crystal Heart -> Monarch Wings -> Shade Cloak (권장) |
| 게이팅 방식 | 순차적 의존: Mothwing Cloak 필요 -> Mantis Claw 접근 가능, Crystal Heart 필요 -> Monarch Wings 접근 가능 |
| 전투 연동 | Shade Cloak으로 적 관통 대시, Dream Nail로 보스 재전투 |
| 탐험 연동 | 각 능력이 새 구역 접근 + Shade Gate 등 전용 장벽 해제 |
| 수집물 | Mask Shard 16개(HP+4), Vessel Fragment 9개(Soul+3) -- 4조각/3조각 단위 합산 |

**ECHORIS 시사점:** 7개라는 적은 수의 능력으로 깊은 게이팅 구조 달성. 능력 간 의존 체인이 핵심. ECHORIS의 6개 렐릭(대시, 벽타기, 이단점프, 안개변신, 수중호흡, 역중력)과 규모가 유사합니다.

### 2. Castlevania: Symphony of the Night -- Relics

| 항목 | 내용 |
|------|------|
| 렐릭 수 | 약 30개 |
| 카테고리 | 변신(Bat/Wolf/Mist), 이동(Leap Stone/Gravity Boots), 패밀리어(5종), 유틸리티(Spirit Orb/Cube of Zoe), 스토리(Vlad 보물 5개) |
| 핵심 이동 렐릭 | Soul of Bat(비행), Soul of Wolf(고속이동), Form of Mist(벽 통과), Leap Stone(이단점프), Gravity Boots(하이점프) |
| 게이팅 방식 | 변신 능력으로 맵 진행 장벽 해제, 일부는 보스 처치 후 획득 |
| 특이점 | ON/OFF 토글 방식, 자원 소모 없음. 변신은 완전한 조작 체계 변경 |

**ECHORIS 시사점:** 변신 렐릭이 단순 게이트를 넘어 플레이 방식 자체를 변경합니다. ECHORIS의 "안개 변신"이 이와 유사한 역할을 합니다. 30개 중 실제 게이팅용은 5-6개이며, 나머지는 유틸리티/수집 성격입니다.

### 3. Metroid -- Suit Upgrades

| 항목 | 내용 |
|------|------|
| 업그레이드 카테고리 | Suit(방어), Beam(공격), Movement(이동), Morph Ball(변형) |
| 핵심 이동 능력 | Morph Ball, Bomb, Space Jump, Screw Attack, Speed Booster, Grapple Beam |
| 설계 원칙 | 모든 업그레이드가 Power Suit 위에 누적 장착. Sequence Breaking 허용 |
| 게이팅 방식 | 특정 능력 없이는 물리적으로 진행 불가 (문/장벽 타입 매칭) |
| 특이점 | 능력이 탐험/전투 양쪽에 동시 활용 (Screw Attack = 점프 + 공격) |

**ECHORIS 시사점:** 능력의 이중 용도(탐험+전투) 설계가 핵심. ECHORIS에서도 벽타기가 전투 회피에, 이단점프가 공중 콤보에 연동되는 설계가 필요합니다.

### 4. Dead Cells -- Rune System

| 항목 | 내용 |
|------|------|
| 룬 수 | 5개 |
| 룬 목록 | Vine Rune(덩굴 생성), Spider Rune(벽타기), Teleportation Rune(벽 관통), Ram Rune(바닥 파괴), Homunculus Rune(분리체) |
| 획득 방식 | 특정 구역의 Elite/보스 처치 시 즉시 영구 해금 |
| 영구성 | 사망해도 유지. Roguelite 구조에서 영구 진행 역할 |
| 게이팅 방식 | 각 룬이 새 Biome 경로를 개방 |

**ECHORIS 시사점:** Roguelite에서 영구 능력을 분리한 좋은 사례. ECHORIS 아이템계에서 사망해도 월드 렐릭은 유지되는 구조와 동일한 철학입니다. 5개라는 매우 적은 수로 충분한 분기를 만들어냅니다.

### 5. Blasphemous -- Rosary Beads

| 항목 | 내용 |
|------|------|
| 비즈 수 | 다수 (장착 슬롯 초기 2개 -> 최대 8개 확장) |

**ECHORIS 시사점:** 장착 슬롯 점진적 확장 메커닉이 흥미롭습니다. 기억 단편 슬롯이 레어리티에 따라 2-8개인 ECHORIS 구조와 개념적으로 유사합니다.

### 6. Ori -- Ability/Shard System

| 항목 | 내용 |
|------|------|
| 시스템 (Will of the Wisps) | Ancestral Tree(능력 해금) + Spirit Shards(패시브 장착) |
| Active 능력 | 3개 동시 장착, 실시간 교체 가능 |
| Spirit Shards | 기존 스킬트리를 대체하는 장착식 패시브 (마테리아 영감) |

**ECHORIS 시사점:** Active 능력 선택 + Passive Shard 조합의 이중 구조가 빌드 다양성을 만듭니다. ECHORIS의 렐릭(영구) + 기억 단편(장착 교체)와 구조적으로 닮아 있습니다.

## B. 세트 아이템 / 빌드 정의 아이템 (Set Items / Build-Defining Items)

### 1. Diablo 시리즈 -- Set Items

#### Diablo 2

| 항목 | 내용 |
|------|------|
| 세트 수 | 16개 (오리지널) + 16개 (LoD 확장) = 약 32개 |
| 세트 구성 | 2-6 피스 세트 |
| 보너스 구조 | 부분 보너스(2피스부터) + 완전 세트 보너스 |
| 설계 철학 | 수집가를 위한 보상. 특정 빌드 강제보다는 점진적 보상 |

#### Diablo 3

| 항목 | 내용 |
|------|------|
| 세트 구성 | 클래스별 4-5개 세트 |
| 보너스 구조 | 2/4/6 피스 3단계 보너스 |
| 배율 | 6피스 보너스가 스킬 데미지 1000-13000% 증가 -- 빌드를 완전히 정의 |
| 문제점 | 세트 의존도가 너무 높아 세트 없이는 엔드게임 불가 |

#### Diablo 4

| 항목 | 내용 |
|------|------|
| Unique 등급 | 일반 Unique + Mythic Unique (Uber Unique) |
| Mythic Unique | 12개, 드랍률 ~1/2000, 항상 Greater Affix 4개 보유 |
| 설계 방향 | 세트 의존 탈피 -> 개별 Unique가 빌드 정의 |
| 획득 | 극히 희귀한 드랍 + Resplendent Spark 2개로 크래프팅 |

**ECHORIS 시사점:** D3의 세트 배율 문제를 피하려면, 세트 보너스는 "스킬 변형"(질적 변화)이지 "수치 배율"(양적 폭증)이어서는 안 됩니다. D4 방식의 개별 유니크 빌드 정의가 아이템계 스파이크와 더 잘 어울립니다.

---

### 2. Path of Exile -- Build-Enabling Uniques

| 항목 | 내용 |
|------|------|
| 유니크 수 | 1000개 이상 |
| 설계 원칙 | "Rare보다 약하지만, 불가능한 빌드를 가능하게 함" |
| 대표 예시 | Facebreaker(무기 없이 물리), Whispering Ice(INT 스택), Hollow Palm(소켓 없이 맨손) |
| 드랍 시스템 | 빌드 활성화 유니크는 비교적 흔하게, 강력한 유니크는 극히 희귀하게 |
| 빌드 정의 방식 | 메커닉 규칙 자체를 변경 (예: "주문이 공격이 된다") |

**ECHORIS 시사점:** "규칙 변경" 유니크가 야리코미 동기를 극대화합니다. 아이템계에서 드랍하는 Legendary/Ancient 장비에 "기억 단편 조합 규칙을 변경하는" 특수 속성을 부여하는 방향이 적합합니다.

---

### 3. Monster Hunter -- Armor Set Skills

| 항목 | 내용 |
|------|------|
| 스킬 시스템 | 각 방어구 파츠에 스킬 포인트 부여, 합산으로 효과 발동 |
| MH:World 이후 | 1포인트부터 효과 발동 (이전작: 10포인트 임계값) |
| Mixed Set | 다른 몬스터 방어구를 조합하여 원하는 스킬 조합 달성 |
| 세트 보너스 | MH:World에서 동일 세트 착용 시 추가 세트 스킬 (2/3/4피스) |
| 설계 핵심 | Full Set vs Mixed Set의 트레이드오프 |

**ECHORIS 시사점:** "포인트 합산" 방식은 파츠 단위 자유도와 세트 보너스의 균형을 잡습니다. ECHORIS 기억 단편 시스템에 "같은 종류 기억 단편 N마리 이상 보유 시 세트 보너스" 구조를 적용할 수 있습니다.

---

### 4. Nioh 2 -- Grace Sets

| 항목 | 내용 |
|------|------|
| Grace 수 | 10개 (Divine) + DLC 추가 Ethereal Grace |
| 시스템 | Non-Set 아이템에 랜덤 Grace가 부여되어 사실상 세트 형성 |
| 보너스 | 피스 수에 따라 단계별 세트 보너스 |
| 특이점 | 명시적 세트 없이도 Grace 매칭으로 "즉석 세트" 형성 |
| 엔드게임 | Grace + 기존 Set 조합으로 하이브리드 빌드 |

**ECHORIS 시사점:** "랜덤 부여 Grace로 세트를 형성하는" 방식은 아이템계에서의 파밍 동기와 매우 잘 맞습니다. 아이템계 클리어 시 장비에 "결(Grain)" 같은 속성이 랜덤 부여되어 같은 결을 모으면 세트 효과가 발동하는 구조를 고려할 수 있습니다.

### 5. Grim Dawn -- Set Items + Monster Infrequent (MI)

| 항목 | 내용 |
|------|------|
| Set Items | 전통적 명시 세트, 다수 존재 |
| Monster Infrequent | 특정 몬스터만 드랍하는 고유 명칭/고유 고정 스탯 + 랜덤 접사 |
| MI 색상 | 녹색 (Green) |
| MI 설계 | 고정 속성이 빌드 방향 제시 + 랜덤 접사가 God Roll 파밍 동기 |
| MI 활용 | 크래프팅 재료로도 사용, "잘 구른 MI"가 세트보다 강할 수 있음 |

**ECHORIS 시사점:** MI의 "고정 + 랜덤" 구조가 아이템계 보스 드랍 설계에 직접 적용 가능합니다. 아이템 장군/왕/신 처치 시 고유 고정 속성 + 랜덤 기억 단편 구성의 특수 장비를 드랍시키는 방식입니다.

| Cluster Jewel | 패시브 트리 외곽 전용 | 패시브 트리 자체를 확장, 내부에 추가 소켓 가능 |
| Unique Jewel | 패시브 트리 소켓 | 주변 패시브를 변환하는 등 규칙 변경 효과 |

**ECHORIS 시사점:** "아이템 내부에 소켓 -> 소켓에 보너스 존재를 장착"하는 구조가 ECHORIS 기억 단편 슬롯과 직접 대응합니다. Cluster Jewel의 "시스템 자체를 확장하는" 개념이 Ancient 등급 아이템의 심연 지층과 연결될 수 있습니다.

### 4. Hades -- Mirror of Night

| 항목 | 내용 |

| 업그레이드 수 | 12개 카테고리 (각 2가지 선택지) |
| 통화 | Darkness |

**ECHORIS 시사점:** Mirror of Night의 "A/B 택1" 구조가 빌드 정체성을 만듭니다. 월드 세이브 포인트에서의 영구 강화 시스템에 이 구조를 차용할 수 있습니다.

### 5. Dead Cells -- Blueprints

| 항목 | 내용 |

**ECHORIS 시사점:** "획득 -> 안전지대 전달 -> 비용 지불 -> 영구 해금" 루프가 아이템계 귀환 동기를 강화합니다. 아이템계에서 발견한 설계도를 월드 세이브 포인트(대장간)에서 해금하는 구조가 가능합니다.

## D. 인벤토리/보관 시스템 (Inventory / Storage Systems)

### 비교표

| 게임 | 인벤토리 방식 | 크기 | 보관소 | 특이점 |
|------|--------------|------|--------|--------|
| Diablo 2/3 | 그리드 (아이템 크기 가변) | 10x4 (D2), 60칸 (D3) | 공유 보관함 | 아이템 크기가 공간 관리 퍼즐 |
| Path of Exile | 12x12 그리드 탭 | 탭당 144칸, Quad=576 | Stash Tab (무제한 구매) | 특수 탭(Currency/Map/Unique), Affinity 자동 분류 |
| Elden Ring | 무한 인벤토리 | 제한 없음 | 보관함 (Sites of Grace) | 장착만 제한, 소지는 무한 |
| Monster Hunter | 리스트 + 장비 Box | 장비 Box (수천 칸) | 하나의 통합 Box | 소재와 장비 분리 관리 |
| Disgaea | 아이템 Bag + Warehouse | Bag 16개(D1) -> 2000개(D6) | Warehouse 256개(D1) | 시리즈 진행에 따라 대폭 확장 |

### 설계 관점 분석

#### 그리드 vs 리스트 vs 무한

| 방식 | 장점 | 단점 | 적합한 게임 |
|------|------|------|------------|
| 그리드 (Diablo) | 시각적 직관, 공간 관리 자체가 게임플레이 | 정리 피로, 대량 아이템 관리 곤란 | Loot 빈도 낮고 개별 가치 높은 게임 |
| 리스트/탭 (PoE) | 대량 아이템 관리 용이, 분류 체계화 | 시각적 재미 부족 | Loot 빈도 높고 종류 다양한 게임 |
| 무한 (Elden Ring) | 관리 스트레스 제로 | 아이템 가치 희석, 정리 동기 부족 | 아이템이 핵심이 아닌 게임 |

#### ECHORIS 인벤토리 설계 방향

ECHORIS의 핵심 루프(아이템 획득 -> 아이템계 진입 -> 강화)를 고려하면:

1. **장비 인벤토리**: 제한적 슬롯 (장비의 개별 가치가 높으므로). 레어리티별 시각 구분 필수
2. **기억 단편 보관**: 별도 탭/구역. 종류별 자동 분류 (PoE Affinity 참고)
3. **소재/설계도**: 무한 또는 매우 넓은 공간 (수집 스트레스 제거)
4. **세이브 포인트 접근**: Disgaea Warehouse처럼 안전 지대에서만 정리 가능

## 종합 시사점 및 ECHORIS 적용 방향

### 렐릭 (능력 게이트)
- **현재 6개 렐릭은 적정 규모** (Hollow Knight 7개, Dead Cells 5개와 유사)
- 각 렐릭의 **전투 연동** 설계 필요 (Metroid의 이중 용도 원칙)
- 렐릭 간 **의존 체인** 명확화 (Crystal Heart -> Monarch Wings 같은 구조)

### 기억 단편 (아이템 내부 존재)
- Disgaea 3계층 분류 채택: **스탯 직접(기본) + 성장 가속(유틸리티) + 메커닉 변경(유니크)**
- PoE Jewel의 "아이템 소켓에 장착" 구조와 완전 일치 -- 검증된 설계
- MH 스킬 포인트처럼 "같은 종류 기억 단편 N마리 = 세트 보너스" 가능성

### 빌드 정의
- D3 배율 폭증 방식 **회피** -> D4/PoE의 "질적 변화" 방식 채택
- Nioh 2 Grace처럼 **아이템계 클리어 시 랜덤 속성(결/Grain) 부여 -> 같은 결 수집 = 세트 효과** 고려
- Grim Dawn MI처럼 **아이템계 보스 전용 드랍 = 고정 속성 + 랜덤 기억 단편**

### 메타 진행 (영구 수집물)
- Hades Mirror 방식의 **영구 강화 + A/B 택1** 구조 (세이브 포인트 업그레이드)
- Dead Cells Blueprint 방식의 **"발견 -> 귀환 -> 해금" 루프**로 아이템계 귀환 동기 강화
- Hollow Knight 조각 수집 방식의 **HP/자원 영구 확장** (월드 탐험 보상)

### 인벤토리
- **장비: 제한적** (가치 보존) / **기억 단편: 분류 탭** (PoE 참고) / **소재: 넉넉하게**
- 세이브 포인트에서만 정리/정비 (Disgaea Warehouse 원칙)
