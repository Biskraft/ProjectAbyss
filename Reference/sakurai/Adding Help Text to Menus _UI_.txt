[English Translation]
[Music]
Menu items are, of course, expressed in words—most often a single word. But there are times when you have no idea what a function does just by looking at that one word.
In those cases, what's useful is a functional description, which we call "help text." Sometimes we place descriptive text at the bottom of the screen. This is something that should be proactively included. Except for titles from a time when it wasn't possible, almost all of my works have this feature. It's a function I consider quite important.
While I'm not entirely sure how the translated versions abroad turned out, there is one thing I particularly care about when setting the help text:
Never explain it using the same words as the menu item itself. This is important, so I'll repeat it: don't explain using the same words.
For example, it might sound like a joke, but it actually happens quite often: a description for "Options" that simply says "Go to options."
Stop doing that. It's meaningless help text for someone who doesn't understand what "Options" means. Try to explain the help text using different words as much as possible. In this case, for Options, you could say "Adjust various in-game settings."
While the ideal is for the meaning to enter the mind smoothly, sometimes that's difficult with help text alone. So, once the creator has thought of the help text, it's important to break it down even further. I hope you continue to refine it so you can view the meaning objectively, strip away explanatory filler, and convey it more simply.
I think the point is to make it feel natural rather than formal. For the mode names themselves, for example, "Play Alone" might be easier to understand than "Single Mode." And, of course, avoid using technical jargon. Terms like "Config" or "Key Repeat" won't be understood by those who aren't familiar with them.
