# 프로젝트 기획 개요: 메트로베니아 x 아이템계 온라인 액션 RPG

> **프로젝트명:** (가칭) ECHORIS
> **장르:** 웹 기반 횡스크롤 온라인 액션 RPG (메트로베니아 + 야리코미)
> **플랫폼:** 웹 브라우저 (PC 우선)
> **타겟:** 탐험과 파밍을 즐기는 코어~미드코어 유저
> **레퍼런스:** BLAME!/메이드 인 어비스(세계관/스케일) + 디스가이아(아이템계/야리코미) + 월하의 야상곡(탐험/전투) + 스펠렁키(절차적 생성)
> **작성일:** 2026-03-23

---

## 1. 핵심 컨셉

### 1.1 한 줄 요약

> **"거대 빌더가 누비는 수직 구조물을 탐험하고, 말을 거는 무기 속으로 들어가 끝없이 강화하며, 다른 플레이어와 함께 싸우는 횡스크롤 온라인 액션 RPG"**

### 1.2 핵심 판타지 (Core Fantasy)

| 판타지 | 설명 |
|:---|:---|
| **탐험가** | 능력을 하나씩 얻으며 갈 수 없던 곳을 뚫어 세계의 비밀을 밝혀낸다 |
| **장인** | 아이템 속에 들어가 잊혀진 기억 단편을 회상시키고, 세상에 하나뿐인 최강 장비를 만든다 |
| **모험가** | 친구와 함께 끝없는 심연의 던전에 도전하고, 위기를 함께 극복한다 |

### 1.3 장르 결합의 핵심 질문과 해답

| 질문 | 해답 |
|:---|:---|
| 메트로베니아의 탐험감과 온라인 멀티플레이가 양립 가능한가? | 2-Space 분리 모델로 해결. 월드는 개인 탐험 (대장간/상점은 세이브 포인트), 아이템계는 협동 (URL 링크 파티 합류) |
| 야리코미(무한 파밍)가 탐험의 가치를 훼손하지 않는가? | 순환 구조로 해결. 더 깊은 아이템계를 위해 더 넓은 월드를 탐험해야 함 |

## 2. 2-Space 분리 모델

게임 세계를 **월드(World)**와 **아이템계(Item World)** 두 공간으로 분리하여, 각 공간이 고유한 규칙과 목적을 갖도록 설계한다.

> **변경 이력 (2026-04-04):** 허브(Hub)는 독립 공간에서 폐기. 대장간/상점은 월드 세이브 포인트에 통합. 파티 합류는 URL 링크 공유로 아이템계 직접 진입.

### 2.1 공간 비교표

| 항목 | 월드 (World) | 아이템계 (Item World) |
|:---|:---|:---|
| **핵심 목적** | 탐험, 능력 획득, 스토리, 대장간/상점 (세이브 포인트) | 아이템 강화, 야리코미 |
| **인원** | 솔로(1인) | 1 ~ 4인 |
| **맵 유형** | 핸드크래프트 + 절차적 혼합 | 완전 절차적 생성 |
| **진행 방식** | 능력 게이트 (메트로베니아) | 지층 클리어 (레어리티별 2~4 지층) |
| **사망 페널티** | 세이브 포인트 복귀 | 진행 지층 손실 (탈출 아이템 미사용 시) |
| **고유 자원** | 능력 렐릭, 맵 데이터, 월드 소재 | 기억 단편, 핵심 기억, 아이템 경험치, 레벨 구슬 |

### 2.2 공간 구조도

```
┌─────────────────────┐     ┌─────────────────────────┐
│    월드 (World)       │     │   아이템계 (Item World)   │
│  - 메트로베니아 탐험   │────▶│  - 기억의 지층 던전       │
│  - 능력 게이트        │     │  - 기억 단편 회상         │
│  - 보스/미니보스      │     │  - 지층별 보스            │
│  - 아이템 획득        │     │  - 순환 진입 (월드 경유)    │
│  - 스탯 게이트        │     │  - 1~4인 협동             │
│  - 세이브 포인트      │     │  (URL 링크로 파티 합류)    │
│    (대장간/상점)      │◀────│                           │
└─────────────────────┘     └─────────────────────────┘
         │                             │
         └──────────┐  ┌───────────────┘
                    ▼  ▼
              ┌──────────────┐
              │  플레이어 성장  │
              │  장비 + 능력   │
              └──────────────┘
```

## 3. 월드 (World) 설계

### 3.1 월드 구조 원칙

월하의 야상곡의 성(城) 구조를 레퍼런스로, **허브 & 스포크 + 원형 순환** 구조를 채택한다.

```
                        ┌──────────┐
                 ┌──────┤ 천공의 탑  ├──────┐
                 │      └──────────┘      │
          ┌──────┴──────┐          ┌──────┴──────┐
          │  마법 연구소  │          │  빙결 동굴   │
          └──────┬──────┘          └──────┬──────┘
                 │      ┌──────────┐      │
                 ├──────┤ 중앙 성채  ├──────┤
                 │      │ (세이브PT) │      │
                 │      └──────────┘      │
          ┌──────┴──────┐          ┌──────┴──────┐
          │  지하 수로   │          │  묘지/카타콤  │
          └──────┬──────┘          └──────┬──────┘
                 │      ┌──────────┐      │
                 └──────┤ 심연의 구  ├──────┘
                        └──────────┘
```

### 3.2 스탯 게이트 시스템

월하의 야상곡의 렐릭 시스템 + 디스가이아의 스탯 시스템을 결합한 **스탯 게이트**를 도입한다. 장비의 스탯이 월드 진행의 열쇠가 된다.

| 스탯 | 게이트 유형 | 예시 |
|:---|:---|:---|
| **ATK** | 물리 장벽 | 금이 간 벽 파괴, 바리케이드 돌파, 중문/쇠사슬/바위 |
| **INT** | 마법 봉인 | 룬 문양 장벽 해제, 마법 봉인 문, 아케인 잠금장치 |

> **설계 변경:** 기존 6대 스탯 게이트(STR/INT/DEX/VIT/SPD/LCK)는 ATK/INT 이중 게이트로 재설계. ATK는 물리 장벽(파괴), INT는 마법 봉인(해제). 기동성/환경 저항/시간 제한 등은 능력 게이트나 환경 퍼즐로 재설계 예정.

**핵심 설계 의도:** 아이템계에서 장비를 강화하면 월드의 새로운 구역이 열린다. 이것이 월드 ↔ 아이템계 순환의 핵심 동력이다.

### 3.3 능력 게이트 시스템

전통적인 메트로베니아 능력 게이트도 병행한다. 보스를 처치하거나 숨겨진 렐릭을 발견하여 획득한다.

| 능력 | 해금 조건 | 열리는 구역 |
|:---|:---|:---|
| 이단 점프 | 묘지 보스 처치 | 높은 절벽 위 구역 |
| 벽 타기/벽 점프 | 지하 수로 렐릭 | 수직 통로, 좁은 갱도 |
| 안개 변신 | 마법 연구소 보스 | 격자/좁은 틈새 |
| 수중 호흡 | 빙결 동굴 렐릭 | 수중 구역 |
| 역중력 | 천공의 탑 보스 | 반전 구역 (역전 성) |

### 3.4 맵 생성 방식: 스펠렁키 하이브리드

- **매크로 구조:** 핸드크래프트. 구역 간 연결, 보스 방, 세이브/워프 룸은 고정 배치
- **마이크로 구조:** 절차적 생성. 각 구역 내부의 방(Room)은 스펠렁키 방식의 템플릿 조립
- **시드:** 서버 고정 시드
- **Always Winnable:** 모든 시드에서 클리어 가능한 Critical Path 보장

```
[매크로 - 고정]                    [마이크로 - 절차적]
구역 A ──── 구역 B                 ┌───┬───┬───┬───┐
  │            │                   │ R1│ R2│ R3│ R4│ ← 4x4 Room Grid
구역 C ──── 구역 D                 ├───┼───┼───┼───┤
  │            │                   │ R5│ R6│ R7│ R8│
구역 E ──── (보스)                 ├───┼───┼───┼───┤
                                   │R9 │R10│R11│R12│
각 구역은 여러 Room Grid로 구성     ├───┼───┼───┼───┤
Room Grid 내부는 템플릿 조립       │R13│R14│R15│R16│
                                   └───┴───┴───┴───┘
                                   각 Room은 핸드크래프트 Chunk 조합
```

## 4. 아이템계 (Item World) 설계

### 4.1 핵심 개념

> 모든 장비 아이템은 내부에 **던전**을 가지고 있다. 플레이어는 아이템 속에 들어가 층을 클리어하며, 아이템을 직접 강화한다.

### 4.2 기본 구조

| 항목 | 사양 |
|:---|:---|
| 총 층수 | 100층 (레어리티에 따라 진입 가능 최대 층 상이) |
| 보스 층 | 10, 20, 30, ... 100층 (10층마다) |
| 세이브 포인트 | 보스 처치 후 (10층 단위) |
| 탈출 | Mr. Gency's Exit (소모품) 또는 보스 층 클리어 후 |
| 시드 | `hash(itemID + level + floor)` → 동일 아이템의 동일 층은 항상 같은 맵 |
| 인원 | 1~4인 (파티 리더의 아이템에 진입) |
| 순환 진입 | 아이템계에서 획득한 아이템은 월드 귀환 후 해당 아이템의 아이템계에 진입 가능 (중첩 진입 없음) |

### 4.3 층 생성 규칙

스펠렁키의 절차적 생성을 횡스크롤 액션에 맞게 변형한다.

```
[층 생성 파이프라인]

1. 시드 결정
   seed = hash(itemID + itemLevel + floorNumber)

2. 레이아웃 생성 (Phase 1)
   - Room Grid 크기 결정 (난이도에 따라 3x3 ~ 5x5)
   - Critical Path 생성 (입구 → 출구 보장)
   - Room Type 배정 (출입구 방향)

3. 지형 조립 (Phase 2)
   - 각 Room에 Chunk 템플릿 삽입
   - 아이템 레어리티에 따라 Chunk 풀 변경 (고급 = 복잡한 지형)

4. 오브젝트 배치 (Phase 3)
   - 적 배치 (층수 기반 난이도 스케일링)
   - 기억 단편 배치 (잊혀진 단편)
   - 보상 오브젝트 배치 (레벨 구슬, 보물상자)
   - 지오 이펙트 패널 배치 (선택적)

5. 특수 이벤트 (Phase 4)
   - 미스터리 룸 생성 확률 체크
   - 기억 단편 군집 출현 체크 (특정 지층)
```

### 4.4 보스 시스템

| 보스 등급 | 출현 층 | 처치 보상 |
|:---|:---|:---|
| 아이템 장군 | 10, 20, 30층 | 아이템 스탯 +5% |
| 아이템 왕 | 40, 50, 60층 | 아이템 스탯 +10%, 기억 슬롯 +1 |
| 아이템 신 | 70, 80, 90층 | 아이템 스탯 +15%, 특수 핵심 기억 드랍 |
| 아이템 대신 | 100층 | 아이템 레어리티 승급 기회, 대량 보상 |

### 4.5 순환 진입 (Sequential Entry Loop)

> **변경 (2026-04-05):** 재귀적(중첩) 진입은 삭제되었습니다. 아이템계 진입은 항상 월드에서만 가능합니다.

```
[순환 진입 루프]
  월드 → 아이템 A의 아이템계 진입 → 아이템 B 획득 → 월드 귀환
  월드 → 아이템 B의 아이템계 진입 → 아이템 C 획득 → 월드 귀환
  월드 → 아이템 C의 아이템계 진입 → ...
  (무한 순환)
```

**핵심 원칙:** 아이템계 내부에서 다른 아이템의 아이템계에 중첩 진입할 수 없다. 아이템계에서 획득한 아이템은 인벤토리에 저장되며, 월드로 귀환한 뒤 세이브 포인트/대장간에서 해당 아이템의 아이템계에 새로 진입한다. 스파이크("아이템 속으로 들어간다")는 유지되며, 순환 구조가 끝없는 파밍 루프를 제공한다.

## 5. 기억 단편 시스템 (Memory Shard System)

> **DEC-036 (2026-04-28):** 기억 단편 시스템은 검 Ego(DEC-033)와 통합되어 기억 단편 시스템으로 흡수됨. 인사이드 아웃의 기억 모델을 척추로 한다. SSoT: `Documents/System/System_Memory_Core.md`.

### 5.1 개요

기억 단편(Memory Shard)은 무기 Ego의 잊혀진 기억 조각. 무기는 자신의 기억 일부를 잃은 채로 발견되며, 플레이어는 아이템계에서 그 기억을 회상시켜 무기를 본래의 모습으로 복원한다.

### 5.2 상태

| 상태 | 한글명 | 효과 | 설명 |
|:---|:---|:---|:---|
| Forgotten | 잊혀진 | 50% 효과 | 지층 내 적 NPC로 출현 |
| Recalled | 회상된 | 100% 효과 | Ego가 받아들임. 다른 무기로 전이 가능 |

격파 = **회상(Recall)**. 잊혀진 기억을 Ego가 되찾는 행위.

### 5.3 위계 (2-Tier)

| 위계 | 획득 채널 | 슬롯 | 전이 |
|:---|:---|:---|:---|
| 일반 단편 (Memory Shard) | 지층 내 몹/체스트/시크릿 | 기억 슬롯 (Active/Passive) | 자유 (변이) |
| 핵심 기억 (Core Memory) | 지층 보스 100% 드롭 | 정체성 슬롯 전용 | 가능 (정체성 결 붕괴) |

### 5.4 5색 기질 (Temperament)

| 색 | 한글명 | 기질 | 대표 단편 계열 |
|:---|:---|:---|:---|
| Forge | 단조 | 분노·열정·공격성 (주황) | Gladiator/Berserker/Burner |
| Iron | 강철 | 결연함·냉정 (청록) | Ironclad/Freezer |
| Rust | 부식 | 비통·체념·세월 (회색) | Vampire/Leech/Dietician |
| Spark | 섬광 | 호기심·경이 (흰빛) | Tutor/Sprinter/Shocker |
| Shadow | 그림자 | 의심·교활·은밀 (자주) | Ghost |

각 무기 Ego는 *주색 1 + 부색 1*. 핸드크래프트 무기(Rustborn 등)는 고정 색 조합. 절차 드롭 무기는 시드 결정.

### 5.5 단편 카탈로그 (5색 분포)

| 기질 | 단편 | 효과 | 비고 |
|:---|:---|:---|:---|
| Forge | Gladiator (글래디에이터) | ATK +N (1/레벨) | 물리 빌드, ATK 게이트 해금 |
| Forge | Berserker (광전사) | 공격 속도 +0.33%/레벨, 소프트 캡 Lv60 → +20% | 빠른 콤보, DPS |
| Forge | Burner (버너) | 화 인챈트 데미지 +20%, 화상 지속 +30% | 아이템계 드랍 |
| Iron  | Ironclad (철벽) | 넉백 저항 +0.8%/레벨, 소프트 캡 Lv60 → ~48% | 보스전 안정 |
| Iron  | Freezer (프리저) | 빙 인챈트 데미지 +20%, 빙결 지속 +30% | 아이템계 드랍 |
| Rust  | Dietician (영양사) | MaxHP +N (1/레벨) | 생존력 |

### 5.6 슬롯 모델

총 슬롯 수는 기존 레어리티별 카운트를 보존하되, 내부를 *정체성 슬롯* + *기억 슬롯*으로 분할한다.

| 레어리티 | 정체성 슬롯 (Core 전용) | 기억 슬롯 (자유) | 합계 |
|:---|:---:|:---:|:---:|
| Normal | 2 | 0 | 2 |
| Magic | 3 | 0 | 3 |
| Rare | 3 | 1 | 4 |
| Legendary | 4 | 2 | 6 |
| Ancient | 5 (4 + 심연) | 3 | 8 |

- 정체성 슬롯 = 무기의 지층 수와 일치. 보스 처치 핵심 기억과 1대1 매칭.
- 기억 슬롯은 *Active*(전투 중 발현) 또는 *Passive*(상시) 두 역할 중 하나로 끼움.
- 같은 단편이 어느 역할에 들어가느냐에 따라 다른 효과 발현.

### 5.7 전이 시 효과 변이

전이는 모루(Anvil)에서 수행. 원본 단편 소모. 수신 Ego의 *기질색*에 따라 단편이 3축에서 변형:

1. **수치 변동** — 효과 강도 ±
2. **이름 변형** — 단편 이름 재명명 (예: "분노의 단편" → 차분한 Iron Ego 손에서 "삭여낸 분노")
3. **효과 종류 변이** — 효과 카테고리 자체 변환 (불 → 연기, 빛 → 그림자 등)

핵심 기억은 전이 시 정체성 결이 붕괴. 무거운 결정.

### 5.8 폐기 (재도입 금지)

- ~~기억 단편~~ → 기억 단편으로 통합
- 단편 직접 발화 (화자는 Ego 단 1명)
- 단편 거래 시장
- 무기 외 사용처
- 단편 분신·복제
- 자동 일괄 흡수

## 7. 순환 구조 (Core Loop)

### 7.1 메인 순환

```
        ┌─────────────────────────────────────────┐
        │                                         │
        ▼                                         │
  [월드 탐험] ──아이템 획득──▶ [아이템계 진입]        │
        ▲                         │               │
        │                    아이템 강화            │
        │                    기억 단편 수집         │
        │                         │               │
        │                         ▼               │
        │                   [장비 스탯 상승] ───────┘
        │                         │
        │                         ▼
        │                   [스탯 게이트 해금]
        │                         │
        └─────── 새 구역 탐험 ◀────┘
```

### 7.2 순환 동력 분석

| 순환 요소 | 동력 | 효과 |
|:---|:---|:---|
| 월드 → 아이템계 | 새 장비 획득, 더 강한 장비 필요 | 아이템계 진입 동기 |
| 아이템계 → 장비 강화 | 층 클리어, 기억 단편 수집 | 파밍/야리코미 만족 |
| 장비 강화 → 월드 | 스탯 게이트 해금 | 새 구역 탐험 동기 |
| 월드 탐험 → 능력 획득 | 보스 처치, 렐릭 발견 | 이동 범위 확대 |
| 능력 → 월드 접근성 | 이단점프, 변신 등 | 미탐험 구역 접근 |

### 7.3 자원 순환표

| 자원 | 획득처 | 소비처 | 역할 |
|:---|:---|:---|:---|
| 장비 아이템 | 월드 드랍, 아이템계 보상 | 아이템계 진입 소재 | 순환의 핵심 매개 |
| 기억 단편 | 아이템계 내 전투 (회상) | 장비에 장착, 합성(Bond) | 야리코미 깊이 |
| 핵심 기억 | 지층 보스 처치 | 정체성 슬롯 장착 | 무기 정체성 결 가동 |
| 골드 | 적 처치, 아이템 판매 | 상점, 합성, 단편 팜 | 기본 경제 |
| 월드 소재 | 월드 채집, 보스 드랍 | 장비 제작, 세이브 포인트 대장간 | 월드 탐험 동기 |

## 8. 멀티플레이어 설계

### 8.1 공간별 인원 구조

### 8.2 멀티플레이 설계 원칙

1. **월드는 개인의 영역:** 탐험의 발견감을 보존하기 위해 솔로 전용
2. **아이템계는 협동의 영역:** 높은 난이도의 깊은 지층은 파티 플레이로 설계. 역할 분담 (탱커, 딜러, 서포터). URL 링크 공유로 친구가 아이템계에 직접 합류
3. **비동기 멀티:** 다른 플레이어의 유령(고스트)이 월드에 나타나 힌트를 남기는 시스템 (Dark Souls 메시지)

### 8.3 파티 역할과 시너지

| 역할 | 무기 예시 | 아이템계 기여 |

| 서포터 | 지팡이, 마법서 | 회복, 버프, 기억 단편 탐지 |

## 11. RPG 성장 시스템

### 11.1 캐릭터 스탯

| 스탯 | 약어 | 역할 |
|:---|:---|:---|
| 공격력 | ATK | 물리 데미지 산출. 장비 ATK + 캐릭터 기본 ATK. ATK 게이트(물리 장벽) 해금 |
| 지력 | INT | 원소/인챈트 데미지 산출. 장비 INT + 캐릭터 기본 INT. INT 게이트(마법 봉인) 해금. 상태이상 지속시간 스케일 |
| 체력 | HP | 생존. MaxHP는 레벨 + 기억 단편(Dietician, Rust 기질)으로 성장 |
| 방어력 | DEF | 물리 피해 감소. 장비에서 직접 제공 |
| 마법저항 | RES | 원소 피해 감소. 장비에서 직접 제공 |

> **설계 변경:** 기존 9개 스탯(HP/MP/STR/INT/DEF/RES/DEX/SPD/LCK) → ATK/INT/HP 3스탯 + DEF/RES(장비 제공)로 단순화. MP 삭제 (스킬은 쿨다운). 크리티컬/드랍률/이동속도는 기억 단편으로 보정. 원소 적중률 100% (상태이상 항상 발동, 지속시간은 INT 스케일).

### 11.2 성장 공식

```
Final_ATK = Base_ATK + Equipment_ATK + Shard_ATK_Bonus
Final_INT = Base_INT + Equipment_INT + Shard_INT_Bonus
Final_DEF = Equipment_DEF + Buff_DEF
Final_RES = Equipment_RES + Buff_RES
Final_MaxHP = Base_HP + Level_Bonus_HP + Shard_HP_Bonus

- Base_ATK: 캐릭터 기본 공격력, Base_INT: 캐릭터 기본 지력
- Equipment_ATK/INT: 장비 ATK/INT × 레어리티 배율
- Shard_ATK_Bonus: Gladiator(Forge) 합산, Shard_INT_Bonus: Tutor(Spark) 합산
```

### 11.3 데미지 공식

```
물리 데미지 = (ATK × 스킬 배율 - DEF × 0.5) × 크리티컬 배율 × 랜덤(0.9~1.1)
원소 데미지 = (INT × 원소 배율 - RES × 0.5) × 크리티컬 배율 × 랜덤(0.9~1.1)

- 최소 데미지: 1 (0 이하 방지)
- 크리티컬 배율: 고정 1.5x
- 크리티컬 확률: 고정 5% (기억 단편으로 보정 가능, 최대 50%)
```
