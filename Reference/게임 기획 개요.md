# 프로젝트 기획 개요: 메트로베니아 x 아이템계 온라인 액션 RPG

> **프로젝트명:** (가칭) Project Abyss
> **장르:** 웹 기반 횡스크롤 온라인 액션 RPG (메트로베니아 + 야리코미)
> **플랫폼:** 웹 브라우저 (PC 우선)
> **타겟:** 탐험과 파밍을 즐기는 코어~미드코어 유저
> **레퍼런스:** 월하의 야상곡(탐험/전투) + 디스가이아(아이템계/야리코미) + 스펠렁키(절차적 생성)
> **작성일:** 2026-03-23

---

## 1. 핵심 컨셉

### 1.1 한 줄 요약

> **"거대한 고딕 세계를 탐험하고, 획득한 아이템 속으로 들어가 끝없이 강화하며, 다른 플레이어와 함께 싸우는 횡스크롤 온라인 액션 RPG"**

### 1.2 핵심 판타지 (Core Fantasy)

| 판타지 | 설명 |
|:---|:---|
| **탐험가** | 능력을 하나씩 얻으며 갈 수 없던 곳을 뚫어 세계의 비밀을 밝혀낸다 |
| **장인** | 아이템 속에 들어가 이노센트를 사냥하고, 세상에 하나뿐인 최강 장비를 만든다 |
| **모험가** | 친구와 함께 끝없는 심연의 던전에 도전하고, 위기를 함께 극복한다 |

### 1.3 장르 결합의 핵심 질문과 해답

| 질문 | 해답 |
|:---|:---|
| 메트로베니아의 탐험감과 온라인 멀티플레이가 양립 가능한가? | 2-Space 분리 모델로 해결. 월드는 개인 탐험 (대장간/상점은 세이브 포인트), 아이템계는 협동 (URL 링크 파티 합류) |
| 야리코미(무한 파밍)가 탐험의 가치를 훼손하지 않는가? | 순환 구조로 해결. 더 깊은 아이템계를 위해 더 넓은 월드를 탐험해야 함 |

## 2. 2-Space 분리 모델

게임 세계를 **월드(World)**와 **아이템계(Item World)** 두 공간으로 분리하여, 각 공간이 고유한 규칙과 목적을 갖도록 설계한다.

> **변경 이력 (2026-04-04):** 허브(Hub)는 독립 공간에서 폐기. 대장간/상점은 월드 세이브 포인트에 통합. 파티 합류는 URL 링크 공유로 아이템계 직접 진입.

### 2.1 공간 비교표

| 항목 | 월드 (World) | 아이템계 (Item World) |
|:---|:---|:---|
| **핵심 목적** | 탐험, 능력 획득, 스토리, 대장간/상점 (세이브 포인트) | 아이템 강화, 야리코미 |
| **인원** | 솔로(1인) | 1 ~ 4인 |
| **맵 유형** | 핸드크래프트 + 절차적 혼합 | 완전 절차적 생성 |
| **진행 방식** | 능력 게이트 (메트로베니아) | 지층 클리어 (레어리티별 2~4 지층) |
| **사망 페널티** | 세이브 포인트 복귀 | 진행 지층 손실 (탈출 아이템 미사용 시) |
| **고유 자원** | 능력 렐릭, 맵 데이터, 월드 소재 | 이노센트, 아이템 경험치, 레벨 구슬 |

### 2.2 공간 구조도

```
┌─────────────────────┐     ┌─────────────────────────┐
│    월드 (World)       │     │   아이템계 (Item World)   │
│  - 메트로베니아 탐험   │────▶│  - 기억의 지층 던전       │
│  - 능력 게이트        │     │  - 이노센트 사냥          │
│  - 보스/미니보스      │     │  - 지층별 보스            │
│  - 아이템 획득        │     │  - 재귀적 진입 (최대 3중)  │
│  - 스탯 게이트        │     │  - 1~4인 협동             │
│  - 세이브 포인트      │     │  (URL 링크로 파티 합류)    │
│    (대장간/상점)      │◀────│                           │
└─────────────────────┘     └─────────────────────────┘
         │                             │
         └──────────┐  ┌───────────────┘
                    ▼  ▼
              ┌──────────────┐
              │  플레이어 성장  │
              │  장비 + 능력   │
              └──────────────┘
```

## 3. 월드 (World) 설계

### 3.1 월드 구조 원칙

월하의 야상곡의 성(城) 구조를 레퍼런스로, **허브 & 스포크 + 원형 순환** 구조를 채택한다.

```
                        ┌──────────┐
                 ┌──────┤ 천공의 탑  ├──────┐
                 │      └──────────┘      │
          ┌──────┴──────┐          ┌──────┴──────┐
          │  마법 연구소  │          │  빙결 동굴   │
          └──────┬──────┘          └──────┬──────┘
                 │      ┌──────────┐      │
                 ├──────┤ 중앙 성채  ├──────┤
                 │      │ (세이브PT) │      │
                 │      └──────────┘      │
          ┌──────┴──────┐          ┌──────┴──────┐
          │  지하 수로   │          │  묘지/카타콤  │
          └──────┬──────┘          └──────┬──────┘
                 │      ┌──────────┐      │
                 └──────┤ 심연의 구  ├──────┘
                        └──────────┘
```

### 3.2 스탯 게이트 시스템

월하의 야상곡의 렐릭 시스템 + 디스가이아의 스탯 시스템을 결합한 **스탯 게이트**를 도입한다. 장비의 스탯이 월드 진행의 열쇠가 된다.

| 스탯 | 게이트 유형 | 예시 |
|:---|:---|:---|
| **ATK** | 물리 장벽 | 금이 간 벽 파괴, 바리케이드 돌파, 중문/쇠사슬/바위 |
| **INT** | 마법 봉인 | 룬 문양 장벽 해제, 마법 봉인 문, 아케인 잠금장치 |

> **설계 변경:** 기존 6대 스탯 게이트(STR/INT/DEX/VIT/SPD/LCK)는 ATK/INT 이중 게이트로 재설계. ATK는 물리 장벽(파괴), INT는 마법 봉인(해제). 기동성/환경 저항/시간 제한 등은 능력 게이트나 환경 퍼즐로 재설계 예정.

**핵심 설계 의도:** 아이템계에서 장비를 강화하면 월드의 새로운 구역이 열린다. 이것이 월드 ↔ 아이템계 순환의 핵심 동력이다.

### 3.3 능력 게이트 시스템

전통적인 메트로베니아 능력 게이트도 병행한다. 보스를 처치하거나 숨겨진 렐릭을 발견하여 획득한다.

| 능력 | 해금 조건 | 열리는 구역 |
|:---|:---|:---|
| 이단 점프 | 묘지 보스 처치 | 높은 절벽 위 구역 |
| 벽 타기/벽 점프 | 지하 수로 렐릭 | 수직 통로, 좁은 갱도 |
| 안개 변신 | 마법 연구소 보스 | 격자/좁은 틈새 |
| 수중 호흡 | 빙결 동굴 렐릭 | 수중 구역 |
| 역중력 | 천공의 탑 보스 | 반전 구역 (역전 성) |

### 3.4 맵 생성 방식: 스펠렁키 하이브리드

- **매크로 구조:** 핸드크래프트. 구역 간 연결, 보스 방, 세이브/워프 룸은 고정 배치
- **마이크로 구조:** 절차적 생성. 각 구역 내부의 방(Room)은 스펠렁키 방식의 템플릿 조립
- **시드:** 서버 고정 시드
- **Always Winnable:** 모든 시드에서 클리어 가능한 Critical Path 보장

```
[매크로 - 고정]                    [마이크로 - 절차적]
구역 A ──── 구역 B                 ┌───┬───┬───┬───┐
  │            │                   │ R1│ R2│ R3│ R4│ ← 4x4 Room Grid
구역 C ──── 구역 D                 ├───┼───┼───┼───┤
  │            │                   │ R5│ R6│ R7│ R8│
구역 E ──── (보스)                 ├───┼───┼───┼───┤
                                   │R9 │R10│R11│R12│
각 구역은 여러 Room Grid로 구성     ├───┼───┼───┼───┤
Room Grid 내부는 템플릿 조립       │R13│R14│R15│R16│
                                   └───┴───┴───┴───┘
                                   각 Room은 핸드크래프트 Chunk 조합
```

## 4. 아이템계 (Item World) 설계

### 4.1 핵심 개념

> 모든 장비 아이템은 내부에 **던전**을 가지고 있다. 플레이어는 아이템 속에 들어가 층을 클리어하며, 아이템을 직접 강화한다.

### 4.2 기본 구조

| 항목 | 사양 |
|:---|:---|
| 총 층수 | 100층 (레어리티에 따라 진입 가능 최대 층 상이) |
| 보스 층 | 10, 20, 30, ... 100층 (10층마다) |
| 세이브 포인트 | 보스 처치 후 (10층 단위) |
| 탈출 | Mr. Gency's Exit (소모품) 또는 보스 층 클리어 후 |
| 시드 | `hash(itemID + level + floor)` → 동일 아이템의 동일 층은 항상 같은 맵 |
| 인원 | 1~4인 (파티 리더의 아이템에 진입) |
| 재귀 진입 | 아이템계 내에서 획득한 아이템의 아이템계 진입 가능 (최대 깊이 3) |

### 4.3 층 생성 규칙

스펠렁키의 절차적 생성을 횡스크롤 액션에 맞게 변형한다.

```
[층 생성 파이프라인]

1. 시드 결정
   seed = hash(itemID + itemLevel + floorNumber)

2. 레이아웃 생성 (Phase 1)
   - Room Grid 크기 결정 (난이도에 따라 3x3 ~ 5x5)
   - Critical Path 생성 (입구 → 출구 보장)
   - Room Type 배정 (출입구 방향)

3. 지형 조립 (Phase 2)
   - 각 Room에 Chunk 템플릿 삽입
   - 아이템 레어리티에 따라 Chunk 풀 변경 (고급 = 복잡한 지형)

4. 오브젝트 배치 (Phase 3)
   - 적 배치 (층수 기반 난이도 스케일링)
   - 이노센트 배치 (야생 이노센트)
   - 보상 오브젝트 배치 (레벨 구슬, 보물상자)
   - 지오 이펙트 패널 배치 (선택적)

5. 특수 이벤트 (Phase 4)
   - 미스터리 룸 생성 확률 체크
   - 이노센트 타운 출현 체크 (특정 층)
```

### 4.4 보스 시스템

| 보스 등급 | 출현 층 | 처치 보상 |
|:---|:---|:---|
| 아이템 장군 | 10, 20, 30층 | 아이템 스탯 +5% |
| 아이템 왕 | 40, 50, 60층 | 아이템 스탯 +10%, 이노센트 슬롯 +1 |
| 아이템 신 | 70, 80, 90층 | 아이템 스탯 +15%, 특수 이노센트 드랍 |
| 아이템 대신 | 100층 | 아이템 레어리티 승급 기회, 대량 보상 |

### 4.5 재귀적 아이템계

```
[현실 세계]
  └── 내 검의 아이템계 (깊이 1)
       ├── 30층에서 레어 갑옷 획득
       │   └── 그 갑옷의 아이템계 (깊이 2)
       │        ├── 15층에서 반지 획득
       │        │   └── 그 반지의 아이템계 (깊이 3, 최대)
       │        │        └── 더 이상 재귀 불가
       │        └── 탈출 → 깊이 2로 복귀
       └── 탈출 → 깊이 1로 복귀
```

**재귀 보너스:** 깊은 층에서 얻는 이노센트/아이템에 깊이 보너스 적용
- 깊이 2: 이노센트 효과 +20%
- 깊이 3: 이노센트 효과 +50%, 희귀 이노센트 출현율 증가

## 5. 이노센트 시스템

### 5.1 개요

이노센트는 아이템에 거주하며 추가 능력치를 부여하는 존재. 디스가이아의 핵심 야리코미 요소를 온라인 게임에 맞게 재설계한다.

### 5.2 상태

| 상태 | 표시 | 효과 | 설명 |
|:---|:---|:---|:---|
| 야생 (Wild) | 빨간색 | 기본 효과 | 아이템계에서 적으로 등장, 격파 시 복종 |
| 복종 (Tamed) | 파란색 | 효과 2배 | 다른 아이템으로 이동 가능 |

### 5.3 이노센트 종류

| 분류 | 이노센트 | 효과 | 비고 |
|:---|:---|:---|:---|
| 기본형 | Gladiator (글래디에이터) | ATK +N (1/레벨) | 물리 빌드, ATK 게이트 해금 |
| 기본형 | Tutor (튜터) | INT +N (1/레벨) | 원소 빌드, INT 게이트 해금 |
| 기본형 | Dietician (영양사) | MaxHP +N (1/레벨) | 생존력 |
| 행동형 | Berserker (광전사) | 공격 속도 +15% | 빠른 콤보, DPS |
| 행동형 | Vampire (뱀파이어) | 적 처치 시 HP 5% 회복 | 공격적 생존 |
| 행동형 | Ironclad (철벽) | 넉백 -50% | 보스전 안정 |
| 행동형 | Ghost (유령) | 대시 i-frame +50ms, 거리 +20% | 회피 특화 |
| 행동형 | Sprinter (질주자) | 이동 속도 +15% | 탐험/스피드런 |
| 행동형 | Leech (거머리) | 공격 적중 시 HP 1 회복 (0.5s CD) | 보스 지구전 |
| 원소형 | Burner (버너) | 화 인챈트 데미지 +20%, 화상 지속 +30% | 아이템계 드랍 |
| 원소형 | Freezer (프리저) | 빙 인챈트 데미지 +20%, 빙결 지속 +30% | 아이템계 드랍 |
| 원소형 | Shocker (쇼커) | 뇌 인챈트 데미지 +20%, 감전 지속 +30% | 아이템계 드랍 |
| 파밍형 | Statistician (통계사) | EXP +N% | 레벨링 효율 |
| 파밍형 | Scavenger (수집가) | 드랍률 +N% | 파밍 효율 |
| 서사형 | No.1 (넘버 원) | P1: 없음. P2: 서사 진행 | 슬롯 0 (에코 부착) |

> **설계 변경:** ATK/INT/HP 3스탯 체계에 맞춰 5분류 15종 이노센트로 재설계. Tutor(INT+) 복원, Sentry(DEF+) DEPRECATED, Collector → Scavenger 통합. 행동형 6종 신규 추가.

## 6. 아이템/장비 시스템

### 6.1 레어리티

| 등급 | 색상 | 스탯 배율 | 이노센트 슬롯 | 아이템계 층수 | 드랍 확률 |
|:---|:---|:---|:---|:---|:---|
| Common | 흰색 | x1.0 | 2 | 30층 | 60% |
| Uncommon | 초록 | x1.3 | 3 | 50층 | 25% |
| Rare | 파랑 | x1.7 | 4 | 70층 | 10% |
| Legendary | 보라 | x2.2 | 6 | 100층 | 4% |
| Mythic | 금색 | x3.0 | 8 | 100층 + 보너스 | 1% |

### 6.2 장비 슬롯

| 슬롯 | 종류 | 비고 |
|:---|:---|:---|
| 무기 | 검, 대검, 단검, 활, 지팡이 | 5종 무기 체계 (근접 3 + 원거리 2) |
| 보조무기 | 방패, 마법서, 횃불 | 서브웨폰 슬롯 겸용 가능 |
| 머리 | 투구, 모자, 서클릿 | |

## 7. 순환 구조 (Core Loop)

### 7.1 메인 순환

```
        ┌─────────────────────────────────────────┐
        │                                         │
        ▼                                         │
  [월드 탐험] ──아이템 획득──▶ [아이템계 진입]        │
        ▲                         │               │
        │                    아이템 강화            │
        │                    이노센트 수집          │
        │                         │               │
        │                         ▼               │
        │                   [장비 스탯 상승] ───────┘
        │                         │
        │                         ▼
        │                   [스탯 게이트 해금]
        │                         │
        └─────── 새 구역 탐험 ◀────┘
```

### 7.2 순환 동력 분석

| 순환 요소 | 동력 | 효과 |
|:---|:---|:---|
| 월드 → 아이템계 | 새 장비 획득, 더 강한 장비 필요 | 아이템계 진입 동기 |
| 아이템계 → 장비 강화 | 층 클리어, 이노센트 수집 | 파밍/야리코미 만족 |
| 장비 강화 → 월드 | 스탯 게이트 해금 | 새 구역 탐험 동기 |
| 월드 탐험 → 능력 획득 | 보스 처치, 렐릭 발견 | 이동 범위 확대 |
| 능력 → 월드 접근성 | 이단점프, 변신 등 | 미탐험 구역 접근 |

### 7.3 자원 순환표

| 자원 | 획득처 | 소비처 | 역할 |
|:---|:---|:---|:---|
| 장비 아이템 | 월드 드랍, 아이템계 보상 | 아이템계 진입 소재 | 순환의 핵심 매개 |
| 이노센트 | 아이템계 내 전투 | 장비에 장착, 합성 | 야리코미 깊이 |
| 골드 | 적 처치, 아이템 판매 | 상점, 합성, 이노센트 팜 | 기본 경제 |
| 월드 소재 | 월드 채집, 보스 드랍 | 장비 제작, 세이브 포인트 대장간 | 월드 탐험 동기 |
| 능력 렐릭 | 월드 보스/숨겨진 방 | 소비 없음 (영구) | 탐험 진행 |
| Mr. Gency's Exit | 상점, 아이템계 드랍 | 아이템계 긴급 탈출 | 리스크 관리 |

## 8. 멀티플레이어 설계

### 8.1 공간별 인원 구조

| 공간 | 최소 인원 | 최대 인원 | 매칭 방식 |
|:---|:---|:---|:---|
| 월드 | 1 | 1 (솔로) | — |
| 아이템계 | 1 | 4 | URL 링크 공유로 직접 합류 |

### 8.2 멀티플레이 설계 원칙

1. **월드는 개인의 영역:** 탐험의 발견감을 보존하기 위해 솔로 전용
2. **아이템계는 협동의 영역:** 높은 난이도의 깊은 지층은 파티 플레이로 설계. 역할 분담 (탱커, 딜러, 서포터). URL 링크 공유로 친구가 아이템계에 직접 합류
3. **비동기 멀티:** 다른 플레이어의 유령(고스트)이 월드에 나타나 힌트를 남기는 시스템 (Dark Souls 메시지)

### 8.3 파티 역할과 시너지

| 역할 | 무기 예시 | 아이템계 기여 |
|:---|:---|:---|
| 탱커 | 대검+방패 | 보스 어그로 관리, 팀 보호 |
| 딜러 | 검, 단검 | 빠른 층 클리어 |
| 레인저 | 활, 투척 | 원거리 안전 딜, 함정 해제 |
| 서포터 | 지팡이, 마법서 | 회복, 버프, 이노센트 탐지 |

## 11. RPG 성장 시스템

### 11.1 캐릭터 스탯

| 스탯 | 약어 | 역할 |
|:---|:---|:---|
| 공격력 | ATK | 물리 데미지 산출. 장비 ATK + 캐릭터 기본 ATK. ATK 게이트(물리 장벽) 해금 |
| 지력 | INT | 원소/인챈트 데미지 산출. 장비 INT + 캐릭터 기본 INT. INT 게이트(마법 봉인) 해금. 상태이상 지속시간 스케일 |
| 체력 | HP | 생존. MaxHP는 레벨 + 이노센트(Dietician)로 성장 |
| 방어력 | DEF | 물리 피해 감소. 장비에서 직접 제공 |
| 마법저항 | RES | 원소 피해 감소. 장비에서 직접 제공 |

> **설계 변경:** 기존 9개 스탯(HP/MP/STR/INT/DEF/RES/DEX/SPD/LCK) → ATK/INT/HP 3스탯 + DEF/RES(장비 제공)로 단순화. MP 삭제 (스킬은 쿨다운). 크리티컬/드랍률/이동속도는 이노센트로 보정. 원소 적중률 100% (상태이상 항상 발동, 지속시간은 INT 스케일).

### 11.2 성장 공식

```
Final_ATK = Base_ATK + Equipment_ATK + Innocent_ATK_Bonus
Final_INT = Base_INT + Equipment_INT + Innocent_INT_Bonus
Final_DEF = Equipment_DEF + Buff_DEF
Final_RES = Equipment_RES + Buff_RES
Final_MaxHP = Base_HP + Level_Bonus_HP + Innocent_HP_Bonus

- Base_ATK: 캐릭터 기본 공격력, Base_INT: 캐릭터 기본 지력
- Equipment_ATK/INT: 장비 ATK/INT × 레어리티 배율
- Innocent_ATK_Bonus: Gladiator 합산, Innocent_INT_Bonus: Tutor 합산
```

### 11.3 데미지 공식

```
물리 데미지 = (ATK × 스킬 배율 - DEF × 0.5) × 크리티컬 배율 × 랜덤(0.9~1.1)
원소 데미지 = (INT × 원소 배율 - RES × 0.5) × 크리티컬 배율 × 랜덤(0.9~1.1)

- 최소 데미지: 1 (0 이하 방지)
- 크리티컬 배율: 고정 1.5x
- 크리티컬 확률: 고정 5% (이노센트로 보정 가능, 최대 50%)
```
