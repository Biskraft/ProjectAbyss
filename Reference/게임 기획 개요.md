# 프로젝트 기획 개요: 메트로베니아 x 아이템계 온라인 액션 RPG

> **프로젝트명:** (가칭) Project Abyss
> **장르:** 웹 기반 횡스크롤 온라인 액션 RPG (메트로베니아 + 야리코미)
> **플랫폼:** 웹 브라우저 (PC 우선)
> **타겟:** 탐험과 파밍을 즐기는 코어~미드코어 유저
> **레퍼런스:** 월하의 야상곡(탐험/전투) + 디스가이아(아이템계/야리코미) + 스펠렁키(절차적 생성)
> **작성일:** 2026-03-23

---

## 1. 핵심 컨셉

### 1.1 한 줄 요약

> **"거대한 고딕 세계를 탐험하고, 획득한 아이템 속으로 들어가 끝없이 강화하며, 다른 플레이어와 함께 싸우는 횡스크롤 온라인 액션 RPG"**

### 1.2 핵심 판타지 (Core Fantasy)

| 판타지 | 설명 |
|:---|:---|
| **탐험가** | 능력을 하나씩 얻으며 갈 수 없던 곳을 뚫어 세계의 비밀을 밝혀낸다 |
| **장인** | 아이템 속에 들어가 이노센트를 사냥하고, 세상에 하나뿐인 최강 장비를 만든다 |
| **모험가** | 친구와 함께 끝없는 심연의 던전에 도전하고, 위기를 함께 극복한다 |

### 1.3 장르 결합의 핵심 질문과 해답

| 질문 | 해답 |
|:---|:---|
| 메트로베니아의 탐험감과 온라인 멀티플레이가 양립 가능한가? | 3-Space 분리 모델로 해결. 월드는 개인 탐험, 아이템계는 협동, 허브는 소셜 |
| 야리코미(무한 파밍)가 탐험의 가치를 훼손하지 않는가? | 순환 구조로 해결. 더 깊은 아이템계를 위해 더 넓은 월드를 탐험해야 함 |
| 자동 사냥이 게임의 재미를 해치지 않는가? | 수동 정복의 보상으로 자동 사냥 해금. 효율 차등으로 직접 플레이 유인 |

## 2. 3-Space 분리 모델

게임 세계를 **월드(World)**, **아이템계(Item World)**, **허브(Hub)** 세 공간으로 분리하여, 각 공간이 고유한 규칙과 목적을 갖도록 설계한다.

### 2.1 공간 비교표

| 항목 | 월드 (World) | 아이템계 (Item World) | 허브 (Hub) |
|:---|:---|:---|:---|
| **핵심 목적** | 탐험, 능력 획득, 스토리 | 아이템 강화, 야리코미 | 사교, 거래, 준비 |
| **인원** | 솔로(1인) | 1 ~ 4인 | 무제한 |
| **맵 유형** | 핸드크래프트 + 절차적 혼합 | 완전 절차적 생성 | 고정 맵 |
| **진행 방식** | 능력 게이트 (메트로베니아) | 층 클리어 (100층) | 자유 이동 |
| **사망 페널티** | 세이브 포인트 복귀 | 진행 층 손실 (탈출 아이템 미사용 시) | 없음 |
| **고유 자원** | 능력 렐릭, 맵 데이터, 월드 소재 | 이노센트, 아이템 경험치, 레벨 구슬 | 상점, 합성, 창고 |

### 2.2 공간 구조도

```
                    ┌─────────────────────┐
                    │      허브 (Hub)       │
                    │  - 마을, 상점, 대장간  │
                    │  - 이노센트 팜        │
                    │  - 파티 매칭          │
                    │  - 거래소             │
                    └─────┬─────┬─────────┘
                          │     │
              ┌───────────┘     └───────────┐
              ▼                             ▼
┌─────────────────────┐     ┌─────────────────────────┐
│    월드 (World)       │     │   아이템계 (Item World)   │
│  - 메트로베니아 탐험   │────▶│  - 절차적 던전 100층      │
│  - 능력 게이트        │     │  - 이노센트 사냥          │
│  - 보스/미니보스      │     │  - 10층마다 보스          │
│  - 아이템 획득        │     │  - 재귀적 진입 (최대 3중)  │
│  - 스탯 게이트        │     │  - 1~4인 협동             │
└─────────────────────┘     └─────────────────────────┘
         │                             │
         └──────────┐  ┌───────────────┘
                    ▼  ▼
              ┌──────────────┐
              │  플레이어 성장  │
              │  장비 + 능력   │
              └──────────────┘
```

## 3. 월드 (World) 설계

### 3.1 월드 구조 원칙

월하의 야상곡의 성(城) 구조를 레퍼런스로, **허브 & 스포크 + 원형 순환** 구조를 채택한다.

```
                        ┌──────────┐
                 ┌──────┤ 천공의 탑  ├──────┐
                 │      └──────────┘      │
          ┌──────┴──────┐          ┌──────┴──────┐
          │  마법 연구소  │          │  빙결 동굴   │
          └──────┬──────┘          └──────┬──────┘
                 │      ┌──────────┐      │
                 ├──────┤ 중앙 성채  ├──────┤
                 │      │  (허브)   │      │
                 │      └──────────┘      │
          ┌──────┴──────┐          ┌──────┴──────┐
          │  지하 수로   │          │  묘지/카타콤  │
          └──────┬──────┘          └──────┬──────┘
                 │      ┌──────────┐      │
                 └──────┤ 심연의 구  ├──────┘
                        └──────────┘
```

### 3.2 스탯 게이트 시스템

월하의 야상곡의 렐릭 시스템 + 디스가이아의 스탯 시스템을 결합한 **스탯 게이트**를 도입한다. 장비의 스탯이 월드 진행의 열쇠가 된다.

| 스탯 | 게이트 유형 | 예시 |
|:---|:---|:---|
| **STR** | 물리적 장벽 | 무거운 문 밀기, 바위 파괴, 무거운 레버 당기기 |
| **INT** | 마법 봉인 | 마법 장벽 해제, 룬 퍼즐, 마법진 활성화 |
| **DEX** | 기동성 장벽 | 좁은 틈새, 빠른 함정 구간, 정밀 점프 |
| **VIT** | 환경 저항 | 독 지대 통과, 용암 구간, 저온 지역 |
| **SPD** | 시간제한 구간 | 닫히는 문, 무너지는 바닥, 추적 장치 |
| **LCK** | 숨겨진 경로 | 행운 수치에 따른 비밀 방 발견 확률 |

**핵심 설계 의도:** 아이템계에서 장비를 강화하면 월드의 새로운 구역이 열린다. 이것이 월드 ↔ 아이템계 순환의 핵심 동력이다.

### 3.3 능력 게이트 시스템

전통적인 메트로베니아 능력 게이트도 병행한다. 보스를 처치하거나 숨겨진 렐릭을 발견하여 획득한다.

| 능력 | 해금 조건 | 열리는 구역 |
|:---|:---|:---|
| 이단 점프 | 묘지 보스 처치 | 높은 절벽 위 구역 |
| 벽 타기/벽 점프 | 지하 수로 렐릭 | 수직 통로, 좁은 갱도 |
| 안개 변신 | 마법 연구소 보스 | 격자/좁은 틈새 |
| 수중 호흡 | 빙결 동굴 렐릭 | 수중 구역 |
| 역중력 | 천공의 탑 보스 | 반전 구역 (역전 성) |

### 3.4 맵 생성 방식: 스펠렁키 하이브리드

- **매크로 구조:** 핸드크래프트. 구역 간 연결, 보스 방, 세이브/워프 룸은 고정 배치
- **마이크로 구조:** 절차적 생성. 각 구역 내부의 방(Room)은 스펠렁키 방식의 템플릿 조립
- **시드:** 서버 고정 시드. 시즌 리셋(3~6개월)으로 세계 재생성
- **Always Winnable:** 모든 시드에서 클리어 가능한 Critical Path 보장

```
[매크로 - 고정]                    [마이크로 - 절차적]
구역 A ──── 구역 B                 ┌───┬───┬───┬───┐
  │            │                   │ R1│ R2│ R3│ R4│ ← 4x4 Room Grid
구역 C ──── 구역 D                 ├───┼───┼───┼───┤
  │            │                   │ R5│ R6│ R7│ R8│
구역 E ──── (보스)                 ├───┼───┼───┼───┤
                                   │R9 │R10│R11│R12│
각 구역은 여러 Room Grid로 구성     ├───┼───┼───┼───┤
Room Grid 내부는 템플릿 조립       │R13│R14│R15│R16│
                                   └───┴───┴───┴───┘
                                   각 Room은 핸드크래프트 Chunk 조합
```

## 4. 아이템계 (Item World) 설계

### 4.1 핵심 개념

> 모든 장비 아이템은 내부에 **던전**을 가지고 있다. 플레이어는 아이템 속에 들어가 층을 클리어하며, 아이템을 직접 강화한다.

### 4.2 기본 구조

| 항목 | 사양 |
|:---|:---|
| 총 층수 | 100층 (레어리티에 따라 진입 가능 최대 층 상이) |
| 보스 층 | 10, 20, 30, ... 100층 (10층마다) |
| 세이브 포인트 | 보스 처치 후 (10층 단위) |
| 탈출 | Mr. Gency's Exit (소모품) 또는 보스 층 클리어 후 |
| 시드 | `hash(itemID + level + floor)` → 동일 아이템의 동일 층은 항상 같은 맵 |
| 인원 | 1~4인 (파티 리더의 아이템에 진입) |
| 재귀 진입 | 아이템계 내에서 획득한 아이템의 아이템계 진입 가능 (최대 깊이 3) |

### 4.3 층 생성 규칙

스펠렁키의 절차적 생성을 횡스크롤 액션에 맞게 변형한다.

```
[층 생성 파이프라인]

1. 시드 결정
   seed = hash(itemID + itemLevel + floorNumber)

2. 레이아웃 생성 (Phase 1)
   - Room Grid 크기 결정 (난이도에 따라 3x3 ~ 5x5)
   - Critical Path 생성 (입구 → 출구 보장)
   - Room Type 배정 (출입구 방향)

3. 지형 조립 (Phase 2)
   - 각 Room에 Chunk 템플릿 삽입
   - 아이템 레어리티에 따라 Chunk 풀 변경 (고급 = 복잡한 지형)

4. 오브젝트 배치 (Phase 3)
   - 적 배치 (층수 기반 난이도 스케일링)
   - 이노센트 배치 (야생 이노센트)
   - 보상 오브젝트 배치 (레벨 구슬, 보물상자)
   - 지오 이펙트 패널 배치 (선택적)

5. 특수 이벤트 (Phase 4)
   - 미스터리 룸 생성 확률 체크
   - 이노센트 타운 출현 체크 (특정 층)
```

### 4.4 보스 시스템

| 보스 등급 | 출현 층 | 처치 보상 |
|:---|:---|:---|
| 아이템 장군 | 10, 20, 30층 | 아이템 스탯 +5% |
| 아이템 왕 | 40, 50, 60층 | 아이템 스탯 +10%, 이노센트 슬롯 +1 |
| 아이템 신 | 70, 80, 90층 | 아이템 스탯 +15%, 특수 이노센트 드랍 |
| 아이템 대신 | 100층 | 아이템 레어리티 승급 기회, 대량 보상 |

### 4.5 재귀적 아이템계

```
[현실 세계]
  └── 내 검의 아이템계 (깊이 1)
       ├── 30층에서 레어 갑옷 획득
       │   └── 그 갑옷의 아이템계 (깊이 2)
       │        ├── 15층에서 반지 획득
       │        │   └── 그 반지의 아이템계 (깊이 3, 최대)
       │        │        └── 더 이상 재귀 불가
       │        └── 탈출 → 깊이 2로 복귀
       └── 탈출 → 깊이 1로 복귀
```

**재귀 보너스:** 깊은 층에서 얻는 이노센트/아이템에 깊이 보너스 적용
- 깊이 2: 이노센트 효과 +20%
- 깊이 3: 이노센트 효과 +50%, 희귀 이노센트 출현율 증가

## 5. 이노센트 시스템

### 5.1 개요

이노센트는 아이템에 거주하며 추가 능력치를 부여하는 존재. 디스가이아의 핵심 야리코미 요소를 온라인 게임에 맞게 재설계한다.

### 5.2 상태

| 상태 | 표시 | 효과 | 설명 |
|:---|:---|:---|:---|
| 야생 (Wild) | 빨간색 | 기본 효과 | 아이템계에서 적으로 등장, 격파 시 복종 |
| 복종 (Tamed) | 파란색 | 효과 2배 | 다른 아이템으로 이동 가능 |

### 5.3 이노센트 종류

| 이노센트 | 효과 | 최대 레벨 | 비고 |
|:---|:---|:---|:---|
| 검사 (Gladiator) | ATK +N | 19998 | 물리 공격력 |
| 마법사 (Tutor) | INT +N | 19998 | 마법 공격력 |
| 수호자 (Sentry) | DEF +N | 19998 | 물리 방어력 |
| 의술사 (Physician) | RES +N | 19998 | 마법 방어력 |
| 레인저 (Marksman) | HIT +N | 19998 | 명중률 |
| 코치 (Coach) | SPD +N | 19998 | 속도 |
| 축복사 (Broker) | 판매가 +N% | 300 | 상점 판매 보너스 |
| 사냥꾼 (Hunter) | 적 드랍률 +N% | 100 | 파밍 효율 |
| 경험의 왕 (Statistician) | EXP +N% | 300 | 경험치 보너스 |
| 도적 (Thief) | 도둑 확률 +N% | 100 | 아이템 탈취 |

### 5.4 이노센트 팜 (허브 시설)

| 기능 | 설명 |
|:---|:---|
| 합성 | 같은 종류의 이노센트 2마리 합성 → 레벨 합산 (상한 존재) |
| 번식 | 이노센트 2마리 배치 → 시간 경과 후 새 이노센트 탄생 (확률적) |
| 듀얼 이노센트 | 서로 다른 종류 합성 → 2가지 스탯을 가진 특수 이노센트 (희귀) |
| 이동 | 복종 상태 이노센트를 다른 장비로 이동 |

## 6. 아이템/장비 시스템

### 6.1 레어리티

| 등급 | 색상 | 스탯 배율 | 이노센트 슬롯 | 아이템계 층수 | 드랍 확률 |
|:---|:---|:---|:---|:---|:---|
| Common | 흰색 | x1.0 | 2 | 30층 | 60% |
| Uncommon | 초록 | x1.3 | 3 | 50층 | 25% |
| Rare | 파랑 | x1.7 | 4 | 70층 | 10% |
| Legendary | 보라 | x2.2 | 6 | 100층 | 4% |
| Mythic | 금색 | x3.0 | 8 | 100층 + 보너스 | 1% |

### 6.2 장비 슬롯

| 슬롯 | 종류 | 비고 |
|:---|:---|:---|
| 무기 | 검, 창, 도끼, 채찍, 지팡이, 너클, 활, 대검 | 무기별 고유 모션 |
| 보조무기 | 방패, 마법서, 단검, 횃불 | 서브웨폰 슬롯 겸용 가능 |
| 머리 | 투구, 모자, 서클릿 | |
| 갑옷 | 중갑, 경갑, 로브 | |
| 망토 | 망토, 날개 | 특수 이동 능력 부여 가능 |
| 장신구 x2 | 반지, 목걸이, 부적 | 2슬롯 |

### 6.3 아이템 성장 경로

```
[아이템 획득] → [아이템계 진입] → [층 클리어로 아이템 레벨업]
                                        │
                              ┌─────────┼─────────┐
                              ▼         ▼         ▼
                        보스 처치    이노센트    레벨 구슬
                        스탯 증가    슬롯 채움    아이템 Lv +5
```

## 7. 순환 구조 (Core Loop)

### 7.1 메인 순환

```
        ┌─────────────────────────────────────────┐
        │                                         │
        ▼                                         │
  [월드 탐험] ──아이템 획득──▶ [아이템계 진입]        │
        ▲                         │               │
        │                    아이템 강화            │
        │                    이노센트 수집          │
        │                         │               │
        │                         ▼               │
        │                   [장비 스탯 상승] ───────┘
        │                         │
        │                         ▼
        │                   [스탯 게이트 해금]
        │                         │
        └─────── 새 구역 탐험 ◀────┘
```

### 7.2 순환 동력 분석

| 순환 요소 | 동력 | 효과 |
|:---|:---|:---|
| 월드 → 아이템계 | 새 장비 획득, 더 강한 장비 필요 | 아이템계 진입 동기 |
| 아이템계 → 장비 강화 | 층 클리어, 이노센트 수집 | 파밍/야리코미 만족 |
| 장비 강화 → 월드 | 스탯 게이트 해금 | 새 구역 탐험 동기 |
| 월드 탐험 → 능력 획득 | 보스 처치, 렐릭 발견 | 이동 범위 확대 |
| 능력 → 월드 접근성 | 이단점프, 변신 등 | 미탐험 구역 접근 |

### 7.3 자원 순환표

| 자원 | 획득처 | 소비처 | 역할 |
|:---|:---|:---|:---|
| 장비 아이템 | 월드 드랍, 아이템계 보상 | 아이템계 진입 소재 | 순환의 핵심 매개 |
| 이노센트 | 아이템계 내 전투 | 장비에 장착, 합성 | 야리코미 깊이 |
| 골드 | 적 처치, 아이템 판매 | 상점, 합성, 이노센트 팜 | 기본 경제 |
| 월드 소재 | 월드 채집, 보스 드랍 | 장비 제작, 허브 시설 | 월드 탐험 동기 |
| 능력 렐릭 | 월드 보스/숨겨진 방 | 소비 없음 (영구) | 탐험 진행 |
| Mr. Gency's Exit | 상점, 아이템계 드랍 | 아이템계 긴급 탈출 | 리스크 관리 |

## 8. 멀티플레이어 설계

### 8.1 공간별 인원 구조

| 공간 | 최소 인원 | 최대 인원 | 매칭 방식 |
|:---|:---|:---|:---|
| 월드 | 1 | 2 (초대) | 친구 초대 전용 |
| 아이템계 | 1 | 4 | 파티 매칭 / 초대 |
| 허브 | - | 제한 없음 | 자동 (채널) |

### 8.2 멀티플레이 설계 원칙

1. **월드는 개인의 영역:** 탐험의 발견감을 보존하기 위해 솔로 또는 2인 협동만 허용
2. **아이템계는 협동의 영역:** 높은 난이도의 깊은 층은 파티 플레이로 설계. 역할 분담 (탱커, 딜러, 서포터)
3. **허브는 사교의 영역:** 다른 플레이어의 장비 구경, 거래, 길드, 파티 모집
4. **비동기 멀티:** 다른 플레이어의 유령(고스트)이 월드에 나타나 힌트를 남기는 시스템 (Dark Souls 메시지)

### 8.3 파티 역할과 시너지

| 역할 | 무기 예시 | 아이템계 기여 |
|:---|:---|:---|
| 탱커 | 대검+방패 | 보스 어그로 관리, 팀 보호 |
| 딜러 | 검, 채찍, 너클 | 빠른 층 클리어 |
| 레인저 | 활, 투척 | 원거리 안전 딜, 함정 해제 |
| 서포터 | 지팡이, 마법서 | 회복, 버프, 이노센트 탐지 |

## 9. 자동 사냥 시스템

### 9.1 3단계 자동 사냥

자동 사냥은 **수동 정복의 보상**으로만 해금된다. 직접 클리어하지 않은 구역은 자동 사냥 불가.

| 단계 | 이름 | 조건 | 효율 | 방식 |
|:---|:---|:---|:---|:---|
| Tier 1 | 사냥터 자동사냥 | 월드 구역 완전 탐험 | 100% (실시간) | 해당 구역에 캐릭터 배치, 실시간 파밍 |
| Tier 2 | 아이템계 자동순찰 | 아이템계 해당 층 클리어 | 60% (실시간) | 클리어한 층까지 자동 반복, 일일 횟수 제한 |
| Tier 3 | 오프라인 원정 | 허브 원정대 시설 해금 | 30% (오프라인) | 로그아웃 중 자원 수집, 시간 비례 보상 |

### 9.2 자동 사냥 설계 원칙

| 원칙 | 설명 |
|:---|:---|
| **정복 선행** | 직접 클리어한 구역/층만 자동 사냥 가능 |
| **효율 차등** | 직접 플레이 > 실시간 자동 > 오프라인 원정 |
| **상한 존재** | 자동 사냥에 일일/주간 상한 설정 |
| **희귀 아이템 제외** | Legendary 이상 등급은 직접 플레이에서만 드랍 |
| **이노센트 불가** | 이노센트는 직접 전투로만 획득 |

## 11. RPG 성장 시스템

### 11.1 캐릭터 스탯

| 스탯 | 약어 | 역할 |
|:---|:---|:---|
| 체력 | HP | 생존 |
| 마력 | MP | 스킬 사용 |
| 공격력 | STR | 물리 데미지 |
| 마력 | INT | 마법 데미지 |
| 방어력 | DEF | 물리 피해 감소 |
| 마방 | RES | 마법 피해 감소 |
| 민첩 | DEX | 명중률, 회피율 |
| 속도 | SPD | 이동속도, 공격속도 |
| 행운 | LCK | 크리티컬, 드랍률, 비밀방 |

### 11.2 성장 공식

```
최종 스탯 = (기본 스탯 + 레벨 보너스) × 장비 배율 + 이노센트 보너스

- 기본 스탯: 캐릭터 기본값
- 레벨 보너스: 레벨당 고정 증가
- 장비 배율: 장비 스탯 × 레어리티 배율
- 이노센트 보너스: 장착된 이노센트 수치 합산
```

### 11.3 데미지 공식

```
데미지 = (ATK × 스킬 배율 - DEF × 0.5) × 원소 배율 × 크리티컬 배율 × 랜덤(0.9~1.1)

- 최소 데미지: 1 (0 이하 방지)
- 크리티컬 배율: 1.5 + (LCK × 0.001)
- 크리티컬 확률: 5% + (LCK × 0.1%) (최대 50%)
```
