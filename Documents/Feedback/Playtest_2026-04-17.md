## 3. 핵심 통찰

### 3.1 스파이크는 성공했고, 스파이크 연출은 실패했다

| 신호 | 증거 |
|------|------|
| 스파이크 컨셉 검증 (긍정) | "무기 세계로 가는게 신박함", "무기와 상호작용 기분 좋다" |
| 스파이크 연출 차단 (부정) | "납치된 기분" 2회 반복, "이해가 안된다" |

**판정:** "아이템 속으로 들어가면 던전이 된다"는 판타지 자체는 플레이어에게 **먹힌다**. 문제는 *들어가는 순간의 연출*이 플레이어 행위성(agency)을 박탈한다는 것.

### 3.2 성장 피드백 루프가 닫혀 있지 않다

반복 플레이 지루 + 도파민 부족 + 강화량 가시성 부족은 **같은 문제의 세 얼굴**이다. 성장이 **존재**하지만 **인지되지 않는다**. 수치·청각·시각·마일스톤 4계층 중 어느 하나도 플레이어에게 명확한 피드백을 제공하지 못하고 있다.

## 4. 근본 원인 분석 (Root Cause)

### 4.1 납치감의 3요소

| 원인 | 설명 |
|------|------|
| 동의 모멘트 부재 | "E 눌러서 들어간다"는 명시적 행위가 없거나 너무 약함 |
| 예고 부재 | 진입 전 "이 무기에 뭐가 있을까?" 빌드업 없음 |
| 문맥 부재 | 왜 아이템 속으로 들어가는가? 대사 0줄 원칙으로 설명 전무 |

### 4.2 성장 체감 부재의 4요소

| 레이어 | 현재 상태 | 결과 |
|--------|---------|------|
| 수치 | 데미지 팝업 없음 / DPS 비교 UI 없음 | "얼마나 강해졌는지 모름" |
| 청각 | 강화 성공 사운드 없음 / 임계점 돌파 사운드 없음 | "도파민 적음" |
| 시각 | 무기 외형이 강화 단계마다 변하지 않음 (레어리티만 변경) | "성장 체감 적음" |
| 마일스톤 | "이 층 보스 잡으면 +X% 강해진다" 사전 고지 없음 | "반복이 지루함" |

### 4.3 온보딩 실패의 공통 원인

제단/아이템 변경/아이템계 진입 모두 **동일한 패턴**:

1. 환경적 교수(Silent Tutorial) 철학을 따랐으나 **affordance(행동 단서)가 약함**
2. 에르다 대사 0줄 + 튜토리얼 UI 없음 → **완전 무언(silent)** 상태
3. 플레이어가 "이게 뭐지?" 하기도 전에 **존재 자체를 놓침**

```
[아이템 선택] → [즉시 텔레포트]
```

**개선 플로우:**
```
[아이템 선택]
  → [아이템 클로즈업 + 금속 결 줌인]
  → [프롬프트: "E — 이 검의 기억 속으로"]
  → [플레이어 입력(E)]
  → [다이브 연출 3초]
  → [층 로딩]

**핵심 원칙:**
- "납치"를 "다이브(의식적 행위)"로 전환

| 대상 | 개선 |

## 9. 튜토리얼 판정 (Tutorial Necessity Analysis)

> 질문: 인벤토리 / 제단 진입 / 무기 교체로 게이트 돌파에 튜토리얼이 필요한가?

### 9.1 결론

**전통적 튜토리얼 불필요. 기존 Silent Tutorial 철학 + 대사 0줄 원칙 유지.**

단, 현재 실패는 "조용히 가르침"이 아니라 **"가르치지 않음"** 상태. 철학은 옳으나 **affordance 구현이 미완**.

### 9.2 현재 상태 재확인 (중요)

| 항목 | 현재 상태 |
|------|---------|
| 인벤토리 키(I) | **상시 우상단 HUD 노출** (조치 불필요) |
| 대부분의 게임 키 | **HUD에 노출됨** |
| 핵심 교수 루프 | **"제단 → 업그레이드 → 게이트 돌파" 구조 이미 존재** |
| 게이트 시스템 | **제단 후 업그레이드를 요구하는 스탯 게이트 이미 구현** |

### 9.3 메커닉별 판정 매트릭스

| 메커닉 | 심각도 | 발견 용이성 | 현재 상태 | 처방 |
|--------|-------|-----------|---------|------|
| 인벤토리 조작 | 중 | 높음 (키 HUD 상시 노출) | 양호 | 첫 획득 시 1회 강조만 추가 |
| 제단 진입 | 치명적 (Spike) | 낮음 (affordance 부족) | 실패 | 맥동·파티클·심볼 프롬프트 |
| 무기 교체 → 게이트 | 치명적 (Core Loop) | 중 (인과 연결 단절) | 실패 | **게이트→제단 시각적 연결** |

### 9.4 잘못된 가설 폐기

> 초기 가설: "제단 전에 추가 게이트를 둬서 무기 교체 교수"

**폐기 사유:** 제단 전 게이트를 세우면 **순환 모순** 발생 — 제단으로 가서 업그레이드하라고 유도해야 하는데 그 앞에 게이트가 있으면 제단에 도달 불가. 현재의 "제단 → 업그레이드 → 게이트" 루프가 **설계상 옳음**.

### 9.5 진짜 문제: 인과 연결 단절 (Causal Disconnect)

```
현재 플레이어 머릿속:
  [제단] 뭔지 모름 → 지나침
  [게이트] 왜 막혔지? → 막힘 → 이탈

플레이어가 봐야 할 머릿속:
  [게이트] ATK 부족 + 망치 아이콘 보임 →
  [제단] 복귀 → 업그레이드 →
  [게이트] 돌파 → "이것이 게임 루프구나" 학습 완료
```

제단은 자기가 뭔지 말하지 않고, 게이트는 제단을 가리키지 않음. **두 지점 사이에 인과의 실이 없음.**

  - 키 아이콘 + 동작 아이콘(검/자물쇠/망치 등)만 허용
  - 첫 조우 시 3초 페이드인, 이후 상시 또는 즉시 표시
  - 텍스트가 아니므로 대사 0줄 원칙 훼손 아님
  - 다국어 로컬라이제이션 비용 0
```

### 10.2 코옵(2인) 호환성 분석

ECHORIS는 Phase 3에서 1~2인 코옵을 지원. 모든 도파민 아이디어는 코옵 환경에서도 작동해야 한다.

**핵심 원칙: "일반 킬 연출은 엔티티 로컬, 보스 킬 연출만 글로벌."**

| # | 아이디어 | 코옵 호환 | 처리 방식 |
|---|---------|----------|----------|
| D1 | 킬 히트스탑 | O | **엔티티 단위 프리즈.** 킬한 플레이어 + 죽은 적만 50ms 정지. 상대 플레이어는 영향 없음. 글로벌 타임스케일 변경 금지. 화면 흔들림은 공유 카메라이므로 양쪽 약하게 적용. |
| D2 | 콤보 카운터 | O+ | **플레이어별 독립 카운터.** 각자 머리 위에 개인 콤보 수 표시. 두 플레이어가 동시에 콤보 유지 시 "Sync Combo" 보너스. 코옵에서 오히려 시너지. |
| D3 | 아이템 레벨업 세레모니 | O | **로컬 연출.** 레벨업한 플레이어에게만 배너 + 빛 기둥 + 짧은 무적(300ms). 상대에게는 해당 위치에 작은 빛 기둥만 표시. 타임스케일 변경 없음. |
| D4 | 사망 파티클 | O | **순수 비주얼.** 월드 좌표에 스폰. 양 플레이어 동일 렌더링. 게임플레이 영향 없음. |

### 10.3 기술 구현 메모

**D1 킬 히트스탑 — 글로벌 vs 로컬 비교:**

| 방식 | 설명 | 코옵 적합성 |
|------|------|-----------|
| 글로벌 타임스케일 | 전체 월드 느려짐 -> 상대방 조작 방해 | **부적합** |
| 엔티티 단위 프리즈 | `entity.timeScale = 0`을 킬러+피격 적에만 적용 | **채택** |

**D2 Sync Combo — 보상 설계:**
- 개인 콤보: EXP 배율 증가 (5킬=x1.1, 10킬=x1.2 ...)
- Sync Combo: 추가 EXP 보너스 (양 플레이어 동시 콤보 유지 시)
- 스파이크 관점: 아이템 EXP 배율이므로 아이템계 경험 직접 강화
