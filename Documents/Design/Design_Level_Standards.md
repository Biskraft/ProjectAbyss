## 1. 개요 (Overview)

### 1-1. 문서 목적

본 문서는 Project Abyss의 **월드(World)** 및 **아이템계(Item World)** 레벨을 설계할 때 준수해야 할 물리적 규격, 배치 원칙, 난이도 설계 가이드를 정의한다. Room Template과 Chunk를 제작하는 모든 작업자(인간/AI)가 이 문서를 기준으로 레벨을 구성한다.

### 1-2. 핵심 원칙

| 원칙 | 설명 |
| :--- | :--- |
| 물리 기반 설계 | 모든 지형은 캐릭터 물리 파라미터(점프 높이, 이동 속도)에서 역산하여 배치한다 |
| Always Winnable | 기본 능력만으로 Critical Path를 반드시 통과할 수 있어야 한다 |
| 능력 보상 탐험 | 능력 게이트 너머에는 반드시 보상(아이템, 숏컷, 비밀)이 존재해야 한다 |
| 읽히는 공간 | 플레이어가 시각적으로 경로와 위험을 판단할 수 있어야 한다 |
| 2-Space 구분 | 월드와 아이템계의 레벨 규격은 분리하여 정의한다 |

### 2-5. 화면 가시 영역 (Screen Visibility)

기준 해상도(480 x 270 px)에서 한 화면에 표시되는 요소의 수량이다. 레벨 배치 시 플레이어가 **한 눈에 인지할 수 있는 범위**를 기준으로 설계한다.

| 항목 | 수치 | 산출 근거 |
| :--- | :--- | :--- |
| 화면 크기 (타일) | 30 x 16.875 타일 | 480 ÷ 16 = 30, 270 ÷ 16 = 16.875 |
| 화면 크기 (픽셀) | 480 x 270 px | 기준 해상도 (1080p의 1/4, ×4 정수배 스케일) |
| 화면 내 Chunk 수 | 6 x 5.625 (약 34개) | 30 ÷ 5 = 6, 16.875 ÷ 3 = 5.625 |
| 화면 대비 Room 비율 | 가로 50%, 세로 약 50% | Room 60x34 중 30x16.875 표시 |
| 화면 내 캐릭터 높이 수 | 약 8.4체 | 16.875 ÷ 2 = 8.4375 |
| 화면 내 캐릭터 폭 수 | 30체 | 30 ÷ 1 = 30 |
| 화면 내 최대 점프 높이 비율 | 약 24% | 기본 점프 4타일 ÷ 화면 높이 16.875타일 |
| 화면 내 수평 점프 거리 비율 | 약 23% | 점프 수평 7타일 ÷ 화면 폭 30타일 |

#### 화면 가시 영역 설계 지침

| 지침 | 설명 |
| :--- | :--- |
| 적 스폰 가시성 | 적은 화면 안(30 x 17 타일)에서만 스폰하거나, 화면 밖 1~2타일 이내에서 진입해야 한다 (기습 금지) |
| 함정 예고 | 함정은 플레이어가 반응할 수 있도록 최소 7타일(화면 폭 약 23%) 전에 시각적 단서를 배치한다 |
| 갈림길 인지 | 분기점은 한 화면 내에서 최소 2개 경로가 보여야 한다 |
| 플랫폼 착지점 | 점프 목적지는 점프 시작점에서 한 화면 내에 보여야 한다 (맹점프 금지) |
| 게이트 티저 | 능력 게이트 너머 보상은 같은 화면 내에서 확인 가능해야 한다 |
| 수직 스크롤 여유 | 상하 5타일 이내의 지형 변화는 카메라 이동 없이 확인 가능하다 |

## 3. 플랫폼 배치 규격 (Platform Placement Standards)

### 3-1. 수직 간격 규칙

캐릭터의 고정 점프 높이(4타일)를 기준으로 모든 수직 간격을 설계한다.

| 구간 | 수직 간격 | 난이도 | 요구 능력 |
| :--- | :--- | :--- | :--- |
| 쉬운 오르막 | 2 타일 | 초급 | 기본 점프 |
| 표준 오르막 | 3 타일 | 중급 | 기본 점프 |
| 도전 오르막 | 3.5 타일 | 상급 | 기본 점프 (여유 0.5타일) |
| 불가 구간 | 5+ 타일 | - | 이단 점프 필요 (능력 게이트) |
| 이단 점프 표준 | 5~6 타일 | 중급 | 이단 점프 |
| 이단 점프 도전 | 6~6.5 타일 | 상급 | 이단 점프 (여유 없음) |

**핵심 규칙:**
* Critical Path의 수직 간격은 **3 타일 이하**로 제한한다 (기본 점프로 100% 도달 가능).
* 3.5 타일 간격은 보상 경로에만 배치한다 (숙련자 보상).
* 4 타일 간격은 배치하지 않는다 (이론상 가능하나 실패 확률이 높아 스트레스 유발).
* 5 타일 이상은 반드시 이단 점프 능력 게이트로 분류한다.

### 3-2. 수평 간격 규칙

점프 중 수평 이동 거리(약 7타일)를 기준으로 설계한다.

| 구간 | 수평 간격 | 난이도 | 비고 |
| :--- | :--- | :--- | :--- |
| 좁은 갭 | 2~3 타일 | 초급 | 달리기 없이 점프로 통과 |
| 표준 갭 | 4~5 타일 | 중급 | 달리기 + 점프 필요 |
| 넓은 갭 | 6 타일 | 상급 | 최대 거리 점프 (여유 1타일) |
| 불가 갭 | 8+ 타일 | - | 능력 게이트 (벽 타기, 안개 변신 등) |

**핵심 규칙:**
* Critical Path의 수평 간격은 **5 타일 이하**로 제한한다.
* 7 타일 간격은 배치하지 않는다 (이론상 한계치로 스트레스 유발).
* 8 타일 이상은 능력 게이트 구간으로만 사용한다.

### 3-3. 천장 높이 규칙

| 구간 | 천장 높이 | 용도 |
| :--- | :--- | :--- |
| 표준 통로 | 4 타일 | 점프 가능, 전투 가능 |
| 넓은 공간 | 6+ 타일 | 보스 방, 전투 아레나 |
| 낮은 통로 | 2 타일 | 서서 통과 가능, 점프 불가 |
| 좁은 틈새 | 1 타일 | 안개 변신 전용 (능력 게이트) |

## 4. 게이트 배치 설계 원칙 (Gate Placement Philosophy)

### 4-1. 능력 게이트 설계

플레이어가 "저기 못 간다" → 능력 획득 → "이제 갈 수 있다!"를 경험하는 구조.

| 능력 | 레벨 디자인 표현 | 시각적 단서 |
| :--- | :--- | :--- |
| 이단 점프 | 5~6 타일 수직 간격 | 높은 곳에 빛나는 아이템/통로 |
| 벽 타기/벽 점프 | 수직 벽면 양쪽에 발판 없음, 벽면에 긁힌 흔적 | 벽면 텍스처 변화 (잡을 수 있는 벽) |
| 안개 변신 | 1타일 높이 격자/좁은 틈새 | 안개가 새어 나오는 틈새 |
| 수중 호흡 | 긴 수중 구간 (산소 게이지 초과) | 물속 깊은 곳의 빛/보물 |
| 역중력 | 천장에 통로/아이템 배치 | 천장의 반전된 구조물 |

**핵심 규칙:**
* 능력 게이트 너머는 반드시 **보상**이 존재해야 한다 (빈 방 금지).
* 게이트 직전에 **티저(Teaser)**를 배치한다: 능력이 없어도 게이트 너머를 **볼 수 있게** 설계한다.
* 한 층위에 동일 능력 게이트는 **최대 3개**까지 배치한다 (과잉 차단 방지).

### 4-2. 스탯 게이트 설계

장비 강화(아이템계)의 동기를 부여하는 게이트.

| 스탯 | 레벨 디자인 표현 | 시각적 단서 |
| :--- | :--- | :--- |
| ATK (물리 장벽) | 파괴 가능한 벽/바리케이드/중문/바위 | 금이 간 벽면, 쇠사슬, 부서질 듯한 바리케이드 |
| INT (마법 봉인) | 룬 장벽/마법 봉인 문/아케인 잠금장치 | 룬 문양, 발광하는 봉인, 마법 진 |

> **설계 변경:** 기존 6대 스탯 게이트(STR/INT/DEX/VIT/SPD/LCK)는 ATK/INT 이중 게이트로 재설계. ATK는 물리 장벽(파괴), INT는 마법 봉인(해제). 기동성/환경 저항/시간 제한 등은 능력 게이트나 환경 퍼즐로 재설계 예정.

**핵심 규칙:**
* 스탯 게이트는 Critical Path에 배치하지 않는다 (선택적 도전).
* 스탯 게이트 너머의 보상 가치는 요구 ATK/INT에 비례한다.
* 스탯 게이트는 수치로 표시한다: "ATK 150 필요" 또��� "INT 80 필요" (모호함 금지).

| 각 지층 | 보스 | 아레나 형태 | 아이템 장군/왕/신/대신 |

## 6. 난이도 곡선 설계 (Difficulty Curve)

### 6-1. 층위별 난이도 매핑

`System_World_ProcGen.md`의 층위 난이도와 일치시킨다.

| 층위 | 난이도 | 전투 강도 | 플랫폼 난이도 | 게이트 밀도 |
| :--- | :---: | :--- | :--- | :--- |
| 중앙 성채 | 1 | 낮음 | 초급 (2타일 간격 중심) | 없음 (튜토리얼) |
| 지하 수로 | 2 | 보통 | 초~중급 + 수중 요소 | 스탯 게이트 소수 |
| 묘지/카타콤 | 2 | 보통 | 중급 + 어둠/시야 제한 | 능력 게이트 1~2개 |
| 연구소 폐허 | 3 | 높음 | 중~상급 + 마법 함정 | 능력+스탯 게이트 혼합 |
| 빙결 보존소 | 3 | 높음 | 중~상급 + 미끄러움 | 능력+스탯 게이트 혼합 |
| 천공의 정원 | 4 | 매우 높음 | 상급 (수직 구조 중심) | 능력 게이트 다수 |
| 심연의 구 | 5 | 극한 | 극상급 (복합 위험) | 스탯 게이트 고수치 |

### 6-2. 난이도 곡선 원칙

1. **점진적 상승**: 층위 내에서 입구 → 보스 방향으로 난이도가 상승한다.
2. **안식처 리듬**: 고난이도 구간 후에는 반드시 안전 구간(세이브 포인트, 빈 통로)을 배치한다.
3. **스파이크 제한**: 한 Room 내에서 전투 + 플랫폼 + 함정이 동시에 최고 난이도인 구간은 보스 방에만 허용한다.
4. **학습 공간**: 새로운 위험 요소는 안전한 환경에서 먼저 노출한다 (독립 소개 → 조합).

## 7. 시각적 설계 규칙 (Visual Design Rules)

### 7-1. 가독성 우선순위

| 우선순위 | 요소 | 규칙 |
| :---: | :--- | :--- |
| 1 | 캐릭터 실루엣 | 배경과 명확히 구분. 어두운 배경 + 선명한 캐릭터 |
| 2 | 발판 경계 | 밟을 수 있는 타일과 배경 타일의 시각적 구분 필수 |
| 3 | 위험 요소 | 함정, 가시, 독 지대는 경고 색상(빨강/보라) 사용 |
| 4 | 상호작용 대상 | 부술 수 있는 벽, 레버 등은 하이라이트 또는 미세 애니메이션 |
| 5 | 비밀 경로 단서 | 의도적으로 미묘하지만 주의 깊은 플레이어가 발견 가능한 수준 |

### 7-2. 바이옴별 시각 테마

| 바이옴 | 주요 색조 | 조명 특성 | 분위기 키워드 |
| :--- | :--- | :--- | :--- |

## 8. Room Template 제작 체크리스트

Room Template 또는 Chunk를 제작할 때 다음 항목을 검증한다.

* [ ] 기본 점프(4타일)만으로 Critical Path를 통과할 수 있는가
* [ ] 수직 간격이 3타일을 초과하는 구간은 Critical Path 밖에 배치했는가
* [ ] 수평 갭이 5타일을 초과하는 구간은 Critical Path 밖에 배치했는가
* [ ] 천장 높이 4타일 미만인 구간에서 전투가 요구되지 않는가 (점프 불가 전투 회피)
* [ ] 능력 게이트 너머에 보상이 배치되어 있는가
* [ ] 게이트 직전에서 게이트 너머를 시각적으로 확인할 수 있는가 (티저)
* [ ] 새로운 위험 요소가 안전 환경에서 먼저 소개되는가
* [ ] 고난이도 구간 후 안식처(세이브/빈 통로)가 존재하는가
* [ ] 밟을 수 있는 발판과 배경이 시각적으로 명확히 구분되는가
* [ ] 함정/위험 요소에 시각적 경고가 존재하는가
* [ ] 비밀 경로에 미묘하지만 발견 가능한 단서가 있는가
* [ ] 화면 표시 영역(30x17 타일) 기준으로 적 스폰이 화면 밖에서 기습하지 않는가

## 검증 기준 (Verification Checklist)

* [ ] 아이템계 지층별 난이도 스케일링이 레어리티에 비례하는가
