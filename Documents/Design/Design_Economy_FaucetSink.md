## 1. 개요 (Overview)

### 1.1. 문서 목적

본 문서는 ECHORIS의 자원 경제가 "왜 이렇게 설계되는가"를 정의하는 상위 철학 문서이다.

"에르다가 수리비를 얼마 받을까?"라는 질문이 이 문서의 출발점이다. 에르다는 대장장이다. 그녀의 세계에서 골드(HL)는 기억을 고치는 비용이고, 기억 단편는 아이템 속에 살아있는 가치이며, 장비의 레어리티는 그 기억이 얼마나 깊이 살아있는가의 척도다. 경제는 단순한 수치 관리가 아니라, 이 세계의 가치 체계를 반영하는 설계다.

수도꼭지(Faucet)는 자원이 게임 세계로 들어오는 모든 경로다. 배수구(Sink)는 자원이 소비되고 사라지는 모든 경로다. 이 두 흐름이 균형을 이룰 때 플레이어는 "적당히 쌓이고, 적당히 쓴다"는 경제적 쾌감을 경험한다. 균형이 깨지면 인플레이션(쌓이기만 하는 경제) 또는 고갈(쓰기만 하는 경제)이 발생하여 성장의 의미가 사라진다.

### 1.2. 한 줄 요약

> "기억은 가치 있다. 그 가치를 순환시키는 것이 경제이고, 순환이 멈추지 않는 한 야리코미는 끝나지 않는다."

### 1.3. 3대 기둥 정렬

| 기둥 | 경제 철학에서의 구현 |
| :--- | :--- |
| 메트로베니아 탐험 | 월드 탐험이 경제의 주요 Faucet이다. 탐험할수록 장비 드랍과 HL이 유입된다. 경제는 탐험 동기를 강화해야 한다 |
| 아이템계 야리코미 | 아이템계가 경제의 핵심 Sink이자 변환 장치다. 야생 기억 단편 복종, 지층 클리어, 레어리티 승급에 자원이 소비된다. 소비할수록 더 강해지는 구조가 야리코미의 지속 동기를 만든다 |
| 온라인 멀티플레이 | 파티 협동이 자원의 분배를 최적화하고, 세이브 포인트의 NPC 상점과 소비 시스템이 유통량을 제어한다 |

### 2.2. 분석 (Analyze)

업계 사례로부터 도출한 경제 설계의 핵심 본질:

| 재미의 본질 | 리스크와 리턴 분석 |
| :--- | :--- |
| 소비 선택권 | 강제 소모가 아닌 더 강해지기 위한 자발적 소비여야 파밍 동기가 살아난다. 플레이어가 "내가 선택해서 썼다"고 느껴야 한다 |
| 공급 예측 가능성 | 세션당 기대 수입이 어느 정도 예측 가능해야 "오늘 이만큼 파밍하면 무엇을 살 수 있다"는 목표 설정이 가능하다 |
| 희소성 유지 | 인플레이션이 쌓이면 모든 자원의 가치가 사라진다. 최상위 자원(Ancient 장비, 고레벨 기억 단편)의 희소성이 장기 야리코미의 동기다 |
| 파밍 대체 방지 | 구매로 모든 것을 살 수 있으면 파밍할 이유가 사라진다. D3 실물 경매장의 실패가 이를 증명했다 |

> **설계 결정: ECHORIS는 플레이어 간 거래소(Auction House / Marketplace)를 도입하지 않는다.**
>
> 근거 세 가지:
>
> 1. **D3 경매장 실패 교훈:** 디아블로 3 런칭 시 실물 경매장(RMAH)과 Gold 경매장을 도입했고, 그 결과 파밍 동기가 완전히 붕괴되었다. 플레이어는 게임 내에서 직접 파밍하는 대신 경매장에서 구매하는 것이 더 효율적임을 인식하고, 게임 플레이 자체가 "경매장 자금 마련"으로 변질되었다. 블리자드는 2013년 경매장을 폐지하며 "게임의 근본적인 재미를 훼손했다"고 공식 인정했다.
> 2. **"내가 키운 가치" 보존:** ECHORIS의 핵심 판타지(장인)는 "내가 직접 파밍하여 세상에 하나뿐인 장비를 만든다"이다. 거래소가 존재하면 장비의 가치가 "내 노력"이 아닌 "시장 가격"으로 정의된다. Legendary+ 장비의 귀속 시스템은 이 판타지를 제도적으로 보호한다.
> 3. **스코프 단순화:** 플레이어 간 거래소는 경매 시스템, 가격 조작 방지, 인플레이션 관리, 봇 대응 등 별도 엔지니어링과 경제 모니터링 부담을 수반한다. 인디/소규모 프로젝트 스코프에서 이 비용은 핵심 게임 플레이 개발에 투자되어야 할 자원을 소모한다.

## 3. 핵심 자원 체계 (Core Resource System)

### 3.1. 자원 목록과 역할 정의

ECHORIS의 경제는 다섯 가지 핵심 자원으로 구성된다. 각 자원은 명확한 역할과 흐름 방향을 가진다. HL 구매력 기준값은 Section 3.2를 참조한다.

| 자원 | 명칭 | 역할 | 주 Faucet | 주 Sink | 인플레이션 위험도 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| HL | 홀로우 링 (Hollow Ring) | 범용 화폐, 기억의 값어치 | 몹 처치, 아이템 판매, 탐험 보상 | 기억 단편 복종, 아이템 수리, 상점 구매, 거래 수수료 | 높음 — 주요 Sink 다수 필요 |
| 장비 아이템 | 드랍 장비 | 아이템계 진입 소재 + 착용 | 월드 몹 드랍, 보스 처치 | 아이템계 진입(소모 없음), 분해(소재 변환) | 낮음 — 착용 슬롯이 자연 상한 |
| 기억 단편 | Memory Shard | 아이템 스탯 보너스 부여자 | 아이템계 직접 전투(야생 격파 후 복종) | 합성(소재로 흡수), 이식(슬롯 이전) | 중간 — 슬롯 수 = 자연 상한 |
| 기억 단편 분해 소재 | Remnant Fragment | 레어리티 승급 재료. **잉여 기억 단편를 분해하여 획득.** 기억 단편 경제와 장비 강화 경제를 직접 연결하는 이중 역할 자원 (PoE 커런시 모델 참조) | 복종 기억 단편 분해 (레어리티·레벨 비례), 아이템계 보스 처치, 지층 클리어 보너스 | 레어리티 승급 소비(단계적 지수 증가) | 중간 — 승급 비용 증가로 자동 억제. 분해를 통한 추가 HL Sink 역할 겸임 |

### 3.2. HL 구매력 기준값 (HL Purchasing Power Anchor)

> 이 절은 세션 수입 수치(Section 8)와 Sink 비용이 플레이어에게 어떤 의미인지 해석하는 기준이다. 모든 HL 금액은 이 기준값에서 파생된다.

| 항목 | HL 비용 | 비고 |
| :--- | ---: | :--- |
| Basic Healing Potion (HP 회복 소모품) | 50 HL | 가장 빈번한 소비 단위. 1 세션 순수입의 약 1/8 |
| Mr. Gency Exit (탈출권, 1회용) | 200 HL | 아이템계 리스크 관리의 핵심 소모품. 세션당 1-2개 소비 예상 |
| Normal 등급 무기 (NPC 상점) | 500 HL | 대략 1회 솔로 세션(순수입 0-400 HL) + α. 초반 파밍의 단기 목표 |

> 설계 원칙: 솔로 Normal 아이템계 세션 기대 순수입(0-400 HL, Section 8.1)을 기준으로 소모품 하나가 세션 수입의 1/8-1/2 범위에 위치하도록 설계한다. Legendary 파티 세션(순수입 500-1500 HL)을 기준으로 하면 소모품 가격이 체감상 더 낮아져 "여유"가 생기는 경제 성장감을 의도한다.

### 3.3. 자원 계층 구조

자원 간에는 변환 경로가 존재한다. 낮은 자원에서 높은 자원으로의 변환에는 항상 비용이 따른다.

```
[기본 자원]          [중간 자원]                [고급 자원]
HL (몹 처치)    →    Remnant Fragment       →    레어리티 승급 재료
                     (잉여 기억 단편 분해,            (Ancient 진입권)
                      아이템계 보스 처치)

야생 기억 단편    →    복종 기억 단편          →    합성 기억 단편
(아이템계 전투)       (×2 레벨 효과)               (레벨 누적)
                          ↓ 분해 시
                     Remnant Fragment

Normal 장비     →    Magic/Rare 장비        →    Legendary/Ancient 장비
(월드 드랍)           (아이템계 강화)               (최심층 클리어)
```

> 설계 원칙 (기억 단편 이중 역할): 복종 기억 단편는 장비 슬롯에 장착하거나, 분해하여 Remnant Fragment로 변환할 수 있다. 이 선택이 플레이어의 핵심 경제 판단이다 — "이 기억 단편를 지금 쓸까, 레어리티 승급 재료로 쓸까?" 기억 단편 경제와 장비 강화 경제가 하나의 자원으로 연결되어 두 시스템 간의 상충 교환(Trade-off)이 발생한다.

## 5. 드랍 경제 설계 (Drop Economy)

### 5.1. 레어리티별 기대 드랍률

업계 벤치마크(`Documents/Research/Equipment_DropRate_Economy_Research.md` 1.6절)를 기반으로 ECHORIS의 드랍률 기준을 설계한다.

드랍은 "공간별 드랍 풀"과 "레어리티 확률"을 분리 롤(2단계)로 처리한다.

```yaml
# 월드 일반 몹 드랍 기준
drop_table_world_normal:
  no_drop:     0.15   # 15% — 드랍 없음
  Normal:      0.55   # 55% — 흰색 장비
  Magic:       0.20   # 20% — 파란색 장비
  Rare:        0.07   # 7%  — 노란색 장비
  Legendary:   0.025  # 2.5% — 주황색 장비
  Ancient:     0.005  # 0.5% — 초록색 (월드 일반몹: 이론상 존재, 매우 희귀)

# 월드 층위 보스 드랍 기준
drop_table_world_boss:
  no_drop:     0.0    # 0% — 보스는 항상 드랍
  Normal:      0.05   # 5%
  Magic:       0.15   # 15%
  Rare:        0.35   # 35%
  Legendary:   0.35   # 35% — 보스의 주요 드랍
  Ancient:     0.10   # 10% — 보스 전용 강화 드랍

# 아이템계 보스 드랍 기준 (아이템 왕 기준)
drop_table_itemworld_boss:
  no_drop:     0.0
  Rare:        0.20
  Legendary:   0.45
  Ancient:     0.25
  unique_memory shard: 0.10  # 보스 전용 특수 기억 단편
```

> 설계 원칙: 레어리티가 높은 공간일수록 높은 레어리티 드랍이 집중된다. 월드 탐험은 파밍의 출발점이고, 아이템계 보스 클리어가 최상위 보상의 집중 경로다.

### 5.2. 배드 럭 보정 시스템 (Pity)

순수 랜덤 드랍만으로는 특정 플레이어가 장시간 Legendary를 얻지 못하는 극단적 케이스가 발생한다. Pity 시스템이 이를 완화한다.

```yaml
pity_system:
  # Legendary Pity — 일반 몹 기준
  legendary_pity:
    trigger_after_n_kills_without_legendary: 200  # 200킬 무드랍 시 발동
    pity_drop_rate_increase_per_kill: 0.005        # 킬마다 +0.5% 추가
    cap_rate: 0.30                                  # 최대 30%에서 상한
    reset_on_legendary_drop: true

  # Ancient Pity — 아이템계 보스 기준
  ancient_pity:
    trigger_after_n_boss_clears_without_ancient: 10
    guaranteed_ancient_at_n: 20  # 20회 보스 클리어 시 Ancient 보장
    reset_on_ancient_drop: true

  # 기억의 방랑자 Pity — 일정 시간 미출현 시 강제 출현
  memory_wanderer_pity:
    base_interval_minutes: 35
    max_wait_minutes: 60  # 60분 초과 시 강제 출현
```

> 설계 원칙: Pity는 "보장"이 아니라 "기대값의 하한선"이다. Legendary 이상은 보장하지 않고 확률 상한만 제공하여 희소성을 유지한다. 단, Ancient는 아이템계 보스 클리어에서 20회 무드랍 시 1회 보장으로 최소한의 확실성을 제공한다.

### 5.3. 보스 전용 드랍 풀 (Dedicated Drop)

Borderlands 3의 "Dedicated Drop" 방식을 차용한다. 각 아이템계 보스는 1-2개의 전용 기억 단편 타입을 높은 확률로 드랍한다.

| 보스 등급 | 전용 드랍 | 전용 확률 | 일반 풀 포함 여부 |
| :--- | :--- | :--- | :--- |
| 아이템 장군 (Item General) | 특정 전투 계열 기억 단편 | 25% | 예 |
| 아이템 왕 (Item King) | 특정 방어/회복 계열 기억 단편 | 30% | 예 |
| 아이템 신 (Item God) | 특정 속성/원소 계열 기억 단편 | 35% | 예 |
| 아이템 대신 (Item Overlord) | 유니크 타입 기억 단편 전용 | 40% | 아니오 (전용 풀만) |

목표 지향 파밍을 가능하게 하는 정보 공개 원칙: 아이템계 진입 전, 해당 장비의 지층 보스 드랍 정보를 아이템 정보 UI에서 확인할 수 있다.

## 6. 거래 제한 설계 (Trade Restriction)

### 6.0. 거래소 없음 — 의도적 설계 결정 (No Marketplace — Intentional Design Decision)

ECHORIS는 **플레이어 간 거래소(Marketplace / Auction House)를 설계에서 영구 제외**한다. 이것은 기능 부재가 아니라 명시적 설계 선택이다.

| 항목 | 내용 |
| :--- | :--- |
| 결정 | 플레이어 간 자유 거래소 없음 |
| 허용되는 거래 | Normal/Magic/Rare 장비의 직접 교환(아이템계 파티 드랍 공유 한정), NPC 상점 매각 |
| 금지되는 거래 | Legendary/Ancient 장비의 모든 플레이어 간 거래, Lv 100+ 기억 단편 거래, 유니크 기억 단편 거래 |
| 1차 근거 | D3 실물 경매장 실패 — 거래소가 게임을 쇼핑몰로 변질시킴 |
| 2차 근거 | 핵심 판타지(장인) 보호 — "내가 직접 파밍한 장비"의 가치가 시장 가격으로 환원되면 안 됨 |
| 3차 근거 | 스코프 단순화 — 거래소 운영·관리 비용(봇, 조작, 인플레이션)을 핵심 개발에 재투자 |

이 결정은 2.2절 "파밍 대체 방지" 원칙의 제도적 구현이며, Project_Vision_Abyss의 핵심 금지 규칙 "Pay-to-Win 과금 금지"와 동일한 철학 위에 놓인다.

### 6.1. 설계 원칙: 파밍 동기 보존

디아블로 3 실물 경매장 폐지 사례가 증명한다. "돈으로 최강 장비를 살 수 있으면 파밍할 이유가 없어진다." 거래 제한은 규제가 아니라 파밍 동기를 보호하는 안전 장치다.

| 등급 | 거래 가능 여부 | 제한 방식 | 설계 의도 |
| :--- | :--- | :--- | :--- |
| Normal | 완전 거래 가능 | 제한 없음 | 초반 경제 윤활. 낮은 가치 |
| Magic | 완전 거래 가능 | 제한 없음 | 중반 진입 장벽 완화 |
| Rare | 1회 거래 가능 | 거래 후 계정 귀속 | 제한적 거래 허용하되, 무한 유통 방지 |
| Legendary | 귀속 조건부 거래 | 드랍 시 미착용이면 1회 거래 가능. 착용 또는 아이템계 진입 후 계정 귀속 | 파밍 동기의 핵심 보호 구간 |
| Ancient | 계정 귀속 | 드랍 즉시 귀속. 거래 불가 | 최강 장비는 반드시 직접 파밍 — 핵심 판타지(장인) 보호 |

> 설계 원칙: Legendary 이상은 직접 파밍으로만 완전한 소유권이 생긴다. 아이템에 에코를 두드려 아이템계에 진입한 순간, 그 아이템은 에르다의 기억과 섞인다 — 이것이 귀속의 서사적 근거이다.

### 6.2. 기억 단편 거래 제한

기억 단편는 장비보다 더 엄격한 제한이 필요하다. 기억 단편가 자유롭게 거래되면 "아이템계에 직접 들어가 기억 단편를 잡는" 핵심 야리코미 행위의 동기가 훼손된다.

| 기억 단편 유형 | 거래 가능 여부 | 이유 |
| :--- | :--- | :--- |
| 야생 기억 단편 (미복종) | 거래 불가 | 야생 기억 단편는 아이템계 내부에서만 존재. 외부 추출 불가 |
| 복종 기억 단편 Lv 1-99 | 거래 가능 (NPC 상점 매각) | 신규 플레이어 진입 지원. 낮은 레벨은 파밍 동기에 영향 미미 |
| 복종 기억 단편 Lv 100+ | 거래 불가 | 고레벨 기억 단편는 장기 야리코미의 결과물. 거래 허용 시 직접 파밍 동기 소멸 |
| 유니크 타입 기억 단편 | 거래 불가 | 아이템 대신 전용 드랍. 직접 도전의 증표 |

## 7. 세션 경제 (Session Economy)

### 7.1. 10-15분 아이템계 세션 기대값

플레이어가 아이템계에서 10-15분을 보낼 때 얻을 수 있는 기대 수입을 정의한다. 이 수치는 세션 설계와 밸런스 기준값으로 사용된다.

```yaml
session_economy_itemworld_10min:
  # 기준: Normal 등급 아이템계, 솔로, 2 지층 완주 가정
  expected_HL_income:
    from_mob_kills: 300-500   # 지층당 몹 약 15-20마리 처치
    from_memory shard_taming: -100 to -300  # 기억 단편 복종 비용 (Sink)
    net_HL_gain: 0-400        # 순수입 (복종 비용 차감)

  expected_memory shard_gain:
    wild_memory shard_encounters: 1-3       # 지층당 야생 기억 단편 1-3마리
    taming_success_rate: 1.0            # 복종은 항상 성공 (비용만 소모)
    net_memory shard_gain_per_session: 2-6  # 세션당 복종 기억 단편 2-6개

  expected_salvage_gain:
    stratum_clear_memory shard_salvage: 1-2  # 지층 클리어당 Remnant Fragment
    boss_memory shard_salvage: 2-4           # 보스 처치 시 추가 Remnant Fragment
    # 참고: 잉여 기억 단편 분해 시 Remnant Fragment 추가 획득 (세션당 변동)

  # 기준: Legendary 등급 아이템계, 2인 파티 (Phase 3), 4 지층 완주 가정 (25-35분)
  expected_HL_income_legendary_party:
    from_mob_kills: 1500-2500
    from_memory shard_taming: -800 to -2000
    net_HL_gain: 500-1500
    party_bonus_multiplier: 1.2         # 파티 플레이 HL 보너스 +20%

  expected_memory shard_gain_legendary:
    wild_memory shard_encounters_per_stratum: 2-4
    total_memory shards_4_strata: 8-16
    party_sharing: "파티원별 독립 드랍 (중복 획득 가능)"
```

### 7.2. 세션 자연 종료 신호

플레이어가 아이템계에서 자연스럽게 탈출을 결정하도록 유도하는 경제적 신호를 설계한다.

| 신호 | 발생 조건 | 플레이어 판단 유도 |
| :--- | :--- | :--- |
| 탈출권 소진 임박 | 탈출권 1개 남음 | "지금 나가면 안전. 더 가면 사망 리스크" |
| HP 소모 임계점 | HP 30% 이하 | "여기서 실패하면 지층 손실" |
| 기억 단편 슬롯 포화 | 인벤토리 기억 단편 슬롯 90% | "더 잡아도 합성 전에 보관 공간 없음" |
| 최심층 지층 완료 | 마지막 지층 클리어 | "자연스러운 사이클 종료" |

> 설계 원칙: 강제 탈출 없이 플레이어 스스로 "지금이 나갈 때"라고 느끼게 하는 경제적 신호를 설계한다. 야리코미의 "한 지층만 더"와 "지금 나가야 해"의 긴장이 세션의 핵심 재미이다.

## 8. 인플레이션 장기 억제 메커니즘 (Long-Term Inflation Control)

장기 운영에서 HL 인플레이션이 심화되지 않도록 다층 억제 장치를 설계한다.

| 억제 장치 | 작동 방식 | 효과 |
| :--- | :--- | :--- |
| 기억 단편 복종 비용 레벨 비례 | 레벨 높을수록 복종 비용 지수 증가 | 야리코미 유저의 HL 소비 자동 확대 |
| 레어리티 승급 비용 (Remnant Fragment 소비) | Remnant Fragment 지수 증가 소비. NPC 상점과 장비 강화가 유일한 HL Sink 경로임을 보완하는 별도 Sink 레이어 | 강력한 단발성 Sink. 기억 단편 분해 수요가 자발적 HL 소비로 이어지는 이중 구조 |
| 기억 단편 분해 수수료 누진 | 고레벨 기억 단편 분해 시 HL 수수료 증가 | 고인물 야리코미 유저의 HL 소비 채널 확보 |
| NPC 상점 + 강화 비용 (유일한 HL Sink) | NPC 상점 구매(소모품, 탈출권)와 기억 단편 복종/이식/합성 비용이 HL의 주요 배수구. **거래소 없음 설계로 플레이어 간 HL 유통 없음 — Sink가 NPC 경로에만 집중됨을 전제로 Sink 규모를 보정한다** | 거래소 부재 시 HL 소비 채널이 좁아지므로, Sink 단가(복종 비용, 이식 비용)를 충분히 높게 설정하여 인플레이션 억제

## 9. 내러티브 경제 통합 (Narrative Economy Integration)

### 9.1. 에르다의 대장간과 경제

에르다는 대장장이다. 그녀의 경제관은 "수리비를 얼마 받을까"라는 실용주의에서 출발한다.

| 경제 요소 | 서사적 프레이밍 | 플레이어 체감 |
| :--- | :--- | :--- |
| HL (홀로우 링) | 기억의 화폐. 고딕 세계에서 잊힌 기억의 가치가 통화가 되었다 | "이 전투에서 얼마짜리 기억을 얻었나" |
| 기억 단편 복종 비용 | 야생 기억 단편를 달래는 비용. 에르다의 망치 에코로 두드려 "이 아이템에 머물러도 좋다"고 설득하는 과정 | 대장장이가 재료를 다루는 느낌 |
| 레어리티 승급 비용 | 아이템의 기억을 더 깊이 깨우는 의식 비용. 얕은 기억을 깊은 기억으로 변환하는 에르다의 장인 작업 | 장인이 명품을 만드는 과정 |

### 9.2. 기억의 방랑자와 경제적 기회비용

기억의 방랑자(Memory Wanderer)는 경제적 의미에서 "제한 시간 드랍률 부스터"이다. 하지만 내러티브 맥락에서는 아이템계에서 돌아오지 못한 자들의 잔상이다. 이 두 의미가 겹치는 지점이 중요하다.

방랑자 아이템계는 일반 아이템계 대비 +50% 드랍량을 제공한다. 이 추가 보상은 서사적으로 정당화된다. "이 방랑자는 자신이 갇혀있는 아이템의 기억을 더 잘 안다. 그래서 그 내부의 보상이 더 풍부하게 드러난다."

기회비용 구조:
- 방랑자를 발견하면 지금 하던 월드 탐험을 멈춰야 한다
- 30-45분에 1회 출현이므로, 무시하면 다음 기회는 30-45분 후
- 방랑자 아이템계 진입은 현재 인벤토리 장비 1개를 소비 (그 장비의 아이템계로 진입)

## 저주받은 문제 (Cursed Problems)

경제 설계에는 근본적으로 해결 불가능한 상충 관계가 존재한다. 이를 숨기지 않고 설계 결정으로 명시한다.

| 상충 관계 | 어느 쪽을 선택했는가 | 이유 |
| :--- | :--- | :--- |
| 야리코미 무한 파밍 vs 경제 인플레이션 방지 | 무한 파밍을 허용하되 Sink를 지수적으로 확장 — 강해질수록 더 많이 소비 | 에너지/피로도 제한은 핵심 금지 규칙. Sink 확장으로 수요 측에서 인플레이션 억제 |
| 거래 자유 vs 파밍 동기 보존 | Legendary 이상 귀속. 낮은 등급은 자유 거래 | D3 실물 경매장 실패 교훈. 최강 장비는 직접 파밍이 핵심 판타지 |
| 신규 유저 접근성 vs 고인물 성취감 보존 | Pity 시스템 + 낮은 등급 자유 거래로 신규 진입 지원. 상위 등급 귀속으로 고인물 희소성 유지 | 두 집단을 다른 경제 레이어로 분리 |
