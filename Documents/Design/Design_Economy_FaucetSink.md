## 1. 개요 (Overview)

### 1.1. 문서 목적

본 문서는 Project Abyss의 자원 경제가 "왜 이렇게 설계되는가"를 정의하는 상위 철학 문서이다.

"에르다가 수리비를 얼마 받을까?"라는 질문이 이 문서의 출발점이다. 에르다는 대장장이다. 그녀의 세계에서 골드(HL)는 기억을 고치는 비용이고, 이노센트는 아이템 속에 살아있는 가치이며, 장비의 레어리티는 그 기억이 얼마나 깊이 살아있는가의 척도다. 경제는 단순한 수치 관리가 아니라, 이 세계의 가치 체계를 반영하는 설계다.

수도꼭지(Faucet)는 자원이 게임 세계로 들어오는 모든 경로다. 배수구(Sink)는 자원이 소비되고 사라지는 모든 경로다. 이 두 흐름이 균형을 이룰 때 플레이어는 "적당히 쌓이고, 적당히 쓴다"는 경제적 쾌감을 경험한다. 균형이 깨지면 인플레이션(쌓이기만 하는 경제) 또는 고갈(쓰기만 하는 경제)이 발생하여 성장의 의미가 사라진다.

### 1.2. 한 줄 요약

> "기억은 가치 있다. 그 가치를 순환시키는 것이 경제이고, 순환이 멈추지 않는 한 야리코미는 끝나지 않는다."

### 1.3. 3대 기둥 정렬

| 기둥 | 경제 철학에서의 구현 |
| :--- | :--- |
| 메트로베니아 탐험 | 월드 탐험이 경제의 주요 Faucet이다. 탐험할수록 장비 드랍과 HL이 유입된다. 경제는 탐험 동기를 강화해야 하며, 자동 사냥이 탐험의 대체재가 되어서는 안 된다 |
| 아이템계 야리코미 | 아이템계가 경제의 핵심 Sink이자 변환 장치다. 야생 이노센트 복종, 지층 클리어, 레어리티 승급에 자원이 소비된다. 소비할수록 더 강해지는 구조가 야리코미의 지속 동기를 만든다 |
| 온라인 멀티플레이 | 허브 거래소가 경제의 균형 교환소다. 플레이어 간 거래가 자원의 분배를 최적화하고, 수수료 Sink가 유통량을 제어한다. 단, 거래가 파밍을 대체해서는 안 된다 |

### 2.2. 분석 (Analyze)

업계 사례로부터 도출한 경제 설계의 핵심 본질:

| 재미의 본질 | 리스크와 리턴 분석 |
| :--- | :--- |
| 소비 선택권 | 강제 소모가 아닌 더 강해지기 위한 자발적 소비여야 파밍 동기가 살아난다. 플레이어가 "내가 선택해서 썼다"고 느껴야 한다 |
| 공급 예측 가능성 | 세션당 기대 수입이 어느 정도 예측 가능해야 "오늘 이만큼 파밍하면 무엇을 살 수 있다"는 목표 설정이 가능하다 |
| 희소성 유지 | 인플레이션이 쌓이면 모든 자원의 가치가 사라진다. 최상위 자원(Ancient 장비, 고레벨 이노센트)의 희소성이 장기 야리코미의 동기다 |
| 파밍 대체 방지 | 거래소로 모든 것을 살 수 있으면 파밍할 이유가 사라진다. D3 실물 경매장의 실패가 이를 증명했다 |

### 2.3. 재구축 (Rebuild)

레퍼런스 게임의 경제 원리를 Project Abyss의 3-Space + 내러티브 구조에 맞게 재구축한다.

| 원형 | Project Abyss의 독창적 변형 |
| :--- | :--- |
| PoE Currency 소모 크래프팅 | 이노센트 복종 비용(HL 소모)이 크래프팅 Sink 역할. 단, 실패 확률 없음 — 복종은 반드시 성공한다. 리스크는 탈출 실패(진행 손실)에서 온다 |
| D4 Masterworking 지수 소비 | 레어리티 승급(Normal → Magic → ... → Ancient) 비용의 단계적 증가. 높은 단계일수록 요구 이노센트 수와 HL이 지수적으로 증가한다 |
| 워프레임 팜 자동 누적 | 허브의 이노센트 팜이 오프라인 자동 성장을 담당. 단, 팜은 낮은 등급 이노센트만 생산 — 고등급은 아이템계 직접 전투로만 획득 |
| 디스가이아 HL 순환 | 에르다의 대장간 수리비 프레임. "기억을 수리하는 비용"으로 서사 통합. HL은 기억의 화폐이다 |
| MH 투명한 드랍 테이블 | 아이템계 보스별 전용 드랍 풀을 플레이어에게 공개. "이 지층 보스를 잡으면 이 이노센트가 나온다"는 목표 지향 파밍 구조 |

### 2.4. 3대 기둥 정렬 확인

| 기둥 검증 질문 | 경제 설계의 답변 |
| :--- | :--- |

## 3. 핵심 자원 체계 (Core Resource System)

### 3.1. 자원 목록과 역할 정의

Project Abyss의 경제는 다섯 가지 핵심 자원으로 구성된다. 각 자원은 명확한 역할과 흐름 방향을 가진다. HL 구매력 기준값은 Section 3.2를 참조한다.

| 자원 | 명칭 | 역할 | 주 Faucet | 주 Sink | 인플레이션 위험도 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| HL | 홀로우 링 (Hollow Ring) | 범용 화폐, 기억의 값어치 | 몹 처치, 아이템 판매, 탐험 보상 | 이노센트 복종, 아이템 수리, 상점 구매, 거래 수수료 | 높음 — 주요 Sink 다수 필요 |
| 장비 아이템 | 드랍 장비 | 아이템계 진입 소재 + 착용 | 월드 몹 드랍, 보스 처치, 거래소 | 아이템계 진입(소모 없음), 분해(소재 변환) | 낮음 — 착용 슬롯이 자연 상한 |
| 이노센트 | Innocent | 아이템 스탯 보너스 부여자 | 아이템계 직접 전투(야생 격파 후 복종) | 합성(소재로 흡수), 이식(슬롯 이전) | 중간 — 슬롯 수 = 자연 상한 |
| 강화 소재 | Memory Fragments | 레어리티 승급, 특수 강화 재료 | 아이템계 보스 처치, 지층 클리어 보너스, 기억의 방랑자 보상 | 레어리티 승급 소비(지수 증가), 이노센트 이식 비용 | 중간 — 승급 비용 증가로 자동 억제 |
| 스킬 포인트 | Skill Points | 스킬 트리 성장 | 캐릭터 레벨업, 특정 보스 처치 | 스킬 트리 투자(비가역), 리스펙 비용(HL 소모) | 낮음 — 레벨 상한이 자연 제한 |

### 3.2. HL 구매력 기준값 (HL Purchasing Power Anchor)

> 이 절은 세션 수입 수치(Section 8)와 Sink 비용이 플레이어에게 어떤 의미인지 해석하는 기준이다. 모든 HL 금액은 이 기준값에서 파생된다.

| 항목 | HL 비용 | 비고 |
| :--- | ---: | :--- |
| Basic Healing Potion (HP 회복 소모품) | 50 HL | 가장 빈번한 소비 단위. 1 세션 순수입의 약 1/8 |
| Mr. Gency Exit (탈출권, 1회용) | 200 HL | 아이템계 리스크 관리의 핵심 소모품. 세션당 1-2개 소비 예상 |
| Normal 등급 무기 (NPC 상점) | 500 HL | 대략 1회 솔로 세션(순수입 0-400 HL) + α. 초반 파밍의 단기 목표 |
| Respec 기본 비용 (1회차) | 1,000 HL | 약 2-3회 풀 세션의 순수입. 빌드 교체의 경제적 마찰 |
| Respec 상한 비용 (4회차+) | 4,000 HL | 세션 약 10-15회 분량. 남용 억제 상한 |

> 설계 원칙: 솔로 Normal 아이템계 세션 기대 순수입(0-400 HL, Section 8.1)을 기준으로 소모품 하나가 세션 수입의 1/8~1/2 범위에 위치하도록 설계한다. Legendary 파티 세션(순수입 500-1500 HL)을 기준으로 하면 소모품 가격이 체감상 더 낮아져 "여유"가 생기는 경제 성장감을 의도한다.

### 3.3. 자원 계층 구조

자원 간에는 변환 경로가 존재한다. 낮은 자원에서 높은 자원으로의 변환에는 항상 비용이 따른다.

```
[기본 자원]          [중간 자원]           [고급 자원]
HL (몹 처치)    →    강화 소재         →    레어리티 승급 재료
                     (아이템계 보스)         (Ancient 진입권)

야생 이노센트    →    복종 이노센트     →    합성 이노센트
(아이템계 전투)       (×2 레벨 효과)         (레벨 누적)

Normal 장비     →    Magic/Rare 장비   →    Legendary/Ancient 장비
(월드 드랍)           (아이템계 강화)         (최심층 클리어)
```

## 4. 공간별 Faucet/Sink 매핑 (3-Space Economy Map)

### 4.1. 월드 (World) 경제

월드는 경제의 주요 Faucet 공간이다. 탐험할수록 자원이 유입된다.

| 유입 경로 (Faucet) | 유입 자원 | 유입량 특성 | 설계 의도 |
| :--- | :--- | :--- | :--- |
| 일반 몹 처치 | HL, 장비 아이템(Normal-Rare) | 소량, 빈번 | 매 전투의 즉각 보상. 탐험의 기본 수익 |
| 중간 네임드 몬스터 처치 | HL, 장비 아이템(Rare-Legendary), 강화 소재 소량 | 중간량, 중간 빈도 | 선택적 전투의 추가 보상 |
| 구역 보스 처치 | HL 다량, 장비(Rare-Legendary 중점), 능력 렐릭, 강화 소재 | 대량, 저빈도(주 1회) | 탐험의 클라이맥스 보상. 기억의 방랑자 유발 가능 |
| 비밀 방/숨겨진 구역 발견 | HL, 장비 아이템(레어리티 +1 보정), 로어 조각 | 중간량, 극저빈도 | LCK 스탯의 경제적 가치. 탐험 동기 강화 |
| 필드 균열 제단 발견 | 특수 없음(진입 지점 해금) | N/A | 아이템계 접근성 향상 = 간접 경제 이득 |
| 기억의 방랑자 접촉 | 아이템계 진입 (보상 강화 변형) | 특수, 30-45분마다 1회 | 월드 탐험 중 아이템계로 자연 연결 |
| 캐릭터 레벨업 | SP (Skill Points) 1개/레벨 | 레벨당 1 SP, 총 Lv1-10 기준 9 SP | 스킬 트리 투자의 원천. 주 SP Faucet |
| 보스 처치 / 비밀방 / NPC 교환 | SP 보너스 (각 경로별 1-2 SP) | Phase 2 구현, 저빈도 | SP 총량 약간 확장. 탐험 동기 추가 강화 |

| 소비 경로 (Sink) | 소비 자원 | 소비량 특성 | 설계 의도 |
| :--- | :--- | :--- | :--- |
| 탐험 중 사망 (세이브 포인트 복귀) | 없음 (경제적 페널티 없음) | 없음 | 탐험 리스크는 시간/진행 손실. HL 페널티는 불필요 마찰 |
| NPC 상점 구매 (허브 경유) | HL | 선택적 소비 | 4.3절 허브 Sink 참조 |

> 설계 원칙: 월드의 사망 페널티는 HL이 아닌 세이브 포인트 복귀이다. 탐험 실패에 경제적 처벌을 추가하면 위험 회피(Risk Aversion) 행동이 강화되어 탐험 동기가 위축된다.

### 4.2. 아이템계 (Item World) 경제

아이템계는 경제의 핵심 Sink이자 변환 장치다. 자원이 유입되기도 하지만, 더 중요하게는 낮은 가치 자원이 높은 가치 자원으로 변환되는 공간이다.

| 유입 경로 (Faucet) | 유입 자원 | 유입량 특성 | 설계 의도 |
| :--- | :--- | :--- | :--- |
| 야생 이노센트 격파 | 야생 이노센트 (복종 대기 상태) | 지층당 1-3개 | 아이템계 진입의 핵심 보상. 복종 후 4배 실효 점프 |
| 지층 클리어 보상 | 강화 소재, HL 소량, 드랍 장비 | 지층당 고정 보상 | 진행 보상. 보스 지층은 추가 보상 |
| 아이템 보스 처치 | 강화 소재, 고레어리티 이노센트, 이노센트 슬롯 +1 (아이템 왕 이상) | 보스 등급별 차등 | 보스 도전의 명확한 리턴 |
| 아이템 보스 처치 (Phase 2) | SP 보너스 드랍 (희귀, 보스 등급별) | 극저빈도 — 아이템 신 이상에서 드랍 | 아이템계 깊이 도전의 추가 리턴. 탐험-전투-스킬 성장의 연결 |
| 미스터리 룸 | HL 다량, 희귀 이노센트, 레벨 구슬 | 저빈도, 고보상 | 탐색 보상. 플레이어 지식이 경제적 우위로 전환 |
| 기억의 방랑자 아이템계 (보상 강화) | 일반 아이템계 대비 +50% 드랍량 | 방랑자 접촉 시에만 | 월드 탐험과 아이템계 보상을 연결하는 접점 |
| 재귀적 진입 보상 | 상위 아이템계 드랍 적용 (재귀 레벨 비례 보정) | 재귀 깊이에 비례 | 재귀 진입의 리스크에 상응하는 리턴 |

| 소비 경로 (Sink) | 소비 자원 | 소비량 특성 | 설계 의도 |
| :--- | :--- | :--- | :--- |
| 이노센트 복종 비용 | HL (`Innocent_Taming_Cost` 참조) | 이노센트 레벨 비례 | 아이템계의 핵심 HL Sink. 복종할수록 HL 소모 |
| 탈출 아이템 소비 | Mr. Gency Exit (소모품, 200 HL 구매) | 긴급 탈출 시 소비 | 탈출 vs 진행의 리스크-리턴 판단. 일회용 |
| 레어리티 승급 트리거 | 강화 소재 (`Rarity_Upgrade_Cost` 참조) | 단계적 지수 증가 | 최심층 클리어의 보상이자 다음 목표로의 연결 |

> **진행 리스크 (Progress Risk) — 자원 소비 아님:** 아이템계에서 사망 시 현재 지층 진행이 손실된다. 이것은 자원(HL, 소재)이 소비되는 것이 아니라 시간·기회비용이 손실되는 것이다. 따라서 Sink 테이블이 아닌 별도 리스크 항목으로 관리한다. 설계 원칙: HL 페널티보다 진행 손실이 더 강렬하고 덜 처벌적인 리스크 메커니즘이다. HL을 잃으면 "돈을 잃었다"는 느낌이고, 진행을 잃으면 "도전이 실패했다"는 느낌이다 — 야리코미의 재도전 동기는 후자에서 나온다.

## 5. 드랍 경제 설계 (Drop Economy)

### 5.1. 레어리티별 기대 드랍률

업계 벤치마크(`Documents/Research/Equipment_DropRate_Economy_Research.md` 1.6절)를 기반으로 Project Abyss의 드랍률 기준을 설계한다.

드랍은 "공간별 드랍 풀"과 "레어리티 확률"을 분리 롤(2단계)로 처리한다.

```yaml
# 월드 일반 몹 드랍 기준
drop_table_world_normal:
  no_drop:     0.15   # 15% — 드랍 없음
  Normal:      0.55   # 55% — 흰색 장비
  Magic:       0.20   # 20% — 파란색 장비
  Rare:        0.07   # 7%  — 노란색 장비
  Legendary:   0.025  # 2.5% — 주황색 장비
  Ancient:     0.005  # 0.5% — 초록색 (월드 일반몹: 이론상 존재, 매우 희귀)

# 월드 구역 보스 드랍 기준
drop_table_world_boss:
  no_drop:     0.0    # 0% — 보스는 항상 드랍
  Normal:      0.05   # 5%
  Magic:       0.15   # 15%
  Rare:        0.35   # 35%
  Legendary:   0.35   # 35% — 보스의 주요 드랍
  Ancient:     0.10   # 10% — 보스 전용 강화 드랍

# 아이템계 보스 드랍 기준 (아이템 왕 기준)
drop_table_itemworld_boss:
  no_drop:     0.0
  Rare:        0.20
  Legendary:   0.45
  Ancient:     0.25
  unique_innocent: 0.10  # 보스 전용 특수 이노센트
```

> 설계 원칙: 레어리티가 높은 공간일수록 높은 레어리티 드랍이 집중된다. 월드 탐험은 파밍의 출발점이고, 아이템계 보스 클리어가 최상위 보상의 집중 경로다.

### 5.2. 배드 럭 보정 시스템 (Pity)

순수 랜덤 드랍만으로는 특정 플레이어가 장시간 Legendary를 얻지 못하는 극단적 케이스가 발생한다. Pity 시스템이 이를 완화한다.

```yaml
pity_system:
  # Legendary Pity — 일반 몹 기준
  legendary_pity:
    trigger_after_n_kills_without_legendary: 200  # 200킬 무드랍 시 발동
    pity_drop_rate_increase_per_kill: 0.005        # 킬마다 +0.5% 추가
    cap_rate: 0.30                                  # 최대 30%에서 상한
    reset_on_legendary_drop: true

  # Ancient Pity — 아이템계 보스 기준
  ancient_pity:
    trigger_after_n_boss_clears_without_ancient: 10
    guaranteed_ancient_at_n: 20  # 20회 보스 클리어 시 Ancient 보장
    reset_on_ancient_drop: true

  # 기억의 방랑자 Pity — 일정 시간 미출현 시 강제 출현
  memory_wanderer_pity:
    base_interval_minutes: 35
    max_wait_minutes: 60  # 60분 초과 시 강제 출현
```

> 설계 원칙: Pity는 "보장"이 아니라 "기대값의 하한선"이다. Legendary 이상은 보장하지 않고 확률 상한만 제공하여 희소성을 유지한다. 단, Ancient는 아이템계 보스 클리어에서 20회 무드랍 시 1회 보장으로 최소한의 확실성을 제공한다.

### 5.3. 보스 전용 드랍 풀 (Dedicated Drop)

Borderlands 3의 "Dedicated Drop" 방식을 차용한다. 각 아이템계 보스는 1-2개의 전용 이노센트 타입을 높은 확률로 드랍한다.

| 보스 등급 | 전용 드랍 | 전용 확률 | 일반 풀 포함 여부 |
| :--- | :--- | :--- | :--- |
| 아이템 장군 (Item General) | 특정 전투 계열 이노센트 | 25% | 예 |
| 아이템 왕 (Item King) | 특정 방어/회복 계열 이노센트 | 30% | 예 |
| 아이템 신 (Item God) | 특정 속성/원소 계열 이노센트 | 35% | 예 |
| 아이템 대신 (Item Overlord) | 유니크 타입 이노센트 전용 | 40% | 아니오 (전용 풀만) |

목표 지향 파밍을 가능하게 하는 정보 공개 원칙: 아이템계 진입 전, 해당 장비의 지층 보스 드랍 정보를 허브 도서관 NPC 또는 아이템 정보 UI에서 확인할 수 있다.

## 6. 거래 제한 설계 (Trade Restriction)

### 6.1. 설계 원칙: 파밍 동기 보존

디아블로 3 실물 경매장 폐지 사례가 증명한다. "돈으로 최강 장비를 살 수 있으면 파밍할 이유가 없어진다." 거래 제한은 규제가 아니라 파밍 동기를 보호하는 안전 장치다.

| 등급 | 거래 가능 여부 | 제한 방식 | 설계 의도 |
| :--- | :--- | :--- | :--- |
| Normal | 완전 거래 가능 | 제한 없음 | 초반 경제 윤활. 낮은 가치 |
| Magic | 완전 거래 가능 | 제한 없음 | 중반 진입 장벽 완화 |
| Rare | 1회 거래 가능 | 거래 후 계정 귀속 | 거래소 참여 허용하되, 무한 유통 방지 |
| Legendary | 귀속 조건부 거래 | 드랍 시 미착용이면 1회 거래 가능. 착용 또는 아이템계 진입 후 계정 귀속 | 파밍 동기의 핵심 보호 구간 |
| Ancient | 계정 귀속 | 드랍 즉시 귀속. 거래 불가 | 최강 장비는 반드시 직접 파밍 — 핵심 판타지(장인) 보호 |

> 설계 원칙: Legendary 이상은 직접 파밍으로만 완전한 소유권이 생긴다. 아이템에 에코를 두드려 아이템계에 진입한 순간, 그 아이템은 에르다의 기억과 섞인다 — 이것이 귀속의 서사적 근거이다.

### 6.2. 이노센트 거래 제한

이노센트는 장비보다 더 엄격한 제한이 필요하다. 이노센트가 자유롭게 거래되면 "아이템계에 직접 들어가 이노센트를 잡는" 핵심 야리코미 행위의 동기가 훼손된다.

| 이노센트 유형 | 거래 가능 여부 | 이유 |
| :--- | :--- | :--- |
| 야생 이노센트 (미복종) | 거래 불가 | 야생 이노센트는 아이템계 내부에서만 존재. 외부 추출 불가 |
| 복종 이노센트 Lv 1-99 | 거래 가능 (허브 거래소) | 신규 플레이어 진입 지원. 낮은 레벨은 파밍 동기에 영향 미미 |
| 복종 이노센트 Lv 100+ | 거래 불가 | 고레벨 이노센트는 장기 야리코미의 결과물. 거래 허용 시 직접 파밍 동기 소멸 |
| 유니크 타입 이노센트 | 거래 불가 | 아이템 대신 전용 드랍. 직접 도전의 증표 |

## 7. 자동 사냥 경제 (Auto-Hunt Economy)

### 7.1. 자동 사냥의 경제적 포지션

자동 사냥은 야리코미(무한 파밍)와 충돌하는 것처럼 보이지만, 올바르게 설계하면 오히려 야리코미를 강화한다. 자동 사냥의 경제적 역할은 "탐험한 구역의 기억을 지속적으로 상기시키는 낮은 강도의 수익 흐름"이다.

```yaml
auto_hunt_economy:
  prerequisite: "해당 구역을 수동으로 최초 클리어 완료"
  efficiency_vs_manual:
    gold_HL: 0.50       # 수동 대비 50% HL 수입
    equipment_drop: 0.50 # 수동 대비 50% 드랍률
    exp: 0.60            # 수동 대비 60% 경험치
  restrictions:
    legendary_drop: false   # Legendary 이상 드랍 금지 — 핵심 금지 규칙 준수
    ancient_drop: false     # Ancient 드랍 금지
    innocent_drop: false    # 이노센트 드랍 금지 (이노센트는 직접 전투만)
  allowed_drops:
    - Normal 장비
    - Magic 장비
    - Rare 장비 (확률 적용)
    - HL
    - 소모품 소재
```

### 7.2. 자동 사냥 Faucet/Sink 균형

자동 사냥이 과도한 Faucet이 되지 않도록 시간당 수입 상한을 설계한다.

| 항목 | 수동 전투 (1시간) | 자동 사냥 (1시간) | 비율 |
| :--- | :--- | :--- | :--- |
| HL 수입 | `Manual_HL_Per_Hour` | `Manual_HL_Per_Hour` × 0.5 | 50% |
| 장비 드랍 수 | `Manual_Drop_Per_Hour` | `Manual_Drop_Per_Hour` × 0.5 | 50% |
| 최상위 드랍 | Legendary + Ancient 가능 | Normal - Rare만 가능 | 근본 차이 |

> 설계 원칙: 자동 사냥은 편의 기능이지, 파밍의 왕도가 아니다. 플레이어가 "직접 하는 것이 효율적이고, 자동은 쉬는 동안 수익이 유지되는 것"으로 인식해야 한다.

## 8. 세션 경제 (Session Economy)

### 8.1. 10-15분 아이템계 세션 기대값

플레이어가 아이템계에서 10-15분을 보낼 때 얻을 수 있는 기대 수입을 정의한다. 이 수치는 세션 설계와 밸런스 기준값으로 사용된다.

```yaml
session_economy_itemworld_10min:
  # 기준: Normal 등급 아이템계, 솔로, 2 지층 완주 가정
  expected_HL_income:
    from_mob_kills: 300-500   # 지층당 몹 약 15-20마리 처치
    from_innocent_taming: -100 to -300  # 이노센트 복종 비용 (Sink)
    net_HL_gain: 0-400        # 순수입 (복종 비용 차감)

  expected_innocent_gain:
    wild_innocent_encounters: 1-3       # 지층당 야생 이노센트 1-3마리
    taming_success_rate: 1.0            # 복종은 항상 성공 (비용만 소모)
    net_innocent_gain_per_session: 2-6  # 세션당 복종 이노센트 2-6개

  expected_material_gain:
    stratum_clear_materials: 1-2        # 지층 클리어당 강화 소재
    boss_materials: 2-4                 # 보스 처치 시 추가 소재

  # 기준: Legendary 등급 아이템계, 4인 파티, 4 지층 완주 가정 (25-35분)
  expected_HL_income_legendary_party:
    from_mob_kills: 1500-2500
    from_innocent_taming: -800 to -2000
    net_HL_gain: 500-1500
    party_bonus_multiplier: 1.2         # 파티 플레이 HL 보너스 +20%

  expected_innocent_gain_legendary:
    wild_innocent_encounters_per_stratum: 2-4
    total_innocents_4_strata: 8-16
    party_sharing: "파티원별 독립 드랍 (중복 획득 가능)"
```

### 8.2. 세션 자연 종료 신호

플레이어가 아이템계에서 자연스럽게 탈출을 결정하도록 유도하는 경제적 신호를 설계한다.

| 신호 | 발생 조건 | 플레이어 판단 유도 |
| :--- | :--- | :--- |
| 탈출권 소진 임박 | 탈출권 1개 남음 | "지금 나가면 안전. 더 가면 사망 리스크" |
| HP 소모 임계점 | HP 30% 이하 | "여기서 실패하면 지층 손실" |
| 이노센트 슬롯 포화 | 인벤토리 이노센트 슬롯 90% | "더 잡아도 합성 전에 보관 공간 없음" |
| 최심층 지층 완료 | 마지막 지층 클리어 | "자연스러운 사이클 종료" |

> 설계 원칙: 강제 탈출 없이 플레이어 스스로 "지금이 나갈 때"라고 느끼게 하는 경제적 신호를 설계한다. 야리코미의 "한 지층만 더"와 "지금 나가야 해"의 긴장이 세션의 핵심 재미이다.

## 9. 장기 경제: 전생 루프 (Long-Term Economy: Reincarnation)

### 9.1. 전생의 경제적 기능

전생(Reincarnation)은 장기 경제의 리셋 밸브이자 재탄생 엔진이다. PoE의 리그 리셋이 거시 경제를 리셋하듯, 전생은 개인 경제를 리셋한다.

| 전생 전 상태 | 전생 후 상태 | 경제적 의미 |
| :--- | :--- | :--- |
| 아이템 스탯 최대치 도달 | 스탯 리셋, 잠재력 상승 | 성장 정체 해소. 새로운 Faucet 사이클 시작 |
| 이노센트 슬롯 한계 | 보너스 슬롯 +1 (전생 보너스) | 더 많은 이노센트 수용 = Sink 확장 |
| HL 과잉 축적 (인플레이션) | 전생 비용으로 HL 대량 소비 | 강력한 단발성 Sink. 인플레이션 해소 |
| 탐험 동기 감소 (구역 다 열림) | 전생 후 초기 스탯으로 되돌아가 스탯 게이트 재활성화 | 탐험 재동기화 |

```yaml
reincarnation_economy:
  cost:
    HL: "현재 보유 HL의 30% (Sink 기능)"
    memory_fragments: "해당 장비 레어리티 승급 비용의 2배"
  benefit:
    base_stat_multiplier: 1.1    # 전생 후 기본 스탯 +10% 상승
    innocent_slot_bonus: 1       # 이노센트 슬롯 +1
    latent_potential: "잠재 특수 어피닉스 해금 (랜덤 1개)"
  restriction:
    max_reincarnation_count: null  # 제한 없음 — 야리코미 무한 루프
    cooldown_days: 0               # 쿨다운 없음 — 즉시 재진입 가능
```

### 9.2. 인플레이션 장기 억제 메커니즘

장기 운영에서 HL 인플레이션이 심화되지 않도록 다층 억제 장치를 설계한다.

| 억제 장치 | 작동 방식 | 효과 |
| :--- | :--- | :--- |
| 전생 비용 HL 소비 | 보유 HL 30% 소비 | 고인물의 과잉 HL을 주기적으로 흡수 |
| 이노센트 복종 비용 레벨 비례 | 레벨 높을수록 복종 비용 지수 증가 | 야리코미 유저의 HL 소비 자동 확대 |
| 거래소 수수료 | 거래 금액의 `Trade_Fee_Rate`% | 유통 HL 지속 흡수 |
| 리스펙 비용 누진 | 리스펙 횟수 비례 비용 증가 | 반복 리스펙으로 인한 의도치 않은 Sink |
| 시즌/이벤트 콘텐츠 (Phase 4) | 새로운 Sink 콘텐츠 추가 | 운영 단계의 인플레이션 직접 대응 |

## 10. 내러티브 경제 통합 (Narrative Economy Integration)

### 10.1. 에르다의 대장간과 경제

에르다는 대장장이다. 그녀의 경제관은 "수리비를 얼마 받을까"라는 실용주의에서 출발한다.

| 경제 요소 | 서사적 프레이밍 | 플레이어 체감 |
| :--- | :--- | :--- |
| HL (홀로우 링) | 기억의 화폐. 고딕 세계에서 잊힌 기억의 가치가 통화가 되었다 | "이 전투에서 얼마짜리 기억을 얻었나" |
| 이노센트 복종 비용 | 야생 이노센트를 달래는 비용. 에르다의 망치 에코로 두드려 "이 아이템에 머물러도 좋다"고 설득하는 과정 | 대장장이가 재료를 다루는 느낌 |
| 이노센트 팜 유지비 | 이노센트를 키우는 공간 유지. 에르다의 대장간 한쪽을 이노센트 거처로 내어준 것 | 작은 생태계를 관리하는 감각 |
| 레어리티 승급 비용 | 아이템의 기억을 더 깊이 깨우는 의식 비용. 얕은 기억을 깊은 기억으로 변환하는 에르다의 장인 작업 | 장인이 명품을 만드는 과정 |
| 허브 거래소 수수료 | 세계의 중개상이 가져가는 수수료. 기억의 거래에는 증인이 필요하다 | 자연스러운 경제 마찰 |

### 10.2. 기억의 방랑자와 경제적 기회비용

기억의 방랑자(Memory Wanderer)는 경제적 의미에서 "제한 시간 드랍률 부스터"이다. 하지만 내러티브 맥락에서는 아이템계에서 돌아오지 못한 자들의 잔상이다. 이 두 의미가 겹치는 지점이 중요하다.

방랑자 아이템계는 일반 아이템계 대비 +50% 드랍량을 제공한다. 이 추가 보상은 서사적으로 정당화된다. "이 방랑자는 자신이 갇혀있는 아이템의 기억을 더 잘 안다. 그래서 그 내부의 보상이 더 풍부하게 드러난다."

기회비용 구조:
- 방랑자를 발견하면 지금 하던 월드 탐험 또는 허브 작업을 멈춰야 한다
- 30-45분에 1회 출현이므로, 무시하면 다음 기회는 30-45분 후
- 방랑자 아이템계 진입은 현재 인벤토리 장비 1개를 소비 (그 장비의 아이템계로 진입)

## 저주받은 문제 (Cursed Problems)

경제 설계에는 근본적으로 해결 불가능한 상충 관계가 존재한다. 이를 숨기지 않고 설계 결정으로 명시한다.

| 상충 관계 | 어느 쪽을 선택했는가 | 이유 |
| :--- | :--- | :--- |
| 야리코미 무한 파밍 vs 경제 인플레이션 방지 | 무한 파밍을 허용하되 Sink를 지수적으로 확장 — 강해질수록 더 많이 소비 | 에너지/피로도 제한은 핵심 금지 규칙. Sink 확장으로 수요 측에서 인플레이션 억제 |
| 거래 자유 vs 파밍 동기 보존 | Legendary 이상 귀속. 낮은 등급은 자유 거래 | D3 실물 경매장 실패 교훈. 최강 장비는 직접 파밍이 핵심 판타지 |
| 신규 유저 접근성 vs 고인물 성취감 보존 | Pity 시스템 + 낮은 등급 자유 거래로 신규 진입 지원. 상위 등급 귀속으로 고인물 희소성 유지 | 두 집단을 다른 경제 레이어로 분리 |
| 이노센트 파밍 동기 vs 허브 이노센트 팜 | 팜은 낮은 등급만 자동 생산. 고등급은 아이템계 직접 전투만 | 편의 기능이 핵심 콘텐츠를 대체하지 않는 원칙 |
| 자동 사냥 편의 vs 직접 플레이 우위 | 자동 50% 효율 + Legendary 이상 드랍 금지 | "쉬는 동안 수익 유지"는 허용, "대신 파밍"은 불허 |
