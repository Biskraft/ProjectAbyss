## 1. 개요 (Overview)

### 1-1. 문서 목적

본 문서는 Project Abyss의 난이도 설계 철학을 정의한다. 메트로베니아(수평적 탐험 숙련도)와 야리코미(수직적 성장 수치) 두 축의 난이도 곡선이 어떻게 공존하며, 3-Space 각 공간에서 난이도가 어떤 성격을 갖는지를 명확히 한다.

### 1-2. 한 줄 요약

> "죽음은 항상 플레이어의 실수여야 한다. 세계는 공정하게 도전하고, 성장은 끝없이 보상한다."

## 2. 설계 의도 (Design Intent)

### 2-1. 분해 (Deconstruct)

| 참고 게임 | 재미 요소 |
| :--- | :--- |
| 월하의 야상곡 | 패턴 학습을 통한 보스 공략, 능력 획득으로 열리는 새 층위의 발견감 |
| 디스가이아 | 수치 성장의 쾌감, 아이템계 깊은 층으로 갈수록 극한의 스탯 도전 |
| 스펠렁키 | 절차적 생성에서도 유지되는 공정한 난이도, 실패에서 배우는 구조 |
| 할로우 나이트 | 높은 난이도에서도 "반응 가능한 공격"으로 공정성 확보, 안식처 리듬 |

### 2-2. 분석 (Analyze)

| 재미의 본질 | 리스크와 리턴 분석 |
| :--- | :--- |
| 메트로베니아의 난이도 쾌감 | 리스크: 패턴 미숙지로 사망 / 리턴: 패턴 학습 후 무피격 클리어의 성취감 |
| 야리코미의 난이도 쾌감 | 리스크: 깊은 지층에서 전멸 시 진행 손실 / 리턴: 극한 스탯 장비 완성의 자부심 |
| 공정한 도전의 본질 | 플레이어가 "내가 못해서 졌다"고 납득할 때 재도전 동기가 발생한다. "게임이 불공정해서 졌다"고 느끼면 이탈한다 |

### 2-3. 재구축 (Rebuild)

| 독창적 변형 | Project Abyss의 강점 |
| :--- | :--- |
| 기술 난이도와 성장 난이도의 분리 | 월드에서는 액션 숙련도로, 아이템계에서는 장비 수치로 난이도를 극복. 두 축을 분리함으로써 액션이 약한 플레이어는 장비를 더 키워서, 파밍이 약한 플레이어는 실력으로 돌파할 수 있다 |
| 스탯 게이트를 통한 두 축의 교차 | 아이템계에서 장비를 강화(수직축)하면 월드의 스탯 게이트(수평축)가 열린다. 이것이 두 난이도 축을 순환으로 엮는 핵심 장치이다 |

## 3. 난이도 철학 선언: 공정한 도전 (Fair Challenge)

### 3-1. 핵심 선언

모든 전투, 플랫포밍, 함정은 다음 원칙을 준수해야 한다:

| 원칙 | 정의 | 위반 사례 |
| :--- | :--- | :--- |
| 반응 가능한 공격 | 적의 모든 공격에는 플레이어가 인지하고 대응할 수 있는 예고 동작(Tell)이 존재한다 | 예고 없는 즉발 공격, 화면 밖에서 날아오는 투사체 |
| 읽을 수 있는 패턴 | 적의 행동은 학습 가능한 패턴으로 구성된다. 동일 조건에서 동일 행동이 나와야 한다 | 완전 랜덤 공격 순서, 패턴 없는 즉흥 행동 |
| 정직한 히트박스 | 시각적 이펙트와 실제 판정 영역이 일치한다 | 이펙트보다 넓은 판정, 이펙트가 끝난 후에도 남아있는 판정 |
| 플레이어 귀책 사망 | 플레이어의 사망은 항상 플레이어의 판단 착오 또는 조작 실수에 기인해야 한다 | 카메라 밖 함정, 초견살, 탈출 불가능한 상황 |

### 3-2. 초견살 금지 원칙

초견에서 회피 불가능한 사망을 만들지 않는다. 새로운 위험 요소는 반드시 다음 절차를 거친다:

1. 소개 (Introduction): 안전한 환경에서 위험 요소를 시각적으로 노출한다
2. 학습 (Learning): 낮은 리스크 상황에서 대처법을 체득하게 한다
3. 시험 (Test): 다른 요소와 조합하여 숙련도를 검증한다
4. 극복 (Mastery): 보스전 또는 도전 구간에서 극한 조합으로 제시한다

## 4. 두 축의 난이도 곡선

### 4-1. 하강축: 메트로베니아 탐험 난이도

월드에서의 난이도는 플레이어의 액션 숙련도에 의존한다. 층위를 내려갈수록 난이도가 상승한다.

| 난이도 요소 | 정의 | 숙련도 대상 |
| :--- | :--- | :--- |
| 플랫포밍 | 점프, 이동, 능력 활용의 정밀도 | 타이밍, 거리 감각, 능력 조합 |
| 전투 패턴 | 적의 공격 패턴 인지 및 회피/반격 | 패턴 학습, 반응 속도, 위치 선정 |
| 환경 인식 | 함정, 지형 위험, 시야 제한 대처 | 관찰력, 기억력, 경로 판단 |
| 탐색 판단 | 갈림길에서 올바른 경로 선택, 숏컷 발견 | 맵 구조 이해, 단서 해석 |

수평축 난이도는 층위가 진행될수록 점진적으로 상승한다. 층위별 상세 난이도 매핑은 `Design_Level_Standards.md`의 6-1항을 참조한다.

### 4-2. 수직축: 야리코미 성장 난이도

아이템계에서의 난이도는 장비 수치와 이노센트 최적화에 의존한다.

| 난이도 요소 | 정의 | 성장 대상 |
| :--- | :--- | :--- |
| 적 스탯 스케일링 | 지층이 깊어질수록 적의 HP, ATK, DEF가 상승 | 장비 레벨, 이노센트 보너스 |
| 보스 체크포인트 | 각 지층의 보스. 보스 등급이 상승할수록 요구 스탯 증가 | 레어리티 승급, 이노센트 합성 |
| 고레어리티 순환 진입 | 더 높은 레어리티 아이템의 아이템계일수록 적 스탯 상승 | 순환 진입으로 획득한 상위 장비 강화 |
| 효율 최적화 | 이노센트 배치, 장비 조합, 파티 구성의 최적해 | 시스템 이해도, 빌드 설계 능력 |

수직축 난이도의 지층별 상세 스케일링은 `Design_Level_Standards.md`의 5-2항을 참조한다.

### 4-3. 교차점: 스탯 게이트

두 축이 만나는 지점이 스탯 게이트이다. 수직적 성장(아이템계 파밍)이 수평적 진행(월드 탐험)을 가능케 하는 구조적 장치이다.

```
[수직축: 야리코미]
     ^
     |
     |    아이템계에서 장비 강화
     |         |
     |         v
     |    스탯 상승
     |         |
     |         v
     |    스탯 게이트 해금 -------> 새 월드 층위 접근
     |                              |
     |                              v
     |                         새 장비 획득
     |                              |
     +------------------------------+-------> [수평축: 탐험]
```

| 교차 방향 | 메커니즘 | 플레이어 경험 |
| :--- | :--- | :--- |
| 수직 -> 수평 | 아이템계에서 장비 스탯 상승 -> 월드 스탯 게이트 해금 | "드디어 STR이 충분해서 저 바위를 부술 수 있다" |
| 수평 -> 수직 | 새 월드 층위에서 더 높은 레어리티 장비 획득 -> 더 깊은 아이템계 도전 가능 | "이 층위에서 Rare 장비를 얻었으니 아이템계 3지층까지 갈 수 있다" |

스탯 게이트의 상세 설계(스탯 유형별 게이트 표현, 배치 원칙)는 `Design_Level_Standards.md`의 4-2항을 참조한다.

## 5. 3-Space별 난이도 성격

### 5-1. 월드 (World): 기술 난이도

| 항목 | 정의 |
| :--- | :--- |
| 난이도 유형 | 기술 난이도 (액션 숙련도, 패턴 학습) |
| 핵심 도전 | 보스 패턴 공략, 정밀 플랫포밍, 환경 위험 극복 |
| 성장의 역할 | 보조적. 장비 스탯이 높으면 적을 빨리 처치할 수 있지만, 공격을 피하는 것은 플레이어 실력 |
| 난이도 곡선 | 층위별 점진적 상승. 층위 내에서 입구 -> 보스 방향으로 상승 |
| 실패 비용 | 낮음 (세이브 포인트 복귀, 진행 일부 재시도) |

월드의 난이도는 "실력이 느는 쾌감"에 초점을 맞춘다. 같은 보스를 다시 만나면 이전보다 쉽게 느껴져야 한다. 이는 장비가 강해져서가 아니라 플레이어의 패턴 이해도가 높아졌기 때문이어야 한다.

### 5-2. 아이템계 (Item World): 성장 난이도

| 항목 | 정의 |
| :--- | :--- |
| 난이도 유형 | 성장 난이도 (장비 수치, 이노센트 최적화) |
| 핵심 도전 | 깊은 지층의 적 스탯 돌파, 보스 DPS 체크, 고레어리티 아이템계의 난이도 관리 |
| 기술의 역할 | 보조적. 조작이 숙련되면 자신의 스탯보다 약간 높은 지층도 돌파 가능하지만, 근본적 해결은 장비 강화 |
| 난이도 곡선 | 지층 깊이에 비례하여 상승. 각 지층 보스에서 스파이크 |
| 실패 비용 | 중간-높음 (탈출 아이템 미사용 시 진행 지층 손실) |

아이템계의 난이도는 "성장의 한계를 느끼고 돌파하는 쾌감"에 초점을 맞춘다. "2번째 지층에서 막혔다 -> 이노센트를 합성하고 다시 도전 -> 3번째 지층까지 돌파"라는 성장-도전 순환이 핵심이다.

### 5-3. 세이브 포인트 (Rest Area): 난이도 없음

| 항목 | 정의 |
| :--- | :--- |
| 난이도 유형 | 없음 (안전지대) |
| 핵심 기능 | 준비, 장비 정비, 대장간/상점 이용 |
| 설계 의도 | 도전과 도전 사이의 안식처. 월드/아이템계에서 돌아온 플레이어가 긴장을 풀고 다음 모험을 계획하는 공간 |

세이브 포인트는 감정 곡선에서 "안도"와 "계획"의 역할을 담당한다. 전투 스트레스 없이 장비를 정리하고, 대장간을 이용하고, 다음 도전을 설계할 수 있어야 한다.

## 6. 난이도 곡선 원칙

### 6-1. 점진적 상승 (Gradual Escalation)

난이도는 급격한 점프 없이 점진적으로 상승해야 한다.

| 적용 공간 | 점진적 상승 규칙 |
| :--- | :--- |
| 월드 | 층위 난이도 1 -> 2 -> 3 -> 4 -> 5 순서. 한 층위 내에서도 입구에서 보스 방향으로 점진 상승 |
| 아이템계 | 지층 깊이에 비례한 적 스탯 스케일링. 급격한 점프 없이 선형-완만지수 혼합 곡선 |

한 층위에서 다음 층위로 넘어갈 때, 이전 층위의 최고 난이도가 다음 층위의 최저 난이도보다 약간 높거나 같아야 한다. 이 구간이 "성장 확인 구간"으로 기능한다: 플레이어가 새 층위 초반에서 "나 강해졌구나"를 체감할 수 있다.

### 6-2. 안식처 리듬 (Rest Rhythm)

고난이도 구간 후에는 반드시 안전 구간을 배치한다.

| 트리거 | 안식처 유형 | 설계 의도 |
| :--- | :--- | :--- |
| 보스 처치 후 | 세이브 포인트 + 안전 통로 | 성취감 확인, 긴장 해소 |
| 난이도 3+ 전투 연속 3회 후 | 적 없는 탐험 방 또는 전환 통로 | 피로 누적 방지 |
| 아이템계 보스 지층 클리어 후 | 탈출/계속 선택지 + 보상 화면 | 리스크 재평가 기회 |
| 새 층위 진입 직후 | 낮은 난이도 도입부 | 새 환경 적응 시간 |

안식처 리듬은 `Design_Level_Standards.md`의 6-2항에서 정의한 원칙과 동기화한다.

### 6-3. 학습 공간 (Teaching Space)

새로운 위험 요소는 안전한 환경에서 먼저 소개한다.

| 단계 | 조건 | 예시 |
| :--- | :--- | :--- |
| 독립 소개 | 해당 위험만 단독으로 등장, 다른 위험 요소 없음 | 가시 함정이 통로에 하나만 배치, 적 없음 |
| 안전한 실패 | 실패해도 치명적이지 않은 환경 | 낙하해도 바로 아래 발판으로 복귀 가능 |
| 점진적 조합 | 학습한 위험 + 이전에 학습한 위험의 조합 | 가시 함정 + 이동 플랫폼 (각각은 이미 학습 완료) |
| 극한 조합 | 다수의 학습 완료 위험이 동시에 등장 | 보스전에서 가시 + 이동 플랫폼 + 적 투사체 |

### 6-4. 스파이크 제한 (Spike Limitation)

한 Room 내에서 전투, 플랫포밍, 함정이 동시에 최고 난이도인 구간은 보스전에만 허용한다.

| 구간 유형 | 전투 난이도 | 플랫폼 난이도 | 함정 난이도 | 허용 여부 |
| :--- | :--- | :--- | :--- | :--- |
| 일반 Room | 높음 | 높음 | 높음 | 금지 |
| 일반 Room | 높음 | 높음 | 없음 | 허용 |
| 일반 Room | 높음 | 없음 | 높음 | 허용 |
| 보스 Room | 높음 | 높음 | 높음 | 허용 (보스전 한정) |

일반 Room에서는 3가지 난이도 축(전투/플랫폼/함정) 중 최대 2개만 동시에 높은 수준으로 배치할 수 있다. 이는 플레이어의 인지 과부하를 방지하기 위함이다.

## 7. 난이도와 보상의 비례 원칙 (Risk and Reward)

### 8-1. 기본 원칙

리스크가 높은 행동에는 반드시 그에 비례하는 리턴이 존재해야 한다. 리스크 없는 보상은 성취감을 훼손하고, 보상 없는 리스크는 좌절감을 유발한다.

| 행동 | 리스크 | 리턴 | 비례 검증 |
| :--- | :--- | :--- | :--- |
| 월드 미지의 층위 진입 | 사망 시 세이브 포인트 복귀, 시간 손실 | 새 능력 발견, 새 장비 획득, 맵 데이터 | 낮은 리스크 - 중간 리턴 (탐험 초반) |
| 월드 보스 도전 | 사망 시 세이브 포인트 복귀 + 보스전 재도전 시간 | 능력 렐릭 획득 (영구 진행) | 중간 리스크 - 높은 리턴 |
| 아이템계 깊은 지층 도전 | 전멸 시 진행 지층 손실 (탈출 아이템 미사용) | 이노센트 획득, 아이템 EXP, 레어리티 승급 기회 | 높은 리스크 - 높은 리턴 |
| 고레어리티 순환 진입 | 더 높은 레어리티 아이템계의 높은 난이도 | 상위 레어리티 전용 이노센트, 레어리티 승급 기회 | 높은 리스크 - 매우 높은 리턴 |

### 8-2. 보상 경로 난이도 원칙

선택적 보상 경로(비밀 방, 도전 구간)의 난이도는 보상 가치에 비례해야 한다.

| 경로 유형 | 난이도 | 기대 보상 |
| :--- | :--- | :--- |
| Critical Path 인접 | 기본 | 소량의 골드, 소비 아이템 |
| 숙련자 보상 경로 | 상급 (3.5타일 간격 등) | Magic 이상 장비, 월드 소재 |
| 능력 게이트 너머 | 해당 능력 필요 | Rare 이상 장비, 능력 렐릭, 숏컷 |
| 스탯 게이트 너머 | 요구 스탯 달성 필요 | 요구 스탯 수치에 비례하는 보상 |
| 비밀 방 | 발견 난이도 (탐색 능력) | 고유 장비, 로어 단편, 희귀 소재 |

## 9. 멀티플레이 시 난이도 스케일링 원칙

### 9-1. 아이템계 파티 스케일링

아이템계에서 파티 인원이 증가하면 적의 스탯이 동적으로 조정된다.

| 파티 인원 | 적 HP 배율 | 적 ATK 배율 | 보상 배율 | 설계 의도 |
| :--- | :--- | :--- | :--- | :--- |
| 1인 | x1.0 | x1.0 | x1.0 | 솔로 기준선 |
| 2인 | 파라미터(`Party_2P_HP_Scale`) | 파라미터(`Party_2P_ATK_Scale`) | 파라미터(`Party_2P_Reward_Scale`) | 두 명이 하면 더 쉽지만 보상도 증가 |
| 3인 | 파라미터(`Party_3P_HP_Scale`) | 파라미터(`Party_3P_ATK_Scale`) | 파라미터(`Party_3P_Reward_Scale`) | 역할 분담 필요 시작 |
| 4인 | 파라미터(`Party_4P_HP_Scale`) | 파라미터(`Party_4P_ATK_Scale`) | 파라미터(`Party_4P_Reward_Scale`) | 풀 파티. 가장 높은 보상 |

구체적 배율 수치는 `Sheets/` CSV에서 관리한다.

### 9-2. 스케일링 설계 방침

| 방침 | 설명 |
| :--- | :--- |
| 혼자서도 재미있고 함께하면 더 재미있다 | 솔로 플레이로 모든 콘텐츠를 클리어할 수 있어야 한다. 파티는 "필수"가 아닌 "더 효율적이고 재미있는 선택지"이다 |
| 깊은 지층에서 파티 유리 | 아이템계 후반 지층 이후 적 스탯 증가 곡선이 가팔라진다. 솔로로도 가능하지만 파티가 효율적이 되도록 설계한다 |
| 인원 감소 시 동적 하향 | 파티 플레이 중 인원이 이탈하면 적 스탯이 남은 인원에 맞게 즉시 하향 조정된다. 이탈로 인한 전멸을 방지한다 |
| 역할 시너지 보상 | 탱커/딜러/레인저/서포터 역할이 고르게 편성된 파티에 추가 보상 배율을 부여한다 |

### 9-3. 월드 협동 (2인)

월드는 솔로가 기본이지만, 친구 초대로 2인 협동이 가능하다. 2인 협동 시:

| 항목 | 규칙 |
| :--- | :--- |
| 적 스탯 | 변경 없음 (월드 적은 고정 난이도) |
| 보스 스탯 | 파라미터(`World_Coop_Boss_HP_Scale`) 적용 |
| 추가 보상 | 2인 전용 퍼즐/보상 방 접근 가능 |
| 난이도 체감 | 전투는 쉬워지지만, 2인 전용 퍼즐이 새로운 도전을 제공 |

## 10. 실패 페널티 설계 철학

### 10-1. 기본 원칙

실패 페널티는 "학습 기회를 제공하되 좌절감을 최소화"하는 수준으로 설정한다. 과도한 페널티는 재도전 동기를 꺾고, 페널티가 없으면 도전의 긴장감이 사라진다.

### 10-2. 월드: 세이브 포인트 복귀

| 항목 | 규칙 |
| :--- | :--- |
| 사망 시 | 최근 세이브 포인트로 복귀 |
| 보존 항목 | 획득한 아이템, 맵 데이터, 능력 렐릭은 유지 |
| 손실 항목 | 세이브 포인트 이후 진행한 구간의 적 처치 상태 리셋 |
| 시간 손실 | 세이브 포인트 간격이 Room 3-5개(5-10분)이므로 최대 10분 분량의 재도전 |
| 설계 의도 | 진행 자체를 잃지 않으므로 재도전 부담이 낮다. "다시 해보면 이번엔 패턴을 알고 있으니 클리어할 수 있다"는 자신감을 유지한다 |

세이브 포인트 간격의 상세 규격은 `Design_Level_Standards.md`의 5-1항을 참조한다.

### 10-3. 아이템계: 진행 보존 vs 전체 손실의 선택

아이템계에서의 실패 페널티는 플레이어에게 "리스크 관리"라는 의사결정을 요구한다.

| 상황 | 결과 | 리스크 관리 수단 |
| :--- | :--- | :--- |
| 보스 지층 클리어 후 자발적 탈출 | 현재까지의 모든 진행(아이템 EXP, 이노센트) 보존 | 보스 클리어마다 탈출 기회 제공 |
| Mr. Gency's Exit 사용 | 현재까지의 모든 진행 보존, 즉시 탈출 | 소모품. 상점 구매 또는 아이템계 내 드랍으로 획득 |
| 전멸 (탈출 아이템 없음) | 마지막 보스 클리어 지층까지의 진행만 보존. 그 이후 진행은 손실 | 보스 클리어 시마다 자동 세이브. 보스 사이 구간(최대 10지층)이 손실 범위 |
| ~~재귀적 진입 중 전멸~~ | ~~DEPRECATED. 재귀적 중첩 진입 삭제~~ | ~~순환 구조로 대체: 아이템계 전멸 시 월드 귀환, 지층 진행 손실~~ |

이 구조의 핵심은 "언제 탈출할 것인가"라는 의사결정이다. 더 깊이 가면 더 큰 보상을 얻지만, 탈출 아이템 없이 전멸하면 보스 클리어 이후 진행분을 잃는다. 이것이 아이템계의 핵심 리스크-리턴 긴장감이다.

## 11. 저주받은 문제 (Cursed Problems)

### 11-1. 기술 난이도 vs 성장 난이도의 균형

| 문제 | 상세 |
| :--- | :--- |
| 상충 | "액션 실력이 좋으면 장비가 약해도 클리어할 수 있어야 한다" vs "장비를 키워야 진행할 수 있어야 한다(스탯 게이트)" |
| 해결 방향 | Critical Path는 적정 장비 수준에서 기술로 돌파 가능하게 설계. 스탯 게이트는 Critical Path가 아닌 선택적 경로에만 배치하여 "반드시 파밍해야 진행 가능"한 상황을 방지 |
| 타협 | 스탯 게이트 층위는 "추가 보상"이지 "필수 진행"이 아니다. 단, 후반 층위(심연의 구)는 스탯 게이트가 주요 경로에 포함되어 야리코미를 유도한다 |

### 11-2. 공정한 난이도 vs 절차적 생성의 불확실성

| 문제 | 상세 |
| :--- | :--- |
| 상충 | "모든 죽음은 플레이어 실수여야 한다" vs "절차적 생성은 예측 불가능한 조합을 만들 수 있다" |
| 해결 방향 | Room Template과 Chunk를 사전 검증된 풀에서만 조립. 조합 규칙으로 위험 요소 동시 배치 상한을 제한. Always Winnable 보장 알고리즘 적용 |
| 타협 | 완전한 공정성 보장은 불가. 대신 "부당한 사망" 발생 시 페널티를 최소화(세이브 포인트 간격 단축, 아이템계 보스 지층 자동 세이브)하여 좌절감을 완화 |

### 11-3. 솔로 플레이 난이도 vs 파티 플레이 난이도

| 문제 | 상세 |
| :--- | :--- |
| 상충 | "솔로로 모든 콘텐츠를 클리어할 수 있어야 한다" vs "파티 플레이가 의미 있으려면 솔로로는 어려운 구간이 필요하다" |
| 해결 방향 | 솔로 클리어는 가능하되 "효율"에서 차이를 둔다. 아이템계 깊은 지층(70+)은 솔로로 클리어 가능하지만 시간이 오래 걸리고, 파티로 하면 빠르고 보상도 더 많다 |
| 타협 | 파티 필수 콘텐츠는 만들지 않는다. 대신 파티로 하면 "더 재미있는" 전용 메커닉(역할 시너지, 협동 패턴)을 제공한다 |

### 11-4. 실패 페널티의 긴장감 vs 접근성

| 문제 | 상세 |
| :--- | :--- |
| 상충 | "실패에 페널티가 있어야 긴장감이 있다" vs "과도한 페널티는 캐주얼 유저를 이탈시킨다" |
| 해결 방향 | 공간별로 페널티 수준을 차등화. 월드는 낮은 페널티(세이브 포인트 복귀), 아이템계는 높은 페널티(지층 진행 손실)지만 리스크 관리 수단(탈출 아이템, 보스 지층 세이브) 제공 |
| 타협 | 페널티의 최대치를 제한한다. 아이템계에서도 보스 클리어마다 자동 세이브되므로 최대 손실은 10지층분의 진행이다. "수 시간의 진행이 한순간에 날아가는" 상황은 발생하지 않는다 |

## 12. 검증 체크리스트 (Verification Checklist)

### 12-1. 공정한 도전 검증

* [ ] 모든 적 공격에 플레이어가 인지 가능한 예고 동작(Tell)이 존재하는가
* [ ] 적의 행동이 학습 가능한 패턴으로 구성되어 있는가
* [ ] 시각적 이펙트와 실제 히트박스가 일치하는가
* [ ] 초견에서 회피 불가능한 사망이 존재하지 않는가
* [ ] 새로운 위험 요소가 안전 환경에서 먼저 소개되는가

### 12-2. 난이도 곡선 검증

* [ ] 층위 간 난이도가 점진적으로 상승하는가 (급격한 스파이크 없음)
* [ ] 고난이도 구간 후 안식처(세이브 포인트, 안전 통로)가 배치되어 있는가
* [ ] 일반 Room에서 전투+플랫폼+함정이 동시에 최고 난이도인 구간이 없는가
* [ ] 아이템계 층별 난이도가 `Design_Level_Standards.md`의 스케일링과 일치하는가

### 12-3. 두 축 교차 검증

* [ ] 스탯 게이트가 Critical Path에 배치되지 않았는가 (후반 예외 층위 제외)
* [ ] 아이템계에서 장비를 강화하면 월드의 새 층위가 열리는 순환이 작동하는가
* [ ] 기술이 뛰어난 플레이어가 적정 장비 수준에서 Critical Path를 클리어할 수 있는가

### 12-4. 리스크-리턴 검증

* [ ] 모든 도전 행동에 리스크와 리턴이 명확히 정의되어 있는가
* [ ] 보상 경로의 난이도가 보상 가치에 비례하는가
* [ ] 리스크 없이 얻을 수 있는 고가치 보상이 존재하지 않는가

### 12-5. 멀티플레이 스케일링 검증

* [ ] 솔로로 모든 콘텐츠를 클리어할 수 있는가
* [ ] 파티 인원 변동 시 적 스탯이 동적으로 조정되는가
* [ ] 파티 플레이의 보상이 솔로 대비 효율적인가 (파티 유인)

### 12-6. 실패 페널티 검증

* [ ] 월드 사망 시 최대 시간 손실이 10분 이내인가 (세이브 포인트 간격 준수)
* [ ] 아이템계 전멸 시 최대 손실이 10지층분의 진행으로 제한되는가
* [ ] 아이템계에 리스크 관리 수단(Mr. Gency's Exit, 보스 지층 탈출)이 충분히 제공되는가
* ~~[ ] 재귀적 진입의 리스크가 보상(깊이 보너스)에 비례하는가~~ (DEPRECATED — 재귀 진입 삭제)

### 12-7. Project Abyss 정렬

* [ ] 3대 기둥 정렬: 이 문서가 탐험(월드 기술 난이도) + 야리코미(아이템계 성장 난이도)에 기여하는가
* [ ] 3-Space 명시: 각 공간의 난이도 성격이 명확히 분리 정의되었는가
* [ ] 순환 구조 강화: 스탯 게이트를 통한 두 축 교차가 순환 루프를 강화하는가
* [ ] 금지 규칙 준수: Pay-to-Win, PvP 강제, 에너지 시스템 등 금지 규칙을 위반하지 않는가
