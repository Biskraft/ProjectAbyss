## 1. 개요 (Overview)

### 1.1. 문서 목적

본 문서는 Project Abyss의 수익화 전략 전체를 정의하는 상위 설계 문서이다.

수익화는 게임의 적이 아니다. 잘못 설계된 수익화가 적이다. "야리코미(무한 파밍)"를 핵심 판타지로 삼는 게임에서 플레이 시간을 돈으로 사야 한다면, 핵심 판타지 자체가 붕괴된다. 본 문서는 "플레이어의 시간과 성취를 보호하면서 지속 가능한 수익을 창출하는 구조"를 정의한다.

선택한 모델은 **Model D: 웹 무료 + Steam 프리미엄**이다. 웹 버전이 바이럴 마케팅 엔진이 되고, Steam 버전이 수익을 회수한다. 두 버전은 서로를 잠식하지 않고 강화한다. 웹은 "스파이크(아이템계 진입)"를 충분히 맛보게 해 "더 깊이 가고 싶다"는 갈망을 만들고, 그 갈망이 Steam 구매로 자연스럽게 전환된다.

### 1.2. 한 줄 요약

> "웹에서 아이템계의 맛을 보여주고, Steam에서 심연의 끝까지 내려가게 한다."

### 1.3. 3대 기둥 정렬

| 기둥 | 수익화 전략에서의 역할 |
| :--- | :--- |
| 메트로베니아 탐험 | 웹 무료 구간은 Tier 1~2 탐험을 포함하여 탐험가 판타지를 충분히 전달한다. Tier 3 이상의 새로운 층위와 비밀은 Steam 전용 콘텐츠로 전환 욕구를 자극한다 |
| 아이템계 야리코미 | 웹에서 아이템계 첫 체험(1지층)을 제공하여 스파이크를 전달한다. Legendary/Ancient 레어리티와 심층 지층(3~4층)은 Steam 전용으로 야리코미의 깊이를 유료화의 동기로 연결한다 |
| 온라인 멀티플레이 | 멀티플레이(파티 아이템계 공략)는 Steam 전용 기능이다. 혼자서는 한계가 느껴지는 지층에서 "친구를 불러야 한다"는 시점이 Steam 전환을 자극하는 자연스러운 게이트가 된다 |

## 2. 수익화 모델 정의 (Monetization Model)

### 2.1. 모델 선택 근거

Project Abyss가 고려한 4가지 수익화 모델과 선택 근거:

| 모델 | 구조 | 채택 여부 | 기각 이유 |
| :--- | :--- | :---: | :--- |
| **Model A: 완전 유료** | 웹/Steam 모두 유료 | X | 첫 플레이까지 마찰이 너무 크다. 인디 신작이 구매를 설득하기 어렵다 |
| **Model B: 완전 무료 + 부분 유료** | 인게임 과금 | X | Pay-to-Win 구조 불가피. 야리코미 핵심 판타지와 충돌 |
| **Model C: 구독형** | 월정액 | X | 작은 스튜디오에서 구독 유지를 위한 지속적 콘텐츠 공급 불가 |
| **Model D: 웹 무료 + Steam 프리미엄** | 웹=무료 체험, Steam=일회성 구매 | **O** | 바이럴 가능성 + Pay-to-Win 없음 + 인디 성공 레퍼런스 다수 |

**Model D 채택.**

### 2.2. 이원 구조 정의

```yaml
monetization_model:
  label: "Model D — Web Free + Steam Premium"

  web:
    price: 0
    content_gate: "Tier 1~2, 아이템계 Normal~Rare 레어리티, 1지층"
    purpose: "바이럴 마케팅 엔진. 스파이크(아이템계 진입) 체험 제공"
    save_system: "웹 계정 세이브 (로컬 + 서버 선택)"
    conversion_hook: "Legendary 첫 드랍 시 Steam 전환 유도 메시지"

  steam:
    price_usd: 15  # 범위: $15~$20, 초기 EA는 $15
    content_gate: "전체 콘텐츠 (Tier 1~7, 전 레어리티, 전 멀티플레이)"
    purpose: "주 수익원. 야리코미 끝까지 제공"
    save_transfer: "웹 세이브 원클릭 Steam 이전"
    cosmetic_dlc: "출시 후 코스메틱 DLC (옵션)"
```

### 2.3. 가격 전략

| 항목 | 값 | 근거 |
| :--- | :--- | :--- |
| Steam EA 가격 | $15 USD | Hollow Knight($15) 기준선 — 콘텐츠량 대비 신뢰도 있는 가격대 |
| Steam 정식 출시 가격 | $15~$20 USD | 콘텐츠 완성도에 따라 결정. EA 구매자는 차액 없이 업그레이드 |
| EA 할인율 | 없음 | 가격 희석보다 품질 인식 유지 우선 |
| 런치 할인 | 20% 1주 | Steam 알고리즘 부스팅 + 위시리스트 전환 |

**가격 결정 원칙:** "콘텐츠량으로 가격을 정당화하는" Hollow Knight 모델을 따른다. 낮은 가격을 무기로 삼지 않고, 압도적인 콘텐츠로 가격을 정당화한다.

## 3. 웹 무료 구간 설계 (Web Free Tier Design)

### 3.1. 포함 콘텐츠

웹 무료 구간은 "스파이크를 충분히 맛보게 하되, '더 깊이'에 대한 갈망을 만드는" 수준으로 설계한다.

| 카테고리 | 웹 무료 포함 | 기준 |
| :--- | :--- | :--- |
| 월드 층위 | Tier 1~2 (전체 7Tier 중 2개 층위) | 탐험가 판타지 충분히 전달 |
| 장비 레어리티 | Normal / Magic / Rare | 성장 체험 가능. "더 좋은 것"에 대한 기대감 형성 |
| 아이템계 지층 | 1지층 (Normal~Rare 아이템 기준) | 스파이크(아이템계 진입)의 핵심 체험 1회 제공 |
| ~~재귀적 진입~~ | ~~DEPRECATED~~ | ~~재귀적 중첩 진입 삭제. 순환 진입(월드 경유)으로 대체~~ |
| 이노센트 | 야생 이노센트 포획, 기본 복종 | 야리코미 맛보기. 슬롯 풀 활용은 Rare 이하만 |
| 무기 카테고리 | 전체 5개 카테고리 접근 가능 | 플레이스타일 탐색 허용 |
| 캐릭터 레벨 | Lv 1~15 (전체 Lv 1~50 중) | 초반 성장 쾌감 제공 |
| 멀티플레이 | 없음 | 솔로 체험으로 게임 이해 후 Steam 전환 유도 |
| NPC / 스토리 | Act 1 전체 | 서사 진입점 제공, Act 2 이후 호기심 유발 |

### 3.2. 제외 콘텐츠

| 카테고리 | 웹 무료 제외 | 제외 이유 |
| :--- | :--- | :--- |
| 월드 층위 | Tier 3~7 | Steam 전환의 핵심 동기 |
| 장비 레어리티 | Legendary / Ancient | 야리코미 깊이의 정점. Steam 전환 타이밍의 자연스러운 게이트 |
| 아이템계 지층 | 2~4지층 (+ 심연) | 파티 플레이가 필요한 깊은 지층은 Steam 전용 |
| ~~재귀적 진입~~ | ~~DEPRECATED~~ | ~~재귀적 중첩 진입 삭제~~ |
| 멀티플레이 | 전체 | 사회적 연결은 Steam 구매 후 허브에서 시작 |
| 캐릭터 레벨 | Lv 16~50 | 성장의 정점은 Steam에서 |
| 코스메틱 | 전체 | 코스메틱 상점은 Steam 전용 수익원 |
| 스토리 | Act 2~4 | 서사 해소 욕구를 Steam 전환 동기로 활용 |

### 3.3. 무료 → 유료 전환 트리거

전환 트리거는 강요가 아닌 **자연스러운 욕망**으로 설계한다. 플레이어가 "막혀서 사는 것"이 아니라 "더 하고 싶어서 사는 것"이어야 한다.

```yaml
conversion_triggers:
  - id: CT-01
    trigger: "Legendary 아이템 최초 드랍"
    moment: "드랍 연출 이후, 아이템 상세 창에서"
    message: "이 무기의 4지층에는 무엇이 있을까? Steam 버전에서 확인하세요."
    tone: "호기심 유도. 압박 없음."

  - id: CT-02
    trigger: "1지층 최초 클리어"
    moment: "클리어 보상 화면 이후"
    message: "여기서 멈추지 마세요. 2지층부터 이야기가 달라집니다."
    tone: "다음 챕터 예고처럼. 서사적 유도."

  - id: CT-03
    trigger: "Tier 2 마지막 보스 처치"
    moment: "능력 렐릭 획득 연출 이후"
    message: "Tier 3의 문이 보입니다. 저 너머에 무엇이 있는지 알고 싶지 않으신가요?"
    tone: "탐험 욕구 자극."

  - id: CT-04
    trigger: "친구 초대 링크 수신 (아이템계)"
    moment: "링크 클릭 시 웹 버전 진입"
    message: "이 파티에 합류하려면 Steam 버전이 필요합니다. [Steam 페이지 바로가기]"
    tone: "사회적 소속 욕구 활용. 단 1회만 표시."
```

**금지 패턴:** 팝업 반복 표시, 플레이 중 강제 중단, 카운트다운 타이머, "지금 구매하지 않으면 진행 불가" 메시지.

## 4. Steam 프리미엄 구간 설계 (Steam Premium Tier Design)

### 4.1. 전체 콘텐츠 보장

Steam 버전은 출시 시점의 모든 콘텐츠를 제공한다. "숨겨진 추가 구매"가 없어야 한다.

| 카테고리 | Steam 포함 콘텐츠 |
| :--- | :--- |
| 월드 층위 | Tier 1~7 전체 (웹 구간 포함) |
| 장비 레어리티 | Normal~Ancient 전체 5등급 |
| 아이템계 지층 | 전 레어리티 × 최대 4지층 + 심연 |
| 순환 진입 | 모든 레어리티 아이템의 아이템계 순환 진입 |
| 이노센트 | 전체 이노센트 타입 + 심층 이노센트 |
| 멀티플레이 | 1~4인 파티 아이템계 공략 전체 |
| 허브 | 전체 허브 시설, NPC 상점, 창고 |
| 스토리 | Act 1~4 + 엔딩 전체 |
| 캐릭터 레벨 | Lv 1~50 전체 성장 경로 |
| 무기 | 전체 무기 카테고리 + 고급 무기 |
| 기술 스택 | 성능 최적화 빌드 (WebGL → 최적화) |

### 4.2. 세이브 이전 시스템

웹 → Steam 이전은 **원클릭**으로 설계한다. 마찰이 없어야 한다.

```yaml
save_transfer:
  method: "계정 기반 자동 동기화"
  trigger: "Steam 버전 첫 실행 시 웹 계정 로그인 프롬프트"
  transferred_data:
    - character_level
    - equipment_inventory
    - map_exploration_data
    - item_world_progress
    - innocent_collection
    - story_progress_act1
  not_transferred:
    - cosmetics  # 웹에는 코스메틱 없음
  rollback: "이전 후 웹 세이브는 읽기 전용으로 유지 (30일)"
  conflict_resolution: "Steam 세이브가 항상 우선"
```

### 4.3. Steam 전용 최적화

| 최적화 항목 | 내용 |
| :--- | :--- |
| 렌더링 | Steam 버전은 PixiJS 고급 렌더 설정 활성화. 웹 버전은 호환성 우선 |
| 로딩 | Steam 버전은 에셋 로컬 캐싱으로 웹 버전 대비 로딩 50% 이상 단축 목표 |
| 입력 | Steam Input API 지원 (게임패드 전체 커스터마이징) |
| 해상도 | Steam 버전은 풀스크린 + 다중 해상도 지원 |

## 5. 코스메틱 전략 (Cosmetic Strategy)

### 5.1. 코스메틱 원칙

코스메틱은 "표현의 자유"이지 "성능의 차이"가 아니다.

```yaml
cosmetic_design_principles:
  must:
    - "순수 시각적 변경만 적용. 스탯/이노센트/대미지에 영향 없음"
    - "기본 코스메틱은 게임 플레이로 획득 가능 (Steam 구매 없이)"
    - "유료 코스메틱은 심미적으로 매력적이되 게임 내 식별 우위 없음"
  must_not:
    - "Pay-to-Win 요소 (스탯 부스트, 이노센트 추가, 경험치 가속)"
    - "확률형 박스 (가챠) 형태 판매"
    - "시간 제한 구매 압박 (카운트다운, 한정판 강조)"
    - "기본 외형 고의 열화 후 유료 복원 구조"
```

### 5.2. 코스메틱 카테고리

| 카테고리 | 항목 수 | 가격대 | 획득 방법 |
| :--- | :--- | :--- | :--- |
| 에르다 의상 | 4~6벌 (Phase 4) | $3~$5 | Steam DLC 개별 구매 |
| 무기 스킨 | 무기 카테고리당 2~3종 | $2~$3 | Steam DLC 개별 구매 |
| 에코 이펙트 | 인챈트 원소별 시각 효과 8종 | $1~$2 | Steam DLC 개별 구매 |
| 이노센트 스킨 | 이노센트 외형 변경 | $1~$2 | 게임 내 달성 또는 DLC |
| 허브 꾸미기 | 플레이어 거처 데코 | $1~$3 | 게임 내 달성 또는 DLC |

**번들 전략:** 개별 구매 외에 "에르다 컬렉션 번들" 등 20% 할인 번들 제공. 번들이 개별 합산보다 항상 저렴하도록 설계.

### 5.3. 게임 내 코스메틱 획득 (무료)

유료 코스메틱과 병렬로 존재하는 **게임 플레이 보상 코스메틱**을 설계하여 "돈 없이도 꾸밀 수 있다"는 인식을 유지한다.

| 획득 조건 | 보상 코스메틱 |
| :--- | :--- |
| Ancient 아이템 최초 획득 | 에르다 "심연 방랑자" 의상 |
| 아이템계 전 지층 클리어 (1개 아이템) | 에코 "지층의 빛" 이펙트 |
| 이노센트 100마리 복종 | 이노센트 "수집가" 칭호 뱃지 |
| Ancient 아이템계 전 지층 클리어 | 무기 스킨 "심층 각인" |

## 6. 바이럴 퍼널 설계 (Viral Funnel Design)

### 6.1. 퍼널 구조

```
[발견] SNS/Discord에서 URL 공유 링크 수신
    ↓
[즉시 플레이] 브라우저 열기 → 10초 내 플레이 시작 (설치 없음)
    ↓
[스파이크 체험] Tier 1 탐험 → 첫 아이템 획득 → 아이템계 진입 (약 15분)
    ↓
[갈망 형성] 1지층 클리어 → "2지층에는 뭐가 있어?" → Legendary 드랍 연출
    ↓
[전환 결정] Steam 위시리스트 등록 또는 즉시 구매
    ↓
[세이브 이전] 원클릭 세이브 이전 → Steam 버전에서 이어서 플레이
    ↓
[재바이럴] Steam 버전에서 아이템계 URL 생성 → 친구 초대 → 퍼널 재시작
```

### 6.2. URL 공유 즉시 플레이

Project Abyss의 핵심 바이럴 메커닉은 **URL 하나로 게임 시작**이다. 설치 없이, 로그인 없이, 즉시 플레이 가능한 것이 웹 버전의 최대 강점이다.

```yaml
url_sharing:
  base_share:
    url_format: "https://projectabyss.io/play"
    load_time_target: "< 5초 (첫 로딩)"
    login_required: false
    install_required: false

  item_world_invite:
    url_format: "https://projectabyss.io/join/{session_id}"
    description: "파티원이 현재 진행 중인 아이템계에 직접 합류"
    web_behavior: "웹 버전으로 진입 → 관전 또는 Tier 2 이하 솔로 구간 체험"
    steam_behavior: "Steam 버전으로 진입 → 파티에 즉시 합류"
    conversion_message: "이 파티에 합류하려면 Steam 버전이 필요합니다."

  social_card:
    auto_generate: true
    content: "아이템명 + 레어리티 + 아이템계 층위 스크린샷"
    platform: "Twitter/X, Discord 링크 프리뷰 최적화"
```

### 6.3. KPI 및 전환율 목표

| 지표 | 목표 | 측정 방법 |
| :--- | :--- | :--- |
| 웹 → 첫 플레이 전환율 | URL 클릭 대비 60%+ | 클라이언트 로그 |
| 첫 플레이 → 아이템계 진입 | 플레이 개시 대비 70%+ | 아이템계 진입 이벤트 |
| 아이템계 체험 → Steam 위시리스트 | 아이템계 클리어 대비 15%+ | Steam Wishlist API |
| Steam 위시리스트 → 구매 | 위시리스트 대비 25%+ | Steam 판매 데이터 |
| Steam 구매 → 친구 초대 | 구매자 대비 30%+ | URL 공유 이벤트 |
| D7 잔존율 (Steam) | 40%+ | 로그인 로그 |

**복합 전환율 목표:** URL 클릭 → Steam 구매 = 약 2~3% (업계 평균 1~2% 대비 높은 목표치. 스파이크 기반 바이럴에 의한 질 높은 트래픽 가정)

## 7. 금지 규칙 (Anti-Patterns)

모든 수익화 결정에서 아래 항목을 위반하는 것은 Project Vision의 핵심 금지 규칙과 동일한 효력을 갖는다. 어느 기둥에도 정렬되지 않는 수익화 기능은 절대 추가하지 않는다.

| # | 금지 항목 | 위반 시 훼손되는 가치 |
| :--- | :--- | :--- |
| 1 | **Pay-to-Win** (스탯 구매, 이노센트 직구매, 레어리티 직구매) | 야리코미 핵심 판타지 붕괴. 장인 판타지는 "내가 직접 만든 것"에 있다 |
| 2 | **에너지/스태미나 시스템** (플레이 시간 제한 과금) | 야리코미(무한 파밍)가 핵심인 게임에서 플레이 제한은 근본적 모순 |
| 3 | **확률형 아이템 박스 (가챠)** | 심리적 착취. 투명하지 않은 확률은 코어 유저의 신뢰를 파괴한다 |
| 4 | **자동 사냥 이용권** | 아이템계 직접 플레이가 핵심 판타지. 자동화로 대체하면 게임이 아니다 |
| 5 | **거래소/경매장** | 경제 인플레이션, 실력 차이 무력화, 봇/작업장 유발 |
| 6 | **시즌 패스 / 배틀 패스** | 시간 압박 구조. "완료하지 않으면 손해" 심리는 야리코미 자유도와 충돌 |
| 7 | **광고 삽입** (배너, 동영상, 보상형 광고) | 게임 몰입 파괴. 웹 게임이라도 광고 기반 수익화는 채택하지 않는다 |
| 8 | **VIP/월정액 스탯 부스트** | 동일 출발선 보장 원칙 위반. 야리코미 성취감은 조건 평등에서 출발한다 |
| 9 | **경험치 부스트 / 골드 부스트 판매** | 성장의 의미 희석. 시간을 돈으로 대체하는 구조는 Pay-to-Win의 변형이다 |
| 10 | **웹 버전 반복 광고성 팝업** | 단발 전환 유도 메시지 1회만 허용. 반복 팝업은 이탈을 유발한다 |

## 9. 레퍼런스 분석 (Reference Analysis)

### 9.1. Vampire Survivors — "무료 체험의 폭발력"

| 항목 | 내용 |
| :--- | :--- |
| 모델 | itch.io 무료 체험 → Steam $3 유료 |
| 결과 | Steam 1,000만 카피+ (2022~2023) |
| 핵심 교훈 | 낮은 진입 장벽이 바이럴을 만들었다. itch.io 무료 버전이 마케팅 비용 없이 SNS 확산을 이끌었다. $3의 낮은 가격은 구매 결정 마찰을 극소화했다 |
| Project Abyss 적용 | URL 즉시 플레이 = itch.io 무료 체험 기능. 단 가격은 $15로 더 높게 설정 — 콘텐츠 볼륨으로 정당화 |

### 9.2. Hollow Knight — "콘텐츠로 가격을 정당화"

| 항목 | 내용 |
| :--- | :--- |
| 모델 | Steam $15 일회성 구매 |
| 결과 | 1,000만 카피+ (2017~현재), 인디 게임 가격 대비 콘텐츠 압도 |
| 핵심 교훈 | "$15가 아깝지 않다"는 인식이 입소문을 만들었다. 무료 DLC 정책이 "이 개발자를 믿는다"는 신뢰를 쌓았다. 높은 품질이 곧 마케팅이었다 |
| Project Abyss 적용 | Steam 가격 기준선 $15. 아이템계 깊이와 야리코미 콘텐츠 볼륨으로 "가격 이상의 가치" 인식 구축 목표 |

### 9.3. Hades — "플랫폼 독점 → 점진적 확장"

| 항목 | 내용 |
| :--- | :--- |
| 모델 | Epic 독점 얼리 액세스 → Steam 출시 → 멀티 플랫폼 확장 |
| 결과 | 100만 카피+ (EA 기간), 정식 출시 후 급등 |
| 핵심 교훈 | EA 기간 동안 유저 피드백 기반 콘텐츠 수정이 정식 출시 품질을 높였다. EA 구매자를 "개발 파트너"로 대우하는 투명한 소통이 충성도를 만들었다 |
| Project Abyss 적용 | Steam EA 단계에서 플레이어 피드백을 반영한 공개 패치 노트 운영. EA 구매자에게 인게임 뱃지와 개발 참여 인식 부여 |

### 9.4. Among Us — "무료 모바일 → 유료 PC 전환"

| 항목 | 내용 |
| :--- | :--- |
| 모델 | 모바일 무료 → PC $5 유료 (2020 바이럴 이후) |
| 결과 | PC Steam 판매 폭증. 기존 $5 가격 그대로 유지 |
| 핵심 교훈 | 무료 버전이 게임을 "알리는" 역할을 하고, 유료 버전이 "더 편하게 하는" 역할을 했다. 무료와 유료의 경험 차이가 충분히 명확했다 |
| Project Abyss 적용 | 웹(무료)과 Steam(유료)의 경험 차이를 "콘텐츠 깊이"로 명확히 설정. 웹은 "더 깊어질 수 없다"는 한계를, Steam은 "끝이 없다"는 야리코미를 제공 |

## 10. 검증 체크리스트 (Validation Checklist)

### 10.1. 설계 검증

- [ ] 웹 무료 구간이 "스파이크(아이템계 진입)"를 충분히 전달하는가?
  - 검증 기준: 플레이테스터가 웹 버전만으로 "아이템계가 핵심"임을 인지 → 세션 내 아이템계 재진입 시도
- [ ] 무료 → 유료 전환 트리거가 자연스러운가?
  - 검증 기준: 전환 메시지에 대한 플레이테스터 반응이 "강요받는 느낌"이 아닌 "기대감" 위주
- [ ] 웹 무료 구간이 Steam 버전의 가치를 훼손하지 않는가?
  - 검증 기준: 웹 버전 플레이 후 "Steam 버전에 뭐가 더 있어?"라는 질문이 나오는가
- [ ] Pay-to-Win 요소가 설계 어디에도 없는가?
  - 검증 기준: 코드 리뷰 + 설계 문서 전수 검토로 스탯 영향 과금 항목 0건

### 10.2. 기술 검증

- [ ] URL 공유 → 즉시 플레이가 5초 내 시작되는가?
  - 검증 기준: Lighthouse 성능 점수 85+, 첫 인터랙티브 < 5초 (3G 기준)
- [ ] 웹 → Steam 세이브 이전이 원클릭으로 동작하는가?
  - 검증 기준: 계정 연동 이후 이전 시간 < 30초, 데이터 손실 0건
- [ ] 아이템계 URL 초대가 올바른 버전으로 분기하는가?
  - 검증 기준: 웹 사용자 → 관전 모드 또는 안내, Steam 사용자 → 즉시 파티 합류

### 10.3. 경험 검증 (플레이테스트)

- [ ] 15분 세션 내 "더 하고 싶다"는 반응이 나오는가?
  - 측정: 세션 종료 후 설문 "계속 플레이하고 싶었는가?" 80%+ YES
- [ ] 웹 버전 플레이 후 Steam 위시리스트 등록 의향이 있는가?
  - 측정: 플레이테스트 후 설문 "Steam 버전 구매 의향" 40%+
- [ ] 코스메틱이 Pay-to-Win처럼 느껴지지 않는가?
  - 측정: "코스메틱이 게임 플레이에 영향을 준다고 느꼈는가?" 5%- YES

## 설계 의도 (Design Intent)

### 분해 (Deconstruct)

| 참고 게임/모델 | 재미/효과 요소 |
| :--- | :--- |
| Vampire Survivors | 무료 체험이 바이럴을 만들고, 낮은 가격이 구매 마찰을 없앤다 |
| Hollow Knight | 콘텐츠 볼륨으로 가격을 정당화한다. "아깝지 않다"는 입소문이 마케팅이다 |
| Hades EA 모델 | EA 기간의 투명한 소통이 충성 유저를 만들고, 충성 유저가 정식 출시를 폭발시킨다 |

### 분석 (Analyze)

인디 게임이 성공하는 수익화의 핵심은 세 가지다. 첫째, 플레이어가 게임을 경험하기까지의 마찰을 최소화한다. 둘째, 경험의 질이 구매를 설득한다(광고가 아니라). 셋째, 구매한 사람이 다시 "전도사"가 된다.

Project Abyss의 스파이크(아이템계 진입)는 이 세 가지를 모두 충족할 수 있는 메커닉이다. URL 하나로 즉시 플레이 가능(마찰 최소화), 아이템계의 독창성이 "이 게임 미쳤다"는 반응 유발(경험의 질), 아이템계 URL 초대로 친구를 끌어들이는 구조(전도사 루프).
