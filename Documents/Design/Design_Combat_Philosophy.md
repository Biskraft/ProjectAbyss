## 1. 개요 (Overview)

### 1.1. 문서 목적

본 문서는 Project Abyss의 전투가 "어떤 느낌이어야 하는가", "왜 이렇게 설계하는가"를 정의하는 상위 철학 문서이다. System 문서(`System_Combat_Action.md`, `System_Combat_Damage.md` 등)가 구체적인 메커닉과 수치를 다룬다면, 이 문서는 그 메커닉들이 추구해야 할 방향과 원칙을 제시한다. 모든 전투 관련 System 문서의 설계 결정은 이 철학 문서의 원칙에 정렬되어야 한다.

### 1.2. 한 줄 요약

> "읽을 수 있고, 반응할 수 있고, 정복할 수 있는 전투"

플레이어는 적의 행동을 읽고(Read), 적절한 타이밍에 반응하며(React), 반복을 통해 전투를 정복한다(Master). 이 세 단계가 모든 전투 설계의 근간이다.

## 2. 설계 의도 (Design Intent)

### 2.1. 분해 (Deconstruct)

| 참고 게임 | 재미 요소 |
| :--- | :--- |
| 월하의 야상곡 | 무기별 고유 모션, 타격 시 히트스탑과 넉백, 적마다 다른 패턴, 보스전의 긴장감 |
| 디스가이아 | 반복 전투에서도 레벨/장비 성장이 체감되는 수치적 쾌감, 극한 육성 후 압도하는 카타르시스 |
| 할로우 나이트 | 정밀한 히트박스, 공격 예고 동작(Tell)의 일관성, 보스 패턴 학습의 성취감 |
| 데드 셀즈 | 빠른 전투 리듬, 무기 전환의 전략적 선택, 적 조합에 따른 전투 밀도 변화 |

### 2.2. 분석 (Analyze)

| 재미의 본질 | 리스크와 리턴 분석 |
| :--- | :--- |
| 타격감 | 공격이 "맞았다"는 감각이 명확할수록 전투 행위 자체가 보상이 된다. 시청각 피드백이 약하면 같은 수치의 전투도 무감각해진다 |
| 패턴 학습 | 적의 행동을 관찰하고 틈을 찾는 과정이 리스크(피격 위험)이며, 성공적 반격이 리턴(데미지 + 성취감)이다 |
| 성장 체감 | 장비 강화 전후의 데미지 차이가 숫자로 보이는 것이 야리코미의 핵심 동력이다 |
| 무기 선택 | 상황에 맞는 무기를 고르는 판단이 리스크(잘못된 선택 = 비효율)이며, 적합한 선택이 리턴(효율적 전투)이다 |

### 2.3. 재구축 (Rebuild)

| 독창적 변형 | Project Abyss만의 강점 |
| :--- | :--- |
| 타격감 + 접근성 | 월하의 타격감을 원터치 스킬 슬롯 체계에서 구현. 복잡한 커맨드 없이도 묵직한 전투 경험 제공 |
| 패턴 학습 + 아이템계 반복 | 보스 패턴을 익히면 아이템계 깊은 지층의 강화된 보스도 동일 패턴으로 대응 가능. 학습이 무한 성장 루프에서 지속적으로 보상 |
| 성장 체감 + 이중 게이트 | 데미지 숫자 상승이 곧 스탯 게이트 해금으로 연결. 전투 성장 = 탐험 범위 확장 |
| 무기 다양성 + 아이템계 순환 | 무기마다 다른 아이템계를 보유. 무기 교체 = 새로운 던전 경험 |

### 2.4. 3대 기둥 정렬 (Pillar Alignment)

| 기둥 | 전투 철학에서의 구현 |
| :--- | :--- |
| 메트로베니아 탐험 | 전투는 탐험의 장벽이자 보상. 새 층위의 적은 새로운 패턴을 제시하고, 정복은 탐험 진행의 증거가 된다 |
| 아이템계 야리코미 | 전투 효율이 장비 강화의 속도를 결정한다. 수동 전투의 숙련도가 곧 야리코미 깊이의 열쇠이다 |
| 온라인 멀티플레이 | 역할 분담(탱커/딜러/서포터)이 전투 깊이를 확장한다. 파티 전투에서 개인의 기여가 명확히 느껴져야 한다 |

## 3. 전투 미학 (Combat Aesthetics)

### 3.1. 타격감(Juice)의 3요소

전투의 "손맛"을 구성하는 세 가지 핵심 피드백 요소이다. 이 세 요소는 항상 함께 작동하며, 하나라도 빠지면 타격감이 현저히 저하된다.

#### 히트스탑: 타격의 "무게"

히트스탑은 공격이 적중한 순간 시간을 멈춰 타격의 무게감을 전달한다.

| 원칙 | 설명 |
| :--- | :--- |
| 비례의 법칙 | 공격의 위력이 강할수록 히트스탑이 길어진다. 플레이어는 정지 시간만으로 공격의 위력을 직감한다 |
| 양자 적용 | 공격자와 피격자 모두 정지한다. 공격자만 정지하면 부자연스럽고, 피격자만 정지하면 무게감이 사라진다 |
| 로직 분리 | 히트스탑 중에도 쿨다운, 버프 지속 시간 등 게임 로직은 정상 진행한다. 정지하는 것은 물리(이동, 중력)뿐이다 |

구체적인 프레임 수치는 `System_Combat_Action.md`의 히트스탑 섹션을 따른다.

#### 화면 흔들림: 충격의 "규모"

화면 흔들림은 타격의 충격이 게임 세계 전체에 파급되는 느낌을 준다.

| 원칙 | 설명 |
| :--- | :--- |
| 강도 비례 | 기본 공격은 미세한 흔들림, 보스 처치는 강렬한 흔들림. 일상적 공격에서 과도한 흔들림은 피로를 유발한다 |
| 방향성 | 공격 방향과 일치하는 방향으로 흔들림이 시작된다. 수평 공격이면 수평 흔들림이 주 성분이다 |
| 감쇠 곡선 | 흔들림은 시작 시 최대 진폭이고 시간에 따라 감쇠한다. 급격한 감쇠(빠른 수렴)가 묵직한 느낌을 준다 |

#### 넉백: 공격의 "힘"

넉백은 공격의 물리적 힘을 시각적으로 표현한다.

| 원칙 | 설명 |
| :--- | :--- |
| 무게 반비례 | 가벼운 적은 멀리 밀려나고, 무거운 적은 조금 밀려난다. 이 차이가 적의 "체급"을 체감하게 한다 |
| 방향 제어 | 넉백 방향은 공격자에서 피격자를 향하는 벡터를 따른다. 특수 공격(어퍼컷 등)은 수직 성분을 포함한다 |
| 감쇠와 정지 | 넉백 속도는 매 프레임 감쇠하며, 벽에 충돌하면 즉시 정지한다. 관성이 남아 미끄러지는 느낌은 배제한다 |

### 3.2. 시청각 피드백 철학

> **설계 원칙:** "모든 행동에는 반응이 있어야 한다"

플레이어의 모든 전투 행동은 즉각적인 시청각 피드백을 동반해야 한다. 피드백이 없는 행동은 플레이어에게 "입력이 무시되었다"는 불안감을 준다.

| 행동 | 시각 피드백 | 청각 피드백 |
| :--- | :--- | :--- |
| 공격 시전 | 무기 궤적 이펙트(트레일), 캐릭터 모션 | 무기별 고유 스윙 사운드 |
| 적중 | 히트 이펙트(스파크/혈흔), 히트스탑, 넉백, 데미지 넘버 | 무기별 고유 타격음, 적 피격음 |
| 빗나감 | 공격 모션 완료, 피드백 없음 | 스윙 사운드만 재생 (타격음 없음) |
| 피격 | 캐릭터 점멸, 붉은 플래시, 넉백 | 피격 사운드, 캐릭터 음성 |
| 크리티컬 | 확대된 히트 이펙트, 강화된 히트스탑, 확대 데미지 넘버 | 크리티컬 전용 타격음(높은 음역) |
| 적 처치 | 소멸 이펙트, 아이템 드랍 연출 | 처치 사운드, 경험치 획득음 |
| 보스 처치 | 히트스탑 + 카메라 줌인 + 강렬한 화면 흔들림 + 전용 처치 연출 | 전용 보스 처치 사운드 + BGM 전환 |

### 3.3. 월하의 야상곡 타격감 분석

Project Abyss의 타격감 레퍼런스로 월하의 야상곡을 채택한 이유와 적용 방향을 정리한다.

| 월하의 특성 | 분석 | Project Abyss 적용 |
| :--- | :--- | :--- |
| 무기별 모션 차이 | 검은 빠르고 가볍고, 대검은 느리지만 넓다. 무기 교체만으로 전투 체감이 완전히 달라진다 | 5종 무기 카테고리(검/대검/단검/활/지팡이) 각각에 고유한 콤보 모션, 히트박스, 타격음을 부여한다 |
| 짧은 히트스탑 | 1-2프레임의 미세한 정지가 빈번하게 발생하여 "경쾌하지만 묵직한" 이중 감각을 형성한다 | 기본 공격은 2-3프레임, 스킬은 3-4프레임, 크리티컬은 6프레임의 히트스탑을 적용한다 |
| 적의 피격 리액션 | 적마다 피격 시 고유한 리액션(흔들림, 후퇴, 점멸)이 있어 "이 적에게 데미지를 주고 있다"는 확신을 준다 | 적 유형별 피격 애니메이션을 분리하고, 무게 계수에 따른 넉백 차이를 명확히 한다 |

## 4. 전투 리듬 (Combat Rhythm)

### 4.1. 공격-회피-반격 순환의 파형 설계

전투는 긴장과 이완이 반복되는 파형이다. 이 파형이 단조로우면 전투가 지루해지고, 너무 빠르면 피로해진다.

```
[일반 전투 파형]

긴장 ──┐   ┌──┐   ┌──┐
        │   │  │   │  │
이완 ───┴───┘  └───┘  └───── 전투 종료

       접근  공격 회피 반격 정리   휴식
       (1초) (1초)(1초)(1초)(1초) (2-3초)
```

```
[보스전 파형]

긴장 ──┐      ┌────┐         ┌──────────┐
        │      │    │         │          │
이완 ───┴──────┘    └─────────┘          └── 처치
       관찰     회피+반격  패턴 전환 대응   페이즈2      카타르시스
       (3-5초)  (2-3초)   (2-3초)         (장기전)
```

### 4.2. Moment-to-Moment: 1-3초 단위의 긴장/이완

전투의 최소 단위는 1-3초이다. 이 시간 안에 플레이어는 하나의 의사결정을 완료한다.

| 단계 | 시간 | 플레이어 행동 | 감정 |
| :--- | :--- | :--- | :--- |
| 인식 | 0.3-0.5초 | 적의 행동을 읽는다 (Tell 확인) | 긴장 |
| 판단 | 0.2-0.3초 | 대시로 회피할지, 공격할지, 스킬을 쓸지 결정한다 | 집중 |
| 실행 | 0.5-1.0초 | 선택한 행동을 수행한다 (대시 i-frame 회피, 공격, 스킬) | 행동 |
| 결과 | 0.5-1.0초 | 성공/실패를 확인한다 (피드백 수신) | 쾌감/긴장 |

이 순환이 매끄럽게 반복되어야 "흐름(Flow)"에 진입할 수 있다. 판단 시간이 부족하면 불공정하게 느끼고, 너무 길면 지루해진다.

### 4.3. 일반 전투 vs 보스전의 리듬 차이

| 항목 | 일반 전투 | 보스전 |
| :--- | :--- | :--- |
| 전투 시간 | 3-10초 | 60-180초 |
| 의사결정 빈도 | 낮음 (공격 위주) | 높음 (회피/반격 반복) |
| 리스크 수준 | 낮음 (피격 1-2회 허용) | 높음 (피격 3-4회로 사망 위험) |
| 긴장/이완 비율 | 3:7 (이완 위주) | 7:3 (긴장 위주) |
| 핵심 감정 | 효율적 돌파의 쾌감 | 패턴 학습과 정복의 성취감 |

### 4.4. "숨 쉴 틈"의 중요성

연속 공격 사이에 반드시 휴식 프레임이 존재해야 한다. 숨 쉴 틈이 없는 전투는 플레이어를 지치게 만들고, 학습 기회를 박탈한다.

| 적용 대상 | 원칙 |
| :--- | :--- |
| 적 공격 패턴 | 연속 공격 후 반드시 빈 프레임(공격하지 않는 시간)을 배치한다. 이 틈이 플레이어의 반격 기회이다 |
| 보스 페이즈 전환 | 페이즈 전환 시 짧은 연출 시간을 두어 플레이어가 상황을 재파악할 수 있게 한다 |
| 적 웨이브 | 한 웨이브 처리 후 다음 웨이브 등장까지 짧은 공백을 둔다 |
| 3타 후딜 | 플레이어의 3타 콤보 완료 후 경직(`combo_end_lag_ms`)은 "공격의 여운"이자 "적의 반격 기회"이다. 이 후딜이 전투에 리듬감을 부여한다 |
| 대시 회피 | 적의 공격 Tell을 인식한 후 대시(i-frame)로 회피하는 행동이 전투의 핵심 긴장감을 만든다. 3타 후딜 중 대시 캔슬은 "공격 지속 vs 안전 이탈"의 의사결정을 생성한다 |

### 4.5. 대시의 전투 깊이 역할 (대시 렐릭 획득 후)

> **전제:** 대시는 기본 조작이 아니라 렐릭(능력 게이트)으로 해금되는 능력이다. 획득 전에는 점프만으로 회피해야 하며, 대시 획득 후 전투 깊이가 한 단계 확장된다.

대시는 단순한 이동 수단이 아니라 전투 리듬의 핵심 메커닉이다.

| 상황 | 대시의 역할 | 생성되는 의사결정 |
| :--- | :--- | :--- |
| 적 공격 Tell 인지 | i-frame 회피 | 회피 타이밍 판단 (너무 빠르면 적의 추적 공격에 당함) |
| 3타 후딜 중 | 후딜 캔슬 + 이탈 | 대시로 빠지기 vs 스킬 연계로 추가 딜 |
| 보스 페이즈 전환 | 안전 거리 확보 | 대시 쿨다운 관리 (2초 후 다음 위기에 대비) |
| 적 사이 돌파 | 적 관통 이동 | 포위 상황 탈출 vs 쿨다운 소비의 트레이드오프 |
| 공중 전투 | 공중 대시 (1회) | 공중 대시 타이밍 판단 (착지 전까지 재사용 불가) |

> **설계 원칙:** 대시 쿨다운(2초)은 "매 순간 대시를 아무 때나 쓸 수 있는 게 아니라, 적절한 순간에 써야 한다"는 자원 관리 긴장감을 만든다. 보스의 다단 히트 공격은 대시 1회로 완전 회피가 불가능하도록 설계하여, 대시 이후의 위치 선정까지 고려해야 하는 깊이를 확보한다.

### 4.6. 전투 자원 관리의 깊이 (Combat Resource Depth)

전투 자원은 "스킬을 아무 때나 쓸 수 없게 만드는 장치"이다. 자원 관리가 없으면 최강 스킬을 반복 사용하는 단조로운 전투가 된다.

#### MP + 쿨다운 이중 비용의 설계 의도

Project Abyss는 MP와 쿨다운을 동시에 적용하는 이중 비용 구조를 채택한다.

| 비용 축 | 역할 | 없을 때의 문제 |
| :--- | :--- | :--- |
| 쿨다운 | 짧은 시간 단위의 스킬 사용 간격을 강제한다. "지금 이 스킬을 쓸 수 있는가?"라는 순간적 판단을 생성한다 | 같은 스킬을 연타하여 전투가 단조로워진다 |
| MP | 긴 시간 단위의 자원 총량을 관리한다. "이 전투에서 스킬을 몇 번 더 쓸 수 있는가?"라는 전략적 판단을 생성한다 | 쿨다운만 기다리면 무한히 스킬을 사용할 수 있어 자원 관리 긴장감이 사라진다 |
| 이중 비용 조합 | 순간 판단(쿨다운)과 장기 전략(MP)이 동시에 작동한다. 쿨다운이 돌아왔더라도 MP가 부족하면 기본 공격으로 버텨야 한다 | - |

#### 3-Space별 MP 긴장감 차이

| 공간 | MP 회복 방식 | 긴장감 수준 | 설계 의도 |
| :--- | :--- | :--- | :--- |
| 월드 (World) | Out-of-Combat 자연 회복 + 포션 | 낮음 | 탐험 흐름을 끊지 않기 위해 MP가 자연 회복된다. MP 관리보다 탐험 진행이 우선이다 |
| 아이템계 (Item World) | 포션 + 미스터리 룸(병원) + 이노센트만 가능. 자연 회복 없음 | 높음 | 모든 지층을 관통하는 자원 관리가 야리코미의 핵심 긴장감이다. "스킬을 아껴야 할까, 지금 전력을 다해야 할까?"라는 판단이 매 지층마다 발생한다 |
| 세이브 포인트 | 전투 없음. MP 전회복 | 없음 | 안전한 휴식 공간. 아이템계 진입 전 MP 관리 부담 없이 준비에 집중한다 |

#### 자원 관리와 전투 전략

MP 관리 능력은 전투의 전략적 깊이를 만드는 핵심 요소이다.

| 항목 | 설명 |
| :--- | :--- |
| MP 효율 | 플레이어가 상황에 맞는 스킬을 선택하여 MP를 효율적으로 사용한다 |
| MP 고갈 대응 | MP 부족 시 기본 공격과 대시(렐릭 획득 후)를 활용한 전술적 전투를 지속한다 |
| 포션 관리 | 보스전 직전에 MP를 온존하는 등 전략적 포션 사용이 가능하다 |

> 설계 원칙: MP는 "스킬을 쓸지 아낄지"라는 의사결정을 전투에 추가한다. 쿨다운만으로는 "쿨다운이 돌면 쓴다"는 기계적 판단만 남지만, MP가 더해지면 "지금 쓸 여유가 있는가?"라는 전략적 판단이 생긴다. 이 전략적 판단이 숙련 플레이어와 초보의 차이를 만든다.

## 5. 적 조우 설계 (Enemy Encounter Design)

### 5.1. 전투 아레나 진입/이탈 흐름

전투는 "시작"과 "끝"이 명확해야 한다. 전투 아레나는 탐험과 전투의 경계를 정의한다.

| 단계 | 설명 |
| :--- | :--- |
| 접근 | 플레이어가 적의 존재를 인식한다. 적의 시각적 실루엣과 움직임이 구별되어야 한다 |
| 진입 | 적의 감지 범위에 진입하면 적이 활성화된다. 갑작스러운 활성화는 최소화하고, 플레이어가 먼저 인식할 수 있는 여유를 준다 |
| 교전 | 전투 상태(In-Combat)로 전환된다. HP 자동 회복이 비활성화되고 전투 집중 모드에 진입한다 |
| 정리 | 모든 적을 처치하면 보상(아이템 드랍, 경험치)이 즉시 지급된다 |
| 이탈 | 전투 종료 후 Out-of-Combat로 전환되며, 탐험 모드로 복귀한다 |

### 5.2. 적 조합 철학: 역할 분담

적 그룹은 단일 유형으로 구성하지 않는다. 역할이 다른 적을 조합하여 플레이어에게 우선순위 판단을 요구한다.

| 적 역할 | 전투 기능 | 플레이어에 대한 압박 |
| :--- | :--- | :--- |
| 근접형 (Melee) | 플레이어에게 접근하여 근거리 공격 | 거리 관리 압박. 뒤로 밀리면 다른 적과 끼일 위험 |
| 원거리형 (Ranged) | 안전 거리에서 투사체 발사 | 회피 행동 강제. 근접 적과 동시에 상대할 때 판단 복잡도 증가 |
| 탱커형 (Tank) | 높은 HP와 슈퍼 아머로 전선 유지 | 시간 압박. 빠르게 처리하지 못하면 다른 적이 합류 |
| 자폭형 (Kamikaze) | 플레이어에게 돌진, 접촉 시 폭발 | 즉각적 위협. 최우선 제거 대상 판단을 강제 |
| 서포터형 (Support) | 아군 적에게 버프 부여 또는 회복 | 전략적 판단. 서포터를 먼저 처리할지, 앞의 적을 먼저 처리할지 선택 |

기본 조합 원칙: 근접형 1-2마리 + 원거리형 1마리의 기본 조합에서 시작하여, 층위 난이도에 따라 탱커형이나 서포터형을 추가한다.

### 5.3. "읽을 수 있는 적": 공격 예고 동작(Tell)

모든 적의 공격에는 반드시 예고 동작(Tell)이 존재해야 한다. Tell은 플레이어가 적의 공격을 읽고 대응할 시간을 보장하는 장치이다.

| Tell 원칙 | 설명 |
| :--- | :--- |
| 시각적 명확성 | 예고 동작은 통상 동작과 확연히 구별되어야 한다. 팔을 크게 뒤로 젖히는 동작, 몸체 발광, 자세 변화 등 |
| 충분한 반응 시간 | Tell 시작부터 피해 판정까지 최소 0.3초 이상의 시간을 보장한다. 이 시간 안에 플레이어가 회피 또는 거리 확보를 결정할 수 있어야 한다 |
| 공격 유형 예고 | Tell의 형태가 공격 유형을 암시해야 한다. 수평으로 팔을 펼치면 넓은 범위 공격, 위로 들어올리면 내려찍기 등 |
| 일관성 | 동일한 Tell은 항상 동일한 공격으로 연결된다. 같은 예고 동작이 때로는 A 공격, 때로는 B 공격으로 이어지면 신뢰가 무너진다 |

## 6. 보스전 철학 (Boss Battle Philosophy)

### 6.1. 보스의 3가지 역할

모든 보스는 다음 세 가지 역할 중 최소 하나, 가능하면 전부를 수행해야 한다.

| 역할 | 설명 | Project Abyss 적용 |
| :--- | :--- | :--- |
| 시험관 (Test) | 플레이어가 해당 층위에서 습득한 능력과 전투 기술을 시험한다 | 월드 보스는 해당 층위에서 획득한 능력을 활용해야 효율적으로 공략할 수 있도록 설계한다 |
| 서사 장치 (Narrative) | 세계관의 핵심 정보를 전달하거나, 스토리의 전환점이 된다 | 보스 처치 후 능력 렐릭 획득과 함께 세계관 단편을 전달한다 |
| 페이스 조절자 (Pace) | 탐험의 흐름에 "정점"을 만들어 감정 곡선에 변화를 준다 | 긴 탐험 구간 후에 보스를 배치하여 긴장감의 최고점을 형성한다 |

### 6.2. 패턴 학습과 마스터리 곡선

보스전은 "처음에는 불가능해 보이지만, 시도할수록 가능해지는" 경험을 제공해야 한다.

| 단계 | 플레이어 상태 | 설계 목표 |
| :--- | :--- | :--- |
| 1차 조우 | 패턴을 모르고, 피격을 많이 당한다 | 사망하더라도 "다음엔 저 공격을 피할 수 있겠다"는 학습 단서를 남겨야 한다 |
| 반복 도전 | 일부 패턴을 기억하고, 회피 성공률이 높아진다 | 회피 성공 시 명확한 반격 기회를 제공하여 "성장하고 있다"는 감각을 준다 |
| 정복 | 모든 패턴을 숙지하고, 안정적으로 공략한다 | 보스 체력을 깎는 과정이 "지배하고 있다"는 카타르시스를 제공해야 한다 |

### 6.3. 페이즈 전환: 위기감 - 적응 - 정복

보스는 HP 구간에 따라 행동 패턴이 변화하며, 각 페이즈 전환이 새로운 긴장을 제공한다.

| 페이즈 | HP 구간 | 전투 성격 | 플레이어 감정 |
| :--- | :--- | :--- | :--- |
| 페이즈 1 | 100%-50% | 기본 패턴. 학습용. 공격 간격이 넉넉하다 | 관찰, 학습 |
| 전환 연출 | 50% 도달 시 | 보스 형태 변화 또는 분노 연출. 짧은 무적 구간 | 위기감, 긴장 고조 |
| 페이즈 2 | 50%-0% | 강화 패턴. 새 공격 추가, 공격 간격 단축 | 적응, 집중 |
| 처치 연출 | HP 0 | 히트스탑 + 카메라 연출 + 보상 드랍 | 정복의 카타르시스 |

### 6.4. 보스 처치의 카타르시스 연출

보스 처치 순간은 전투의 최고 정점이다. 이 순간의 연출이 약하면 보스전 전체의 인상이 희미해진다.

| 연출 요소 | 설명 |
| :--- | :--- |
| 확장된 히트스탑 | 최종 타격 시 통상 히트스탑보다 긴 정지를 적용한다. "결정적 일격"의 무게를 극대화한다 |
| 카메라 줌인 | 최종 타격 순간 카메라가 보스에게 줌인하여 처치의 순간을 강조한다 |
| 강렬한 화면 흔들림 | 보스 소멸과 함께 강한 화면 흔들림이 발생한다 |
| BGM 전환 | 전투 BGM이 정지되고, 승리 팡파르 또는 정적이 찾아온다. 청각적 대비로 해방감을 제공한다 |
| 보상 연출 | 보스 처치 후 능력 렐릭 또는 아이템이 화면 중앙에서 천천히 드랍된다. 보상 획득의 의식감(Ritual)을 부여한다 |

### 6.5. 아이템계 보스의 특수성

아이템계 보스(아이템 장군, 왕, 신, 대신)는 월드 보스와 다른 설계 철학을 따른다.

| 항목 | 월드 보스 | 아이템계 보스 |
| :--- | :--- | :--- |
| 등장 빈도 | 층위당 1체 (희소) | 각 지층마다 1체 (빈번) |
| 전투 시간 | 60-180초 (장기전) | 30-90초 (짧은 전투) |
| 패턴 복잡도 | 높음 (고유 패턴) | 중간 (보스 등급별 공유 패턴 + 변형) |
| 핵심 역할 | 서사 + 시험 + 페이스 조절 | 진행 확인 + 보상 지급 |
| 반복 대응 | 첫 정복의 성취감 중심 | 효율적 처리의 만족감 중심 |

## 7. 무기 다양성 원칙 (Weapon Diversity)

### 7.1. 무기 간 차별화의 3축

모든 무기는 다음 세 가지 축에서 서로 다른 위치를 점유해야 한다. 두 무기가 세 축 모두에서 유사하다면, 하나는 존재 이유가 없다.

| 축 | 정의 | 극단 예시 |
| :--- | :--- | :--- |
| 사거리 (Range) | 공격이 도달하는 거리 | 단검(최단) vs 활(최장) |
| 속도 (Speed) | 공격 모션의 빠르기 (DPS와 직결) | 대검(최저) vs 단검(최고) |
| 범위 (Area) | 한 번의 공격이 커버하는 면적 | 단검(최소, 직선) vs 대검(최대, 넓은 호) |

### 7.2. 플레이스타일 분기

무기 선택이 곧 플레이스타일을 결정한다. 무기의 3축 배치가 자연스럽게 전투 방식을 유도한다.

| 플레이스타일 | 특성 | 해당 무기 경향 |
| :--- | :--- | :--- |
| 공격적 (Aggressive) | 적에게 밀착하여 높은 DPS로 제압 | 단검, 검 (짧은 사거리, 빠른 속도) |
| 방어적 (Defensive) | 안전 거리를 유지하며 꾸준히 딜링 | 활, 지팡이 (긴 사거리, 원거리) |
| 기술적 (Technical) | 넓은 범위와 강력한 일격으로 다수 적 처리 | 대검 (넓은 범위, 충격파) |

### 7.3. 무기 전환의 전략적 깊이

장비 교체 자체가 전략적 선택이 되어야 한다.

| 원칙 | 설명 |
| :--- | :--- |
| 상황 적합성 | 좁은 통로에서는 창의 직선 공격이 유리하고, 넓은 아레나에서는 대검의 범위 공격이 유리하다. 지형과 적 조합에 따라 최적 무기가 달라진다 |
| 아이템계 연계 | 각 무기는 고유한 아이템계를 보유한다. 무기 교체는 "이 무기를 더 강화할지, 저 무기를 새로 키울지"라는 성장 전략의 선택이기도 하다 |
| 원소 매칭 | 무기에 부여된 원소와 적의 약점을 매칭하는 것이 전략적 무기 전환의 핵심 동기이다 |

무기별 구체적 모션, 히트박스, 피해 배율은 `System_Combat_Action.md`의 무기별 기본 공격 모션 섹션과 `Content_Stats_Weapon_List.csv`를 따른다.

## 8. 공정한 전투 (Fair Combat)

### 8.1. 핵심 원칙: "죽음은 플레이어의 실수"

플레이어가 사망했을 때 "내가 실수했다"고 납득할 수 있어야 한다. "게임이 불공정했다"고 느끼는 사망은 이탈의 가장 큰 원인이다.

### 8.2. 공정성 보장 규칙

| 규칙 | 설명 |
| :--- | :--- |
| 히트박스 정직성 | 공격의 시각적 이펙트와 실제 히트박스가 일치해야 한다. 이펙트 밖인데 맞거나, 이펙트 안인데 빗나가면 신뢰가 무너진다 |
| Tell 보장 | 모든 적의 공격에는 예고 동작이 있어야 한다. Tell 없는 즉발 공격은 반응이 불가능하므로 금지한다 |
| 화면 밖 공격 금지 | 적은 플레이어의 시야(카메라 프레임) 밖에서 공격을 시작하지 않는다. 시야 밖의 적은 비활성화 상태를 유지한다 |
| 넉백 회복 시간 보장 | 피격 후 무적 시간(`invincibility_after_hit_ms`) 동안 추가 피격을 받지 않는다. 무한 콤보(적이 플레이어를 영구히 경직시키는 상황)를 방지한다 |
| 슈퍼 아머 파괴 보상 | 슈퍼 아머를 깨뜨린 플레이어에게 명확한 반격 기회(무방비 시간)를 보장한다. 수고에 대한 리턴이 있어야 한다 |
| 사망 원인 가독성 | 플레이어가 "왜 죽었는지" 이해할 수 있어야 한다. 데미지 넘버, 피격 이펙트, 사망 화면에서 원인 정보를 제공한다 |

### 8.3. 히트박스 설계 원칙

| 원칙 | 설명 |
| :--- | :--- |
| 관대한 플레이어 허트박스 | 플레이어의 허트박스(피격 판정)는 시각적 캐릭터보다 약간 작게 설정한다. "아슬아슬하게 피한" 경험을 자주 만들기 위함이다 |
| 정직한 적 히트박스 | 적의 공격 히트박스는 시각적 이펙트와 정확히 일치하거나 약간 작게 설정한다. 이펙트보다 큰 히트박스는 금지한다 |
| AABB 단순성 | 모든 히트 판정은 AABB(축 정렬 경계 상자) 방식으로 처리한다. 복잡한 다각형 판정은 성능과 예측 가능성을 모두 저해한다 |

## 9. 3-Space별 전투 성격 차이 (Combat by Space)

### 10.1. 월드 (World) 전투

| 항목 | 설계 방향 |
| :--- | :--- |
| 전투 밀도 | 낮음-중간. 탐험이 주목적이며, 전투는 탐험 흐름의 일부이다 |
| 전투 목적 | 경험치/아이템 획득, 보스 처치를 통한 능력 해금, 층위 정복 |
| 지형 활용 | 고저차, 좁은 통로, 수중 등 지형이 전투에 직접 영향을 준다. 높은 곳에서 하방 공격, 좁은 통로에서 대검 충격파 활용 등 |
| 적 배치 | 층위 테마와 일치하는 적이 등장한다. 적 조합이 층위의 난이도를 정의한다 |
| 사망 페널티 | 세이브 포인트 복귀. 가혹하지 않지만 탐험 진행을 되돌리므로 긴장감을 유지한다 |
| 핵심 감정 | "이 층위를 정복하고 있다"는 탐험가의 성취감 |

### 10.2. 아이템계 (Item World) 전투

| 항목 | 설계 방향 |
| :--- | :--- |
| 전투 밀도 | 높음. 지층 전체가 전투 공간이다. 적 조우 간격이 짧다 |
| 전투 목적 | 효율적 전투를 통한 빠른 지층 클리어, 이노센트 사냥, 아이템 경험치 축적 |
| 지형 활용 | 절차적 생성 지형에 따라 전투 공간이 매번 달라진다. 지형 적응력이 전투 효율에 영향을 준다 |
| 적 배치 | 지층 깊이에 따라 적 강도가 스케일링된다. 각 지층의 보스로 아이템 장군/왕/신/대신이 등장한다 |
| 사망 페널티 | 진행 지층 손실. 탈출 아이템을 사용하지 않으면 클리어한 지층도 잃을 수 있어 긴장감이 높다 |
| 핵심 감정 | "효율적으로 돌파하여 장비를 강화하고 있다"는 장인의 파밍 쾌감 |

### 10.3. 세이브 포인트 (Rest Area)

세이브 포인트에서는 전투가 발생하지 않는다. 세이브 포인트는 안전한 공간으로, 전투의 긴장에서 벗어나 휴식하고 준비하는 곳이다. 이 대비가 월드와 아이템계의 전투 긴장감을 강화한다.

## 11. 저주받은 문제 (Cursed Problems)

| 문제 | 상충 구조 | 해결 방향 |
| :--- | :--- | :--- |
| 타격감 vs ���작 단순화 | 묵직한 타격감은 복잡한 조작(커맨드, 타이밍)에서 나오지만, 접근성 높은 설계는 단순 조작을 요구한다 | 타격감의 원천을 "입력 복잡도"가 아닌 "피드백 품질"(히트스탑, 화면 흔들림, 넉백, 사운드)에 집중한다. 원터치 스킬이라도 피드백이 풍부하면 타격감을 확보할 수 있다 |
| 패턴 학습 보스 vs 반복 파밍 보스 | 월드 보스는 학습과 정복의 성취감이 핵심이지만, 아이템계 보스는 반복 처치 효율이 핵심이다 | 월드 보스와 아이템계 보스의 설계 철학을 분리한다. 월드 보스는 고유 패턴과 서사를, 아이템계 보스는 등급별 공유 패턴과 빠른 전투를 추구한다 |
| 공정한 전투 vs 야리코미 압도감 | 적이 항상 공정하면 충분히 성장한 뒤에도 긴장해야 하지만, 야리코미의 쾌감은 "압도적으로 쓸어버리는" 데 있다 | 이미 정복한 층위(월드 기존 층위, 아이템계 저층)에서는 플레이어의 성장에 따라 적이 상대적으로 약해져 "압도감"을 체험한다. 새 층위/고층에서는 항상 도전적인 전투를 유지한다 |
| 솔로 밸런스 vs 파티 밸런스 | 솔로로도 모든 콘텐츠를 클리어할 수 있어야 하지만, 파티 플레이에도 의미가 있어야 한다 | 솔로 기준으로 난이도를 설계하고, 파티 시 적 HP 스케일링을 적용한다. 파티의 이점은 "더 쉬움"이 아니라 "더 높은 효율(보상 배율)"과 "더 깊은 전략(역할 분담)"이다 |
| MP 고갈의 좌절감 vs 자원 관리 긴장감 | MP가 부족하면 스킬을 못 써서 답답하지만, MP가 무한하면 전략적 깊이가 사라진다 | Out-of-Combat MP 회복과 포션으로 월드에서의 좌절감을 최소화한다. 아이템계에서는 의도적으로 자원 관리 긴장감을 유지하되, 미스터리 룸(병원)이 완충 장치 역할을 한다 |

## 13. 정합성 노트 (Alignment Notes)

기존 System 문서와의 정합성을 검토한 결과를 기록한다.

### 13.1. System_Combat_Action.md와의 정합성

| 항목 | 상태 | 비고 |
| :--- | :--- | :--- |
| 히트스탑 프레임 수치 | 정합 | 본 문서의 "비례의 법칙"과 Action 문서의 약/중/강/극강 4단계 히트스탑이 일치한다 |
| 넉백 무게 반비례 | 정합 | 본 문서의 "무게 반비례" 원칙과 Action 문서의 `actual_knockback = knockback_force / weight` 공식이 일치한다 |
| 히트스탑 중 로직 진행 | 정합 | 본 문서의 "로직 분리" 원칙과 Action 문서의 "물리만 정지, 게임 로직 정상 진행" 규칙이 일치한다 |
| 3타 후딜의 의미 | 정합 | 본 문서의 "숨 쉴 틈" 원칙과 Action 문서의 `combo_end_lag_ms`(600ms) 경직이 일치한다. 다만 Action 문서에 "3타 후딜이 적의 반격 기회이기도 하다"는 철학적 의도를 명시하면 설계 의도가 더 명확해질 것이다 |
| 콤보 보너스 부재 | 주의 필요 | Action 문서에서 "별도의 콤보 카운터나 콤보 보너스 시스템은 두지 않는다"고 명시했으나, 본 문서의 전투 리듬 관점에서 스킬 연계(원소 시너지)가 사실상 콤보 보너스 역할을 한다. Action 문서의 해당 규칙에 "스킬 시너지가 콤보 보너스를 대체한다"는 보충 설명을 추가하면 혼동을 방지할 수 있다 |

| 데미지 넘버 "약점/저항" 표시 | 정합 | 본 문서의 "읽을 수 있는 전투" 원칙과 Damage 문서의 "WEAK!"/"RESIST" 텍스트 표시가 일치한다. 적의 약점을 시각적으로 확인할 수 있어 무기 전환 전략을 지원한다 |
