## 1. 개요 (Overview)

### 한 줄 요약

> "메트로베니아의 고독한 탐험과 온라인 협동 파밍, 그리고 소셜 교류를 양립시키기 위해 게임 세계를 월드/아이템계/허브 세 공간으로 분리한다."

### 문서 목적

이 문서는 3-Space 분리 모델이 "왜" 이렇게 설계되었는지의 철학적 기반을 정의한다. 각 공간이 존재하는 이유, 공간 분리가 해결하는 설계 문제, 공간 간 전환이 만들어내는 감정 리듬, 그리고 공간 간 연결이 순환 구조와 맺는 관계를 다룬다.

System 문서들이 "어떻게 작동하는가"를 다룬다면, 이 문서는 "왜 이렇게 나누었는가"를 다룬다. 모든 시스템 설계자는 새로운 메커닉을 추가하기 전에 이 문서의 원칙을 확인하여, 3-Space 구조를 훼손하지 않는지 검증해야 한다.

### 이 문서가 다루지 않는 것

- 각 공간의 구체적 메커닉 사양 (System 문서 영역)
- 공간별 수치 데이터 (`Sheets/` CSV 영역)
- 순환 구조의 구체적 메커닉 (`Design_CoreLoop_Circulation.md` 영역)

## 2. 설계 의도 (Design Intent)

### 분해 (Deconstruct)

3-Space 모델은 세 가지 레퍼런스 게임의 핵심 경험을 분해하는 것에서 출발한다.

| 레퍼런스 | 핵심 경험 | 공간적 특성 |
| :--- | :--- | :--- |
| 월하의 야상곡 | 혼자서 거대한 성을 탐험하며 능력을 획득하는 고독한 발견의 쾌감 | 단일 플레이어, 핸드크래프트 맵, 비선형 탐험 |
| 디스가이아 | 아이템 속으로 들어가 끝없이 파고드는 야리코미의 중독성 | 절차적 생성, 반복 가능한 던전, 무한 성장 |
| 스펠렁키 | 매번 다른 절차적 레벨에서 긴장감 있는 도전을 반복하는 쾌감 | 절차적 생성, 높은 리스크, 숙련 보상 |

여기에 Project Abyss만의 요구사항인 온라인 멀티플레이가 추가된다. 이 네 가지 요소를 하나의 공간에 담으려 하면 필연적으로 충돌이 발생한다.

### 분석 (Analyze)

각 경험이 "왜" 재미있는지를 분석하면, 서로 양립하기 어려운 조건들이 드러난다.

| 경험 | 재미의 본질 | 필수 조건 | 충돌 지점 |
| :--- | :--- | :--- | :--- |
| 메트로베니아 탐험 | 혼자만의 발견, 경이감, 미지의 공포 | 솔로 플레이, 핸드크래프트 맵, 비가역적 진행감 | 다른 플레이어가 보이면 "미지"가 사라진다 |
| 야리코미 파밍 | "한 층만 더"의 중독, 끝없는 성장 | 반복 가능한 절차적 콘텐츠, 무한 성장 루프 | 핸드크래프트 맵에서는 콘텐츠 고갈이 발생한다 |
| 온라인 협동 | 역할 분담, 위기 극복의 유대감 | 다인 플레이, 실시간 상호작용 | 솔로 탐험의 고독감을 훼손한다 |
| 소셜 교류 | 자랑, 거래, 공동체 소속감 | 안전한 공간, 다수의 플레이어 | 전투/탐험의 긴장감과 양립하지 않는다 |

핵심 통찰: 이 네 가지 경험은 각각 다른 플레이어 수, 다른 맵 유형, 다른 감정 상태를 요구한다. 하나의 공간에서 모두 충족시키려는 시도는 모든 경험을 희석시킨다.

### 재구축 (Rebuild)

해법은 "공간을 나누는 것"이다. 각 경험이 요구하는 조건을 완벽히 충족하는 전용 공간을 설계하고, 공간 사이를 순환 구조로 연결하여 하나의 게임으로 통합한다.

| 공간 | 담당 경험 | 독창적 변형 |
| :--- | :--- | :--- |
| 월드 (World) | 메트로베니아 탐험 | 스탯 게이트를 추가하여 아이템계와의 순환을 생성 |
| 아이템계 (Item World) | 야리코미 파밍 + 온라인 협동 | 메트로베니아 능력을 아이템계 내에서도 활용 |
| 허브 (Hub) | 소셜 교류 + 준비 | 이노센트 팜, 거래소 등 순환을 촉진하는 시설 배치 |

이것이 3-Space 분리 모델의 핵심이다. 분리 자체가 목적이 아니라, 분리를 통해 각 경험의 순도를 보장하면서도 순환 구조로 하나의 게임 경험을 만들어내는 것이 목적이다.

## 3. 왜 3개의 공간인가

### 공간 분리의 근본 원칙

> **원칙:** 서로 다른 감정 상태를 요구하는 경험은 같은 공간에 공존할 수 없다.

영화관에서 영화를 보는 사람과 수다를 떠는 사람이 같은 공간에 있을 수 없는 것과 같다. 탐험가의 고독한 경이감과 파티원들의 활기찬 협동은 같은 공간에서 동시에 성립하지 않는다.

3-Space는 이 원칙의 최소 구현이다. 2개로는 부족하고(탐험+파밍을 분리하면 소셜 공간이 없다), 4개 이상은 공간 간 전환 비용이 증가하여 순환의 흐름이 끊긴다.

### 각 공간의 존재 이유

#### 월드 (World) - "저기 뭔가 있다"

존재 이유: 메트로베니아 탐험감은 "혼자서" 미지의 공간을 발견하는 경험에서 나온다. 다른 플레이어가 이미 지나간 흔적이 보이면, 미지는 더 이상 미지가 아니다.

월드가 솔로 공간이어야 하는 이유:

- 탐험의 핵심 감정은 "나만 이것을 발견했다"는 독점적 발견감이다
- 핸드크래프트 맵의 비밀은 한 번 발견하면 소진된다. 혼자 발견해야 가치가 보존된다
- 능력 게이트("아직 못 간다" -> "드디어 갈 수 있다!")는 개인의 성장 서사이다
- 고딕 세계의 분위기(고독, 경이, 공포)는 파티 채팅과 양립하지 않는다

월드가 해결하는 설계 문제:

| 문제 | 월드의 해법 |
| :--- | :--- |
| 멀티플레이 환경에서 탐험감 보존 | 월드를 솔로 전용으로 격리 |
| 핸드크래프트 맵의 콘텐츠 고갈 | 시즌 리셋 + 절차적 마이크로 구조로 재방문 가치 유지 |
| 탐험과 파밍의 목적 충돌 | 월드는 탐험과 발견에 집중, 파밍은 아이템계로 분리 |

#### 아이템계 (Item World) - "한 층만 더"

존재 이유: 야리코미(끝없는 파밍)는 무한히 반복 가능한 콘텐츠를 요구한다. 핸드크래프트 맵으로는 불가능하며, 절차적 생성만이 이를 충족한다. 동시에, 깊은 층의 도전은 혼자서는 불가능한 난이도를 제공하여 협동 플레이의 동기를 자연스럽게 만든다.

아이템계가 협동 공간이어야 하는 이유:

- 깊은 층(70층 이상)의 난이도는 파티 구성과 역할 분담을 요구한다
- "함께 위기를 극복했다"는 유대감은 높은 리스크 환경에서 극대화된다
- 파티 보상 보너스가 사회적 플레이를 유인한다
- 절차적 생성 맵은 매번 새로우므로, 파티원과 공유해도 발견감이 소진되지 않는다

아이템계가 해결하는 설계 문제:

| 문제 | 아이템계의 해법 |
| :--- | :--- |
| 무한 파밍 콘텐츠 부족 | 모든 장비에 100층 던전 내장 + 재귀적 진입 |
| 멀티플레이의 반복 동기 | 매번 다른 절차적 맵 + 파티 보상 보너스 |
| 장비 성장의 체감 부족 | 직접 아이템 속에 들어가 강화하는 물리적 행위감 |

#### 허브 (Hub) - "모험 전 한숨 돌리기"

존재 이유: 긴장과 도전만으로는 플레이어가 지친다. 안전한 공간에서 숨을 고르고, 다음 모험을 준비하며, 다른 플레이어와 교류하는 시간이 게임 경험의 리듬을 완성한다.

허브가 안전한 소셜 공간이어야 하는 이유:

- 사망 페널티가 없는 공간이 있어야 플레이어가 심리적으로 "쉴 수 있다"
- 장비 자랑, 거래, 파티 모집 등 사회적 활동은 전투 중에 할 수 없다
- 이노센트 팜, 대장간, 상점 등 준비 시설은 안전한 환경에서만 의미가 있다
- 허브에서의 교류가 "다음에 같이 아이템계 가자"라는 멀티플레이 동기를 생성한다

허브가 해결하는 설계 문제:

| 문제 | 허브의 해법 |
| :--- | :--- |
| 전투/탐험 피로 | 안전한 휴식 공간 제공 |
| 소셜 기능과 게임플레이 충돌 | 소셜 활동을 전용 공간으로 분리 |
| 순환 구조의 "준비" 단계 부재 | 시설을 통해 다음 순환을 촉진 |

## 4. 각 공간의 정체성과 감정적 목표

### 감정 설계 원칙

> **원칙:** 각 공간은 고유한 감정 색채를 가져야 하며, 공간에 진입하는 순간 플레이어는 감정 상태의 전환을 느껴야 한다.

이 전환이 명확할수록 각 공간의 정체성이 강해지고, 공간 간 이동이 의미 있는 경험이 된다.

### 공간별 감정 프로파일

| 항목 | 월드 | 아이템계 | 허브 |
| :--- | :--- | :--- | :--- |
| 핵심 감정 | 경이감, 고독, 발견의 전율 | 집중, 긴장, 성장의 쾌감 | 안도, 따뜻함, 소속감 |
| 플레이어 내면의 독백 | "저기 뭔가 있다... 갈 수 있을까?" | "한 층만 더... 이번엔 이노센트 나올 거야" | "드디어 돌아왔다. 장비 정리하고 다음 준비하자" |
| 시각 톤 | 어둡고 신비로운 고딕 (월하의 야상곡) | 추상적이고 몰입적인 던전 (레어리티별 테마 변화) | 따뜻하고 경쾌한 마을 (디스가이아) |
| 청각 톤 | 고딕 오케스트라 + 앰비언트 | 긴장감 있는 전투 루프 (깊을수록 BPM 증가) | 편안한 마을 테마 |
| 긴장도 | 중-높음 (미지에 대한 기대와 불안) | 높음 (사망 시 진행 손실) | 낮음 (사망 페널티 없음) |
| 리스크 수준 | 중간 (세이브 포인트 복귀) | 높음 (층 진행 손실, 탈출 아이템 소모) | 없음 |

### 감정 곡선과 공간 전환

플레이어가 한 세션 동안 경험하는 감정의 흐름:

```
[허브: 안도/준비] ──진입──> [월드: 긴장/호기심] ──탐험──> [월드: 발견/경이]
──보스──> [월드: 도전/성취] ──아이템 획득──> [아이템계: 집중/파밍]
──깊은 층──> [아이템계: 스릴/위험] ──클리어──> [아이템계: 성취/환희]
──귀환──> [허브: 안도/자랑/교류] ──준비──> [다음 순환]
```

이 곡선에서 핵심은 "이완-긴장-이완"의 반복이다. 허브에서 긴장을 풀고, 월드에서 긴장을 높이고, 아이템계에서 최고조에 달한 뒤, 다시 허브로 돌아와 이완한다. 이 리듬이 무너지면 플레이어는 피로감을 느끼거나 몰입이 끊긴다.

## 5. 공간 분리가 해결하는 설계 문제들

### 문제 1: 메트로베니아 탐험감과 멀티플레이의 공존

전통적 메트로베니아 게임은 솔로 경험이다. 온라인 멀티플레이를 추가하면 다음 문제가 발생한다:

- 다른 플레이어가 이미 보스를 잡고 지나간 흔적이 보이면, 탐험의 미지감이 사라진다
- 파티원이 앞서 나가면 자기 페이스로 탐색할 수 없다
- 채팅창과 다른 플레이어의 존재가 고딕 분위기를 깨뜨린다

3-Space 해법: 월드를 솔로 전용으로 격리함으로써 메트로베니아 탐험감을 100% 보존한다. 멀티플레이 욕구는 아이템계와 허브에서 충족한다. 월드에서의 비동기 멀티(고스트 메시지)는 고독감을 해치지 않으면서 "나 혼자가 아니다"는 최소한의 연결감만 제공한다.

### 문제 2: 핸드크래프트 맵과 절차적 생성의 공존

핸드크래프트 맵은 정교한 레벨 디자인과 숨겨진 비밀을 제공하지만 콘텐츠 양에 한계가 있다. 절차적 생성은 무한한 콘텐츠를 제공하지만 정교한 설계가 어렵다.

3-Space 해법: 월드는 매크로 구조를 핸드크래프트로, 마이크로 구조를 절차적으로 설계하여 탐험의 정교함을 유지한다. 아이템계는 완전 절차적 생성으로 무한 반복 콘텐츠를 제공한다. 두 접근법이 각자의 공간에서 최대 효과를 발휘한다.

### 문제 3: 솔로 경험과 협동 경험의 분리

"혼자서도 재미있고 함께하면 더 재미있다"는 말은 쉽지만 구현은 어렵다. 단일 공간에서 솔로와 파티를 모두 지원하면, 솔로 플레이어는 파티 플레이어에 비해 불이익을 느끼고, 파티 플레이는 솔로용 난이도에 맞춰져 도전감이 사라진다.

3-Space 해법: 월드는 솔로에 최적화된 난이도와 보상을 제공한다. 아이템계는 1-4인 스케일링으로 솔로도 가능하지만 파티가 유리하도록 설계한다. 솔로 플레이어는 월드에서 충분한 재미를 얻고, 파티 플레이어는 아이템계에서 추가적 재미를 얻는다. 두 경험이 서로의 영역을 침범하지 않는다.

### 문제 4: 야리코미(무한 파밍)와 탐험 가치의 보존

무한 파밍이 가능하면 플레이어는 한 곳에서 레벨을 올려 모든 장벽을 무력화할 수 있다. 이렇게 되면 탐험과 발견의 가치가 사라진다.

3-Space 해법: 아이템계에서의 파밍은 장비 스탯을 올려 스탯 게이트를 해금하지만, 능력 게이트는 월드 탐험으로만 해금된다. 이중 게이트 구조가 파밍만으로는 진행할 수 없게 만들어, "파밍을 위해 탐험이 필요하고, 탐험을 위해 파밍이 필요한" 순환을 강제한다. 스탯 게이트와 능력 게이트의 상세 사양은 `Documents/Terms/Project_Vision_Abyss.md`의 [이중 게이트] 섹션을 참조한다.

## 6. 공간 간 전환의 리듬 설계

### 전환 리듬의 원칙

> **원칙:** 공간 전환은 단순한 로딩 화면이 아니라, 플레이어의 감정 상태를 전환시키는 의식(ritual)이다.

각 전환은 "이전 공간의 경험을 정리"하고 "다음 공간의 기대를 설정"하는 역할을 한다.

### 전환 경로와 감정 흐름

| 전환 | 감정 흐름 | 설계 의도 |
| :--- | :--- | :--- |
| 허브 -> 월드 | 안도 -> 기대/긴장 | 준비를 마치고 미지로 나아가는 모험의 시작 |
| 월드 -> 아이템계 | 발견의 흥분 -> 집중/도전 | 획득한 아이템의 잠재력을 끌어내려는 의지 |
| 아이템계 -> 허브 | 성취/피로 -> 안도/만족 | 도전을 마치고 안전한 곳으로 돌아온 안도 |
| 월드 -> 허브 | 탐험 피로 -> 휴식 | 긴 탐험을 마치고 쉬러 돌아오는 귀환 |
| 허브 -> 아이템계 | 준비 완료 -> 도전 의지 | 파티를 구성하고 깊은 층에 도전하는 결의 |

### 전환 빈도와 세션 구조

한 세션(목표 평균 45분) 내에서 공간 전환은 자연스러운 흐름의 일부여야 하며, 너무 잦거나 너무 드물어서는 안 된다.

| 세션 유형 | 전형적 흐름 | 전환 횟수 |
| :--- | :--- | :--- |
| 탐험 중심 세션 | 허브 -> 월드(탐험) -> 허브 | 2회 |
| 파밍 중심 세션 | 허브 -> 아이템계(반복) -> 허브 | 2-4회 |
| 혼합 세션 | 허브 -> 월드 -> 아이템계 -> 허브 | 3회 |
| 소셜 세션 | 허브(거래/교류) | 0회 |

## 7. 공간 간 연결 원칙

### 순환 구조와의 관계

3-Space는 분리만으로는 의미가 없다. 세 공간이 순환 구조로 연결되어 "탐험 -> 획득 -> 강화 -> 해금 -> 탐험"의 루프를 형성할 때 비로소 완성된다. 순환 구조의 구체적 메커닉은 `Documents/Terms/Project_Vision_Abyss.md`의 [순환 구조] 섹션을 참조한다.

### 연결의 3대 원칙

원칙 1 - 양방향 동기 부여: 각 공간은 다른 공간으로 가야 할 이유를 제공해야 한다.

| 방향 | 동기 |
| :--- | :--- |
| 월드 -> 아이템계 | 월드에서 획득한 장비를 강화하고 싶다 |
| 아이템계 -> 월드 | 강화한 장비로 스탯 게이트를 해금하고 싶다 |
| 월드 -> 허브 | 탐험에서 얻은 소재를 가공하고 싶다 |
| 허브 -> 월드 | 준비를 마치고 새 구역을 탐험하고 싶다 |
| 아이템계 -> 허브 | 획득한 이노센트를 정리하고 싶다 |
| 허브 -> 아이템계 | 파티를 구성해 깊은 층에 도전하고 싶다 |

원칙 2 - 자원의 공간 귀속: 각 공간에서만 획득 가능한 고유 자원이 있어야 한다. 고유 자원의 상세 목록은 `Documents/Terms/Project_Vision_Abyss.md`의 [3-Space 분리 모델] 섹션을 참조한다. 한 공간에서 모든 것을 얻을 수 있다면, 다른 공간으로 이동할 이유가 사라진다.

원칙 3 - 병목 없는 순환: 순환의 어떤 단계에서도 과도한 병목이 발생해서는 안 된다. 한 공간에서 지나치게 오래 머물러야 다음 공간으로 넘어갈 수 있다면, 그것은 순환이 아니라 정체이다.

## 8. 각 공간의 규칙 독립성

### 규칙 독립성 원칙

> **원칙:** 각 공간은 고유한 규칙 체계를 갖는다. 한 공간의 규칙 변경이 다른 공간의 규칙에 연쇄적 영향을 미쳐서는 안 된다.

### 공간별 독립 규칙 영역

| 규칙 영역 | 월드 | 아이템계 | 허브 |
| :--- | :--- | :--- | :--- |
| 사망 페널티 | 세이브 포인트 복귀 | 진행 층 손실 | 없음 |
| 맵 생성 | 핸드크래프트 + 절차적 혼합 | 완전 절차적 | 고정 |
| 진행 방식 | 능력/스탯 게이트 | 층 클리어 | 자유 이동 |
| 플레이어 수 | 1인 | 1-4인 | 무제한 |
| 시간 압박 | 없음 (자유 탐험) | 있음 (층별 시간 제한 가능) | 없음 |
| 적 리스폰 | 구역 재진입 시 | 매 층 새로 생성 | 없음 |

### 규칙 공유 영역 (예외)

다음 규칙은 공간을 넘어 일관성을 유지해야 한다:

- 전투 메커닉: 공격, 회피, 스킬 사용법은 모든 공간에서 동일하다
- 장비 스탯: 장비의 능력치는 어떤 공간에서든 동일하게 적용된다
- 능력 사용: 월드에서 획득한 능력(이단 점프, 벽 타기 등)은 아이템계에서도 사용 가능하다
- 인벤토리: 아이템 소지 규칙은 모든 공간에서 동일하다

### 규칙 독립성의 이유

규칙을 공간별로 분리하는 이유는 각 공간의 감정적 목표가 다르기 때문이다. 월드의 사망 페널티가 "세이브 포인트 복귀"인 이유는 탐험의 부담을 낮추기 위해서이고, 아이템계의 사망 페널티가 "진행 층 손실"인 이유는 야리코미의 리스크-리턴을 높이기 위해서이다. 만약 두 공간의 사망 페널티를 통일하면, 한쪽의 감정적 목표가 훼손된다.

## 9. 저주받은 문제 (Cursed Problems)

3-Space 구조에서 발생하는 상충 문제와 그 해결 방향:

| 문제 | 상충하는 약속 | 해결 방향 | 희생/제약 |
| :--- | :--- | :--- | :--- |
| 탐험의 고독 vs 함께하는 재미 | 월드는 혼자여야 탐험감이 살지만, 플레이어는 함께하고 싶다 | 월드는 솔로 격리. 고스트 메시지로 최소한의 연결감만 제공 | 월드에서의 실시간 협동을 포기한다 |
| 파밍 효율 vs 탐험 동기 | 아이템계에서 무한 파밍하면 월드 탐험의 이유가 사라진다 | 이중 게이트(스탯+능력)로 양쪽 모두 필요하게 설계 | 순수 파밍만으로는 게임을 진행할 수 없다 |
| 자유로운 이동 vs 공간 정체성 | 공간 전환이 너무 자유로우면 공간의 경계가 흐려진다 | 전환에 의식적 절차 부여 (아이템계: 아이템 선택 후 진입, 허브: 워프 포인트 사용) | 즉시 전환의 편의성을 일부 포기한다 |
| 허브의 활력 vs 게임 진행 | 허브에 머무는 것이 너무 편하면 모험을 나가지 않는다 | 허브에서는 성장 자원을 직접 획득할 수 없도록 설계 | 허브 전용 콘텐츠(미니게임 등)를 제한한다 |
| 공간별 규칙 차이 vs 학습 비용 | 공간마다 규칙이 다르면 플레이어가 혼란스럽다 | 전투/조작의 핵심 규칙은 통일. 차이나는 부분(사망 페널티, 맵 유형)은 공간 진입 시 명시적으로 안내 | 공간별 고유 규칙의 복잡도에 상한을 둔다 |
