## 1. 스프라이트 크기 규격

### 1.1. 기준 해상도 및 타일 단위

ECHORIS 기준 해상도: **640×360 px**
타일 크기: **16×16 px** (1타일)
에르다는 화면 높이(360px) 기준으로 약 13% 높이. 배경 구조물의 1/8 ~ 1/12 크기를 유지하여 메가스트럭처의 압도적 스케일을 표현한다.

### 1.2. 캐릭터 유형별 캔버스 규격

| 캐릭터 유형 | 캔버스 크기 | 실제 스프라이트 점유 | 타일 비율 | 비고 |
|:-----------|:----------:|:------------------:|:--------:|:-----|
| 에르다 (플레이어) | 48×64 px | 32×48 px | 2×3타일 | 앞치마/에코/여백 포함 |
| 소형 적 (Slime) | 32×32 px | 16×20 px | 1×1타일 | 점프 시 스쿼시/스트레치 여백 필수 |
| 중형 적 (Skeleton, Ghost) | 48×48 px | 32×40 px | 2~3타일 | 에르다와 유사 높이 |
| 대형 적 (GoldenMonster) | 64×64 px | 48×56 px | 3~4타일 | 에르다보다 1.5배 |
| Guardian (소형 보스 - 아이템 장군) | 80×80 px | 64×64 px | 4×4타일 | |
| 아이템 왕 (중형 보스) | 128×128 px | 96×96 px | 6×6타일 | Phase 2 |
| 아이템 신 (대형 보스) | 192×192 px | 160×160 px | 10×10타일 | Phase 2 |

**캔버스 여백(Padding) 규칙:**
캔버스는 실제 스프라이트보다 최소 8px 이상 크게 설정한다. 이 여백에 무기 이펙트, 앞치마/에코 보조 동작, 공격 범위 VFX가 들어간다. 여백 없이 작업하면 공격 VFX와 캐릭터 바운드가 충돌한다.

### 1.3. 640×360 기준 캐릭터 비율

```
[화면 높이 360px 기준 상대 비율]

에르다 실제 높이  : 48px  = 화면 높이의 13.3%
Guardian 보스    : 64px  = 화면 높이의 17.8%
아이템 왕 보스   : 96px  = 화면 높이의 26.7%
아이템 신 보스   : 160px = 화면 높이의 44.4%
배경 구조물 기준 : ~400px = 에르다의 약 8~12배

[타일 그리드 대응]
640×360 = 40타일 × 22.5타일 (타일 16×16px 기준)
에르다 = 2×3타일 — 화면에 가로 20명, 세로 7~8명 정도 들어오는 스케일
```

## 2. 키프레임 우선 원칙

### 2.1. 픽셀아트 전용 Key-First 방법론

픽셀아트 애니메이션은 **키프레임(핵심 포즈)을 먼저 확정하고, 보간 프레임을 나중에 추가**하는 방식으로 제작한다. 보간 프레임을 먼저 채우면 동작이 흐릿해지고 타이밍이 약해진다.

**Saint11(Pedro Medeiros, Celeste) 원칙 기반:**

| 단계 | 작업 | 완료 기준 |
|:-----|:-----|:---------|
| 1단계: 키포즈 | 동작의 극단적 포즈 2~3장만 드로잉 | 이 포즈만 봐도 어떤 동작인지 알 수 있어야 함 |
| 2단계: 타이밍 | 각 포즈의 hold time 결정 | 실제 60fps 게임에서 재생하여 리듬 확인 |
| 3단계: 중간 포즈 | 필요한 경우에만 보간 포즈 1~2장 추가 | 키포즈 사이가 너무 끊겨 보일 때만 |
| 4단계: 보조 동작 | 앞치마, 고글 끈, 에코 잔상 반응 추가 | 주 동작이 완성된 후에만 작업 |

**프레임 수 상한선:**
- 에르다: 상태별 2~10프레임. 총 애니메이션 상태 15종
- 소형 적: 상태별 2~6프레임. 총 5종 상태
- Guardian 보스: 상태별 2~12프레임. 총 11종 상태

4~8프레임을 기본 목표로 삼는다. 이 범위를 초과하면 제작 비용 대비 품질 향상이 급감한다.

### 2.2. 스쿼시 & 스트레치 규칙

Disney 12원칙 중 픽셀아트에서 가장 효과 대비 비용이 높은 원칙.

| 동작 | 스쿼시 (납작) | 스트레치 (길쭉) | 지속 프레임 |
|:-----|:----------:|:----------:|:----------:|
| 착지 | 착지 순간 1f | 없음 | 1f 이하. 2f 초과 금지 |
| 점프 도약 | 없음 | 도약 정점 향해 1f | 1f |
| Slime 점프 | 도약 직전 1f | 공중 정점 1f | 각 1f |
| 대시 | 없음 | 대시 방향으로 1f | 1f |

스쿼시/스트레치는 "의식하지 못할 정도"로 빠르게 지나가야 한다. 2프레임 이상 유지하면 만화적으로 과장되어 톤이 어긋난다.

## 3. 애니메이션별 프레임 수 표

### 3.1. 에르다 (플레이어 캐릭터)

캔버스 48×64 px. 실제 스프라이트 32×48 px. 60fps 기준.

| 애니메이션 | 프레임 수 | 재생 속도 | 루프 | 비고 |
|:----------|:--------:|:--------:|:----:|:-----|
| idle | 4f | 100~120ms/frame | Loop | 미세 호흡 상하 1~2px. 에코 은은한 맥동 |
| run | 6f | 60ms/frame | Loop | Celeste 참고. 4f도 허용 |
| jump | 2f | - | No | 도약(1f: 길쭉) → 정점/하강(1f: 중립) |
| fall | 2f | - | Loop | 하강 중 루프 |
| land | 2f | 50ms/frame | No | 스쿼시(1f: 납작) → 회복(1f) |
| attack_1 | 4f | 60ms/frame | No | 예비(1f) + 행동(2f) + 회복(1f) |
| attack_2 | 4f | 60ms/frame | No | 예비(1f) + 행동(2f) + 회복(1f) |
| attack_3 | 6f | 50ms/frame | No | 예비(2f) + 행동(2f) + 회복(2f). 후딜 600ms |
| dash | 3f | 40ms/frame | No | 잔상 별도 스프라이트 2~3장 추가 |
| hit | 2f | 80ms/frame | No | 히트스탑 연동. 뒤로 기울어짐 |
| death | 6f | 80ms/frame | No | 무릎 꿇기(2f) → 쓰러짐(3f) → 정지(1f) |
| wall_slide | 2f | - | Loop | 벽 마찰감. 파편 파티클로 보강 |

**3타 콤보 타임라인 (60fps 기준):**

```
1타: 예비 1f (17ms) | 행동 2f (33ms) | 히트스탑 3f (50ms) | 회복 1f (17ms)
2타: 예비 1f (17ms) | 행동 2f (33ms) | 히트스탑 4f (67ms) | 회복 1f (17ms)
3타: 예비 2f (33ms) | 행동 2f (33ms) | 히트스탑 6f (100ms) | 회복 2f (33ms) + 후딜 600ms
```

## 4. 예비동작(Anticipation) 프레임 규칙

### 4.1. AAR 3상 구조 (Anticipation-Action-Recovery)

모든 공격 애니메이션은 3상 구조를 따른다.

```
[ 예비동작 ]       →  [ 행동 ]          →  [ 회복 ]
  Anticipation         Action (히트박스 ON)   Recovery (취약)
  반대 방향 기울기      극단적 과장 포즈        오버익스텐드 자세
  플레이어의 회피 단서  1~2f 유지 권장          idle 복귀 전 정착 포즈
```

**예비동작 방향 규칙:**
예비동작은 공격 방향의 반대편으로 몸이 기울어진다. 검 오른쪽 스윙 → 예비동작은 왼쪽으로 기움. 이것이 플레이어가 회피 타이밍을 잡는 유일한 시각적 단서이다.

### 4.2. 공격 속도별 예비동작 프레임 수

| 공격 속도 | 무기/적 | 예비동작 프레임 | 시각적 텔레그래프 |
|:---------|:--------|:------:|:-----|
| 빠름 (Fast) | 단검 (Build 1+) | 1f | 몸이 앞으로 기우는 자세 + 잔상 시작 |
| 보통 (Medium) | 검, Skeleton | 2~3f | 무기를 뒤로 당기는 동작 명확 |
| 느림 (Slow) | 대검 (Build 1+) | 4~5f | 크게 뒤로 젖히기 + 지면 충격 파티클 |
| Guardian 일반 | 스윙/돌진 | 4~6f | 기 모임 이펙트 + 플래시. 500ms 텔레그래프 |
| Guardian 격노 | 모든 패턴 | 6~8f | 화면 전체 영향. 붉은 색상 변화 필수 |

### 4.3. 적 공격 텔레그래프 원칙

| 원칙 | 구현 |
|:-----|:-----|
| 예비동작 최소 시간 | 약공격도 최소 2프레임(33ms). Guardian은 250ms 이상 |
| 범위 표시 | 광역 공격 전 바닥/공간에 VFX 경고 존 |
| 색상 코딩 | 격노 패턴은 색상이 다른 에너지 이펙트로 구분 |
| 음향 전조 | 애니메이션보다 1프레임 앞서 차징 사운드 재생 |

## 5. 히트스탑과 애니메이션 프리즈 상호작용

### 5.1. 히트스탑 원칙

히트스탑(Hitstop)은 타격 순간 게임 시간을 일시 정지하여 타격감을 증폭한다. 픽셀아트 애니메이션과의 상호작용 규칙:

| 히트스탑 적용 대상 | 애니메이션 처리 |
|:-----------------|:-------------|
| 피격자 | 히트 리액션 애니메이션 첫 프레임에서 프리즈. 히트스탑 종료 후 재생 재개 |
| 공격자 | 히트스탑 중에도 애니메이션 계속 진행 (Dead Cells 방식). 반응성 우선 |
| VFX (히트스파크) | 히트스탑과 무관하게 독립 재생. 타이밍에 영향받지 않음 |
| 배경/타일 | 히트스탑 영향 없음 |

### 5.2. 히트스탑 프레임 수 연동표

공격 강도별 히트스탑 수치는 `System_Combat_HitFeedback.md`가 SSoT. 애니메이션 팀 참고용 요약:

| 타격 유형 | 히트스탑 (프레임) | 히트스탑 (ms) | 피격자 프리즈 포즈 |
|:---------|:-----------:|:-----------:|:---------------|
| 에르다 1타 | 3f | 50ms | hit 애니메이션 1번 프레임 |
| 에르다 2타 | 4f | 67ms | hit 애니메이션 1번 프레임 |
| 에르다 3타 피날레 | 6f | 100ms | hit 애니메이션 1번 프레임 |
| Guardian 돌진 | 8f | 133ms | 에르다 hit 1번 프레임 + 넉백 시작 |
| Guardian 슬램 | 10f | 167ms | 에르다 hit 1번 프레임 + 강한 넉백 |

### 5.3. 피격 시 피격자 애니메이션 우선도

피격 상태(hit)는 모든 현재 재생 중인 애니메이션을 즉시 중단하고 최우선으로 재생된다. 단, death 애니메이션이 재생 중이면 hit로 전환하지 않는다.

```
우선도: death > hit > attack > run/walk/idle
```

## 6. 스프라이트시트 레이아웃 규격

### 6.1. 행 기반 균일 셀 원칙

모든 스프라이트시트는 **애니메이션 상태별 가로 행** 구조를 따른다. 한 행 = 한 애니메이션 상태.

**균일 셀 규칙:** 한 시트 내 모든 셀의 크기는 동일해야 한다. 캔버스 크기 = 셀 크기. 각 상태의 최대 프레임 수를 기준으로 행 너비가 결정된다.

**장점:**
- 엔진에서 행 번호로 애니메이션 상태 직접 참조
- 새 애니메이션 추가 시 행만 추가 (기존 좌표 불변)
- 스프라이트시트 파싱 코드 단순화

### 6.2. 에르다 스프라이트시트 레이아웃

캔버스: 48×64 px 셀 기준. 최대 프레임(death: 6f, run: 6f 등) 기준 시트 너비 결정.

```
Row 0:  idle        (4프레임)  — 4×48 = 192px
Row 1:  run         (6프레임)  — 6×48 = 288px
Row 2:  jump        (2프레임)  — 2×48 = 96px
Row 3:  fall        (2프레임)  — 2×48 = 96px
Row 4:  land        (2프레임)  — 2×48 = 96px
Row 5:  dash        (3프레임)  — 3×48 = 144px
Row 6:  wall_slide  (2프레임)  — 2×48 = 96px
Row 7:  attack_1    (4프레임)  — 4×48 = 192px
Row 8:  attack_2    (4프레임)  — 4×48 = 192px
Row 9:  attack_3    (6프레임)  — 6×48 = 288px
Row 10: hit         (2프레임)  — 2×48 = 96px
Row 11: death       (6프레임)  — 6×48 = 288px
```

**시트 총 크기 권장:** 288×(12×64) = 288×768 px (최대 행 너비 기준)
단, 실제 구현 시 행별 필요 너비만 사용하고 나머지는 투명 패딩 처리.

### 6.3. Guardian 보스 스프라이트시트 레이아웃

캔버스: 80×80 px 셀 기준.

```
Row 0:  idle             (4프레임)
Row 1:  move             (6프레임)
Row 2:  telegraph        (4프레임)
Row 3:  charge           (4프레임)

Row 6:  slam_land        (4프레임)

Row 8:  hit              (2프레임)

```

### 7.1. 히트스파크 (Hit Spark)

| 타입 | 캔버스 | 프레임 수 | 총 지속 시간 | 구성 |

| Heavy Hitspark | 48×48 px | 6f | 약 250ms | 중심 폭발(2f) + 라인 스파크 방사(3f) + 잔광(1f) |

중심: 하이라이트 #FFFFFF. 외곽: 채도 있는 어두운 색 (#3A1A00 등).

### 7.3. 대시 잔상 (Dash Afterimage)

에르다 스프라이트를 복사하여 알파와 색상을 변환한다.

| 잔상 | 알파 | 색조 | 위치 오프셋 |
|:-----|:----:|:----:|:----------:|
| 1번 (가장 앞) | 40% | 청록 #00B4B4 | 대시 방향 -12px |
| 2번 | 25% | 청록 #00B4B4 | 대시 방향 -24px |
| 3번 (가장 뒤) | 10% | 진청 #005A7A | 대시 방향 -36px |

잔상 색상에 청록을 사용하는 이유: 에르다 작업복 색상과 연동되어 "에르다의 잔상"임을 직관적으로 전달하며, PA 기본 팔레트(청록+주황)와 일치한다.

## 8. 팔레트 제한

### 8.1. 양공간 팔레트 규칙

ECHORIS는 두 공간(월드/아이템계)이 같은 색상을 역할 반전하여 사용한다.

| 색상 | 월드에서의 역할 | 아이템계에서의 역할 |
|:-----|:-------------|:----------------|
| 청록 #2A6B6B~#1A4A4A | 배경, 심연, 안개 | 산화 디테일, 차가운 금속 패턴 |
| 주황 #C45A2C~#E87830 | 구조물, 컨테이너 | 배경 발광, 단조열 (기억의 열기) |
| 콘크리트 회색 #6B6B6B~#8A8A8A | 빌더 구조물 | 부식된 금속 표면 |
| 황색 #D4A855 | 등불, 화톳불 | (아이템계 미사용) |
| 백열 백색 #FFF5E0 | 단조 불꽃 | 코어 중심점, 에코 타격 |
| 심연 흑색 #0A0A12 | 깊이, 공허 | 탄소 흑색 그림자 |

**규칙: 어떤 화면이든 청록과 주황이 동시에 보여야 한다.** 청록 또는 주황 중 하나가 완전히 없는 화면은 PA의 시각적 정체성을 잃는다.

### 8.2. 스프라이트 팔레트 제한

| 캐릭터 유형 | 권장 색상 수 | 필수 포함 색 |
|:----------|:----------:|:----------:|
| 에르다 | 12~16색 | 청록 계열(작업복), 주황 계열(앞치마) |
| 소형 적 | 6~8색 | 해당 Tier 팔레트 내 색상 |
| Guardian 보스 | 14~18색 | 격노 전 기본 팔레트 + 격노 후 붉은 팔레트 |

### 8.3. 아이템계 레어리티별 팔레트 악센트

아이템계 내부에서 등장하는 적/환경은 해당 아이템의 레어리티 색상을 악센트로 반드시 포함한다.

| 레어리티 | 악센트 색 | 적용 위치 |
|:--------|:--------:|:---------|
| Normal | #FFFFFF (강철 백) | 이노센트 실루엣 |
| Magic | #6969FF (마법 청) | 이노센트 마법 글로우 |
| Rare | #FFFF00 (룬 황) | 환경 조명 악센트 |
| Legendary | #FF8000 (심연 주황) | 보스 에너지 이펙트 |
| Ancient | #00FF00 (고대 녹) | 환경 전체 색조 시프트 |

### 8.4. Tier별 배경 팔레트 시프트

Tier(지층)가 깊어질수록 청록 비율이 증가하고 주황이 감소한다.

```
Surface : 주황 50% / 청록 50%
Mid     : 주황 70% / 청록 30%  — 단조열 강화
Deep    : 적색 계열 전환 (동맥 적색 + 산화 흑)
Core    : 백열 백색 중심 + 탄소 흑색 외곽
```

## 9. 아웃라인 원칙

Blasphemous 레퍼런스 기반 픽셀아트 아웃라인 규칙.

- **기본 아웃라인:** 1px 검은 아웃라인 (#0A0A12 — 순수 검정 대신 채도 있는 심연 흑)
- **선택적 컬러 아웃라인:** 빛을 받는 면 = 밝은 아웃라인. 그림자 면 = 어두운 아웃라인
- **에르다 고글 앞면:** 청록 컬러 아웃라인 #00B4B4
- **Guardian 방패 엣지:** 황금색 컬러 아웃라인 #D4A855
- **이노센트:** 레어리티 악센트 색상 컬러 아웃라인
