## 1. 개요 (Overview)

### 1.1. 문서 목적

본 문서는 Project Abyss의 탐험 설계가 "어떤 느낌이어야 하는가", "왜 이렇게 설계하는가"를 정의하는 상위 철학 문서이다. `Design_Level_Standards.md`가 Room 규격, Chunk 크기, 게이트 수치 등 물리적 규격과 배치 원칙을 다룬다면, 이 문서는 그 규격들이 추구해야 할 탐험의 방향과 감정을 정의한다. 모든 레벨 설계 결정은 이 철학 문서의 원칙에 정렬되어야 한다.

게이트 유형별 수치 기준(이단 점프 5타일 이상, 안개 변신 1타일 틈새 등)과 배치 규칙은 `Design_Level_Standards.md` 섹션 3~4를 따른다. 본 문서는 해당 규격의 "왜"를 다룬다.

### 1.2. 한 줄 요약

> "보이지만 갈 수 없는 곳이 있고, 갈 수 있게 되면 세계가 다시 태어난다. 이것이 탐험의 전부이다."

## 2. 설계 의도 (Design Intent)

### 2.1. 분해 (Deconstruct)

| 참고 게임 | 탐험 재미 요소 |
| :--- | :--- |
| 월하의 야상곡 | 거대한 성 전체가 하나의 퍼즐. 능력을 하나씩 얻으며 성의 구석을 뚫는 발견의 쾌감. 거꾸로 된 성(Inverted Castle)으로 맵 달성률 200%를 달성한 재귀 구조 |
| 할로우 나이트 | 지도를 직접 채우는 "개척자" 느낌. 메인 경로 이후에는 탐험 순서를 플레이어가 자유롭게 결정하는 극도의 비선형성. 소울라이크와의 결합으로 탐험에 죽음의 무게를 부여 |
| 오리와 눈먼 숲 | 이동(Movement) 자체가 핵심 게이팅 요소. 탈출 시퀀스가 획득한 기동 능력의 숙달 시험 역할. "아름답고 타이트한 메트로베니아"라는 정체성을 처음부터 끝까지 유지 |
| 데드 셀즈 | 로그라이트 + 메트로베니아 융합. 반복 도전 속에서도 맵의 구조적 일관성이 탐험 기억을 축적시킴. 숙련도가 탐험 속도로 직결되는 숙련 보상 |
| 메트로이드 | 무기 자체가 탐험 도구. 미사일은 빨간 문을 열고, 아이스 빔은 발판을 만드는 등 전투와 탐험 해금이 동일 시스템에서 작동 |

### 2.2. 분석 (Analyze)

| 재미의 본질 | 리스크와 리턴 분석 |
| :--- | :--- |
| 미지의 공간 | "저 문 너머에 무엇이 있을까"라는 질문이 탐험을 계속하게 만드는 원동력이다. 답이 명확히 보이면 탐험의 긴장이 사라진다 |
| 능력 기반 해금 | 현재 능력으로 갈 수 없는 곳을 미리 보여주면 "미해결 과제"가 뇌리에 각인된다. 능력을 얻은 후 재방문 시 그 각인이 해소되며 강렬한 성취감이 발생한다 |
| 세계의 확장 | 능력 하나를 얻을 때마다 접근 가능한 공간이 기하급수적으로 늘어난다. 성장이 숫자가 아니라 "갈 수 있는 곳"으로 체감된다 |
| 백트래킹의 가치 | 이미 지나온 층위을 다시 방문할 때 "그때는 몰랐던 것"을 발견하는 쾌감이다. 재방문이 의무가 아니라 보상이어야 한다 |
| 스탯 게이트의 의미 | 순수 메트로베니아의 능력 게이트만으로는 "강화할 동기"가 부족하다. 스탯 게이트는 "더 강해야 갈 수 있다"는 압박을 탐험에 추가하여 아이템계로의 순환을 만든다 |

### 2.3. 재구축 (Rebuild)

| 독창적 변형 | Project Abyss만의 강점 |
| :--- | :--- |
| 이중 게이트 시스템 | 능력 게이트(새 능력 획득) + 스탯 게이트(장비 강화) 동시 적용. 탐험가 판타지와 장인 판타지가 같은 공간에서 충돌하지 않고 서로를 강화한다 |
| 스탯 게이트 = 아이템계 진입 신호 | 스탯 게이트 앞에서 막히는 순간이 "아이템계에 들어갈 때가 됐다"는 자연스러운 유도. 탐험의 벽이 곧 야리코미의 문이다 |
| 월드(솔로) vs 아이템계(1~4인) 분리 | 3-Space 구조가 메트로베니아 탐험의 고독함을 보호한다. 파티 멤버가 보이지 않는 월드에서 플레이어는 진정한 탐험가이다 |
| 절차적 생성 + 핸드크래프트 혼합 | 핸드크래프트된 랜드마크와 서사적 요소가 절차적 맵의 방향 감각 상실을 방지한다. 절차적 변화가 재방문의 신선함을 공급한다 |

### 2.4. 3대 기둥 정렬 (Pillar Alignment)

| 기둥 | 탐험 철학에서의 구현 |
| :--- | :--- |
| 탐험가 | 능력 게이트가 탐험의 진행을 정의한다. 능력 하나를 얻을 때마다 세계가 넓어지며, 플레이어는 세계의 비밀을 하나씩 밝혀가는 탐험가가 된다 |
| 장인 | 스탯 게이트가 탐험에 아이템계를 연결한다. "저 벽을 부수려면 STR이 더 필요하다"는 인식이 아이템계 파밍의 목적을 탐험 세계에 심어준다 |
| 모험가 | 아이템계에서도 능력 게이트의 원칙이 적용된다. 함께 탐험한 파티가 능력 역할을 분담하여 솔로로는 불가능한 루트를 열 수 있다 |

## 3. 메트로베니아 핵심 원칙 (Core Principles)

### 3.1. 비선형 탐험의 정의

비선형 탐험은 "어디로 갈지 플레이어가 결정한다"는 의미가 아니다. 오히려 다음을 의미한다.

> "목적지는 하나이지만, 그곳에 도달하는 시점과 경로가 플레이어에 따라 달라진다."

선형 게임은 A → B → C 순서를 강제한다. 메트로베니아는 A, B, C가 모두 존재하되, 플레이어가 B를 먼저 방문할 수도 있고, A에서 C로 바로 갈 수도 있다. 단, 어떤 경로를 택하든 반드시 통과 가능한 Critical Path가 보장된다.

| 항목 | 선형 게임 | 메트로베니아 |
| :--- | :--- | :--- |
| 진행 구조 | A → B → C (강제) | A, B, C 상호 연결 (탐험 순서 자유) |
| 성장 방식 | 고정 능력치, 이벤트 획득 | 탐험 중 능력 획득, 이것이 진행 조건 |
| 탐험의 의미 | 숨겨진 보너스 아이템 | 새로운 경로 확보 + 세계 이해 |
| 난이도 조절 | 피지컬에 전적 의존 | 피지컬 + 장비 강화(스탯 게이트)의 조화 |

### 3.2. 능력 기반 게이팅의 본질 — "보이지만 갈 수 없다 → 능력 → 갈 수 있다"

능력 게이트는 자물쇠가 아니라 "가능성의 예고"이다.

> "보이지만 갈 수 없다. 그 사실이 목적지를 만든다."

능력 게이트는 세 가지 심리적 기능을 동시에 수행한다.

| 기능 | 설명 | 레벨 디자인 표현 |
| :--- | :--- | :--- |
| 미해결 과제 각인 | 갈 수 없는 곳을 보여줌으로써 "저기 나중에 가야겠다"는 기억을 심는다 | 게이트 너머 보상을 같은 화면에서 보이게 배치 |
| 성장의 직접 체감 | 능력 획득 후 이전에 막혔던 곳을 통과할 때 캐릭터 성장이 수치가 아닌 공간 확장으로 느껴진다 | 능력 획득 층위과 게이트 층위을 지리적으로 연결 |
| 세계 몰입 강화 | 도달 불가한 장소는 세계를 실제보다 더 광대하고 신비롭게 만든다 | 능력 게이트 너머는 독립적 시각 정체성을 가진 층위으로 설계 |

능력 게이트의 구체적 수치(이단 점프 5타일 이상, 안개 변신 1타일 틈새 등)와 배치 규칙은 `Design_Level_Standards.md` 섹션 4-1을 따른다.

### 3.3. 백트래킹의 쾌감 — 재방문이 보상인 이유

백트래킹은 메트로베니아의 숙명이지만, "이미 본 것을 다시 보는 것"이 아니다.

> "같은 장소이지만 다른 눈으로 보는 것. 그 눈이 새로운 능력이다."

| 지루한 백트래킹 | 흥미로운 백트래킹 |
| :--- | :--- |
| 새로운 발견 없이 이동만 한다 | 이전에 보이지 않던 숨겨진 통로가 드러난다 |
| 이미 처치한 약한 적들만 다시 만난다 | 환경 변화로 새로운 적 조합이 등장한다 |
| 목적지까지의 동선이 길고 반복적이다 | 숏컷이 열려 동선이 극적으로 단축된다 |
| 방문할 이유가 없다 | 새 능력으로 열리는 문, 이전에 못 깼던 장벽이 있다 |

백트래킹의 페인 포인트는 이동 시간이다. 이를 완화하기 위해 Project Abyss는 세 가지 구조적 장치를 사용한다.

1. **영구 숏컷**: 능력 획득 후 돌아가는 길에 반대편에서 열리는 문 또는 파괴 가능한 벽. 이후 재방문 시 이동 시간을 대폭 단축한다.
2. **루프 구조**: 직선이 아닌 원형 동선으로 설계하여 시작점 또는 다음 목표 지점으로 자연스럽게 귀환한다. 수직 하강 + 가지 경로 토폴로지 상세는 섹션 5.4를 참조한다.
3. **환경적 변화**: 보스 처치 또는 이벤트 이후 기존 층위에 새로운 요소(새 적, 열린 문, 변화한 지형)가 등장하여 익숙한 공간에 신선함을 제공한다.

## 4. 이중 게이트 철학 (Dual Gate System)

### 4.1. 능력 게이트 × 스탯 게이트 공존 원칙

Project Abyss의 탐험은 두 종류의 게이트가 공존한다. 이 두 게이트는 서로 다른 보상 회로를 자극하며 상호 보완적으로 작동한다.

| 항목 | 능력 게이트 (Ability Gate) | 스탯 게이트 (Stat Gate) |
| :--- | :--- | :--- |
| 해금 조건 | 특정 능력 획득 (이단 점프, 벽 타기, 안개 변신 등) | ATK ≥ X 또는 INT ≥ X 수치 달성 |
| 해금 방법 | 월드 탐험, 보스 처치 후 능력 렐릭 획득 | 아이템계에서 장비 강화, 이노센트 수집 |
| 핵심 감정 | "드디어 갈 수 있다!" 능력 성장의 직접 체감 | "이제 충분히 강해졌다" 수치 성장의 목적지 확인 |
| 플레이어 행동 유도 | 새 능력을 가르쳐 준 후 즉시 적용할 수 있는 게이트 배치 | 장비 강화를 위한 아이템계 진입 동기 부여 |
| 연결 공간 | 월드 내부 순환 | 월드 ↔ 아이템계 순환 |

두 게이트를 같은 층위에 동시 배치할 때의 원칙: 능력 게이트와 스탯 게이트가 같은 입구를 막으면 플레이어는 "무엇이 없어서 막혔는지" 알 수 없다. 두 게이트는 시각적으로 명확히 구분된 다른 입구에 배치한다.

### 4.2. 포사이트닝과 티징 — "보이지만 갈 수 없는 곳" 설계

> "도달할 수 없는 곳을 보여주는 것이 가장 강력한 탐험 동기이다."

게이트 너머를 "볼 수 있게" 설계하는 것이 원칙이다. 시각적 티저(Teaser)가 없는 게이트는 존재를 인지하지 못한 채 지나쳐 버린다.

| 티징 기법 | 설명 | 예시 |
| :--- | :--- | :--- |
| 직접 가시 | 게이트 너머 보상이 같은 화면 내에서 보인다 | 이단 점프 게이트 위로 빛나는 능력 렐릭 상자 |
| 간접 암시 | 특이한 지형, 빛 처리, 안개 등이 "저기 무언가 있다"는 신호를 보낸다 | 안개가 새어 나오는 1타일 틈새 |
| 소리 단서 | 보이지 않지만 소리가 들려 접근을 유도한다 | 봉인된 문 너머에서 들려오는 기계음 또는 물소리 |
| 지도 공백 | 탐험 완료된 층위 옆 지도 공백이 미탐험 공간의 존재를 암시한다 | 인접 Room이 탐험됐는데 연결 통로가 보이지 않는 경우 |

GDC 인사이트 적용 — Discovery Beat 구조: 분기점(Discovery Beat)에서 2~3개 경로를 제시하고, 각 경로(Exploration Beat)에서 고유한 보상/스토리를 제공한 뒤 재합류한다. 티저는 Discovery Beat에서 Exploration Beat로의 "약속"이다.

### 4.3. 게이트 해금 순서 — 비선형 속 의도된 경로

비선형이라고 해서 어떤 순서로든 모든 곳에 갈 수 있다는 의미가 아니다. 탐험에는 의도된 경로가 있되, 그 경로에서 이탈하는 자유가 있다.

| 경로 유형 | 설명 | 게이트 요건 |
| :--- | :--- | :--- |
| Critical Path | 게임 클리어를 위해 반드시 통과해야 하는 최소 경로 | 기본 능력만으로 통과 가능. 스탯/능력 게이트 없음 |
| 보상 경로 | Critical Path를 벗어난 탐험 경로. 장비, 로어, 숏컷 보상 존재 | 능력 게이트 1~2개 요구. 해당 능력 없으면 진입 불가 |
| 극한 탐험 경로 | 숙련 플레이어 전용. 예외적 기동이나 높은 스탯 요구 | 능력 게이트 + 스탯 게이트 복합. Ancient 등급 아이템 등 최고 보상 |
| 숨겨진 경로 | 시각 언어 단서를 읽어야 발견 가능. 맵에 표시 없음 | 시퀀스 브레이킹 또는 지식 게이팅 (Metroidbrainia) |

게이트 해금 순서 설계 원칙:
- 같은 층위에 동일 능력 게이트는 최대 3개까지 배치한다. 과도한 차단은 플레이어를 방치감으로 몰아붙인다.
- 스탯 게이트는 층위 진입 이전이 아니라 층위 내 깊숙한 곳에 배치한다. 입구에서 막히면 탐험 욕구가 꺾인다.

### 4.4. 스탯 게이트 = 아이템계 연결 — 탐험→야리코미 순환

스탯 게이트는 단순한 "입장 조건"이 아니다. 탐험 공간(월드)과 강화 공간(아이템계)을 연결하는 순환의 교점이다.

> "저 벽을 부수려면 ATK가 더 필요하다. 저 봉인을 풀려면 INT가 더 필요하다. 아이템계에 들어가야 한다."

이 인식이 발생하는 순간, 스탯 게이트는 플레이어를 아이템계로 자연스럽게 유도한다. 강화의 목적이 "더 강해지는 것"이 아니라 "저 벽 너머를 탐험하는 것"으로 구체화된다.

| 스탯 | 탐험에서의 표현 | 아이템계에서의 강화 경로 |
| :--- | :--- | :--- |
| ATK (물리장벽) | 파괴 가능한 바리케이드, 두꺼운 문, 바위, 쇠사슬 | 무기 아이템계에서 Gladiator(ATK+) 이노센트 수집 |
| INT (마법봉인) | 룬 문양 장벽, 봉인된 마법 문, 아케인 잠금장치 | 무기/장신구 아이템계에서 Tutor(INT+) 이노센트 수집 |

스탯 게이트의 구체적 시각 표현과 배치 규칙은 `Design_Level_Standards.md` 섹션 4-2를 따른다.

## 5. 맵 상호연결성 (Map Interconnectedness)

### 5.1. 루프 구조 vs 분기 구조

메트로베니아 맵 설계의 두 기본 구조. 두 구조를 상황에 맞게 조합하는 것이 핵심이다.

| 항목 | 루프 구조 (Loop) | 분기 구조 (Branch) |
| :--- | :--- | :--- |
| 정의 | 출발점과 귀환점이 같은 원형 동선 | 하나의 경로가 여러 하위 경로로 갈라짐 |
| 장점 | 백트래킹 부담 최소화. 자연스러운 재방문 | 선택의 자유. 각 분기에 고유 보상 배치 가능 |
| 단점 | 플레이어가 루프 구조를 인식하면 탐험 경이감 감소 | 막힌 분기가 많으면 답답함을 유발 |
| 적합한 사용처 | 주요 층위 간 연결. 보스 방 전후 동선 | 보상 경로. 능력 게이트 너머 탐험 공간 |

Project Abyss 원칙: 메인 동선은 루프 구조로 설계하여 백트래킹 부담을 줄이고, 보상 경로는 분기 구조로 설계하여 탐험의 선택감을 제공한다.

### 5.2. 숏컷과 영구 개방

숏컷(Shortcut)은 백트래킹의 시간 비용을 낮추는 핵심 장치이다.

> "새 능력을 얻고 돌아가는 길이 처음 왔던 길보다 짧아야 한다."

| 숏컷 유형 | 설명 | 해금 조건 |
| :--- | :--- | :--- |
| 반대편 개방 | 이전에 한 방향에서만 열리던 문이 반대편에서도 열린다 | 해당 층위의 능력 게이트 돌파 |
| 파괴 통로 | 특정 스탯 이상 시 부술 수 있는 벽. 파괴 후 영구 개방 | STR/INT 스탯 게이트 해금 |
| 낙하 루트 | 올라온 루트를 거꾸로 내려가면 더 빠른 경로가 된다 | 낙하 데미지 없음 (기본 설정) |
| 전송 장치 | 특정 층위을 순간이동으로 연결하는 장치. 발견과 활성화가 별개 조건 | 레벨 탐험 + 활성화 아이템/능력 |

숏컷 설계 규칙: 숏컷이 Critical Path를 우회하지 않도록 한다. 숏컷은 이미 탐험한 층위으로의 귀환을 빠르게 할 뿐, 미탐험 층위으로의 진입 조건을 우회해서는 안 된다.

### 5.3. 랜드마크와 내비게이션

2D 환경에서는 3D와 달리 원거리 경치를 볼 수 없다. 플레이어가 지도를 보지 않고도 자신의 위치를 감각적으로 알 수 있어야 한다.

| 랜드마크 유형 | 역할 | 구현 방식 |
| :--- | :--- | :--- |
| 시각적 랜드마크 | 층위의 정체성을 상징하는 배경 구조물 | 각 층위마다 반복되지 않는 대형 배경 오브젝트 |
| 청각적 랜드마크 | BGM이 층위 진입을 감각적으로 알림 | 층위마다 고유 BGM. 전환점에서 크로스페이드 |
| 시각 언어 | 배경 색 팔레트와 타일 스타일이 층위을 구분 | 바이옴별 명확히 다른 시각 정체성 |
| 체크포인트 | 맵 탐험의 앵커(기준점) 역할 | 주요 분기점 인근에 배치 |

GDC 인사이트 — Cue Focus 적용: 숨겨진 통로나 보물 근처에 시각적 어트랙터를 배치하여 카메라가 미묘하게 해당 방향을 비추게 한다. 관찰력 있는 플레이어에게 보상이 되는 내비게이션 힌트이다.

### 5.4. 수직 하강 + 가지 경로 토폴로지

Project Abyss 월드 맵의 거시적 구조는 수직 하강을 주축으로 하고, 각 층위에서 수평 가지 경로가 뻗어나가는 형태이다.

```
[수직 하강 + 가지 경로 구조]

[Tier 1: 천공의 정원] ← 후반 해금 (역중력)
        ↑
[Tier 2: 중앙 성채 — HUB] ← 시작점
        ↓ 격벽 ① (이단 점프)
[Tier 3: 묘지/카타콤] ←→ 가지 경로
        ↓ 격벽 ② (벽 타기)
[Tier 4: 지하 수로] ←→ 가지 경로
        ↓ 격벽 ③ (안개 변신)
[Tier 5: 연구소 폐허] ←→ 가지 경로
        ↓ 격벽 ④ (수중 호흡)
[Tier 6: 빙결 보존소] ←→ 가지 경로
        ↓ 대격벽
[Tier 7: 심연의 구] ← 최종

각 층위는 능력 격벽으로 구분되며, 하강할수록 난이도와 보상이 상승한다.
가지 경로는 해당 층위에서 수평으로 뻗어나가 보상 경로·숨겨진 경로를 제공한다.

[원형 순환 구조 (층위 내부)]

  입구 --> 전투 복도 --> 능력 게이트 --> 보상 방
   ^                                        |
   +------- 숏컷 (능력 획득 후 개방) <------+
```

이 구조의 핵심 효과: 플레이어는 "아래로 내려간다"는 일관된 방향 감각을 유지하면서, 각 층위에서 수평으로 가지를 뻗으며 탐험의 폭을 넓힌다. 탐험의 성과(숏컷, 개방된 문)가 맵에 물리적으로 새겨지며, 상위 층위로의 귀환은 숏컷과 전송 장치를 통해 보장된다.

## 6. 탐험의 보상 경제 (Exploration Reward Economy)

### 6.1. Hot Spot / Cool Spot 밀도 원칙

GDC 수집물 배치 이론(Leah Miller)의 Hot Spot / Cool Spot 개념을 2D 메트로베니아에 적용한다.

| 층위 유형 | 정의 | 보상 밀도 전략 |
| :--- | :--- | :--- |
| Hot Spot | 플레이어가 필수적으로 지나가는 메인 경로 층위 | 낮은 보상 밀도. Critical Path 진행에 필요한 최소 보상만 배치 |
| Cool Spot | 플레이어가 의도적으로 탐험해야 발견되는 우회 층위 | 높은 보상 밀도. 고등급 장비, 로어, 숏컷, 능력 업그레이드 집중 배치 |
| 보스 방 전실 | 보스 전 마지막 안전 공간 | 중간 밀도. 포션, 체크포인트, 서사 단서 배치 |

원칙: Hot Spot에서 보상이 많으면 탐험 동기가 사라진다. Cool Spot의 보상이 Hot Spot보다 질적으로 우수해야 "탐험이 효율적"이라는 인식이 생긴다.

### 6.2. 탐험 효율 보장 — 메인 경로 대비 1:1 이상

> "탐험은 항상 직진보다 이득이어야 한다."

탐험 경로에서 얻는 보상의 시간 대비 가치가 메인 경로 대비 동등하거나 더 높아야 한다. 탐험 비효율 = 탐험 포기.

| 경로 유형 | 소요 시간 | 예상 보상 수준 | 기준 |
| :--- | :--- | :--- | :--- |
| Critical Path (직진) | 기준 시간 | Magic 등급 장비 + 기본 경험치 | 기준값 |
| 보상 경로 (우회) | 기준 + 30~50% | Rare 이상 장비 또는 영구 능력 업그레이드 | Critical Path 이상 |
| 극한 탐험 경로 | 기준 + 100% 이상 | Legendary/Ancient 등급 또는 숨겨진 서사 | 최고 등급 보상 |

검증 기준: 레벨 설계 완료 후 직진 경로와 탐험 경로를 각각 플레이하여 단위 시간당 보상 가치를 비교하는 탐험 효율 검증 테스트를 실시한다.

### 6.3. 보상 유형 분류

탐험 보상은 즉각적 보상과 장기적 보상으로 나뉜다. 두 유형이 균형 있게 배치되어야 탐험 동기가 지속된다.

| 보상 유형 | 예시 | 핵심 감정 | 배치 원칙 |
| :--- | :--- | :--- | :--- |
| 즉각 보상 | 포션, 소모품, 골드, 경험치 오브 | 즉각적 도파민 | 탐험 경로 전반에 분산 배치 |
| 장비 보상 | Magic~Ancient 등급 장비 | 아이템계 진입 동기 | Cool Spot 집중 배치 |
| 영구 능력 업그레이드 | 이단 점프, 벽 타기 등 능력 렐릭 | 세계 확장의 쾌감 | 보스 처치 또는 능력 게이트 심부 |
| 서사 보상 | 환경 텍스트, 로어 아이템, NPC 대화 | 세계 이해의 만족감 | 맥락에 맞는 위치. Flow 중단 금지 |
| 메타 보상 | 숏컷 개방, 지도 업데이트, 비밀 방 발견 | 지도 정복의 성취감 | 탐험 깊이와 비례하여 배치 |

## 7. 월드 상태와 진화 (World State Evolution)

### 7.1. 재방문 시 변화

탐험이 세계에 흔적을 남겨야 한다. 플레이어가 지나간 자리는 영구적으로 변한다.

| 사건 | 월드 변화 | 목적 |
| :--- | :--- | :--- |
| 보스 처치 | 해당 층위 BGM 변화, 보스 방 외관 변화, 새로운 NPC 등장 | 보스 처치의 무게감과 세계 변화 연출 |
| 능력 게이트 해금 | 게이트 물리적 개방 (문이 열린 채로 고정) | 성장의 증거를 지도에 새김 |
| 스탯 게이트 파괴 | 바리케이드/봉인 파편이 남아있는 상태로 고정 | 강화의 결과를 탐험 세계에 투영 |
| 숏컷 개방 | 새 경로 표시, 지도 업데이트 | 탐험 효율 향상이 즉각 체감됨 |
| 이벤트 NPC 상호작용 | NPC 위치 또는 대화 변화, 주변 환경 미세 변화 | 세계가 플레이어를 기억하는 느낌 |

### 7.2. 서사적 게이트

일부 게이트는 능력이나 스탯이 아니라 서사적 진행이 해금 조건이다. 이 게이트는 세계의 맥락 속에서만 의미가 있다.

| 서사 게이트 유형 | 해금 조건 | 예시 |
| :--- | :--- | :--- |
| 신뢰 게이트 | NPC와의 관계 형성 | 특정 NPC의 신뢰를 얻어야 열리는 비밀 통로 |
| 지식 게이트 (Metroidbrainia) | 세계의 메커니즘 이해 | 특정 순서로 동상을 조작하면 열리는 고대 문. 능력이 아니라 방법을 아는 플레이어만 가능 |
| 희생 게이트 | 특정 자원 소비 | 대량의 골드 또는 특수 아이템을 봉납해야 열리는 신전 |

서사적 게이트 설계 원칙: 해금 단서는 반드시 게임 세계 내에 존재해야 한다. 외부 정보 없이는 해결 불가능한 퍼즐은 금지한다. 관찰력 있는 플레이어가 내부 단서만으로 발견할 수 있어야 한다.

### 7.3. 역전 구조 — 심연 미러

월하의 야상곡 "거꾸로 된 성"에서 영감을 얻은 Project Abyss의 고유 구조. 특정 임계점 이후 월드의 일부 층위이 "심연 미러" 형태로 변환된다.

> "100% 탐험했다고 생각했을 때, 세계가 다시 태어난다."

심연 미러는 단순한 맵 뒤집기가 아니다. 기존 맵의 물리적 구조를 활용하되 새로운 적 조합, 해금 조건, 서사적 맥락을 부여하여 완전히 다른 탐험 경험을 제공한다. 구체적 발동 조건과 서사적 맥락은 narrative-director가 정의한다.

## 8. 아이템계에서의 메트로베니아 (Item World as Metroidvania)

### 8.1. 절차적 생성에서 탐험 느낌 유지

아이템계(기억의 지층)는 완전 절차적 생성이지만, 탐험의 경이감을 유지해야 한다.

| 문제 | 원인 | 해결 방향 |
| :--- | :--- | :--- |
| 방향 감각 상실 | 모든 방이 비슷하게 생겨 기억이 안 됨 | 각 지층마다 고유한 시각 테마(타일 팔레트, 조명) 적용 |
| 탐험 동기 부재 | 어느 방이나 비슷한 보상이라 탐색 의미가 없음 | 지층별 특수 방(미스터리 룸, 이노센트 집결지)을 고정 비율로 배치 |
| 진행 방향 불명확 | 어디로 가야 다음 지층인지 알 수 없음 | Critical Path(보스 방까지의 최소 경로) 보장. 시각적 방향 단서 |
| 반복 피로 | 같은 지층을 여러 번 반복하면 지루함 | 지층마다 랜덤 시드로 다른 방 조합. 최소 40개 Chunk Pool |

### 8.2. 능력 게이트의 절차적 배치

아이템계에서도 월드와 동일한 능력 게이트가 의미를 가져야 한다.

| 아이템계 게이트 | 설명 | 해금 시 보상 |
| :--- | :--- | :--- |
| 이단 점프 요구 방 | 높은 플랫폼 위에만 존재하는 특수 방 | 고등급 이노센트 또는 희귀 장비 |
| 벽 타기 요구 통로 | 수직 벽면 사이의 좁은 연결 통로 | 보너스 지층 입구 또는 이노센트 군집 |
| 안개 변신 요구 틈새 | 1타일 틈새로만 진입 가능한 숨겨진 방 | 아이템계 내 숨겨진 이노센트 보스 |

원칙: 아이템계에서 능력 게이트로 막힌 방은 "이번 플레이에서는 포기"가 아니라 "다음 진입 시 도전"의 목표가 된다. 순환 진입(월드 → 아이템계 → 월드)을 반복하며 축적된 능력이 이전에 막혔던 곳을 여는 경험을 제공한다.

### 8.3. 시퀀스 브레이킹과 숙련자 보상

> "게임의 규칙을 가장 잘 이해하는 플레이어는 그 규칙을 깨뜨릴 수 있어야 한다."

시퀀스 브레이킹(Sequence Breaking)이란 설계된 순서를 의도적으로 우회하여 더 빠르거나 독창적인 경로로 목표를 달성하는 것이다.

| 시퀀스 브레이킹 유형 | 허용 여부 | 조건 |
| :--- | :--- | :--- |
| 스킬 기반 우회 (높은 플랫폼 오르기) | 허용 | 플레이어의 기술로 의도된 게이트를 우회 |
| 지식 기반 우회 (숨겨진 통로 발견) | 허용 | 관찰과 탐험으로 비밀 경로 발견 |
| 아이템 기반 우회 (고성능 장비로 강적 조기 격파) | 허용 (제한적) | Critical Path를 완전히 건너뛰면 서사가 망가짐. 스토리 트리거는 우회 불가 |
| 버그/익스플로잇 활용 | 의도하지 않은 우회 | 발견 후 패치. 단, 의도된 숙련자 경로처럼 만들어 자연스럽게 통합 가능 |

## 9. 저주받은 문제 (Cursed Problems)

메트로베니아 설계에서 피할 수 없는 근본적 딜레마들이다. 완벽한 해결은 존재하지 않으며, 의식적인 타협이 필요하다.

| 문제 | 상충 | 해결 방향 | 타협 |
| :--- | :--- | :--- | :--- |
| 탐험 자유도 vs 서사 일관성 | 플레이어가 스토리 순서를 뒤집으면 서사가 의미를 잃는다 | 서사 트리거는 능력 게이트 뒤에 배치하여 탐험 순서와 서사 순서를 최소한으로 묶는다 | 핵심 서사 3~5개만 순서를 강제하고, 나머지는 자유 탐험에 위임한다 |
| 백트래킹 부담 vs 맵 규모 | 맵이 넓어질수록 동일 층위을 다시 가는 시간 비용이 증가한다 | 숏컷과 전송 장치를 전략적으로 배치하여 이동 시간을 제어한다 | 맵 규모에 비례하여 전송 장치 밀도를 높인다. 모든 층위이 최대 3회 전송으로 도달 가능하게 설계 |
| 비선형성 vs 진행 방향 안내 | 방향을 자유롭게 주면 플레이어가 길을 잃고, 안내를 강화하면 탐험 경이감이 사라진다 | 지도와 랜드마크로 위치를 파악하게 하되, 목적지는 알려주지 않는다 | "어디에 있는지"는 알 수 있고, "어디로 가야 하는지"는 스스로 판단한다 |
| 절차적 생성 vs 탐험 기억 | 절차적 맵은 재방문 시 달라져 지리적 기억이 무의미해진다 | 아이템계(절차적)와 월드(핸드크래프트)를 엄격히 분리한다 | 아이템계에서는 "방 기억"이 아닌 "패턴 기억"이 숙련도의 축적 |
| 스탯 게이트 vs 탐험 흐름 | 스탯 게이트가 너무 많으면 탐험이 "파밍 숙제"로 변질된다 | 스탯 게이트는 층위 진입점이 아닌 층위 심부에 배치한다 | 한 층위에 스탯 게이트는 최대 2개. 입구 게이트는 능력 게이트로만 구성 |

## 10. 검증 체크리스트 (Verification Checklist)

레벨 설계 완료 후 아래 체크리스트로 탐험 철학 정합성을 검증한다.

### 비선형 탐험
- [ ] Critical Path를 기본 능력만으로 통과 가능한가?
- [ ] 탐험 경로에서 Critical Path보다 질 좋은 보상이 존재하는가?
- [ ] 탐험 순서가 바뀌어도 서사의 핵심이 훼손되지 않는가?
- [ ] 경로의 시각적 티저(Teaser)가 존재하는가?

### 능력 게이트
- [ ] 게이트 너머가 같은 화면에서 보이는가? (가시성 원칙)
- [ ] 게이트 너머에 반드시 보상이 존재하는가? (빈 방 금지)
- [ ] 동일 능력 게이트가 한 층위에 4개 이상 배치되지 않았는가?
- [ ] 게이트의 시각적 표현이 요구 능력을 암시하는가?

### 스탯 게이트
- [ ] 스탯 게이트가 층위 입구가 아닌 심부에 위치하는가?
- [ ] 스탯 게이트 앞에서 아이템계 진입 필요성이 자연스럽게 인식되는가?
- [ ] 한 층위에 스탯 게이트가 2개 이하인가?

### 맵 상호연결성
- [ ] 루프 구조로 주요 동선이 연결되어 있는가?
- [ ] 보스 처치 또는 능력 획득 후 숏컷이 개방되는가?
- [ ] 각 층위의 시각적 랜드마크가 지도 없이 위치를 파악하게 하는가?
- [ ] 모든 층위이 최대 3회 전송으로 도달 가능한가?

### 보상 경제
- [ ] Hot Spot의 보상 밀도가 Cool Spot보다 낮은가?
- [ ] 탐험 경로의 시간 대비 보상 가치가 직진 대비 1:1 이상인가?
- [ ] 탐험 보상이 즉각 보상과 장기 보상을 모두 포함하는가?

### Project Abyss 정렬
- [ ] 탐험 공간(월드)에서 다른 플레이어가 탐험을 방해하지 않는가? (솔로 탐험 보호)
- [ ] 스탯 게이트가 아이템계 진입의 자연스러운 동기가 되는가?
- [ ] 아이템계에서 월드 탐험 능력이 유효하게 작동하는가?
- [ ] 탐험, 강화, 재탐험의 순환 고리가 끊기지 않는가?

## 11. 정합성 노트 (Alignment Notes)

### Design_Level_Standards.md와의 역할 분담

| 항목 | 본 문서 | Design_Level_Standards.md |
| :--- | :--- | :--- |
| 게이트 수치 | 다루지 않음 | 이단 점프 5타일, 안개 변신 1타일 등 구체 수치 정의 |
| 게이트 배치 원칙 | "왜 이렇게 배치하는가" 철학 정의 | "어떻게 배치하는가" 물리적 규칙 정의 |
| 보상 배치 | 밀도 원칙과 감정 목표 | 구체적 배치 위치와 수량 기준 |
| Room/Chunk 규격 | 다루지 않음 | 60x34 타일, 5x3 Chunk 등 물리 규격 |

### Design_Architecture_3Space.md와의 관계

본 문서의 탐험 철학은 월드(World) 공간에 적용된다. 아이템계에서의 메트로베니아 원칙(섹션 8)은 3-Space 분리 모델과 충돌하지 않는 범위 내에서 적용된다. 아이템계의 협동 플레이가 월드 탐험의 고독함을 침범하지 않도록, 두 공간 간 탐험 원칙의 경계를 명확히 유지한다.

### Design_CoreLoop_Circulation.md와의 관계

스탯 게이트 = 아이템계 연결(섹션 4.4)은 Core Loop의 순환 교점이다. 탐험 → 스탯 게이트 → 아이템계 → 강화 → 스탯 게이트 해금 → 재탐험의 고리가 본 문서에서 정의한 철학과 `Design_CoreLoop_Circulation.md`의 대순환 구조와 일치하는지 정기적으로 검증한다.

### System_World_ProcGen.md와의 관계

섹션 5.4(토폴로지)와 섹션 8(아이템계 메트로베니아)에서 정의한 절차적 생성 원칙은 `System_World_ProcGen.md`의 구체적 생성 알고리즘과 정합성을 유지해야 한다. 철학 문서에서 정의한 "랜드마크 보장", "Critical Path 보장" 원칙은 절차적 생성 알고리즘의 필수 제약 조건으로 전달되어야 한다.
