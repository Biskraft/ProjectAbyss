## 1. 업계 표준: 세이프 존 및 마진

### 1.1 TV/콘솔 세이프 존 (Broadcast Standard)

| 구분 | 화면 비율 | 1920x1080 기준 마진 | 용도 |
|:-----|:---------|:-------------------|:-----|
| **Action Safe** | 화면의 95% (각 변 2.5%) | 각 변 48px | 모든 인터랙티브 요소가 이 안에 있어야 함 |
| **Title Safe** | 화면의 90% (각 변 5%) | 각 변 96px | 텍스트/중요 정보가 이 안에 있어야 함 |

### 1.2 실전 게임 UI 마진

| 권장 사항 | 값 |
|:---------|:--|
| 상단 마진 (헤더/타이틀) | 48px (1080p 기준) |
| 좌우 마진 (인터랙티브) | 16-32px |
| 하단 마진 (내비게이션) | 32px |
| HUD 요소 간 그룹 간격 | 8-16px |

### 1.3 ECHORIS 640x360 환산

| 1080p 기준 | 640x360 환산 (÷3) | 비고 |
|:----------|:------------------|:-----|
| Action Safe 48px | 16px | 최소 마진 |
| Title Safe 96px | 32px | 텍스트 마진 |
| 16px 간격 | 5-6px | 요소 간 간격 |
| 32px 하단 마진 | 11px | 하단 패딩 |

## 2. Hollow Knight

### 2.1 기술 사양

| 항목 | 값 |
|:-----|:--|

| **네이티브 해상도** | 1920x1080 (16:9 고정) |

| **울트라와이드** | 지원 안 함 (HUD가 16:9로만 그려짐, Asset Bundle 수정 필요) |

### 2.2 HUD 요소 위치 및 크기 (1920x1080 기준 추정)

### 2.3 설계 철학

- **미니멀 + 디에게틱 느낌:** 마스크/소울이 게임 세계관 내 오브젝트를 직접 표현
- **좌상단 집중:** 모든 전투 정보가 좌상단 ~20% 영역에 집중
- **고딕 장식 프레임:** UI 요소 주변에 손그림 풍 장식 테두리
- **애니메이션 피드백:** 소울 충전 시 액체 채움 이펙트, 마스크 파괴 시 깨짐 이펙트
- **화면 점유율:** HUD 전체가 화면의 ~5-7% 미만

## 5. Celeste

### 5.1 기술 사양

| 항목 | 값 |
|:-----|:--|
| **엔진** | FNA / XNA (C#, MonoGame 계열) |
| **내부 렌더 해상도** | **320x180** (16:9) |
| **타일 크기** | 8x8px |
| **출력 해상도** | 정수 스케일링 (4x→720p, 6x→1080p, 8x→1440p) |
| **렌더링 파이프라인** | 3-레이어 분리: Game(픽셀아트) / UI(고해상도) / Map(3D) |
| **UI 렌더링** | 게임 월드와 별도 고해상도 레이어 |

### 5.2 HUD 요소 위치 및 크기

> Celeste는 "미니멀 HUD" 접근법의 극단적 사례. 게임플레이 중 HUD 거의 없음.

#### 게임플레이 중 표시 요소

| 항목 | 320x180 기준 | 1080p 환산 (6x) | 비고 |
|:-----|:------------|:---------------|:-----|
| 딸기 카운터 | x: ~280px, y: ~8px (우상단) | x: ~1680px, y: ~48px | 챕터 내 수집 딸기 수 |
| 아이콘 크기 | ~8x8px | ~48x48px | 딸기 아이콘 |
| 폰트 크기 | ~6-8px (네이티브) | ~36-48px (1080p) | 고해상도 렌더링 |
| 타이머 | 없음 (기본 HUD) | - | 옵션에서 활성화 가능 |
| 사망 카운터 | 없음 (기본 HUD) | - | 챕터 완료 화면에서 표시 |

#### 챕터 완료 화면 통계

| 항목 | 비고 |
|:-----|:-----|
| 표시 정보 | 딸기 수, 사망 수, 클리어 시간 |
| 레이아웃 | 화면 중앙, 고해상도 UI 레이어 |
| 폰트 | 게임 내 픽셀 폰트와 별도의 깔끔한 폰트 사용 |

#### 대시/에너지 표시

| 항목 | 비고 |
|:-----|:-----|
| 대시 잔량 | HUD가 아닌 캐릭터 색상으로 표시 (빨강→파랑) |
| 체력 | 없음 (즉사 메카닉) |

### 5.3 설계 철학

- **극단적 미니멀리즘:** 체력 바, 마나 바, 스킬 슬롯 없음. 즉사 메카닉이므로 HP 자체가 불필요
- **디에게틱 피드백:** 대시 상태를 캐릭터 색상 변화로 전달 (HUD 불필요)
- **분리 렌더링:** 게임 월드(320x180 픽셀아트)와 UI(1080p 벡터)를 완전 분리하여 픽셀 일관성 유지
- **크리스탈 하트 이중 렌더링:** 같은 오브젝트가 게임 내에서는 픽셀아트, UI에서는 고해상도로 표시
- **정보는 결과 화면에서:** 통계 정보를 게임플레이가 아닌 챕터 종료 시 모아서 표시

### 5.4 3-레이어 렌더링 시스템 (ECHORIS 참고용)

```
Layer 1: Game World (320x180)
  - 타일맵, 캐릭터, 파티클
  - Nearest Neighbor 업스케일

Layer 2: UI (1920x1080 네이티브)
  - 딸기 카운터, 대화 텍스트
  - 벡터/고해상도 폰트

Layer 3: Map (3D 렌더링)
  - 오버월드 맵
  - 별도 셰이더
```

### 6.4 설계 철학 (및 비판)

- **숫자 중심 표시:** HP/MP를 바가 아닌 숫자로만 표시하는 독특한 설계. 직관성 부족 (Zero Punctuation 비판: "HP 라벨이 3픽셀 높이의 배경보다 약간 밝은 글자")
- **정보 과밀:** 좁은 256x240에서 HP, MP, 하트, EXP, 서브웨폰, 레벨 등 다수 정보를 상단 바에 우겨넣음
- **상단 고정:** 모든 정보가 화면 최상단 ~10-12%에 집약 (하단은 완전히 비움)
- **유일한 바 = EXP:** HP/MP가 바가 아닌데 EXP만 바 형태라는 비일관성
- **레거시 설계:** 1997년 PS1 메모리 제약 하 설계. 현대 리마스터에서도 원본 레이아웃 유지

## 8. ECHORIS 적용 지침

### 8.1 현재 상태 vs 레퍼런스 비교

| 항목 | ECHORIS 현재 | 레퍼런스 평균 | 권장 |
|:-----|:-----------|:-----------|:-----|
| HP 바 너비 | 50px (캔버스) = 150px (1080p) = 7.8% | 14-25% | **80-120px (캔버스)** = 240-360px @1080p |
| HP 바 높이 | 5px = 15px @1080p | 20-32px @1080p | **6-8px (캔버스)** |
| 좌측 마진 | 4px = 12px @1080p | 32-80px @1080p | **8-16px (캔버스)** |
| 상단 마진 | 4px = 12px @1080p | 32-60px @1080p | **8-16px (캔버스)** |
| 폰트 크기 | 8px = 24px @1080p | 16-32px @1080p | **8px 유지 (적정)** |
| 아이콘 크기 | 없음 | 32-48px @1080p | **12-16px (캔버스)** |

### 8.2 미니멀 vs 정보 밀집 접근법

ECHORIS의 아이템계 야리코미 특성상, Dead Cells에 가까운 **중간~높은 정보 밀도**가 적합하되, 월드 탐험 시에는 Hollow Knight급 미니멀로 전환하는 **컨텍스트 적응 HUD**를 권장함.

| 공간 | 권장 접근법 | 레퍼런스 | 표시 정보 |
|:-----|:----------|:--------|:---------|
| 월드 (탐험) | 미니멀 | Hollow Knight | HP + 능력 게이지 + 골드 |
| 아이템계 (파밍) | 정보 밀집 | Dead Cells | HP + 장비 슬롯 + 이노센트 + 지층 + 미니맵 |

### 8.3 Celeste 스타일 분리 렌더링 고려

ECHORIS (640x360)도 Celeste (320x180)와 유사하게 **HUD를 별도 고해상도 레이어로 분리** 가능:

```
현재: 모든 것이 640x360 캔버스 위에 렌더링
권장: Game Layer (640x360) + UI Layer (1920x1080)
```

이점:
- 8px 픽셀 폰트 대신 더 깔끔한 벡터 폰트 사용 가능
- 아이콘 디테일 향상
- 게임 월드 픽셀아트 일관성 유지

단점:
- 렌더링 파이프라인 복잡도 증가
- PixiJS에서 이중 캔버스 관리 필요

### 8.4 핵심 수치 권장안 (640x360 캔버스 기준)

```
세이프 존:
  Action Safe 마진: 10px (모든 변)
  Title Safe 마진: 20px (모든 변)

HP 바:
  너비: 100px (화면 너비의 15.6%)
  높이: 6px
  위치: (10, 10)

아이콘 크기:
  주요 (장비 슬롯): 16x16px
  보조 (이노센트/버프): 10x10px
  최소 (상태 아이콘): 8x8px

폰트:
  주요 수치 (HP, Gold): 8px
  보조 정보 (지층, 타이머): 6px
  라벨: 6px

간격:
  요소 내부 간격: 2px
  그룹 간 간격: 6px
  텍스트-아이콘 간격: 2px
```
