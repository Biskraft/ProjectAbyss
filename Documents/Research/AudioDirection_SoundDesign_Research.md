## 1. 히트 SFX 설계 — 2D 액션 게임 사례 분석

### 1-1. 핵심 원리: "사운드가 히트스탑의 절반이다"

PA의 타격 피드백 시스템(`System_Combat_HitFeedback.md`)은 사쿠라이 8기법을 기반으로 시각·운동감각 채널을 이미 구현하고 있다. 그러나 CMB-07-R~V(SFX 레이어 4종)는 전부 `📅 대기` 상태다. GDC 연구 및 플레이테스트 메타분석에 따르면 **타격감의 30~50%는 사운드에서 온다.** 시각 피드백이 완성된 지금, SFX 없이는 시스템이 절반의 효력만 발휘한다.

"타격음"은 단일 사운드가 아니다. Dead Cells의 오디오 팀이 공개한 기법에 따르면, 좋은 히트 SFX는 **3개의 독립 레이어가 시간차를 두고 겹치는 구조**다:

| 레이어 | 역할 | 타이밍 | 음역 |
| :--- | :--- | :--- | :--- |
| L1. 슬래시 (Whoosh) | 무기의 궤적 소리. 공격 모션 시작과 함께 | 0ms (공격 시작) | 중고음 (2–8 kHz) |
| L2. 임팩트 (Thud/Crack) | 금속이 살/갑옷에 닿는 순간 | 히트 판정과 동시 | 저중음 (200–1200 Hz) |
| L3. 피격 반응 (Grunt/Echo) | 적의 피격 반응. 임팩트에서 30~50ms 지연 | +30~50ms | 중음 + 소멸 잔향 |

이 구조는 CMB-07-R/S/T와 직접 대응한다. L3의 30~50ms 지연은 이미 CMB-07-T 명세에 기록되어 있다.

### 1-2. Hollow Knight — "손톱 소리가 나이트의 목소리다"

Team Cherry가 인터뷰에서 밝힌 핵심 원칙: **The Knight는 대사가 없으므로, 모든 감정 표현이 액션 사운드를 통해 이루어진다.** 이것은 에르다와 완벽하게 같은 상황이다.

| 기법 | 내용 | PA 적용 |
| :--- | :--- | :--- |
| 손톱(Nail) 히트의 금속성 | 단단한 금속 "챙" 소리 + 짧은 공명 잔향. 콤보마다 피치가 미세하게 상승 | 검 3타: 1타 피치 0%, 2타 +3%, 3타 +7%. 피니셔가 "완성"처럼 들림 |
| 마법(Spell) 히트와의 대조 | Nail은 딱딱하고 직접적. 마법은 공명이 길고 울림이 큰 소리. 두 종류가 공존 | 근접 3종(딱딱) vs 원거리 2종(울림) 대비 원칙으로 적용 |
| 환경별 히트음 변화 | Greenpath는 유기적 메아리, Deepnest는 납작하고 둔탁한 히트음 | 아이템계 테마별 히트음 EQ 프리셋 적용 (T-FORGE는 금속 공명, T-FOREST는 흡수적) |
| 킬 사운드의 분리 | 적 사망 시 별도 "파열" 사운드 재생. 히트음과 다른 레이어 | CMB-07-U 킬 SFX와 동일 설계 |

**Hollow Knight 교훈:** 캐릭터에게 목소리가 없을 때, 무기 사운드가 캐릭터의 감정과 의도를 전달한다. 에코(Echo)의 히트 사운드는 에르다의 "말"이다.

---

### 1-3. Dead Cells — 무기별 완전 차별화

Dead Cells의 사운드 디자이너 Yoann Laulan은 GDC 2018에서 다음을 발표했다:

> "각 무기는 고유한 '사운드 페르소나'를 가져야 한다. 플레이어가 눈을 감고 소리만 들어도 무기 종류를 알 수 있어야 한다."

| 무기 유형 | Dead Cells 접근 | PA 적용 방향 |
| :--- | :--- | :--- |
| 단검류 | 빠른 "틱-틱" 금속음. 어택이 짧고 서스테인 없음. 4타의 리듬이 마치 타자기 소리처럼 일관됨 | PA 단검: 4타 모두 짧고 건조. 마지막 배후 강타만 "쾅" 저음으로 끝냄 |
| 대검류 | 무겁고 긴 "쿵" + 공기 마찰음. 히트 직전 0.1초 "휙" 선행음이 중량감을 예고 | PA 대검: 선행 "쉬익" + "쿵" 충격파. 2타 충격파 SFX는 별도 "파동" 레이어 |
| 마법 계열 | 마찰음 없이 순수 에너지음. "윙~충" 구조 (발사음 + 폭발음) | PA 지팡이: 투사체 "윙" + 원소별 충돌음 (화염=폭발, 냉기=유리 파열) |
| 활 계열 | 현 당기는 텐션 사운드 + 발사 "팅" + 관통 "쑥" | PA 활: 텐션→발사→충돌 3단 구조. Dead Cells 접근 직접 적용 |

**Dead Cells 교훈:** 무기 SFX 차별화는 전략 다양성을 청각으로 강화한다. 플레이어가 단검에서 대검으로 교체할 때, 시각 변화 전에 사운드 정체성 변화가 먼저 느껴져야 한다.

### 1-4. Hades — 부운 수정 히트음

Darren Korb와 Greg Kasavin의 사운드 접근에서 중요한 기법:

**부운(Boon) 수정 히트음:** Aphrodite 부운 = 따뜻한 "핑크 노이즈" 오버레이, Poseidon = 물결 "출렁" 레이어. 원래 히트음 위에 부운 색감의 레이어가 얹히는 구조다. 이것은 단순히 무기가 아닌 "이 캐릭터의 현재 빌드"를 청각으로 표현한다.

PA 적용: **원소 인챈트(에코)가 활성화되면 기본 히트음 위에 원소 레이어가 추가된다.** 화염 인챈트는 히트에 "직직" 연소음이 얹히고, 냉기는 "갈라지는" 크리스탈 음이 얹힌다. 이것은 CMB-07-O(원소 피드백)의 오디오 버전이다.

---

### 1-5. Celeste — 대시의 "마침표"

Celeste의 SFX 디자이너 Power Up Audio의 접근:

**단일 액션 = 단일 완결 사운드:** 대시는 "쉭" 한 음이지만, 그 안에 어택(0ms), 서스테인(없음), 릴리즈(10ms 잔향)가 완결된다. 플레이어는 대시 SFX만으로 "완전히 성공했음"을 안다. 불완전한 대시(벽에 막힘)는 약간 다른 피치로 구분된다.

PA 적용: **에코의 단발 강타(Echo Strike)는 완결된 단일 사운드여야 한다.** 에코 스트라이크 전용 SFX는 PA의 가장 임팩트 있는 순간(아이템계 진입 트리거, 강력한 단발)이므로 다른 모든 히트음보다 명확하게 "다름"을 청각으로 선언해야 한다.

---

### 1-6. Hyper Light Drifter — 미니멀리즘 임팩트

Disasterpeace의 SFX 접근: **적을수록 많다(Less is more).** HLD의 히트음은 놀라울 정도로 짧고 건조하다. 어택 5ms, 릴리즈 거의 없음. 이 "건조함"이 오히려 날카로운 충격감을 준다.

PA 교훈: **단검 계열은 HLD 접근을 채택한다.** 길고 울리는 히트음은 단검의 스피드 정체성과 맞지 않는다. 빠른 단검 4타는 각각 짧고 건조해야 한다. 반면 대검은 HLD와 반대 방향으로—길고 울림이 있는 사운드로 차별화.

### 1-7. PA 무기별 SFX 프로파일 (CMB-07-V 명세)

| 무기 | 슬래시 사운드 | 임팩트 사운드 | 특수 SFX | 설계 레퍼런스 |
| :--- | :--- | :--- | :--- | :--- |
| **검** | 중간 "휙" (금속 마찰) | 1타: "챙" / 2타: "챙+" / 3타: "쾅" (피치 상승 계단) | 3타 피니셔 짧은 공명 잔향 0.3초 | Hollow Knight Nail |
| **대검** | 묵직한 "쉬익" + 공기 압력 | "쿵" 저주파 중심. 땅이 울리는 느낌 | 2타 충격파: "파동" 저음 레이어 | Dead Cells 대검 |
| **단검** | 매우 짧은 "틱" × 4 | 건조하고 날카로운 "탁" | 배후 강타: 피치 하강 "쾅" 1회 | Hyper Light Drifter |
| **활** | 현 당김 "흐음" (차징) | 발사 "탁" + 충돌 "퍽" | Dead Cells 활 3단 구조 | Dead Cells 활 |
| **지팡이** | 마법 충전 "윙~" | 원소별 충돌음 (화염/냉기/전격 분기) | 투사체 비행음 (짧은 "씽") | Hades 마법 계열 |

**콤보 피치 계단 원칙 (검/단검/지팡이 공통):** 같은 무기의 콤보는 타격이 누적될수록 임팩트 음의 피치가 미세하게 (3~7%) 상승한다. 3타 피니셔 직전은 플레이어가 "완성"을 청각으로 예감한다.

**히트 컨펌 원칙:** "내가 뭔가를 맞췄다"는 사실은 L2(임팩트)의 **저음 성분이 충분히 크냐**로 결정된다. 임팩트음에서 150~400 Hz 대역이 빠지면 타격감이 사라진다. 보조 모니터나 저가 노트북 스피커를 쓰는 플레이어를 고려하면 이 대역은 반드시 강조해야 한다.

## 2. 앰비언트 / 환경 오디오

### 2-1. Hollow Knight — 구역별 앰비언트 레이어 시스템

Hollow Knight의 앰비언트는 **세 개의 독립 오디오 레이어**가 항상 동시에 흐른다:

```
Layer A: 환경 기저 (Environmental Bed)
  — 항상 재생. 공간 특성을 정의. Greenpath=풀벌레, Deepnest=저음 맥동
Layer B: 거리 이벤트 (Distance Events)
  — 랜덤 간격으로 발생. "저 멀리 뭔가 있다" 암시. 시각화 없음
Layer C: 근접 트리거 (Proximity Triggers)
  — 특정 오브젝트/구역 진입 시 활성화. 물 소리, 상인 소리
```

**Greenpath vs Deepnest 대비:**

| 요소 | Greenpath | Deepnest |
| :--- | :--- | :--- |
| 기저음 | 풀벌레 합창, 바람, 물소리 | 저음 맥동, 산발적 스크리밍 |
| 거리 이벤트 | 새소리, 잎 흔들림 | 알 수 없는 저음, 벽 안에서 스크래핑 |
| 공간 잔향 | 중간. 개방적 | 없음에 가까움. 음향이 흡수됨 ("막힌 느낌") |
| 음악과의 관계 | 음악과 앰비언트가 조화 | 음악이 점차 줄고 앰비언트가 지배 |

**PA 교훈:** 앰비언트의 잔향(Reverb) 길이는 공간 규모의 직접 번역이다. The Shaft의 깊은 층위일수록 잔향이 길어지고("광활한 공동"), 아이템계 지층 내부는 잔향이 짧아진다("금속 상자 안").

---

### 2-2. Dead Cells — 바이옴 오디오 전환

Dead Cells의 환경 오디오 시스템은 **스냅(Snap) 전환이 아닌 크로스페이드(Crossfade) 기반**이다. 두 바이옴의 경계를 지날 때 약 2초에 걸쳐 앰비언트가 교차 페이드된다. 이것이 Hard Cut보다 공간 이동이 자연스럽게 느껴지는 이유다.

PA 적용: 월드 층위 간 이동 시 앰비언트 크로스페이드 2~3초. 아이템계 진입 시 앰비언트 페이드아웃 1초 + 새 앰비언트 페이드인 1초.

---

### 2-3. PA 월드 층위별 앰비언트 프로파일

The Shaft는 수직 하강 구조다. 층위가 깊어질수록 산업적·기계적 소리가 줄고 원시적·압박적 사운드가 증가한다.

| 층위 | 공간 특성 | 앰비언트 기저 | 거리 이벤트 | 잔향 설정 |
| :--- | :--- | :--- | :--- | :--- |
| Tier 2 (허브/세이브) | 따뜻한 대장간. 유일한 안전 지대 | 화덕 소리, 망치 원거리 타격, 낮은 기계 허밍 | 동료 소리, 거래 소리, 물 방울 | 중간 잔향 (돌 공간) |
| Tier 3~4 | 산업 구조물. 탐험 시작 | 금속 구조물 진동, 통풍구 바람, 케이블 긴장음 | 멀리서 들리는 기계 작동음 | 긴 잔향 (대공동) |
| Tier 5~6 | 구조물이 노후화. 이질감 증가 | 금속 부식 소리, 저음 맥동, 간헐적 진동 | 원인 불명의 스크래핑, 알 수 없는 흐름 | 매우 긴 잔향 + 플랜지 |
| Tier 7 (최심층) | 빌더의 영역. 세계의 끝 | 거의 침묵. 에르다의 발소리만 들림 | 없음 | 극장 급 잔향 (무한 공동) |

**세이브 포인트 사운드 정체성:** 프로젝트 비전(`Project_Vision_Abyss.md §5`)에 명시된 "거대한 차가움 속 유일한 따뜻함." 세이브 포인트에 진입하는 순간, 외부의 산업적 앰비언트가 페이드아웃되고 화덕 소리와 낮은 대장간 허밍이 페이드인된다. 이것이 플레이어에게 "이곳은 안전하다"를 전달하는 가장 강력한 신호다.

## 3. 음악 디렉션

### 3-1. 레퍼런스 분석

#### SotN (Symphony of the Night) — 구역 전용 테마의 정전

SotN의 음악 디렉터 미치루 야마네(Michiru Yamane)의 접근:
- 각 구역은 그 구역의 **건축적·감정적 정체성**을 반영한 전용 테마를 보유
- Marble Gallery: 클래식 하프시코드 + 현악 앙상블 = "낡은 귀족의 영화(榮華)"
- Abandoned Pit: 미니멀 타악기 + 베이스 드론 = "무너진 것들의 적막"
- 음악은 탐험을 방해하지 않는다. 루프 포인트를 자연스럽게 처리하여 무한 루프가 이물감 없음

**PA 적용:** 월드 각 층위가 전용 탐험 BGM을 가진다. 루프 포인트는 반드시 음악적으로 해결한다 (멜로디 종지 후 루프).

#### Hollow Knight — 크리스토퍼 라킨의 미니멀 오케스트라

크리스토퍼 라킨(Christopher Larkin) 인터뷰에서:

> "나는 각 구역의 '심리적 날씨'를 먼저 썼다. Greenpath는 '경이로움+약간의 우울'. City of Tears는 '아름다움+거대한 슬픔'. 그 감정이 오케스트레이션을 결정했다."

| 구역 | 핵심 악기 | 감정 | 동적 요소 |
| :--- | :--- | :--- | :--- |
| Greenpath | 피아노 + 첼로 피치카토 | 경이로움 + 약간의 우울 | 탐험 깊이에 따라 오케스트레이션 두께 증가 |
| City of Tears | 피아노 + 현악 앙상블 | 아름다움 + 슬픔 | 비 사운드가 BGM 레이어로 통합 |
| Deepnest | 거의 없음 + 저음 드론 | 공포 + 불안 | BGM 점차 사라지고 앰비언트 지배 |

**PA 적용:** Hollow Knight 방식의 "심리적 날씨 우선" 접근 채택. 각 월드 층위의 BGM 작업 지시서에 감정 목표를 먼저 기술한다.

#### Dead Cells — 전자음 + 오케스트레이션 하이브리드

Dead Cells의 음악은 칩튠 전통(8비트 느낌)과 현대 오케스트레이션이 합쳐진 하이브리드다. 이것이 "복고적인 느낌인데 웅장하다"는 감각을 만든다.

**강도 스케일링 시스템:** Dead Cells는 플레이어가 전투 중일 때 BGM의 타악기 레이어(드럼 트랙)가 페이드인되고, 전투 종료 후 30초 경과 시 페이드아웃된다. 동일 음악이 탐험 버전과 전투 버전으로 전환된다.

PA 적용: **월드 BGM = 탐험 레이어 + 전투 레이어.** 전투 탐지 시 타악기/베이스 레이어가 1.5초에 걸쳐 페이드인.

#### Hades — 다렌 코브의 "수퍼제트 아그리안" 장르 블렌딩

다렌 코브(Darren Korb)는 하데스의 음악을 "수퍼제트 아그리안"이라 불렀다: 지중해 민속음악 + 현대 헤비 메탈 + 전자음의 혼합.

**핵심 기법:** 각 지역이 독립된 음악 정체성을 갖지만, 모든 음악이 공통 리듬 패턴을 공유하여 게임 전체가 하나의 세계처럼 들린다.

PA 적용: **아이템계 BGM은 월드 BGM과 공통 리듬 패턴을 공유하되, 음색(Timbre)이 완전히 다르다.** 월드는 멜로딕, 아이템계는 리드미컬+타악기 중심.

#### Hyper Light Drifter — Disasterpeace의 앰비언트 전자음

Disasterpeace(Rich Vreeland)는 HLD에서 전통적인 음악 구조(기승전결)를 버리고 **무드를 흐르는 음악 텍스처(Music as Texture)** 접근을 채택했다. 명확한 멜로디가 없고, 음색의 흐름이 감정을 만든다.

PA 교훈: **Tier 7(최심층) BGM은 Disasterpeace 접근을 참조한다.** 깊어질수록 구조화된 음악에서 앰비언트 텍스처로 이행한다.

| :--- | :--- | :--- |
| T-HOME | 목관악기 (따뜻한 멜로디) | 리듬이 요리 리듬(왈츠풍) |
| T-MILITARY | 군악 스네어 드럼 | 행진 리듬. 정확한 박자감 |
| T-FORGE | 대형 앤빌 타격 (베이스 리듬) | 리듬이 망치 소리 자체가 됨 |
| T-FOREST | 타악 없음. 자연 앰비언트 | 리듬 구조 해체. 자연음이 BGM 대체 |
| T-SCHOLAR | 하프시코드 + 낮은 현악 | 중세 학술 느낌. 느린 템포 |
| T-SACRED | 파이프 오르간 드론 | BGM 거의 없음. 오르간이 앰비언트화 |
| T-MARKET | 만돌린 + 짧은 멜로디 | 가볍고 빠른 템포. 거래 리듬 |
| T-PRISON | 없음 + 저음 맥동 | BGM 극소화. 앰비언트 압도 |
| T-NATURE | 타악 기반 원시 리듬 | 박자 불규칙. 자연의 비주기성 |
| T-WAR | 전쟁 북 + 현악 불협 | 리듬 붕괴 직전의 긴장 |

#### "음악이 없어야 할 때" 원칙

Hollow Knight의 Deepnest 접근에서 가장 중요한 교훈: **음악의 부재 자체가 음악이다.** PA에서 음악이 페이드아웃되어야 하는 순간:

1. Tier 7 최심층 탐험
2. T-FOREST 지층에서 오래 머물 때 (자연이 BGM을 대체)
3. 아이템계 진입 직전 1초 (침묵이 진입의 무게를 만든다)
4. 에르다가 마르타의 기억을 마주칠 때 (내러티브 순간)

### 4-2. THE 스파이크 사운드 — 아이템계 진입

**아이템계 진입은 PA의 가장 중요한 단일 사운드 이벤트다.** "이 검 속으로 들어간다"는 게임의 핵심 스파이크이므로, 이 SFX는 게임에서 가장 공들여 설계해야 한다.

진입 시퀀스는 `ItemWorldEntry_TechImplementation_Research.md`에서 정의된 8페이즈 바닥 붕괴 + 다이빙 연출과 동기화된다:

```
Phase 1: Echo Strike 충전 (0~0.5초)
  → SFX: 에너지 충전 "윙~" (피치 상승)

Phase 2: 바닥 균열 생성 (0.5~1.0초)
  → SFX: 금속 균열 "쩍쩍" + 저음 공명

Phase 3: 바닥 붕괴 (1.0~2.0초)
  → SFX: 금속 파열 + 낙하 바람 소리 급증

Phase 4: 에르다 다이빙 (2.0~2.5초)
  → SFX: 바람 소리 최대 + 아이템 고유 "흡수" 음

Phase 5: 씬 전환 블랙아웃 (2.5~3.4초)
  → SFX: 모든 소리 페이드아웃 0.5초
  → 이후: 새 테마 앰비언트 페이드인
```

**"흡수" 음의 방향:** 에르다가 아이템 속으로 빨려 들어가는 순간, 모든 외부 소리가 갑자기 "진공처럼" 사라지는 효과. 기술적으로는 모든 오디오에 급격한 LP 필터(저역 통과)를 걸면서 동시에 음량을 줄이는 방식으로 구현. "귀가 멍해지는" 기압 변화 느낌.

레어리티별 진입 SFX 강도:
| 레어리티 | 균열 SFX | 다이빙 SFX | 흡수 음 |
| :--- | :--- | :--- | :--- |
| Normal | 작고 건조한 균열 | 짧은 바람 | 기본 |
| Magic | 전기 스파크 추가 | 보통 바람 | 1.2x |
| Rare | 황금빛 "챙" 추가 | 강한 바람 | 1.5x |
| Legendary | 불꽃 폭발 추가 | 포효 수준 바람 | 2.0x |
| Ancient | 현실이 왜곡되는 느낌 (피치 시프트 모든 SFX) | 차원 붕괴 수준 | 2.5x + 반향 |

| 이노센트 복종 | "딸랑" 링 SFX + 짧은 옥타브 상승 | 수집의 만족. 디스가이아 분위기 |
| 이노센트 도주 | 낮은 "삐익" 실패음 | 패널티 인식. 단, 짜증보다는 코믹하게 |
| 레어리티 드랍 | 등급별 차별화된 드랍 SFX | (하단 표) |

**레어리티 드랍 SFX 차별화:**

| 레어리티 | 드랍 SFX | 지속 시간 | 특징 |
| :--- | :--- | :--- | :--- |
| Normal | 작은 "탁" | 0.2초 | 존재감 최소 |
| Magic | 가벼운 "챙" | 0.4초 | 청량한 금속음 |
| Rare | 황금빛 "챙챙챙" | 0.8초 | 3음 짧은 팡파르 |
| Legendary | 낮은 "쿵" + 오렌지 불꽃 사운드 | 1.5초 | 땅이 울리는 느낌 |
| Ancient | 시간이 멈추는 느낌 (리버브 급증) + 팡파르 | 2.5초 | 게임 내 최대 이벤트급 |

## 5. Howler.js 기술 구현

### 5-1. Howler.js 핵심 기능 및 한계

**Howler.js 2.x 기능 요약:**

| 기능 | 지원 여부 | 비고 |
| :--- | :--- | :--- |
| 다중 동시 재생 | 지원 | `pool` 옵션으로 동시 인스턴스 수 설정 |
| 오디오 스프라이트 | 지원 | `sprite` 옵션으로 단일 파일 내 다수 사운드 정의 |
| 3D 공간 오디오 | 지원 | `pannerAttr`, `pos()`, `orientation()` |
| Web Audio API 폴백 | 지원 | Web Audio 불가 시 HTML5 Audio 자동 폴백 |
| 루프 | 지원 | `loop: true` |
| 페이드 | 지원 | `fade(from, to, duration)` |
| 오디오 그룹 | 지원 | `Howler.volume()` 전역 / 개별 `volume()` |
| 자동 인코딩 선택 | 지원 | `['webm', 'ogg', 'mp3']` 형식 배열로 브라우저별 선택 |

**주요 한계:**

1. **Web Audio 레이턴시 (30~100ms):** `WebGameFeel_Optimization_Research.md §3`에서 분석된 문제. AudioContext는 초기화 후에도 OS 오디오 버퍼로 인한 지연이 존재한다.
2. **백그라운드 탭 자동 정지:** 브라우저는 백그라운드 탭의 AudioContext를 자동 `suspend`한다. Howler는 `visibilitychange` 이벤트로 이를 처리하지만, 재포커스 시 첫 사운드가 지연될 수 있다.
3. **iOS Safari 제약:** 사용자 인터랙션 없이 AudioContext를 시작할 수 없다. 첫 클릭/터치 이후에만 오디오 재생 가능. (PA는 PC 웹 타겟이라 우선순위 낮음)

### 5-2. 오디오 스프라이트 전략

Web 게임에서 HTTP 요청 최소화를 위해 **카테고리별 스프라이트 번들**을 권장한다. 파일 하나에 여러 SFX를 타임라인으로 배치하고 Howler의 `sprite` 옵션으로 섹션을 정의한다.

**권장 번들 구조:**

| 번들 파일 | 포함 SFX | 예상 크기 | 로드 타이밍 |
| :--- | :--- | :--- | :--- |
| `sfx_combat_bundle.ogg` | 무기 슬래시 5종 × 3변형, 임팩트 5종 × 3변형, 킬 SFX | ~2MB | 게임 시작 시 |
| `sfx_ui_bundle.ogg` | 메뉴 조작, 아이템 장착, 레벨업, 드랍 SFX | ~0.5MB | 게임 시작 시 |
| `sfx_world_bundle.ogg` | 발소리 (지면 재질 8종), 점프, 착지, 대시 | ~1MB | 게임 시작 시 |
| `sfx_itemworld_bundle.ogg` | 진입 시퀀스 전체, 이노센트 SFX, 보스 전환 | ~1.5MB | 첫 아이템계 진입 전 |

**스프라이트 정의 예시:**
```javascript
const combatSounds = new Howl({
  src: ['sfx_combat_bundle.ogg', 'sfx_combat_bundle.mp3'],
  sprite: {
    sword_whoosh_01: [0, 150],
    sword_whoosh_02: [200, 150],
    sword_impact_light: [400, 200],
    sword_impact_heavy: [650, 300],
    sword_combo_finish: [1000, 500],
    // ...
  },
  pool: 8  // 동시 재생 가능 인스턴스 수
});
```

### 5-3. Web Audio 레이턴시 보상

`WebGameFeel_Optimization_Research.md`의 분석을 바탕으로, SFX와 히트스탑 동기화를 위한 실용적 방법:

**레이턴시 측정 + 오프셋 보정:**
```javascript
// AudioContext 초기화 시 레이턴시 측정
const ctx = Howler.ctx;
const measuredLatency = ctx.baseLatency + ctx.outputLatency;
// measuredLatency를 SFX 트리거 오프셋에 반영

// 히트 SFX 트리거 시:
const adjustedDelay = Math.max(0, HITSTOP_ONSET_MS - measuredLatency);
setTimeout(() => combatSounds.play('sword_impact_light'), adjustedDelay);
```

**AudioContext 사전 워밍(Pre-warming):** 게임 시작 첫 클릭 이후, 무음 버퍼를 1회 재생하여 AudioContext 파이프라인을 초기화한다. 이후 실제 SFX의 첫 번째 재생에서 "콜드 스타트 레이턴시"가 제거된다.

### 5-4. 오디오 풀링 (동시 히트 처리)

아이템계에서 파티 4인이 동시에 전투할 경우, 프레임당 다수의 히트가 동시에 발생한다. Howler의 `pool` 옵션으로 동시 인스턴스를 관리한다:

| 사운드 카테고리 | 권장 pool 크기 | 이유 |
| :--- | :--- | :--- |
| 무기 임팩트 | 8 | 4인 파티 × 콤보 중복 |
| 무기 슬래시 | 8 | 위와 동일 |
| 피격 반응 | 8 | 다수 적 동시 피격 |
| 환경 SFX | 4 | 동시 발생 빈도 낮음 |
| 앰비언트 루프 | 3 | 레이어 수 고정 |
| BGM | 2 | 크로스페이드용 |

### 5-5. BGM 크로스페이드 구현

Howler의 `fade()` 메서드를 사용한 구역 간 BGM 전환:

```javascript
// 현재 BGM 페이드아웃 + 새 BGM 페이드인 동시 실행
function transitionBGM(nextTrack: string, duration = 2000) {
  if (currentBGM) {
    currentBGM.fade(currentBGM.volume(), 0, duration);
    setTimeout(() => currentBGM.stop(), duration);
  }
  const newBGM = new Howl({
    src: [nextTrack],
    loop: true,
    volume: 0
  });
  newBGM.play();
  newBGM.fade(0, TARGET_VOLUME, duration);
  currentBGM = newBGM;
}
```

### 5-6. 전투/탐험 강도 레이어 시스템

Dead Cells 방식의 동적 BGM 강도 시스템. Howler는 복수의 Howl 인스턴스를 동시에 `play()`로 시작하고 각 볼륨을 독립 제어한다:

```javascript
// BGM을 탐험/전투 두 레이어로 로드
const bgmExplore = new Howl({ src: ['mus_explore_tier3_loop.ogg'], loop: true });
const bgmCombat  = new Howl({ src: ['mus_combat_tier3_loop.ogg'],  loop: true, volume: 0 });

// 탐험 시작 시 동기 재생 (두 트랙이 정확히 같은 BPM, 루프포인트를 공유해야 함)
bgmExplore.play();
bgmCombat.play();  // 볼륨 0으로 재생 중. 타이밍 동기화 유지

// 전투 시작
bgmCombat.fade(0, 0.8, 1500);

// 전투 종료 (30초 후)
setTimeout(() => bgmCombat.fade(0.8, 0, 3000), 30000);
```

**주의:** 두 트랙은 정확히 같은 BPM, 같은 마디 수, 같은 루프포인트를 가져야 한다. 오디오 에셋 제작 단계에서 반드시 검토.

## 7. 침묵 주인공 오디오 디자인

### 7-1. 왜 침묵 주인공에게 오디오가 더 중요한가

에르다는 대사 0줄이다. Hollow Knight의 The Knight, BLAME!의 킬리와 같이 말하지 않는 주인공이다. 내러티브 전달이 대사가 아닌 **행동과 환경**으로 이루어지므로, 에르다의 모든 움직임 사운드가 그녀의 "말"이 된다.

플레이어는 에르다가 무엇을 느끼는지 알 수 없다. 그러나 에르다가 **어떻게 움직이고, 에코가 어떤 소리를 내는지**를 통해 그녀의 성격과 감정 상태를 추론한다.

---

### 7-2. 에르다의 "발성" 디자인

침묵 주인공이라도 **비언어적 발성(Non-verbal Vocalizations)** 은 허용된다. 이것은 대사가 아니라 신체 반응이다.

| 상황 | 비언어적 발성 | 음량/피치 | 목표 |
| :--- | :--- | :--- | :--- |
| 대쉬 | 짧은 숨 내뱉기 ("후") | 작음, 낮음 | 에너지 소비 표현 |
| 피격 | 짧은 충격 숨소리 ("읏") | 중간 | 피해 인식. 단, 과하지 않게 |
| 치명타 피격 | 더 강한 충격음 ("윽") | 중간-큰 | 위험 신호 |
| 사망 | 짧은 쓰러짐 소리 | 중간 | 드라마틱하되 길지 않게 |
| 높은 곳 낙하 착지 | 충격 숨소리 + 무릎 구부림 관절음 | 작음 | 물리적 무게감 |
| 아이템계 진입 다이빙 | 숨 참기 (무음) → 진입 후 공기 흡입 | 없음 → 중간 | 긴장감 극대화 |

**중요 원칙:** 에르다의 비언어적 발성은 **절제되어야 한다.** 너무 자주, 너무 크게 나오면 "말 없는 주인공"의 정체성을 희석시킨다. 발성은 아껴야 의미가 생긴다.

---

### 7-3. 발소리 시스템 — 에르다의 존재감

에르다는 메가스트럭처를 걷는다. 발소리는 **그녀가 어디 있는가, 어떤 공간에 있는가**를 지속적으로 전달한다.

**지면 재질별 발소리:**

| 지면 재질 | 발소리 특성 | 적용 층위/테마 |
| :--- | :--- | :--- |
| 금속 격자 | 딱딱한 "탁탁". 짧은 금속 공명 | 월드 구조물 전반 |
| 돌/콘크리트 | 단단하고 낮은 "툭툭" | Tier 5~7 |
| 흙/유기물 | 부드럽고 흡수적인 "퍽퍽" | T-FOREST, T-NATURE |
| 금속 판 (대형) | "탁" + 공명이 길게 울림 | T-FORGE, T-MILITARY |
| 나무 | "삐걱" + 건조한 "탁" | T-HOME |
| 물/진흙 | "첨벙" + 낮은 퍽 | T-NATURE 일부 |

**달리기 vs 걷기:** 달리기는 발소리 간격이 짧고 충격이 강하다. 걷기(잠입)는 발소리 음량이 -12dB. 이 차이가 "조심히 움직이는" 느낌을 만든다.

---

### 7-4. 에코(Echo)의 사운드 정체성

에코는 에르다의 우측 손에 항상 들려 있는 도구/무기다. GBE(BLAME!) 오마주. 에코의 사운드는 **에르다의 의도와 감정을 가장 직접적으로 표현하는 음향 채널**이다.

**에코 상태별 사운드:**

| 에코 상태 | 사운드 | 빈도 |
| :--- | :--- | :--- |
| 평상시 대기 | 낮고 안정적인 전기 허밍. 거의 들리지 않음 | 상시 (음량 매우 낮음) |
| 새 구역 스캔 | 짧은 "핑" + 잔향 (소나 효과) | 탐험 중 가끔 |
| 균열 탐지 | 빠른 "비이~" 알림음 | 균열 근처 |
| 아이템계 진입 충전 | 에너지 상승 "윙~" (충전 중 피치 상승) | 진입 트리거 시 |
| 원소 인챈트 활성 | 원소별 지속 허밍 (화염=따뜻한 징잉, 냉기=차갑고 높은 윙) | 인챈트 지속 중 |
| 전투 중 | 허밍 볼륨 약간 상승. 타격 시 각 히트에 에코 공명 추가 | 전투 중 |

**에코 사운드 디자인 원칙:** 에코는 기계 + 유기물의 혼성체다. 순수 전자음이 아니라 **따뜻한 유기적 울림이 있는 전기음**이어야 한다. BLAME!의 GBE 사격음처럼, "이것은 단순한 기계가 아니다"를 사운드로 표현한다.

### 7-5. "기억 속에서" — 아이템계 테마와 침묵 서사

`Project_Vision_Abyss.md §5`의 스토리 톤 진행:

> Act 2: "아이템 안에서 이전 주인의 삶 목격. 이건 그냥 던전이 아니었어"

이 순간의 오디오 디자인이 중요하다. 에르다는 아무 말도 하지 않는다. 그러나 플레이어는 무언가를 느껴야 한다.

**기억 오브젝트 근접 시 사운드 이벤트:**
- 배경 BGM이 점차 낮아지며 앰비언트 레이어로 이행
- 해당 아이템의 이전 주인과 관련된 "소리의 잔향"이 짧게 재생됨 (예: T-HOME에서 아이 웃음소리 1초, T-MILITARY에서 불명확한 군인의 목소리)
- 에르다 발소리가 잠시 느려짐 (플레이어 이동 속도와 무관한 발소리 레이어)
- 에코가 낮은 공명음을 냄 ("감지했다")

이것이 대사 없이 "에르다가 이 기억을 인식했다"를 전달하는 오디오 내러티브다.

## 8. PA 오디오 아키텍처 요약

### 8-1. 오디오 레이어 구조

```
L1: BGM (탐험/전투 2레이어, 크로스페이드)
L2: 앰비언트 루프 (기저/거리이벤트/근접트리거 3레이어)
L3: 에르다 이동 SFX (발소리, 점프, 착지, 대시)
L4: 에코 상태 SFX (허밍, 스캔, 인챈트)
L5: 전투 SFX (슬래시, 임팩트, 피격반응, 킬)
L6: 게임플레이 SFX (아이템 획득, 스탯게이트, 레어리티드랍)
L7: UI SFX (메뉴, 커서, 장착)
```

### 8-2. 우선 구현 순서 (Phase 1 기준)

CMB-07-R~V (HitFeedback SFX 대기 항목)를 포함한 Phase 1 오디오 구현 우선순위:

1. **L5 전투 SFX — 검 4종** (슬래시, 임팩트, 킬, 에코스트라이크) → 히트스탑과 동기화
2. **L3 에르다 발소리** (금속 격자 2종) → 존재감 확립
3. **L1 BGM** — 세이브 포인트, Tier 3 탐험, 아이템계 기본 3트랙
4. **L2 앰비언트** — 세이브 포인트 1종 (세계관 첫 인상)
5. **L6 아이템계 진입 SFX** — Normal 레어리티 1종
6. **L7 UI SFX** — 메뉴, 선택, 장착 최소 세트
