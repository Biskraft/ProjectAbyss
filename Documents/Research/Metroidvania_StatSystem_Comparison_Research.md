## 1. 리서치 개요 및 프레임워크

### 1.1. 조사 기준

각 게임에 대해 다음 5가지 항목을 기준으로 분석한다.

| 항목 | 설명 |
| :--- | :--- |
| **스탯 종류 및 수** | 어떤 스탯이 몇 개 존재하는가 |
| **게임플레이 영향** | 스탯이 전투 공식과 탐험 게이팅에 어떻게 작용하는가 |
| **장비 연동** | 장비가 스탯과 어떻게 상호작용하는가 |
| **작동 이유 분석** | 무엇이 이 시스템을 효과적으로 만드는가 |
| **ECHORIS 적용점** | 무엇을 참고/회피해야 하는가 |

### 1.2. 평가 기준 프레임워크

**MDA 관점에서 스탯 시스템을 평가한다.**

- **Mechanics**: 스탯의 규칙은 명확한가? 공식이 투명한가?
- **Dynamics**: 어떤 플레이 패턴을 만들어내는가? 빌드 다양성이 존재하는가?
- **Aesthetics**: 플레이어가 무엇을 느끼는가? 성장감, 통제감, 탐험 욕구?

## 2. 핵심 메트로베니아 — RPG 스탯 계열

### 2-1. Castlevania: Symphony of the Night (월하의 야상곡, 1997)

**ECHORIS의 1차 레퍼런스. 이 게임이 스탯 시스템의 출발점이다.**

#### 스탯 종류 (총 6개)

| 스탯 ID | 이름 | 역할 |
| :--- | :--- | :--- |
| STR | Strength (힘) | 물리 공격력 계산에 더해짐 |
| CON | Constitution (체력) | 방어력(DEF) 결정, 저주/독 지속시간 저항 |
| INT | Intelligence (지력) | 마법/서브 무기 대미지 결정 |
| LCK | Luck (행운) | 아이템 드랍률, 크리티컬 확률 결정 |
| ATT | Attack (공격력) | STR과 장비 최대 ATT를 합산한 최종 공격 수치 |
| DEF | Defense (방어력) | CON에서 파생, 피해 감소량 |

> 주의: ATT와 DEF는 파생 수치이며 플레이어가 직접 투자하는 원본 스탯은 STR/CON/INT/LCK 4개다.

#### 스탯 성장 메커닉

- 레벨 업 시 STR, CON, INT, LCK은 각각 0, 1, 2 중 하나씩 상승
- 한 스탯이 2 오르면 나머지는 모두 0 (제로섬 성장 메커닉)
- MP는 레벨당 4 또는 5 상승

#### 전투 공식 (ATT 계산)

```
// STR < 장비 Max ATT 경우
ATT = STR + (MaxATT - STR / 2)

// STR >= 장비 Max ATT 경우
ATT = STR + MaxATT
```

이 공식의 함의:
- STR이 낮을 때 강한 무기를 들면 페널티 발생 (ATT 수치 50% 패널티)
- STR이 장비 요구치를 초과하면 장비 ATT를 100% 활용
- 즉, **장비 + 스탯의 이중 게이팅**이 자연스럽게 발생

#### 방어 공식

```
Damage Taken = Enemy_ATT - DEF
```

DEF 1 = 피해 1 감소. DEF 100 도달 시 적과 접촉해도 피격 경직 발생하지 않음.

#### 장비 연동

- 각 무기에는 Max ATT가 정의되어 있음
- 방어구는 DEF 수치 직접 증가
- 특수 장비: 특정 원소 내성, 특수 능력 부여
- 장비 착용 요구치(Min STR 등) 없음 — 착용 후 스탯 보정으로 조절

#### 게이팅 방식 분석

SotN은 **능력 게이팅이 주, 스탯 게이팅이 사실상 없다.** 이중 점프, 변신, 벽 타기 같은 능력이 탐험을 열고, 스탯은 단순히 전투 쾌적도를 높이는 역할.

| 게이팅 유형 | 존재 여부 | 비고 |
| :--- | :---: | :--- |
| 능력 게이팅 | O | 핵심 진행 구조 |
| 스탯 게이팅 | X (사실상 없음) | 낮은 스탯으로도 진입 가능, 난이도 차이만 발생 |
| 레벨 게이팅 | X | 모든 구역 언제든 진입 가능 |

#### 작동 이유

1. **STR-ATT 공식의 자연스러운 장비 요구**: 공식 자체가 "이 무기를 최대 활용하려면 STR이 필요해"를 만들어냄. 인위적 잠금이 아닌 수학적 손익 구조.
2. **LCK의 이중 역할**: 전투(크리티컬)와 성장(드랍률) 양쪽에 영향. 투자 이유가 2개인 스탯.
3. **CON의 질감 변화**: 단순 HP 증가가 아닌 저주/독 저항까지 맡아 CON 투자에 의미 부여.

#### 문제점

- 스탯 분배 자동화(레벨 업 시 RNG 분배)로 플레이어의 스탯 빌드 통제권이 약함
- INT가 마법 사용자에게만 유효 — 물리 특화 시 3개 스탯(STR/CON/LCK)만 의미 있음
- 스탯 게이팅이 없어 탐험 구조가 순전히 능력에 의존

### 2-2. Bloodstained: Ritual of the Night (2019)

**SotN의 정신적 후계작. IGA 감독 작품으로 스탯 시스템도 SotN을 계승하되 더 복잡하게 발전.**

#### 스탯 종류 (총 9개)

| 스탯 ID | 이름 | 역할 |
| :--- | :--- | :--- |
| STR | Strength (힘) | 물리 근접 무기 성능 |
| INT | Intelligence (지력) | 마법 주문 성능 |
| CON | Constitution (체력) | 물리 피해 감소 |
| MND | Mind (정신력) | 마법 피해 감소 |
| LCK | Luck (행운) | 크리티컬, 드랍률, 코인 가치 |
| DEF | Defense | CON에서 파생 |
| RES | Resistance | MND에서 파생 |
| HP | Hit Points | 레벨 및 CON에서 파생 |
| MP | Mana Points | 레벨 및 INT에서 파생 |

> SotN 대비 CON을 MND(정신력)로 분리해 물리/마법 방어를 이원화. 빌드 결정이 더 구체화됨.

#### SotN 대비 차이점

| 구분 | SotN | Bloodstained |
| :--- | :--- | :--- |
| 스탯 수 | 6개 | 9개 |
| 방어 이원화 | X (CON 하나) | O (CON + MND 분리) |
| 스탯 투자 방식 | 자동 (레벨 업 RNG) | 자동 (레벨 업) + 장비로 조정 |
| 게이팅 방식 | 능력 게이팅 중심 | 능력 게이팅 중심 |

### 2-3. Dead Cells (2018)

**로그라이크 메트로베니아. 런(run) 기반의 스탯 시스템으로 전통적 캐릭터 성장과 다른 접근법.**

#### 스탯 구조 (총 3개 스탯 컬러)

Dead Cells는 전통적 스탯 대신 **빌드 정체성을 결정하는 컬러 스탯** 시스템을 채택한다.

| 스탯 컬러 | 이름 | 빌드 정체성 | 대표 무기 유형 |
| :--- | :--- | :--- | :--- |
| 빨강 | Brutality (잔인함) | 근접, 고속 DPS | 대부분의 근접 무기 |
| 보라 | Tactics (전술) | 원거리, 군중 통제 | 활, 총기, 트랩 |
| 초록 | Survival (생존) | 방어, 실드, 강타 | 쉴드, 방어 도구 |

#### 스케일링 공식

```
// 스탯 5 증가마다 DPS 약 2배
DPS_Multiplier = f(StatValue) [로그함수적 스케일]

// Survival 스탯과 HP 관계
ExtraHP = 70% * (Survival - 1)  // 두 번째 Survival 레벨부터
```

#### 스탯 획득 방법

1. **두루마리 of Power (Scroll of Power)**: 단색 — 선택한 스탯 +1
2. **두루마리 듀얼 (Dual Scroll)**: 두 가지 스탯 중 하나 선택
   - Assassin's: Brutality/Tactics
   - Minotaur's: Brutality/Survival
   - Guardian's: Tactics/Survival
3. **부적 (Amulet)**: 착용 시 해당 스탯 +1~+4

#### 핵심 설계 원리: "빌드 정체성"

- 모든 무기에 컬러 태그가 있음 (Brutality 무기, Tactics 무기 등)
- 해당 컬러 스탯이 높을수록 그 무기의 피해량 증가
- **듀얼 스크롤은 낮은 스탯 쪽으로 더 자주 출현** — 빌드 수정/전환 기회를 제공
- 고수 플레이: 처음부터 스탯 집중 방향 결정 → 일관된 빌드 구성

#### 작동 이유

1. **직관적 색상 코딩**: "빨간 무기는 빨간 스탯에 강함"을 시각적으로 즉시 전달
2. **런 내 유연성**: 영구 성장 없이 런마다 다른 빌드 — 반복 플레이 동기 내재화
3. **강제 트레이드오프**: 세 스탯을 고르게 올리면 모두 약해짐. 집중이 강요됨

#### 게이팅 방식

Dead Cells는 **파워 셀(Boss Cell)로 난이도를 게이팅**하며 스탯 자체는 게이트 역할을 하지 않는다. 스탯은 오로지 전투 출력 향상에만 집중.

### 2-4. Blasphemous 1 & 2 (2019, 2023)

**다크 고딕 메트로베니아. 전통적 RPG 스탯 최소화, 독창적인 Guilt 시스템 채택.**

#### 스탯 구조

Blasphemous는 전통적 스탯 수치 시스템을 거의 사용하지 않는다.

| 리소스 | 역할 | 특징 |
| :--- | :--- | :--- |
| HP (Health) | 생명력 | 특정 아이템으로 최대치 증가 |
| Fervour (열기) | 스킬/기도 사용 자원 | 적 공격 시 축적, 특정 아이템으로 최대치 증가 |

#### Guilt 시스템 (Blasphemous 2)

Blasphemous 2의 핵심 독창적 메커닉. **사망 → 스탯 페널티 → 고위험 고보상 루프**를 만든다.

```
죽을 때마다 Guilt 축적
Guilt 레벨 상승 → DEF/저항 감소
Guilt 상태에서 적 처치 → 더 많은 Tears of Atonement (재화) + Martyrdom Points 획득
죄의 단편 회수 or 고해(Confess) → Guilt 초기화, Fervour 최대치 복구
```

**위험-보상 메커닉 분석:**
- 높은 Guilt = 더 큰 리스크(피해 증가) + 더 큰 리턴(재화 + 포인트 증가)
- 플레이어가 의도적으로 사망 후 고죄 상태 유지를 선택할 수 있음
- Dark Souls의 인간성/저주 시스템과 유사하나 리워드 극대화에 초점

#### 장비 연동

- **Rosary Bead**: 여러 개 장착 가능한 액세서리. 원소 저항, 특정 공격 시 피해 증가 등 다양한 효과
- 장비가 스탯 수치를 직접 올리는 것이 아닌 **특정 상황 강화** 방식

#### 작동 이유 / 실패 요인

**작동하는 이유:**
- Guilt 시스템이 사망을 "실패"가 아닌 "전략적 선택"으로 재정의 — 블랙소울 피로감 해소
- 장비가 수치 올리기가 아닌 빌드 개성 부여 — 숫자 보다 텍스처

**한계:**
- Fervour/HP 두 리소스 외 성장감이 빈약 — 야리코미 욕구를 자극하지 못함
- 장비 다양성이 부족해 "최강 조합"이 금방 수렴됨

### 2-5. Ender Lilies / Ender Magnolia (2021, 2025)

**영혼 소환 메커닉을 중심으로 한 소울라이크 메트로베니아. 스탯은 미니멀하나 장비(Spirit) 다양성이 핵심.**

#### 스탯 구조

| 스탯 | 역할 |
| :--- | :--- |
| ATK (공격력) | Spirit(영혼) 피해량 기반. 레벨 업 시 자동 증가 |
| HP (생명력) | 레벨 업 시 자동 증가 |
| Relic Slots | 유물(Relic) 장착 슬롯 수. 탐험으로 Chain of Sorcery를 찾아 확장 |

> 최대 레벨: 100. STR/INT/DEX 같은 세분화 스탯 없음.

#### Spirit 시스템 (사실상의 장비 시스템)

- 물리친 보스와 미니보스의 영혼을 최대 6개 동시 장착
- 각 Spirit는 고유 공격 패턴, 개별 쿨다운, 업그레이드 경로를 가짐
- 업그레이드 재료: Stagnant Blight, Furious Blight (적 드랍)
- "이른 시기 Spirit이 후반에도 쓸모있음" — 수직 성장 유지

#### Relic 시스템

- 여러 Relic 동시 장착 가능 (슬롯 수에 제한)
- Relic은 패시브 보너스 부여: 원소 저항, 특정 Spirit 강화, 생명력 추가 등
- 슬롯 수 자체가 성장 지표 — Chain of Sorcery 찾기 = 탐험 동기

#### 게이팅 방식

| 게이팅 유형 | 방식 |
| :--- | :--- |
| 능력 게이팅 | 특정 Spirit 획득으로 봉인 해제 |
| 스탯 게이팅 | 없음 (ATK/HP는 전투 쾌적도만 조절) |

#### 작동 이유

1. **Spirit가 "장비이자 스킬"**: 장비 교체가 곧 전투 스타일 전환. 스탯 숫자 없이도 빌드 정체성 부여
2. **Relic Slot 확장 = 유형적 탐험 보상**: 슬롯 +1은 즉시 빌드 가능성 확장 → 탐험 동기 명확
3. **조기 Spirit의 지속 가치**: 아이템을 버리지 않아도 되는 구조 — 애착 형성

## 3. 스탯 없는 메트로베니아 계열

### 3-1. Hollow Knight (2017)

**전통적 스탯이 전혀 없는 메트로베니아의 대표작. 대신 Charm + Notch 시스템으로 커스터마이징 제공.**

#### 리소스 구조

| 리소스 | 역할 | 성장 방식 |
| :--- | :--- | :--- |
| HP (Masks) | 생명력. 최대 9개까지 증가 | 탐험/구매로 Mask Shard 수집 |
| Soul | 스킬 사용 자원 | 공격 시 자동 축적 |
| Notches | 참 장착 슬롯 수 | 탐험으로 총 11개 확보 가능 |

#### Charm 시스템 (스탯을 대체하는 커스터마이징)

- 총 45개 Charm 존재 (한 번에 최대 40개 보유)
- 각 Charm은 1~4 Notch 소모
- 가능한 조합: **약 446만 가지** (수학적 계산 기준)

#### Overcharming (오버참밍)

```
// 슬롯 초과 장착 시
조건: 장착된 Charm의 총 Notch 소모량 > 현재 Notch 수
효과: 받는 피해 2배
장점: 슬롯 이상의 강력한 빌드 가능

활성화 조건: 슬롯 초과 시도를 5번 반복
```

**게임 디자인 관점에서 Overcharming은 "고위험 고보상"의 완벽한 구현.**

#### 게이팅 방식

- **완전 능력 게이팅**: Monarch Wings(이중 점프), Mantis Claw(벽 타기), Crystal Heart(가속 비행) 등
- 스탯 게이팅: **없음**

#### 작동 이유

1. **스탯 수치 제거 = 인지 부하 최소화**: 플레이어는 "STR 15→16" 같은 추상적 수치 대신 "Quickslash 참을 달았으니 빠르게 공격됨"이라는 직관적 결과에 집중
2. **Notch = 경제 시스템**: 슬롯이 화폐처럼 기능. "이 참을 낄까 저 참을 낄까"의 트레이드오프
3. **능력 게이팅의 '아하!' 순간**: STR이 2 오른 것이 아닌 "이제 이중 점프됨"이라는 명확한 기능 해금이 탐험 쾌감 극대화

#### Hollow Knight로부터 얻는 교훈

> 스탯 시스템 없이도 높은 빌드 복잡도와 탐험 동기를 동시에 달성할 수 있다.
> 핵심은 "숫자의 성장"이 아닌 "가능성의 확장"이다.

## 4. 장비 중심 액션 RPG 계열

### 4-1. Diablo 2 (2000)

**ECHORIS 장비 레어리티 시스템의 직접 레퍼런스. "장비 스탯 요구치" 개념의 원형.**

#### 스탯 구조 (총 4개 — 플레이어 직접 배분)

| 스탯 | 역할 | 주요 투자 이유 |
| :--- | :--- | :--- |
| Strength (힘) | 장비 착용 요구치, 물리 피해 +1%/포인트 | 고급 장비 착용 최소 투자 |
| Dexterity (민첩) | 명중률, 원거리 피해, 방어 확률 | 궁수/투척 빌드 |
| Vitality (체력) | HP, 스태미나 | **대부분의 빌드에서 가장 많이 투자** |
| Energy (에너지) | 마나 | 약한 스탯 — 마나 회복 아이템이 더 효율적 |

레벨 업마다 5 포인트 배분. Act 3 퀘스트 완료 시 +5 포인트 추가 (최대 3회, 15 포인트).

#### 장비 요구치 시스템 (ECHORIS 스탯 게이팅의 원형)

```
장비 착용 가능 조건:
  Character_STR >= Item_MinSTR
  Character_DEX >= Item_MinDEX
  Character_Level >= Item_MinLevel
```

**이것이 Diablo 2 스탯 시스템의 핵심이다.** 스탯은 전투 성능보다 "더 좋은 장비를 입기 위한 키"로 주로 기능한다.

#### 작동 이유

1. **STR/DEX는 장비 잠금을 여는 키**: "이 장비를 쓰려면 STR 156이 필요해" → 명확한 성장 목표 제시
2. **VIT 집중의 자연스러운 수렴**: STR/DEX는 최소한만 투자, 나머지는 VIT → 단순하고 강력한 의사결정 구조
3. **Energy의 안락사**: 의도적으로 에너지를 약하게 만들어 아이템 의존도 증가 → 파밍 동기 강화

#### 문제점

- Energy 스탯이 사실상 무가치 — 나쁜 스탯의 존재가 시스템 신뢰도 저하
- STR/DEX가 장비 요구치 충족 이후 가치 급락 — 레이트게임 스탯 투자 의미 약화

### 4-2. Diablo 4 (2023)

**Diablo 2의 4 스탯 체계를 유지하되, 각 스탯의 역할을 클래스별로 다르게 정의. 더 유연하고 의미있는 스탯 설계.**

#### 스탯 구조 (총 4개 + 클래스별 보너스)

| 스탯 | 공통 효과 | 클래스 보너스 예시 |
| :--- | :--- | :--- |
| Strength (힘) | 방어도 +1/포인트 | Barbarian: 스킬 피해 +0.1%/포인트 |
| Intelligence (지력) | 전 저항 +0.05%/포인트 | Sorcerer/Necromancer: 스킬 피해 +0.1%/포인트 |
| Willpower (의지) | 회복량 +0.1%, 과부하 피해 +0.25%/포인트 | Druid: 스킬 피해 +0.1%/포인트 |
| Dexterity (민첩) | 회피 확률 +0.025%/포인트 | Rogue: 스킬 피해 +0.1%/포인트 (나머지는 크리티컬) |

레벨 업 시 4개 스탯 **자동으로 각 +1**. 장비에서 개별 스탯 or 전체 스탯 롤 획득.

#### D2 대비 개선점

1. **쓸모없는 스탯 제거**: 모든 스탯이 클래스별로 1개씩은 "스킬 피해 증가"에 기여 — Energy처럼 버리는 스탯이 없음
2. **공통 효과 + 클래스 효과 이원화**: 클래스가 다르면 같은 스탯이 다른 의미를 가짐 → 재플레이 가치
3. **장비 스탯 롤이 핵심**: 기반 스탯보다 장비에서 굴리는 스탯 접두사가 빌드를 결정

### 4-3. Path of Exile (2013~)

**가장 복잡한 스탯 시스템. 스탯 자체보다 "스탯이 장비 착용 요건을 결정하는 열쇠" 설계의 극한.**

#### 스탯 구조 (총 3개 속성)

| 속성 | 기본 효과 | 장비 요건 역할 |
| :--- | :--- | :--- |
| Strength (힘) | STR 1 = 생명력 +2 | 전사 계열 장비/스킬 젬 착용 조건 |
| Dexterity (민첩) | DEX 1 = 정확도 +5 | 민첩 계열 장비/스킬 젬 착용 조건 |
| Intelligence (지력) | INT 1 = 마나 +2 | 지식 계열 장비/스킬 젬 착용 조건 |

> **핵심**: PoE에서 STR/DEX/INT의 직접 효과(생명력+, 정확도+, 마나+)는 **부차적**이다. 이 속성의 진짜 역할은 **패시브 트리 진입과 장비/스킬 젬 착용 요건 충족**이다.

#### PoE의 속성 시스템 설계 원리

```
// 장비/스킬 젬에 최소 속성 요건 존재
Red Skill Gem: min_STR >= X
Green Skill Gem: min_DEX >= X
Blue Skill Gem: min_INT >= X

// 패시브 트리 영역이 속성으로 구분됨
Left Area: STR 노드 밀집
Right Area: DEX 노드 밀집
Top Area: INT 노드 밀집
```

#### 속성 변환 (Attribute Conversion)

특정 패시브 주얼을 패시브 트리에 박으면 해당 구역의 속성을 다른 속성으로 변환 가능. 예: STR → DEX로 변환하는 주얼을 사용.

이것이 PoE 빌드 복잡성의 핵심 축이다.

#### 작동 이유 / 복잡도 경고

**작동하는 이유:**
- 속성이 패시브 트리 탐색 비용 역할 → "이 방향으로 트리를 짜면 이 속성이 올라가고, 그 속성으로 이 장비/젬을 쓸 수 있음"이라는 긴 인과 사슬
- 속성 변환으로 크로스-아키타입 빌드 가능

**ECHORIS에 적용 시 경고:**
- PoE의 복잡도는 수백 시간의 플레이 기반 위에서 작동함
- ECHORIS의 미드코어 타겟에는 이 수준의 간접 연결 구조가 과다 복잡도로 작용할 위험

## 5. 야리코미 레퍼런스: Disgaea 시리즈

**ECHORIS 아이템계 시스템의 1차 레퍼런스. 스탯 극대화가 게임의 목표 자체인 야리코미 구조.**

#### 스탯 구조 (핵심 6개)

| 스탯 ID | 이름 | 역할 |
| :--- | :--- | :--- |
| ATK | Attack (공격력) | 물리 공격 피해량 증가 |
| DEF | Defense (방어력) | 물리 피해 감소 |
| INT | Intelligence (지력) | 마법 공격 피해량 증가 |
| RES | Resistance (저항력) | 마법 피해 감소 |
| SPD | Speed (속도) | 행동 순서, 회피 |
| LCK | Luck (행운) | 크리티컬, 회피 |

레벨 업 + **장비 적성(Aptitude)** + **무기 마스터리(Weapon Mastery)** + **아이템계 기억 단편(Memory Shard)**의 3중 배율 시스템.

#### Memory Shard 시스템과 스탯 연동

```
// 최종 스탯 계산 (단순화)
FinalStat = BaseStat * LevelMultiplier * AptitudeMultiplier * WeaponMasteryMultiplier + Memory ShardBonus

// Memory ShardBonus 예시
각 기억 단편: ATK +2% (기본값)
기억 단편 복종 완료 후 스택: ATK +20%~수백%
```

기억 단편 개별 보너스는 작게 시작하지만 스택이 쌓이면 수백 배 차이로 귀결된다. 이것이 아이템계 야리코미의 수학적 기반이다.

#### Disgaea 스탯 규모

Disgaea에서 스탯은 **레벨 1~9999**에 걸쳐 지수 성장한다. 최고 레벨 캐릭터의 ATK가 수백만에 달하는 것은 버그가 아닌 의도된 디자인이다.

| 단계 | 스탯 규모 | 게임플레이 단계 |
| :--- | :--- | :--- |
| Lv 1~100 | 수백~수천 | 일반 스토리 진행 |
| Lv 100~1000 | 수만 | 아이템계 진입, 기억 단편 수집 |
| Lv 1000~9999 | 수십만~수백만 | 극한 야리코미, 모든 기억 단편 복종 |

#### Memory Shard 유형 (Disgaea 7 기준)

| 유형 | 강화 스탯 |
| :--- | :--- |
| Foundation Memory Shards | HP or SPD |
| Physical Memory Shards | ATK or DEF |
| Brain Memory Shards | INT or RES |
| Ambition Memory Shards | 전체 스탯 조합 보너스 |

#### 작동 이유

1. **수치 극대화 자체가 목표**: 디스가이아에서 플레이어는 "더 강해지는 것"이 목적이 됨. 야리코미의 정수.
2. **기억 단편 = 복잡한 최적화 퍼즐**: 어떤 기억 단편를 어떤 슬롯에 넣느냐가 전략적 깊이 생성
3. **레어리티가 기억 단편 슬롯 수를 결정**: 고레어리티 = 더 많은 기억 단편 슬롯 = 더 큰 성장 잠재력 → ECHORIS에 직접 채택된 구조

## 6. Hades / Hades 2 — 런-베이스드 빌드 시스템

**전통적 스탯 없이 "Boon(축복)" 조합으로 빌드를 형성하는 독창적 시스템.**

#### 스탯 구조

Hades는 전통적 STR/DEX/INT 스탯이 없다. 대신:

| 리소스 | 역할 | 영구 성장 여부 |
| :--- | :--- | :--- |
| HP | 생명력 | Mirror of Night으로 기본 HP 영구 증가 |
| Darkness | 영구 업그레이드 재화 | 런 간 지속 |
| Boon | 런 내 임시 강화 | 런 종료 시 소멸 |

#### Boon 시스템 (스탯을 대체하는 신의 축복)

- 각 방 클리어 후 3개의 Boon 중 하나 선택
- Boon은 신(Zeus, Poseidon, Athena 등)에 따라 계열이 다름
- **시너지 조합**: 같은 신 Boon을 여러 개 모으거나, 특정 신 조합이 특수 효과 발동

```
// 피해 스케일링 예시
느린 무기에는 퍼센트 증가형 Boon이 유리 (Aphrodite, Apollo 계열)
빠른 무기에는 추가 피해원 Boon이 유리 (Zeus, Hephaestus 계열)
```

#### 영구 성장 (Mirror of Night)

스탯이 아닌 **키 패시브 효과**를 잠금 해제:
- 최대 HP +25
- 어둠 획득량 +20%
- 대쉬 능력 개선
- 등 25가지 패시브

이것이 Hades의 "런간 성장감"을 담당한다.

#### 작동 이유

1. **시너지 발견의 쾌감**: "Zeus + Poseidon 조합이 이렇게 강하다니!"의 발견 Aesthetic
2. **런마다 다른 빌드**: 무작위 Boon 제공 → 매 런이 다른 경험
3. **단순한 수치 없이 복잡한 표현**: HP 수치를 올리지 않고도 "강해진 느낌"을 Boon의 질적 변화로 전달

## 8. 핵심 인사이트 및 트렌드 분석

### 8-1. 현대 메트로베니아의 스탯 트렌드

**트렌드 1: 스탯 수치 최소화 또는 제거**

2015년 이후 성공한 메트로베니아의 다수가 전통적 STR/DEX/INT 스탯을 축소하거나 제거했다.

- Hollow Knight (2017): 스탯 없음, Charm으로 대체
- Celeste (2018): 스탯 없음, 순수 능력 게임
- Ori 시리즈: 스탯 최소화
- Blasphemous (2019): HP/Fervour만 존재

이유: **수치 성장이 "진짜 성장"을 희석시킨다.** "공격력이 15에서 16이 됐어"보다 "이제 이중 점프가 됐어"가 훨씬 강력한 탐험 쾌감을 준다.

**트렌드 2: 장비/스킬 시스템으로 복잡도 흡수**

스탯 대신 장비, 차암, Spirit, Boon이 커스터마이징을 담당한다. 숫자 대신 "무엇을 할 수 있는가"가 성장 지표.

**트렌드 3: Knowledge Gating (지식 게이팅) 부상**

Outer Wilds, Return of the Obra Dinn 등 "메트로이드브레이니아" 장르의 출현. 스탯도 능력도 아닌 "이 비밀을 알고 있느냐"로 진행 결정. ECHORIS의 직접 레퍼런스는 아니나 트렌드로 주시 필요.

### 8-2. 스탯 게이팅 vs 능력 게이팅: 플레이어 경험 차이

| 구분 | 스탯 게이팅 | 능력 게이팅 |
| :--- | :--- | :--- |
| 성장 표현 | 점진적 수치 상승 ("STR 85 → 120") | 순간적 기능 해금 ("이중 점프 획득!") |
| 플레이어 감정 | 파밍 달성감, 야리코미 만족감 | "아하!" 발견 쾌감, 공간 재해석 |
| 적합 장르 | 파밍 RPG, 야리코미 게임 | 순수 메트로베니아 |
| 위험 요소 | 그라인드 피로감, 게이트 답답함 | 능력 없으면 완전 차단, 죽었다 깨나도 불가 |
| SDT 충족 | Competence (능력 성장), Achievement | Autonomy (탐험 선택), Discovery |

**ECHORIS의 이중 게이팅 설계는 이 두 경험을 동시에 제공하려는 시도다.**
능력 게이팅: "이 능력이 있어야 갈 수 있음" (SotN 정통 MV)
스탯 게이팅: "이만큼 강해야 갈 수 있음" (디스가이아 야리코미 동기)

### 8-3. Diablo 2 장비 요구치 시스템 분석

Diablo 2의 장비 요구치는 **스탯 게이팅의 가장 성공적인 구현 사례**다.

성공 이유:
1. **명확한 숫자**: "이 갑옷을 입으려면 STR 156"이라는 명시적 목표 제시
2. **트레이드오프 강제**: STR에 투자할지 VIT에 투자할지 결정
3. **장비가 목표가 됨**: 파밍의 최종 목적이 스탯 수치가 아니라 "이 장비 착용"

ECHORIS 스탯 게이팅과의 차이:
- D2: 장비 착용 조건 (인벤토리 → 캐릭터 방향)
- ECHORIS: 월드 탐험 조건 (장비 스탯 → 월드 게이트 방향)

## 9. ECHORIS 설계 시사점

### 9-1. 현재 ECHORIS 설계의 강점 (레퍼런스 검증 완료)

| 설계 결정 | 근거 게임 | 유효성 |
| :--- | :--- | :---: |
| STR/INT/DEX/VIT/SPD/LCK 6개 스탯 | SotN의 4코어 + 확장, Disgaea의 6코어와 일치 | 검증됨 |
| FinalStat = Base + Equip + Memory Shard 공식 | Disgaea의 3중 배율 구조 단순화 버전 | 검증됨 |
| 비-크리티컬 경로에만 스탯 게이팅 배치 | D2 장비 요건과 SotN 능력 게이팅의 하이브리드 | 검증됨 |
| 레어리티가 기억 단편 슬롯 수 결정 | Disgaea 고레어리티 무기 = 더 많은 기억 단편 슬롯 | 직접 채택 |
| 스탯 게이트에 숫자 명시 ("STR 120 필요") | D2 장비 요구치의 투명성 원칙 | 검증됨 |

### 9-2. 주의해야 할 위험 요소

**위험 1: 스탯 게이팅과 능력 게이팅의 밸런스**

레퍼런스 게임들이 보여주는 트렌드: 메트로베니아에서 스탯 게이팅은 **탐험 흐름을 방해**할 위험이 있다. "저 구역에 가려면 STR 120이 필요해"는 "저 구역에 가려면 이중 점프가 필요해"보다 탐험 동기를 약화시킬 수 있다.

**권고사항**: 비-크리티컬 경로 스탯 게이팅 원칙을 엄격히 유지. 스탯 게이팅 구역의 보상을 능력 게이팅으로 얻은 보상보다 더 가시적으로 티징.

**위험 2: SPD 스탯 이원화 혼란**

SotN과 Bloodstained에서 SPD와 무브먼트 스피드는 직접 연결되지 않는다 (SotN은 이동속도 장비 효과이지 SPD 스탯이 아님). ECHORIS의 SPD 스탯이 "플랫폼 타이밍 게이팅" 역할을 하는 경우, SPD가 이동속도인지 액션 반응속도인지 플레이어에게 명확히 전달 필요.

**위험 3: 스탯 6개의 "버려지는 스탯" 문제**

D2에서 Energy 스탯이 사실상 버려진 것처럼, 6개 스탯 중 일부가 특정 빌드에서 완전히 무관할 경우 시스템 신뢰도 저하. **각 스탯이 최소 2개 이상의 탐험 게이팅에 연결되어야 한다.**

**위험 4: 기억 단편 복잡도 조기 노출**

Disgaea의 기억 단편 시스템은 수십~수백 시간 플레이어를 대상으로 한다. ECHORIS에서 기억 단편 최적화가 초반 30분 내에 요구될 경우 온보딩 장벽이 됨. **기억 단편는 심화 콘텐츠로 도입 시점을 Phase 2 이후로 설계함이 타당.**

### 9-3. 채택 권장 요소

| 채택 요소 | 출처 게임 | 이유 |
| :--- | :--- | :--- |
| 스탯 게이트에 "현재값 vs 필요값" 비교 UI | D2 장비 요건 UX | 목표 명확화로 야리코미 동기 강화 |
| LCK 스탯의 드랍률 + 크리티컬 이중 역할 | SotN의 LCK | 투자 이유가 2개 → 버려지는 스탯 방지 |
| 고위험 고보상 Guilt 스타일 메커닉 옵션 | Blasphemous 2 | 어려운 구역을 무리하게 도전할 이유 부여 |
| Spirit/Charm 형태의 능력 장비 시스템 | Hollow Knight, Ender Lilies | 능력 게이팅의 "아하!" 쾌감 유지 |
| 런 내 빌드 정체성 (Dead Cells 컬러 스탯) | Dead Cells | 아이템계 내 빌드 선택 구조에 적용 가능 |

### 9-4. 최종 검증: ECHORIS 6개 스탯의 유효성

현재 설계된 6개 스탯(STR/INT/DEX/VIT/SPD/LCK)을 레퍼런스와 비교하면:

| 스탯 | SotN 대응 | Disgaea 대응 | Diablo 2 대응 | 게이팅 표현 | 판정 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| STR | STR | ATK | Strength | 파괴 가능한 벽, 바리케이드 | 유효 |
| INT | INT | INT | Intelligence | 마법 봉인, 룬 장벽 | 유효 |
| DEX | (없음) | SPD 일부 | Dexterity | 타이밍 플랫폼, 기어 | 유효 |
| VIT | CON | DEF/RES | Vitality | 독/용암/빙결 환경 | 유효 |
| SPD | (없음) | SPD | (없음) | 시간 제한 통로 | 주의 필요* |
| LCK | LCK | LCK | (없음) | 비밀 경로, 숨겨진 트리거 | 유효 |

> * SPD 스탯의 "시간 제한 통로" 게이팅은 레퍼런스 게임에 선례가 없어 플레이테스트 필수. SPD가 이동속도를 직접 높이는지 여부가 게이팅 느낌을 결정함.
