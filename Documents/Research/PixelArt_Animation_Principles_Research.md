## 1. 픽셀아트 애니메이션 핵심 원칙

### 1-1. 레퍼런스 게임 분석

**Celeste (Madeline) — 최소 프레임 최대 표현**

Celeste는 픽셀아트 애니메이션의 "효율 극대화" 사례다. Pedro Medeiros(Saint11)가 직접 참여했으며, 핵심 원칙은 다음과 같다.

- 달리기: 4프레임. 수십 프레임 사이클 없이도 리듬감 전달 가능
- 점프: 점프 중 스프라이트 변화를 최소화하고, 대신 파티클·화면 흔들림·속도 라인으로 공중감 전달
- 스쿼시 & 스트레치(Squash & Stretch): 대시, 점프, 벽 그랩 시 스프라이트 변형을 1~2프레임 동안 과장되게 적용. "의식하지 못할 정도"로 빠르지만, 제거하면 즉시 이질감 발생
- 결론: **프레임 수보다 타이밍과 키프레임 선택이 더 중요**

**Dead Cells — 3D → 2D 파이프라인**

Motion Twin은 3DS Max로 3D 모델을 제작하고, 자체 툴로 2D 스프라이트를 렌더링했다. 이 방식의 의미는 다음과 같다.

- 애니메이션 수정이 재드로잉 없이 가능 → 게임플레이 변화에 빠른 대응
- 30fps 애니메이션 기준 (60fps 게임에서 2프레임마다 1장)
- 핵심 키프레임 확정 후 보간 프레임 추가하는 방식 ("Key-first, Interpolate-second")
- 피격자만 히트스탑, 공격자는 계속 움직임 → 반응성 우선
- PA 적용: 스프라이트 제작 시 3D 파이프라인은 현실적이지 않으나, **키프레임 우선 드로잉 방식**은 그대로 채용 가능

**Hollow Knight — 프레임 바이 프레임, 무게감 우선**

Team Cherry의 핵심 원칙:

- 공격 모션에 1~2프레임 극단적 히트 포즈(Action Pose) 삽입 → 짧아도 임팩트 강함
- 회복(Recovery) 프레임에서 캐릭터가 오버익스텐드(overextend)된 자세 유지 → 취약감 시각화
- 히트스탑 자체가 매우 짧음(1~4프레임) — 반응성과 타격감 균형 최적화
- 피격 히트스탑: 피격자만 정지, 공격자는 계속 이동 → "후속 공격 기회" 직관적 인지

**Hyper Light Drifter — 픽셀 인상주의**

- 키프레임(apex 포즈)만 그리고, 나머지는 플레이어의 "인지적 갭 채우기"에 의존
- 평면 색상(flat color) 기반 + 대형 그라디언트 오버레이 → 고딕 감성 without 과도한 그라데이션 드로잉
- 대시: 잔상(afterimage) 2~3장 + 모션 블러 대체 이펙트로 속도감 표현

**Blasphemous — 고딕 픽셀아트의 교과서**

PA와 가장 가까운 톤 레퍼런스:

- 스프라이트에 "퓨리스트 픽셀아트" 원칙 적용: 모든 픽셀이 의도적
- 아웃라인: 1px 검은 아웃라인 + 선택적 컬러 아웃라인(빛을 받는 면 = 밝은 아웃라인)
- 캐릭터 비율: 5~6.5 헤드 (리얼리스틱, 치비 아님) — PA 에르다와 동일 방향
- 배경: 어두운 팔레트 + 스프라이트의 높은 대비로 실루엣 가독성 확보
- 애니메이션: 세밀한 옷감·망토 움직임이 중량감 부여 → **에르다의 앞치마·고글 움직임**에 직접 적용 가능

### 1-2. Saint11 픽셀아트 애니메이션 원칙 (Celeste 제작자)

Disney 12원칙을 픽셀아트에 적용한 Saint11의 핵심 교훈:

| 원칙 | 픽셀아트 적용법 | PA 우선순위 |
|:-----|:-----|:-----:|
| 스쿼시 & 스트레치 | 착지 시 1프레임 납작, 점프 시 1프레임 길쭉. 연속 2프레임 이상 유지 금지 | 필수 |
| 예비동작 | 공격 전 반대 방향으로 1~3프레임 | 필수 |
| 후속동작 | 공격 후 오버슈트 → 수렴 포즈 1~2프레임 | 필수 |
| 타이밍 | 프레임 수보다 각 프레임의 유지 시간(hold time)이 더 중요 | 필수 |
| 과장 | 액션 포즈를 "실제보다 30% 과장"해야 정상처럼 보임 | 중요 |
| 직선 vs 호형 동작 | 공격은 직선, 유기적 동작(호흡·걸음)은 호형 | 중요 |
| 보조 동작 | 에르다 앞치마 · 고글 끈 · 에코 잔상이 주 동작에 따라 반응 | 권장 |

## 3. 예비동작-행동-회복 (Anticipation-Action-Recovery)

### 3-1. 구조 정의

공격 애니메이션의 3상 구조는 픽셀아트 전투 가독성의 핵심이다.

```
[ 예비동작 ] → [ 행동 (히트박스 활성) ] → [ 회복 (취약) ]
  Anticipation      Action (Active)          Recovery (Vulnerable)
```

**예비동작 (Anticipation):**
- 공격 방향의 반대편으로 몸이 기울어짐
- "무기에 힘이 실리는" 느낌 전달
- 이것이 플레이어가 회피 타이밍을 잡는 유일한 단서

**행동 (Action):**
- 히트박스가 활성화되는 구간
- 가능하면 1~2프레임 유지 (짧을수록 임팩트 강함)
- 극단적 포즈(과장): 정상보다 30% 과장해야 "정상처럼" 보임

**회복 (Recovery):**
- 캐릭터가 오버익스텐드된 자세 — 취약함 시각적 표현
- 3타 피날레는 600ms 후딜이므로 이 구간을 애니메이션으로 명확히 표현 필수
- Idle로 복귀 전 중간 "정착 포즈" 1~2f 삽입

### 3-2. 공격 속도별 예비동작 프레임 가이드

| 공격 속도 | 무기 | 예비동작 프레임 | 시각적 텔레그래프 |
|:---------|:-----|:------:|:-----|
| 빠름 (Fast) | 단검 | 1~2f | 몸이 앞으로 기우는 자세 + 잔상 시작 |
| 보통 (Medium) | 검 | 2~3f | 검을 뒤로 당기는 동작 명확 |
| 느림 (Slow) | 대검 | 4~5f | 크게 뒤로 젖히기 + 지면 충격 파티클 |
| 보스 일반 | 모든 타입 | 6~8f | 기 모임 이펙트 + 소리 전조 |
| 보스 분노 | 모든 타입 | 10~14f | 화면 전체 영향. 색상 변화 필수 |

### 3-3. 적 공격 텔레그래프 원칙

"죽는 것이 부당하게 느껴지면 플레이어는 컨트롤러를 내려놓는다." — Dead Cells 팀 인터뷰

| 원칙 | 구현 방법 |
|:-----|:-----|
| 예비동작은 항상 명확해야 한다 | 약공격도 최소 2프레임 예비동작 |
| 범위 표시 | 광역 공격 전 바닥/공간에 VFX 경고 존 표시 |
| 색상 코딩 | 분노 패턴은 색상이 다른 에너지 이펙트로 구분 |
| 음향 전조 | 애니메이션보다 1프레임 앞서 차징 사운드 재생 |

## 4. 스프라이트 시트 규격

### 4-1. 캐릭터별 권장 스프라이트 크기

| 캐릭터 유형 | 캔버스 크기 | 실제 스프라이트 점유 | 타일 비율 | 비고 |
|:-----------|:----------:|:------------------:|:--------:|:-----|
| 에르다 (플레이어) | 48×64 px | 32×48 px | 3×4타일 | 앞치마·에코 포함. 캔버스 여백 필수 |
| 소형 적 (Slime, Bat) | 32×32 px | 16×16~20×20 px | 1×1타일 | 타일과 동일 크기대 |
| 중형 적 (Skeleton, Ghost) | 48×48 px | 32×40 px | 2~3타일 | 에르다와 유사 높이 |
| 대형 적 (GoldenMonster) | 64×64 px | 48×56 px | 3~4타일 | 에르다보다 1.5배 |
| 소형 보스 (아이템 장군) | 80×80 px | 64×64 px | 4×4타일 | |
| 중형 보스 (아이템 왕) | 128×128 px | 96×96 px | 6×6타일 | |

**핵심 원칙 — 캔버스 여백(Padding):**
스프라이트 캔버스는 실제 그림보다 최소 8px 이상 크게 설정한다. 이 여백에 무기 이펙트, 보조 동작(앞치마·에코)이 들어간다. 여백 없이 작업하면 이후 공격 범위 VFX와 충돌한다.

## 5. VFX 스프라이트 가이드라인

### 5-1. 히트 스파크 (Hit Spark)

현재 `HitSparkManager.spawn()`이 코드로 Graphics 생성. Phase 2에서 스프라이트로 교체 권장.

| 타입 | 캔버스 크기 | 프레임 수 | 총 지속 시간 | 구성 |
|:-----|:----------:|:--------:|:-----------:|:-----|
| Light Hitspark | 32×32 px | 4f | 약 180ms | 중심 플래시(1f) + 라인 스파크 방사(2f) + 잔광(1f) |
| Heavy Hitspark | 48×48 px | 6f | 약 250ms | 중심 폭발(2f) + 라인 스파크 방사(3f) + 잔광(1f) |
| Kill Hitspark | 64×64 px | 8f | 약 320ms | 대형 방사 + 파편 + 링 웨이브 |

**히트 스파크 색상 원칙 (사쿠라이 인용):**
"어두운 요소를 섞어야 밝은 이펙트가 눈에 띈다." 히트 스파크의 중심은 하이라이트(#FFFFFF), 외곽은 어두운 채도의 색상(#3A1A00 등)을 혼합한다. 순수 흰색 + 순수 검정이 아닌, 채도가 있는 어두운 색(dark saturated) 사용.

트레일은 알파값 100% → 0%로 선형 감쇠. 마지막 2프레임은 50% 이하 알파. 급격한 소멸(pop-off) 금지.

### 5-3. 대시 잔상 (Dash Afterimage)

대시 잔상은 에르다 스프라이트를 동일하게 복사하되 알파와 색상을 변환.

| 2번 | 25% | 청록 (#00B4B4) | 대시 방향 -24px |
| 3번 (가장 뒤) | 10% | 진청 (#005A7A) | 대시 방향 -36px |

잔상 색상에 청록을 사용하는 이유: 에르다의 작업복 색상과 연동되어 "에르다의 잔상"임을 직관적으로 전달. 또한 PA 세계의 기본 색조(청록+주황)와 일치.

## 6. 적 유형별 애니메이션 지침

### 6-1. 아키타입별 포즈 언어

애니메이션만으로 적의 행동 패턴을 전달한다. 플레이어는 적을 1~2초 관찰하고 "이 적을 어떻게 다뤄야 하는가"를 직관적으로 파악해야 한다.

| 적 | Idle 포즈 | 어택 자세 | 시각적 특이점 |
|:--|:---------|:---------|:------------|
| Skeleton | 약간 앞으로 기울어진 공격적 자세. 칼을 든 팔 미세 흔들림 | 칼을 뒤로 크게 당기는 예비동작 | 뼈대 클랙(clack) 느낌. 관절 끊기는 듯한 딱딱한 애니메이션 |
| Slime | 상하 팽창-수축. 둥글고 귀여운 위협감 없는 외형 | 점프 전 납작하게 눌리는 예비동작 | 점프 정점에서 길쭉하게 변형. 착지 시 퍼짐 이펙트 |
| Ghost | 유령 특유의 부유. 투명도 60~80% 교대 맥동 | 플레이어 향해 확 팽창하는 예비동작 | 공격 시 일시적으로 완전 불투명. 텔레그래프 명확 |
| GoldenMonster | 방패를 앞세운 굳건한 자세. 무게감 있는 발 구름 | 방패를 뒤로 젖히는 예비동작 (대형) | 황금빛 반짝임. 방패 엣지에 선택적 컬러 아웃라인 |
| Spark Bat | 날갯짓 2~3프레임 루프. 고도 미세하게 오르내림 | 플레이어 방향으로 급하강 예비동작 | 날개에서 파란 전기 파티클 상시 방출 |
| Cinder Imp | 뒤로 물러서는 소심한 자세. 작은 키 강조 | 화염구를 양손으로 모으는 예비동작 | 투사체 발사 후 뒤로 점프 도주 동작 |
| Memory Shard | 빠르게 좌우 두리번거림. 긴장한 포즈 | 없음 (도주 특화) | 발견되면 즉시 전력 도주. 작고 빠른 발걸음 |

### 6-2. 아키타입별 핵심 설계 원칙

**근접 돌격형 (Skeleton, GoldenMonster):**
- Idle에서 공격 자세가 이미 보여야 함 — "이 적은 나를 노린다"는 즉시 전달
- 예비동작이 공격 방향을 명확히 지시
- 회복 구간에 취약 포즈 유지 (측면 공격 기회 창)

**비행형 (Spark Bat):**
- 날갯짓은 2~3프레임 루프. 모든 비행 적의 호버 진폭: 상하 3~5px
- 공격 패턴: 직선 급하강 → 호형 복귀. 급하강 시 날개 접음 (1f)
- 전기 파티클: 상시 방출이지만 공격 전 밀도 증가 (텔레그래프)

**원거리 사격형 (Cinder Imp):**
- 항상 플레이어와 거리를 유지하려는 후퇴 동작이 Idle에서도 보임
- 차징 → 발사 → 도주 3단계 리듬을 명확히 시각화
- 투사체는 별도 스프라이트: vfx_fireball_cinder_16x16.png

**도주형 (Memory Shard):**
- 다른 적과 달리 공격 애니메이션 없음
- 도주 속도는 플레이어의 70~80% — "따라잡을 수 있을 것 같은" 긴장감
- 포획 시 반짝임 이펙트 + 작은 복종 포즈 (굴복하는 것 같은 짧은 애니메이션)

### 6-3. 사망 애니메이션 원칙

| 원칙 | 이유 |
|:-----|:-----|
| 최소 4프레임 | 너무 빠른 소멸은 "제거된 느낌" 미약 |
| 보스는 8프레임 이상 | 클라이맥스 연출. 스테이지 클리어 피드백 |
| 마지막 1~2프레임 알파 감쇠 | 팝-오프 없이 자연스러운 소멸 |
| 사망 시 파티클 방출 | 아이템 드롭 VFX와 연동 |
| 기억 단편 포획 사망 없음 | 포획 = 복종 애니메이션으로 대체 |

## 7. 일관된 아트 스타일 가이드라인

### 7-1. 팔레트 원칙

PA 비주얼 아이덴티티의 핵심: **청록(배경/심연) + 주황(구조물/단조열)** 이중 톤

| 영역 | 주 색상 | 보조 색상 | 금지 색상 |
|:-----|:-------:|:---------:|:---------:|
| 월드 배경 | 청록 계열 (#003040~#006080) | 회색 금속 (#404040~#808080) | 채도 높은 그린, 핑크 |
| 에르다 의상 | 청록 작업복 (#006080) + 주황 앞치마 (#C04000) | 회색 금속 부품 | 밝은 파스텔 |
| 적 | 각 테마 팔레트 | 공통 어두운 그림자 색 | 에르다와 동일 팔레트 충돌 금지 |
| UI | 청록 프레임 + 밝은 텍스트 | 레어리티 색상 | 빨간(플레이어 HP 제외) |
| VFX | 히트 스파크: 흰+어두운 채도 색 | 원소별 정의 색상 | 무채색 전용 VFX |

**팔레트 제한 규칙:**
각 스프라이트(캐릭터 단위)는 최대 16색을 사용한다. 이는 픽셀아트 가독성 원칙이자 SunnyLand 아틀라스가 준수한 방식이다. 그라데이션은 금지하되, 3단계 색조(하이라이트 / 미드톤 / 그림자)를 통한 선택적 셀 셰이딩은 허용.

### 7-2. 아웃라인 스타일

Blasphemous의 "퓨리스트 픽셀아트" 접근법을 채용한다.

| 유형 | 규칙 |
|:-----|:-----|
| 표준 아웃라인 | 1px 검은 아웃라인. 외부 경계 전체 |
| 선택적 컬러 아웃라인 | 빛을 받는 면(상단, 앞면)에 한해 밝은 컬러 아웃라인 1px 추가 |
| 내부 디테일 선 | 의상 접힘·무기 질감 등 내부 선은 검정보다 밝은 어두운 색 사용 (#1A1A2E 등) |
| VFX 아웃라인 | VFX 스프라이트는 아웃라인 없음. 단독 알파 채널로 배경과 분리 |
| 보스 아웃라인 | 분노 모드 시 아웃라인 색이 빨강/오렌지로 변경 → 위협 상태 시각화 |

### 7-3. 캐릭터 비율

| 캐릭터 유형 | 비율 | 참고 게임 |
|:-----------|:----:|:---------:|
| 에르다 (주인공) | 5.5~6 헤드 | Blasphemous 페니텐트 원 |
| 일반 적 | 4~5 헤드 | SotN 일반 적 |
| 보스 | 비율 의도적 왜곡 가능 | 크기가 위협 전달 |
| Memory Shard | 2~3 헤드 (작고 귀여운) | 디스가이아 프리니 |

**비율 강제 이유:**
에르다가 5.5~6 헤드 비율을 유지하면 Blasphemous/Hollow Knight류의 성숙하고 무게감 있는 분위기가 유지된다. 치비 비율(2~3 헤드)로 가면 디스가이아 톤이 강해져 PA의 BLAME! + 메이드 인 어비스 분위기와 충돌한다. 단, Memory Shard는 의도적으로 작고 귀여운 비율로 설계 — 적이지만 포획 대상이라는 이중성을 시각적으로 표현.

### 7-4. 실루엣 가독성 원칙

640×360 해상도에서는 작은 스프라이트가 화면을 채운다. 실루엣 가독성이 세부 묘사보다 중요하다.

| 규칙 | 이유 |
|:-----|:-----|
| 에르다 실루엣 = 배경과 고대비 유지 | 배경이 청록이면 에르다는 주황·밝은 색 강조 |
| 각 적의 실루엣이 서로 달라야 함 | 16×16~32×32 크기에서 형태로 구분 |
| 보스 실루엣 = 화면의 1/4 이상 | 크기 자체가 위협 |
| 무기 실루엣이 캐릭터 실루엣에서 돌출 | 무기 장착 여부를 원거리에서도 인식 |

## 8. Phase 1 최소 애니메이션 세트

### 8-1. 절대 필수 (없으면 게임 불가)

Phase 1 목표: "핵심 루프가 재미있는가?" 검증. 최소한 이것이 있어야 타격감 검증이 가능하다.

| 대상 | 애니메이션 | 프레임 수 | 이유 |
|:-----|:---------|:--------:|:-----|
| 에르다 | Idle | 4f | 시작 상태 필수 |
| 에르다 | Run | 6f | 이동 검증 |
| 에르다 | Attack 1~3타 | 4 / 4 / 6f | 3타 콤보 + 히트스탑 검증 |
| 에르다 | Hit Reaction | 3f | 피격 피드백 검증 |
| 에르다 | Death | 6f | 사망 처리 필수 |
| 에르다 | Jump / Fall / Land | 3 / 2 / 2f | 플랫폼 이동 검증 |
| 적 (1종) | Idle + Walk + Attack + Death | 4/6/4/5f | 전투 루프 검증 |

### 8-2. Phase 1 플레이스홀더 허용 목록

| 애니메이션 | 플레이스홀더 방법 |
|:---------|:--------------|
| Dash (대시) | 에르다 Run 스프라이트 + 코드 잔상 효과만 (현재 구현 방식 유지) |
| Wall Slide / Wall Jump | 에르다 Idle 스프라이트 재사용 |
| Echo Fire / Echo Scan | 에르다 Attack 스프라이트 재사용 |
| 적 2~7번 | 1종 스프라이트에 색상 변경만 적용 |
| 보스 | 적 스프라이트 2배 스케일 임시 사용 |
| 모든 VFX | 현재 코드 Graphics 방식 유지 |
| 무기 종류별 | Sword 단일 스프라이트로 전체 무기 임시 대응 |

### 8-3. 애니메이션 우선순위 로드맵

| 단계 | 우선순위 | 대상 | 기대 효과 |
|:----|:-------:|:-----|:---------|
| P1-A | 즉시 | 에르다 기본 7종 (Idle/Run/Jump/Fall/Land/Attack1~3/HitReact/Death) | 핵심 루프 검증 가능 |
| P1-B | 2주 내 | 에르다 Dash + 적 1종 완성 애니메이션 | 전투 쾌감 + 회피 검증 |
| P1-C | Phase 1 완료 전 | 적 3종 고유 애니메이션 (Skeleton/Slime/Ghost) | 적 다양성 검증 |
| P2-A | Phase 2 시작 | 에르다 무기별 변형 + VFX 스프라이트화 | 무기 차별화 |
| P2-B | Phase 2 중반 | 보스 완성 애니메이션 | 보스 전투 검증 |
| P2-C | Phase 2 후반 | 모든 적 7종 고유 애니메이션 | 아이템계 시각적 완성도 |

## 9. 교훈 요약

| 인사이트 | 적용 원칙 |
|:---------|:---------|
| 프레임 수보다 키프레임 포즈의 질이 타격감을 결정한다 | 4프레임으로도 Celeste가 증명. 포즈 극단화 우선 |
| 예비동작은 공정한 전투의 기반이다 | 적 공격에 반드시 2f 이상 텔레그래프. 보스는 6~14f |
| 스쿼시&스트레치는 짧을수록 효과적이다 | 1~2프레임 이상 유지하면 오히려 이상해짐 |
| 잔상과 파티클은 프레임 부족을 보완한다 | 대시·빠른 공격에서 추가 스프라이트 없이 속도감 전달 |
| 실루엣 가독성이 세부 묘사보다 우선이다 | 640×360 해상도에서 형태로 적을 구분 |
| 팔레트 통일이 곧 아트 스타일이다 | 각 스프라이트 16색 제한. 청록+주황 기조 유지 |
| Phase 1에서는 플레이스홀더를 두려워하지 않는다 | 타격감 검증에 필요한 최소한만 제작 |
| 아웃라인은 맥락이 있다 | 빛 방향 기반 선택적 컬러 아웃라인이 입체감 부여 |
