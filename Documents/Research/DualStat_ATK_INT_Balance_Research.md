## 1. 레퍼런스 게임 분석

### 1.1. Dark Souls / Elden Ring — 4축 스케일링 (STR/DEX/INT/FTH)

**구조 요약:**
FromSoft는 STR(완력), DEX(기교), INT(지성), FTH(신앙)의 4축 스케일링을 운용한다. 각 무기는 A/B/C/D/E/S 등급으로 해당 스탯에 대한 스케일링 강도를 명시한다.

**물리 vs 마법 균형 메커니즘:**

| 메커니즘 | 물리(STR/DEX) | 마법(INT/FTH) |
| :--- | :--- | :--- |
| 강화 재료 | 타이타나이트 계열 (범용) | 트윈클링 타이타나이트 (희귀) |
| 적 저항 | 물리 저항 (보편적) | 마법/화염/번개 저항 (적별 편차 큼) |
| 버프 메커니즘 | 전사 탈리스만, 전쟁의 표식 | 마법사 탈리스만, 테라 마기카 |
| 초반 접근성 | 즉시 사용 가능 | 인챈트 선행 투자 필요 |
| 후반 천장 | 안정적이나 낮음 | 리소스 소모 후 높음 |

**핵심 인사이트:**
- Elden Ring의 물리/마법 균형은 **적 별 저항 분포**로 유지된다. 전체 적군의 약 40%는 물리 저항이 높고, 30%는 마법 저항이 높으며, 30%는 중립에 가깝다. 어떤 단일 빌드도 모든 적을 최효율로 처리하지 못한다.
- **파워 커브 교차:** 물리 빌드는 초반(레벨 1~60)에 안정적 우위를 가진다. 마법 빌드는 레벨 60 이후 "결정 마법" 등 고스케일 주문으로 역전한다. 이 교차점이 새 빌드 도전의 동기가 된다.
- **치명적 약점:** STR 무기 + DEX 스케일링 혼합 빌드("질량 무기에 DEX 버프")는 종종 순수 빌드를 초과한다. 과도한 하이브리드 효율이 전문화의 의미를 희석시킨 사례다.

**PA 적용점:** 에코 인챈트가 INT 스케일링의 진입 장벽이다. 초반 화속성은 기본 제공하되, 고원소 빌드는 월드 보스 처치 후 해금되는 구조 — 이 설계가 FromSoft의 초반 물리 우위/후반 마법 역전 패턴과 유사하게 작동한다.

### 1.2. Dead Cells — 3색 스탯 (Brutality/Tactics/Survival)

**구조 요약:**
Dead Cells는 빨강(Brutality=공격), 보라(Tactics=스킬/罠), 초록(Survival=방어)의 3색 스탯 체계로 운영된다. 각 무기와 스킬은 단일 색상 스케일링을 가진다.

**균형 유지 메커니즘:**

| 메커니즘 | 설명 |
| :--- | :--- |
| 색상 유전 | 아이템이 스크롤 흡수 시 해당 색 스탯 상승. 획득 아이템 색상이 스탯 방향을 결정 |
| 복수 색상 무기 | 일부 무기는 2색 스케일링 (예: 빨강+초록). 단, 단일 색 전문화가 최대 효율 |
| 변이 시스템 | 변이 아이템이 동일 색상 스케일 강화. 같은 색 아이템 3개 보유 시 동색 스탯 추가 보너스 |
| 빌드 강제 락인 | 한번 색상 방향을 정하면 스크롤 분배가 그 방향으로 누적. 중간 방향 전환 비효율 |

**핵심 인사이트:**
- Dead Cells의 성공 요인은 **아이템 획득 자체가 스탯 방향을 결정**하는 강제 전문화 메커니즘이다. 플레이어가 "어느 스탯을 올릴지" 선택하는 것이 아니라, "어떤 아이템을 장착할지" 선택함으로써 자연스럽게 전문화된다.
- 하이브리드는 2~3색 분산으로 스탯이 희석되어 단일 전문화 대비 약 20~30% 낮은 데미지 천장을 가진다. 이것이 설계 의도다.
- **범용성의 비용:** 하이브리드는 더 많은 상황에 대응 가능하지만 각 상황에서 최고가 아니다. 이 트레이드오프가 뚜렷하게 체감된다.

**PA 적용점:** PA의 Gladiator(ATK+) vs Tutor(INT+) 이노센트 슬롯 배분이 Dead Cells의 색상 전문화와 같은 역할을 한다. 슬롯 수가 제한(Normal 2 ~ Ancient 8)되어 있으므로, 동일 슬롯에 ATK형과 INT형을 섞으면 양쪽 모두 낮아지는 페널티를 내재한다.

### 1.3. Diablo 2 — 물리 지배 현상의 교훈

**구조 요약:**
Diablo 2는 물리 데미지(STR 스케일)와 원소 데미지(각 직업 스탯)가 공존했다. 하지만 후반(Hell 난이도)에서 물리가 원소를 압도적으로 지배했다.

**물리 지배의 구조적 원인:**

| 원인 | 설명 |
| :--- | :--- |
| 물리 면역 부재 | 대부분의 적에게 물리 데미지는 항상 유효했다 |
| 원소 면역 존재 | "화염 면역", "냉기 면역" 몬스터가 Hell에 다수 등장 |
| 원소 면역 해제 불가 | 면역은 저항 감소 스킬(Lower Resist 등)로만 부분 해제 가능. 이 접근법이 번거로움 |
| 물리 데미지 증폭 | 아이템 모디파이어(ED%, CB, OW)가 물리에만 적용되는 것이 다수 |
| 원소 데미지 캡 | Blessed Hammer 같은 예외를 제외하면 원소 데미지의 스케일 상한이 낮았음 |

**결과:** 소서리스, 네크로만서, 드루이드 등 원소 특화 직업은 Hell 난이도에서 물리 면역 적을 만나면 "벽"에 부딪혔다. 보편적 해법은 물리 보조 빌드로의 전환이었고, 이는 직업 정체성의 붕괴를 의미했다.

**핵심 인사이트:**
- 원소 면역/물리 면역 비대칭이 물리 지배의 직접 원인이다.
- 면역은 "저항 100%"가 아닌 "저항 초과(100+%)"로 설계되어 완전 차단됐다. 저항 침투(Penetration) 메커니즘이 없으면 원소 빌드가 붕괴된다.
- **교훈:** 면역은 절대로 채용하지 말 것. 대신 "높은 저항(75~90%)"과 "낮은 저항(10~30%)"의 분포로 실질적 다양성을 만들 것.

**PA 적용점:** PA의 최소 데미지 1 보장 원칙은 이 교훈을 이미 반영하고 있다. 확장하여, 어떤 적도 DEF 또는 RES가 100%에 도달하지 않도록 상한을 설정해야 한다. 권장 최대치: DEF 85%, RES 80%.

### 1.4. Path of Exile — 물리↔원소 전환 메커니즘

**구조 요약:**
PoE는 물리와 원소 사이에 **데미지 전환(Conversion)** 메커니즘을 도입했다. 물리 데미지의 일정 비율을 특정 원소 데미지로 전환할 수 있다.

**핵심 메커니즘:**

| 메커니즘 | 효과 |
| :--- | :--- |
| Physical to Lightning (50%) | 물리 데미지의 50%가 번개 데미지로 전환. 높은 물리 ATK가 원소 데미지로도 기여 |
| Elemental Equilibrium | 한 원소 공격 후 해당 원소 저항 +25%, 나머지 원소 저항 -50%. 원소 교체의 전략적 가치 |
| Penetration | 원소 저항 침투. 원소 빌드가 고저항 적을 상대할 수 있는 핵심 메커니즘 |
| Conversion 상한 100% | 전환 합산이 100%를 초과할 수 없어 무한 배가 방지 |

**핵심 인사이트:**
- 저항 침투(Penetration)가 없으면 원소 빌드는 후반에 취약해진다. PA에도 INT 빌드에 적합한 "RES 침투" 메커니즘이 필요하다.
- PoE의 저항 무효화(Exposure)와 PA의 상태이상(Freeze, Shock, Burn)은 유사한 역할 — 적의 유효 저항을 낮춰 원소 데미지 효율을 높인다.
- **균형의 핵심:** 원소 빌드의 최고 천장(high ceiling)을 물리보다 높게 설정하되, 그 천장에 도달하는 경로에 조건(상태이상 적용, 원소 교체 타이밍)을 부여하면 복잡성 없이 전략성이 생긴다.

**PA 적용점:** 상태이상이 INT 빌드의 "숨겨진 DPS"로 작동한다. 화상의 지속 피해, 빙결의 데미지 증폭(freeze_damage_bonus), 감전의 행동 속도 감소는 모두 INT 기반 원소 빌드에 간접 데미지/CC를 제공한다. 이것이 PA의 자연스러운 Penetration 대안이다.

### 1.5. Hollow Knight — 손톱 vs 주문 빌드 (Shaman Stone vs Quick Slash)

**구조 요약:**
Hollow Knight는 단순한 공격(Nail) 기반 빌드와 주문(Spell) 기반 빌드를 참/거짓 장신구 조합으로 구현했다.

**균형 분석:**

| 빌드 | 핵심 장신구 | 장점 | 단점 |
| :--- | :--- | :--- | :--- |
| Nail 빌드 | Quick Slash, Fragile Strength, Mark of Pride | 안정적 근접 DPS, 보스 직접 공격 | 근접 위험성, 리버 몸드 같은 근접 불리 보스 |
| Spell 빌드 | Shaman Stone, Spell Twister, Soul Eater | 안전거리 공격, 단일 히트 고데미지 | 소울 자원 관리, 공격 모션 경직 |

**핵심 인사이트:**
- Team Cherry의 균형 철학: "Nail은 쉽게 시작, Spell은 숙련자 보상." Nail은 리소스 관리 없이 기본으로 작동하고, Spell은 소울 관리라는 추가 게임플레이 레이어를 요구한다.
- 주문 빌드는 "단일 타격의 폭발력"이 높고, 손톱 빌드는 "지속 화력의 안정성"이 높다. 이 구분이 각 빌드에 고유한 사용 이유를 부여한다.
- **Weaversong 문제:** 특정 장신구 조합이 두 시스템을 모두 강화해 "하이브리드가 항상 유리"한 상황이 발생했다. 패치로 수정됐다.

**PA 적용점:** ATK는 안정적 기본 DPS(콤보 지속), INT는 인챈트 활성화 후 폭발적 원소 버스트라는 구분이 Hollow Knight 모델과 일치한다. INT 빌드에 "에코 인챈트 유지"라는 리소스 관리 레이어(쿨다운, 원소 해금)가 Spell의 소울 관리와 유사한 숙련도 요구 사항으로 기능한다.

### 1.6. Hades — 공격/특수/주문 삼각 구도

**구조 요약:**
Hades는 직접 공격(Attack), 특수기(Special), 주문(Cast)을 별도 축으로 강화하며 제우스/포세이돈/아테나 등 올림포스 신의 버프가 각 축에 서로 다르게 적용된다.

**균형 메커니즘:**

| 메커니즘 | 효과 |
| :--- | :--- |
| Boon 특이화 | 같은 신의 버프 3개 이상 시 추가 보너스 → 단일 신 특화가 보상 |
| 듀오 버프 | 두 신의 조합으로만 해금되는 강력한 버프. 하이브리드에도 유일한 인센티브 제공 |
| 무기별 기본 특성 | 활(=원거리+Cast 시너지), 창(=근접+Attack 시너지) 등 무기가 빌드 방향을 자연 유도 |
| 대쉬 방어 기반 생존 | 생존이 스탯이 아닌 기술에 의존 → 스탯 집중 투자가 가능 |

**핵심 인사이트:**
- Hades의 성공 비결은 **각 빌드가 독자적인 플레이 패턴을 요구**한다는 점이다. Cast 빌드는 거리 관리와 타이밍, Attack 빌드는 근접 압박, Special 빌드는 히트앤런이 다르다. 수치 균형이 아닌 "경험의 차별화"로 공존을 실현했다.
- 하이브리드(듀오 버프)는 두 경로를 각각 어느 정도 발전시켜야 해금된다는 진입 장벽이 있다. 하이브리드에 접근하기 위한 조건이 전문화와 결합된다.

**PA 적용점:** PA에서 ATK 빌드(콤보 리듬, 근접 압박)와 INT 빌드(인챈트 유지, 원소 타이밍)의 플레이 패턴 차이가 수치 균형을 보완하는 핵심 레이어다. 두 빌드의 수치가 같아도 플레이 스타일이 다르면 공존할 수 있다.

## 2. 핵심 실패 패턴 — 물리 지배 현상

레퍼런스 게임 분석에서 추출한 물리 지배 현상의 구조적 원인과 대응:

| 실패 패턴 | 원인 | PA 예방 설계 |
| :--- | :--- | :--- |
| 원소 면역 몬스터 | 원소 빌드가 특정 적에게 0 데미지 | 면역 금지. 최소 데미지 1 보장. RES 상한 80% |
| 물리 버프 아이템 편중 | 물리 강화 이노센트/아이템이 원소보다 많음 | Gladiator : Tutor = 1:1 비율 유지. 원소 이노센트 3종 동등 지원 |
| 원소 투자 비용 > 물리 투자 비용 | 원소 빌드의 준비 단계가 너무 길고 복잡 | 에코 화속성 기본 내장으로 원소 빌드 진입 비용 최소화 |
| 하이브리드 최적해 수렴 | 두 스탯 동시 투자가 항상 유리 | 슬롯 제약 + 소프트 캡으로 하이브리드 비효율화 |
| 후반 원소 스케일 부족 | INT 투자 대비 데미지 상승이 ATK에 비해 낮음 | INT 스케일 팩터를 ATK보다 약 15~20% 높게 설정 (상태이상 DPS 포함 기대값 동등화) |

## 3. 손익분기점 분석 — DEF vs RES 비율

### 3.1. 이론 모델

PA의 현재 공식:
```
Physical_Damage = max(1, floor((ATK * Skill_Mul - DEF * DEF_Factor) * Crit * Variance))
Elemental_Damage = max(1, floor((INT * Element_Mul - RES * RES_Factor) * Crit * Variance))
```

**손익분기점 조건:** ATK 빌드와 INT 빌드의 기대 데미지가 동등해지는 지점.

DEF_Factor = RES_Factor = 0.5로 가정하면:

```
ATK * Skill_Mul - DEF * 0.5 = INT * Elem_Mul - RES * 0.5
```

**손익분기점 (BEP):**
```
BEP: ATK * Skill_Mul - INT * Elem_Mul = (DEF - RES) * 0.5
```

### 3.2. 구체 수치 시뮬레이션

아래는 Lv 1 기준 대표 수치다. ATK = INT = 50, Skill_Mul = 1.0 (기본 공격), Elem_Mul = 1.0 (무상성)으로 가정.

| DEF | RES | Physical_Dmg | Elemental_Dmg | 유리한 빌드 |
| --- | --- | --- | --- | --- |
| 20 | 20 | 50*1.0 - 20*0.5 = **40** | 50*1.0 - 20*0.5 = **40** | 동등 |
| 20 | 10 | 50*1.0 - 20*0.5 = **40** | 50*1.0 - 10*0.5 = **45** | INT +12.5% |
| 20 | 30 | 50*1.0 - 20*0.5 = **40** | 50*1.0 - 30*0.5 = **35** | ATK +14.3% |
| 30 | 20 | 50*1.0 - 30*0.5 = **35** | 50*1.0 - 20*0.5 = **40** | INT +14.3% |
| 40 | 20 | 50*1.0 - 40*0.5 = **30** | 50*1.0 - 20*0.5 = **40** | INT +33.3% |

**결론:**
- DEF > RES인 적에게는 INT 빌드가 유리
- RES > DEF인 적에게는 ATK 빌드가 유리
- DEF = RES인 중립 적에게는 동등

### 3.3. 상태이상 DPS 포함 기대값

INT 빌드에는 상태이상 지속 피해가 추가된다. 화상(Burn)을 예시로:

```
화상 기대 데미지/초 = MaxHP * burn_damage_per_sec * burn_duration_s
```

`burn_damage_per_sec = 0.02`, `burn_duration_s = 3초`이면:
- 적 MaxHP 200 기준: 200 * 0.02 * 3 = **12 추가 피해**

이 값은 INT 빌드의 "숨겨진 DPS"로, 총 기대 데미지 계산에 포함해야 한다. 순수 원소 히트 데미지만으로 비교하면 INT 빌드가 약해 보이는 착시가 발생한다.

**보정된 INT 기대 데미지:**
```
Adjusted_INT_Damage = Elemental_Damage + StatusEffect_DPS * EffectiveDuration
```

## 4. PA 권장 데미지 공식 파라미터

### 4.1. 현재 공식 확인

현재 System_Combat_Damage.md의 공식:
```
Physical: max(1, floor((ATK * Skill_Mul - DEF * DEF_Factor) * Crit * LvlCorr * Rand(0.9,1.1)))
Elemental: max(1, floor((INT * Elem_Mul - RES * RES_Factor) * Crit * LvlCorr * Rand(0.9,1.1)))
```

### 4.2. 권장 파라미터 값

| 파라미터 | 현재 값 | 권장 값 | 근거 |
| :--- | :--- | :--- | :--- |
| DEF_Factor | 0.5 | **0.5** | 유지. ATK의 절반이 DEF에 상쇄. 직관적 |
| RES_Factor | (미정) | **0.4** | RES 상쇄를 DEF보다 약간 낮게. 원소 빌드가 방어 관통에서 소폭 유리 |
| 최대 DEF 효과 상한 | (미정) | **ATK * 0.85** | DEF가 아무리 높아도 ATK의 85% 이상을 상쇄할 수 없음. 최소 15% 피해 보장 |
| 최대 RES 효과 상한 | (미정) | **INT * 0.80** | RES 상한을 DEF보다 낮게. 원소 빌드가 고저항 적에게 상대적으로 유리 |
| 약점 원소 배율 | 2.0x | **2.0x** | 유지. 확실한 약점 체감. 단, 보스는 1.5x로 하향 가능 |
| 저항 원소 배율 | 0.5x | **0.5x** | 유지. 단, 저항 최하한은 0.5x. 면역 금지 |
| 상태이상 지속피해 계수 | burn_damage_per_sec 미정 | **0.02 (MaxHP%/초)** | 화상 3초 = 총 HP의 6%. 위험하지 않되 INT 기여 가시화 |
| 상태이상 지속시간 INT 스케일 | 0.005 per INT | **0.005 per INT** | 유지. INT 100 시 지속시간 1.5배 |
| 크리티컬 배율 | 1.5x (고정) | **1.5x** | 유지 |
| 랜덤 분산 | 0.9~1.1 | **0.9~1.1** | 유지 |

### 4.3. RES_Factor 비대칭 설계 근거

RES_Factor를 DEF_Factor(0.5)보다 낮게(0.4) 설정하는 이유:

1. **INT 빌드의 "기술 세금"을 수치로 보전:** 에코 인챈트 유지, 원소 타이밍 등 INT 빌드의 조작 복잡성은 ATK 빌드보다 높다. 이 추가 기술 비용에 대한 소폭의 수치 보상이 공정하다.
2. **상태이상 DPS와의 통합:** 순수 히트 데미지만 비교하면 INT가 불리해 보이지만, 상태이상 지속 피해를 포함하면 동등하다. 순수 히트 비교를 약간 유리하게 잡되 상태이상이 없으면 동등 이하가 되도록 설계한다.
3. **PoE Penetration 대안:** 별도 저항 침투 스탯 없이 RES_Factor를 DEF_Factor보다 낮게 설정하는 것으로 같은 효과를 구현한다.

## 5. 빌드 아키타입 파워 프로파일

### 5.1. 3대 빌드 아키타입 정의

| 아키타입 | 스탯 방향 | 이노센트 구성 | 무기 | 게이트 경로 |
| :--- | :--- | :--- | :--- | :--- |
| 순수 ATK | ATK 극대화, INT 최소 | Gladiator 중심, Berserker 조합 | 검/대검/단검/활 | ATK 게이트 우선 해금 |
| 순수 INT | INT 극대화, ATK 최소 | Tutor 중심, Burner/Freezer/Shocker 조합 | 지팡이 전용 | INT 게이트 우선 해금 |
| 하이브리드 | ATK/INT 균등 투자 | Gladiator+Tutor 혼합 | 검+Echo Enchant | 두 게이트 모두 느리게 |

### 5.2. 단계별 파워 프로파일 (Lv 1~Max)

```
DPS 지수 (기준값 100 = Lv 1 ATK 빌드 기본 공격)

게임 단계:    초반(Lv1~3)    중반(Lv4~7)    후반(Lv8~10)    야리코미 심층
──────────────────────────────────────────────────────────────────────────
순수 ATK:     100            130            155             190
순수 INT:     85             130            165             210
하이브리드:   90             115            140             165
──────────────────────────────────────────────────────────────────────────
```

**설계 의도:**
- 초반: ATK가 약 15% 앞선다. 진입 장벽이 낮고 이해하기 쉬운 ATK가 초반 우세를 가져 새 플레이어를 자연스럽게 게임에 적응시킨다.
- 중반: 두 빌드가 교차한다. 원소 해금(월드 보스 처치)이 INT 빌드를 본격 강화한다. 이 교차점이 "INT 빌드를 시도해볼까?"라는 동기를 만든다.
- 후반/심층: INT 빌드가 ATK를 앞선다. 상태이상 DPS, 원소 상성 관리, 고레벨 Tutor 이노센트의 복합 효과가 높은 천장을 만든다.
- 하이브리드: 항상 순수 빌드 이하. 단, 두 개의 게이트 유형을 모두 해금하는 실용적 이점이 있다.

### 5.3. 하이브리드가 최선이 되지 않는 이유

| 제약 요인 | 상세 |
| :--- | :--- |
| 이노센트 슬롯 분할 | Gladiator 3 + Tutor 3 = 각 +15 vs Gladiator 6 = +30. 단일 특화가 2배 효율 |
| 소프트 캡 위치 | ATK와 INT 각각 소프트 캡(체감 효율 감소 시작점)이 존재. 하이브리드는 두 소프트 캡 모두 낮은 수준에서 도달 |
| 지팡이 전용 INT 스케일 | INT 빌드는 지팡이 + 원소 증폭 ×1.5의 시너지를 받지만, 하이브리드가 검을 쓰면 이 시너지를 포기 |
| 게이트 해금 속도 | ATK 게이트: ATK 집중 빌드 = 2회 IW 클리어 / 하이브리드 = 3~4회. 탐험 속도 지연 |

## 8. ATK vs INT 성장 곡선 설계

### 8.1. 장비 기여 (Equipment Scaling)

장비 ATK와 장비 INT의 레어리티별 기본값이 동등하게 설계되어야 한다. 아래는 검(ATK)과 지팡이(INT)의 Normal 등급 ATK/INT 기준값 권장:

```
검 Normal:    ATK_Base = 15~20
지팡이 Normal: INT_Base = 15~20
```

레어리티별 스탯 배율(`System_Equipment_Rarity.md`):
- Normal: x1.0 → 검 ATK ≈ 17, 지팡이 INT ≈ 17
- Magic: x1.3 → 검 ATK ≈ 22, 지팡이 INT ≈ 22
- (이하 동일 배율 적용)

> **설계 원칙:** 레어리티별 장비 ATK와 INT 기본값이 같은 배율을 공유해야 어느 빌드를 선택해도 동등한 성장 경험이 보장된다.

### 8.2. 이노센트 기여 (Innocent Scaling)

Gladiator(ATK+)와 Tutor(INT+)의 단위 레벨당 기여 수치를 동등하게 설계한다.

```
Gladiator Lv 1 = +ATK 1
Tutor Lv 1     = +INT 1

(단위 레벨당 기여량 1:1 고정)
```

> **주의:** 만약 Gladiator의 단위 기여가 Tutor보다 높으면 ATK 빌드가 이노센트 파밍에서 유리해진다. 1:1 비율을 파라미터로 명시하고 튜닝 시 함께 조정한다.

### 8.3. ATK vs INT 소프트 캡

| 스탯 | 소프트 캡 시작 | 소프트 캡 이후 효율 | 하드 캡 |
| :--- | :--- | :--- | :--- |
| ATK | 200 | 신규 ATK 2 당 실효 +1 (50% 효율) | 500 |
| INT | 180 | 신규 INT 2 당 실효 +1 (50% 효율) | 500 |

> INT 소프트 캡을 ATK보다 10% 낮게(200→180) 설정하는 이유: 상태이상 DPS가 INT 고투자 구간에서도 지속적으로 기여하므로, 순수 히트 데미지 소프트 캡을 더 빨리 도달시켜 상태이상 기여 비중을 자연스럽게 높인다.

```
│                   ──────────
│         ──────────      ATK ──────────
│─────────
│
└──────────────────────────────────────────→ 게임 진행
 Lv1  Lv2  Lv3 | Lv4  Lv5 | Lv6  Lv7 | Lv8  Lv9  Lv10
  초반           중반교차    중반        후반
```

| 구간 | ATK vs INT | 핵심 이벤트 |
| :--- | :--- | :--- |
| 초반 (Lv1~3) | ATK +15% 우위 | 에코 화속성 기본, 지팡이는 있지만 INT 이노센트 미확보 |
| 교차점 (Lv4~5) | 동등 | 첫 월드 보스 처치 → 빙/뇌 원소 해금. Tutor 이노센트 확보 시작 |
| 중반 (Lv6~7) | INT +5~10% 우위 | Tutor 이노센트 복종 누적, 원소 상성 지식 활용 시작 |
| 후반 (Lv8+) | INT +15~20% 우위 (이론 천장) | 고레벨 Tutor + 원소 이노센트 조합. 상태이상 DPS 극대화 |

> **주의:** 이 교차점은 "최적 플레이" 기준이다. 실제 플레이어는 이노센트 파밍 수준, 원소 상성 활용도에 따라 편차가 크다. 교차점이 고정된 레벨이 아닌 "행동"(월드 보스 처치, Tutor 획득)에 연동되도록 설계된 것이 의도적이다.

### 9.2. 교차점을 행동 연동으로 설계하는 이유

FromSoft 분석에서 교훈: 파워 커브 교차가 레벨에만 연동되면 "레벨만 올리면 자동으로 강해짐"이 된다. PA는 교차점을 월드 보스 처치(원소 해금)와 이노센트 파밍(Tutor 확보)에 연동시켜 플레이어의 능동적 선택이 파워 커브를 결정하게 한다. 이것이 야리코미 철학과 직접 정렬된다.

## 10. 전문화 강제 메커니즘

하이브리드가 항상 최선이 되지 않도록 하는 설계 레이어:

### 10.1. 이노센트 슬롯 제약 (기존 설계 활용)

레어리티별 슬롯 수(2~8)는 이미 전문화 압력을 내재한다. 이 압력을 명시적으로 수치화:

```
전문화 지수 = (주력 이노센트 수) / (전체 이노센트 슬롯 수)

순수 ATK: Gladiator 6 / 슬롯 6 = 1.0 (완전 전문화) → ATK 소프트캡까지 풀 이용
하이브리드: Gladiator 3 + Tutor 3 / 슬롯 6 = 0.5 → 각 스탯 소프트캡의 절반 지점
```

### 10.2. 지팡이 인챈트 증폭 (기존 설계 활용)

지팡이의 "원소 증폭 ×1.5" 시그니처 메커닉은 INT 빌드에서만 완전한 가치를 발휘한다. 검 + Echo Enchant 하이브리드는 이 증폭을 받지 못한다.

```
INT 빌드 원소 데미지: INT * Elem_Mul * 1.5 (지팡이 시그니처)
하이브리드 원소 데미지: INT * Elem_Mul * 1.0 (검 사용, 시그니처 없음)
```

33% 차이가 하이브리드의 원소 DPS를 억제한다.

### 10.3. 스탯 게이트 해금 속도 차이

§7.2 임계값 곡선 기준:
- 층위 3 ATK 게이트 (임계값 100): ATK 집중 빌드는 IW 2회 클리어로 도달. 하이브리드는 3~4회.
- 층위 3 INT 게이트 (임계값 70): INT 집중 빌드는 IW 2회 클리어로 도달. 하이브리드는 3~4회.

탐험 속도 지연이 하이브리드의 실질적 비용이다.

### 10.4. 원소 이노센트 전문화 시너지

Burner/Freezer/Shocker(원소 이노센트)는 INT 빌드에서만 완전한 가치를 발휘한다. ATK 빌드가 원소 이노센트를 장착하면 슬롯 낭비가 된다(인챈트를 사용하지 않으므로).

이 자연 시너지가 "ATK 빌드는 ATK 이노센트만, INT 빌드는 INT + 원소 이노센트"라는 명확한 전문화 경로를 형성한다.

## 11. 튜닝 파라미터 레지스터

밸런스 조정 시 건드려야 할 파라미터 목록과 안전 범위:

| 파라미터 | 현재/권장값 | 안전 범위 | 영향 범위 | 조정 방향 |
| :--- | :--- | :--- | :--- | :--- |
| `DEF_Factor` | 0.5 | 0.3~0.6 | ATK 빌드 전체 DPS | ATK 빌드가 약하면 낮춤 |
| `RES_Factor` | 0.4 | 0.25~0.55 | INT 빌드 전체 DPS | INT 빌드가 약하면 낮춤 |
| `DEF_Absorption_Cap` | ATK * 0.85 | 0.75~0.90 | 고DEF 적 vs ATK 빌드 | 벽 느낌 강하면 낮춤 |
| `RES_Absorption_Cap` | INT * 0.80 | 0.70~0.88 | 고RES 적 vs INT 빌드 | INT 빌드 막힘 강하면 낮춤 |
| `Elem_Weakness_Mul` | 2.0x | 1.5x~2.5x | 약점 타격 임팩트 | 약점이 너무 강하면 1.5x로 |
| `Elem_Resist_Mul` | 0.5x | 0.3x~0.7x | 불리 원소 패널티 | 패널티 과하면 0.6x로 |
| `burn_damage_per_sec` | 0.02 (MaxHP%) | 0.01~0.03 | 화상 지속 DPS | INT 빌드 약하면 올림 |
| `burn_duration_s` | 3.0 | 2.0~5.0 | 화상 총 피해량 | INT 빌드 약하면 늘림 |
| `Staff_Enchant_Amplifier` | 1.5x | 1.2x~2.0x | 지팡이 INT 빌드 천장 | INT 후반 너무 강하면 낮춤 |
| `ATK_SoftCap` | 200 | 150~250 | ATK 빌드 야리코미 진입점 | 파밍 루프가 짧으면 낮춤 |
| `INT_SoftCap` | 180 | 130~230 | INT 빌드 야리코미 진입점 | 파밍 루프가 짧으면 낮춤 |
| `Gladiator_per_Level` | 1 ATK | 0.8~1.5 | Gladiator 이노센트 기여 | ATK 이노센트 효율 조정 |
| `Tutor_per_Level` | 1 INT | 0.8~1.5 | Tutor 이노센트 기여 | INT 이노센트 효율 조정 |

### 11.1. 단일 파라미터 조정 시 체크리스트

어떤 파라미터를 조정하든 아래를 검증:
1. Lv 5 ATK 빌드 vs INT 빌드 DPS 기대값 격차가 ±15% 이내인가?
2. 야리코미 심층(Ancient 장비 + 고레벨 이노센트)에서 INT 빌드 DPS 천장이 ATK보다 높은가?
3. 어떤 적도 DEF 또는 RES 상한이 ATK/INT의 100%를 초과하지 않는가?
4. 하이브리드 빌드 DPS 기대값이 순수 빌드 기대값의 85% 미만인가?

## 12. 시뮬레이션 검증 스펙

Phase 1 프로토타입 밸런스 검증을 위한 시뮬레이션 파라미터:

### 12.1. 기준 시나리오 (Baseline Scenario)

| 항목 | 값 |
| :--- | :--- |
| 테스트 레벨 | Lv 1, Lv 5, Lv 10 (3개 단계) |
| ATK 빌드 테스트 캐릭터 | ATK = 레벨 × 18 + 이노센트 Gladiator 6 × 레벨, INT = 10 (최소) |
| INT 빌드 테스트 캐릭터 | INT = 레벨 × 18 + 이노센트 Tutor 6 × 레벨, ATK = 10 (최소) |
| 하이브리드 테스트 캐릭터 | ATK = 레벨 × 10, INT = 레벨 × 10, 이노센트 Gladiator 3 + Tutor 3 |
| 테스트 무기 | ATK 빌드: 검(Normal~Ancient), INT 빌드: 지팡이(Normal~Ancient) |
| 테스트 적 | 중립형 적 (DEF = ATK×0.4, RES = INT×0.4), 물리형 적, 원소형 적 각각 |
| 반복 횟수 | 1000회 (랜덤 분산 통계 수렴) |

### 12.2. 합격 기준 (Pass Criteria)

| 검증 항목 | 합격 조건 |
| :--- | :--- |
| 초반 DPS 격차 | ATK 빌드가 INT 빌드 대비 +10~20% 이내 |
| 중반 DPS 격차 | ±10% 이내 (교차 구간) |
| 후반 DPS 격차 | INT 빌드가 ATK 빌드 대비 +10~25% 이내 |
| 하이브리드 DPS | 순수 빌드 기대값 대비 80~90% 이내 (너무 약하면 개선, 초과하면 제약 강화) |
| 면역 체크 | 어떤 빌드/적 조합에서도 연속 5회 이상 1 데미지 발생하지 않음 |
| 상태이상 기여 | INT 빌드 총 데미지에서 상태이상 DPS가 10~25% 비중 |
