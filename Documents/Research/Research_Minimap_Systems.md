## 0. 리서치 동기

ECHORIS는 현재 전체 맵을 항상 노출하는 방식으로 작동한다. 이는 탐험 기둥의 핵심 감정인 "저기 뭔가 있다... 갈 수 있을까?"를 근본적으로 훼손한다. 미지(未知)가 없으면 탐험은 이동으로 전락한다.

본 리서치는 대표적인 메트로베니아 타이틀 6개의 맵 시스템을 분석하여 공통 패턴을 추출하고, ECHORIS의 수직 메가스트럭처에 최적화된 맵 시스템 설계안을 도출하는 것을 목적으로 한다.

## 1. 게임별 맵 시스템 분석

### 1-1. 캐슬바니아: 월하의 야상곡 (Symphony of the Night, 1997)

#### 기본 구조

- **맵 표시 방식:** 일시정지 메뉴의 전용 맵 화면. HUD에 실시간 미니맵 없음.
- **방 단위 공개:** 방(Room)에 진입하는 순간 해당 방의 형태가 맵에 표시된다.
- **미탐험 영역:** 표시 자체가 없음. 맵은 탐험한 영역만 보인다.

#### 색상 코딩

| 색상  | 의미 |
| :---- | :--- |
| 파란색 (Blue) | 탐험 완료된 일반 방 |
| 빨간색 (Red) | 세이브 방 (Save Room) |
| 노란색 (Yellow) | 워프 방 (Warp Room) |
| 회색 (Gray) | 구입한 Castle Map 아이템이 공개한 방 (아직 미탐험) |

#### 완료율 시스템

- 전체 맵은 정상 성과 역전 성(Inverted Castle)으로 구성된다.
- 정상 성: 약 942개의 맵 블록. 99% 달성 가능.
- 역전 성: 948개 추가 블록.
- 최대 완료율: 200.6% (양쪽 성 모두 완탐 기준).
- Castle Map 아이템(도서관 사서 구매): 기본 방 구조를 회색으로 미리 표시. 비밀 방은 미포함.

#### 설계 인사이트

- **맵 = 탐험 성취의 시각적 기록.** 파란 영역이 넓어질수록 플레이어는 자신의 진전을 눈으로 확인한다.
- 색상 코딩이 단순(3색)하여 즉각적 판독이 가능하다.
- 미니맵이 없어도 불편함이 적은 이유: 성의 구조 자체가 직관적이고, 방 간 이동이 짧다.
- 역전 성으로 맵 개념을 재활용하여 콘텐츠를 2배로 확장한 사례.

### 1-2. 할로우 나이트 (Hollow Knight, 2017)

#### 기본 구조

- **맵 표시 방식:** 일시정지 화면의 전체 맵 + 빠른 맵(홀드 단축키로 반투명 오버레이).
- **이중 공개 시스템:** 탐험하면 맵이 자동 갱신되지 않는다. 반드시 벤치(Bench)에서 휴식해야 탐험한 영역이 맵에 반영된다.
- **영역 맵 구매:** 지도 제작자 Cornifer를 각 구역에서 만나 영역 지도를 구매해야 한다. 구매 전에는 탐험해도 맵이 표시되지 않는다. Cornifer를 놓친 경우 마을 상점에서 더 비싼 가격으로 구매 가능.

#### 핀 마커 시스템

- 지도 판매 NPC Iselda가 마커(핀)를 판매한다.
- 마커 종류: 세이브 포인트, 샵, 보스, 수집품 등.
- 마커는 이미 탐험한 구역에만 배치 가능하다.
- Collector 아이템 획득 시: 46개의 Grub 위치가 녹색 핀으로 자동 표시.

#### Fog of War 구현 방식

- 영역 지도 미구매 상태: 완전한 안개. 아무것도 표시되지 않는다.
- 영역 지도 구매 후 + 탐험: 탐험한 방의 외곽선이 표시된다.
- 벤치 휴식: 축적된 탐험 데이터가 맵에 반영된다.
- 탐험 완료 여부와 별개로, 방에 아이템이 남아있으면 맵에 반짝이는 표시가 나타난다.

#### 설계 인사이트

- **탐험의 불확실성을 적극적으로 활용.** 맵을 "발견하는 과정" 자체가 게임플레이의 일부다.
- 벤치 갱신 딜레이는 "기록"의 의식화다. 벤치 도착 = 안도 + 진전 확인의 의례.
- 영역 맵 구매를 강제함으로써 탐험 동기와 경제 시스템을 결합했다.
- 단점: 초반 플레이어에게 혼란스럽다는 비판이 있다. 맵 없이 탐험하는 구간의 학습 비용이 높다.

### 1-3. 슈퍼 메트로이드 (Super Metroid, 1994) / 메트로이드 드레드 (Metroid Dread, 2021)

#### 기본 구조

- **HUD 미니맵:** 화면 우상단 상시 표시 (슈퍼 메트로이드 이후 2D 시리즈 표준).
- **전체 맵:** 일시정지 화면에서 접근. 구역별 맵 표시.
- **셀 기반 공개:** 방에 진입하면 해당 방의 형태(셀 집합)가 맵에 표시된다.
- **Map Station:** 특정 방에 있는 맵 스테이션을 활성화하면 해당 구역의 미탐험 방 외곽선이 모두 표시된다. (단, 내부 정보는 미공개)

#### 드레드의 색상 코딩 시스템

| 색상 | 의미 |
| :--- | :--- |
| 빨간색 (Red) | 미탐험 방 |
| 주황색 (Orange) | 탐험 완료 방 |
| 파란색 (Blue) | 초저온 구역 방 |
| 구역별 고유색 | 구역을 구분하는 색상 테마 |

#### 마커 시스템 (드레드)

- 플레이어가 직접 최대 6개의 색상 마커를 맵 어디에나 배치 가능.
- 마커는 HUD 미니맵에도 표시된다.
- 아이콘 하이라이트: 같은 종류의 아이콘을 전체 맵에서 동시에 강조 표시.
- 커서를 숨겨진 아이템이 있는 구역 위에 올리면 맵이 빛난다(힌트).

#### 드레드의 지형 세부 표시

- 이전 시리즈는 단순 셀 격자였으나, 드레드는 지형의 윤곽선을 세밀하게 표시한다.
- 방의 형태(높낮이, 통로 위치)가 맵에 반영되어 "공간감"이 크게 향상됐다.

#### 설계 인사이트

- **HUD 미니맵은 탐험 중 길 잃음을 방지하는 안전망이다.** 일시정지 없이 방향 감각을 유지할 수 있다.
- Map Station은 "준비 구간"을 만든다. 스테이션 발견 후 맵을 파악하고 루트를 계획하는 과정 자체가 플레이어 경험을 풍부하게 한다.
- 능력 획득 후 아이콘 하이라이트로 "이제 갈 수 있는 곳"을 시각적으로 알려주는 방식은 백트래킹 동기를 명확히 제공한다.

### 1-4. 데드 셀 (Dead Cells, 2018)

#### 기본 구조

- **맵 표시 방식:** HUD 미니맵 상시 표시 (화면 코너).
- **절차적 생성 맵:** 레벨 전체가 매번 새로 생성되므로, 맵은 "탐험 기록"이 아닌 "현재 레벨 내비게이션 도구"다.
- **완전 공개 방식:** 데드 셀의 미니맵은 탐험 여부와 관계없이 현재 레벨의 전체 구조를 보여준다(관심 지점 표시 포함). 이는 의도적인 설계 선택이다.

#### 관심 지점 시스템

- 탐험 중 발견되는 아이템/상점/이벤트가 미니맵에 아이콘으로 추가된다.
- 중요 지점은 미니맵에 강조 표시되어 목표 지향 탐험을 유도한다.

#### 설계 인사이트

- **로그라이크에서 Fog of War는 탐험 동기가 아닌 복잡도 증가다.** 데드 셀은 명시적으로 이를 피했다.
- 절차적 생성 레벨에서는 "이번 판에 뭐가 나왔나"가 핵심이지, "어디를 아직 못 갔나"가 아니다.
- ECHORIS의 아이템계(절차적 생성)에 적용 시 참고할 접근법이다. 아이템계의 미니맵은 Fog of War 없이 현재 지층 구조를 전체 노출하는 것이 합리적이다.

### 1-5. 오리와 눈먼 숲 / 오리와 도깨비불 (Ori and the Blind Forest / Will of the Wisps, 2015/2020)

#### 기본 구조

- **맵 표시 방식:** 일시정지 메뉴의 전체 맵. 자유 스크롤 및 줌 지원.
- **실시간 갱신:** 오리가 탐험하면 실시간으로 맵이 갱신된다 (벤치 갱신 딜레이 없음).
- **지도 제작자 Lupo:** 각 구역에 위치한 Lupo에게 영역 지도를 구매하면 해당 구역의 전체 외곽선과 주요 수집품 위치가 표시된다.

#### Silent Map 퀘스트 (Will of the Wisps)

- Silent Woods 구역은 Lupo도 지도를 가지고 있지 않다. 플레이어가 직접 구역 전체를 탐험하여 지도를 완성하는 퀘스트가 존재한다.
- 완성 보상: 이후 모든 Lupo의 지도 가격 50% 할인.

#### Spirit Wells (세이브 포인트)

- Spirit Well은 세이브 + HP/에너지 회복 + 텔레포트의 기능을 통합한다.
- 지도에 Spirit Well 위치가 마커로 표시된다.
- Spirit Well 자체가 "안전 구역"과 "이동 거점"의 역할을 동시에 한다.

#### 설계 인사이트

- **유기적 맵 외관이 탐험감을 강화한다.** 격자 기반이 아닌 자유형 외곽선이 세계의 생동감을 높인다.
- Lupo의 영역 지도는 할로우 나이트 Cornifer보다 덜 강제적이다. 지도 없이도 탐험은 가능하나, 구매하면 효율이 높아지는 구조.
- 실시간 갱신은 "탐험의 즉각적 보상"을 제공한다. 벤치 갱신 딜레이보다 만족감이 직접적이다.

---

### 1-6. 블래스퍼머스 (Blasphemous, 2019)

#### 기본 구조

- **맵 표시 방식:** 일시정지 메뉴의 전체 맵.
- **자동 공개:** 새로운 방에 진입하면 해당 방이 맵에 자동으로 추가된다. 즉각적이며 별도 조건 없음.
- **미탐험 영역:** 표시되지 않는다. 탐험한 영역만 보인다.

#### 마커 시스템

- Prie Dieu 제단 (세이브 포인트): 자동 마커.
- 패스트 트래블 포인트: 자동 마커.
- 퀘스트/목표 지점: 별도 마커.
- 지역별 숨겨진 아이템 방은 탐험 전까지 표시되지 않는다.

#### 설계 인사이트

- **단순하고 명확한 시스템이 장점이다.** 조건 없이 자동 공개되므로 학습 비용이 없다.
- Prie Dieu 마커가 세이브 포인트의 위치를 항상 시각화하여 "어디로 돌아가야 하는가"를 명확히 한다.
- 수직 구조의 고딕 세계(Cvstodia)를 표현하는 맵이 ECHORIS와 유사한 세계관 구조를 가진다.

## 2. 공통 패턴 추출 — 좋은 메트로베니아 맵의 핵심 요소

### 패턴 1: 방 단위 공개 (Room-by-Room Reveal)

6개 게임 모두 방 단위로 맵을 공개한다. 픽셀/타일 단위가 아닌 방이라는 의미 단위가 공개의 원자 단위다. 방에 "발을 들이는" 행위 자체가 맵 탐험의 보상 단위다.

### 패턴 2: 색상 코딩으로 방의 기능 구분

탐험 상태(미탐험/탐험 완료) + 기능 중요도(세이브, 보스, 상점)를 색상으로 구분한다. 3~5색 이내로 단순하게 유지한다. 색이 많아질수록 가독성이 낮아진다.

### 패턴 3: 이중 표시 계층 (HUD 미니맵 + 전체 맵)

- **HUD 미니맵:** 게임플레이 중 끊임없이 참조. 작고 단순하게. 현재 위치와 인접 방만 표시.
- **전체 맵:** 일시정지 상태에서 전략적 파악. 크고 상세하게. 전체 진행 상황 확인.

두 계층이 독립적으로 기능하며 상호 보완한다. 데드 셀과 메트로이드 드레드가 이 이중 구조의 정석이다.

### 패턴 4: 세이브 포인트의 명시적 마킹

모든 게임에서 세이브 포인트는 맵에 명확히 표시된다. 이는 "돌아갈 곳"에 대한 심리적 안전망이다. 특히 사망 페널티가 있는 게임에서 세이브 포인트 마킹은 필수다.

### 패턴 5: 수집/탐험 완료도 시스템

월하의 야상곡의 200.6%, 할로우 나이트의 그럽/에센스 수집, 메트로이드의 아이템 완료율. 숫자로 표현되는 완료도는 달성욕(Achievement)을 자극하는 강력한 동기다.

### 패턴 6: 선택적 맵 구매 vs 자동 공개의 양분

- **구매형 (Hollow Knight, Ori):** 탐험과 경제를 결합. 구매 전까지 안개. 높은 미지감, 높은 학습 비용.
- **자동형 (Blasphemous, SotN):** 진입 즉시 공개. 낮은 학습 비용, 낮은 미지감.
- **혼합형 (Metroid):** Map Station 발견 시 구역 미리보기. 탐험 후 자동 공개.

### 패턴 7: 수직 세계에서의 특수 고려사항

월하의 야상곡과 블래스퍼머스는 수직 구조가 강한 세계를 가지고 있다. 공통적으로 관찰되는 특징:

- 맵이 세로로 길다. 스크롤이 필수다.
- 아래로 내려갈수록 위험도가 증가하는 구조를 맵 색상이 암시한다.
- 수직 이동(상승/하강)이 명확히 표시되어야 하므로 방 연결 구조가 수평 맵보다 복잡하다.

## 3. ECHORIS 맵 시스템 설계안

### 3-1. 설계 원칙 (Design Principles)

ECHORIS의 맵 시스템은 다음 원칙에 따라 설계된다.

**원칙 1 — 탐험 기둥 우선:** 맵은 "발견의 기록"이어야 한다. 미지의 영역은 완전히 숨겨진다. 플레이어가 "저기 뭔가 있다"는 감정을 느끼려면, 지도가 미완성 상태여야 한다.

**원칙 2 — 수직 메가스트럭처 표현:** 세계는 수직으로 뻗은 구조다. 맵은 세로 스크롤이 자연스럽고, 하강할수록 심연에 가까워지는 느낌을 시각적으로 전달해야 한다.

**원칙 3 — 2-Space 분리 반영:** 월드 맵과 아이템계 맵은 별개의 시스템이다. 월드 맵은 영구적이고 누적적이다. 아이템계 맵은 진입마다 새로 생성되며, 탐험 기록이 지속되지 않는다.

**원칙 4 — 세이브 포인트가 앵커:** 세이브 포인트 방은 맵에서 항상 명확히 식별 가능해야 한다. 사망 후 복귀 지점이 어디인지 항상 알 수 있어야 한다.

**원칙 5 — 스파이크 강화:** 아이템계 진입 포인트(아이템 속으로 들어가는 순간)는 맵에서 특별하게 표시되어, 플레이어가 새로운 공간 차원으로 진입한다는 감각을 강화한다.

### 3-2. 수직 메가스트럭처에 맞는 맵 표현

#### 층위(Layer) 기반 수직 맵

ECHORIS의 세계는 7개 층위로 구성된다. 맵은 층위를 수직으로 쌓아 표시한다.

```
[상단] 천공의 정원 ───────────
       중앙 성채   ─────────── ← 시작점
       연구소 폐허 ─────────
       빙결 보존소 ─────────
       지하 수로   ──────────
       묘지/카타콤 ─────────
[하단] 심연의 구   ──────────── ← 최종 목적지
```

- **전체 맵 (일시정지):** 수직 방향으로 전체 층위가 표시된다. 현재 위치가 강조된다. 스크롤로 상하 이동.
- **HUD 미니맵:** 현재 층위만 표시. 인접 방과 현재 방을 보여주는 축소 격자.
- **층위 전환 표시:** 층위 연결 포인트(포털/계단)가 맵에 화살표 또는 기호로 표시된다.

#### 층위별 색상 테마

각 층위는 고유한 색상 테마를 가져 맵에서 즉각적으로 구분된다. 이는 세계의 다양성을 맵에서 시각적으로 전달하는 역할을 한다.

| 층위 | 테마 | 맵 색상 |
| :--- | :--- | :--- |
| 중앙 성채 | 고딕 성 | 회청색 (#5A7A8C) |
| 연구소 폐허 | 마법 연구소 | 보라색 (#6A4A8C) |
| 빙결 보존소 | 얼음 | 빙청색 (#4AACCC) |
| 지하 수로 | 수중/습지 | 남색 (#2A4A6C) |
| 묘지/카타콤 | 무덤 | 어두운 갈색 (#4A3A2A) |
| 천공의 정원 | 하늘/정원 | 연두색 (#4A8A4A) |
| 심연의 구 | 공허 | 심홍색 (#8C2A2A) |

탐험 완료 방: 위 테마색의 밝은 버전.
미탐험 방(Map Station 공개): 위 테마색의 매우 어두운 버전 (외곽선만).

### 3-3. HUD 미니맵 vs 전체 맵 사양

#### HUD 미니맵

- **위치:** 화면 우상단 코너.
- **크기:** 화면 높이의 약 20% 크기의 정사각형 패널.
- **표시 범위:** 현재 방 + 인접 방 (상하좌우 2칸 이내).
- **갱신 주기:** 실시간 (방 진입 즉시).
- **표시 정보:**
  - 현재 방: 밝은 강조색으로 테두리 표시.
  - 탐험 완료 방: 층위 테마색으로 채움.
  - 미탐험 방: 표시 안 함.
  - 세이브 포인트 방: 빨간색 별 아이콘.
  - 아이템계 진입 포인트: 금색 아이콘.
  - 보스 방: 해골 아이콘 (보스 처치 전).
- **투명도:** 평소 70% 불투명. 전투 중 40% 불투명 (시야 방해 최소화).
- **토글:** 단축키로 표시/숨김 전환 가능.

#### 전체 맵 (일시정지 화면)

- **접근:** 일시정지 메뉴 → 맵 탭.
- **표시 범위:** 전체 세계. 수직 스크롤 가능. 핀치/휠로 줌 지원.
- **현재 위치:** 깜박이는 점으로 표시.
- **층위 전환:** 층위 경계선이 맵에 표시됨. 클릭으로 해당 층위로 뷰 이동.
- **완료율:** 맵 우하단에 "탐험 완료: XX%" 표시 (탐험한 방 / 총 방 수).
- **마커 시스템:** 플레이어가 직접 마커를 배치할 수 있음 (상세 내용은 3-5절 참조).

### 3-4. 방 발견 및 탐험 상태 관리

#### 방의 4가지 상태

| 상태 | 조건 | 맵 표시 | HUD 표시 |
| :--- | :--- | :--- | :--- |
| `UNDISCOVERED` | 진입한 적 없음. Map Station도 미활성화 | 표시 안 함 | 표시 안 함 |
| `OUTLINED` | Map Station 활성화로 외곽선 공개 | 어두운 층위색 외곽선 | 표시 안 함 |
| `VISITED` | 한 번 이상 진입 | 층위 테마색으로 채움 | 표시 O |
| `CLEARED` | 방 내의 모든 적 처치 | VISITED와 동일 색. 별도 기호 추가 | 작은 체크 기호 |

#### 상태 전환 규칙

- `UNDISCOVERED` → `OUTLINED`: 해당 층위의 Map Station 활성화 시.
- `UNDISCOVERED` / `OUTLINED` → `VISITED`: 방에 처음 진입하는 순간. 즉각적으로 맵에 반영.
- `VISITED` → `CLEARED`: 방의 모든 일반 적 처치 시 (보스 방은 보스 처치 시).
- 상태는 영구적으로 저장된다. 사망, 로그아웃 후에도 유지.

#### Map Station 시스템

- 각 층위에 1개의 Map Station 방이 존재한다.
- Map Station 활성화 시: 해당 층위의 모든 방이 `OUTLINED` 상태로 공개된다. 방의 외곽선(형태)만 보이고, 내부 정보(적, 아이템)는 미공개.
- Map Station 활성화 전까지 플레이어는 탐험한 방만 볼 수 있다.
- Map Station 방 자체는 세이브 포인트와 인접하도록 배치한다 (안도 구간에서 준비하는 플레이 패턴 유도).

### 3-5. 마커 시스템

#### 자동 마커 (시스템이 자동으로 배치)

| 마커 | 아이콘 | 색상 | 조건 |
| :--- | :--- | :--- | :--- |
| 세이브 포인트 | 별 | 빨간색 | 방 최초 진입 시 자동 |
| Map Station | 지도 기호 | 흰색 | 발견 시 자동 |
| 보스 방 (미처치) | 해골 | 주황색 | 방 진입 시 자동 |
| 보스 방 (처치 완료) | 해골에 X | 회색 | 보스 처치 시 자동 변경 |
| 층위 게이트 | 자물쇠 | 노란색 | 게이트 발견 시 자동 |
| 스탯 게이트 | 방패 | 파란색 | 게이트 접근 시 자동 |
| 능력 게이트 | 열쇠 | 초록색 | 게이트 접근 시 자동 |

#### 수동 마커 (플레이어가 직접 배치)

- 최대 10개의 사용자 마커를 배치할 수 있다.
- 마커 종류: 방문 필요, 미수집 아이템, 미해결 퍼즐, 위험 구역, 메모 (텍스트 입력 가능).
- 탐험한 방에만 배치 가능하다.
- 사용자 마커는 HUD 미니맵에도 표시된다.

### 3-6. 아이템계 맵 시스템 (별도 규칙)

아이템계는 절차적으로 생성되며, 진입마다 새로운 지층이 생성된다. 데드 셀의 접근법을 참조한다.

- **전체 공개 방식:** 아이템계 미니맵은 현재 지층의 전체 구조를 처음부터 표시한다. Fog of War 없음.
- **이유:** 절차적 생성 레벨에서 Fog of War는 탐험 동기가 아닌 복잡도 증가다. 야리코미의 핵심은 "이번 지층에서 무엇이 나왔나"이며, 전략적 루트 결정을 위해 전체 구조를 알아야 한다.
- **표시 내용:** 현재 지층의 Room 그리드. 현재 위치. 보스 방. 이노센트 출현 방(탐험 후 추가). 탈출 포인트.
- **지층 완료 후:** 해당 지층의 맵 기록은 삭제된다. 다음 지층은 새로운 맵으로 시작.
- **재귀 진입 불가:** CLAUDE.md 금지 항목에 따라 재귀 진입은 삭제된 기능이다. 단일 아이템계 내 지층만 고려한다.

### 3-7. 구현 난이도와 우선순위

#### 구현 복잡도 평가

| 기능 | 복잡도 | 이유 |
| :--- | :--- | :--- |
| 방 상태 저장 및 공개 | 낮음 | Room ID별 상태 플래그 저장 |
| HUD 미니맵 렌더링 | 중간 | 실시간 업데이트, PixiJS Graphics API |
| 전체 맵 (일시정지) | 중간 | 스크롤/줌 UI, 전체 맵 데이터 렌더링 |
| 색상 코딩 시스템 | 낮음 | 상태별 색상 매핑 테이블 |
| 자동 마커 시스템 | 낮음 | 특수 방 유형별 마커 배치 로직 |
| 수동 마커 시스템 | 중간 | UI 입력, 저장, HUD 연동 |
| Map Station 시스템 | 중간 | 층위별 방 외곽선 일괄 공개 |
| 완료율 계산 | 낮음 | 방 수 집계 |
| 아이템계 미니맵 | 낮음 | 전체 공개 방식으로 단순화 |

#### 우선순위 구분

**Build 0 (현재 — 프로토타입):**

현재 전체 맵이 항상 노출되는 문제를 먼저 해결한다.

1. **방 상태 시스템 구현:** 각 Room에 `UNDISCOVERED | VISITED` 두 상태만 먼저 구현한다. (OUTLINED, CLEARED는 후순위)
2. **HUD 미니맵 기본 구현:** 현재 방 + 인접 방 표시. 탐험 완료 여부에 따른 색상 구분만.
3. **전체 맵 Fog of War:** 미탐험 방은 맵에서 완전히 숨긴다.
4. **세이브 포인트 자동 마커:** 단일 빨간 점으로 표시.

**Build 1 (알파):**

5. **층위 색상 테마 적용:** 7개 층위별 고유 색상.
6. **Map Station 시스템:** 층위 외곽선 일괄 공개.
7. **보스 방/게이트 자동 마커:** 기능별 아이콘 도입.
8. **전체 맵 UI 완성:** 수직 스크롤, 현재 위치 표시, 완료율.
9. **아이템계 미니맵:** 전체 공개 방식으로 구현.

**Build 2 (베타):**

10. **수동 마커 시스템:** 플레이어 메모 기능.
11. **CLEARED 상태 추가:** 방 내 적 처치 완료 표시.
12. **아이콘 하이라이트:** 같은 종류 마커 전체 강조 (메트로이드 드레드 방식).
13. **지형 세부 표시:** 방의 형태(높낮이)를 맵에 반영 (메트로이드 드레드 방식).

### 3-8. 데이터 구조 개요 (구현 참고용)

```typescript
// 방 탐험 상태
type RoomState = 'UNDISCOVERED' | 'OUTLINED' | 'VISITED' | 'CLEARED';

// 맵 데이터 (월드)
interface WorldMapData {
  zones: {
    [zoneId: string]: {
      rooms: {
        [roomId: string]: {
          state: RoomState;
          markers: MarkerData[];
        };
      };
      mapStationActivated: boolean;
    };
  };
  userMarkers: UserMarkerData[];
}

// 마커
interface MarkerData {
  type: 'SAVE' | 'MAP_STATION' | 'BOSS' | 'BOSS_CLEARED' | 'GATE_ABILITY' | 'GATE_STAT';
  position: { x: number; y: number };
}

// 사용자 마커
interface UserMarkerData {
  type: 'TODO' | 'ITEM' | 'PUZZLE' | 'DANGER' | 'NOTE';
  roomId: string;
  note?: string;
}
```

이 구조는 서버 저장 키-값 형태로 직렬화 가능하다. Room ID는 월드 절차적 생성 시스템(`System_World_ProcGen.md`)의 Grid 좌표 기반 ID와 일치해야 한다.

## 4. 요약 및 결론

### 핵심 설계 결정

| 항목 | 선택한 방향 | 근거 |
| :--- | :--- | :--- |
| 방 공개 방식 | 진입 즉시 자동 공개 (블래스퍼머스/월하 방식) | 학습 비용 없음. 탐험 기둥과 직접 연결. |
| 구매형 지도 여부 | 미도입 (Build 1까지 보류) | 스코프 관리. Lupo/Cornifer 방식은 NPC 시스템 필요. |
| HUD 미니맵 | 도입 (메트로이드 방식) | 수직 세계에서 방향 감각 유지 필수. |
| Map Station | 도입 (메트로이드 방식) | 탐험 동기 보강. 층위 외곽선 미리 공개로 기대감 생성. |
| 아이템계 맵 | 전체 공개 (데드 셀 방식) | 절차적 생성 레벨에서 Fog of War는 비효율. |
| 수직 표현 | 층위별 수직 적층 맵 | 수직 메가스트럭처의 정체성 강화. |
| 완료율 | 도입 (월하의 야상곡 방식) | 야리코미(탐험 완성)를 수치화하여 달성 동기 부여. |

### 탐험 기둥과의 정렬 검증

- 미지(未知)의 보존: 미탐험 방은 완전히 숨겨짐. "저기 뭔가 있다"의 감정 가능. ✓
- 발견의 보상: 방 진입 즉시 맵에 반영. 즉각적인 시각적 보상. ✓
- 성장의 기록: 탐험 완료율이 수치로 누적. 야리코미 달성 동기. ✓
- Map Station으로 기대감 생성: 층위의 형태를 미리 보여줌으로써 "저기 가고 싶다"는 동기 강화. ✓
