## 1. 타격감(Game Feel) 핵심 요소

### 1.1 히트스탑(Hit Stop / Freeze Frame)

타격 순간 게임 로직을 일시 정지시켜 충격의 무게를 시각적으로 전달하는 기법. 사쿠라이 마사히로의 "공격 위력 비례 히트스탑"이 업계 표준.

**레퍼런스 게임별 프레임 수치 비교 (60fps 기준)**

| 게임 | 약공격 | 중공격 | 강공격 | 킬 타격 | 비고 |
|:-----|:------:|:------:|:------:|:------:|:-----|
| **Smash Bros. Ultimate** | 6~8f | 9~11f | 12~15f | 동일 | 공식: `⌊(d×0.65+6)×h×e⌋` |
| **Dead Cells** | 2~3f | 4~5f | 6~8f | 없음 | 피격자만 정지, 공격자 지속 |
| **Hollow Knight** | 1~2f | 2~3f | 3~4f | 없음 | 극도로 짧음, 반응성 우선 |
| **SotN** | 없음 | 없음 | 없음 | 없음 | 히트스탑 미적용, 넉백으로 대체 |
| **Project Abyss (현재)** | 1타 3f | 2타 4f | 3타 6f | +5f | 사쿠라이 기법 기반 |

**스매시 브라더스 히트스탑 공식:**
```
hitlag = ⌊(damage × 0.65 + 6) × hitlag_multiplier × electric_modifier⌋
최소값: 3프레임
예시: 15% 공격 = ⌊15×0.65+6⌋ = 15프레임
```

**설계 원칙:**
- 약공격(1~2타): 리듬 유지를 위해 2~4f
- 강공격(3타): 피날레 카타르시스를 위해 6~8f
- 원소 상태이상 적용 시: +1~2f 추가
- 킬 타격: +5f 보너스

### 1.2 화면 흔들림(Screen Shake)

**Jan Willem Nijman "Art of Screenshake" 핵심 원칙:**
1. 충격 벡터 방향으로 편향 (무방향 = 혼란)
2. 지수적 감쇠 곡선 필수 (선형 = 부자연스러움)
3. 강도 범위: 1~8px (그 이상 = 멀미)
4. 지속 시간: 200~400ms

| 게임 | 약공격 | 강공격 | 보스 피격 | 감쇠 | 방향성 |
|:-----|:------:|:------:|:--------:|:-----|:------|
| **Dead Cells** | 1~2px | 3~5px | 8~12px | 지수적 | 넉백 방향 |
| **Hollow Knight** | 0~1px | 2~3px | 5~7px | 지수적 | 타격 방향 |
| **Project Abyss** | 1.5px | 4px | 미설계 | 0.88/f | 넉백 방향 ✓ |

### 1.3 넉백(Knockback)

| 게임 | 약공격 | 강공격 | 벽 충돌 | 무게 영향 |
|:-----|:------|:------|:-------|:---------|
| **SotN** | ~30px 수평 | ~60px 수평 | 정지 | 없음 |
| **Dead Cells** | 80~120px | 200~300px | 바운스/스턴 | 체급별 감소 |
| **Hollow Knight** | 20~40px | 50~100px | 정지 | 없음 |
| **Project Abyss** | 120px/s | 240px/s | 미설계 | 1.0 고정 |

**벽 충돌 처리 옵션:**
1. **정지 방식** (SotN): 단순, 안정적
2. **바운스 방식** (Dead Cells): 동적, 구현 복잡
3. **슬라이드 방식**: 수직 전환, 공중 콤보 연계

### 1.4 이펙트(VFX)

| 게임 | 파티클 수 | 스파크 형태 | 수명 |
|:-----|:--------:|:-----------|:----:|
| **Dead Cells** | 8~15개 | 선형+원형 혼합 | 150~250ms |
| **Hollow Knight** | 4~8개 | 동심원 스파크 | 100~180ms |
| **Project Abyss** | 라이트4/헤비7 | 선형+중앙플래시 | 180ms |

### 1.5 사운드(SFX)

**Dead Cells 3레이어 구조:**
1. 임팩트 레이어 (저주파, 0ms): 물리적 충격감
2. 슬래시 레이어 (고주파, 0ms): 날카로운 절삭감
3. 히트 반응 레이어 (30~50ms 지연): "맞았다" 확인

**타이밍 설계:**
```
공격 판정 (0ms)
├── 임팩트음 + 슬래시음: 즉시 (0ms)
├── 피격체 반응음: +30~50ms
├── 데미지 넘버 팝업: +50ms (음향 선행 → 시각 후행)
└── 킬 사운드: 별도 레이어
```

## 2. 공격 시스템 비교 분석

### 2.1 월하의 야상곡 (SotN)

- 두 손 슬롯 독립 입력, 백대시 단일 버튼
- **무기 속도:** 단검 3~5f / 일반 검 8~12f / 대검 15~20f / 카타나 6~8f
- **서브웨폰:** 하트 소비 5종 (단검/도끼/성수/십자가/회중시계)
- **백대시 캔슬(BDC):** 스피드런 발견 기법 → 공격 후딜 취소. **교훈:** 의도적 캔슬 시스템을 설계에 명시하면 공식 시스템이 됨
- **넉넉한 입력 버퍼** → "접근성 있는 심화 전투"의 핵심

### 2.2 데드셀(Dead Cells)

- **4슬롯:** 무기 2 + 스킬 2 + 부적 1
- **롤링 닷지:** 0.4초 지속, 0.37초 쿨다운, i-frame 보유, 히트박스 1타일 축소
- **패링:** 적 공격 타이밍에 블록 → 반격 + i-frame
- **Auto-aim:** 빠른 무기 → 소형 적, 느린 무기 → 대형 적 우선
- **핵심:** 무기×스킬 시너지가 빌드의 핵심 (냉동+헤비 등)

### 2.3 Hollow Knight

- **네일 공격:** 전방/상방/하방 3방향
- **네일 아트:** Great Slash(x2.5), Dash Slash(x2.5), Cyclone Slash(x1.25×7히트)
- **소울 시스템:** 네일 히트 1회 = 11 Soul / 스펠/힐링 = 33 Soul 소비
  - **"공격하지 않으면 회복도 없다"** = Push Forward Combat (DOOM Glory Kill과 동일 철학)
- **패링:** 흰색 예비 동작 공격만 가능. 타이밍 창 매우 좁음 → 스킬 표현 공간
- **i-frame:** 피격 후 ~1.5초 / Shadow Dash 전 구간 / 패링 0.25초

## 3. 피격/방어 시스템

### 3.1 무적 프레임(i-frame) 비교

| 게임 | 피격 후 | 대시 | 설계 의도 |
|:-----|:------:|:----:|:---------|
| **SotN** | 0.5~1초 | 없음 | 탐험 쾌적성 |
| **Dead Cells** | ~0.5초 | 롤 전구간 0.4초 | 롤이 핵심 회피 |
| **Hollow Knight** | ~1.5초 | 없음→Shadow Dash 전구간 | 업그레이드 보상 |
| **Project Abyss** | hitstun 중 | **수치 미명시** | **갭 존재** |

### 3.2 패링 3요소 원칙

1. **텔레그래프:** 패링 가능 공격에 예비 동작 표시
2. **타이밍 창:** 150~300ms가 적정
3. **보상:** 데미지+i-frame 또는 MP 회복

### 3.3 상태이상 시너지 (Dead Cells 참조)

| 상태이상 | 효과 | 빌드 시너지 |
|:--------|:-----|:-----------|
| 냉동 | 이동불가, 피해 취약 | 헤비 무기 "냉동+강타" 콤보 |
| 독 | 지속 피해 | 마크 무기+독으로 폭발적 DPS |
| 출혈 | 이동 시 피해 증폭 | 빠른 이동 적에게 효과적 |
| 화상 | 지속 피해+방어력 감소 | 약점 연계 |
| 마비 | 짧은 행동 불능 | 범용 시너지 |

### 3.4 체력 회복 메커니즘 비교

| 게임 | 기본 회복 | 전투 중 | 설계 의도 |
|:-----|:---------|:-------|:---------|
| **SotN** | 포션 아이템 | 자동 MP 회복 | 아이템 관리 |
| **Dead Cells** | 특정 변이/스킬 | 적 처치 시 일부 | Push Forward 약화 |
| **Hollow Knight** | 벤치 | Soul 소비 힐링 | 공격=회복 연료 |
| **Project Abyss** | **미명시** | **미명시** | **설계 갭** |

## 4. 적 전투 AI 패턴

### 4.1 일반 적 행동 트리 (Dead Cells)

```
[순찰] → 감지 → [추격] → 사거리 진입
→ [텔레그래프 0.3~0.5초] → 공격 → [회복 0.3~0.8초] → 복귀
```

**DOOM "한 문장 설명" 원칙:**
- 근접 러셔: "돌진하여 공격"
- 원거리 포지셔너: "거리 유지, 투사체"
- 방어형: "반격 창이 열림"
- 유틸형: "다른 적 강화/힐링"

### 4.2 보스 페이즈 설계

**공통 원칙:**
- HP 임계점(50%, 33%)에서 페이즈 전환
- 페이즈마다 새 패턴 추가 + 속도 증가
- **Delta of Chance:** 공격 파동 → 안전 창(1~2초) → 반복 (안전 창 없으면 "불공평")

**Project Abyss 아이템계 보스 페이즈 권장:**
- 아이템 장군: 1~2페이즈
- 아이템 왕: 2~3페이즈
- 아이템 신: 3페이즈
- 아이템 대신: 4페이즈 + 최종 형태

## 5. 멀티플레이어 전투 과제

### 5.1 히트스탑 동기화 — 3가지 방식

| 방식 | 설명 | 장점 | 단점 |
|:-----|:-----|:-----|:-----|
| A: 클라이언트 로컬 | 자기 공격만 로컬 히트스탑 | 레이턴시 없음 | 타인 공격 일관성 깨짐 |
| B: 서버 동기화 | 서버 브로드캐스트 | 전원 동일 경험 | 50~100ms 지연 |
| **C: 하이브리드 (권장)** | 공격자 즉시 로컬 + 서버 확인 후 피격자 동기화 | 즉각적 피드백 + 동기화 | 롤백 필요 |

### 5.2 어그로 시스템

```
위협도 = (누적 데미지 × 1.0) + (도발 스킬 × 3.0) + (근접 여부 × 2.0)
→ 1~2초마다 재평가, 최고 위협도 플레이어를 타겟
```

### 5.3 협동 시너지

- 역할 분담 **유도**하되 **강제하지 않음**
- 원소 연계: 플레이어 A 디버프 → 플레이어 B 반응 스킬 → 폭발 데미지

### 6.2 웹 환경(60fps) 히트스탑 구현 주의점

1. **프레임→밀리초 기반 전환 권장:** `requestAnimationFrame`이 정확한 60fps 미보장
2. **히트스탑 중 서버 이벤트:** 큐에 쌓아두면 "텔레포트" 현상 → 실시간 보간 필요
3. **모바일 프레임 드랍:** 최소 지속 시간 보장 로직 필수

```typescript
// 권장 구현 패턴
const HITSTOP_MS = { light: 50, heavy: 100, kill: 183 };

if (this.hitstopRemaining > 0) {
  this.hitstopRemaining -= app.ticker.deltaMS;
  return; // 물리 업데이트 스킵
}
```
