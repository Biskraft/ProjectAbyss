# 인벤토리 온보딩 UX 리서치 — 첫 인벤토리 열기+장착의 허들 해소

> **작성일:** 2026-04-26
> **목적:** ECHORIS에서 "첫 인벤토리 열기 + 아이템 장착"이 플레이어에게 허들이 되는 원인을 분석하고, 레퍼런스 게임의 해법을 종합하여 개선안을 도출한다.
> **핵심 문제:** 인벤토리 관리 자체가 허들. 색상/UX가 직관적이지 않음. "무엇을 해야 하는지" 불분명.

---

## 1. 왜 인벤토리가 허들인가 — 5대 원인

| 원인 | 설명 | ECHORIS 해당 여부 |
|:-----|:-----|:-----------------|
| **몰입 파괴** | 인벤토리를 열면 전투/탐험 흐름이 끊김. "갑자기 영웅이 아니라 회계사가 된다" | **해당.** I키 → 그리드 → 방향키 탐색 → X키 장착 = 4단계 |
| **선택 마비** | 첫 오픈 시 너무 많은 옵션/탭/스탯이 보이면 즉시 닫아버림 | **해당.** 5x4 그리드 + Info Box + 레어리티 5색 + 스탯 |
| **"이게 뭘 하는 건데?"** | 장착 전후의 변화가 시각적으로 보이지 않으면 행위의 의미를 모름 | **해당.** 장착해도 에르다 외형 변화 없음 (픽셀아트 제약) |
| **색상 의미 불명** | 레어리티 색상(흰/파/노/주/초)의 의미를 첫 오픈에서는 모름 | **해당.** 색상 학습은 2-3번째 아이템 획득에서 자연 발생해야 함 |
| **키 조작 불확실** | "어떤 키를 눌러야 장착하지?" — 키보드 전용 게임의 고유 문제 | **해당.** Z/X/C/I/ESC 중 어떤 키인지 불분명 |

## 2. 레퍼런스 게임 비교 — "첫 인벤토리 순간"

| 게임 | 첫 오픈 시점 | 아이템 수 | 교육 방식 | 핵심 패턴 |
|:-----|:-----------|:---------|:---------|:---------|
| **Dead Cells** | 0초 (시작 시 자동장착) | 1개 | 바닥 무기 위에서 비교 툴팁 자동 표시 | **인벤토리 제거** |
| **Hades** | 5-15분 (첫 사망 후) | 1개 | 디스플레이 케이스에 1개뿐 → 선택 마비 없음 | **하나의 슬롯, 하나의 선택** |
| **BotW** | 2-5분 | 2-3개 | 상자에서 획득 시 "Press + to equip" 안내 | **빈 슬롯 자동 장착** |
| **Diablo 3/4** | 1-3분 | 시작 장비+드롭 1-2개 | 드롭 위 마우스 → 자동 비교 + 녹색/빨강 화살표 | **자동 비교 툴팁** |
| **Hollow Knight** | 15-30분 | 0-1개 | 벤치(세이브 포인트)에서만 장착 가능 | **전용 장착 장소** |
| **SotN** | 프롤로그 | 6개→0개 | 최강 장비 체험 → Death가 박탈 → 재획득 동기 | **박탈 후 재획득** |
| **Elden Ring** | 즉시 | 3-6개 | **없음** (의도적 불친절) | **안티 패턴** |

### ECHORIS 현재 상태

| 항목 | 현재 | 문제 |
|:-----|:-----|:-----|
| 첫 오픈 시점 | Broken Blade 자동 장착 후, 적 드롭 시 I키 | **I키를 눌러야 한다는 것을 모름** |
| 아이템 수 | Broken Blade 1개 + 드롭 1개 = 2개 | 적절 |
| 교육 방식 | 없음. 심볼 프롬프트도 없음 | **핵심 문제** |
| 장착 방법 | I키 → 방향키 → X키 | **3단계. 키 불명확** |
| 시각 피드백 | 없음 (스탯 수치만 변화) | **장착 의미 불분명** |

## 3. 해결 패턴 6종

### 패턴 A: 인벤토리 제거 (Dead Cells)

```
월드에서 무기 위를 지나감 → 비교 툴팁 자동 → 상호작용 버튼 → 즉시 교체
```

- 장점: 허들 자체가 없음
- 단점: ECHORIS의 20슬롯 인벤토리 + 아이템계 진입 선택과 충돌
- **판정: 아이템계 내부 드롭에만 부분 적용 가능**

### 패턴 B: 하나의 슬롯, 하나의 선택 (Hades)

```
첫 장비 획득 → 슬롯 1개뿐 → 선택 마비 없음 → 자동 장착
```

- 장점: 첫 인터랙션 의사결정 비용 0
- **판정: ECHORIS 첫 무기(Broken Blade) 자동 장착에 이미 적용 중. 두 번째 무기부터가 문제**

### 패턴 C: 전용 장착 장소 (Hollow Knight)

```
벤치(세이브 포인트)에서만 장착 → 전투 중 변경 불가 → 안전 공간에서 실험
```

- 장점: 실패 공포 제거, 전략적 선택
- 단점: 즉각 피드백 지연
- **판정: ECHORIS의 세이브 포인트 기반 아이템계 진입과 잘 맞음**

### 패턴 D: 자동 비교 툴팁 (Diablo)

```
드롭 아이템 위 → 현재 장비와 자동 비교 → ↑(녹색)/↓(빨강) 즉시 표시
```

- 장점: 인벤토리를 열지 않고도 "이게 더 좋은가?" 즉시 판단
- **판정: ECHORIS에 가장 시급한 추가 기능**

### 패턴 E: 박탈 후 재획득 (SotN)

```
프롤로그: 강한 무기 체험 → 무기 상실 → 약한 무기로 재시작 → 되찾고 싶다
```

- 장점: 파워 판타지 맛보기 → 동기 생성
- **판정: 오프닝 거대 빌더 씬과 결합 가능**

### 패턴 F: 점진적 공개 (Progressive Disclosure)

```
첫 오픈: 장착 버튼만 보임
두 번째: 비교 모드 추가
세 번째: 상세 스탯 추가
...
```

- 장점: 인지 부하 최소화
- **판정: ECHORIS의 Level 1/2/3 정보 계층(UI_Inventory.md)과 정확히 일치하는 구조**

### 4.2 진짜 남은 허들 — "왜"와 "교체 vs 보관"의 혼란

기능은 구현되어 있다. **문제는 플레이어의 멘탈 모델이다:**

| 허들 | 상세 | 왜 문제인가 |
|:-----|:-----|:-----------|
| **"왜 장착해야 하지?"** | Lore Popup에 스탯은 보이지만 현재 장비와의 비교가 없음. "이게 내 현재 무기보다 좋은 건지" 모름 | 장착 동기 불분명 |
| **"교체하면 이전 무기는?"** | 무기를 교체하면 이전 무기가 인벤토리에 남는데, "아이템계에 들어갈 수 있는 무기를 잃는 건 아닌지" 불안 | 앤빌 시스템과의 관계 불명확 |
| **"장착 무기 vs 앤빌 무기"** | 장착 중인 무기는 앤빌에 올릴 수 없음. "왜?"라는 질문에 게임이 답하지 않음 | 시스템 규칙 불투명 |
| **비교가 수동(Z키)** | 비교를 보려면 Z키를 눌러야 하는데, Z키가 비교라는 것을 모름 | 핵심 정보 접근 허들 |

### 4.3 ECHORIS 고유 제약 — "즉시 교체"가 안 되는 이유

Dead Cells/Diablo식 "픽업 시 즉시 교체" 패턴은 ECHORIS에 **맞지 않는다:**

```
ECHORIS의 무기 = 단순 스탯 도구가 아니라 "던전 그 자체"

  무기 A (장착 중, ATK 20) ←── 이걸로 전투한다
  무기 B (인벤토리, ATK 15) ←── 이걸 앤빌에 올려서 아이템계에 진입한다

  만약 B가 더 강해서 즉시 교체하면:
  → 무기 A가 인벤토리로 감
  → 이제 A를 앤빌에 올릴 수 있음 (장착 해제됐으니까)
  → 하지만 플레이어는 "방금 장착을 바꿨는데, 전투 무기와 다이브 무기가 뒤바뀐 거야?"라고 혼란
```

**핵심:** ECHORIS에서 무기 교체는 **전투력 변경 + 다이브 대상 변경**이 동시에 일어나는 전략적 결정이다. 월드에서 무심코 교체하면 안 된다.

## 5. 개선안 — 실제 구현에 기반

### 5.1 Lore Popup에 비교 정보 추가 (가장 시급)

현재 Lore Popup은 아이템의 스탯만 표시한다. **현재 장착 무기와의 비교 한 줄**을 추가하면, "이게 더 좋은지"를 Popup 단계에서 즉시 판단할 수 있다.

```
[현재 Lore Popup]
  ┌──────────────────────┐
  │  Iron Blade           │
  │  MAGIC  Lv.1          │
  │  ATK: 20              │
  │  "Forged in memory.." │
  │                       │
  │  [C] CLOSE            │
  └──────────────────────┘

[개선안]
  ┌──────────────────────────┐
  │  Iron Blade               │
  │  MAGIC  Lv.1              │
  │  ATK: 20  (+5 ↑ vs 장착)  │  ← 녹색, 현재 장착과 비교
  │  "Forged in memory..."    │
  │                           │
  │  [C] CLOSE                │
  └──────────────────────────┘
```

- 교체를 강제하지 않음 (Lore Popup은 닫기만 가능)
- "이 무기가 더 강하다/약하다"는 정보만 전달
- 교체 결정은 플레이어가 나중에 인벤토리에서 직접 수행
- **무기 = 던전**이라는 특성상 "강한 무기 = 좋은 던전"이 아닐 수 있으므로, 교체를 강제하지 않는 것이 올바름

### 5.2 비교 자동 표시 (Z키 토글 → 선택 시 자동)

현재 Z키로 Compare Mode를 토글해야 한다. **아이템을 선택하는 것만으로 장착 무기와의 비교가 Info Box에 항상 표시**되도록 변경한다.

```
[현재]
  아이템 선택 → Info Box에 스탯만 표시
  Z키 → Compare Mode 활성화 → 비교 표시

[개선안]
  아이템 선택 → Info Box에 스탯 + 비교가 항상 표시
  (Z키는 Detail View용으로 유지)
```

변경 범위: `InventoryUI.ts:388-396` — `if (this.compareActive` 조건 제거. 장착 무기가 있고 선택 아이템이 장착 아이템과 다르면 항상 비교 표시.

### 5.3 "장착 무기는 앤빌 불가" 규칙의 시각적 설명

현재 장착 중인 무기를 앤빌에 올리려 하면 "Unequip first" 토스트가 뜬다. 하지만 **왜 장착 무기가 앤빌에 올라가지 않는지**를 모른다.

```
[현재]
  앤빌 모드에서 장착 무기 선택 → C키 → "Unequip first" 토스트

[개선안]
  앤빌 모드에서 장착 무기 선택 시:
  → Info Box에 "[E] Equipped — 장착 해제 후 배치 가능" 표시
  → 또는 무기 Ego: "I'm in your hands. Choose another."
  → 장착 슬롯에 자물쇠(🔒) 아이콘 오버레이
```

### 5.4 무기 Ego가 온보딩을 담당

CLAUDE.md 스파이크: **"무기는 Ego를 가지고 에르다에게 말을 건다."**

이것을 인벤토리 온보딩에 활용:

| 시점 | Ego 대사 | 기능 |
|:-----|:---------|:-----|
| 두 번째 무기 획득 시 | *"Compare us. Open your pack."* | [I] 키 유도 |
| 인벤토리에서 강한 무기 선택 시 | *"I'm stronger. Equip me."* | 장착 유도 |
| 앤빌 첫 접근 시 | *"Place me on the anvil. I'll show you what's inside."* | 앤빌 사용 유도 |
| 장착 무기를 앤빌에 올리려 할 때 | *"I'm in your hands. Choose another."* | 규칙 설명 |
| 첫 다이브 시 | *"Dive into my memory."* | 다이브 유도 |

**구현:** LorePopup과 동일한 Pattern B (하단 텍스트 박스). 무기 Ego 전용 색상 (레어리티 색). 1회성 (같은 대사는 두 번 표시 안 함).

## 5. 색상/시각 개선

### 5.1 현재 색상 문제

| 요소 | 현재 | 문제 |
|:-----|:-----|:-----|
| 빈 슬롯 | `0x2A2A3E` (짙은 남색) | 배경과 구분 어려움 |
| 선택됨 | `0x4A4A8A` (밝은 남색) | 미묘한 차이. "선택됐는지" 불분명 |
| 장착 중 | 흰색 테두리 1px | **너무 얇아서 인지 어려움** |
| 레어리티 Normal | 흰색 `0xFFFFFF` | 빈 슬롯과 혼동 가능 |

### 5.2 개선안

| 요소 | 개선 | 이유 |
|:-----|:-----|:-----|
| 선택됨 | **노란색 테두리 2px** + 슬롯 배경 밝아짐 | 현재 커서 위치 즉시 인지 |
| 장착 중 | **흰색 테두리 2px + "E" 배지** (우상단 작은 원) | Diablo의 장착 아이콘 참조 |
| 신규 아이템 | **빨간 점(!) 배지** (우상단) | "새 아이템이 있다" 즉시 인지 |
| 비교 상승 | **녹색 화살표 ↑ + 녹색 수치** | Diablo 표준 |
| 비교 하락 | **빨간 화살표 ↓ + 빨간 수치** | Diablo 표준 |
| 첫 오픈 하이라이트 | **선택 가능 슬롯 펄스 (0.8초 주기)** | "여기를 선택하라" 어포던스 |

## 6. 키 조작 개선

### 현재 키 매핑 문제

| 상황 | 현재 키 | 문제 |
|:-----|:-------|:-----|
| 인벤토리 열기 | I | "I가 뭔지" 모름. 심볼 프롬프트 없음 |
| 아이템 장착 | X | **게임 중 X=대시인데, 인벤토리에서 X=장착.** 혼란 |
| 비교 | C | 인벤토리에서 C=Compare인데, 게임 중 C=공격. 혼란 |

### 개선안

| 상황 | 개선 키 | 이유 |
|:-----|:-------|:-----|
| 인벤토리 열기 | I (유지) | 표준. 단, **첫 획득 시 [I] 프롬프트 표시** |
| **아이템 장착** | **C** (Attack=Confirm 통일) | Pattern A 규약: C=범용 확인. X/Z는 UI 사용 금지 |
| 상세 보기 | Z | 유지 (Level 3 Detail) |
| 닫기 | ESC / I | 유지 |
| 비교 | 자동 (선택 시 장착 아이템과 자동 비교) | **별도 키 불필요.** 선택만 하면 Info Box에 비교 표시 |

**핵심 변경:** 장착 키를 X→C로 변경. 이유:
1. `UI_Interaction_Patterns.md`의 **Pattern A: C=확인** 규약과 일치
2. X키(Dash)가 UI에서 다른 의미를 갖는 것은 근육 기억 충돌 (피드백 이미 존재: `feedback_ime_input.md`)
3. C=공격=확인=장착. 하나의 키가 "실행/확인"의 일관된 의미

## 7. 비교 모드 자동화

### 현재: C키로 Compare Mode 수동 토글

```
아이템 선택 → C 누름 → Compare Mode 활성화 → 비교 표시
```

### 개선: 선택만 하면 자동 비교

```
아이템 선택 (방향키 이동) → Info Box에 자동으로 장착 아이템과 비교 표시
  ATK: 20 → 25 (+5 녹색)
  INT: 0 → 0
  HP:  0 → +10 (+10 녹색)
```

- **별도 Compare Mode 키가 불필요해짐**
- C키가 Compare에서 해방되어 **장착(Equip)으로 전환 가능**
- Diablo/PoE/D4 모두 이 방식 (자동 비교가 표준)

| **P2** | 무기 Ego 온보딩 대사 시스템 | 3시간 | 스파이크 연동 ("말을 거는 무기") |
| **P2** | 선택 슬롯 하이라이트 강화 (테두리 2px) | 1시간 | 시각 인지 개선 |

## 핵심 교훈 5가지

1. **ECHORIS에서 무기 교체는 전략적 결정이다.** Dead Cells식 "즉시 교체"는 맞지 않음. 무기 = 던전이므로, 교체 = 전투력 변경 + 다이브 대상 변경이 동시에 일어남. 교체를 강제하지 말고, **비교 정보만 제공**하여 플레이어가 판단하게 해야 한다.

2. **비교 정보는 획득 순간부터 보여야 한다.** Lore Popup에서 현재 장비와의 비교 한 줄을 추가하면, 인벤토리를 열기 전에 "이게 더 좋은지" 판단할 수 있다. 인벤토리 내 비교도 Z키 토글이 아닌 자동 표시로 전환해야 한다.

3. **"장착 무기는 앤빌 불가" 규칙을 설명해야 한다.** 현재 "Unequip first" 토스트만 표시. 왜 장착 무기가 앤빌에 올라가지 않는지를 자물쇠 아이콘 + 무기 Ego 대사로 설명해야 한다. ("I'm in your hands. Choose another.")

4. **색상보다 형태.** 레어리티 색상은 경험자에게 강력하지만 첫 사용자에게는 무의미. 장착 상태("[E]" 배지), 신규 아이템("!" 배지) 등 **형태 기반 정보 전달**이 선행되어야 한다.

5. **무기의 Ego가 온보딩을 담당한다.** 에르다는 대사 0줄이지만, 무기가 말을 건다 (CLAUDE.md 스파이크). 인벤토리 온보딩의 모든 안내를 무기 Ego 대사로 처리하면, 스파이크와 UX가 동시에 해결된다.
