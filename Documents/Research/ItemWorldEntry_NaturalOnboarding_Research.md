### 1.2 핵심 질문

사용자의 요청: "플레이타임 1분 내에 아이템계에 도달해서 들어가는데, 어색함을 느끼고 있어."

이 문장에는 두 가지 가설이 내포되어 있다:
- **가설 A:** 1분이 너무 빠르다 (시간 문제)
- **가설 B:** 1분이든 5분이든, 동기/동의/문맥 없이 들어가면 어색하다 (구조 문제)

본 리서치는 두 가설을 모두 검증한다.

## 3. 레퍼런스 분석 -- 첫 차원 진입의 타이밍과 동기

### 3.1 "아이템/장비 속으로 들어가기" 계열

#### R-01. 디스가이아 시리즈 -- 아이템계

| 항목 | 내용 |
|:-----|:-----|
| 첫 해금 타이밍 | **챕터 3** (약 3-5시간 후) |
| 해금 조건 | 스토리 진행으로 NPC 등장 |
| 동기 부여 | 스토리: "이 아이템을 강화하지 않으면 다음 보스를 못 이긴다" + 튜토리얼 강제 진입 |
| 동의 메커니즘 | NPC에게 말 걸기 -- 아이템 선택 -- 확인 (3단계 명시적 동의) |
| 예고 | NPC 대사로 "아이템 안에 세계가 있다, 클리어하면 강해진다" 설명 |
| 문맥 | 이미 3챕터 분량의 전투 경험 -- 장비 스탯의 중요성 체득 완료 |
| 첫 진입 시 무료 탈출 아이템 지급 | Mr. Gency's Exit 여러 개 지급 -- "갇히지 않는다" 안전망 |

**핵심 교훈:**
- 디스가이아는 아이템계를 **게임의 핵심 시스템**이면서도 **수 시간 후에** 해금한다
- 해금 전까지 플레이어는 "장비가 중요하다"를 충분히 학습한다
- 3단계 동의 + NPC 설명 + 탈출 아이템으로 안전망 제공
- SRPG라 "시스템 이해 후 진입"이 자연스러움. 액션 RPG와는 다른 맥락

#### R-02. 사이코너츠 -- 정신 세계 진입

| 항목 | 내용 |
|:-----|:-----|
| 첫 진입 타이밍 | **약 15분** (기본 탐색 후 첫 훈련) |
| 동기 부여 | "초능력 훈련을 위해 코치의 정신 속으로 들어가라" -- 목적이 명확 |
| 동의 메커니즘 | 사이코 포탈을 대상에게 **직접 던지는** 능동적 행위 |
| 예고 | 캠프 NPC들이 "정신 속에 들어가면 훈련할 수 있다" 반복 언급 |
| 탈출 | 언제든 메뉴에서 탈출 가능 |

**핵심 교훈:** "남의 머릿속에 들어간다"는 기이한 컨셉을 **훈련이라는 익숙한 프레이밍**으로 감쌌다.

### 3.2 "절차적 던전에 즉시 진입" 계열

#### R-03. Dead Cells -- 감옥에서 깨어남

| 항목 | 내용 |
|:-----|:-----|
| 첫 진입 타이밍 | **즉시 (0초)** -- 깨어나면 이미 절차적 레벨 안 |
| 동기 부여 | 필요 없음. "여기서 나가야 한다"가 동기 그 자체 |
| 동의 메커니즘 | 없음. 시작이 곧 진입 |
| 어색함? | **없음** -- 탈옥이라는 내러티브가 즉시 진입을 정당화 |

**핵심 교훈:** "이미 안에 있다" 패턴. 차원 전환이 없으므로 어색할 수가 없다.

#### R-04. Hades -- 지하세계 탈출

| 항목 | 내용 |
|:-----|:-----|
| 첫 진입 타이밍 | **약 15초** -- 대사 2줄 후 첫 방 진입 |
| 동기 부여 | "아버지 집에서 탈출하고 싶다" -- 보편적 감정 |
| 동의 메커니즘 | 첫 문을 **직접 열기** |
| 죽음 후 재진입 | 하데스 집에서 NPC 대화 -- 무기 선택 -- 문 열기 (3단계) |
| 어색함? | **없음** -- 매 런이 "다시 도전"이라는 명확한 의지 |

**핵심 교훈:** 로그라이크의 "런 시작"은 **플레이어의 의지(agency)**로 시작된다. Hades는 이를 서사적으로 "탈출 의지"에 연결했다.

#### R-05. Enter the Gungeon -- 건전 강하

| 항목 | 내용 |
|:-----|:-----|
| 첫 진입 타이밍 | **즉시** -- 오프닝에서 캐릭터가 건전에 뛰어듦 |
| 동기 부여 | 캐릭터 선택 화면에서 각 캐릭터의 "과거를 바꾸고 싶다"는 동기 제시 |
| 동의 메커니즘 | 게임 시작 = 진입 동의. 입구 광장에서 NPC 탐색 가능 |

**핵심 교훈:** "게임을 시작한 것 자체가 동의"로 작동하려면, 게임의 **전체가** 그 던전이어야 한다.

#### R-06. Slay the Spire -- 첨탑 등반

| 항목 | 내용 |
|:-----|:-----|
| 첫 진입 타이밍 | **즉시** -- 맵 선택이 곧 게임 |
| 동기 부여 | "첨탑 꼭대기에 도달하라" -- 목표가 게임 자체 |
| 동의 메커니즘 | 첫 경로 노드를 **직접 클릭** |

**핵심 교훈:** 카드 로그라이크는 "런 시작이 게임 시작"이므로 차원 전환 문제가 없다.

| 동기 부여 | "보물이 있다" -- 탐험가 판타지 |

### 3.3 "월드 탐험 후 특수 공간 진입" 계열 (ECHORIS와 가장 유사)

#### R-08. 할로우 나이트 -- 드림 네일

| 항목 | 내용 |
|:-----|:-----|
| 첫 해금 타이밍 | **약 8-15시간** (Crystal Peak 이후 Resting Grounds) |
| 동기 부여 | 스토리 + 꿈의 세계에서만 얻을 수 있는 보상(Essence) |
| 동의 메커니즘 | 드림 네일로 대상을 **직접 베는** 능동적 행위 |
| 빌드업 | 수 시간의 탐험으로 "이 세계에 숨겨진 것이 더 있다"는 기대 축적 |
| 진입 전 정보 | 대상에게 근접하면 말풍선(꿈 대사)이 미리 표시 |

**핵심 교훈:** 할로우 나이트는 꿈 세계를 **중반 이후에** 해금한다. 플레이어가 "일반 세계에서 할 수 있는 모든 것"을 충분히 경험한 후에 새로운 차원을 열어준다.

#### R-09. 마리오 64 -- 그림 속 세계

| 항목 | 내용 |
|:-----|:-----|
| 첫 진입 타이밍 | **약 2분** (피치 성 도착 후 첫 그림) |
| 동기 부여 | "성 안에 잠겨있는 문을 열려면 스타가 필요" -- 목표 명확 |
| 동의 메커니즘 | 그림을 향해 **직접 뛰어드는** 물리적 행위 |
| 빌드업 | 성 밖 잔디밭에서 이동/점프 학습 (30초-1분) -- 성 안에서 "이 그림이 수상하다" 관찰 -- 뛰어들기 |
| 탈출 | 스타 획득 또는 사망으로 즉시 복귀 |
| 어색함? | **없음** -- "그림이 물결친다"는 시각적 단서가 "여기 뭔가 있다"를 전달 |

**핵심 교훈:**
- 2분 만에 차원 전환하지만 어색하지 않은 이유:
  1. **물리적 단서**: 그림이 물결친다 (시각적 affordance)
  2. **능동적 행위**: 뛰어드는 것은 플레이어의 선택
  3. **학습 완료**: 그 전에 이동/점프를 이미 배움
  4. **탈출 보장**: 스타 얻으면 자동 복귀

### 3.4 젤다 시리즈 -- 온보딩 진화의 교과서

> 상세 분석: `Documents/Research/Zelda_Onboarding_Evolution_Research.md`

젤다 시리즈는 25년에 걸쳐 온보딩 설계가 **과잉 교습 -> 환경 교습**으로 진화한 가장 명확한 사례다. 8작의 핵심 타이밍:

| 작품 | 첫 전투 | 첫 에이전시 | 첫 던전 | Tutorial 침습도 |
|:-----|:--------|:-----------|:--------|:---------------|
| Ocarina of Time (1998) | ~8분 | ~2분 | ~8분 | 중간 |
| Majora's Mask (2000) | ~3분 | ~2분 | ~120분 | 중간 |
| Wind Waker (2002) | ~9분 | ~2분 | ~75분 | 중간 |
| Twilight Princess (2006) | ~20분 | ~5분 | ~180분 | **최고** |
| Skyward Sword (2011) | ~60분 | ~3분 | ~120분 | **최고** |
| A Link Between Worlds (2013) | ~3분 | ~10분 | ~15분 | 낮음 |
| Breath of the Wild (2017) | ~3분 | ~1분 | ~60분* | **최저** |
| Tears of the Kingdom (2023) | ~2분 | ~7분 | ~90분* | 낮음 |

> *BotW/TotK의 "첫 던전"은 Great Plateau/Sky Island 탈출 후 접근 가능 시점. 사당을 미니던전으로 보면 ~7-10분.

#### 3단계 진화 모델

**1기: 구조적 교습기 (1998-2002)** -- OoT, MM, WW. NPC가 서사적 맥락 안에서 가르침. 안전 마을 -> 위기 사건 -> 던전 패턴 확립.

**2기: 과잉 교습기 (2006-2011)** -- TP, SS. Wii 리모컨 불안감이 튜토리얼 팽창 유발. TP 3시간, SS 2시간. 미야모토의 TP 오르돈 마을 1일->3일 연장 지시가 역효과의 대표 사례. **ECHORIS의 반면교사.**

**3기: 환경 교습기 (2013-2023)** -- ALBW, BotW, TotK. BotW가 완성형: 환경 자체가 교사, NPC 설명 최소화. **ECHORIS의 직접 레퍼런스.**

#### ECHORIS에 직결되는 핵심 발견

| 발견 | 젤다 근거 | ECHORIS 적용 |
|:-----|:---------|:-------------|
| **환경이 유일한 교사** | BotW Chemistry Engine + 삼각형 법칙 (CEDEC 2017) | 대사 0줄이므로 지형/시각 피드백만 사용 |
| **Gate가 곧 교습** | OoT 미도("검+방패 없으면 불가"), BotW Rune 사당 | ATK Gate 앞 "막힘" = 강화 필요성 자각 |
| **Hand-holding 금지** | TP/SS 반면교사 (3시간/2시간 튜토리얼) | UI 팝업, 동반자 설명, 강제 미니게임 전부 배제 |
| **격리된 교실 -> 월드 응용** | TotK 사당 (Ultrahand/Fuse/Ascend/Recall 각 1사당) | 아이템계 = 격리된 강화 교실, 월드 = 응용 공간 |
| **삼각형 법칙의 2D 변환** | BotW 삼각형 = 시야 차단 + 호기심 유발 | 2D 횡스크롤에서 **수직 단차(Ledge)**가 동일 역할 |
| **Great Plateau 패러다임** | 울타리 공간 -> 능력 학습 -> 자유 | ECHORIS 첫 20 Screen: 발코니 -> Gate -> 모루 -> 돌파 |

**결정적 차이:** BotW는 Great Plateau에서 "능력(What)"을 가르치지만, ECHORIS는 "순환(How) = 막힘->강화->돌파"를 가르친다. 교습 대상이 다르므로 **1시간이 아닌 5분 이내**에 1회전을 완료할 수 있다.

## 4. 패턴 추출 -- 자연스러운 차원 진입의 5조건

레퍼런스 11개 게임에서 추출한 공통 패턴:

### 조건 1: 동기 (Motivation) -- "왜 들어가야 하는가?"

| 패턴 | 예시 | ECHORIS 현재 |
|:-----|:-----|:------------|
| 생존 동기 | Dead Cells: 탈옥 | 없음 |
| 성장 동기 | 디스가이아: "이거 강화 안 하면 진행 불가" | 있으나 **전달 안 됨** |
| 호기심 동기 | 마리오 64: "이 그림 안에 뭐가 있지?" | 있으나 **단서 없음** |
| 구조 동기 | 포켓몬 MD: "도와달라고 한다" | 해당 없음 |

**진단:** ECHORIS는 성장 동기(무기 강화)가 존재하지만, 1분 내에는 "강화가 필요하다"는 인식이 형성되기 전이다.

**동기 프레이밍 5유형 (Motivation Framing Taxonomy):**

| 유형 | 프레이밍 | 사례 | ECHORIS 적합도 |
|:-----|:---------|:-----|:-------------|
| A. 명시적 목표 | "검을 강화해라" | 디스가이아 Ch.3 강제 | 중 -- 대사 0줄과 충돌 |
| B. 호기심 | "안에 뭔가 있다..." | HK Dream Nail, Inscryption | **상** -- 대사 없이 시각으로 전달 가능 |
| C. 장애물 | "이 문은 ATK 50 이상만 통과" | 디스가이아, 포켓몬 레벨 벽 | **상** -- 스탯 게이트와 정합 |
| D. 사회적/윤리적 | "이 아이를 구해야 한다" | 포켓몬 MD, 페르소나 5 | 하 -- 서사 준비 시간 필요 |
| E. 존재론적 | "너는 이미 안에 있다" | Hades, Dead Cells | 중 -- 2-Space 모델과 충돌 |

**ECHORIS 최적 조합: B(호기심) + C(장애물) 하이브리드**
- 무기 맥동 = 호기심 ("안에 뭔가 있다")
- 스탯 게이트 = 장애물 ("강화해야 통과 가능")
- 두 동기가 동시에 작동하면 강력한 복합 견인력

**젤다 교차 검증:**
- OoT 코키리 마을 = C 장애물 (미도의 "검+방패 Gate"). 플레이어가 **자연스럽게** 탐색하여 아이템을 찾음
- BotW Great Plateau = B 호기심 (삼각형 법칙으로 "저 너머에 뭐가 있지?") + C 장애물 (추위/비/절벽이 능력 학습 강제)
- ALBW = B 호기심 (벽화 능력 체험) -- 유가에게 패배하여 **직접 벽화가 됨** = 능력을 체험으로 교습
- TP/SS = A 명시적 목표의 과잉 적용 사례. "Fi가 모든 것을 말해줌" -> 역효과

### 조건 2: 동의 (Consent) -- "내가 선택해서 들어간다"

| 패턴 | 예시 | ECHORIS 현재 |
|:-----|:-----|:------------|
| 물리적 행위 | 마리오 64: 뛰어들기 / HK: 베기 | **없음** -- 자동 텔레포트 |
| UI 확인 | 디스가이아: 아이템 선택 -- 확인 | **없음** |
| 문 열기 | Hades: 첫 문 열기 | **없음** |
| 의식적 행위(Ritual) | 사이코너츠: Psycho-Portal 설치 / Dark Souls: 안개 벽 통과 | **없음** |

**진단:** 현재 빌드는 특정 조건 충족 시 자동으로 아이템계에 전송된다. 플레이어의 **명시적 동의 행위가 없다**.

**젤다 교차 검증:**
- 마리오 64 = 그림을 향해 **뛰어드는** 물리적 행위. BotW 사당 = 조사석에 **시카 슬레이트를 갖다 대는** 의식적 행위
- Hades/Dead Cells/BotW 모두 "플레이어가 무언가를 **직접 한다**"는 공통점. 자동 전송은 어디에도 없음
- ECHORIS Sacred Pickup: 앵빌에 무기를 놓는 행위 = 마리오의 그림 뛰어들기에 대응하는 의식

**의식(Ritual) 행위의 힘:** 게임 디자인에서 "의식"은 특정 순서로 수행하는 특정 행동이 특정 결과를 낳는 패턴이다. 의식적 행위가 강력한 이유는 **플레이어의 사고방식을 전환**시키기 때문이다. ECHORIS에서 "대장간 앵빌에 무기를 올려놓는 행위"는 단조의 판타지와 차원 진입을 하나로 묶는 의식이 될 수 있다.

**리미널 공간(Liminal Space)의 역할:** 라틴어 "limen"(문턱)에서 유래. 두 상태 사이의 전환 공간이 존재하면 플레이어에게 심리적 준비 시간을 제공한다.
- Mario 64: 성 로비 -> 방 -> 그림 앞 (3단계 접근)
- Persona 5: 현실 -> 앱 발동 -> 왜곡 시작 -> 팰리스
- ECHORIS: 세이브 포인트 -> 대장간 UI -> 무기 선택 -> DIVE 프롬프트 -> 진입 (4단계)

### 조건 3: 예고 (Preview) -- "안에 뭐가 있는지 안다"

| 패턴 | 예시 | ECHORIS 현재 |
|:-----|:-----|:------------|
| 시각적 미리보기 | 마리오 64: 그림이 목적지를 보여줌 | **없음** |
| 텍스트 정보 | 디스가이아: "30층, 적 Lv.X" 표시 | **없음** |
| 환경적 단서 | HK: 꿈 대사가 미리 표시 | **없음** |

**진단:** 플레이어가 "들어가면 뭐가 나올지" 전혀 모른다. 미지의 장소에 설명 없이 보내진다.

**젤다 교차 검증:**
- 마리오 64 = 그림이 목적지의 **풍경을 보여줌** (시각적 예고의 교과서)
- BotW = 절벽 위 **하이럴 파노라마**가 세계 전체를 1초에 예고. 사당 입구에서 내부 조명이 새어나옴
- TotK = 각 사당이 능력 이름(Ultrahand, Fuse)을 입구에 표시
- ECHORIS Sacred Pickup: Lore Popup의 "Memory Strata: 2 Floors" = 마리오 그림에 대응하는 텍스트 예고

### 조건 4: 안전망 (Safety Net) -- "돌아올 수 있다"

| 패턴 | 예시 | ECHORIS 현재 |
|:-----|:-----|:------------|
| 자동 복귀 | 마리오 64: 스타 획득 시 복귀 | 존재 (층 클리어 시) |
| 탈출 아이템 | 디스가이아: Mr. Gency's Exit | 존재 (미구현) |
| 메뉴 탈출 | 사이코너츠: 언제든 탈출 | **미전달** |

**진단:** 탈출 메커니즘은 존재하지만 "돌아올 수 있다"는 정보가 진입 전에 전달되지 않는다.

### 조건 5: 학습 완료 (Readiness) -- "기본은 알고 들어간다"

| 패턴 | 예시 | ECHORIS 현재 |
|:-----|:-----|:------------|
| 기본 전투 학습 | 마리오 64: 성 밖에서 이동/점프 연습 | 1분으로는 **불충분** |
| 시스템 이해 | 디스가이아: 3챕터 전투 경험 | 1분으로는 **불충분** |
| 최소 학습 | Dead Cells: 첫 방에서 이동+공격 학습 | 가능 (30초) |

**진단:** 1분은 이동+공격 학습에 충분하지만, "장비가 중요하다"까지 학습하기엔 부족하다.

### 5조건 달성 현황

| 조건 | 마리오 64 | Hades | Dead Cells | 디스가이아 | BotW | TP/SS | **ECHORIS** |
|:-----|:---------|:------|:----------|:----------|:-----|:------|:-----------|
| 동기 | O | O | O | O | O (삼각형) | O (NPC) | **X** |
| 동의 | O | O | -- | O | O (사당 조사) | O | **X** |
| 예고 | O | 부분 | -- | O | O (파노라마) | O (과잉) | **X** |
| 안전망 | O | O | O | O | O (자유 사망) | O | **X (미전달)** |
| 학습 | O | O | O | O | O (환경) | O (과잉) | **부분** |
| **합계** | **5/5** | **4.5/5** | **3/5** | **5/5** | **5/5** | **5/5** | **0.5/5** |

ECHORIS는 5조건 중 **0.5개만 충족**. 이것이 납치감의 원인이다.

**젤다 시리즈의 교훈:** BotW와 TP/SS 모두 5/5를 달성하지만, 방법이 정반대다.
- **BotW** = 환경이 5조건을 **암묵적으로** 충족. 삼각형 법칙(동기), 시카 슬레이트(동의), 파노라마(예고), 자유 사망(안전망), Chemistry Engine(학습)
- **TP/SS** = NPC/UI가 5조건을 **명시적으로** 충족. 3시간에 걸쳐 하나하나 설명 -> 플레이어 이탈
- **ECHORIS의 선택:** BotW 방식. 대사 0줄이므로 환경/시각 피드백으로 5조건 압축 충족

## 5. 근본 원인 재진단

### 5.1 "시간"이 문제인가? (가설 A 검증)

| 증거 | 판정 |
|:-----|:-----|
| Dead Cells는 0초에 절차적 레벨에 진입하지만 어색하지 않다 | 시간이 본질이 아님 |
| Hades는 15초에 첫 런을 시작하지만 어색하지 않다 | 시간이 본질이 아님 |
| 마리오 64는 2분에 차원 전환하지만 어색하지 않다 | 시간이 본질이 아님 |
| 디스가이아는 3-5시간 후 해금하고 자연스럽다 | 시간을 늘리면 **자연스러워지긴** 함 |

**젤다 추가 증거:**
| 증거 | 판정 |
|:-----|:-----|
| BotW는 ~1분에 첫 에이전시, ~3분에 첫 전투. 어색하지 않다 | 시간이 본질이 아님 |
| ALBW는 15분에 첫 던전이지만, 관습(아이템 대여) 전복으로 자유도 극대화 | **구조 혁신**이 시간을 압축 |
| TP는 180분에 첫 던전. 시간이 길어도 **과잉 설명**은 부자연스러움 유발 | 시간을 늘려도 방법이 틀리면 실패 |
| SS는 Fi의 개입 때문에 2시간이 "지루하다" 평가 | 시간 + 과잉 설명 = 최악 |

**판정:** 시간은 **필요충분조건이 아니다**. 시간을 늘리면 자연스러워지는 것은 사실이지만, 그것은 시간이 지나면서 5조건이 자연스럽게 충족되기 때문이다. 5조건을 충족하면 1분 내 진입도 자연스러울 수 있다. 젤다 시리즈는 이를 25년에 걸쳐 증명했다: **TP/SS의 과잉(시간은 길지만 방법이 잘못됨) -> BotW의 압축(시간은 짧지만 구조가 올바름)**.

### 5.2 "구조"가 문제인가? (가설 B 검증)

ECHORIS의 현재 구조적 문제:

```
[스폰] -- 이동 -- [적 1-2체 처치] -- [아이템 획득] -- [자동 진입???]
                                                        ^
                                                   여기서 납치감
```

이 플로우에서 결여된 것:
1. 아이템 획득 전에 "이 아이템에 들어갈 수 있다"는 **개념 소개** 자체가 없음
2. 아이템 획득 후 "들어갈까?" 선택이 없음
3. "들어가면 뭐가 있는지" 미리보기가 없음
4. "돌아올 수 있다"는 보장이 없음

**판정: 가설 B가 정답.** 구조적으로 5조건 중 어느 것도 충족하지 않는다.

### 5.3 ECHORIS 특수 상황: 2-Space 모델의 딜레마

ECHORIS는 다른 게임과 구조적으로 다르다:

| 게임 유형 | 공간 구조 | 차원 전환 | 어색함 리스크 |
|:----------|:---------|:---------|:------------|
| 순수 로그라이크 (Dead Cells, Hades) | 1-Space (던전만) | 없음 | 낮음 |
| 순수 메트로베니아 (HK) | 1-Space (월드만) | 매우 늦게 | 낮음 |
| **ECHORIS** | **2-Space (월드 + 아이템계)** | **매우 빨리** | **높음** |
| 디스가이아 | 2-Space (허브 + 아이템계) | 늦게 (Ch.3) | 낮음 |

**ECHORIS의 딜레마:** 아이템계가 스파이크 피처이므로 **빨리 보여주고 싶지만**, 2-Space 전환은 원래 **시간이 걸려야 자연스러운** 패턴이다.

**해결 방향:** Dead Cells/Hades처럼 "이미 안에 있다" 패턴을 차용하거나, 마리오 64처럼 "빠르지만 5조건을 압축 충족"하는 패턴이 필요하다.

## 7. 최종 권장안: 안 G "Sacred Pickup" 패턴

### 7.0 설계 철학

기존 안 A-F의 문제점: 아이템 획득을 **평범한 이벤트**로 취급한 뒤, 앵빌에서 별도로 다이브를 유도한다. 이 구조에서는 "획득 -> 앵빌 이동"이 분리되어 있어 연결감이 약하다.

**안 G의 핵심 전환:**
1. **모든 아이템 획득 자체가 특별하다** -- 바닥에 배치된 아이템은 레어리티 색상 이펙트를 가짐
2. **획득 팝업이 아이템의 정체성을 보여준다** -- Lore = Preview, 무기의 기억이 곧 다이브의 예고
3. **획득 직후 무기 맥동 컷신이 앵빌과의 선 연결을 만든다** -- "이 무기를 저기서 단조하라"는 공간적 유도
4. **앵빌 접근이 자연스러운 의식이 된다** -- 획득부터 앵빌까지 하나의 흐름

이 설계는 **첫 아이템뿐 아니라 게임 전체의 모든 아이템 획득**에 동일하게 적용되므로, 시스템으로서 확장 가능하다.

### 7.2 5조건 충족 검증

### 7.4 "Broken Blade" 시작 무기 사양

| 요소 | 사양 |
|:-----|:-----|
| 이름 | Broken Blade |
| ATK | 3 (Normal Blade의 1/4) |
| 공격 패턴 | 1타 약공격 (콤보 없음) |
| 시각 | 부러진 검 스프라이트 (날 절반 없음) |
| 아이템계 | **없음** -- 부러진 무기에는 Memory Strata가 존재하지 않음 |
| Lore Popup | 게임 시작 시 표시하지 않음 (이미 가지고 시작하므로) |
| 서사적 의미 | 에르다가 The Shaft에서 주운 잔해. 약하지만 싸울 수는 있다. 첫 온전한 무기를 만나면 "이것과는 다르다"는 대비 효과 |

**Broken Blade -> Normal Blade 대비 효과:**
- Broken Blade = glow 없음, Lore 없음, 아이템계 없음, 약함
- Normal Blade = glow 있음, Lore 있음, 아이템계 있음, 강함
- 이 대비가 "온전한 아이템은 특별하다"를 **체험으로** 전달

### 7.8 1회성 vs 다회성 분류

#### 1회성 (Tutorial / First-Time Only)

게임 플레이 중 **단 한 번만** 발생하는 연출. 온보딩 목적으로만 존재하며, 이후에는 재생되지 않는다.

| # | 요소 | 발생 시점 | 목적 | 재발생 조건 |
|:--|:-----|:---------|:-----|:-----------|
| T1 | **Broken Blade 시작** | 게임 최초 스폰 | "부족함" 인식. 온전한 아이템과의 대비 | 없음 (뉴 게임 시에만) |
| T2 | **첫 온전한 무기의 맥동 컷신 (카메라 줌인)** | 첫 Normal 아이템 [X] 닫기 직후 | "이 무기는 특별하다"는 강한 신호 | 없음. 2번째 아이템부터는 맥동만 (줌인 없음) |
| T3 | **앵빌 Tether (빛의 선)** | 첫 온전한 무기 획득 직후 | 앵빌의 존재와 위치를 공간적으로 교수 | 없음. 이후 획득에서는 Tether 생략 |
| T4 | **앵빌 Tether 점선 잔상 유지** | T3 이후 ~ 앵빌 도달까지 | 앵빌로의 이동 유도 (길찾기) | 없음 |
| T5 | **첫 DIVE 시 풀 프리뷰 패널** | 첫 앵빌 DIVE 선택 시 | Memory Strata 개념, 층 수, 적 레벨을 최초 소개 | 없음. 2회차부터 요약 프리뷰 |

#### 다회성 (System / Every-Time)

게임 플레이 전반에 걸쳐 **매번** 발생하는 시스템 연출. Sacred Pickup의 핵심 체계.

| # | 요소 | 발생 조건 | 목적 | 빈도 |
|:--|:-----|:---------|:-----|:-----|
| S1 | **바닥 아이템 glow + 파티클** | 아이템이 월드에 존재하는 동안 | 아이템 발견 유도. 레어리티 시각 구분 | 항상. 모든 배치/드롭 아이템 |
| S2 | **자동 획득 (접근 = 획득)** | 플레이어가 아이템 히트박스 접촉 | 획득의 즉시성. 흐름 유지 | 항상. 모든 아이템 |
| S3 | **Lore Popup** | 아이템 획득 시 | 아이템 정체성 + 아이템계 Preview | 해당 아이템 **첫 획득** 시마다 (동일 아이템 재획득 시 생략. 옵션으로 항상 표시 가능) |
| S4 | **무기 맥동 (glow pulse)** | Lore Popup [X] 닫기 직후 | "이 안에 뭔가 있다" 반복 각인 | 매 아이템 획득. 단, 줌인 컷신은 첫 회만(T2). 이후는 제자리 맥동만 |
| S5 | **앵빌 [E] 프롬프트** | 앵빌 48px 이내 접근 | 앵빌 인터랙션 유도 | 항상. 모든 앵빌 |
| S6 | **대장간 UI + DIVE/BACK 선택** | 앵빌 [E] 입력 | 명시적 동의 + 프리뷰 | 항상. 모든 앵빌 인터랙션 |
| S7 | **다이브 연출 (리플 + 아이리스 + 낙하)** | DIVE [E] 입력 | 차원 전환 연출 | 항상. 단, 8절의 단축 규칙 적용 (풀 -> 단축 -> 초단축) |
| S8 | **레어리티별 glow/파티클 강도 차이** | 아이템 존재 시 | Normal부터 Ancient까지 시각적 위계 | 항상. 레어리티 체계의 일부 |

```
  [Return 아이콘 표시] ─── T6 1회성  ←── 첫 진입만
```

#### 설계 원칙

- **1회성(T)은 교수(Teaching)를 위해 존재한다.** 플레이어가 한 번 이해하면 반복할 필요 없는 것들.
- **다회성(S)은 시스템 정체성을 위해 존재한다.** "아이템 획득이 특별하다"는 느낌은 100번째 아이템에서도 유지되어야 한다.
- **1회성에서 다회성으로의 전환이 자연스러워야 한다.** 첫 무기에서 줌인+Tether를 보여준 후, 두 번째 무기에서 줌인과 Tether가 없어도 "아, 아까처럼 앵빌에 가면 되겠지"라고 플레이어가 스스로 연결할 수 있어야 한다.
