## 0. 리서치 목적

Project Abyss의 월드는 "수직 거대 구조물(Vertical Megastructure)" 형태의 핸드크래프트 매크로 구조를 기반으로 한다. 이 문서는 레퍼런스 게임 8종의 매크로 구조 패턴을 분석하여, 수직 구조에 적용 가능한 설계 원칙과 패턴을 도출한다.

**분석 대상:**
- Castlevania: Symphony of the Night (1997) — SotN
- Hollow Knight (2017) — HK
- Super Metroid (1994) — SM
- Metroid Dread (2021) — MD
- Ori and the Blind Forest (2015) / Will of the Wisps (2020) — Ori
- Dead Cells (2018) — DC
- Blasphemous (2019) — Blas
- Salt and Sanctuary (2016) — SaS

## 1. 월드 구조 유형론 (World Structure Typology)

### 1-1. 주요 구조 패턴

메트로베니아 게임의 월드 매크로 구조는 세 가지 원형 패턴으로 분류된다.

| 패턴 | 설명 | 대표 게임 | 특성 |
| :--- | :--- | :--- | :--- |
| **허브-스포크 (Hub-and-Spoke)** | 중앙 허브에서 여러 구역이 방사형으로 뻗어나가는 구조 | Super Metroid, SotN, Hollow Knight | 항상 돌아올 중심이 있어 방향 감각 유지. 고전적 메트로베니아에 가장 많이 사용 |
| **연결망 (Web / Open Network)** | 구역들이 서로 다중 경로로 촘촘하게 연결된 구조 | Hollow Knight (심화), Blasphemous | 비선형성 극대화. 길 잃기 쉬우나 발견의 쾌감이 크다 |
| **선형 분기 (Linear with Branches)** | 메인 경로는 선형이나 좌우/상하로 선택적 구역이 붙어있는 구조 | Metroid Dread, Ori, Dead Cells | 내러티브 제어 용이. 페이싱 설계가 쉽다 |

**수직 구조에 대한 시사점:** Project Abyss의 "층위(Stratum)" 개념은 선형 분기 패턴의 수직 버전에 가장 가깝다. 각 층위를 노드로 보면 위아래로 연결된 선형 구조이며, 각 층위 내에서는 좌우 분기가 발생한다. 단, 수직 연결부(엘리베이터, 수직 통로)가 허브-스포크의 중심 역할을 대신한다.

### 1-2. 게임별 구역 수와 규모

| 게임 | 구역(Zone) 수 | 연결 방식 | 월드 방향성 |
| :--- | :---: | :--- | :--- |
| SotN | ~10구역 (역전 성 포함 2배) | 복도식 연결, 텔레포트 룸 | 수평 중심, 수직 혼합 |
| Hollow Knight | ~17구역 | 다중 경로 연결망 | 수직 중심 (위에서 아래로 깊어짐) |
| Super Metroid | ~6구역 | 허브-스포크 (수직 샤프트 중심) | 수직 샤프트 + 수평 탐험 |
| Metroid Dread | 9구역 (행성 ZDR) | 구역 내 선형, 구역 간 텔레포트 | 수직 하강 중심 |
| Ori WotW | ~11구역 | 중앙 글레이드 허브 방사형 | 수평 중심 |
| Dead Cells | ~30단계 (런 기반) | 단방향 선형, 분기 경로 선택 | 수직 하강 (탑에서 아래로) |
| Blasphemous | ~18구역 | 연결망, 수직 레이어 | 수직 중심 (지상에서 위아래) |
| SaS | ~20구역 | 수직 레이어 연결 | 수직 강하 (솔스라이크 구조) |

**핵심 인사이트:** Hollow Knight와 Blasphemous처럼 **수직 중심 구조**를 채택한 게임들은 세계가 "깊어질수록 위험해진다"는 직관적 난이도 감각을 자연스럽게 제공한다. 수직 메가스트럭처인 Project Abyss에 직접 적용 가능한 원칙이다.

### 1-3. "연결된 세계" vs "분리된 레벨"의 차이

연결된 세계감을 만드는 요소:

1. **시각적 연속성:** 두 구역이 만나는 경계에서 한 구역의 아트 언어가 서서히 다른 구역의 아트 언어로 전환된다. SotN의 Marble Gallery와 Colosseum 경계, HK의 Fungal Wastes와 City of Tears 경계가 좋은 예시다.
2. **청각적 전환:** 배경음이 구역 경계에서 즉시 바뀌지 않고 자연스럽게 크로스페이드된다.
3. **지형적 참조:** 한 구역에서 다른 구역의 지형지물이 배경에 보인다. (예: HK의 Ancient Basin에서 City of Tears를 위로 볼 수 있음)
4. **일관된 물리 법칙:** 모든 구역에서 같은 이동 규칙이 적용된다.
5. **이전 구역으로의 귀환 경로:** 단방향이 아닌 양방향 연결이 세계의 물리적 일관성을 강화한다.

## 2. 크리티컬 패스 vs 선택 구역

### 2-1. 필수/선택 구역 비율

| 게임 | 필수 구역 비율 (추정) | 선택 구역의 역할 |
| :--- | :---: | :--- |
| Super Metroid | ~40% 필수 | 아이템 수집, 선택적 능력 업그레이드, 숨겨진 경로 |
| Hollow Knight | ~30% 필수 | 매력적인 사이드 퀘스트, 숨겨진 보스, 독특한 서사 |
| Metroid Dread | ~60% 필수 | EMMI 구역, 파워업, 선택적 탱크 업그레이드 |
| SotN | ~50% 필수 | 숨겨진 룸, 장비, 선택적 보스 |
| Ori WotW | ~55% 필수 | 수집 요소, 조각상, 정령 빛 |
| Blasphemous | ~45% 필수 | 사이드 퀘스트, 유물, 기도 |

**시사점:** 건강한 메트로베니아는 필수 경로가 전체의 30~60% 수준이다. 나머지는 선택적이지만 플레이어가 탐색하고 싶게 만드는 보상이 배치된다. 필수 비율이 너무 높으면 선형 게임이 되고, 너무 낮으면 길을 잃는 느낌이 강해진다.

### 2-2. 지표(Marker) 없이 크리티컬 패스로 유도하는 기법

메트로베니아는 UI 마커 없이도 플레이어를 올바른 방향으로 유도한다. 사용되는 기법들:

**환경 언어 (Environmental Language):**
- 빛의 방향: 밝은 곳이 목표 방향이다. (HK의 City of Tears는 빛이 새어 나오는 방향으로 유도)
- 실루엣 배경: 목표 구조물이 배경에 보여 방향 감각을 제공한다.
- NPC/적 밀도: 크리티컬 패스에는 적이 더 촘촘하게 배치되어 "이쪽이 진행 방향"을 암시한다.

**게이트 배치의 역할:**
- 잠긴 문은 "열쇠를 찾아라"는 방향 지시다. 플레이어는 자연스럽게 다른 경로를 탐색하게 된다.
- Super Metroid의 "봄 공" 블록은 아직 배우지 않은 기술을 암시하는 시각적 힌트다.

**지형적 접근성:**
- 초반에는 직관적으로 이동 가능한 방향이 크리티컬 패스다.
- 선택 구역은 약간의 기술적 장벽(약간 높은 점프, 경사진 지형)을 암시로 사용한다.

**Dead Cells의 특수 방식:**
- 선형 런 구조에서 분기 경로는 "다른 레벨로의 탈출구"로 명확히 표시된다. 선택 구역은 "도전적인 대안 경로"로 기능한다.

### 2-3. 백트래킹을 보람 있게 만드는 설계

백트래킹은 메트로베니아의 핵심 메커닉이지만, 잘못 설계하면 지루함의 주범이 된다.

**효과적인 백트래킹 설계 원칙:**

1. **새로운 능력 = 새로운 시선:** 이전에 지나친 막힌 경로가 새 능력으로 열릴 때, 플레이어는 같은 구역을 완전히 새로운 눈으로 본다. HK의 Forgotten Crossroads가 Infected Crossroads로 변화하는 것은 단순 재방문을 감정적으로 다른 경험으로 만드는 극단적 예시다.

2. **빠른 이동 경로 확보:** 재방문 시 지루한 구역을 통과해야 한다면 지름길이나 패스트 트래블이 필수다. SotN의 텔레포트 룸, HK의 스태그 역, SM의 엘리베이터가 이 역할을 한다.

3. **발견의 시간차:** 특정 아이템/비밀은 처음에는 눈에 보이지만 아직 획득 불가능하게 설계한다. 나중에 능력을 얻어 돌아왔을 때 "그게 여기 있었구나!" 하는 보람을 제공한다.

4. **구역 변화:** 단순히 같은 구역을 다시 통과하는 것이 아니라 시간 경과, 서사 변화 등으로 구역 자체가 달라진 것이 있어야 한다.

5. **밀도 있는 연결:** 복귀 경로는 처음 진입 경로와 다른 쪽으로 연결되어, 새로운 부분을 발견하면서 복귀할 수 있어야 한다.

## 3. 구역 게이팅 (Zone Gating)

### 3-1. 능력 게이트 vs 열쇠 게이트 vs 스탯 게이트

| 게이트 유형 | 메커닉 | 장점 | 단점 | 주요 사용 게임 |
| :--- | :--- | :--- | :--- | :--- |
| **능력 게이트 (Ability Gate)** | 특정 이동/전투 능력 보유 시 통과 | 강한 성장감, 직관적 "열쇠-자물쇠" | 능력 획득 순서가 고정됨 | SM, HK, MD, Ori |
| **열쇠 게이트 (Key Gate)** | 특정 아이템/열쇠 보유 시 통과 | 유연한 탐험 순서 | 열쇠 없이 막히면 답답함 | SotN, Blas |
| **스탯 게이트 (Stat Gate)** | 특정 수치 (ATK, INT 등) 달성 시 통과 | 성장 동기 부여, 야리코미 촉진 | 그라인드 강요 느낌 가능 | Blas (방어력), SaS |
| **내러티브 게이트 (Narrative Gate)** | 특정 NPC 대화/이벤트 완료 시 통과 | 서사 통합 | 기억 의존, 무엇을 해야 할지 불명확 | HK, Blas |
| **소프트 게이트 (Soft Gate)** | 기술적으로 가능하나 의도된 시점 전에는 매우 어려움 | 숙련 플레이어에게 보상, 탐험감 | 초보자 혼란, 의도치 않은 루트 가능 | SM (Sequence Breaking) |

**소프트 게이트 vs 하드 게이트:**

- **하드 게이트:** 능력 없이는 물리적으로 불가능. (예: 이중 점프 없이는 닿을 수 없는 높이) 안정적이나 탐험 자유도를 제한.
- **소프트 게이트:** 기술적으로 가능하나 극히 어렵게 설계. (예: SM의 특정 섹션은 특정 아이템 없이도 정밀한 조작으로 통과 가능) 고숙련 플레이어에게 보상을 주고 발견의 쾌감을 극대화하나, 의도치 않은 순서 파괴(Sequence Breaking)가 발생할 수 있다.

**Sirlin의 관점 적용:** 소프트 게이트를 통한 Sequence Breaking은 게임 숙달(Mastery)의 표현이므로 원칙적으로 허용해야 한다. 단, 이것이 전체 서사/페이싱을 완전히 무너뜨리는 경우(Degenerate Strategy)는 하드 게이트로 보완해야 한다.

### 3-2. 게임별 능력 게이트 수

| 게임 | 주요 이동 능력 게이트 수 | 획득 순서 | 비고 |
| :--- | :---: | :--- | :--- |
| Super Metroid | 8~10개 | 비선형 (일부 선택 가능) | 봄 공, 스크류 어택, 스페이스 점프 등 |
| Hollow Knight | 6~8개 | 비선형 | 만티스 발톱, 모스윙 망토, 수정 대시 등 |
| Metroid Dread | 10~12개 | 주로 선형 | EMMI 처치 순서가 능력 획득 순서 |
| Ori WotW | 8~10개 | 비선형 | 각 사당에서 획득하는 능력 |
| SotN | 5~7개 | 비선형 | 변신, 부유 등 주로 아이템 기반 |

**시사점:** 주요 게이트 능력은 6~12개 범위가 적절하다. 너무 적으면 탐험의 진전감이 희박해지고, 너무 많으면 관리 부담이 커진다.

### 3-3. 이중 게이트 (Dual Gate) 패턴

Project Abyss의 핵심 설계 요소인 "능력 게이트 + 스탯 게이트" 이중 구조의 레퍼런스:

**Blasphemous의 조합 게이팅:**
Blasphemous는 특정 구역에 진입하기 위해 특정 기도(Prayer, 아이템 기반)와 충분한 죄 획득(스탯 기반)을 모두 요구하는 구역이 존재한다. 이는 이중 게이트의 좋은 레퍼런스다.

**Salt and Sanctuary의 스탯 게이트:**
솔스라이크 구조를 채택한 SaS는 특정 무기 사용을 위해 능력치(STR, DEX 등)가 필요하다. 스탯이 부족한 플레이어는 우회 경로를 탐색하거나 레벨을 올려야 한다. 이것은 스탯이 능력 자체가 아닌 장비 사용을 통해 구역을 간접 게이팅하는 방식이다.

## 4. 지름길 설계 (Shortcut Design)

### 4-1. 지름길 유형 분류

| 유형 | 설명 | 대표 예시 | Project Abyss 적용성 |
| :--- | :--- | :--- | :--- |
| **일방향 지름길** | 처음에는 한 방향으로만 이동 가능, 이후 양방향 개방 | Dark Souls 엘리베이터, HK 탈것 | 높음 — 층위 간 하강은 쉽고 상승은 능력 필요 |
| **개방형 패스트 트래블** | 특정 지점 발견 후 즉시 이동 가능한 포탈 | SotN 텔레포트 룸, MD 텔레포트 | 높음 — 세이브 포인트 간 패스트 트래블 |
| **루프백 경로** | 두 구역을 연결하는 숨겨진 단방향 통로 | SM 일부 구간, HK 숏컷 | 높음 — 탐험 보상으로 설계 |
| **능력 기반 지름길** | 새 능력 획득 후 이전에 막혔던 경로가 지름길이 됨 | Ori의 새 능력으로 열리는 경로 | 매우 높음 — 능력 게이트 설계의 핵심 |
| **NPC 개방 지름길** | 특정 NPC와 상호작용 후 지름길 개방 | HK의 엘리베이터 수리 | 중간 — 내러티브 통합 필요 |

### 4-2. 어두운 영혼(Dark Souls) 방식 엘리베이터 원칙

Dark Souls의 엘리베이터 지름길 설계는 메트로베니아에도 강력히 적용된다:

1. **목적지 먼저, 경로 나중:** 플레이어는 먼저 힘겨운 경로로 목적지에 도달한다. 그 후 지름길을 발견하여 돌아오는 여정을 단축한다.
2. **지름길 발견의 "아하!" 순간:** 지름길의 반대쪽 끝이 이미 지나온 구역에 연결될 때, 세계의 연결성에 대한 경이감을 제공한다.
3. **되돌아갈 이유:** 지름길은 단순히 편의만 제공하는 것이 아니라, 이전 구역을 재방문할 동기를 만든다.

### 4-3. 패스트 트래블 배치 원칙

| 원칙 | 근거 | 구현 방법 |
| :--- | :--- | :--- |
| 구역당 최소 1개 | 패스트 트래블 거점이 너무 드물면 이동 피로가 누적된다 | 각 층위(Stratum)에 1~2개의 세이브 포인트=패스트 트래블 거점 |
| 보스 도전 전 배치 | 보스 전 패스트 트래블 거점은 반복 도전의 마찰을 줄인다 | 각 층위 보스 방 직전에 세이브 포인트 배치 |
| 핵심 분기점 배치 | 여러 경로가 갈라지는 지점에 패스트 트래블 제공 | 층위 간 연결 통로에 세이브 포인트 |
| 후반부 개방 | 초반에는 패스트 트래블을 제한하여 세계 공간감을 강화한다 | 초기 2~3개 층위는 패스트 트래블 없이 도보 이동 |

## 5. 페이싱 (Pacing)

### 5-1. 구역 내 활동 교대 패턴

Csikszentmihalyi의 플로우 모델에 따르면, 장기간 동일한 강도의 자극은 플레이어를 지루함 또는 불안으로 밀어낸다. 메트로베니아는 구역 내에서 다음 요소를 교대 배치한다:

| 요소 유형 | 역할 | 페이싱 위치 |
| :--- | :--- | :--- |
| **전투 구간** | 긴장감 상승, 자원 소모 | 주요 이동 경로 |
| **플랫폼/이동 구간** | 조작 집중, 긴장 유지 | 전투 구간 후 |
| **탐험/발견 구간** | 호기심 충족, 긴장 해소 | 보상 공간 |
| **휴식 공간 (Breath Room)** | 긴장 완전 해소, 정보 흡수 | 강도 높은 구간 직후 |
| **보스 전 공간** | 긴장 고조, 결전 암시 | 보스 방 직전 |

**"Breath Room" 설계 원칙:**
- 적이 없거나 최소한
- 세이브 포인트, 치유 포인트, 또는 탈출 기회 제공
- 단순한 플랫폼 이동만 요구 (복잡한 퍼즐이나 전투 없음)
- 직전의 도전에 대한 보상 (아이템, 환경적 발견) 배치

### 5-2. 구역 난이도 진행 패턴

모든 레퍼런스 게임이 공통적으로 사용하는 "톱날 곡선(Sawtooth Curve)" 패턴:

```
난이도
  |
  |     /\        /\        /\
  |    /  \      /  \      /  \
  |   /    \    /    \    /    \
  |  /      \  /      \  /      \
  | /        \/        \/        \
  |_________________________ 진행
  [구역A]   [새 구역 진입] [더 깊은 구역]
```

- 각 구역 진입 초반: 상대적으로 쉬운 경험으로 새 환경 학습
- 구역 중반: 점진적 난이도 상승
- 구역 보스: 최고 난이도
- 다음 구역 진입: 난이도 일부 하강 후 새 베이스라인에서 재상승

**중요:** 다음 구역 진입 후의 베이스라인은 이전 구역의 베이스라인보다 높아야 한다. 전반적 상향 추세를 유지해야 성장감이 보존된다.

### 5-3. 수직 구조에서의 난이도 배치

수직 세계에서 "깊어질수록 위험해진다"는 원칙은 플레이어의 직관과 일치한다:

**Hollow Knight의 수직 난이도 원칙:**
- 위(Dirtmouth/King's Pass): 쉬운 입문 구역
- 중간(Forgotten Crossroads, Greenpath): 중간 난이도
- 아래(Ancient Basin, Deepnest, Abyss): 최고 난이도
- 수직 깊이 = 위험도와 보상의 정비례 관계

**Blasphemous의 변주:**
Blasphemous는 순수 수직만이 아니라 수평 레이어를 혼합한다. 그러나 핵심 진행 방향(위 = 신성함, 아래 = 타락)을 유지하여 방향 감각을 제공한다.

**Dead Cells의 수직 구조:**
런 기반이지만 "탑에서 아래로" 내려가는 구조를 사용한다. 각 층이 독립적인 선형 레벨이나, 점점 깊어질수록 적과 함정이 증가한다.

## 6. 수직 메가스트럭처에 대한 적용 패턴 도출

### 6-1. 레퍼런스 게임들로부터의 핵심 원칙 요약

Project Abyss의 "수직 거대 구조물" 월드 설계에 직접 적용할 수 있는 패턴을 도출한다.

**원칙 1: 수직 연결 통로가 허브 역할을 해야 한다**

Super Metroid의 수직 샤프트처럼, 층위와 층위를 연결하는 수직 통로(엘리베이터, 수직 통로, 낙하 구간)가 매크로 구조의 허브 역할을 해야 한다. 플레이어는 항상 "이 통로로 돌아가면 된다"는 기준점을 갖는다.

**원칙 2: 층위(Stratum)는 3~5개의 특색 있는 바이옴으로 구성되어야 한다**

각 층위는 단일 환경이 아니라 소규모 바이옴의 집합이어야 한다. HK가 각 구역을 독특한 시각/청각/기계적 정체성으로 구분하듯, 각 층위는 명확한 환경 언어를 가져야 하며, 층위 경계에서는 시각적 전환이 이루어져야 한다.

**원칙 3: 하강은 쉽고 상승은 능력 또는 스탯이 필요해야 한다**

수직 세계에서 중력을 활용: 새로운 층위로 하강하는 경로를 먼저 발견하지만, 다시 올라오기 위해서는 새로운 능력(이단 점프, 벽 타기)이나 엘리베이터가 필요하도록 설계한다. 이것은 일방향 지름길의 수직 버전이다.

**원칙 4: 층위당 1~2개의 능력 게이트, 0~1개의 스탯 게이트**

층위 진행을 위한 필수 능력 게이트는 1~2개로 제한한다. 스탯 게이트는 선택 구역에 배치하여 야리코미(아이템계 강화)를 유도하되 필수 진행을 막아서는 안 된다. 이 비율은 레퍼런스 분석에서 도출한 6~12개 능력 게이트(7층위 기준 층위당 약 1~1.7개)를 기반으로 한다.

**원칙 5: 층위당 최소 1개의 루프백 연결**

각 층위는 최소 1개의 "나중에 열리는 지름길"을 갖도록 설계한다. 이 지름길은 처음 진입 시에는 한 방향으로만 통과 가능하고(예: 낙하 전용 구간), 새 능력 획득 후 양방향이 된다. Dark Souls 엘리베이터 원칙의 직접 적용이다.

**원칙 6: 세이브 포인트를 페이싱 앵커로 사용**

세이브 포인트의 배치가 페이싱을 직접 통제한다. 보스 전 반드시 배치하고, 고난이도 전투 구간 후에 배치하여 "여기까지 왔다"는 안도감을 제공한다. HK의 벤치, SotN의 세이브 룸이 이 역할을 한다.

**원칙 7: 수직 방향 = 서사적 의미 부여**

BLAME!의 메가스트럭처처럼, 수직 방향이 서사적 의미를 가져야 한다. "위 = 외부 세계/시작점", "아래 = 심연/진실"의 구도는 수직 탐험에 서사적 무게를 더한다. Project Abyss에서 "심연"이라는 이름 자체가 이 원칙을 지지한다.

### 6-2. 구역 연결 패턴 (Project Abyss 적용)

```
[외부 세계 / 입구]
        |
   [층위 1: 초입 구역]
   (능력 게이트 없음, 쉬운 스탯 게이트)
   /          \
[선택 구역 A]  [크리티컬 패스]
               |
        [층위 2: 중간 구역]
        (능력 게이트 1개, 스탯 게이트 1개 선택)
        /           \
[선택 구역 B]   [크리티컬 패스]
                    |
             [층위 3: 심층 구역]
             (능력 게이트 1~2개, 스탯 게이트 1~2개 선택)
             ←→ [아이템계 진입 포인트들]
```

**세이브 포인트 배치 위치:**
- 각 층위 진입점
- 층위 내 주요 분기점
- 보스 방 직전
- 고난이도 플랫폼 구간 직후

### 6-3. 각 게임에서 Project Abyss로의 직접 적용 사례

| 레퍼런스 | 적용 패턴 | Project Abyss에서의 구현 |
| :--- | :--- | :--- |
| Super Metroid — 수직 샤프트 허브 | 중앙 수직 통로가 모든 층위를 연결하는 기준점 | 각 층위를 연결하는 "대심연 수직통로" (중앙 수직 구조) |
| HK — 구역 간 다중 연결 | 각 구역이 최소 2개의 다른 구역과 연결 | 각 층위는 위아래 층위 외에 최소 1개의 수평 연결통로 보유 |
| SotN — 역전 성 | 후반부에 세계의 다른 시점을 제공하는 반전 구조 | 특정 능력 획득 후 이미 탐험한 층위를 다른 방향으로 재탐험 |
| Dark Souls — 엘리베이터 지름길 | 어렵게 도달한 후 발견하는 지름길의 "아하!" | 각 층위에 한 방향 낙하 통로, 이후 엘리베이터/능력으로 역방향 개방 |
| MD — EMMI 구역 | 특정 위협으로 구분되는 위험 구역 | 층위 내 고위험 구간 (특정 적/환경 위험이 지배하는 서브구역) |
| Blas — 이중 게이팅 | 능력 + 스탯 동시 요구 구역 | 스탯 게이트가 능력 게이트와 조합되는 고난이도 선택 구역 |
| SM — 소프트 게이트 | 기술적으로 조기 진입 가능한 숨겨진 경로 | 숙련 플레이어를 위한 Sequence Break 허용 (크리티컬 패스 파괴 없는 범위) |
| Dead Cells — 런별 분기 선택 | 매 세션마다 다른 경로를 선택하는 분기 | 층위 내 마이크로 구조의 절차적 변형으로 재방문 가치 유지 |

## 7. 저주받은 문제 (Cursed Problems) — 수직 구조 특유의 충돌

### 7-1. 수직 구조가 만들어내는 고유한 긴장

| 문제 | 충돌하는 가치 | 해결 방향 | 희생 |
| :--- | :--- | :--- | :--- |
| 하강은 쉽고 귀환은 어렵다 | 탐험 자유도 vs 귀환 마찰 | 세이브 포인트/패스트 트래블로 귀환 마찰 최소화 | 이동의 물리적 실감 일부 포기 |
| 수직 공간이 너무 길면 방향 감각 상실 | 세계 규모감 vs 방향 인지 | 랜드마크(기둥, 조명, 독특한 구조물)로 시각적 앵커 제공 | 자유로운 수직 연장 불가 |
| 위아래 방향만 있으면 세계가 단조롭다 | 수직 테마 vs 탐험 다양성 | 층위마다 수평 탐험 공간 충분히 제공 | 수직 순수성 일부 희석 |
| 스탯 게이트가 수직 진행을 완전 차단한다 | 야리코미 유도 vs 탐험 흐름 | 스탯 게이트는 선택 구역에만 배치, 크리티컬 패스는 능력 게이트만 사용 | 스탯 게이트의 진행 차단력 약화 |
| 층위가 독립적이면 세계 연결감이 약해진다 | 층위별 정체성 vs 통합 세계감 | 층위 경계에서 시각/청각 전환 + 배경에 다른 층위 보이기 | 층위 완전 독립성 포기 |

## 8. 체크리스트 — 수직 매크로 구조 설계 검증

### 세계 구조

- [ ] 각 층위는 위아래 층위와 연결되어 있는가?
- [ ] 각 층위는 최소 1개의 추가 연결(수평 또는 대각선 연결)을 갖는가?
- [ ] 수직 통로/엘리베이터가 매크로 허브 역할을 하는가?
- [ ] 각 층위는 시각적/청각적으로 구분되는 정체성을 갖는가?
- [ ] 층위 경계에서 환경적 전환이 점진적으로 이루어지는가?

### 크리티컬 패스

- [ ] 크리티컬 패스가 전체 맵의 30~60% 수준인가?
- [ ] 크리티컬 패스를 UI 마커 없이 환경 언어만으로 유도하고 있는가?
- [ ] 선택 구역은 크리티컬 패스보다 높은 보상을 제공하는가?
- [ ] 백트래킹 시 새로운 발견이 있는가?

### 게이팅

- [ ] 층위당 필수 능력 게이트가 1~2개 이하인가?
- [ ] 스탯 게이트는 선택 구역에만 배치되어 있는가?
- [ ] 소프트 게이트를 통한 Sequence Breaking이 크리티컬 패스를 파괴하지 않는가?
- [ ] 각 게이트 통과 조건이 시각적으로 명확히 안내되는가?

### 지름길

- [ ] 각 층위에 최소 1개의 루프백/지름길이 존재하는가?
- [ ] 지름길은 어렵게 도달한 후 발견되도록 배치되어 있는가?
- [ ] 보스 방 직전에 세이브 포인트(=패스트 트래블 거점)가 배치되어 있는가?

### 페이싱

- [ ] 고강도 전투/플랫폼 구간 직후 휴식 공간(Breath Room)이 배치되어 있는가?
- [ ] 각 층위 내 난이도가 톱날 곡선(긴장-이완-재긴장) 패턴을 따르는가?
- [ ] 깊어질수록 전반적 난이도가 상승하는 추세를 유지하는가?
