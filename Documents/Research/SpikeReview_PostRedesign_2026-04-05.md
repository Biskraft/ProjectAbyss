## 3. "한 문장 설명" 테스트

> **축소 전:** "고딕 세계를 탐험하고, 아이템 속에 들어가 강화하고, 자동 사냥하고, 거래소에서 팔고, 시즌마다 리셋되고, 8종 무기로 패링하고, 6개 스탯과 스킬트리를 찍고, 5원소 시너지를 활용하고, 허브에서 이노센트를 키우고, 모바일에서도 되는 온라인 RPG"

> **현재:** **"아이템 속으로 들어가면, 그 아이템의 기억이 던전이 된다."**

## 5. 남아있는 시스템 — 스파이크 정렬 검증

### 5.1. 스파이크 핵심 (아이템계 직결)

| 시스템 | 스파이크 기여 |
|:-------|:-------------|
| ItemWorld Core / FloorGen / Boss / Events / Recursion | 아이템계의 뼈대 |
| ItemNarrative Template / MonsterPool / EnvironmentPool | "기억이 던전이 된다" — 서사 스파이크 |
| Innocent Core (15종 행동형) | "아이템계에서 잡은 것이 플레이를 바꾼다" |
| Equipment Rarity / Slots / Growth | 아이템계 진입 이유 (더 좋은 장비) |
| World StatGating (ATK/INT) | 아이템계 → 월드 순환의 동력 |

### 5.2. 필수 인프라

| 시스템 | 역할 |
|:-------|:-----|
| 3C (Character / Control / Camera) | 기본 조작 |
| Combat (Action / Damage / HitFeedback / Weapons 5종) | 전투 — 아이템계 안에서의 경험 |
| Growth (Stats 3개 / LevelExp) | 성장 — ATK/INT 올리기 |
| Enemy (AI / BossDesign) | 아이템계 안의 적 |
| World (Map / Zone / ProcGen / AbilityGating) | 탐험 — 아이템을 얻는 공간 |

### 5.3. 주시 필요

| 시스템 | 우려 | 권고 |
|:-------|:-----|:-----|
| 능력 게이트 5종 | 5종 전부 만들면 월드 층위 5개 필요 | 이단점프 1종으로 시작 |
| 레어리티 5등급 | Normal~Ancient 전체 구현 비용 | N/M/R 3등급으로 축소 가능 |
| 멀티플레이 | URL 초대는 Tier S 스파이크 | 솔로 검증 이후 구현 |

## 6. 빌드 다양성 — 축소했는데 오히려 깊어졌는가?

| 축 | 축소 전 | 현재 | 평가 |
|:---|:--------|:-----|:-----|
| 무기 | 8종 (각각 미완성) | 5종 (각각 정체성 명확) | 더 깊음 |
| 스탯 | 6개 (버려지는 스탯 존재) | ATK vs INT 빌드 분화 | 더 명확 |
| 이노센트 | 스탯+X 4종 (숫자 올리기) | 행동형 6종 + 기본 3종 (플레이 변형) | 훨씬 깊음 |
| 원소 | 5종 (풍/광 역할 불명) | 3종 (화/빙/뇌, 100% 적중, INT 스케일) | 더 명확 |
| 스킬 | 트리 10분기 (미구현) | 무기 내장 (단순) | 동등 (이노센트가 대체) |

**결론: 시스템 수는 줄었지만 빌드 선택의 의미는 더 강해졌다.**

### 빌드 예시 — Rare 검 (슬롯 4)

| 빌드 | 이노센트 구성 | 플레이 스타일 |
|:-----|:-------------|:-------------|
| 물리 DPS | Gladiator ×2 + Berserker + Vampire | 빠른 콤보 + 킬 흡혈 |
| 원소 마법 | Tutor ×2 + Burner + Berserker | 화 인챈트 극대화 |
| 탱커 | Dietician + Ironclad ×2 + Gladiator | 보스전 버티기 |
| 기동형 | Ghost ×2 + Sprinter + Gladiator | 회피 중심 속파 |
| 파밍 | Scavenger ×2 + Statistician + Gladiator | 아이템계 효율 |

같은 검인데 전혀 다른 5가지 플레이. 이것이 "세상에 하나뿐인 내 장비".
