## 1. 문제 진단

### 1.1 증상

플레이어가 다음 방으로 이어지는 출구(룸 전환 타일)에 접근해도 그곳이 통로임을 인지하지 못한다. 결과적으로 탐험이 정체되며 "막다른 길"로 돌아가는 행동이 반복된다.

### 1.2 근본 원인

| 원인 | 증거 | 심각도 |
| :--- | :--- | :--- |
| **뷰포트 레터박스 = 벽 색** | PixiJS Application 배경색과 `iw_*_wall` 그라디언트 최하단(`#0c2028`, `#10202c` 등)이 양쪽 모두 "거의 검정". 웹 브라우저 16:9 보정 레터박스가 벽과 연속되어 보임 | **최고** |
| **출구 Affordance 부재** | 출구 타일이 주변 벽과 시각적으로 구분되지 않음. "지나갈 수 있다"는 신호 없음 | 높음 |
| **구조적 팔레트 설계** | Content_System_Area_Palette.csv 전 지형의 벽이 "거의 검정"으로 설계됨 (Dead Cells Grayscale 팔레트 계승) — 임시 해결(특정 방만 수정)은 구조와 충돌 | 구조적 |
| **Affordance 교습 전무** | BotW 삼각형 법칙 / ALBW 시각 언어와 달리, ECHORIS는 현재 "출구는 이렇게 생겼다"는 약속 언어가 없음 | 높음 |

### 1.3 왜 단순 해결이 어려운가

- 벽 색을 밝게 하면 **"거의 검정 벽 = Dead Cells 팔레트 정체성"** (DEC-021, 2026-04-17)과 충돌
- 레터박스를 제거(CSS `object-fit`)하면 종횡비가 깨짐
- UI 화살표는 대사 0줄/UI 최소화 원칙과 충돌

→ **다층 해결**이 필요하다: 레터박스 대비 회복 + 출구 시각 언어 확립 + 카메라 유도.

## 4. ECHORIS 권장 조합 — 3층 스택

단일 수단은 효과가 약하다. **누적 설계**로 중첩하여 해결한다.

### 4.1 1층 — C2: 레터박스 색 교체 (즉시, 필수)

| 항목 | 값 |
| :--- | :--- |
| **조치** | PixiJS Application 배경색 및 HTML body 배경을 `#3a1a28` 으로 변경 |
| **근거 색상** | `world_shaft_bg` 그라디언트 최하단(`0.00:3a1a28`) — 월드 기본 색 |
| **비용** | 1줄 수정, 1시간 내 배포 |
| **효과** | 검정 = 벽 혼동의 **구조적 원인 제거** |
| **파일** | `game/src/Game.ts`, `game/index.html` body background |

### 4.2 2층 — A2: 출구 Light Bleed (핵심 시각 언어)

| 항목 | 값 |
| :--- | :--- |
| **조치** | 룸 전환 타일(LDtk entity 또는 IntGrid 타입) 양옆에 PixiJS `GlowFilter` 또는 Graphics 그라디언트 부착 |
| **색상** | 주황 `#e07028` → 투명. 강도 `outerStrength=2–3` |
| **반경** | 16–24px, 맥동 1.0s 주기 `alpha = 0.6 + sin(t) * 0.2` |
| **적용 범위** | 모든 방 전환부에 일괄 — 공통 시각 언어로 규칙화 |
| **비용** | 중 — Glow 렌더링 + LDtk 탐지 로직 |
| **효과** | "주황빛이 새어 나오는 곳 = 출구"라는 약속 언어 확립 |

### 4.3 3층 — C1: 카메라 Lookahead (에이전시 회복)

| 항목 | 값 |
| :--- | :--- |
| **조치** | 카메라 추종 로직에 이동 방향 예측 오프셋 추가 |
| **공식** | `targetX = player.x + sign(vx) * LOOKAHEAD_DISTANCE`. LOOKAHEAD_DISTANCE = 40px |
| **조건** | 이동 방향 지속 300ms 이상, 접지 상태, 비전투(최근 3초 내 피격/공격 없음) |
| **Lerp** | `camera.x += (targetX - camera.x) * 0.08` (프레임당) |
| **비용** | 중 |
| **효과** | 플레이어가 직접 다음 방 끄트머리를 확인 → 납치감 제거 |
| **파일** | `game/src/scenes/LdtkWorldScene.ts` 카메라 추종 함수 |

### 4.4 선택 4층 — A1: 아치 도어웨이 타일 (Phase 2 이후)

핸드크래프트 룸에만 적용. 절차적 방은 제외 (타일 일관성 불가). 빌더 구조물 서사를 강화하는 플롯-정당화된 시각 언어.

| 조치 | 예상 "막다른 길" 오인 감소율 |

## 7. 2층 A2의 포탈/출구 색상 역할 분리

출구 Light Bleed와 레어리티 포탈(Normal~Ancient 색상)이 시각적으로 충돌할 위험이 있다. 분리 규칙:

| 요소 | 색상 | 의미 |
| :--- | :--- | :--- |
| **일반 출구 (방 전환)** | 따뜻한 주황 `#e07028` | "지나가는 통로" — 단조열 |
| **아이템계 포탈** | 레어리티 색 | "이곳은 차원 전환점" — 레어리티 고지 |
| **제단 포탈** | 흰색/청록 | "제단에서 생성됨" — 구분 가능 |

주황 = "월드 탐험의 방향성", 레어리티 색 = "아이템계로의 초대". 플레이어는 두 언어를 곧 구별한다.
