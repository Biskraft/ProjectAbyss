# 대화 UI 시스템 리서치: 느낌 좋은 대화창 설계 분석

> **문서 ID:** RES-DLG-01
> **문서 상태:** `초안 (Draft)`
> **작성일:** 2026-04-06
> **작성 목적:** Project Abyss 대화 UI 설계를 위한 레퍼런스 분석

---

## 0. 리서치 목적

Project Abyss의 주인공 에르다는 침묵 주인공(Silent Protagonist)이다. NPC가 에르다에게 일방적으로 말하고, 에르다는 반응하지 않는다. 이 리서치는 침묵 주인공 기반 대화 UI의 사례를 분석하고, 어떤 요소가 대화 경험을 만족스럽게 만드는지를 밝히며, Project Abyss의 구체적 구현 스펙을 제안하는 것을 목적으로 한다.

---

## 1. 게임별 대화 UI 분석

### 1.1. Castlevania: Symphony of the Night (1997)

**침묵 주인공 처리 방식**

알루카드는 엄밀한 의미에서 침묵 주인공이 아니다. 그는 대사를 가지지만 분량이 극히 적으며, 대부분의 대화는 NPC가 알루카드에게 일방적으로 설명하거나 경고하는 형식이다. 이는 Project Abyss의 에르다 구조와 유사하다.

**텍스트 박스 구조**

- 화면 하단에 고정된 가로형 텍스트 박스
- 배경: 반투명 다크 블루 (거의 불투명에 가까운 네이비)
- 테두리: 얇은 청회색 선 + 모서리 장식 (PS1 시대의 바로크풍 UI)
- 폰트: 비트맵 고정폭 폰트, 흰색, 소문자 혼용
- NPC 이름이 텍스트 박스 상단 좌측에 별도 표기
- 포트레이트 없음 — 이름만으로 화자를 식별

**타이핑 연출**

- 글자 단위 타이핑 (문자별 순차 표시)
- 타이핑 속도: 중간 (약 40~50자/초 추정)
- 페이지 진행: 버튼 입력으로 수동 진행
- 다음 페이지 표시: 박스 우측 하단에 깜빡이는 삼각형(▼)

**사운드**

- 타이핑 사운드 없음 (텍스트 표시 시 무음)
- BGM이 분위기를 대신함

**분석 및 인사이트**

SotN의 대화 UI는 기능적이지만 감각적으로 빈약하다. 포트레이트와 타이핑 사운드의 부재로 인해 대화가 단순한 정보 전달로 느껴진다. 그러나 NPC 이름 표기와 다크 배경 색상은 당시 장르에서 정착된 시각 문법이었으며, Project Abyss의 고딕 미학과 호환된다.

### 1.2. Hollow Knight (2017)

**침묵 주인공 처리 방식**

The Knight는 완전한 침묵 주인공이다. 대화 인터랙션 시 플레이어는 NPC 근처에서 상호작용 버튼을 누를 뿐이며, The Knight의 대사는 일절 없다. 모든 텍스트는 NPC의 독백 형식이다. 이것이 Project Abyss 에르다와 가장 유사한 구조다.

**텍스트 박스 구조**

- 화면 하단 중앙에 고정된 텍스트 박스
- 배경: 반투명 검정 (알파 약 75~80%)
- 테두리: 둥근 모서리의 세련된 프레임 — 별도 장식 없음
- 폰트: 커스텀 비트맵 세리프 폰트, 흰색
- 포트레이트 없음 — NPC는 게임 월드 내 실제 스프라이트로 존재하므로 포트레이트가 불필요
- 화자 이름 표기 없음 (상황 맥락으로 파악)

**타이핑 연출**

- 글자 단위 타이핑
- 타이핑 속도: 빠름 (약 60자/초 추정)
- 버튼 홀드 시 전체 텍스트 즉시 표시 (스킵 기능)
- 전체 표시 후 재입력으로 다음 페이지 진행
- 다음 페이지 표시: 박스 하단에 깜빡이는 화살표

**사운드**

- 캐릭터별 고유 음성 블립(Voice Blip)이 있음
- 블립의 음색과 리듬이 캐릭터 성격을 표현:
  - Quirrel: 부드럽고 단조로운 블립 (조용한 여행자)
  - Cornifer: 활기차고 빠른 블립 (흥분한 지도 제작자)
  - Zote: 과장된 낮은 음역의 블립 (거만한 자)
  - Grimm: 느리고 중후한 블립 (위협적인 분위기)
- 블립은 글자가 화면에 나타날 때마다 재생되며, 타이핑 속도와 동기화

**드림네일 대화**

특수 상호작용인 드림네일(Dream Nail)로 적과 NPC의 내면 독백을 읽을 수 있다. 동일한 텍스트 박스 UI이지만 색조가 보라/금빛으로 바뀐다. 이 컬러 변화가 "일반 대화"와 "내면 독백"을 즉시 구분해준다.

**분석 및 인사이트**

Hollow Knight는 침묵 주인공 대화의 모범 사례다. 포트레이트가 없어도 캐릭터별 블립 사운드만으로 화자의 개성이 생생하게 전달된다. 텍스트 박스가 최소주의적으로 설계되어 있어 세계관에서 이탈하는 느낌을 주지 않는다. NPC가 월드 내에 실제로 존재하므로 포트레이트의 공백을 시각적으로 채운다.

### 1.3. Dead Cells (2018)

**침묵 주인공 처리 방식**

The Prisoner(플레이어)는 사실상 침묵 주인공이다. The Collector 등 NPC가 일방적으로 말을 건네며 플레이어의 대사는 없다.

**텍스트 박스 구조**

- 대화창이 아닌 단발성 팝업 텍스트 형식
- 캐릭터 스프라이트 위에 말풍선(Speech Bubble) 형태로 표시
- 배경: 반투명 어두운 회색
- NPC 고유 색상 코딩: The Collector의 대사는 특정 색으로 강조
- 긴 대화 없음 — 대부분 1~3줄 이내의 짧은 독백

**타이핑 연출**

- 즉시 표시 (타이핑 애니메이션 없음)
- 일정 시간 후 자동 소멸 또는 버튼 입력으로 닫기

**사운드**

- 음성 블립 없음
- 배경음악과 효과음이 분위기 전달을 대신함

**분석 및 인사이트**

Dead Cells의 대화 UI는 게임 템포를 방해하지 않기 위한 최소화 전략을 취한다. 하드코어 액션 게임의 특성상 대화에 몰입을 요구하지 않는다. Project Abyss에서 빠른 NPC 반응(상점 NPC, 지나치는 주민)에 응용할 수 있는 패턴이다.

### 1.4. Celeste (2018)

**침묵 주인공 처리 방식**

Madeline은 침묵 주인공이 아니다. 그녀는 적극적으로 대화한다. 그러나 Celeste의 대화 UI 연출 기법은 업계에서 가장 정교하게 분석된 사례 중 하나이므로, 기법 추출 목적으로 분석한다.

**텍스트 박스 구조**

- 화면 하단(또는 상단, 상황에 따라 가변)에 배치
- 배경: 화자별로 고유 색상의 반투명 박스
  - Madeline: 분홍/마젠타 계열
  - Theo: 파랑 계열
  - Badeline(Part of me): 보라/다크 계열
- 포트레이트: 텍스트 박스 좌측에 배치된 캐릭터 얼굴 스프라이트
- 포트레이트는 감정(Expression)에 따라 실시간 교체

**포트레이트 감정 시스템**

- 각 캐릭터마다 10개 이상의 표정 스프라이트 보유
- 대사 중간에도 감정 태그로 포트레이트 교체 가능
- 포트레이트가 텍스트 박스 프레임을 벗어나 튀어나오는 연출 사용 (강한 감정 표현 시)

**타이핑 연출**

- 글자 단위 타이핑
- 텍스트에 웨이브(흔들림) 및 셰이킹(떨림) 효과 적용 가능
  - `{wave}텍스트{/wave}`: 공포, 흥분 시
  - `{shake}텍스트{/shake}`: 극도의 동요 시
- 특정 단어에 강조 색상 적용 가능

**사운드 — Celeste의 핵심 혁신**

Celeste의 오디오 디렉터는 각 캐릭터의 음성을 완전 합성음(Synthesized Voice)으로 설계했다. 구체적 방법론:

1. 각 캐릭터의 감정 범위를 분석 (행복/슬픔/두려움/분노 등)
2. 각 감정별로 3개 카테고리의 음절 샘플을 제작:
   - 빠른 경과 음절(Quick Passing Syllables): 약 20개
   - 강조 음절(Emphasized Syllables): 약 10개
   - 문장 종료 음절(Sentence Endings): 약 10개
3. FMOD를 통해 현재 포트레이트 표정 정보를 오디오 엔진에 전달
4. FMOD가 감정에 맞는 음절 풀에서 랜덤 재생
5. 타이핑 종료 시 FMOD에 "발화 종료" 신호 전달

결과: 실제 사람 목소리처럼 들리지만 어떤 언어도 아닌 "가상 언어" 음성이 각 캐릭터에 부여됨.

**분석 및 인사이트**

Celeste의 감정 기반 오디오 시스템은 대화 UI에서 가장 인상적인 혁신 중 하나다. 포트레이트의 표정과 텍스트의 웨이브 효과, 그리고 합성 음성이 삼위일체로 작동하여 감정을 전달한다. Project Abyss에서 완전히 이 시스템을 구현하기는 어렵지만, 핵심 원리(캐릭터별 고유 블립 + 감정 변화)는 반드시 적용해야 한다.

### 1.5. Undertale (2015) / Deltarune (2018~)

**침묵 주인공 처리 방식**

Frisk(Undertale)와 Kris(Deltarune)는 침묵 주인공이다. 단, Undertale은 일부 대화 선택지가 있어 완전한 침묵은 아니다. Frisk의 선택지는 플레이어가 직접 결정하는 행동의 표현이며, "말하는" 것이 아니라 "행동하는" 것이다.

**텍스트 박스 구조**

- 화면 하단에 배치
- 배경: 검정 (불투명)
- 테두리: 흰색 단순 직사각형 테두리
- 좌측에 캐릭터 포트레이트 배치
- 포트레이트 크기: 약 80×80픽셀 영역 (원본 기준)
- 내러티브(환경 설명) 시에는 포트레이트 없음

**폰트 시스템 — Undertale의 핵심 정체성**

Undertale은 각 캐릭터마다 고유 폰트를 사용한다:

| 캐릭터 | 폰트 | 느낌 |
| :--- | :--- | :--- |
| 기본 내러티브 | 8-Bit Operator (픽셀 고정폭) | 중립적, 게임적 |
| Sans | 변형 Comic Sans | 캐주얼, 반어적 |
| Papyrus | 변형 Papyrus 폰트 | 과장, 자기 과시 |
| Flowey | 표준 폰트 + 왜곡 효과 | 불안, 위협 |
| Toriel | 둥근 폰트 | 따뜻함, 모성애 |

이 폰트 선택은 음향 설계 없이도 "이 문장은 누가 말하는가"를 즉시 전달한다.

**타이핑 사운드**

- 각 캐릭터마다 고유 블립 사운드 (Voice Font)
- 블립 톤의 피치, 음색, 리듬이 캐릭터 성격과 일치
- Sans: 낮고 느린 블립
- Papyrus: 높고 과장된 블립

**텍스트 효과**

- `{shake}`: 공포 또는 불안 시 텍스트 흔들림
- `{wave}`: 마법/신비로운 분위기 시 텍스트 웨이브
- 특정 단어 색상 강조

**분석 및 인사이트**

Undertale은 "폰트가 곧 목소리"라는 원칙을 확립한 게임이다. 포트레이트나 고품질 음성 없이도 폰트 하나로 캐릭터 개성을 즉각 전달한다. Project Abyss는 고딕/메가스트럭처 미학이 강하므로, NPC 계급/직업에 따른 폰트 변형을 검토할 수 있다.

### 1.6. Final Fantasy VI (1994)

**텍스트 박스 구조**

FF6는 포트레이트 없이 텍스트만으로 대화를 구성하는 JRPG의 원형이다.

- 화면 하단 1/3 영역에 고정된 텍스트 박스
- 배경: 다크 블루 (시리즈 전통색)
- 테두리: 얇은 흰색/회색 이중 테두리 + 모서리 장식
- 폰트: 비트맵 고정폭 폰트
- 화자 이름: 텍스트 박스 상단 좌측에 표기
- 포트레이트: 원본 기준 없음. 팬 모드 및 리마스터에서 추가됨

**유산과 영향**

FF6의 "화자 이름 표기 + 하단 텍스트 박스" 구조는 이후 수많은 JRPG와 메트로베니아 게임의 표준이 되었다. SotN의 UI도 이 전통을 계승한다.

**분석 및 인사이트**

포트레이트 없는 대화의 한계와 가능성을 동시에 보여준다. 이름 표기만으로 화자를 명확히 하는 것이 가능하지만, 감정 전달 능력이 제한된다. 현대 게임에서는 보완 수단(포트레이트, 블립, 텍스트 효과)을 추가하는 것이 표준이다.

### 1.7. Persona 5 (2016)

**대화 UI의 미학적 혁신**

Persona 5의 대화 UI는 기능성과 스타일의 균형에 있어 업계 최상위 사례다.

**텍스트 박스 구조**

- 비정형(기울어진) 다각형 형태의 텍스트 박스
- 배경: 검정 + 짙은 빨강 (테마 컬러)
- 굵은 흰색 테두리
- 말풍선 "꼬리(Tail)"가 화자 방향을 가리킴 (화살표 역할)
- 캐릭터 포트레이트: 대형 흉상 일러스트, 텍스트 박스 좌측 배치
- 포트레이트에 제한적 애니메이션(립싱크, 눈 깜빡임) 적용

**색상 시스템**

- 주인공(Joker) 대사: 검정/빨강
- 고유 NPC별 색상 코딩 (세컨더리 컬러로 UI 차별화)
- 고채도 색상(보라, 녹색, 노랑)을 서브 NPC에 사용해 인상 강화

**디자인 철학**

"일관된 테마 유지 — 메뉴를 열어도 게임 세계를 이탈하지 않는다"는 원칙이 관철된다. 대화창, 인벤토리, 지도 모두 동일한 시각 언어를 사용한다.

**분석 및 인사이트**

Persona 5는 "UI가 게임 정체성의 일부"라는 명제를 증명한 게임이다. 기울어진 텍스트 박스와 공격적인 색상 대비는 처음에는 불편하게 느껴질 수 있지만, 플레이 중 익숙해지면 다른 UI가 밋밋하게 보일 정도로 강렬한 인상을 남긴다. Project Abyss에서는 고딕/산업 미학을 UI 레이아웃 자체에 녹여내는 접근을 참고할 수 있다.

---

### 1.8. Shovel Knight (2014)

**대화 UI 구조**

- 화면 하단에 배치된 고전 JRPG 스타일 텍스트 박스
- 포트레이트: 픽셀 아트 반신상, 좌측 배치
- 배경: 어두운 배경 + 황금색 테두리 (중세 판타지 미학 강화)
- 포트레이트가 감정에 따라 교체됨 (기본/분노/슬픔 등)
- 화자 이름은 텍스트 박스 상단 또는 포트레이트 아래에 표기

**타이핑 연출**

- 글자 단위 타이핑
- 짧고 경쾌한 타이핑 블립 사운드
- 버튼 입력으로 전체 즉시 표시 + 페이지 진행

**분석 및 인사이트**

Shovel Knight는 "고전 JRPG 대화 UI의 현대적 재해석" 교과서다. 픽셀 아트 포트레이트 + 황금 테두리 조합이 중세 판타지 미학을 텍스트 박스 레벨에서 강화한다. Project Abyss의 고딕 미학 적용 시 직접 참고 가능한 레이아웃 패턴이다.

---

### 1.9. Blasphemous (2019)

**침묵 주인공과 고딕 미학**

Blasphemous의 The Penitent One은 침묵 주인공이다. 이 게임은 Project Abyss와 가장 유사한 미학(고딕, 다크 판타지, 2D 횡스크롤, 메트로베니아)을 가지고 있어 핵심 레퍼런스다.

**텍스트 박스 구조**

- 화면 하단에 배치
- 배경: 짙은 검정/갈색 계열의 양피지 텍스처
- 테두리: 종교적 장식 모티프 (가시 왕관, 십자가 형태의 엠블럼)
- 커스텀 고딕 픽셀 폰트 사용 (가독성 낮지만 분위기 압도적)
- 포트레이트: NPC의 흉상 픽셀 아트, 좌측 배치
- 포트레이트에 독특한 황금/종교적 프레임 적용

**음성 처리**

- NPC에 일부 실제 음성 연기(Voice Acting) 적용
- 그러나 라틴어/스페인어 혼합의 의도적 불명확한 발음으로 처리
- The Penitent One의 대사는 일절 없음

**분석 및 인사이트**

Blasphemous는 "텍스트 박스가 세계관의 일부"라는 원칙을 실현한 최고의 예시다. UI 테두리만 봐도 어떤 세계관의 게임인지 즉시 알 수 있다. 단, 가독성 희생이 너무 심하다는 비판도 있다 — Project Abyss에서는 미학과 가독성의 균형을 더 신중하게 조율해야 한다.

---

### 1.10. CrossCode (2018)

**독특한 대화 시스템 구조**

CrossCode는 MMORPG를 시뮬레이션하는 설정의 게임으로, 주인공 Lea가 대사를 거의 하지 못하는 설정(음성 합성 장치 고장)을 가지고 있다. 이것은 침묵 주인공 처리의 창의적 변형 사례다.

**주인공의 침묵 처리**

- Lea는 "Emotes(감정 표현)"으로만 소통
- 텍스트 박스에 Lea의 대사 자리는 있지만 대부분 "..."
- 가끔 단어 한두 개를 간신히 발화
- NPC들이 Lea의 제한된 소통에 반응하며 대화를 이끌어감

**텍스트 박스 구조**

- 화면 상단에 배치 (일반적 하단 배치와 차별화)
- 포트레이트: 고해상도 애니메이션 스타일 일러스트
- 좌우 포트레이트 동시 표시 가능 (1:1 대화 연출)

**분석 및 인사이트**

CrossCode의 Lea는 "...와 Emote만으로 소통하는 침묵 주인공"의 흥미로운 변형이다. Project Abyss 에르다가 고개를 끄덕이거나, 에코를 향해 시선을 돌리거나, 아무 반응 없이 그냥 걷기 시작하는 행동 언어로 소통하는 방식과 연결된다.

## 2. "느낌 좋은" 대화 UI의 공통 요소

### 2.1. 타이핑 리듬이 만드는 기대감

글자가 한 번에 표시되지 않고 순차적으로 나타나는 타이핑 효과는 단순 연출이 아니다. 이것은 다음 심리적 메커니즘을 활성화한다:

**기대 형성(Anticipation)**: 글자가 하나씩 나타남으로써 플레이어는 무의식적으로 "다음에 무슨 말이 올까"를 예측하기 시작한다. 이 예측 행위 자체가 참여도(Engagement)를 높인다.

**읽기 속도 제어**: 타이핑 속도가 최적 읽기 속도와 일치하면, 플레이어는 텍스트를 스킵하거나 기다리지 않고 자연스럽게 흡수한다. 너무 느리면 지루하고, 너무 빠르면 버튼 연타를 유발한다.

**최적 타이핑 속도 범위**: 30~60자/초. 감정적으로 강한 대사는 느리게, 일상 대화는 빠르게 조정하는 것이 이상적이다.

### 2.2. 사운드 블립의 존재감

음성 블립(Voice Blip)은 대화 UI에서 음성 연기(Voice Acting)를 대체하는 가장 비용 효율적인 도구다. 효과가 큰 이유:

- **리듬 형성**: 블립이 타이핑과 동기화되면 대화 자체가 리듬감 있는 퍼포먼스가 됨
- **캐릭터 분화**: 피치, 음색, 재생 패턴으로 캐릭터 개성 즉각 전달
- **몰입 유지**: 화면 효과만으로는 줄 수 없는 "누군가 말하고 있다"는 느낌을 제공

블립이 없는 대화(SotN, Dead Cells)는 상대적으로 평면적으로 느껴진다.

### 2.3. 포트레이트의 감정 노동

포트레이트의 핵심 역할은 화자 식별이 아니라 감정 전달이다. 정적 포트레이트(감정 변화 없음)는 화자 식별에는 유효하지만 감정 전달 능력이 0에 가깝다. 느낌 좋은 대화 UI의 포트레이트는 최소 3~5단계의 표정 변화를 가진다.

**표정 교체 타이밍**: 새 대사 블록 시작 시 또는 감정 태그 도달 시

**포트레이트 크기**: 텍스트 박스 높이의 90~110% (텍스트 박스와 비슷하거나 약간 크게)

### 2.4. 진행 표시자(Advance Indicator)

"더 읽을 내용이 있다"는 신호를 플레이어에게 명확히 주어야 한다. 표준 솔루션:

- 깜빡이는 삼각형(▼) 또는 화살표 — 가장 보편적
- 밝아졌다 어두워지는 아이콘 (페이드 인/아웃)
- 작은 화살표의 바운스 애니메이션

이 표시자가 없으면 플레이어가 "더 읽을 게 있는지" 불확실해하며 버튼을 불필요하게 연타하거나 반대로 기다린다.

### 2.5. 텍스트 박스와 세계관 통합성

느낌이 좋은 대화 UI는 텍스트 박스 자체가 세계관 미학을 강화한다:

| 게임 | 텍스트 박스 미학 | 세계관 강화 효과 |
| :--- | :--- | :--- |
| Shovel Knight | 황금 테두리 + 픽셀 아트 | 중세 판타지 기사 세계관 |
| Blasphemous | 종교 장식 + 양피지 텍스처 | 고행 종교 세계관 |
| Persona 5 | 기울어진 형태 + 빨강/검정 | 반항아 팬텀 씨프 |
| Hollow Knight | 최소주의 + 어두운 반투명 | 황폐한 곤충 왕국 |

세계관 통합성이 낮은 대화 UI(일반적인 파란 직사각형 박스)는 플레이어를 게임 세계에서 잠시 이탈시킨다.

### 2.6. 침묵 주인공의 반응 처리

침묵 주인공이 있는 게임들의 공통 전략:

1. **무반응 전략** (Hollow Knight): 주인공의 대화 박스 자체가 없음. NPC만 말함
2. **행동 언어 전략** (CrossCode의 Lea): "..." + 캐릭터 애니메이션으로 소통
3. **플레이어 행동이 반응** (Undertale의 Frisk): 선택지가 말이 아닌 행동 목록

Project Abyss의 에르다는 1번 전략이 가장 적합하다. NPC가 에르다에게 말을 걸고, 에르다는 서 있거나 걸어가거나 에코를 들어 올리는 행동으로 반응한다. 이 행동들은 대화 UI 밖, 게임플레이 레이어에서 발생한다.

## 3. Project Abyss 대화 UI 제안

### 3.1. 설계 목표

| 항목 | 목표 |
| :--- | :--- |
| 미학 통합 | 고딕 / 메가스트럭처 산업 미학을 텍스트 박스에서 구현 |
| 캐릭터 분화 | 포트레이트 + 블립으로 NPC 개성 즉각 전달 |
| 침묵 처리 | 에르다의 대사 박스 없음. 에르다 UI 일절 없음 |
| 가독성 | Blasphemous의 실패(극단적 고딕 폰트)를 반면교사. 분위기 + 가독성 동시 확보 |
| 구현 가능성 | Phase 1 프로토타입 범위 내에서 구현 가능한 스펙 |

### 3.2. 레이아웃 (Layout)

**기본 배치**

```
┌──────────────────────────────────────────────────────────────┐
│                        [게임 화면]                            │
│                                                              │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ ┌──────────┐  ┌──────────────────────────────────────────┐  │
│ │          │  │ NPC 이름                                  │  │
│ │ 포트레이트 │  │─────────────────────────────────────────│  │
│ │  96×96   │  │ 대사 텍스트가 여기 표시됩니다.            │  │
│ │          │  │ 한 줄 최대 약 40자. 최대 3줄.             │  │
│ └──────────┘  │                                    [▼]   │  │
│               └──────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

**수치 스펙 (1280×720 기준)**

| 요소 | 위치 / 크기 |
| :--- | :--- |
| 텍스트 박스 전체 | X: 0, Y: 540, W: 1280, H: 180 |
| 포트레이트 영역 | X: 16, Y: 544, W: 96, H: 96 |
| NPC 이름 | X: 128, Y: 548, 폰트 14px, 강조 색상 |
| 대사 텍스트 | X: 128, Y: 572, W: 1120, 폰트 12px |
| 진행 표시자(▼) | X: 1240, Y: 696 |

**가변 배치 조건**

- 대화 NPC가 화면 하단에 있는 경우: 텍스트 박스를 상단으로 이동 (Y: 0~160)
- 상단 배치 시 포트레이트는 동일하게 좌측 유지

---

### 3.3. 시각 스타일 (Visual Style) — 고딕/메가스트럭처 미학

**배경 (Background)**

- 색상: `#0D0D0F` (거의 검정에 가까운 다크 네이비)
- 알파: 0.92 (약한 반투명)
- 텍스처 옵션: 미세한 금속 스크래치 텍스처 오버레이 (불투명도 10%)

**테두리 (Border)**

- 기본 테두리: `1px solid #3A3A4A` (냉회색)
- 강조 테두리: `2px solid #7A6A3A` (낡은 황동색) — 상단 엣지에만 적용
- 좌측 포트레이트 경계: `1px solid #5A5A6A` 수직선

**포트레이트 프레임**

- 포트레이트 주변: 메가스트럭처 계측 장비 스타일의 직각 프레임
- 코너에 작은 리벳/나사 스프라이트 배치 (16×16px)
- 포트레이트 프레임 색상: `#4A4040` (산화된 철 느낌)

**NPC 이름 색상 시스템**

NPC 계급/역할에 따라 이름 색상을 다르게 적용:

| NPC 유형 | 색상 | 용도 |
| :--- | :--- | :--- |
| 일반 주민 | `#C8C8C8` (흰색에 가까운 회색) | 지하 거주민, 노동자 |
| 상인 / 제공자 | `#F0C060` (황동/금색) | 물건 파는 NPC |
| 위험 / 적대 NPC | `#CC4444` (어두운 빨강) | 경계해야 할 존재 |
| 신비/미지 존재 | `#7070CC` (흐린 보라) | 정체 불명의 존재 |
| 시스템 메시지 | `#60B060` (흐린 녹색) | 게임 시스템 알림 |

**폰트**

- 기본 폰트: 비트맵 픽셀 폰트 (12px 기준, 픽셀 아트 렌더링)
- 조건: 영어 대소문자 + 한국어 지원 (게임 내 텍스트는 영어이지만 개발 과정에서 한국어 테스트 필요)
- 권장 폰트 후보: Pixel Operator, Press Start 2P (타이틀용), Noto Sans (한국어 폴백)
- 줄 간격(Line Height): 18px (폰트 12px + 여백 6px)

### 3.4. 타이핑 연출 (Typewriter Effect)

**기본 사양**

| 항목 | 값 |
| :--- | :--- |
| 기본 타이핑 속도 | 40자/초 (25ms/글자) |
| 감정적 대사 속도 | 20자/초 (50ms/글자) — 느리고 무거운 느낌 |
| 일상 대사 속도 | 60자/초 (17ms/글자) — 경쾌한 느낌 |
| 스킵 방식 | 버튼 홀드(0.2초) → 전체 즉시 표시 |
| 다음 페이지 | 스킵 후 추가 입력 또는 버튼 클릭 |

**구두점 딜레이**

특정 구두점 뒤에 딜레이를 추가하여 읽기 리듬을 조성한다:

| 구두점 | 추가 딜레이 |
| :--- | :--- |
| `,` | +50ms |
| `.` `!` `?` | +150ms |
| `...` | +300ms (침묵 효과) |
| `—` (em dash) | +100ms |

이 딜레이가 없으면 대사가 기관총처럼 쏟아져 나와 감정 리듬이 사라진다.

**텍스트 효과 (선택적 적용)**

| 효과 태그 | 연출 | 사용 상황 |
| :--- | :--- | :--- |
| `[wave]` | 텍스트 상하 웨이브 | 두려움, 불안, 마법적 |
| `[shake]` | 텍스트 랜덤 진동 | 극도의 공포, 분노 |
| `[slow]` | 타이핑 속도 1/2 | 죽어가는 NPC, 엄숙한 발언 |
| `[fast]` | 타이핑 속도 2배 | 흥분, 급박함 |

---

### 3.5. 사운드 설계 (Sound Design)

**블립 시스템 설계 원칙**

각 NPC는 고유한 블립 사운드 프로파일을 가진다. 블립은 글자가 화면에 표시될 때마다 재생되며, 글자 표시 빈도에 따라 재생 빈도가 결정된다.

**블립 파라미터**

| 파라미터 | 범위 | 역할 |
| :--- | :--- | :--- |
| 기본 피치 | 200Hz ~ 800Hz | 캐릭터 음역 설정 |
| 피치 랜덤 범위 | ±5% ~ ±15% | 자연스러움 추가 |
| 지속 시간 | 30ms ~ 80ms | 짧을수록 날카롭고, 길수록 부드럽게 |
| 음색 | 사인파/사각파/톱니파 | 사인=따뜻, 사각=날카롭, 톱니=기계적 |
| 볼륨 | 마스터 SFX의 30~50% | 블립이 효과음보다 작아야 함 |

**NPC 유형별 블립 프로파일 예시**

| NPC 유형 | 피치 | 음색 | 느낌 |
| :--- | :--- | :--- | :--- |
| 지하 거주민(노동자) | 300~400Hz | 사각파 | 거칠고 피곤한 |
| 기술자/수리공 | 500Hz | 사인파 + 고주파 배음 | 날카롭고 전문적인 |
| 신비로운 존재 | 200~250Hz | 사인파 (느리게 변조) | 울림, 비인간적 |
| 아이 / 젊은이 | 600~700Hz | 사인파 | 가볍고 빠른 |
| 위협적 존재 | 150~200Hz | 톱니파 | 저음, 불안감 |

**블립 재생 간격 최적화**

타이핑 속도가 빠를수록 블립이 너무 빠르게 연속 재생되어 소음이 된다. 이를 방지하기 위해:

- 최소 블립 재생 간격: 50ms (초당 20회 이상 재생 방지)
- 모음/자음 판별: 기술적으로 어려울 경우, 일정 간격(N번째 글자마다) 재생으로 단순화

### 3.6. 에르다(침묵 주인공) 처리 방법

**핵심 원칙**

에르다의 대사 박스는 존재하지 않는다. 대화 UI는 오직 NPC의 발화를 표시하는 용도로만 사용된다.

**에르다의 반응 표현 방법 (대화 UI 외부)**

| 상황 | 에르다 행동 | 구현 |
| :--- | :--- | :--- |
| NPC가 경고를 줄 때 | 잠시 멈추고 NPC 방향을 바라봄 | 이동 입력 차단 + 페이싱 로직 |
| NPC가 중요 정보를 줄 때 | 에코를 들어 올림 (에코 발광) | 에코 스프라이트 전환 |
| NPC가 무시해도 되는 말을 할 때 | 계속 걷거나 서 있음 | 기본 상태 유지 |
| NPC가 우호적 인사를 할 때 | 고개를 한 번 끄덕임 | 단발 애니메이션 트리거 |

**대화 중 에르다 이동 제한**

- 대화 시작 시 에르다의 이동 입력 차단 (강제 정지)
- 텍스트 박스 표시 중에는 전투 불가
- 대화 종료 시 이동 입력 복구
- 예외: 짧은 팝업 독백형 NPC(Dead Cells 스타일)는 에르다 이동 중에도 표시

**에르다와 NPC의 공간적 관계**

대화 중 카메라는 에르다와 NPC가 동시에 프레임 안에 들어오도록 조정한다. 이렇게 하면 포트레이트 없이도 "대화 장면"이라는 맥락이 시각적으로 성립한다.

### 3.7. 구현 스펙 요약

**Phase 1 MVP 스펙 (프로토타입 목표)**

| 기능 | 우선순위 | 난이도 |
| :--- | :--- | :--- |
| 기본 타이핑 텍스트 박스 | 필수 | 낮음 |
| 구두점 딜레이 | 필수 | 낮음 |
| 버튼으로 전체 표시 + 페이지 진행 | 필수 | 낮음 |
| 진행 표시자(▼) 깜빡임 애니메이션 | 필수 | 낮음 |
| NPC 이름 표시 + 색상 코딩 | 필수 | 낮음 |
| 정적 포트레이트(고정 이미지) | 권장 | 낮음 |
| 기본 음성 블립(단일 사운드 변형) | 권장 | 낮음 |
| 텍스트 박스 슬라이드 인/아웃 애니메이션 | 권장 | 낮음 |

**Phase 2 추가 스펙 (알파 목표)**

| 기능 | 우선순위 | 난이도 |
| :--- | :--- | :--- |
| NPC별 고유 블립 프로파일 | 권장 | 중간 |
| 포트레이트 감정 교체 (3~5단계) | 권장 | 중간 |
| 텍스트 웨이브/셰이크 효과 | 선택 | 중간 |
| 텍스트 속도 태그 (`[slow]`/`[fast]`) | 선택 | 낮음 |
| 강조 색상 인라인 적용 | 선택 | 낮음 |

**구현 상세 — 텍스트 박스 슬라이드 애니메이션**

```
트리거: 대화 시작
애니메이션: 아래에서 위로 슬라이드 인 (Y: 720 → 540)
시간: 150ms
이징: Ease Out Cubic
트리거: 대화 종료
애니메이션: 위에서 아래로 슬라이드 아웃 (Y: 540 → 720)
시간: 120ms
이징: Ease In Cubic
```

**구현 상세 — 진행 표시자 깜빡임**

```
표시 조건: 현재 페이지 타이핑 완료 후
애니메이션: 페이드 인/아웃 루프
주기: 600ms (300ms 표시 + 300ms 숨김)
위치: 텍스트 박스 우측 하단, 대사 마지막 줄 우측 끝
```

### 3.8. 구현 난이도 평가

| 항목 | 난이도 | 이유 |
| :--- | :--- | :--- |
| 기본 타이핑 시스템 | 낮음 | PixiJS 텍스트 렌더링 + 타이머 로직으로 구현 가능 |
| 구두점 딜레이 | 낮음 | 타이핑 루프에 조건 분기 추가 |
| 포트레이트 표시 | 낮음 | 스프라이트 로드 + 좌측 고정 배치 |
| 음성 블립 | 낮음 | Howler.js로 짧은 오디오 트리거 |
| 텍스트 웨이브/셰이크 | 중간 | 글자별 위치 오프셋 계산 필요 |
| 포트레이트 감정 교체 | 중간 | 태그 파서 + 스프라이트 교체 로직 |
| NPC별 블립 프로파일 | 낮음 | JSON 데이터 파일로 파라미터 관리 |
| 에르다 행동 연동 | 중간 | 대화 이벤트와 플레이어 애니메이션 시스템 연동 |

전체 평가: Phase 1 MVP는 약 2~3일 작업량. 블립 사운드와 포트레이트 에셋이 준비된다면 Phase 2 스펙까지 5~7일 내 구현 가능.

## 4. 핵심 요약

### 반드시 해야 하는 것 (Must Have)

1. 타이핑 효과 + 구두점 딜레이 — 읽기 리듬의 기반
2. 진행 표시자(▼) — 플레이어 불확실성 제거
3. NPC 이름 표기 + 색상 코딩 — 화자 즉각 식별
4. 텍스트 박스 슬라이드 인/아웃 — 폴리쉬의 최소 기준
5. 에르다의 대사 박스 없음 — 침묵 주인공 철학 관철

### 하면 훨씬 좋아지는 것 (Should Have)

6. 캐릭터별 음성 블립 — 감정 전달력과 캐릭터 개성의 핵심
7. 정적 포트레이트 — 화자 시각화
8. 메가스트럭처 미학 테두리 — 세계관 통합성

### 여유가 생기면 하는 것 (Nice to Have)

9. 포트레이트 감정 교체 (3단계 이상)
10. 텍스트 웨이브/셰이크 효과
11. 에르다의 반응 행동 애니메이션 연동
