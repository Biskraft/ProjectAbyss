## 2. 레퍼런스 게임별 스킬 시스템 분석

### 2-1. 캐슬바니아: 월하의 야상곡 (SotN)

| 항목 | 내용 |
|:-----|:-----|
| **스킬 획득** | 마법 주문은 입력 커맨드만 알면 언제든 사용 가능. 서브웨폰은 맵에서 획득 |
| **슬롯 구조** | 서브웨폰 1개 + 마법 주문은 커맨드로 무제한 접근 |
| **트리 구조** | 없음. 플레이어 지식 기반 (커맨드를 아는가 모르는가) |
| **시너지** | 서브웨폰 + Item Crash = 대량 하트 소비로 강화 버전 발동 |
| **리소스** | 서브웨폰=하트, 마법=MP. 이중 리소스 |
| **무기 연동** | 무기 종류(검/봉/너클 등)에 따라 기본 공격 모션/사거리 완전히 다름. 마법은 무기 독립 |
| **스탯 스케일링** | INT=마법 위력, STR=물리 위력 |

**핵심 교훈:** "상시 접근 가능한 마법 + 교체 가능한 서브웨폰"이라는 이중 구조. 스킬 자체보다 사용 타이밍과 상황 판단이 깊이를 만든다.

### 2-2. Dead Cells

| 항목 | 내용 |
|:-----|:-----|
| **스킬 획득** | 런 중 랜덤 드랍/상점. 뮤테이션은 스테이지 사이 NPC에서 선택 |
| **슬롯 구조** | 무기 2슬롯 + 스킬 2슬롯 + 뮤테이션 3슬롯 = 총 7개 활성 요소 |
| **트리 구조** | 뮤테이션 3색 분기: Brutality 12 / Tactics 12 / Survival 12 / Colorless 20 = 총 56개 |
| **시너지** | 뮤테이션이 스탯 색상(빨강/보라/초록)에 스케일링. 같은 색 집중 시 강력한 시너지 |
| **리소스** | 쿨다운 기반. 뮤테이션으로 쿨다운 단축 가능 |
| **무기 연동** | 무기마다 고유 스케일링 색상. 근접=Brutality, 원거리=Tactics, 방어=Survival |
| **스탯 스케일링** | 3색 스크롤이 모든 스킬/뮤테이션 위력에 영향 |

**핵심 교훈:** "56개 중 3개만 선택"이 만드는 긴장감. 리스펙 비용 점진적 증가(1000→2000→4000→8000 상한)로 "가벼운 실험 허용, 남용 억제".

### 2-3. Hollow Knight

| 항목 | 내용 |
|:-----|:-----|
| **스킬 획득** | 맵 탐험 발견 + NPC 상점 구매. 능력은 보스/이벤트 해금 |
| **슬롯 구조** | 참(Charm) 노치 시스템: 시작 3노치 → 최대 11노치 |
| **트리 구조** | 없음. 42개 참 중 노치 한도 내 자유 조합 |
| **시너지** | 숨겨진 참 시너지: 특정 조합 시 기존 효과 변형 |
| **리소스** | 소울(SOUL) = 적 타격 시 충전. 스펠과 힐링에 공유 |
| **무기 연동** | 네일(단일 무기) 중심. 참으로 네일 범위/속도/피해 변경 |
| **스탯 스케일링** | 수치 스케일링 없음. 참 배율 상한 x2 |

**참 시스템 상세:**
- 42개 참, 노치 비용 분포: 1노치(10개), 2노치(16개), 3노치(10개), 4노치(4개)
- 전체 합계 88노치, 최대 장착 11노치 → 전체의 약 **12.5%만 동시 장착**
- 10개 카테고리: 소울 획득, 스펠, 소환, 힐링, 체력, 이동, 방어, 공격, 특수공격, 유틸리티

**핵심 교훈:** "노치 = 예산" 시스템. 강력한 참은 비용이 높아 나머지 빌드를 압박. 수치 배율을 보수적으로(x2 상한) 설정하여 전투 난이도를 보존.

### 2-4. Hades

| 항목 | 내용 |
|:-----|:-----|
| **스킬 획득** | 런 중 방 클리어 보상으로 올림포스 신이 3개 선택지 제공 |
| **슬롯 구조** | 무기 1개 + 부운 부착 위치 5개 (Attack, Special, Cast, Dash, Call) |
| **트리 구조** | 없음. 런 내 선형 축적 + 신별 카테고리 분기 |
| **시너지** | **듀오 부운:** 2개 신의 부운이 결합된 시너지 스킬 → 런의 최종 빌드 결정 |
| **리소스** | 쿨다운(Cast, Special) + 체력 리소스(Death Defiance) |
| **무기 연동** | 6종 무기 x 4 Aspect = 24종 기본 플레이스타일. 부운이 무기 공격에 직접 부착 |
| **스탯 스케일링** | 부운 레어리티(Common~Legendary)로 효과 수치 결정 |

**부운 시스템 상세:**
- 8명 올림포스 신 각각 테마 특화: 제우스(번개/연쇄), 포세이돈(넉백), 아테나(반사), 아레스(출혈), 아르테미스(크리티컬), 아프로디테(약화), 디오니소스(중독), 데메테르(빙결)
- Twin Fists + 제우스 Attack 부운 = 공속 x 고정피해 = 폭발적 시너지

**핵심 교훈:** "신 = 스킬 카테고리"라는 명확한 테마 분류. 부운이 기존 액션에 "부착"되어 슬롯 추가 없이 빌드 다양성 확보. **이노센트가 유사한 역할 가능** — 이노센트가 스킬에 원소/효과를 부여.

### 2-5. Path of Exile

| 항목 | 내용 |
|:-----|:-----|
| **스킬 획득** | 스킬 젬 드랍/퀘스트 보상. 장비 소켓에 장착 |
| **슬롯 구조** | 액티브 젬 274종 + 서포트 젬 177종 = 총 451종. 장비 소켓 6링크 핵심 |
| **트리 구조** | 패시브 스킬 트리 1,325+ 노드 거대 웹 (모든 클래스 공유, 시작점만 다름) |
| **시너지** | 액티브 젬 + 서포트 젬 링크 = 스킬 커스터마이즈 |
| **리소스** | 마나 + 라이프(Blood Magic). 쿨다운은 일부만 |
| **무기 연동** | 무기 소켓 수/색상이 빌드 결정 |
| **스탯 스케일링** | STR/DEX/INT가 서포트 젬 슬롯 개방에 영향 (PoE2) |

**PoE2 설계 변화:** 서포트 젬이 스킬 젬 자체에 내장 슬롯으로 이동. 장비 의존도 감소, 각 스킬이 독립적 "미니 빌드"가 되는 구조.

**핵심 교훈:** "젬 + 서포트 = 스킬 커스터마이즈"는 극한의 빌드 다양성. 다만 접근성이 극도로 낮음. **이노센트(서포트 젬) + 스킬(액티브 젬)의 단순화 버전**으로 적용 가능.

### 2-6. 메이플스토리

| 항목 | 내용 |
|:-----|:-----|
| **스킬 획득** | 전직(Job Advancement)으로 스킬 세트 해금 (Lv10→30→60→100→200) |
| **슬롯 구조** | PC: 키보드 전체 핫키(30개+). 모바일: 6~8개 제한 |
| **트리 구조** | 전직별 선형 + 하이퍼스킬(Lv140) + V매트릭스(Lv200, 노드 30슬롯) |
| **시너지** | V매트릭스 부스트 노드가 기존 1~4차 스킬 강화 |
| **리소스** | MP 소비 + 쿨다운. 버프는 지속시간 기반 |
| **무기 연동** | 직업별 전용 무기. 무기 = 직업 아이덴티티 |
| **스탯 스케일링** | 주스탯이 공격력에 직접 환산 |

**V매트릭스 시스템:**
- Lv200 개방. 노드스톤에서 노드 획득
- 슬롯 12개(Lv200) → 30개(Lv260)
- 부스트 노드: 기존 스킬 Final Damage 증가 (최대 Lv60)

**핵심 교훈:** "전직 = 스킬 세트 갱신"으로 성장 체감 극대화. Project Abyss는 전직 없으므로, **무기 교체가 전직 역할**을 대체해야 한다.

### 2-7. 던전앤파이터 (DFO)

| 항목 | 내용 |
|:-----|:-----|
| **스킬 획득** | 레벨업 해금 + SP(스킬 포인트) 투자로 레벨업 |
| **슬롯 구조** | 퀵슬롯 12개 + 커맨드 입력 보조 (방향키+키 조합) |
| **트리 구조** | 직업별 선형. Lv15/20/25/30... 단위로 새 스킬 해금 |
| **시너지** | 스킬 진화(Skill Evolution): 후반부에서 기존 스킬 기능 자체 변경 |
| **리소스** | MP 소비 + 쿨다운. 고레벨 스킬 = 긴 쿨다운 + 높은 MP |
| **무기 연동** | 무기 종류별 특정 스킬 효과/범위 변화 |
| **스탯 스케일링** | 물리/마법 독립 ATK. 스킬별 물리/마법 비율 다름 |

**핵심 교훈:** "SP 부족"이 빌드를 결정한다 — 모든 스킬을 최대로 올릴 수 없어 주력 스킬 선택이 필수. 4슬롯 제한인 Project Abyss에서는 "슬롯 외 패시브 스킬"로 깊이를 보완해야 한다.

## 3. 스킬 시스템 설계 패턴 분류

### 3-1. 아키텍처 비교

| 패턴 | 대표 게임 | 장점 | 단점 | 적합도 |
|:-----|:---------|:-----|:-----|:------|
| **스킬 트리** (분기형) | 디아블로4, PoE | 전문화 감각, 장기 목표 | 리스펙 문제, 선택 마비 | 중 |
| **스킬 슬롯** (장착형) | Dead Cells, DFO | 빌드 실험 자유, 상황 대응 | 깊이 부족 가능 | **높음** |
| **참/배지** (예산형) | Hollow Knight, Paper Mario | 비용 관리 재미, 트레이드오프 | UI 복잡 | 중~높음 |
| **부운/축적형** | Hades | 런마다 새 경험 | 영구 성장감 부족 | 낮음 (아이템계 한정) |
| **젬 소켓** (조합형) | PoE | 극한 커스터마이즈 | 진입장벽 극높 | 낮음 |
| **전직** (세트 교체형) | 메이플, DFO | 성장 체감 극대화 | 단일 캐릭터 부적합 | 낮음 (무기로 대체) |

### 3-2. 스킬 카테고리 분류 체계

| 카테고리 | 역할 | Project Abyss 예시 | 빌드 역할 |
|:---------|:-----|:-----------------|:---------|
| **공격 (Offensive)** | 직접 피해 | 검기 방출, 파이어볼, 메테오 | 딜러 핵심 |
| **방어 (Defensive)** | 생존, 피해 감소 | 보호막, 흡혈 오라, 도발 | 탱커 핵심 |
| **유틸리티 (Utility)** | 상태이상, 버프/디버프 | 빙결, 공격력 증가, 속도 감소 | 서포터 핵심 |
| **이동 (Mobility)** | 위치 이동, 회피 | 순간이동, 돌진, 공중 대시 | 전 역할 보조 |

### 3-3. 세분화 (기존 전투 액션 문서 기반)

| 하위 카테고리 | 시전 이동 | 쿨다운 범위 | 위험-보상 |
|:-------------|:---------|:----------|:---------|
| 근접 액티브 | free | 3~6초 | 중 (사거리 진입 필요) |
| 원거리 액티브 | free/slow | 5~10초 | 낮~중 |
| 범위 액티브 | slow/lock | 8~15초 | 높 (이동 불가, 피격 위험) |
| 버프 액티브 | free | 10~15초 | 낮 (직접 피해 없음) |
| 소환 액티브 | free | 12~15초 | 낮~중 |

## 4. 핵심 설계 원칙

### 4-1. 의미 있는 스킬 트리의 7가지 열쇠

1. **선택에 의한 헌신(Commitment):** 되돌리기 어려운 선택을 강제
2. **플레이어 역량 강화(Empowerment):** 자기 표현과 자기 투영을 장려
3. **게임플레이 변화(Gameplay Enhancement):** 수치 인플레이션이 아닌 행동 변화를 만드는 스킬
   - 나쁜 예: +5% 경험치
   - 괜찮은 예: 엘리트 적 암살 가능 (제한 해제)
   - 좋은 예: 언데드 소환 (새 메커닉 도입)
4. **선택 마비 방지:** 너무 많지도 적지도 않은 선택지
5. **리스펙 허용:** 충분한 비용으로 전체 리셋. 실수 교정은 허용하되 헌신 유지
6. **진행 게이팅:** 스토리 중에는 전문화 강제, 스토리 후에는 완성주의자를 위해 전체 해금
7. **혁신:** 랜덤 트리, 부정적 효과 스킬, 양면적 메커닉 등 미탐험 영역

### 4-2. "선택의 환상" 문제와 해결책

**문제:** 과도한 밸런싱이 개성을 거세한다. 스프레드시트가 지배하면 어떤 선택도 의미 없어진다.

| 해결 패턴 | 설명 | 게임 예시 |
|:---------|:-----|:---------|
| **비대칭 설계** | 빌드별 명확히 다른 강점/약점 | FFT 직업 시스템 |
| **상호배타적 선택** | A를 고르면 B를 포기 | Hollow Knight: 11노치로 42참 중 12% |
| **환경 반응 차이** | 같은 스킬이 상황에 따라 가치 극변 | Hades: 보스별 유리한 부운이 다름 |
| **시너지 보상** | 특정 조합이 합보다 큰 결과 | Dead Cells: 같은 색 집중 시 급증 |

### 4-3. 쿨다운/리소스 비용 모델 비교

| 모델 | 대표 게임 | 특성 | 적합도 |
|:-----|:---------|:-----|:------|
| **MP + 쿨다운** (이중) | SotN, DFO | 이중 제약으로 남발 억제 | **현재 설계와 일치** |
| **쿨다운만** | Dead Cells | 리소스 관리 부담 감소 | 아이템계 MP회복 불가와 충돌 |
| **소울/히트 충전** | Hollow Knight | 적 타격으로 충전 → 스킬 or 힐 선택 | 공격-보상 루프 강화에 적합 |
| **하트** (소모성) | Classic Castlevania | 수집 리소스 = 탄약 | 아이템계 포션 경제와 유사 |

> **제안:** 현재 MP + 쿨다운 이중 시스템 유지. 추가로 "스킬 히트 시 MP 일부 회복"을 고려하면, 공격적 플레이가 리소스를 생성하는 선순환이 만들어진다.

## 5. 멀티플레이와 스킬 설계

### 5-1. 역할 분화 ("니치 보호" 원칙)

각 플레이어가 파티에서 고유한 역할을 가져야 자신의 존재 가치를 느낀다.

| 역할 | 스킬 빌드 방향 | 주력 스탯 | 파티 기여 |
|:-----|:-------------|:---------|:---------|
| 탱커 | 도발, 보호막, 패링 특화 | VIT, DEF | 어그로 관리, 피해 흡수 |
| 물리 딜러 | 근접 액티브, 넉백, 에어 콤보 | STR, DEX | 단일 타겟 고DPS |
| 마법 딜러 | 원거리/범위 액티브 | INT | AoE 정리, 원소 시너지 시동 |
| 서포터 | 버프 액티브, 디버프, 힐 | INT, LCK | 파티 강화, 적 약화 |

### 5-2. 빌드 다양성 확보

| 레이어 | 역할 | 메커닉 |
|:------|:-----|:------|
| **무기** | 기본 아이덴티티 | 8종 무기가 기본 공격 패턴 결정 |
| **스킬 4슬롯** | 전술적 선택 | 수십 종 스킬 중 4개 선택 = 빌드 핵심 |
| **이노센트** | 스킬 수정자 | 이노센트가 스킬에 원소 부여/효과 변경 (PoE 서포트 젬 역할) |
| **스탯 분배** | 장기 방향 | 6대 스탯 중 어디에 집중하느냐가 게이트 해금 순서와 빌드 결정 |

## 7. Project Abyss 스킬 시스템 제안

### 7-1. 스킬 트리 구조 제안: "무기별 분기 + 공용 트리" 하이브리드

```
[공용 트리]                        [무기별 트리] (8종)
   │                                  │
   ├── 유틸리티 분기                   ├── 검 전용 스킬 분기
   │   ├── 이동 스킬                  ├── 창 전용 스킬 분기
   │   ├── 생존 스킬                  ├── 도끼 전용 스킬 분기
   │   └── 탐험 스킬                  ├── 채찍 전용 스킬 분기
   │                                  ├── 지팡이 전용 스킬 분기
   └── 원소 분기                      ├── 너클 전용 스킬 분기
       ├── 화염 계열                  ├── 활 전용 스킬 분기
       ├── 빙결 계열                  └── 대검 전용 스킬 분기
       ├── 번개 계열
       └── 암흑 계열
```

| 설계 결정 | 근거 | 레퍼런스 |
|:---------|:-----|:---------|
| 무기별 분기 | 무기 교체 = 사실상 전직. 8종 = 8개 빌드 방향 | 메이플(전직), DFO(직업) |
| 공용 트리 | 무기 교체 시에도 유지 = 캐릭터 아이덴티티 | PoE(패시브 트리) |
| 원소 분기 | 이미 정의된 원소 시너지와 직결 | Hades(신별 테마) |
| SP 부족 설계 | 모든 스킬 최대 불가 = 선택 강제 | DFO(SP 부족) |

### 7-2. 스킬 획득 경로

| 획득 경로 | 스킬 유형 | 3-Space | 설계 의도 |
|:---------|:---------|:--------|:---------|
| 레벨업 SP | 무기별 트리 + 공용 트리 기본 스킬 | World | 성장 체감, 매 레벨 새 스킬 접근 |
| 월드 탐험 발견 | 능력 게이트 스킬 (이단점프, 벽타기 등) | World | 메트로베니아 핵심 |
| 아이템계 보상 | 스킬 강화 소재, 스킬 변이 아이템 | Item World | 야리코미 동기 |
| 보스 처치 | 희귀 스킬 스크롤 (특수 스킬 해금) | World + IW | 목표 부여, 가변비율 보상 |
| NPC 교환 | 스킬 서적 (공용 트리 확장) | Hub | 허브 경제 참여 유인 |

### 7-3. 4슬롯의 깊이 확보 방안

| 방안 | 영감 원천 | 구현 방식 |
|:-----|:---------|:---------|
| **스킬 레벨** | DFO SP 투자 | 같은 스킬이 레벨에 따라 효과/범위/피해 상승 |
| **이노센트 스킬 수정** | PoE 서포트 젬 | 이노센트가 스킬에 원소/효과 부여 |
| **무기별 스킬 전환** | 메이플 전직 | 무기 교체 시 장착 가능한 스킬 풀 변경 |
| **패시브 스킬 트리** | PoE 패시브 | 4슬롯 외에 상시 적용 패시브 |
| **시너지 콤보** | Hades 듀오 부운 | 특정 스킬 2개 동시 장착 시 숨겨진 시너지 |
| **공중/지상 차별화** | 기존 설계 확장 | 같은 스킬이 공중 발동 시 다른 효과 |

### 7-4. 스킬 스케일링 공식

기존 System_Growth_Stats.md의 `FinalStat = Base + Equip + Innocent` 구조와 연동:

```
SkillDamage = SkillBase × (1 + PrimaryStat × ScalingCoeff) × SkillLevel × ElementBonus

- SkillBase:     스킬 고유 기본 피해
- PrimaryStat:   스킬별 주 스탯 (STR/INT/DEX)
- ScalingCoeff:  스킬별 스탯 반영 계수
- SkillLevel:    SP 투자로 상승 (1~10, 이후 확장)
- ElementBonus:  원소 시너지 배율 (1.0 기본, 시너지 시 1.2~1.5)
```

### 7-5. 스킬-무기 연동 매트릭스

| 무기 | 주력 스킬 유형 | 보조 스킬 유형 | 파티 역할 경향 |
|:-----|:-------------|:-------------|:-------------|
| 검 (Sword) | 근접 액티브 | 유틸리티 | 밸런스 딜러 |
| 창 (Spear) | 근접 액티브 (장거리) | 범위 액티브 | 리치 딜러 |
| 도끼 (Axe) | 범위 액티브 | 버프 (슈퍼아머) | 탱크 딜러 |
| 채찍 (Whip) | 원거리 액티브 | 유틸리티 (디버프) | 컨트롤러 |
| 지팡이 (Staff) | 원거리/범위 액티브 | 버프 | 마법 딜러/서포터 |
| 너클 (Knuckle) | 근접 액티브 (초근접) | 이동 스킬 | 근접 버스트 |
| 활 (Bow) | 원거리 액티브 | 소환 (트랩) | 원거리 딜러 |
| 대검 (Greatsword) | 범위 액티브 | 근접 액티브 | AoE 탱크 |
