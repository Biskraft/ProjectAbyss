## 1. 아이템계 절차적 생성 메커니즘

### 1.1 기본 생성 원리

디스가이아 아이템계는 **완전 알고리즘 기반 생성**을 사용하며, 사전 제작된 Room Template이 없다. 매 층 진입 시 다음 요소가 전부 새로 생성된다.

| 생성 요소 | 랜덤 여부 | 비고 |
|:---------|:---------|:-----|
| 맵 크기 (가로/세로) | 완전 랜덤 | 소형 3×3부터 대형까지 |
| 지형 형태 | 완전 랜덤 | 구멍, 계단, 돌출 지형 등 |
| 지오 패널 배치 | 완전 랜덤 | 색상 조합과 면적 |
| 지오 심볼 배치 | 완전 랜덤 | 종류와 패널 위치 |
| 적 종류 및 배치 | 완전 랜덤 | 아이템 랭크+층수 기반 |
| 이노센트(미복종) 배치 | 완전 랜덤 | 아이템 내 이노센트 목록 기반 |
| 스킵 게이트(포탈) 위치 | 완전 랜덤 | 도달 가능하도록 보장 [추정] |
| 미스터리 게이트 유무 | 확률적 | ~10~20% 출현 |
| 레벨 구슬 유무 | 확률적 | D3 이후, 21층 이상 |

### 1.2 적 레벨 결정

```
적 레벨 ≈ f(아이템 랭크, 아이템 현재 레벨, 현재 층수)
```

아이템을 강화할수록 적 레벨도 상승 → 자연스러운 난이도 스케일링

### 1.3 적 배치 규칙

| 구분 | 규칙 |
|:-----|:-----|
| 적 종류 | 아이템 유형과 연관 (레어 스태프 → 마법사 계열 증가) |
| 적 수 | 맵 크기에 비례, 랜덤 |
| 이노센트(비복종) | 아이템 내 이노센트 수에 따라 유닛 등장 |
| 해적 | 2턴 이상 경과 시 확률적 침입 |
| 보스 | 10층마다 전용 배치 |

### 1.4 클리어 조건의 이원화

| 방법 | 설명 | 보상 차이 |
|:-----|:-----|:---------|
| 포탈 도달 | 아군 1명이라도 포탈 위치 이동 | 보너스 게이지 부분 충전 |
| 적 전멸 | 맵 내 모든 적 처치 | 보너스 게이지 최대 충전 |

"빠른 층 클리어 vs 완전 클리어" 사이의 플레이어 선택 구조.

## 2. 경로 보장 (Critical Path)

### 2.1 디스가이아의 보장 방식

- 맵 생성 후 스킵 게이트까지의 경로 검증 실행 [추정]
- No Entry 패널이 전체 맵을 덮는 극단적 상황은 방지하지 못하는 경우 존재
- **"높은 확률의 보장"이지 완벽한 보장은 아님**

### 2.2 스펠렁키와의 비교

| 항목 | 디스가이아 | 스펠렁키 |
|:-----|:---------|:--------|
| 보장 원칙 | 높은 확률 | **수학적 완전 보장** |
| 생성 방식 | 완전 알고리즘 후 검증 | Critical Path 먼저 확정 후 나머지 채움 |
| 실패 처리 | 예외 상황 발생 가능 | 검증 실패 시 레벨 전체 재생성 |

## 3. 스펠렁키 절차적 생성 분석

### 3.1 핵심 접근: "Always Winnable"

```
Phase 1: 레이아웃 생성
  4×4 Room Grid에서 Critical Path 먼저 확정
  좌-우-하 이동으로 최상단 → 최하단 경로 생성
  경로 Room에 Type 1(좌우), Type 2(하강), Type 3(상승) 배정
  비경로 Room은 Type 0(막힘)

Phase 2: 장애물 배치
  5×3 타일 Chunk를 Derek Yu가 수동 설계
  Critical Path Room에는 반드시 통과 가능한 Chunk만 배치

Phase 3: 엔티티 배치
  World Config별 몬스터 풀 기반 랜덤 배치
```

**핵심:** 어떤 시드에서도 클리어 가능한 경로가 수학적으로 보장. "실패 = 플레이어 탓"이라는 명확한 책임 귀속.

## 4. 데드셀 절차적 생성 분석

### 4.1 핵심 접근: Concept Graph + Room Pool

```
Concept Graph 예시 (Prisoners' Quarters):
[Start] → [Combat A] → [Exploration] → [Combat B] → [Fork]
                                                      ├→ [Combat C] → [Treasure] → [Exit A]
                                                      └→ [Challenge] → [Exit B]

Room 유형 분류:
  Combat (40~60%): 전투 중심
  Exploration (15~25%): 플랫포밍/탐색
  Treasure (5~10%): 보상
  Secret (5~8%): Rune 게이팅
  Lore (2~5%): NPC/스토리

몬스터 밀도 공식:
  total_monsters = combat_tile_length / density_ratio
  BC(Boss Cell) 높을수록 밀도 증가
```

**핵심:** "Every room feels handmade, but every run is different." 바이옴당 50~100+ 핸드메이드 룸을 de-duplication 알고리즘으로 배치.

## 5. 세 시스템 비교 분석

| 항목 | 디스가이아 | 스펠렁키 | 데드셀 |
|:-----|:---------|:--------|:------|
| 생성 방식 | 완전 알고리즘 | Template 조립 | Concept Graph + Room Pool |
| Critical Path | 사후 검증 (약한 보장) | 사전 확정 (수학적 보장) | Graph 내재 (바이옴 보장) |
| 맵 크기 | 완전 랜덤 | 고정 (4×4 Grid) | 바이옴별 Fixed + 가변 |
| 핸드크래프트 | 없음 | Room Template + Chunk | 바이옴당 50~100+ 프리팹 |
| 전략 레이어 | 지오 패널/심볼 | 함정/장애물 | Rune 게이트 |
| 반복성 방지 | 완전 랜덤 (무한 조합) | Chunk 조합 (유한하지만 다양) | de-duplication 알고리즘 |

## 6. 디스가이아 아이템계의 고유 차별점

### 6.1 콘텐츠 컨테이너로서의 레벨

스펠렁키/데드셀의 레벨은 "통과해야 하는 공간"이지만, 아이템계 층은 **"강화할 아이템 자체의 내부"**. 탐험 대상 = 성장 대상이라는 동일성이 심리적 투자를 극대화.

### 6.2 전장 규칙의 물리적 조작

지오 패널/심볼은 플레이어가 심볼을 이동/파괴하여 **전장 규칙 자체를 실시간 재배치**. 스펠렁키의 함정(일방적 위협)이나 데드셀의 Rune 게이트(경로 결정)와 근본적으로 다름.

### 6.3 비소모성 축적

스펠렁키/데드셀은 런 단위(사망 시 맵 초기화). 아이템계는 층별 영구 강화 + 죽어도 보스 처치 보너스 보존. 이 비소모성이 수백 시간 리텐션의 핵심.

### 6.4 세션 자율성

스펠렁키는 클리어 or 사망 강제 종료. 아이템계는 **10층 단위로 언제든 탈출 가능**. "오늘은 30층만" 식의 세션 자율 조절.

## 7. 시리즈 진화가 보여주는 레벨 디자인 트렌드

```
D1 (2003): 단순 랜덤 맵 + 고정 규칙
     ↓
D2 (2006): 미스터리 룸 + 해적 → "예측 불가 서프라이즈 삽입"
     ↓
D3 (2008): 역해적 + 레벨 구슬 → "플레이어가 침입자가 되는 역전"
     ↓
D4 (2011): 분기 루트 → "동일 아이템의 다중 진행 경로"
     ↓
D5 (2015): 무한 층 + 보너스 스테이지 → "상한 철폐 + 이벤트 삽입"
     ↓
D6 (2021): 자동 탐험 → "방치형 파밍"
     ↓
D7 (2023): 30층 + 전생 반복 → "짧은 루프 + 무한 반복"
```

트렌드: **"탐험 공간의 물리적 설계" → "세션 설계와 루프 구조 최적화"**

## 8. ECHORIS 기억의 지층 설계 권장사항

### 8.1 생성 방식 비교 및 권장

| 접근법 | 장점 | 단점 | 적합성 |
|:------|:-----|:-----|:------|
| 디스가이아식 완전 랜덤 | 무한 조합, 구현 간단 | 품질 불균형 | 가능하나 품질 제어 어려움 |
| 스펠렁키식 Template + Critical Path | Always Winnable, 디자이너 의도 반영 | 제한된 조합 수 | 횡스크롤에 가장 적합 |
| 데드셀식 Concept Graph + Room Pool | 바이옴별 유니크 질감, 핸드메이드 느낌 | 구현 복잡도 높음 | 지층별 테마 차별화에 이상적 |

**권장:** 데드셀식 + 스펠렁키식 하이브리드
- 지층별 테마에 맞는 Concept Graph와 Room Pool → 체감 차이
- Critical Path 사전 확정 → Always Winnable 수학적 보장

### 8.2 핵심 설계 원칙

1. **층 내 이원적 클리어 조건:** "출구 통과(빠른)" vs "이노센트 전원 복종(완전)" 선택 구조
2. **10층 체크포인트 리듬:** 긴장→결정점→이완의 페이싱 패턴
3. **미스터리 룸 서프라이즈:** "항상 이득" X → "기대와 위험의 혼합"
4. **해적 = 계획되지 않은 침입자:** "빠르게 움직여야 하는 압박" → "지층 침입자" 개념
5. **환경 효과 구역:** 지오 패널의 횡스크롤 대응물 (중력 역전, 속성 변환 등)
