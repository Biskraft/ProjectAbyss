## 1. 방 구성 (Room Composition)

### 1-1. 레퍼런스별 방 치수

#### Symphony of the Night (1997, Konami)
- **타일 크기:** 16x16 픽셀
- **최소 방 크기:** 256x256 픽셀 (16x16 타일)
- **화면 해상도:** 256x208 픽셀 (플레이 영역 기준)
- **방 구조:** 고정 크기 없음. 타일 크기의 배수이면 어떤 치수든 가능. 좁은 복도(2~3타일 높이)부터 거대한 홀(20타일 이상)까지 혼용.
- **특징:** 룸간 스크롤링 전환이 없고, 자유 스크롤 방식. 하나의 "방"이 수십 화면에 걸쳐 이어진다. 방 경계는 플레이어가 인식하기 어렵고, 지도 시스템을 통해 간접 제공.

#### Hollow Knight (2017, Team Cherry)
- **타일 크기:** 16x16 픽셀 (추정, 스프라이트 기준)
- **방 구조:** 단일 화면(Celeste식)이 아닌 자유 스크롤. 방 크기 편차가 극단적.
  - 좁은 통로형: 가로 5~8타일
  - 전투 아레나형: 가로 30~50타일, 세로 20~30타일
  - 수직 탐강형: 세로 60타일 이상
- **특징:** 배경 레이어와 전경 레이어 분리. 배경에서 적의 존재를 미리 암시. 거대한 방 크기가 탐험의 광활감을 만든다.

#### Super Metroid (1994, Nintendo)
- **타일 크기:** 16x16 픽셀
- **방 치수:** 1~12 화면 분량. 가로 확장이 주. 고정 스크롤 화면 단위.
- **방 유형:**
  - 단화면 방: 256x192 픽셀 (1 화면)
  - 다화면 방: 최대 약 3584x192 픽셀 (14 화면 가로 확장)
  - 수직 방: 256x960 픽셀 이상 (5화면 수직)
- **특징:** 화면 스크롤이 명확하게 "다음 공간"을 알려줌. 방 전환 시 도어 애니메이션으로 경계를 인식시킴.

#### Celeste (2018, Maddy Makes Games)
- **타일 크기:** 8x8 픽셀
- **방 크기:** 320x180 픽셀 (40x22.5 타일) — 단일 화면, 고정
- **특징:** 모든 방이 동일한 단일 화면. 화면 전환 = 방 전환. 이 제약이 방 단위 퍼즐 디자인을 가능하게 함.

#### Mega Man X (1993, Capcom)
- **타일 크기:** 16x16 픽셀
- **화면:** 256x224 픽셀 (가로 스크롤 고정)
- **방 구조:** 엄밀한 "방" 개념 없음. 가로 스크롤 연속 구조. 체크포인트와 보스 방이 구조적 분절을 만듦.
- **특징:** 스테이지가 "도입부 → 중간 미니 챌린지 → 중간 보스 → 클라이맥스 → 보스방"의 단계적 구조를 따름.

#### Dead Cells (2018, Motion Twin)
- **타일 크기:** 16x16 픽셀
- **Room 크기:** 바이옴별 핸드메이드 타일(Room Template). 가로 40~120타일, 세로 20~60타일 범위 추정.
- **특징:** 방이 Concept Graph에 의해 조립됨. 각 방은 역할(전투/전이/보상/휴식)이 명확히 분리되어 있음.

#### Spelunky (2012, Mossmouth)
- **타일 크기:** 32x32 픽셀
- **방(Room) 크기:** 10x8 타일 (320x256 픽셀) — 고정
- **레벨 구조:** 4x4 방 그리드 = 40x32 타일 레벨
- **Chunk 크기:** 5x3 타일 (160x96 픽셀) — 방 내부 장애물 단위

### 1-2. 방 유형 분류

메트로베니아와 플랫포머에서 반복적으로 등장하는 방 유형은 5가지로 분류된다.

| 방 유형 | 역할 | 디자인 특성 | 대표 게임 |
| :--- | :--- | :--- | :--- |
| **전투 아레나 (Combat Arena)** | 전투 경험 제공 | 적 배치 중심, 진입/탈출 동선 명확, 높이와 폭이 전투 허용 범위 확보 | Hollow Knight, Super Metroid |
| **이동 챌린지 (Traversal Challenge)** | 플랫폼 기술 측정 | 이동 능력을 정밀하게 요구하는 장애물 배치, 낙하 패널티 존재 | Celeste, Mega Man X |
| **퍼즐 방 (Puzzle Room)** | 사고력 요구 | 스위치/환경 요소 상호작용, 단일 해법 또는 복수 해법 | SotN, Super Metroid |
| **휴식 방 (Rest Room)** | 긴장 해소, 리소스 보충 | 적 없음 또는 약한 적, 세이브 포인트/상점/힐링 요소 | 모든 게임 |
| **전이 통로 (Transition Corridor)** | 공간 간 연결 | 짧고 직선적, 분위기 전환 역할, 장치 없음 | SotN, Super Metroid |

### 1-3. 기쇼텐케츠 구조 (起承転結) — 방 내 4단계 흐름

닌텐도 레벨 디자인 철학에서 파생된 방 구성 원칙. 갈등 없이 대비(Contrast)와 반전(Surprise)으로 흥미를 만든다.

| 단계 | 일본어 | 역할 | 게임 예시 |
| :--- | :--- | :--- | :--- |
| **기 (起, Ki)** | 도입 | 새로운 요소 소개. 안전한 환경에서 메커닉 첫 등장 | 발판 1개, 적 1마리를 격리된 공간에 배치 |
| **승 (承, Sho)** | 발전 | 같은 요소의 조합과 응용. 위험도 점진적 상승 | 같은 메커닉 + 소규모 변형 2~3회 반복 |
| **전 (転, Ten)** | 반전 | 예상 밖의 요소 추가. 새로운 도전 맥락 | 기존 메커닉의 "트위스트" — 이동 플랫폼, 적 조합 |
| **결 (結, Ketsu)** | 해소 | 반전 요소의 마스터리 요구 및 해소 | 학습한 모든 것을 통합한 최종 도전 |

**Celeste 적용 예시:**
- Ki: 대시 아이템(크리스탈)이 혼자 배치됨. 플레이어가 시도하고 학습.
- Sho: 같은 크리스탈이 2~3개 연속 배치. 연속 대시 연습.
- Ten: 이동 방향이 꺾이는 크리스탈 배치 + 가시 장애물 추가.
- Ketsu: 이 방에서 학습한 대시 타이밍을 요구하는 정밀 구간으로 마무리.

## 2. 적 배치 (Enemy Placement)

### 2-1. 적 배치의 핵심 원칙

#### 원칙 1 — 격리 후 조합 (Isolation Before Combination)

새로운 적 유형은 반드시 단독으로 먼저 소개된 후에 조합 배치된다.

- **Super Metroid:** 각 구역에서 새로운 적이 처음 등장할 때는 열린 공간, 충분한 반응 시간, 다른 적 없는 환경에서 등장.
- **Hollow Knight:** 새로운 적 유형이 등장하는 방은 대부분 다른 적 없이 단독으로, 아레나 구조에서 소개됨.
- **Mega Man X:** 각 스테이지의 첫 번째 적 등장은 플레이어가 대처법을 발견하기 충분한 공간과 시간을 제공.

#### 원칙 2 — 복합 배치의 시너지 설계

단순 숫자 추가가 아닌, 서로 다른 적 특성이 **상호보완적 압박**을 만들어야 한다.

**효과적인 조합 패턴:**
- **원거리 + 근거리:** 원거리 적이 회피를 강제하고, 근거리 적이 도망갈 공간을 좁힘.
- **지상 + 공중:** 지면 이동을 방해하는 지상 적 + 공중에서 아래로 공격하는 적이 입체적 압박 생성.
- **이동형 + 고정형:** 쫓아오는 적이 플레이어를 고정형 위험물(함정, 가시) 쪽으로 유도.
- **Explosive Blob + Aiming Blob (Hollow Knight 유사 사례):** Aiming Blob은 숨는 전략을 유도하고, Explosive Blob은 숨을 공간을 없애 — 두 전략이 서로 모순됨. 이 모순이 창의적 해결을 요구.

#### 원칙 3 — 공정한 배치 (Fair Placement)

"공정성"은 플레이어에게 반응할 수 있는 충분한 정보와 시간을 제공하는 것이다.

| 공정한 배치 | 불공정한 배치 |
| :--- | :--- |
| 적이 시야각 안에서 등장 | 화면 밖에서 갑자기 등장 |
| 첫 등장 전 시각/청각 단서 존재 | 전혀 예고 없이 낙하/기습 |
| 회피 가능한 공간 확보 | 회피 불가능한 코너에 배치 |
| 패턴이 학습 가능한 속도 | 반응 불가능한 속도의 투사체 |

**Super Metroid의 "투명한 손(Invisible Hand)":**
죽음의 원인이 항상 플레이어의 실수이지 게임의 불공정함이 아니어야 한다. Super Metroid는 과열 지역에서 Samus의 에너지가 감소하는 것을 미리 표시하여, 새로운 위험 유형을 예고 없이 즉사로 처리하지 않는다.

### 2-2. 방 유형별 적 밀도 기준

| 방 유형 | 권장 동시 활성 적 수 | 엘리트/미니보스 | 비고 |
| :--- | :--- | :--- | :--- |
| 전투 아레나 | 3~6마리 | 가능 | 방 크기에 비례 |
| 이동 챌린지 | 0~2마리 | 불가 | 이동 집중이 목적 |
| 퍼즐 방 | 0~1마리 | 불가 | 사고 집중이 목적 |
| 휴식 방 | 0마리 | 불가 | 긴장 해소 목적 |
| 전이 통로 | 0~2마리 | 불가 | 이동 중 단순 장애물 |

### 2-3. 수직 방 vs 수평 방 적 배치 차이

**수평 방 (Horizontal Room):**
- 이동 방향이 좌우로 명확. 적이 "통행을 방해"하는 역할.
- 지상 적은 플레이어와 같은 레인에서 압박.
- 공중 적은 이동 경로 위에 배치하여 점프를 강제.
- 시야선(Sight Line): 가로 방향으로 길어 원거리 투사체가 위협적.

**수직 방 (Vertical Room):**
- 이동 방향이 상하로 명확. 적이 "낙하를 위협"하는 역할.
- 아래에서 올라오는 적이 상승을 방해.
- 공중 부유 적이 올라가는 경로를 막음.
- 낙하 발판이 안전 구역과 위험 구역을 나눔.
- Hollow Knight의 수직 통로: 위에서 내려오는 적이 낙하 착지점을 위협.

## 3. 플랫폼 레이아웃 (Platform Layout)

### 3-1. 플랫폼 표준 수치

아래 수치는 레퍼런스 게임 분석 및 2D 플랫포머 설계 원칙에서 도출한 기준이다. 실제 구현 시 프로토타입으로 검증 필요.

| 항목 | 권장 범위 | 비고 |
| :--- | :--- | :--- |
| 발판 폭 (Platform Width) | 최소 2타일 (32px), 권장 3~6타일 | 2타일 미만은 착지 신뢰도 저하 |
| 수평 간격 (Gap Width) | 1~4타일 | 5타일 이상은 대시/이중점프 필요 |
| 수직 간격 (Step Height) | 1~3타일 | 4타일 이상은 점프 능력 필요 |
| 낙하 피트 깊이 | 최대 6타일 (낙하 페널티 없을 경우) | 즉사 피트는 4타일 이상 |
| 천장 높이 (Combat Room) | 최소 8타일 | 점프 + 공중 적 회피 공간 확보 |
| 이동 플랫폼 속도 | 1~3타일/초 | 4타일/초 이상은 타이밍 픽셀 퍼펙트 위험 |

### 3-2. 일방통행 발판 (One-Way Platform) 사용 원칙

일방통행 발판(위에서 내려올 수 있지만 아래서 올라갈 수 없는 발판)은 다음 목적으로 사용된다.

1. **층위 분리:** 아래층과 위층을 나누면서도 양방향 이동을 허용.
2. **퇴각 경로:** 위험한 적 앞에서 아래로 빠르게 이탈 가능.
3. **비밀 탐색 유도:** 발판 아래에 숨겨진 공간을 암시.
4. **경제적 공간 설계:** 적은 타일로 복잡한 이동 구조 생성.

**사용 금지 패턴:**
- 일방통행 발판이 Critical Path의 유일한 경로인 경우 → 되돌아갈 방법이 없어야 하는 경우에만 허용.
- 시각적으로 일반 발판과 구별 불가능한 경우 → 색상/패턴 차이 필수.

### 3-3. 수직 방 vs 수평 방 — 설계 규칙 차이

**수평 방 설계 원칙:**
- 방 전체를 좌에서 우로 이동할 때 평균 이동 시간: 8~15초 (적 없을 때).
- 이동의 흥미는 수직 고저 차이로 만든다 (지루한 평지 이동 금지).
- 중간 높이 발판이 전투 선택지(고지/저지 전략)를 만든다.
- 배경에서 전경으로 이어지는 깊이감이 공간감을 확장.

**수직 방 설계 원칙:**
- 상승과 하강이 서로 다른 경험이어야 한다.
  - 상승: 능동적 탐험, 더 많은 집중과 기술 요구.
  - 하강: 빠른 이동, 위험은 착지 지점에 집중.
- 중간 "쉬는 발판"을 2~3개 배치하여 긴장의 리듬을 만든다.
- 수직 방의 최상부와 최하부는 서로 다른 분위기/보상을 가져야 한다.

### 3-4. 이동 발판 (Moving Platform) 및 사라지는 발판 (Disappearing Block)

| 발판 유형 | 사용 목적 | 설계 규칙 |
| :--- | :--- | :--- |
| 이동 발판 (Moving Platform) | 이동 타이밍 도전, 플랫폼 퍼즐 | 패턴이 예측 가능해야 함. 첫 사이클에 패턴을 관찰할 수 있는 공간 필요 |
| 사라지는 발판 (Disappearing Block) | 긴장감, 기억력 도전 | Celeste: 밟으면 1~2초 후 사라짐. 재생 시간도 시각 표시 필요 |
| 낙하 발판 (Fall-away Platform) | 즉각 행동 강제 | 밟는 순간 진동 → 0.5초 후 낙하. 시각 피드백 필수 |
| 스프링 발판 (Bounce Platform) | 수직 이동 가속 | Super Metroid의 스프링볼. 예측 가능한 반사각 필수 |

## 4. 비밀 디자인 (Secret Design)

### 4-1. 비밀 설계의 3단계 구조

효과적인 비밀은 단순히 "숨겨진 것"이 아니라, 발견의 여정 자체가 보람이어야 한다.

```
단서(Clue) → 탐색(Exploration) → 발견(Discovery) → 보상(Reward)
```

**단계별 디자인 원칙:**
1. **단서 (Clue):** 시각적 이질감. 주변 벽과 다른 재질/색상/패턴. "이 벽은 부서질 것 같다"는 느낌.
2. **탐색 (Exploration):** 단서를 발견한 플레이어가 적극적으로 상호작용을 시도.
3. **발견 (Discovery):** 행동의 결과로 비밀이 개방. 청각/시각 피드백.
4. **보상 (Reward):** 비밀의 크기에 비례한 보상. 작은 비밀 → 소모품/정보. 큰 비밀 → 핵심 아이템/지름길/스토리.

### 4-2. 비밀 힌트 시스템 분석

| 게임 | 힌트 방식 | 발견 난이도 | 설명 |
| :--- | :--- | :--- | :--- |
| **Symphony of the Night** | 부서지는 벽의 시각적 재질 차이 + 벽을 향해 공격 시 효과음 | 보통 | 의도적으로 공격해봐야 알 수 있음. 경험에 의한 발견. |
| **Super Metroid** | 벽에 미세한 균열/색상 차이. 특정 아이템 획득 후 재방문 유도 | 어려움 | "The Invisible Hand" — 게임이 조용히 방향을 안내. |
| **Hollow Knight** | 배경 색과 다른 핑크색 크리스탈이 벽에 박혀 있음 | 낮음~보통 | 환경적으로 눈에 띄지만 튀지 않음. |
| **Celeste** | 대시 수정체(Crystal Heart) 주변에 반짝이는 효과. 분홍빛 광채 | 낮음 | 적극적으로 힌트를 주되 직접 설명하지 않음. |

**핵심 원칙:** 힌트는 "조용한 차이"여야 한다. 너무 명백하면 발견의 성취감이 없고, 너무 모호하면 좌절만 남는다. 힌트 시스템의 목표는 "아, 이건 다르네"라는 순간을 만드는 것이다.

### 4-3. 비밀 밀도 권장값

데이터 기반 정량적 밀도 기준은 공개된 연구가 없으므로, 레퍼런스 게임에서 역산한 추정치이다.

| 게임 | 구역당 비밀 방 수 | 총 비밀 공간 비율 | 비고 |
| :--- | :--- | :--- | :--- |
| Symphony of the Night | 2~4개 | 약 15~20% | 전체 맵의 20% 이상이 선택적 탐험 |
| Hollow Knight | 3~6개 | 약 20~25% | 노치 효과, 비밀 지역 등 밀도 높음 |
| Super Metroid | 1~3개 | 약 10~15% | 핵심 아이템이 비밀에 위치 |

**ECHORIS 권장:** 층위(Zone)당 비밀 방 3~5개. 총 탐험 가능 공간의 15~20%를 선택적 경로로 배분.

### 4-4. 위험-보상 균형 (Risk/Reward Balance)

비밀의 접근 난이도는 보상의 크기와 비례해야 한다.

| 난이도 | 비밀 접근 방법 | 보상 등급 | 예시 |
| :--- | :--- | :--- | :--- |
| **낮음** | 탐험 중 자연스럽게 발견 | 소모품, 골드 | 파괴 가능한 벽 뒤의 단순 방 |
| **보통** | 특정 능력 + 탐색 의지 | 희귀 아이템, 스탯 업그레이드 | 이중점프로만 도달 가능한 상단 비밀 |
| **높음** | 특수 능력 + 퍼즐 해결 | 핵심 장비, 스토리 단편, 지름길 | 스탯 게이트 + 환경 퍼즐 조합 |

## 5. 환경적 내러티브 (Environmental Storytelling)

### 5-1. 환경이 이야기하는 방법

환경적 내러티브는 대사/컷씬 없이 공간 자체로 이야기를 전달하는 기법이다. 플레이어가 스스로 해석하게 함으로써 능동적 참여와 발견의 기쁨을 만든다.

**3가지 레이어로 분석:**

| 레이어 | 도구 | 효과 |
| :--- | :--- | :--- |
| **직접적 단서** | 시체, 파괴된 오브젝트, 혈흔 | "여기서 무슨 일이 있었는가?"를 직접 암시 |
| **구조적 단서** | 방 배치, 문의 방향, 폐쇄된 경로 | "이 공간은 원래 어떤 용도였는가?"를 질문 |
| **분위기적 단서** | 배경 그림, 조명, 음악 전환 | "이 공간이 품고 있는 감정"을 전달 |

### 5-2. 레퍼런스 게임 분석

#### Hollow Knight — 침묵으로 말하는 내러티브

- **배경의 선예고(Foreshadowing):** Forgotten Crossroads에서 적 Goam이 지면을 부수며 등장. 배경 그림에 이미 같은 생물이 그려져 있어 위협을 예고.
- **유해(Remains)의 배치:** 바닥에 흩어진 곤충 껍데기와 해골이 과거의 전쟁을 암시. NPC가 없어도 역사가 느껴짐.
- **규모로 말하는 공간:** 방이 너무 크고 황량할 때, 그 빈 공간 자체가 "한때 여기엔 뭔가가 가득했다"를 전달.

#### Super Metroid — 환경이 곧 게임 규칙

- **함정이 세계관:** 용암, 물, 고온 지역이 단순한 장애물이 아니라 Zebes의 생태계를 나타냄.
- **적의 배치가 생태:** 같은 구역에 존재하는 적들이 먹이사슬 관계를 암시.
- **Kraid의 숨겨진 방:** 표면적으로 엘리베이터 방이지만, 벽 한 블록을 슈퍼 미사일로 파괴해야 접근 가능 → "이 지역은 고의로 숨겨져 있다"는 내러티브.

#### Symphony of the Night — 건축이 역사다

- **방의 용도가 역사:** 도서관, 왕좌실, 예배당 등 실제 성의 공간 구성이 드라큘라 성의 역사를 암시.
- **역전된 성(Inverted Castle):** 뒤집힌 성이 단순한 콘텐츠 추가가 아니라 "악몽 속의 성"이라는 상징성을 가짐.
- **좁은 복도의 긴장감:** 복도가 좁을수록 고딕 건축의 위압감이 강해짐. 플레이어가 "주인이 아닌 침입자"임을 상기시킴.

### 5-3. 환경 위험 요소의 세계관 기능

단순한 장애물을 넘어 세계관을 설명하는 위험 요소.

| 위험 요소 | 메커닉적 역할 | 내러티브 역할 |
| :--- | :--- | :--- |
| 용암/고온 구간 | 스탯 게이트 (VIT 필요) | 이 구역이 인간이 살 수 없는 영역임을 표현 |
| 붕괴하는 발판 | 이동 타이밍 도전 | 건물이 오래되고 방치되었음을 암시 |
| 침수된 방 | 수중 호흡 능력 게이트 | 과거의 재난/홍수를 암시 |
| 가시 함정 | 회피 도전 | 이 공간이 의도적으로 방어되고 있음을 표현 |
| 독 안개 | 독 저항 스탯 게이트 | 생물학적 오염 또는 마법적 타락 암시 |

## 6. 절차적 방 디자인 (Procedural Room Design)

### 6-1. Dead Cells — Concept Graph 기반 하이브리드 PCG

**핵심 구조:**
1. **Fixed Framework (고정 골격):** 바이옴의 전체 크기, 입구/출구 위치는 고정. 모든 시드에서 동일.
2. **Concept Graph (개념 그래프):** 방의 배치 순서와 분기를 정의한 유향 그래프. CastleDB(현 LDtk의 전신)로 편집.
3. **Room Template Pool (방 템플릿 풀):** 각 방 유형에 해당하는 핸드메이드 방 목록. 생성 시 그래프에 따라 무작위 선택.
4. **PCG Fill (절차적 채움):** 방 템플릿 배치 후 남은 공간을 자동으로 채움.

**방 유형 분리의 중요성:**
- 전투방, 보상방, 상인방, 전이방이 별도의 템플릿 풀을 가짐.
- 전투방이 상인방으로 무작위 대체되는 일이 불가능.
- 각 방이 역할에 최적화된 레이아웃을 가질 수 있음.

**Dead Cells 방 디자인 원칙:**
- 방 내부의 주요 전투 구간 길이에 따라 몬스터 수 결정 (Density Ratio).
- 위험 가중치 시스템: 특수 몬스터는 일반 몬스터 10배의 "위험 점수"를 보유.
- 방 시작 전 플레이어가 내부를 잠깐 볼 수 있는 "시야 제공 구간" 설계.

### 6-2. Spelunky — Chunk 기반 Template 조립

**계층 구조:**
```
레벨 (40x32 타일)
  └── 4x4 방 그리드
        └── 각 방 (10x8 타일)
              └── Room Template (출입구 유형 결정)
                    └── Chunk들 (5x3 타일) × 복수 배치
```

**Critical Path 알고리즘 (상세):**

```
1. 최상단 행에서 무작위로 시작 방 선택
2. 현재 방에서 방향 결정: 좌(25%), 우(25%), 하(50%)
3. 그리드 좌우 경계 도달 시 → 강제 하강
4. 하강 방에 Type 2 (LRD: Left-Right-Down) 배정
5. 최하단 행 도달 → 경로 완성
6. 경로 위 모든 방을 "Critical Path 방"으로 마킹
7. 비경로 방은 Type 0 (Dead End) 또는 Type 1 (LR) 배정
```

**Always Winnable 보장 메커니즘:**
- Critical Path 방의 Room Template은 항상 크리티컬 이동이 가능한 템플릿만 사용.
- Chunk는 Critical Path 방에서 통로를 막지 않도록 배치 규칙 보유.
- 검증: 생성 후 BFS/DFS로 입구→출구 경로 존재 확인.

**Chunk 설계 원칙:**
- 5x3 타일 크기가 "다루기 쉬운 최소 의미 단위"를 만든다.
- 각 Chunk는 독립적으로 의미가 있어야 하고, 인접 Chunk와 자연스럽게 연결되어야 한다.
- Derek Yu는 각 Chunk를 "하나의 작은 디자인 아이디어"로 취급: 사다리, 플랫폼 배치, 함정 조합 등.

**Spelunky 2의 이중 레이어 (Dual Layer) 확장:**
- 전경(Front Layer): 충돌 발생. 일반 플랫폼.
- 후경(Back Layer): 통과 가능. 숨겨진 경로, 일방통행.
- 이 구조가 동일 공간에서 "알려진 경로"와 "발견 가능한 경로"를 동시에 만든다.

### 6-3. 조합 가능성 설계 (Combinability Design)

절차적 생성에서 핵심은 각 구성 요소(Chunk, Template)가 어떤 조합에서도 유효한 결과를 만드는 것이다.

**Chunk 설계 체크리스트:**
- [ ] 인접 Chunk와 지형이 자연스럽게 연결되는가? (높이 차이, 벽 연결)
- [ ] 어떤 방향에서 진입해도 탈출 가능한가? (막힌 Chunk 없음)
- [ ] 이 Chunk만으로 "작은 재미"가 존재하는가? (격리 테스트)
- [ ] 극단적 조합(강한 Chunk 4개 연속)에서도 클리어 가능한가?

**Template 설계 체크리스트:**
- [ ] 방의 역할(전투/탐험/휴식)이 Chunk 배치에 반영되는가?
- [ ] 출입구(입구/출구)가 항상 접근 가능한가?
- [ ] 플레이어가 방의 구조를 이해하는 데 3초 이내인가?

## 7. ECHORIS 적용 원칙

### 7-1. 월드 (핸드크래프트) 레벨에 적용

ECHORIS의 월드는 핸드크래프트 방식으로 제작되므로, 레벨 디자이너가 위 원칙을 직접 적용한다.

**방 구성 체크리스트 (Design_Level_Standards.md 보완용):**

```yaml
room_design_checklist:
  composition:
    - 방 유형(전투/이동/퍼즐/휴식/전이) 명확히 결정
    - 기쇼텐케츠 4단계 흐름 적용 여부 확인
    - 방 크기가 의도한 경험을 지원하는가 (전투방 최소 천장 8타일)

  enemy_placement:
    - 새 적 유형은 격리 공간에서 단독 소개
    - 적 조합이 상호보완적 압박을 만드는가
    - 모든 등장이 공정한가 (시야 내 등장, 반응 시간 확보)
    - 방 유형에 맞는 밀도인가 (퍼즐/이동 방에 고밀도 금지)

  platform_layout:
    - 이동 플랫폼 패턴이 관찰 가능한가 (최소 1 사이클 여유)
    - 일방통행 발판이 시각적으로 구별 가능한가
    - 수직/수평 방 설계 규칙 적용

  secrets:
    - 비밀이 있는 경우, 단서가 "조용한 차이"로 존재하는가
    - 비밀 접근 난이도가 보상 등급에 비례하는가
    - 층위당 3~5개 비밀 공간 목표

  environmental_storytelling:
    - 방의 역할이 세계관과 일치하는가 (도서관 = 책장, 묘지 = 석관)
    - 환경 위험이 내러티브 기능을 겸하는가
    - 배경 레이어가 스토리 단서를 포함하는가
```

### 7-2. 아이템계 (절차적 생성) 방 디자인에 적용

아이템계는 Dead Cells + Spelunky의 하이브리드 방식을 채용한다.

**Chunk 설계 표준:**

현재 `System_World_ProcGen.md`에 정의된 Chunk 크기: **5x3 타일 (80x48 픽셀, 16px 타일 기준)**

아이템계 Chunk는 월드 Chunk와 다른 원칙이 적용된다.

| 항목 | 월드 Chunk | 아이템계 Chunk |
| :--- | :--- | :--- |
| 크기 | 5x3 타일 | 5x3 타일 (동일) |
| 바이옴 | 층위 고정 | 아이템 속성 연동 |
| 내러티브 기능 | 강함 | 약함 (기억의 지층이 추상적) |
| 조합 가능성 요구 | 보통 | 매우 높음 (완전 절차적) |
| 적 배치 | Chunk에 포함 | Chunk에 포함 |
| 비밀 비율 | 15~20% | 10~15% (반복 플레이 고려) |

**아이템계 방 유형 (4종):**

| 방 유형 | 아이템계 역할 | 배치 규칙 |
| :--- | :--- | :--- |
| 전투방 | 기억 단편 사냥, 경험치 | 크리티컬 패스 + 비경로 혼용 |
| 기억 단편 방 | 특정 기억 단편 보스 | 크리티컬 패스 종점에 배치 |
| 보물방 | 아이템 보상, 기억 단편 복종 | 비경로 전용, 능력 게이트 뒤 |
| 아이템 장군 방 | 지층 보스 전투 | 크리티컬 패스 최하단 고정 |

### 7-3. LDtk 워크플로우 통합

ECHORIS는 LDtk를 레벨 에디터로 사용한다. 위 원칙의 LDtk 적용 방법:

1. **방 유형 레이어:** LDtk의 Entity 레이어에 방 유형 태그를 붙여 구분.
2. **Chunk 배치:** LDtk의 타일셋 규칙(Auto-Layer)으로 Chunk 조합 검증.
3. **Critical Path 표시:** LDtk Entity로 입구/출구/크리티컬 경로 마킹.
4. **비밀 방 표시:** "SECRET" 태그가 달린 방은 별도 레이어에서 가시성 관리.
5. **적 배치 슬롯:** 각 Chunk에 Enemy Spawn Point Entity를 사전 배치, 런타임에 적 유형 결정.

### 7-4. 기쇼텐케츠 적용 예시 — ECHORIS 중앙 성채

**첫 층위(중앙 성채)에서 이중 점프 메커닉 소개:**

```
방 1 (Ki — 도입):
  - 높이 3타일 갭이 있는 방. 기본 점프로 건너기 불가.
  - 갭 직전에 이중 점프 부츠 아이템 배치.
  - 장애물 없음, 적 없음. 순수하게 "이중 점프 획득" 경험.

방 2 (Sho — 발전):
  - 이중 점프가 필요한 갭 2~3개 연속.
  - 타이밍 요구 없음, 갭 간 거리 충분.
  - 약한 적 1마리가 이중 점프 착지 지점을 지킴 (이동 + 전투 결합).

방 3 (Ten — 반전):
  - 이동 발판 위에서 이중 점프 요구.
  - 발판이 이동 중이므로 타이밍 + 이중 점프 동시 요구.
  - 또는: 이중 점프로만 도달하는 가시 함정 회피 구간.

방 4 (Ketsu — 해소):
  - 이 구역의 능력 게이트: 이중 점프로만 접근 가능한 다음 층위 입구.
  - 이중 점프를 완전히 숙달한 플레이어만 진행 가능 확인.
  - 게이트 통과 후 휴식 방 배치 → 성취감과 긴장 해소.
```
