# 인벤토리 레이아웃 + 아이템 상태 시각 구분 리서치

> **작성일:** 2026-04-26
> **목적:** ECHORIS 인벤토리에서 일반/장착/선택 상태가 구분되지 않는 문제를 해결. Grid vs List 레이아웃 비교 + 15종 게임 레퍼런스 분석.
> **핵심 결론:** 아이템 최대 20개 + 무기 1슬롯 + 스탯 비교 핵심 + 아이템계 진입 통합 → **리스트 + 디테일 패널 하이브리드 (Disgaea/Bloodstained 방식)** 이 그리드보다 우위.

---

## 1. 현재 문제 진단

| 상태 | 현재 시각 처리 | 문제 |
|:-----|:-------------|:-----|
| **Normal** | 짙은 남색 `0x2A2A3E` 배경 + 레어리티 색 사각형 | 기준 |
| **Selected** | 약간 밝은 남색 `0x4A4A8A` | **명도 차이 ~15%. 구분 불가능.** 최소 30% 필요 |
| **Equipped** | 흰색 테두리 1px | **너무 얇아서 사실상 보이지 않음.** 최소 2-3px 필요 |

추가 문제:
- 무기 아이콘 없음 (레어리티 색 사각형만) → "어떤 아이템인지" 식별 불가
- 5x4 그리드는 아이콘 식별이 핵심인데, 아이콘이 없으므로 그리드의 장점을 활용하지 못함
- 아이템이 20개 이하로 적어서 빈 슬롯이 허전함

---

## 2. 15종 게임 레퍼런스 — 상태 구분 방법 비교

### 장착(Equipped) 표시 방법

| 방법 | 빈도 | 대표 게임 |
|:-----|:----:|:---------|
| **별도 슬롯/영역에 분리 배치** | 10/15 | Diablo, PoE, Elden Ring, SotN, HK, Dead Cells, Bloodstained, FE, MHR, Hades |
| **"E" 텍스트 뱃지 / 캐릭터명** | 5/15 | Disgaea, FFT, FE, MHR, Bloodstained |
| **테두리 색상 변경** | 4/15 | Diablo, Hades, PoE, Elden Ring |
| **밝기/색조 차이** | 3/15 | HK, Hades, Bloodstained |

**업계 표준:** "별도 영역 분리" 가 가장 보편적. 리스트 UI에서는 **"E" 뱃지 + 장착자 표시** 가 표준.

### 선택(Selected) 표시 방법

| 방법 | 빈도 |
|:-----|:----:|
| **하이라이트 바 (행 전체 밝아짐)** | 12/15 |
| **커서 화살표 (▶)** | 5/15 |
| **확대/줌** | 2/15 |
| **테두리 변경** | 4/15 |

**업계 표준:** **하이라이트 바** 가 압도적. 리스트에서는 하이라이트 바 + 커서 화살표(▶) 조합이 가장 안전.

## 3. Grid vs List — ECHORIS 조건 대입

### ECHORIS 조건

- 아이템 최대 20개
- 장비 슬롯 1개 (무기)
- 스탯 비교가 핵심 (ATK 한 줄이라도)
- 아이템계(DIVE) 진입 기능이 인벤토리에 통합
- 무기 아이콘 미구현 (레어리티 색 사각형만)
- 키보드 전용 (방향키 탐색)

### 비교

| 기준 | Grid (현재) | List (제안) |
|:-----|:-----------|:-----------|
| 아이콘 없이 아이템 식별 | **불가능** — 색 사각형만으론 구분 안 됨 | **가능** — 이름 텍스트로 즉시 식별 |
| 스탯 비교 | **어려움** — 별도 Info Box 필요 | **쉬움** — 같은 행에 ATK 표시 + 비교 패널 |
| 장착 상태 구분 | **어려움** — 1px 테두리 | **쉬움** — "E" 뱃지 + 주황 세로 바 |
| 선택 상태 구분 | **어려움** — 미묘한 색 차이 | **쉬움** — 행 전체 하이라이트 |
| 아이템계 DIVE 표시 | **작음** — 6x6 아이콘이 그리드 코너 | **명확** — "DIVE ▶" 텍스트를 행 우측에 |
| 빈 공간 낭비 | **있음** — 20슬롯 중 3-5개만 차 있으면 허전 | **없음** — 아이템 수만큼만 표시 |
| 키보드 탐색 | 4방향 (상하좌우) — 5열 그리드에서 혼란 | **2방향 (상하)** — 단순, 직관적 |

### 판정

**리스트가 ECHORIS에 더 적합하다.** 현재 그리드는 아이콘 없이 색 사각형만 보여주고 있어 그리드의 핵심 장점(시각적 아이콘 스캔)을 쓰지 못합니다. 리스트로 전환하면 **이름/스탯/상태가 한 행에 모두 보여** 모든 문제가 동시에 해결됩니다.

## 4. 제안: 리스트 + 디테일 패널 하이브리드

### 레이아웃

```
640x360 (베이스 해상도)

┌═══════════════[ INVENTORY ]═══════════════════════════┐
│                                                        │
│  ⚔ [E] Cosmic Blade      ★★★★  ATK 120              │ ← 장착 중 (주황 [E])
│  ──────────────────────────────────────────────────── │
│  ⚔ [▶] Galaxy Sword      ★★★   ATK 100   [DIVE ▶]  │ ← 선택 중 (하이라이트)
│  ⚔     Bronze Dagger     ★     ATK   5   [DIVE ▶]  │
│  ⚔     Iron Axe          ★★    ATK  25   [DIVE ▶]  │
│  ⚔     Old Cleaver       ★     ATK   8   [CLR ✓]   │ ← 클리어됨
│                                                        │
│  ─────────────────────────────────────────────────── │
│  Galaxy Sword                  vs Equipped:            │
│  RARE (노란색)  Lv.35          ATK: 120 → 100 (▼-20) │
│  Memory Shards: 2/3                                        │
│  Strata: 3 layers  [DIVE ▶]                           │
│  ─────────────────────────────────────────────────── │
│  [C] Equip   [Z] Detail   [ESC] Close                 │
└════════════════════════════════════════════════════════┘
```

### 행 구조 (한 줄에 보이는 정보)

```
[아이콘] [상태] [이름]           [레어리티] [핵심 스탯]  [DIVE 상태]
  ⚔      [E]   Cosmic Blade     ★★★★      ATK 120      --
  ⚔      [▶]   Galaxy Sword     ★★★        ATK 100      [DIVE ▶]
```

| 요소 | 크기 | 설명 |
|:-----|:-----|:-----|
| 아이콘 | 8x8 | 무기 종류 실루엣 (Blade/Cleaver/Shiv 등). 없으면 ⚔ 기본 |
| 상태 | 16px | `[E]` = 장착 (주황), `[▶]` = 선택 (흰색), 빈칸 = 일반 |
| 이름 | 가변 | 레어리티 색상으로 표시. 최대 16자 |
| 레어리티 | 32px | ★ 개수 (N=1, M=2, R=3, L=4, A=5) |
| 핵심 스탯 | 48px | `ATK 120` (흰색) |
| DIVE 상태 | 48px | `[DIVE ▶]` (청록), `[CLR ✓]` (초록), `--` (장착중/불가) |

### 3가지 상태 시각 처리

| 상태 | 배경 | 좌측 마커 | 텍스트 | 추가 |
|:-----|:-----|:---------|:-------|:-----|
| **Normal** | `0x1A1A2E` (기본 어두움) | 없음 | `0xCCCCCC` (밝은 회색) | -- |
| **Selected** | `0x3A3A6E` (밝아짐, +40%) | `▶` 커서 (흰색) | `0xFFFFFF` (순백) | 하단 디테일 패널에 정보 표시 |
| **Equipped** | `0x2A1A10` (어두운 주황 틴트) | `[E]` 뱃지 (주황 `0xFF8C00`) | 이름: 레어리티 색 | 좌측에 주황 세로 바 3px |

### DIVE 상태 아이콘

| 상태 | 표시 | 색상 |
|:-----|:-----|:-----|
| 다이브 가능 (미클리어) | `[DIVE ▶]` | 청록 `0x00CED1` |
| 클리어됨 | `[CLR ✓]` | 초록 `0x44FF44` |
| 장착 중 (다이브 불가) | `--` | 회색 `0x666666` |
| 다이브 불가 (Broken Blade 등) | -- | -- |

## 5. 앤빌 모드 (FORGE)

리스트 레이아웃에서 앤빌 모드는 더 자연스러워집니다:

```
┌═══════════════[ FORGE ]═══════════════════════════════┐
│                                                        │
│  ⚔ [E] Cosmic Blade      ★★★★  ATK 120   🔒        │ ← 장착중=배치 불가
│  ──────────────────────────────────────────────────── │
│  ⚔ [▶] Galaxy Sword      ★★★   ATK 100   [DIVE ▶]  │ ← 선택. C키로 배치
│  ⚔     Bronze Dagger     ★     ATK   5   [DIVE ▶]  │
│  ⚔     Old Cleaver       ★     ATK   8   [CLR ✓]   │ ← 클리어=배치 불가
│                                                        │
│  ─────────────────────────────── ┌────────────────┐  │
│  Galaxy Sword                     │   ┌────────┐   │  │
│  RARE  Lv.35                     │   │ ANVIL  │   │  │
│  ATK: 100  (+5 vs Broken)        │   │  슬롯  │   │  │
│  Strata: 3 layers                │   └────────┘   │  │
│  ──────────────────────────────  │ [C]Place        │  │
│  [C] Place on Anvil              │ [ESC]Back       │  │
└══════════════════════════════════ └────────────────┘  │
```

- `[E]` 아이템: 🔒 자물쇠 표시. 배치 차단. 선택 시 "Unequip first" 안내
- `[CLR ✓]` 아이템: 이미 클리어됨. 배치 차단 (Phase 2 재사이클 전까지)
- `[DIVE ▶]` 아이템만 배치 가능

## 6. 비교: 현재 Grid vs 제안 List

### 시각적 비교

```
[현재 5x4 Grid]                     [제안 List]

┌──┬──┬──┬──┬──┐                   ⚔ [E] Cosmic Blade  ★★★★ ATK120
│■ │■ │■ │  │  │                   ─────────────────────────────
│──┼──┼──┼──┼──│                   ⚔ [▶] Galaxy Sword  ★★★  ATK100
│■ │  │  │  │  │                   ⚔     Bronze Dagger ★    ATK  5
│──┼──┼──┼──┼──│                   ⚔     Iron Axe      ★★   ATK 25
│  │  │  │  │  │
│──┼──┼──┼──┼──│                   어떤 게 장착인지? → [E] 즉시 인지
│  │  │  │  │  │                   어떤 게 선택인지? → [▶] + 밝은 행
└──┴──┴──┴──┴──┘                   각 무기 이름/스탯? → 텍스트로 즉시
                                   다이브 가능? → [DIVE ▶] 즉시
어떤 게 장착인지? → 1px 테두리???
어떤 게 선택인지? → 약간 밝은 색???
각 무기 이름? → Info Box를 봐야 함
다이브 가능? → 6x6 아이콘을 찾아야 함
```

### 구현 공수 비교

| 접근 | 공수 | 위험 |
|:-----|:-----|:-----|
| **기존 Grid 개선** (테두리 두껍게, 색 차이 키우기) | 2시간 | 근본 문제(아이콘 없음) 미해결 |
| **List로 전환** | 6-8시간 | InventoryUI.ts 대폭 변경. 하지만 Info Box 로직 재사용 가능 |
| **Grid 유지 + 무기 아이콘 추가** | 아이콘 제작 시간 미정 | 아이콘이 없으면 여전히 식별 불가 |

## 7. 구현 우선순위

### Option A: Grid 유지 + 즉시 개선 (최소 공수)

지금 당장 그리드를 버리지 않고 **상태 구분만 개선**하는 방안:

| 항목 | 변경 | 공수 |
|:-----|:-----|:-----|
| Selected 배경 | `0x4A4A8A` → `0x5A5AAA` + 노란 테두리 2px | 30분 |
| Equipped 테두리 | 1px 흰색 → **3px 주황 `0xFF8C00`** + "E" 뱃지 | 1시간 |
| 하단 힌트 | 현재 키 안내를 상태별로 갱신 | 30분 |

**총 2시간.** 근본 해결은 아니지만 즉각 체감 개선.

### Option B: List 전환 (근본 해결)

| 항목 | 변경 | 공수 |
|:-----|:-----|:-----|
| InventoryUI.ts 리스트 렌더링 | draw() 전면 재작성 | 4시간 |
| 디테일 패널 | 기존 Info Box 코드 재사용 | 1시간 |
| 앤빌 모드 통합 | 리스트 + 앤빌 슬롯 | 2시간 |
| 테스트 | 키 입력/상태/앤빌 전체 검증 | 1시간 |

**총 8시간.** 하지만 이후 모든 인벤토리 관련 피드백이 한 번에 해결됨.
