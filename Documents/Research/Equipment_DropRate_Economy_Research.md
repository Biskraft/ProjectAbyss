## 1. 레어리티별 드롭률 업계 벤치마크

### 1.1 Diablo 3 — 레어리티별 드롭률

디아블로 3는 아이템 레어리티를 **드롭 풀 가중치**로 관리한다. Blizzard가 공식 공개한 수치는 없으나, 커뮤니티 통계 분석(2015~2023, reddit.com/r/diablo3 및 플레이어 드랍 추적 데이터)을 통해 다음 수치가 확립되었다:

| 레어리티 | 드롭 비중 (일반 적) | 드롭 비중 (GR 상자) | 비고 |
|:---------|:-------------------|:-------------------|:-----|
| Magic (Blue) | ~50% | ~20% | 초중반까지 주류 |
| Rare (Yellow) | ~45% | ~55% | 후반 주류 |
| Legendary (Orange) | ~2~4% | ~20~25% | GR 클리어 상자에서 집중 |
| Set Item (Green) | ~1~2% | ~5% | Legendary 하위 분류 |
| Primal Ancient | 전체 Legendary의 1% | 전체 Legendary의 1% | GR 70 첫 클리어 후 해금 |
| Ancient | 전체 Legendary의 ~10% | 전체 Legendary의 ~10% | Legendary에서 분기 |

**핵심 수치:**
- Primal Ancient: Legendary 풀의 **1%** (일반 적 기준 전체 드랍의 약 0.02~0.04%)
- GR 클리어 보상 상자에서 Legendary 확률이 일반 적 대비 약 5~10배 높음
- "Smart Loot" 적용: 캐릭터 클래스에 맞는 아이템이 85% 확률로 드랍

**설계 교훈:**
- 최상위 등급(Primal Ancient)은 Legendary 내 하위 분류로 설계 → 플레이어가 Legendary를 기대하다가 Primal을 "보너스"로 경험
- GR(던전)에서 드롭률 집중 → 콘텐츠 유도 효과

### 1.2 Diablo 4 — 드롭 설계 변화 (Season 4~)

D4는 Season 4(2024, "Loot Reborn")에서 드롭 구조를 전면 개편했다:

| 레어리티 | 드랍 비중 변화 | 비고 |
|:---------|:-------------|:-----|
| Magic/Rare | 대폭 감소 | 엔드게임에서 거의 드랍 안됨 |
| Legendary | 대폭 증가 | 엔드게임 주류 드랍 |
| Unique | Legendary의 ~5% | 전용 드랍 풀 존재 |
| Mythic Unique | 추정 0.03~0.1% | 계정당 1개만 착용 가능 |
| Sacred / Ancestral | 월드 티어 4 이상 | 같은 종류 아이템의 강화 버전 |

**Tormented 보스 드랍률 (Season 4 기준):**
- 일반 Uber Boss: Unique 100% 보장 (보스 전용 아이템 테이블)
- Tormented Uber Boss: Mythic Unique 드랍 가능 (추정 2~5% / 보스 처치)

**설계 교훈:**
- D4는 "더 많은 Legendary, 더 적은 쓰레기"로 전환 → 필터링 부담 감소
- Masterworking으로 아이템을 완성하는 구조 → Legendary 자체보다 "완성도"에 희소성 집중

### 1.3 Path of Exile — Currency & Unique 드롭률

PoE는 레어리티 개념 대신 **아이템 베이스 + 모드 수(Modifier Count)**로 품질을 정의한다:

| 아이템 유형 | 드롭 비중 | 비고 |
|:-----------|:---------|:-----|
| Normal (White) | ~40% | 베이스 아이템 |
| Magic (Blue) | ~35% | 1~2개 모드 |
| Rare (Yellow) | ~20% | 4~6개 모드 |
| Unique (Orange/Brown) | ~1~3% | 고정 모드 세트, 강력 아이템 |
| Mirror of Kalandra | 약 0.0001% | 업계 최희귀 아이템 중 하나 |

**Currency 드랍 비중 (T16 맵 기준, 2026 Settlers of Kalguur):**

| Currency | 드랍 빈도 | 환율 (Chaos 기준) |
|:---------|:---------|:-----------------|
| Orb of Alteration | 매우 흔함 | ~0.1 Chaos |
| Chaos Orb | 흔함 | 1 Chaos (기준) |
| Divine Orb | 드묾 | ~100~200 Chaos |
| Mirror of Kalandra | 극희귀 | ~50,000~100,000 Chaos |

**Unique 아이템 드랍 구조:**
- 일반 Unique: 레어 기준 풀에서 1~3% → 평균 매핑 30~100회에 1개 획득
- Chase Unique (Mageblood 등): 특정 맵/보스 전용 드랍 풀 + 기본 확률 0.1~0.3%
- Forbidden Tome (최상위): 0.01~0.05% 추정

**Deterministic Crafting (결정론적 크래프팅):**
PoE는 순수 랜덤 드랍 외에도 **Essence / Fossil / Harvest** 등 재료 소비로 원하는 속성을 "강제"하는 시스템을 제공한다. 이는 배드 럭 보정의 대안적 접근이다:

```
[순수 파밍] → 원하는 Unique 드랍 기다리기 (운 의존)
[크래프팅]  → Rare 아이템에 원하는 속성 강제 부여 (재료 소비)
```

**설계 교훈:**
- 순수 드랍과 크래프팅을 병행 → 어느 경로든 목표 달성 가능
- 크래프팅 재료 자체가 경제의 핵심 Sink

### 1.4 Borderlands 시리즈 — 색상별 드롭률

Borderlands 2/3는 **총기 등급 색상(Grade)**으로 아이템 품질을 구분한다:

| 등급 | 색상 | 드롭 비중 (일반) | 드롭 비중 (보스) |
|:-----|:-----|:---------------|:--------------|
| Common | 흰색 | ~50% | ~10% |
| Uncommon | 초록 | ~30% | ~25% |
| Rare | 파란 | ~15% | ~35% |
| Epic | 보라 | ~4~5% | ~25% |
| Legendary | 주황 | ~0.5~1% | ~5% |
| Pearl / Seraph | 하늘/빨강 | DLC 전용 | 전용 보스만 |

**Dedicated Drop 시스템 (BL3 핵심):**
- 각 보스가 1~3개의 전용 Legendary 아이템을 높은 확률로 드랍 (약 10~15%)
- "Mayhem Mode" 적용 시 전반적 Legendary 드랍률 2~3배 증가
- 결과: "이 보스를 잡으면 이 총기가 나온다"는 타겟 파밍 동기 강화

**설계 교훈:**
- Dedicated Drop = 목표 지향적 파밍. "어디서 어떤 아이템이 나온다"는 지식이 플레이어의 전문성으로 전환됨
- Mayhem Scale로 파밍 효율을 난이도에 비례시킴 → 도전 동기 부여

### 1.5 Monster Hunter — 소재 드롭 테이블

MH 시리즈는 가장 투명한 드랍 테이블 공개 정책을 가진 게임 중 하나다:

| 획득 방법 | 드랍률 범위 | 비고 |
|:---------|:---------|:-----|
| 부위 파괴 보상 | 60~90% (특정 소재) | 머리/꼬리 등 부위별 고정 |
| 포획 보상 | 공식 테이블 공개 | 포획 > 토벌 희귀 소재 유리한 경우多 |
| 카빙(해체) | 공식 테이블 공개 | 소재별 40~60% / 레어 소재 5~15% |
| 실버 라운드 (Tempered) | 30~45% | 강화 버전 몬스터 |
| 골드 라운드 (AT) | 15~25% | 최강화 버전 몬스터 |

**MH 핵심 수치 예시 (Monster Hunter World: Iceborne, Rajang 기준):**
- Rajang Hardclaw (희귀 소재): 카빙 18%, 꼬리 파괴 보상 27%, 포획 보상 20%
- Spiritvein Solidbone (Tempered 전용): Tempered G Rank Rajang 17%

**설계 교훈:**
- 드랍 테이블 공개 → 플레이어가 "지식"을 습득하고 효율적 파밍 경로를 선택 → 전문성에서 오는 만족감
- 부위 파괴, 포획, 카빙의 삼중 드랍 경로 → 같은 몬스터를 다양한 전략으로 파밍

## 2. 드롭 테이블 설계 패턴

### 2.1 가중치 기반 드롭 테이블 (Weighted Drop Table)

가장 널리 사용되는 드랍 테이블 구조. 각 항목에 정수 가중치를 부여하고, 전체 가중치 합에서 무작위 추출한다:

```
[드롭 풀 예시 — 일반 월드 적]
Nothing      : weight 200  (드롭 없음)
Normal Equip : weight 600  (60%)
Magic Equip  : weight 250  (25%)
Rare Equip   : weight 100  (10%)
Legendary    : weight 40   (4%)
Ancient      : weight 10   (1%)
----
Total weight : 1200

확률 계산:
P(Normal)    = 600 / 1200 = 50.0%
P(Magic)     = 250 / 1200 = 20.8%
P(Rare)      = 100 / 1200 = 8.3%
P(Legendary) = 40  / 1200 = 3.3%
P(Ancient)   = 10  / 1200 = 0.8%
P(Nothing)   = 200 / 1200 = 16.7%
```

**분리 롤(Separated Roll) 방식:**
단일 룰렛 대신 레어리티 결정 + 아이템 결정을 2단계로 분리:

```
Step 1 - 레어리티 결정 롤:
  Normal(60%) / Magic(25%) / Rare(10%) / Legendary(4%) / Ancient(1%)

Step 2 - 아이템 종류 결정 롤 (레어리티 내 서브 풀):
  Legendary 풀: 검(20%) / 갑옷(20%) / 망토(15%) / 장신구A(10%) / 장신구B(10%) / ...
```

분리 롤 방식의 장점:
- 레어리티 확률과 아이템 종류 확률을 독립적으로 조정 가능
- 특정 아이템 추가/제거 시 레어리티 분포에 영향 없음
- 로컬 드랍 테이블(구역별, 보스별) 적용이 용이

---

### 2.2 조건부 드랍 (Conditional Drop)

**보스 전용 드랍 풀:**

```
[일반 몬스터 드랍 풀]     [보스 전용 드랍 풀]
- Normal 60%              - Rare 40%
- Magic 25%               - Legendary 35%
- Rare 10%                - Ancient 15%
- Legendary 4%            - 보스 전용 유니크 10%
- Ancient 1%
```

보스 전용 풀 설계 원칙:
- Normal/Magic은 보스 풀에서 제거 또는 최소화
- 보스 특화 "Dedicated Drop" 아이템: 해당 보스에서만 드랍되는 전용 아이템 1~3종
- 보스 처치 1회당 1개 이상의 유의미한 아이템을 보장 (예: Rare 이상 1개 확정)

**구역 전용 드랍 풀:**
- 구역마다 별도 아이템 풀 정의 → 특정 구역에서만 드랍되는 아이템 존재
- 플레이어가 "어디서 어떤 장비를 파밍하는지"를 학습하는 재미 부여

---

### 2.3 스마트 루트 (Smart Loot)

Diablo 3가 도입한 캐릭터 클래스/레벨 기반 드랍 필터링:

```
[스마트 루트 알고리즘]
아이템 드랍 결정
  → 레어리티 결정 (가중치 롤)
  → 아이템 타입 결정 (서브 풀 롤)
  → 스마트 루트 필터 적용:
       - P(플레이어 클래스 적합 아이템) = 85%
       - P(다른 클래스 아이템) = 15%
  → 스탯 범위 결정 (플레이어 레벨 기반)
```

**Project Abyss 적용 방향:**
- Phase 2: 장비 타입별 스마트 루트 (착용 중인 슬롯 부재 아이템 드랍률 상승)
- 파티 플레이: 각 파티원의 클래스/레벨 정보를 드랍 계산에 활용
- 단, MVP 단계에서는 완전 랜덤 드랍 → 스마트 루트는 Phase 2 이후 도입

---

### 2.4 드롭 풀 분리 (Drop Pool Segmentation)

아이템 유형별 드롭 풀을 완전히 분리하는 구조:

```
[드랍 이벤트 발생]
        ↓
   [드롭 풀 선택]
  ┌─────┬────────┬──────────┬──────────┐
  ↓     ↓        ↓          ↓          ↓
장비  소모품   재료/광물   이노센트   골드/화폐
풀    풀        풀          풀         풀
```

**드롭 풀 발동 확률 (제안):**

| 드롭 풀 | 일반 적 | 엘리트 적 | 보스 |
|:--------|:--------|:---------|:-----|
| 장비 풀 | 30% | 50% | 70% |
| 소모품 풀 | 25% | 20% | 10% |
| 재료 풀 | 30% | 20% | 15% |
| 이노센트 풀 | 5% | 8% | 20% |
| 골드 풀 | 10% | 2% | 5% |
| 드롭 없음 | 0% | 0% | 0% |

*각 행의 합계가 100%를 초과하는 것은 복수 드롭(Multi-drop) 롤을 허용하기 때문. 각 풀은 독립적으로 롤.*

## 3. 배드 럭 보정 (Bad Luck Protection / Pity System) 설계

### 3.1 겐신 임팩트 — 하드 피티 & 소프트 피티

겐신 임팩트의 가챠 피티 시스템은 업계에서 가장 투명하게 설계된 배드 럭 보정 사례다:

**5성 캐릭터 피티 (표준 배너 기준):**

| 구간 | 확률 |
|:-----|:-----|
| 1~73회 | 0.6% 고정 |
| 74~89회 (소프트 피티) | 매 회 +6%씩 증가 (74회 6.6%, 75회 12.6%, ...) |
| 90회 (하드 피티) | 100% 보장 |

**소프트 피티 확률 공식:**
```
P(n) = 0.006 + 0.06 × max(0, n - 73)   (73 < n < 90)
P(90) = 1.0  (하드 피티)
```

**기대 소요 횟수:**
- 피티 미적용 순수 확률: E[X] = 1 / 0.006 = 약 167회
- 소프트+하드 피티 적용: 실제 평균 약 62~65회

**봉헌 시스템 (Epitomized Path):** 한정 무기 배너에서 2회 5성을 뽑은 후에도 원하는 무기가 아닌 경우, 3번째는 원하는 무기 100% 보장. 이는 "1회의 완전 피티" 외에 "타겟팅 피티"를 추가한 구조.

---

### 3.2 Diablo 4 — 확률 증가형 피티

D4는 고정 횟수 피티가 아닌 **연속 실패 시 확률 선형 증가** 방식을 채택한다:

```
기본 Legendary 드랍률: L%
연속 실패 n회 후 드랍률: L% + (n × ΔL%)
```

단, 정확한 수치는 Blizzard가 공식 비공개. 커뮤니티 분석에 따르면:
- Uber Unique 계열: 연속 처치 시마다 확률 소폭 증가
- Tormented 보스: 처치 횟수 기반 Mythic Unique 증가율 적용 추정

---

### 3.3 PoE — Deterministic Crafting (결정론적 배드 럭 보정)

PoE는 피티 타이머 대신 **재료 소비 확정 크래프팅**으로 배드 럭 보정을 구현한다:

| 크래프팅 방법 | 메커니즘 | 비고 |
|:-------------|:--------|:-----|
| Harvest Crafting | 특정 속성 확정 부여/제거 | 시즌별 가용 범위 변동 |
| Essence | 특정 속성 1개 확정 + 나머지 랜덤 | 재사용 불가 |
| Fossil | 속성 편향 (특정 속성 출현 확률 상향) | 복수 조합 가능 |
| Fracturing | 아이템의 속성 1개 고정 (불변) | 재화 소모 |

```
[배드 럭 보정 수학적 비교]

피티 타이머 방식:
  시도 횟수 누적 → 확률 증가 → 보장

결정론적 크래프팅:
  재료 n개 소비 → 원하는 속성 확정
  재료 획득 자체가 파밍 목표
```

### 3.4 피티 타이머의 수학적 모델링

**기본 피티 타이머 모델:**

```
변수:
  p = 기본 드랍 확률
  n = 현재 연속 실패 횟수
  T = 피티 임계값 (하드 피티 횟수)

선형 증가 모델:
  P(n) = p + (n / T) × (1 - p)   (n < T)
  P(T) = 1.0

이차 가속 모델 (소프트 피티):
  P(n) = p                        (n < S, 소프트 피티 시작점)
  P(n) = p + k × (n - S)²        (S <= n < T)
  P(T) = 1.0
```

**기댓값 계산 (기본 피티 없는 경우 vs 피티 있는 경우):**

```
피티 없는 기하분포:
  E[X] = 1 / p

선형 피티 있는 경우:
  E[X] = Σ(n=1 to T) [n × P(success at n)]
       = Σ(n=1 to T) [n × (1 - P(n-1)) × P(n)]

예시 (p = 0.01, T = 100):
  피티 없음: E[X] = 100회
  선형 피티: E[X] ≈ 52~58회 (피티로 약 45~48% 시도 감소)
```

**연속 시도 중위값(Median) vs 평균(Mean):**
- 평균은 극단값에 취약 → 운 나쁜 플레이어의 실제 경험을 반영 못 함
- 설계 시 **P90 (90번째 백분위수)** 기준으로 피티 설정 권장
- "90%의 플레이어가 N회 이내에 획득"이 목표 기준

### 3.5 Project Abyss 이노센트/레어 장비 피티 적용 방안

**제안 1 — 카운터 기반 선형 피티 (심플)**

```
[이노센트 출현 피티]
기본 확률: 5% (일반 적 처치 시 이노센트 드랍)
연속 비출현 횟수: n
피티 확률: P(n) = 0.05 + (n / 40) × 0.95
하드 피티: 40회 처치 시 100% 이노센트 1개 보장

[Legendary 장비 피티]
기본 확률: 4%
소프트 피티 시작: 50회 연속 비드랍
소프트 피티 확률: P(n) = 0.04 + 0.04 × (n - 50)  (50 <= n < 75)
하드 피티: 75회 연속 비드랍 시 100% 보장
```

**제안 2 — 이노센트 복권 시스템 (레이어드)**

```
[레이어 1: 기본 드랍]
이노센트 등장 확률: 5% / 적 처치

[레이어 2: 복종 피티]
야생 이노센트 복종 실패 시 카운터 누적
10회 실패 시: 복종 확률 100% 보장 (단발성)
카운터 리셋

[레이어 3: Legendary 이노센트 등장 피티]
이노센트 드랍 중 Legendary 이노센트 등장 확률: 2%
30회 이노센트 드랍 누적 시: 1회 Legendary 이노센트 보장
```

**제안 3 — LCK 스탯 연동 소프트 피티**

Project Abyss의 LCK 스탯을 드랍률 보정에 활용:

```
P(rarity) = Base_P × (1 + LCK × 0.001)

예: LCK 100 → 드랍률 +10%
    Ancient 기본 1% → LCK 100 시 1.1%
    LCK 500 → Ancient 드랍률 1.5%

LCK 상한: 999 (최대 드랍률 보정 +99.9%, 즉 최대 2배)
```

**권장:** 제안 1(카운터 기반)을 기본으로, LCK 연동(제안 3)을 레이어로 추가. 이노센트 복종에는 제안 2의 복권 시스템 적용.

## 4. 파밍 효율 차등 설계

### 4.1 수동 파밍 vs 자동 사냥 보상 차이

Project Abyss는 "수동 정복 선행, 자동 사냥 해금" 원칙을 가진다. 이를 경제적으로 뒷받침하는 차등 설계:

| 항목 | 수동 파밍 | 자동 사냥 | 비율 |
|:-----|:---------|:---------|:-----|
| 장비 드랍률 | 100% (기준) | 50% | 2:1 |
| Legendary 이상 드랍 | 허용 | 불가 | - |
| Ancient 드랍 | 허용 | 불가 | - |
| 이노센트 드랍률 | 100% | 40% | 2.5:1 |
| 골드 획득 | 100% | 70% | 1.43:1 |
| 경험치 | 100% | 60% | 1.67:1 |
| 보스 스폰 | 정상 | 없음 | - |
| 배드 럭 보정 카운터 | 누적됨 | 누적 안됨 | - |

**설계 근거:**
- Legendary/Ancient는 수동 플레이에서만 드랍 → 자동 사냥의 근본적 한계 설정
- 이노센트 드랍률 2.5:1 격차 → 아이템계 야리코미는 수동 플레이가 압도적으로 효율적
- 자동 사냥은 소모품/재료/골드 획득 수단으로 포지셔닝 (전략적 용도 유지)

---

### 4.2 타겟 파밍 vs 랜덤 파밍의 균형

**랜덤 파밍의 문제점:**
- 원하는 아이템이 언제 드랍될지 모름 → 불확실성 → 좌절로 이어질 수 있음
- 단, 극복 가능한 불확실성은 도파민 분비의 핵심 (Variable Ratio Schedule)

**타겟 파밍 도입 방법:**

| 방법 | 게임 사례 | Project Abyss 적용 |
|:-----|:---------|:-----------------|
| 보스 Dedicated Drop | Borderlands 3 | 월드 보스별 전용 Legendary 1~2종 |
| 구역 한정 드랍 | Monster Hunter | 특정 구역에서만 나오는 장비 타입 |
| 크래프팅 레시피 | Path of Exile | 소재 수집 → 확정 합성 (Phase 2) |
| 이노센트 트레이드 | Disgaea | 플레이어 간 이노센트 거래 (허브 거래소) |
| 교환 상점 | 대부분의 MMORPG | 파밍 포인트로 원하는 등급 교환 |

**균형 원칙:**
- 타겟 파밍 가능한 아이템: 전체의 30~40%
- 순수 랜덤 드랍(Chase Item): 전체의 60~70%
- Chase Item은 타겟 파밍 불가 대신 피티 타이머 강화로 보정

---

### 4.3 시간당 기대 보상값 (Expected Value) 계산 방법

**단위 시간당 기대 드랍 (EV/h) 공식:**

```
EV(rarity, h) = encounters_per_hour × P(drop) × P(rarity) × item_value

예시:
월드 일반 구역 파밍 (시간당 300 적 처치):
  P(drop) = 0.35 (적 1마리당 아이템 드랍 확률)
  P(Legendary | drop) = 0.04

  Legendary EV/h = 300 × 0.35 × 0.04 = 4.2개/시간

아이템계 파밍 (시간당 200 적 처치, 보너스 적용):
  P(drop) = 0.45
  P(Legendary | drop) = 0.08 (아이템계 드랍률 보너스 2배)

  Legendary EV/h = 200 × 0.45 × 0.08 = 7.2개/시간 (71% 향상)
```

**목표 EV 설계 기준 (권장):**

| 등급 | 초반 (Lv 1~20) | 중반 (Lv 21~60) | 후반 (Lv 61~100) | 엔드게임 |
|:-----|:-------------|:--------------|:--------------|:---------|
| Normal | 5~10개/h | 10~20개/h | 20~40개/h | 분당 1~2개 |
| Magic | 2~4개/h | 5~10개/h | 8~15개/h | 시간당 20~30개 |
| Rare | 0.5~1개/h | 1~3개/h | 3~6개/h | 시간당 8~15개 |
| Legendary | 0.1~0.3개/h | 0.3~0.8개/h | 0.5~1.5개/h | 시간당 2~5개 |
| Ancient | 세션당 0~1개 | 세션당 0~1개 | 시간당 0.1~0.3개 | 시간당 0.5~1개 |

---

### 4.4 "갓 롤(God Roll)" 확률과 야리코미 동기

단순히 Legendary를 드랍받는 것이 아닌, 원하는 스탯 조합(갓 롤)의 확률이 야리코미의 핵심 동기다:

```
[갓 롤 확률 계산 예시]
Legendary 검 드랍 확률: 4%
원하는 주 스탯(ATK): 1/3 (메인 스탯 3종 중)
원하는 보조 스탯 조합 (2개 이상 원하는 것): 1/10
원하는 이노센트 슬롯 속성: 1/5 (이노센트 8슬롯 중 원하는 종류 분포)

갓 롤 Legendary 드랍 확률 = 0.04 × (1/3) × (1/10) = 0.0013%
→ 평균 약 75,000 아이템 드랍마다 1개
→ 단, 이노센트 강화와 재롤(Phase 2)로 갓 롤 "만들기" 가능
```

**야리코미 동기 공식:**
- 순수 드랍 갓 롤: 극히 낮은 확률 → 운의 영역, 강렬한 감정 반응
- 이노센트 파밍 갓 롤: 노력으로 달성 가능 → 성취감, 지속 참여 동기
- 두 경로의 병존이 야리코미의 핵심

## 5. 아이템 싱크 (Item Sink) 설계

### 5.1 강화 재료 소비

아이템을 소비하여 다른 아이템을 강화하는 구조:

| 강화 시스템 | 소비 재료 | 성공률 | 게임 사례 |
|:-----------|:---------|:------|:---------|
| 이노센트 합성 | 저등급 이노센트 3개 → 고등급 1개 | 100% 확정 | Disgaea (이노센트 타운) |
| 레어리티 승급 | 현재 아이템 + 업그레이드 스톤 | 100% (조건 달성 시) | D4 Masterworking |
| 스탯 재롤 | 리롤 스크롤 + 재화 | 100% (결과 랜덤) | PoE Chaos Orb |
| 아이템 분해 | 아이템 자체 | 확정 소멸 → 재료 획득 | 전반적 ARPG |

**Project Abyss 강화 재료 싱크 구조 (제안):**

```
[이노센트 합성]
야생 이노센트(복종 안됨) 3개 + 합성석 1개 → 랜덤 복종 이노센트 1개
복종 이노센트 2개 + 고위 합성석 → 동일 종류 이노센트 (레벨 합산 80%)

[레어리티 승급 소비]
승급 재료 = "기억의 파편" (아이템계 보스 처치 시 드랍)
Normal → Magic: 기억의 파편 5개 + 대상 아이템 아이템계 2지층 클리어
Magic → Rare: 기억의 파편 15개 + 3지층 클리어
Rare → Legendary: 기억의 파편 50개 + 3지층 클리어
Legendary → Ancient: 기억의 파편 200개 + 4지층 + 심연 클리어
```

### 5.2 아이템 분해 시스템

**기본 분해 규칙:**

| 등급 | 분해 결과 | 획득 재료 수량 |
|:-----|:---------|:------------|
| Normal | 기초 재료 조각 | 1~2개 |
| Magic | 마법 재료 조각 | 1~3개 |
| Rare | 정제 재료 조각 | 2~4개 |
| Legendary | 고급 재료 조각 + 기억의 파편 1~2개 | 3~5개 + 파편 |
| Ancient | 고급 재료 조각 + 기억의 파편 3~5개 | 5~7개 + 파편 |

**분해 트레이드오프:**
- Legendary 분해 = "이 장비의 가능성 포기" → 플레이어가 신중하게 결정
- 기억의 파편 획득은 레어리티 승급에 필수 → 필요에 의한 분해 발생

### 5.4 거래소 수수료

허브 거래소를 통한 이코노미 Sink:

| 거래 유형 | 수수료율 | 수수료 행선지 |
|:---------|:--------|:------------|

| 아이템 거래 성사 | 5% | 소멸 (Sink) |
| 즉시 구매 | 8% | 소멸 |
| 이노센트 거래 | 10% | 소멸 |
| 골드 환전 | 3% | 소멸 |

**수수료 소멸 구조의 효과:**
- 거래량이 증가할수록 골드가 자연스럽게 소멸 → 인플레이션 억제
- 대량 거래 자동화 방지 (높은 수수료가 대규모 투기 억제)

### 5.5 인플레이션 방지 전략

**핵심 Faucet vs Sink 균형:**

**인플레이션 경고 지표:**

- Legendary 거래 가격이 30일 평균의 150% 초과 → 드랍률 미세 조정

### 6.5 배드 럭 보정 수치 제안

**레어리티별 피티 카운터:**

| 레어리티 | 소프트 피티 시작 | 하드 피티 | 피티 적용 구간 확률 증가 |
|:--------|:-------------|:---------|:---------------------|
| Magic | 없음 | 없음 | - |
| Rare | 없음 | 20회 (월드 구역 내) | 10회 이상 시 확률 x1.5 |
| Legendary | 30회 | 60회 | 30~59회: 매 5회마다 +1% |
| Ancient | 50회 | 150회 | 50~149회: 매 10회마다 +0.5% |

**피티 카운터 구체 공식:**

```
[Legendary 피티]
0~29회 비드랍: P(Legendary) = 4% (기본)
30~59회 비드랍: P(Legendary) = 4% + floor((n-30)/5) × 1%
  (30회: 5%, 35회: 6%, 40회: 7%, 45회: 8%, 50회: 9%, 55회: 10%, 59회: 11%)
60회 이상: P(Legendary) = 100% (하드 피티)

[Ancient 피티]
0~49회 비드랍: P(Ancient) = 1% (기본)
50~149회 비드랍: P(Ancient) = 1% + floor((n-50)/10) × 0.5%
  (50회: 1.5%, 60회: 2%, 70회: 2.5%, ..., 140회: 5.5%, 149회: 5.5%)
150회 이상: P(Ancient) = 100% (하드 피티)

[이노센트 피티]
연속 40 적 처치 동안 이노센트 미드랍 시 → 다음 적 처치 시 이노센트 1개 확정 드랍
카운터 리셋
```

**피티 카운터 저장 및 이월 규칙:**
- 카운터는 서버에 영구 저장 (세션 종료, 사망에 관계없이 유지)
- 월드 피티 카운터와 아이템계 피티 카운터는 독립 (별도 관리)
- 시즌 초기화 여부: Phase 4 시즌 도입 시 결정 (현재 이월 유지 권장)

### 6.6 LCK 스탯 드롭률 보정 공식

```
[LCK 드롭률 보정]
LCK_Bonus(lck) = lck × 0.001  (단위: 배수)

P_actual(rarity, lck) = P_base(rarity) × (1 + LCK_Bonus(lck))

예시:
  LCK = 0:   Ancient 1.0%
  LCK = 100: Ancient 1.1% (+10%)
  LCK = 500: Ancient 1.5% (+50%)
  LCK = 999: Ancient 2.0% (+99.9%, 최대 2배 상한)

[이노센트 드랍률 LCK 보정]
이노센트 드랍 확률 기본 5%
LCK = 500 → 이노센트 7.5%
LCK 999 → 이노센트 10% (상한 2배)
```

**LCK 획득 방법 (설계 권장):**
- 장비 이노센트 중 "Lucky" 타입 → LCK +n
- 특정 구역 탐험 완료 보상
- 아이템계 특정 이노센트(Statistician 유사) → LCK 보너스 부여

## 8. 요약 및 설계 권고사항

### 8.1 핵심 원칙 요약

1. **희소성 계층 유지:** Ancient(1%)는 드랍 자체가 이벤트. 월드 보스/아이템계 심연에 집중 배치하여 희소성을 구역과 콘텐츠로 강화.

2. **피티는 기대 시간의 1/2 이내로 설계:** 평균 150회에 1번 드랍하는 아이템이라면 피티는 75~100회에 도달해야 플레이어가 "언젠가 얻을 수 있다"고 신뢰.

3. **Legendary/Ancient는 수동 파밍 독점:** 자동 사냥의 역할을 소모품/골드/재료로 제한. "진짜 보상"은 직접 플레이에서만 나온다는 원칙 고수.

4. **아이템계가 드랍률의 핵심 레버:** 지층이 깊을수록 희귀 아이템 집중 → "더 깊이 들어가야 한다"는 행동 유인을 경제 설계로 강화.

5. **이노센트는 파밍 Deep Dive 동기:** 갓 롤 아이템을 드랍으로 기다리는 대신, 이노센트 강화로 "직접 만드는" 경로 제공 → 야리코미 판타지의 경제적 기반.

6. **LCK 스탯의 의미 부여:** 스탯 게이트 6종 중 LCK는 탐험보다 드랍률 보정에 활용. 파밍 특화 빌드(LCK 극대화)가 경제적으로 유효한 전략이 되도록.
