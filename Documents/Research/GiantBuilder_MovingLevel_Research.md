# 거대 빌더 (Giant Builder) — 이동하는 맵 시스템 설계

> **작성일:** 2026-04-24
> **핵심 컨셉:** 대공동(The Shaft)을 수직으로 오가는 거대한 빌더 엔티티. 플레이어가 올라타고, 안에 들어가고, 작동시키는 살아있는 맵.
> **레퍼런스:** Shadow of the Colossus, God of War (크로노스/세계뱀), BotW 신수(Divine Beast), Xenoblade (비오니스), Castlevania OoE (브라키우라), Outer Wilds
> **스파이크 검증:** "아이템에 들어가면 기억이 던전이 된다" + "빌더 자체에 올라타면 빌더의 기억이 세계가 된다" → **스파이크 강화**

---

## 1. 핵심 컨셉

### 한 줄 정의

> **"대공동을 수직으로 이동하는 거대한 빌더의 잔해 — 올라타고, 안에 들어가고, 깨워서, 길을 연다."**

### 빌더란 무엇인가

ECHORIS의 세계관에서 "빌더(Builder)"는 대공동(The Shaft)을 건설한 원초적 존재다. 빌더 자체는 이미 부재하지만, 빌더가 만들고 남긴 **거대한 건설 엔티티(Construction Entity)**가 대공동 내부를 아직도 순회하고 있다. 이 엔티티들은:

- 빌더가 대공동을 건설할 때 사용한 **자동화 건설 기계**의 잔해
- 수백 년간 스스로를 수리하며 대공동 내부를 **수직으로 순환** 이동
- 일부는 **정지 상태** (입구를 막고 있음), 일부는 **느리게 이동 중**
- 내부에 빌더의 작업 공간, 데이터 코어, **모루(Anvil)**가 존재
- 크기: **5-10 스크린 높이** (80-160타일 = 1280-2560px @16px)

### 완다와 거상과의 공명

| Shadow of the Colossus | ECHORIS Giant Builder |
|:----------------------|:---------------------|
| 거상 = 보스 + 레벨 | 빌더 = 이동 플랫폼 + 내부 던전 + 게이트 |
| 거상 위를 등반 | 빌더 외부를 등반 (잔향사, 공명정 활용) |
| 약점 타격으로 격파 | 내부 진입 → 코어 활성화로 이동 시작 |
| 빛나는 인장이 약점 표시 | 빌더 문양(공명점)이 진입로 표시 |
| 거상의 크기가 경외감 | 빌더의 크기가 **대공동의 스케일** 체현 |

### BotW 신수와의 공명

| BotW Divine Beast | ECHORIS Giant Builder |
|:-----------------|:---------------------|
| 신수 자체가 회전/이동하는 퍼즐 던전 | 빌더 내부가 기울어지고 이동하는 퍼즐 공간 |
| 신수를 플레이어가 조작 (맵 회전) | 빌더를 플레이어가 작동 (수직 이동 개시) |
| 신수 내부에 단말기(Terminal) | 빌더 내부에 **모루(Anvil) + 데이터 코어** |
| 신수 해방 = 보스전 | 빌더 활성화 = 경로 해금 + 보스(빌더 수호자)전 |

## 2. 빌더의 5대 역할

### 역할 1: 수직 교통수단 (Vertical Elevator)

- 빌더가 대공동을 수직으로 **왕복 이동**
- 플레이어가 **외부에 올라타서** 이동 (무료 엘리베이터)
- 이동 중 빌더 표면에서 적과 전투 발생 (이동하는 아레나)
- **이동 주기**: Outer Wilds 방식 — 예측 가능한 시간표. 플레이어가 "다음 빌더가 언제 오는지" 학습

```
빌더 이동 시간표 예시:

  Tier 1 (천공) ──── 정차 20초 ────┐
       ↕ 하강 40초                  │
  Tier 3 (수로)  ──── 정차 20초 ────┤ 왕복 약 3분
       ↕ 하강 40초                  │
  Tier 5 (연구소) ──── 정차 20초 ───┘
       ↕ 상승 80초 (복귀)
```

### 역할 2: 통로 차단 (Gate/Obstacle)

- **정지 상태의 빌더**가 대공동의 핵심 통로를 물리적으로 막고 있음
- 거대한 빌더 몸체가 샤프트 전체를 가로막아 **위/아래로 진행 불가**
- 플레이어는 빌더 안에 들어가 **코어를 활성화**해야 빌더가 이동을 시작하고 통로가 열림
- **능력 게이트와 결합**: 빌더 외부 진입로에 공명점이 있어 공명정 없이는 진입 불가

### 역할 3: 내부 던전 (Interior Dungeon)

- 빌더 내부는 **독립된 미니 레벨** (LDtk 별도 레벨 파일)
- 3-5개 방 규모의 퍼즐+전투 공간
- 빌더가 이동 중이면 내부도 함께 기울어짐/흔들림 (카메라 효과)
- 내부 구조:

```
[빌더 내부 구조 — 수직 단면]

  ┌─── 머리 (Head) ─────────────┐
  │  모루(Anvil) + 아이템계 포탈  │  ← 최종 목표
  │  데이터 코어 (활성화 지점)    │
  ├─── 상체 (Upper) ────────────┤
  │  퍼즐 방 (공명정 + 기억각인)  │
  │  전투 방 (빌더 수호자 적)     │
  ├─── 중체 (Core) ─────────────┤
  │  동력실 (지층 관통으로 진입)   │
  │  보스 방 (빌더 코어 수호자)   │
  ├─── 하체 (Lower) ────────────┤
  │  진입구 (외부에서 접근)       │
  │  통로 + 잡졸 전투             │
  └─────────────────────────────┘
```

### 역할 4: 아이템계 허브 (Item World Hub)

- 빌더 머리(Head) 또는 코어에 **모루(Anvil)**가 있음
- 세이브 포인트도 빌더 내부에 존재
- "빌더 안에서 아이템의 기억으로 다이브한다" = **빌더의 기억 + 아이템의 기억이 이중으로 겹침**
- 빌더마다 다른 테마의 모루가 있어, **빌더 종류에 따라 아이템계 진입 시 보너스 효과** 가능

### 역할 5: 세계관 전달 (Narrative Vehicle)

- 각 빌더는 **고유한 이름과 역사**를 가짐
- 빌더 내부의 데이터 코어에서 **빌더의 기억(=대공동 건설의 역사)**을 단편적으로 확인
- 빌더의 외형이 **해당 층위의 역사를 반영** (전쟁 빌더는 방어 장갑, 연구 빌더는 관측 장치)
- Ancient 아이템의 최심층에서 만나는 것이 **빌더의 원형 데이터**와 연결

## 3. 빌더 유형 (3종)

### Type A: 순회 빌더 (Patroller)

> 대공동을 수직으로 왕복하는 이동형. 플레이어의 수직 이동 수단.

| 항목 | 값 |
|:-----|:---|
| 크기 | 5-7 스크린 높이 (약 100타일) |
| 이동 | 수직 왕복. 2-3개 Tier 범위. 예측 가능한 시간표 |
| 내부 | 2-3개 방 (소규모). 모루 1개, 세이브 포인트 1개 |
| 외부 | 상부 플랫폼에서 탑승. 측면에 잔향 앵커 배치 |
| 적 | 외부: 비행형 적이 이동 중 공격. 내부: 잡졸 2-3종 |
| 등장 수 | 대공동 전체에 3-4체 |
| 게이트 역할 | 없음 (자유 탑승). 이동 수단 전용 |
| 첫 등장 | T2 (중앙 성채) — "저게 뭐지?" 경외감 + "올라탈 수 있어?" 발견 |

### Type B: 격벽 빌더 (Blocker)

> 대공동의 핵심 통로를 가로막고 정지해 있는 게이트형. 내부 진입+활성화로 길을 연다.

| 항목 | 값 |
|:-----|:---|
| 크기 | 7-10 스크린 높이 (약 150타일). 샤프트 전체를 가로막음 |
| 이동 | **정지 상태** → 활성화 후 이동 시작 → 영구적으로 길을 연다 |
| 내부 | 3-5개 방 (중규모 던전). 퍼즐+전투+보스. 모루+세이브 포인트 |
| 외부 | 진입로에 **공명점** 배치 (공명정 필수). 측면 등반 루트 |
| 적 | 내부: 빌더 수호자 (전용 적). 보스: 코어 수호자 (미니보스) |
| 등장 수 | Tier 간 경계에 1체씩. 총 4-5체 |
| 게이트 역할 | **핵심 진행 게이트.** 빌더를 활성화해야 다음 Tier로 진행 가능 |
| 첫 등장 | T2→T3 경계 — "거대한 것이 길을 막고 있다. 안에 들어가야 한다." |

### Type C: 심연 빌더 (Abyssal)

> 대공동 최심부에서 발견되는 빌더의 원형. Ancient 아이템계와 연결.

| 항목 | 값 |
|:-----|:---|
| 크기 | 10+ 스크린 높이. 가장 거대 |
| 이동 | 극히 느린 하강 (심연으로 가라앉는 중) |
| 내부 | 5-7개 방 (대규모 던전). 빌더의 원형 데이터 + Ancient 모루 |
| 외부 | 표면이 반쯤 부서져 내부가 노출 |
| 적 | 빌더 원형 수호자 (보스급). 형태가 확정되지 않은 기하학적 존재 |
| 등장 수 | 1체 (유일무이) |
| 게이트 역할 | Ancient 아이템계 전용 진입점. 세계관 최종 퍼즐 |
| 첫 등장 | T7 (심연의 구) — "빌더 자신이 심연으로 가라앉고 있다" |

## 4. 플레이어 인터랙션 상세

### 4.1 외부 — 올라타기

```
[빌더 외부 — 측면도]

     ┌─── 머리 (Head) ──── 모루/포탈 접근 가능
     │
     │  ← 잔향 앵커 (Echo Thread로 등반)
     │
     ├── 측면 플랫폼  ← 적 출현 지점
     │
     │  ← 공명점 (Resonance Nail로 문 열기)
     │
     ├── 진입구  ← 내부로 들어가는 문
     │
     │  ← 잔향 앵커
     │
     └─── 다리 (Legs) ──── 바닥에서 올라타는 최초 접점
```

**탑승 방법:**
1. 빌더가 플레이어 근처를 지나갈 때 **다리/측면 플랫폼에 점프하여 착지**
2. 빌더 표면에 착지하면 **빌더의 Container에 자식으로 편입** → 빌더와 함께 이동
3. 빌더 표면에서 점프하면 **분리** → 월드 좌표로 복귀

**외부 전투:**
- 이동 중 비행형 적(A-05)이 빌더 주변에서 공격
- 빌더 표면의 적(빌더 수호 드론)과 전투
- **낙하 리스크**: 빌더 표면에서 떨어지면 대공동 아래로 낙하 → 지층 관통(Stratum Dive)으로 안전 착지 가능

### 4.2 외부 → 내부 진입

**진입 조건 (빌더 유형별):**

| 유형 | 진입 조건 | 이유 |
|:-----|:---------|:-----|
| 순회 빌더 | 측면 진입구에서 C키 (Pattern D) | 자유 접근 |
| 격벽 빌더 | **공명점에 공명정 사용** → 문 열림 → C키 진입 | 공명정 렐릭 필수 (능력 게이트) |
| 심연 빌더 | 표면 균열로 직접 진입 (파손 상태) | 접근 자체가 T7 도달 필요 |

**진입 연출:**
1. 진입구에 접근 → `[C] Enter` 프롬프트 (Pattern D)
2. C키 입력 → 카메라가 빌더 내부로 전환 (0.5초 페이드)
3. 내부 레벨 로드 (LDtk 별도 레벨)
4. 내부 HUD: 빌더 이름 + "Interior" 표시

### 4.3 내부 — 미니 던전

**내부 게임플레이:**
- 2D 타일맵 기반 방 구조 (기존 월드/아이템계와 동일 시스템)
- 빌더가 이동 중이면 **카메라 미세 흔들림** + 배경 패럴랙스 이동으로 "움직이는 내부" 체감
- 빌더 수호자 적 (전용 아키타입 — 기계적, 빌더의 자기 수리 시스템)
- 퍼즐: 공명정으로 내부 메커니즘 활성화, 기억 각인으로 위험 구역 탐색

**코어 활성화 (격벽 빌더 전용):**
1. 내부 최심부의 **동력실(Power Core)**에 도달
2. 코어 수호자 미니보스 처치
3. 코어에 C키 → **빌더가 움직이기 시작** → 화면 전체가 진동 + 상승 연출
4. 빌더가 이동하면서 가로막던 통로가 열림 → **다음 Tier 접근 가능**
5. 이후 이 빌더는 **순회 빌더로 전환** → 수직 이동 수단으로 활용 가능

### 4.4 모루/아이템계 접근

- 빌더 머리(Head) 또는 코어에 **모루(Anvil)** 배치
- "빌더 안의 모루에서 아이템의 기억으로 다이브" = 이중 기억 체험
- 빌더 종류에 따라 **다이브 보너스**:
  - 순회 빌더: 보너스 없음
  - 격벽 빌더: 첫 클리어 시 해당 Tier의 아이템 드랍율 증가
  - 심연 빌더: Ancient 전용 아이템계 진입 가능

## 5. 시각 설계

### 5.1 외부 실루엣

```
[격벽 빌더 — 대공동 내부 실루엣]

                    ┌─┐
                   ╱│ │╲        ← 머리 (관측탑/안테나)
                  ╱ │ │ ╲
              ┌──╱──┴─┴──╲──┐
              │   ◆ ◆ ◆    │   ← 상체 (공명점 3개 빛남)
              │  ╔═══════╗  │
              │  ║ 내 부  ║  │   ← 중체 (창문에서 빛 새어나옴)
              │  ║       ║  │
              │  ╚═══════╝  │
              │    │   │    │   ← 하체 (다리/지지 구조)
              └──┤ │   │ ├──┘
                 │ │   │ │      ← 다리 (대공동 벽에 걸림)
              ───┘ └───┘ └───   ← 대공동 벽면
```

### 5.2 스케일 전달 방법

| 기법 | 설명 |
|:-----|:-----|
| **패럴랙스 속도차** | 빌더가 앞에 있을 때 배경보다 빠르게 스크롤 → 거대함 체감 |
| **그림자** | 빌더가 대공동을 가로막으면 아래 전체가 그림자 (BlackGoo 확대) |
| **부분만 보임** | 한 화면에 빌더 전체가 안 들어옴. 스크롤해야 전모 파악 |
| **시야 내 진입** | 멀리서 실루엣이 보이다가 점점 가까워짐. 소리가 먼저 들림 |
| **비교 오브젝트** | 빌더 다리에 걸린 일반 건물/파이프가 빌더 대비 작게 보임 |

### 5.3 팔레트

- 빌더 외부: **빌더 전용 팔레트** (LUT) — 기존 층위 팔레트와 다른 고유 색조
- 빌더 내부: 층위 팔레트를 약간 변형 (더 어둡고 기계적, 주황 단조열 악센트)
- 공명점: 빌더 문양이 **펄스하는 청록색** (빌더의 색 = 청록 + 주황 의 청록 축)

### 6.3 빌더 이동 시스템

```typescript
class GiantBuilder {
  route: { y: number; waitTime: number }[];  // 정차 지점 목록
  currentRouteIdx: number;
  speed: number;  // px/frame
  state: 'moving' | 'waiting' | 'dormant';  // dormant = 격벽 빌더 비활성

  update(dt: number): void {
    if (this.state === 'dormant') return;

    if (this.state === 'waiting') {
      this.waitTimer -= dt;
      if (this.waitTimer <= 0) {
        this.currentRouteIdx = (this.currentRouteIdx + 1) % this.route.length;
        this.state = 'moving';
      }
      return;
    }

    // 목표 지점을 향해 이동
    const target = this.route[this.currentRouteIdx];
    const dir = Math.sign(target.y - this.container.position.y);
    this.container.position.y += dir * this.speed * dt;

    // 도착 체크
    if (Math.abs(this.container.position.y - target.y) < 2) {
      this.container.position.y = target.y;
      this.state = 'waiting';
      this.waitTimer = target.waitTime;
    }
  }

  activate(): void {  // 격벽 빌더 활성화
    this.state = 'moving';
    // 진동 이펙트 + 카메라 흔들림
    // 이후 순회 빌더로 전환
  }
}
```

### 빌더가 전하는 서사

> *"이 거대한 기계는 아직도 자신의 임무를 수행하고 있다. 빌더가 떠난 후에도, 건설 프로토콜은 멈추지 않았다. 이것은 충성인가, 아니면 버그인가."*

빌더 내부의 데이터 코어에서 플레이어가 발견하는 것은 **빌더(원초 존재)의 건설 기록**이다. 각 빌더마다 대공동 건설의 다른 단면이 기록되어 있어, 모든 빌더를 활성화하면 대공동의 기원에 대한 전체 그림이 완성된다.

## 9. 스파이크 검증

### Q1: 이것이 아이템계 경험을 강화하는가?

**강화한다.**
- 빌더 내부의 모루에서 다이브하면 "빌더의 기억 안에서 아이템의 기억으로 들어가는" 이중 기억 체험
- 빌더 활성화 보상으로 해당 Tier의 아이템 드랍율 증가 → 아이템계 진입 동기 강화
- 심연 빌더(Architect)가 Ancient 전용 아이템계 유일 진입점 → 야리코미 최종 목표

### Q2: 이것이 없으면 스파이크가 약해지는가?

**약해진다.**
- 빌더 없이 대공동은 "정적인 수직 통로"에 불과
- 빌더가 있으면 "살아있고 움직이는 세계" → 대공동의 규모와 역사가 체감됨
- 빌더 위에서의 전투/탐험은 다른 메트로베니아에서 경험할 수 없는 고유한 순간

### Q3: 이것이 스파이크를 희석시키는가?

**희석하지 않는다.**
- 빌더 시스템은 아이템계와 독립적이면서도 보완적
- "아이템의 기억이 던전이 된다"라는 스파이크에 "빌더의 기억이 세계가 된다"를 더한다
- 복잡도 증가 우려 → 격벽 빌더 4-5체 + 순회 2체 + 심연 1체 = 총 7-8체로 제한
