# 무기 다양성 리서치: 100개의 무기, 100개의 정체성

> **작성일:** 2026-04-16
> **리서치 목적:** ECHORIS 5카테고리 무기 체계에서 100개 무기가 각각 고유한 전투 정체성을 갖추기 위한 설계 프레임워크 도출
> **핵심 레퍼런스:** Elden Ring (411+ 무기), Dead Cells (56+ 무기), Hades (6무기 x 4양상), Castlevania SotN, Hollow Knight, Monster Hunter
> **관련 문서:** `System_Combat_Weapons.md`, `Content_Weapon_List.md`, `System_Combat_Elements.md`, `System_Equipment_Growth.md`

## 1. 현재 ECHORIS 무기 시스템 진단

### 1.1. 현재 구조

| 항목 | 현재 값 | 한계 |
|:---|:---|:---|
| 카테고리 | 5종 (Sword/Greatsword/Dagger/Bow/Staff) | 카테고리당 무브셋 1세트 |
| 무기 수 | 25종 (5카테고리 x 5레어리티) | 같은 카테고리 내 무기 간 차이가 수치(ATK)뿐 |
| 시그니처 메커닉 | 5종 (카테고리당 1개, 패시브 자동 발동) | 카테고리 안에서 무기별 차별화 축 부재 |
| 원소 | 3종 (Fire/Ice/Thunder) + Echo 인챈트 | 원소 자체는 무기 정체성이 아닌 레이어 |
| 성장 | 레벨(0-99) + 보스 영구 보너스 + 레어리티 승급 | 성장이 수치 증가에 국한 |

### 1.2. 핵심 문제

**"같은 카테고리의 Normal 검과 Ancient 검은 ATK 수치만 다르고, 전투 경험은 동일하다."**

현재 25무기 체계에서 플레이어가 체감하는 무기 종류는 사실상 5개(카테고리 수)이다. 레어리티는 수치 스케일링이지 플레이 스타일 변화가 아니다. 100무기 목표를 달성하려면 **카테고리 내부에서 무기별 고유 정체성**을 부여하는 새 축이 필요하다.

### 1.3. 목표 정의

> **"100개의 무기가 있으면, 100개의 무기 특성이 있어야 한다."**
> 모든 무기가 서로 다른 전투 리듬, 포지셔닝, 의사결정을 요구해야 한다.

## 2. 레퍼런스 분석: 대량 무기를 차별화하는 검증된 방법

### 2.1. Elden Ring: 5-Layer 무기 정체성 모델

Elden Ring은 411+ 무기를 40개 카테고리로 분류하되, 개별 무기 정체성은 5개 레이어의 조합으로 형성한다.

| 레이어 | 내용 | 공유/고유 | ECHORIS 적용 가능성 |
|:---|:---|:---|:---|
| L1: 기본 무브셋 | 카테고리 공유 약/강 공격 모션 | 70% 공유 | 이미 보유 (5카테고리 무브셋) |
| L2: 전쟁재(Ash of War) | 무기 스킬. 일반 무기는 교체 가능, 특수 무기는 고정 | 116종 풀 공유 | **핵심 채택 후보** |
| L3: 속성(Affinity) | 스케일링 변경 (Heavy/Keen/Quality/Magic 등) | 13종 전체 공유 | 부분 적용 (ATK/INT 축으로 제한) |
| L4: 스탯 스케일링 등급 | S~E 등급으로 보정 차등 | 무기별 고유 | 레어리티 체계와 충돌. 제한적 |
| L5: 고유 패시브 효과 | 상태이상, HP 흡수, 투사체 생성 등 | 30% 무기만 고유 | **핵심 채택 후보** |

**핵심 교훈:** Elden Ring 무기의 70%는 카테고리 무브셋을 공유하면서도 전쟁재+속성+패시브 조합으로 고유하게 느껴진다. 풀 애니메이션이 아니라 시스템 레이어가 정체성을 만든다.

**일반 무기 vs 특수(Somber) 무기:**

| 구분 | 일반 무기 (184종) | 특수 무기 (127종) |
|:---|:---|:---|
| 전쟁재 교체 | 가능 | 불가 (고정 스킬) |
| 속성 변경 | 가능 | 불가 |
| 성격 | 커스터마이징 캔버스 | 완성된 작품 |

이 이분법은 ECHORIS의 "일반 장비 vs 보스/특수 장비" 구조에 직접 대응 가능하다.

### 2.2. Dead Cells: 2D 횡스크롤에서 56+ 무기를 차별화한 방법

Dead Cells는 ECHORIS와 같은 2D 횡스크롤이면서 56개 이상의 고유 무기를 구현했다.

**핵심 전략: 무기별 고유 크리티컬 조건**

각 무기가 크리티컬(보너스 데미지)을 얻는 조건이 완전히 달라서, 같은 적을 상대해도 전투 접근법 자체가 바뀐다.

| 무기 | 고유 크리티컬 조건 | 플레이 변화 |
|:---|:---|:---|
| Assassin's Dagger | 배후 공격 | 위치 선정 중심 |
| Gold Digger | 골드 수집량 비례 | 파밍+전투 통합 |
| Scythe Claws | 교대 공격 리듬 | 리듬 게임적 전투 |
| War Spear | 다수 적 동시 타격 | 포지셔닝 최적화 |
| Valmont's Whip | 스위트 스팟 거리 | 정확한 간격 유지 |

**Dead Cells의 제작 효율화:**
- 3D 모델 -> 2D 스프라이트 렌더링 파이프라인: 새 무기 프로토타입 30분~1시간
- 프로그래밍 기반 차별화: 시각적 모션이 유사해도 히트박스/타이밍/특수 효과로 완전히 다른 느낌
- 격투 게임 기법 차용: 히트스탑, 히트래그로 타격감 차별화

**ECHORIS 적용 교훈:** 2D 횡스크롤에서 가장 효과적인 무기 차별화 수단은 **고유 발동 조건**이다. 모든 무기에 고유한 "이 무기가 빛나는 순간"을 부여하면 애니메이션 비용 없이 정체성을 확보할 수 있다.

### 2.3. Hades: 적은 무기 x 깊은 변형

| 무기 | Aspect 수 | 변형 방식 |
|:---|:---|:---|
| Stygian Blade | 4 (Zagreus/Nemesis/Poseidon/Arthur) | 이동 보너스 / 크리티컬 보너스 / 자원 회수 / 완전 새 무브셋 |
| Eternal Spear | 4 | 창 회수 러시 / 넓은 범위 / 완전 새 스타일 / 기본형 |
| 총 6무기 | 24 | 기본 무기 6개에서 24개 고유 플레이 스타일 도출 |

**설계 교훈:** "적은 기본 틀 x 깊은 변형" 전략. ECHORIS의 5카테고리도 충분히 넓은 기반이다.

### 2.4. Monster Hunter: 무기 = 캐릭터

14개 무기가 각각 **완전히 다른 조작 체계**를 제공한다. Charge Blade의 "소드 모드로 차지 -> 도끼 모드로 방출" 같은 **전투 루프 자체의 차이**가 핵심이다.

**ECHORIS 적용 교훈:** 무기가 가진 "고유 자원 관리 메커닉"이 전투 루프를 근본적으로 바꾼다.

### 2.5. Hollow Knight: 수식어(Modifier) 조합으로 다양성 확보

무기는 1개(Nail)이지만 45개 Charm 조합으로 근접 DPS/마법 특화/방어형/하이브리드 빌드를 자유롭게 전환한다. **노치 시스템**(강한 Charm = 높은 비용)으로 트레이드오프를 강제한다.

### 2.6. Castlevania SotN: 커맨드 입력 필살기

특정 무기에 격투 게임식 커맨드 입력(QCF + Attack)으로 숨겨진 Critical Art를 발동한다. **발견 자체가 탐험적 재미**이다.

## 3. ECHORIS 100무기 확장 프레임워크

### 3.1. 설계 원칙

1. **5카테고리 유지:** 기존 5종 무기 체계(Sword/Greatsword/Dagger/Bow/Staff)를 변경하지 않는다
2. **카테고리 무브셋 공유:** 같은 카테고리 무기는 기본 콤보 애니메이션을 공유한다 (비용 억제)
3. **3계층 정체성 모델:** 무브셋(공유) + 무기 기술(교체/고정) + 고유 패시브(무기별 1개)
4. **스파이크 강화:** 무기의 아이템계가 해당 무기의 고유 패시브를 반영해야 한다
5. **레어리티 직교:** 무기 정체성(패시브/기술)은 레어리티와 독립. 같은 "Flame Edge"는 Normal이든 Ancient든 같은 패시브를 가진다

### 3.2. 3계층 무기 정체성 모델

```
계층 1: 카테고리 무브셋 (5종, 공유)
  - 기본 콤보 (Sword 3타, Dagger 4타, Greatsword 2타, Bow/Staff 3타)
  - 점프 공격, 대시 공격
  - 카테고리 시그니처 메커닉 (기존 유지)
  → 5세트 풀 애니메이션. 이미 설계 완료.

계층 2: 무기 기술 (Weapon Art) (30~40종, 무기에 1개 장착)
  - 별도 입력(특수 공격 버튼)으로 발동하는 액티브 스킬
  - 일반 무기: 기술 교체 가능 (Elden Ring 전쟁재 방식)
  - 보스/특수 무기: 고유 기술 고정 (Somber 무기 방식)
  - 쿨다운 기반 (MP/자원 없음 — 기존 시스템에 자원 없으므로)
  → 기술당 1~2개 추가 애니메이션. 총 약 40~80개.

계층 3: 고유 패시브 (Weapon Passive) (100종, 무기별 1개 고정)
  - 추가 입력 없이 조건 충족 시 자동 발동 (기존 시그니처와 동일 방식)
  - 코드/데이터로 구현. 추가 아트 불필요.
  - 100무기 = 100개 고유 패시브
  → 제작 비용 최저. 정체성 기여도 최고.
```

### 3.3. 100무기 배분 구조

| 카테고리 | 무기 수 | 일반 무기 (기술 교체) | 특수 무기 (기술 고정) | 근거 |
|:---|:---:|:---:|:---:|:---|
| Sword (검) | 24 | 16 | 8 | 기준 무기. 가장 많은 변형 |
| Greatsword (대검) | 20 | 12 | 8 | 파워형. 보스 무기 다수 |
| Dagger (단검) | 20 | 14 | 6 | 스피드형. 다양한 기습 패턴 |
| Bow (활) | 18 | 12 | 6 | 원거리 물리. 화살 유형으로 변형 |
| Staff (지팡이) | 18 | 10 | 8 | 원소형. 원소별 특수 지팡이 |
| **합계** | **100** | **64** | **36** | 64:36 비율 (Elden Ring 184:127과 유사) |

### 3.4. 레어리티와의 관계

기존 5레어리티(Normal~Ancient) 체계는 유지하되, 무기 정체성과 분리한다.

```
무기 정체성 = 무기 이름 (예: "Flame Edge") = 고유 패시브 + 기술
레어리티 = 수치 스케일 (Normal x1.0 ~ Ancient x3.0) + 지층 수 + 이노센트 슬롯

동일 "Flame Edge"라도:
  - Normal Flame Edge: BaseATK 15, 2지층, 이노센트 2슬롯
  - Ancient Flame Edge: BaseATK 45, 4지층+심연, 이노센트 8슬롯
  → 패시브 "3타 피니셔에 화염 폭발" 동일
```

이렇게 하면 100무기 x 5레어리티 = **500개 장비 아이템**이 되어 야리코미 파밍의 깊이가 급격히 확대된다.

## 4. 계층 2 상세: 무기 기술 (Weapon Art) 시스템

### 4.1. 개요

무기 기술은 기존 시그니처 메커닉(패시브)과 별개인 **액티브 스킬**이다.

| 항목 | 설계 |
|:---|:---|
| 입력 | 특수 공격 버튼 (기존 공격/점프/대시 외 1키 추가) |
| 쿨다운 | 기술별 상이 (8~30초). MP/자원 불필요 |
| 교체 | 일반 무기: 대장간에서 교체. 특수 무기: 고정 |
| 카테고리 제한 | 기술마다 호환 카테고리 지정 (검용, 근접 공용, 전체 공용 등) |

## 6. 아이템계와의 시너지: 무기 패시브가 던전 경험을 바꾼다

### 6.1. 무기 패시브 -> 아이템계 방 유형 가중치

무기의 고유 패시브가 아이템계 절차적 생성에 영향을 줄 수 있다.

| 패시브 축 | 유리한 방 유형 | 생성 가중치 변화 |
|:---|:---|:---|
| 배후 공격 계열 (D-01, S-23) | 복잡한 지형 (벽/장애물 많음) | 통로형 방 +20% |
| 범위 공격 계열 (G-01, S-02) | 밀집 적 + 넓은 방 | 개방형 방 +15% |
| 원소 특화 (T-10, T-14) | 원소 환경 방 | 해당 원소 환경 +25% |
| 이동 연동 (B-11, D-04) | 넓은 플랫폼 구조 | 수직 구조 방 +15% |
| 파밍 보상 (D-05, D-09) | 보물 방 비율 증가 | 보물방 +10% |

**스파이크 검증:** "이 무기를 들고 아이템계에 들어가면, 그 무기의 성격에 맞는 던전이 펼쳐진다" -- 스파이크 강화.

### 6.2. 특수 무기: 보스 드랍 무기의 고정 정체성

아이템계 보스와 월드 보스를 처치하면 해당 보스의 정체성을 담은 특수 무기를 획득한다.

**특수 무기 = 고유 기술(고정) + 고유 패시브 + 보스 로어**

| 보스 | 드랍 무기 | 카테고리 | 고유 기술 | 고유 패시브 |
|:---|:---|:---|:---|:---|
| 월드보스 1층 | Builder's Remnant | Greatsword | Ground Pulse (빌더의 지진파) | 구조물 근처 ATK +20% |
| 월드보스 2층 | Echo Fang | Dagger | Phase Shift (순간 이동) | 음파 적중 시 배후 판정 |
| 아이템 대신 | Memory's Edge | Sword | Memory Slash (기억 참격) | 아이템 레벨에 비례 데미지 +0.5%/lv |

## 7. 구현 비용 분석

### 7.1. 제작 비용 매트릭스

| 구성 요소 | 수량 | 비용 유형 | 비용 규모 | Phase |
|:---|:---:|:---|:---|:---|
| 카테고리 무브셋 | 5종 | 풀 애니메이션 (5~8타 + 공중/대시) | **높음** | Phase 1-2 (이미 설계) |
| 무기 기술 | 37종 (확장 50종) | 기술당 1~2개 애니메이션 | **중간** | Phase 2-3 |
| 고유 패시브 | 100종 | 코드/데이터만 (아트 불필요) | **낮음** | Phase 2-4 (10종/Phase) |
| 무기 외형(스프라이트) | 100종 | 무기 아이콘 + 전투 스프라이트 | **중간** | Phase 2-4 |
| 밸런스 튜닝 | 지속 | 플레이테스트 + 수치 조정 | **지속** | 전 Phase |

### 7.2. 80/20 법칙 적용

**80%의 무기 정체성 = 20%의 콘텐츠가 결정:**

| 영향도 | 요소 | 제작 비용 |
|:---|:---|:---|
| **매우 높음 (핵심 20%)** | 고유 패시브 (조건+효과) | 낮음 (코드) |
| **매우 높음** | 히트박스 형태/크기/활성 프레임 | 중간 |
| **높음** | 타격감 파라미터 (히트스탑, 넉백, 사운드) | 낮음 |
| 중간 | 무기 기술 1개 | 중간 |
| 낮음 | 스탯 수치/스케일링 | 매우 낮음 |
| 낮음 | 외형/색상 차이 | 낮음 |

**결론:** 100개 고유 패시브 구현이 가장 비용 효율적인 차별화 수단이다. 애니메이션 추가 없이 코드만으로 무기 정체성의 50% 이상을 결정할 수 있다.

### 7.3. Phase별 구현 로드맵

## 8. 스프라이트 애니메이션 비용 최적화 전략

### 8.1. 전략 1: 캐릭터-무기 분리 렌더링

에르다(플레이어 캐릭터) 몸체 애니메이션과 무기 스프라이트를 별도 레이어로 렌더링한다.

```
[에르다 몸체 애니메이션] --- 카테고리별 5세트 (공유)
        |
[무기 스프라이트 오버레이] --- 무기별 100종 (교체 가능)
        |
[히트박스 데이터] --- 무기별 미세 조정 (데이터만)
        |
[이펙트 레이어] --- 패시브/기술별 VFX
```

같은 "검 3타 콤보" 모션에 다른 무기 스프라이트를 교체하면, 추가 신체 애니메이션 없이 100개 무기의 시각적 차이를 만들 수 있다.

### 8.2. 전략 2: 프로그래밍 기반 차별화

Dead Cells 개발자의 교훈: "프로그래밍으로 차별화하면 보충 애니메이션이 극적으로 줄어든다."

- 히트박스 크기/모양/타이밍을 데이터로 조정
- 히트스탑/넉백/파티클을 패시브에 따라 변경
- 같은 모션 + 다른 데이터 = 다른 느낌

### 8.3. 총 애니메이션 예상량

| 카테고리 | 기본 무브셋 | 무기 기술 | 합계 |
|:---|:---:|:---:|:---:|
| 카테고리 무브셋 5종 x 8모션 | 40 | - | 40 |
| 무기 기술 37종 x 1.5모션 | - | 56 | 56 |
| 무기 스프라이트 100종 | - | - | 100 (스프라이트) |
| **합계** | **40** | **56** | **96 애니메이션 + 100 스프라이트** |

100개 무기에 대해 96개 애니메이션은 매우 효율적이다. (무기당 평균 1개 미만)

## 10. 핵심 인사이트 요약

### 10.1. 레퍼런스에서 도출한 5대 원칙

1. **70% 공유 + 30% 고유 법칙** (Elden Ring): 대부분의 무기는 카테고리 무브셋을 공유해도 된다. 패시브와 기술이 정체성을 만든다
2. **고유 발동 조건이 전투 접근법을 바꾼다** (Dead Cells): 수치 차이가 아닌 "이 무기가 빛나는 순간"의 차이가 핵심
3. **적은 틀 x 깊은 변형** (Hades): 5카테고리로 충분. 각 카테고리 안에서 패시브/기술 조합이 깊이를 만든다
4. **프로그래밍이 아트를 대체한다** (Dead Cells): 동일 모션 + 다른 데이터 = 다른 경험. 비용 효율 극대화
5. **무기 선택이 전투 루프 자체를 변경해야 한다** (Monster Hunter): 무기가 바뀌면 플레이어의 의사결정 패턴이 바뀌어야 한다

### 10.2. ECHORIS 적용 핵심 결정 사항

| 결정 | 내용 | 근거 |
|:---|:---|:---|
| 5카테고리 유지 | 기존 5종 무기 체계 변경 없음 | 카테고리 추가는 애니메이션 비용 급증 |
| 무기 기술(Weapon Art) 도입 | 액티브 스킬 1키 추가. 교체 가능/고정 이분법 | Elden Ring 전쟁재 검증. 30% 정체성 기여 |
| 고유 패시브 100종 | 무기별 1개 고정 패시브. 7축 분배 | Dead Cells 크리티컬 조건 검증. 50%+ 정체성 기여 |
| 일반/특수 이분법 | 64 일반(기술 교체) + 36 특수(기술 고정) | Elden Ring 일반/Somber 검증 |
| 레어리티 직교 | 무기 정체성과 레어리티 분리 | 100무기 x 5레어리티 = 500장비 |
