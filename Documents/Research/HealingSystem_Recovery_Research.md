## 1. ECHORIS 현재 구현 상태

현재 코드베이스에 존재하는 회복 관련 엔티티:

| 엔티티 | 파일 | 역할 | 비고 |
|:---|:---|:---|:---|
| `HealingPickup` | `game/src/entities/HealingPickup.ts` | 고정 HP 회복 (기본 30), 1회 수거 후 영구 소멸 | 월드 배치용, 초록색 십자 그래픽 |
| `HealthShard` | `game/src/entities/HealthShard.ts` | 최대 HP 영구 증가 (기본 +10), Hollow Knight 마스크 조각 모델 | 탐험 보상, 분홍색 다이아몬드 그래픽 |
| `Healing_potion` | LDtk ItemType enum | 아이템계 드랍 아이템 | 소비 로직 미구현 |

현재는 **고정 배치 + 탐험 보상** 두 가지만 존재. 전투 연동형, 세이브포인트 리필형, 이노센트 연동형은 없음.

## 2. 레퍼런스 게임 체력 회복 방식 비교

### 2.1. 횡스크롤 액션 / 메트로베니아

#### Hollow Knight -- Soul/Focus 모델

| 항목 | 내용 |
|:---|:---|
| 주요 방식 | 적 타격 시 Soul 축적 → Focus 발동으로 HP 회복 (33 Soul = 1 마스크) |
| 캐스팅 시간 | 1.14초 (완전 무방비) |
| 보조 방식 | 벤치(세이브포인트)에서 전체 회복, 참(Charm) 조합으로 커스터마이즈 |
| 참 시너지 | Quick Focus (캐스팅 0.57초), Deep Focus (2배 회복/2배 Soul), Shape of Unn (힐 중 이동 가능) |
| 핵심 설계 | **공격 = 회복 자원 획득**. Soul을 힐에 쓸지 스펠에 쓸지 자원 분배 딜레마 |
| 긴장감 | ★★★★★. Focus 캐스팅 중 무방비 → 보스 패턴 학습 강제. 3:1 법칙 (3타격 vs 1피격) |

#### Dead Cells -- Flask + Rally 하이브리드

| 항목 | 내용 |
|:---|:---|
| 주요 방식 | Health Flask (체력 60% 회복, 체크포인트에서 리필) |
| 난이도 연동 | Boss Cell 0 = Flask 4회, BC1 = 3회, BC2 = 2회, BC3+ = 1회 → **난이도 스케일링의 핵심 레버** |
| Rally 시스템 | 피격 후 주황색 HP 표시 → 즉시 반격 시 잃은 HP 일부 회수 |
| 보조 방식 | 음식 드랍 (15-50% 회복, 확률적), Necromancy 뮤테이션 (킬 시 HP 회복) |
| 핵심 설계 | Flask 감소로 난이도를 올리면서 Rally로 **공격적 플레이를 강제** |
| 긴장감 | ★★★★. BC3+에서 Flask 1회 → Rally와 뮤테이션 의존 → 실력 기반 생존 |

#### Spelunky -- 극소 회복 모델

| 항목 | 내용 |
|:---|:---|
| 주요 방식 | Damsel 구출 → 출구 도달 시 +1 HP |
| Kapala | 적 처치 시 피 방울 드랍 → 8방울 수집 = +1 HP (희귀 아이템) |
| 보조 방식 | Kissing Booth (+1 HP 구매), 제단 헌납 (48포인트 시 +4-8 HP) |
| 핵심 설계 | 회복이 **극도로 희소** → 모든 HP가 귀중. "Damsel을 구하러 위험을 감수할까?" = 리스크/리워드 핵심 |
| 긴장감 | ★★★★★. 즉사 함정 + 제한된 힐 = 실수 비용 극대화 |

### 2.2. 소울라이크 / 로그라이트

#### Dark Souls / Elden Ring -- Estus Flask (리필식)

| 항목 | 내용 |
|:---|:---|
| 주요 방식 | Estus Flask: 본파이어에서 충전, 사용 시 느린 음주 애니메이션 |
| 확장 | DS: Kindle로 5→10→15→20회 확장. ER: Golden Seed로 수 증가, Sacred Tear로 회복량 강화 |
| 보조 방식 | DS: Humanity, 기적 주문. ER: 필드 적 처치 시 Flask 1회 보너스 충전 |
| 핵심 설계 | 본파이어 = 리필 + 적 리스폰 → **탐험 거리와 자원 관리가 직결** |
| 긴장감 | ★★★★. "지금 Flask를 쓸까, 보스전을 위해 아낄까?" 매 순간 판단 |

#### Hades -- 분산 선택지 모델

| 항목 | 내용 |
|:---|:---|
| 주요 방식 | 보스 후 Fountain Room (50% 회복), 상점 음식 구매 |
| Death Defiance | 치사량 피격 시 50% HP 부활 (최대 3회). 메타 강화로 영구 해금 |
| 보조 방식 | Chthonic Vitality (방 입장 시 +1-3 HP), Dionysus After Party, NPC 힐 이벤트 |
| 핵심 설계 | 힐 수단이 **보상 선택지 속에 분산** → "힐을 살까, 파워업을 살까?" 트레이드오프 |
| 긴장감 | ★★★. Death Defiance가 보험 역할. 힐이 풍부하지만 분산 |

#### Vampire Survivors -- 풍부한 회복 모델

| 항목 | 내용 |
|:---|:---|
| 주요 방식 | Floor Chicken (바닥 음식 드랍), 레벨업 시 Floor Chicken 선택지 |
| 보조 방식 | Recovery 스탯 (초당 HP 재생), Pummarola 아이템 (+0.2/레벨) |
| 핵심 설계 | 힐이 **빌드 선택의 일부** → Recovery 투자 vs 공격력 투자 |
| 긴장감 | ★★. 서바이벌 장르 특성상 의사결정보다 빌드 최적화에 가까움 |

### 2.3. 캐슬바니아 SotN (위키 기반 심층 분석)

**출처:** `Reference/castlevania-wiki-md/` (Save Room, Potion, Dark Metamorphosis, HP Max Up 등)

#### 회복 수단 6계층 구조

| 계층 | 수단 | 메커닉 | 비고 |
|:---|:---|:---|:---|
| **1. 세이브룸** | 다면체 → 황금 관 연출 | HP + MP 전회복, Heart 미회복 | 적 출현 불가. 가짜 세이브룸 (Succubus) 존재 |
| **2. 포션 3단계** | Potion / High Potion / EX Potion | 소량 / 대량 / 100% 회복. 즉시 소비 | 음식과 달리 직접 사용 가능 (안전) |
| **3. 음식 드랍** | Beef(100HP), Turkey(80HP), Sushi(100HP), 과일류(6-10HP) | 벽 뒤 숨김 배치 ("벽 고기") + 적 드랍 | 던져서 주워야 해서 전투 중 리스크 존재 |
| **4. HP Max Up** | Life Vessel | 최대 HP +5 영구 증가 + 현재 HP 전회복 | 비밀 장소 / 보스 드랍. 탐험 보상 |
| **5. 전투 회복** | Dark Metamorphosis 주문 | 적 타격 시 피 튀김 → 8 HP 회수 | Bloodstone 장비로 2배. Gurthang/Muramasa 검은 상시 회복 |
| **6. 장비 시너지** | Blessed Ankh (+50% 포션 효과), Topaz Circlet (독 → 힐 변환) | 장비로 회복 특성 변경 | 빌드 다양성 제공 |

#### ECHORIS 적용 포인트

- **세이브룸 전회복** → ECHORIS 세이브포인트(대장간)에 이미 정합
- **Dark Metamorphosis** → "단조열이 상처를 봉합" 테마로 행동 연동 회복의 원형
- **벽 고기** → 탐험 보상 회복 (현재 HealingPickup과 동일 컨셉)
- **포션 3단계** → 소모품 드랍 등급 차등에 적용 가능
- **장비 시너지** → 이노센트 연동 회복의 근거

### 2.4. 디스가이아 시리즈 (위키 기반 심층 분석)

**출처:** `Reference/disgaea-wiki-md/` (Hospital, Healing Power, Innocent Town, Item World 등)

#### 회복 수단 체계

| 수단 | 메커닉 | ECHORIS 시사점 |
|:---|:---|:---|
| **힐 주문** (Heal → Mega → Giga → Omega) | RES 스탯 기반 회복량. SP 소모 (4-108) | ECHORIS는 힐러 클래스 없음 → 주문 회복 부적합 |
| **소모 아이템** | 10HP(ABC Gum) - 20,000HP(Sundae)까지 20단계+ | 등급화된 소모품 드랍의 레퍼런스 |
| **병원(Hospital)** | 유료 회복/부활. **누적 이용량 → 보상 아이템** 지급 | 장기 야리코미 보상 루프로 변환 가능 |
| **이노센트 타운 의료관** | 아이템계 10층마다 출현 확률. 파티 전회복 | 아이템계 체크포인트 회복의 직접 레퍼런스 |
| **Geo Effect: Heal 20%** | 특정 패널 위에 서면 턴마다 20% 회복 | 아이템계 특수 타일 (환경 퍼즐 요소) |
| **Reverse Damage** | 공격 = 회복, 회복 = 데미지 | 아이템계 이벤트 방 변형으로 활용 가능 |

#### 핵심 발견: 아이템계 내 층간 자동 회복 없음

디스가이아의 아이템계는 **층 사이에 자동 회복이 없다.** 플레이어가 직접 힐 주문/아이템으로 회복해야 하며, 10층마다 이노센트 타운(확률적)에서만 전회복 가능. 이 설계가 "이 자원으로 다음 10층을 버틸 수 있는가?" → Mr. Gency's Exit 사용 여부 판단이라는 **핵심 의사결정**을 만든다.

디스가이아 6부터는 이노센트 타운이 아이템계 시작점으로 변경 → 진입 전 반드시 회복 가능. 이전작 대비 접근성 개선.

## 3. 힐링 시스템 디자인 원칙 (범용)

### 원칙 1: 힐링은 "의사결정"이어야 한다

모든 우수한 힐링 시스템은 플레이어에게 **트레이드오프**를 요구한다.

- Hollow Knight: Soul을 힐에 쓸까, 스펠에 쓸까?
- Hades: 상점에서 힐을 살까, Boon을 살까?
- Dark Souls: Flask를 지금 쓸까, 보스전을 위해 아낄까?

**ECHORIS 적용:** 아이템계에서 "남은 회복 자원으로 다음 지층을 도전할까, 안전하게 탈출할까?" = 탈출 판단과 회복 자원이 직결되어야 한다.

### 원칙 2: 힐링 수단은 코어 루프를 강화해야 한다

힐링 방식이 핵심 행동과 분리되면 "회복 파밍"이라는 지루한 루틴이 생긴다.

| 평가 | 게임 | 이유 |
|:---|:---|:---|
| 좋은 예 | Hollow Knight | 적 공격 → Soul 획득 → 힐. 전투 참여 장려 |
| 좋은 예 | Dead Cells Rally | 피격 후 즉시 반격 → HP 회수. 공격적 플레이 강제 |
| 나쁜 예 | Bloodborne Blood Vial 파밍 | 소진 시 이전 스테이지로 돌아가 파밍 → 코어 루프와 무관한 반복 작업 |

**ECHORIS 적용:** 회복 수단이 "전투 → 보상" 루프 안에 있어야 한다. 별도 파밍이 필요한 소모품 의존 모델은 기피.

### 원칙 3: 힐링 희소성 = 긴장감의 핵심 레버

| 희소성 | 대표 게임 | 긴장감 | 특징 |
|:---|:---|:---|:---|
| 극소 | Spelunky | ★★★★★ | 매 HP가 생사를 가름 |
| 한정 리필 | Dark Souls, Dead Cells | ★★★★ | 체크포인트 간 자원 관리 |
| 선택지 분산 | Hades | ★★★ | 힐 vs 파워업 트레이드오프 |
| 풍부 | Vampire Survivors | ★★ | 편안한 생존, 빌드 최적화 |

**ECHORIS 적용:** 아이템계 = 한정 리필(★★★★) 수준이 적합. 지층 깊이/레어리티별 Flask 수 차등으로 Dead Cells식 난이도 스케일링 가능.

### 원칙 4: 회복 유형 분류 -- 리필식 vs 소모식 vs 행동 연동식

| 유형 | 대표 게임 | 장점 | 단점 |
|:---|:---|:---|:---|
| **리필식** (체크포인트 충전) | Dark Souls, Dead Cells, Salt and Sanctuary | 파밍 불가 → 자원 관리 집중 | 체크포인트 간 거리 밸런싱 필요 |
| **소모식** (드랍/구매) | Hades, Spelunky, Castlevania | 확률적 긴장감, 탐색 보상 | 운 의존, 파밍 리스크 |
| **행동 연동식** (공격→회복) | Hollow Knight, Bloodborne, Dead Cells Rally | 코어 루프 강화, 공격적 플레이 유도 | 초보자 진입장벽 높음 |

### 원칙 5: 힐링 애니메이션의 "취약 시간"이 스킬 갭을 만든다

| 게임 | 힐링 취약 시간 | 효과 |
|:---|:---|:---|
| Hollow Knight Focus | 1.14초 무방비 | 보스 패턴 학습 강제 |
| Dark Souls Estus | 느린 음주 애니메이션 | 안전 타이밍 판단 필요 |
| Dead Cells Flask | 짧은 음용 + 이동 가능 | 긴장감 낮지만 접근성 높음 |
| Hades 음식 | 즉시 회복 | 취약 시간 0 → 전략적 힐 판단 없음 |

**ECHORIS 적용:** 취약 시간 0.5-1.0초 수준 권장. "단조 타격감" 테마에 맞는 회복 연출(단조 불꽃이 상처를 봉합하는 모션)과 결합하면 스킬 테스트 + 세계관 몰입 동시 달성.

## 4. ECHORIS 설계 조건 정리

| 조건 | 근거 | 회복 설계에 대한 제약 |
|:---|:---|:---|
| **2-Space 분리** | 월드(탐험) + 아이템계(파밍) | 두 공간에서 회복 방식/자원이 달라야 의미 |
| **세이브포인트 = 대장간/상점** | 허브 폐기 | 세이브포인트가 유일한 안전 거점 + 회복 리필 지점 |
| **아이템계 = 연속 지층** | 4x4 그리드, 레어리티별 2-5지층 | 지층 간 자동 회복 없음 → "다음 지층 도전 vs 탈출" 의사결정 |
| **멀티플레이 1-4인** | 전용 힐러 클래스 없음 (무기 카테고리 기반) | 개인 회복 + 팀 시너지 필요 (이노센트/아이템으로) |
| **스파이크 강화** | 아이템 속으로 들어가면 던전 | 회복력도 아이템계 파밍의 결과물이어야 함 |
| **횡스크롤 액션** | 실시간 전투, 단조 타격감 | 캐스팅 취약 시간 = 스킬 테스트 |
| **야리코미** | 무한 파밍, 장기 리텐션 | 회복 시스템이 빌드 다양성/최적화 축이 되어야 함 |

## 5. 후보 모델 5종 비교

### A. Estus 모델 (리필식 한정 자원)

> 세이브포인트에서 N회 충전. 사용 시 애니메이션 + 취약 시간.

| 장점 | 단점 |
|:---|:---|
| 파밍 불가 → 자원 관리 집중 | 아이템계에서 리필 지점 설계 필요 |
| 세이브포인트 가치 강화 (대장간 경유 = 리필) | 멀티에서 개인 Flask vs 공유 문제 |
| 난이도 스케일링 레버 (Flask 횟수 = 탐험 범위) | 아이템/이노센트와의 시너지 약함 |

### B. Soul/Focus 모델 (행동 연동형)

> 적 타격 → 게이지 축적 → 소모해서 회복 (캐스팅 시간 있음)

| 장점 | 단점 |
|:---|:---|
| 공격 = 자원 획득 → 전투 참여 강제 | 게이지를 힐 vs 스킬에 분배하는 설계 필요 |
| 보스전 긴장감 극대화 (캐스팅 = 리스크) | 아이템계에서 적이 없는 구간 = 회복 불가 |
| ECHORIS "단조 타격감" 테마와 정합 | 초보자 진입장벽 높음 |

### C. Flask + Rally 하이브리드 (Dead Cells형)

> 리필식 기본 + 피격 후 즉시 반격 시 잃은 HP 일부 회수

| 장점 | 단점 |
|:---|:---|
| "안전한 회복"과 "공격적 회복" 이중 레이어 | Rally 윈도우 밸런싱 까다로움 |
| 아이템계 난이도별 Flask 감소 → Dead Cells 검증 모델 | 시스템 2개를 동시에 이해해야 함 |
| 공격적 플레이 보상 (야리코미 페이스에 적합) | -- |

### D. 이노센트 연동형 (ECHORIS 고유)

> 장비에 "회복형 이노센트"가 거주 → 조건 충족 시 자동 회복

| 장점 | 단점 |
|:---|:---|
| **스파이크 직결** -- 회복력도 아이템계 파밍의 결과물 | 아이템계 첫 진입 시 이노센트 없으면 회복 불가 |
| 빌드 다양성 (공격형 vs 회복형 이노센트 트레이드오프) | 시스템 복잡도 증가 |
| 장비 교체 = 회복 전략 변경 → 야리코미 의미 부여 | 밸런싱 난이도 높음 |

### E. 복합 모델 (A + B + D 결합) -- 권장

> 기본 리필 Flask + 전투 연동 부분 회복 + 이노센트로 커스터마이즈

| 계층 | 방식 | 적용 공간 | 레퍼런스 |
|:---|:---|:---|:---|
| **1차: 세이브포인트 Flask** | 세이브포인트에서 N회 충전 | 월드 + 아이템계 | Dark Souls Estus, Dead Cells Flask |
| **2차: 전투 연동 회복** | 연속 타격 시 소량 HP 회수 | 양쪽 | Hollow Knight Soul, Dead Cells Rally, SotN Dark Metamorphosis |
| **3차: 이노센트 패시브** | 회복형 이노센트가 조건부 자동 회복 (킬 시, 방 클리어 시 등) | 아이템계 (파밍 보상) | 디스가이아 이노센트 + Vampire Survivors Recovery |
| **4차: 소모품 드랍** | 적/상자에서 즉시 회복 아이템 확률 드랍 | 양쪽 | SotN 음식/포션, 디스가이아 소모 아이템 |

## 8. 결론 및 권장 방향

### 8.1. 권장 모델: E. 복합 모델 (Flask + 전투 연동 + 이노센트)

**선택 근거:**

1. **Flask = 세이브포인트(대장간) 가치 강화** -- 탐험 범위의 물리적 한계를 설정하고, 세이브포인트 방문 동기를 만든다
2. **전투 연동 = 단조 타격감 테마 정합** -- "때릴수록 단조열이 상처를 녹인다." 코어 루프(전투) 안에 회복이 내장되어 파밍 루틴 방지
3. **이노센트 연동 = 스파이크 직결** -- 더 좋은 회복 이노센트 = 더 깊은 지층 도전 가능 = 아이템계 재진입 동기. "아이템 속에 들어가야 하는 이유"가 하나 더 생긴다
4. **난이도 스케일링** -- 레어리티별 Flask 수 차등 + 지층별 드랍률 감소 → Dead Cells가 검증한 모델

### 8.2. 프로토타입 우선순위

| 순위 | 구현 항목 | 근거 |
|:---|:---|:---|
| 1 | 세이브포인트 Flask 리필 (N회 한정) | 자원 관리의 기본 프레임워크 |
| 2 | 아이템계 보스 클리어 시 부분 회복 | 4x4 그리드 체크포인트 |
| 3 | 전투 연동 소량 회복 (콤보 완료 시) | 코어 루프 강화 검증 |
| 4 | 이노센트 패시브 회복 | Phase 2 이노센트 시스템과 동시 구현 |
