# 오프닝 씬 설계 — 거대 빌더 첫 공개 + 튜토리얼 진입

> **작성일:** 2026-04-25
> **목적:** 게임 시작 30초 안에 "거대 빌더가 움직이는" wow moment를 보여주고, 자연스럽게 튜토리얼로 진입하는 오프닝 씬을 설계한다.
> **근거:** `GiantBuilder_MovingLevel_Research.md`, `Design_Narrative_Worldbuilding.md`, `Design_Tutorial_EnvironmentalTeaching.md`
> **핵심 레퍼런스:** Inside (Playdead), Journey (산 공개), SotC (첫 거상), Made in Abyss (심연 첫 하강), HK (Forgotten Crossroads 첫 진입)

## 레퍼런스 분석 — "첫 30초"의 기술

### 1. Inside (Playdead, 2016) — 행동이 먼저, 경외가 나중

```
0초:  소년이 숲에서 나온다. 조작 설명 없음. 플레이어가 자연스럽게 오른쪽으로 이동.
5초:  배경에 트럭 헤드라이트가 보인다. 아직 위협을 모름.
15초: 첫 번째 장애물 (통나무). 자연스럽게 점프를 배운다.
30초: 배경에서 무언가가 일어나고 있다는 것을 인지. 호기심 발동.
```

**교훈:** 플레이어가 "이동"을 먼저 하게 하고, **배경에서 세계의 규모를 암시**한다. 설명하지 않는다.

### 2. Journey (thatgamecompany, 2012) — 목표를 보여준 후 움직이게 한다

```
0초:  모래사막. 아무것도 없는 공간에 캐릭터만 있다.
3초:  카메라가 천천히 위로 틸트 → 거대한 산의 빛이 보인다.
8초:  산에서 빛이 뿜어져 나온다. "저기에 가야 한다"는 것을 본능적으로 안다.
15초: 카메라가 캐릭터로 돌아온다. 조작 가능.
```

**교훈:** **"저기에 뭐가 있다"를 먼저 보여주고, 그 다음 "가봐"라고 한다.** 대사 없이.

### 3. Shadow of the Colossus — 스케일의 점진적 공개

```
첫 거상 조우:
0초:  넓은 평원. 적이 없다. 멀리 무언가가 보인다.
10초: 가까이 가면 → 땅이 흔들린다. 소리가 먼저 온다.
20초: 거상이 일어선다. 카메라가 위로 올라간다. 전체가 한 화면에 안 들어온다.
30초: 플레이어가 거상을 올려다본다. 크기의 차이가 경외감을 만든다.
```

**교훈:** **소리 → 진동 → 부분 공개 → 전체 공개** 순서. 한 번에 보여주지 않는다.

### 4. Made in Abyss — 수직 세계의 첫인상

```
1화:
- 아비스의 입구에서 아래를 내려다본다
- 바닥이 보이지 않는다
- 새가 아래로 날아가다 점점 작아진다
- "저 아래에 뭐가 있을까" — 수직 세계의 첫인상이 호기심을 만든다
```

**교훈:** **수직 세계는 "아래를 내려다보는 순간"으로 시작**해야 한다. 수평이 아니라 수직으로의 시선 유도.

### 5. Hollow Knight — 환경이 가르친다

```
Forgotten Crossroads 첫 진입:
- 좁은 통로를 지나면 갑자기 넓은 공간이 열린다
- 멀리 배경에 거대한 구조물이 보인다
- 적은 약하고 배치가 단순하다 — 탐험에 집중하게 한다
- "이 세계는 크다"는 것을 환경이 전달한다
```

**교훈:** **좁은 공간 → 넓은 공간의 전환**이 경외감을 만든다. 대비의 힘.

## ECHORIS 오프닝 씬 설계

### 설계 원칙

1. **행동이 먼저** — 플레이어가 이동/점프를 먼저 체험한 후 wow moment
2. **소리 → 진동 → 부분 공개 → 전체 공개** — SotC 순서
3. **수직으로 시선 유도** — Made in Abyss. "아래를 내려다보는 순간"
4. **좁은→넓은 대비** — HK. 좁은 통로를 지나면 대공동이 열린다
5. **대사 0줄** — 에르다는 말하지 않는다. 심볼 프롬프트만

---

### 씬 구성 — 4막 구조 (약 90초)

```
[전체 흐름]

  Act 1: 좁은 통로 (20초) — 기본 이동 학습
    ↓
  Act 2: 대공동 첫 공개 (10초) — 수직 세계의 첫인상
    ↓
  Act 3: 빌더 통과 (30초) — wow moment
    ↓
  Act 4: 착지 + 튜토리얼 시작 (30초) — 본격 게임 진입
```

---

### Act 1: 좁은 통로 (0초~20초)

> **목적:** 이동/점프를 자연스럽게 학습. "좁고 안전한 곳"에서 시작.

```
[씬 레이아웃 — 횡스크롤]

  ████████████████████████████████████████
  ██                                    ██
  ██  에르다 ──→  [2타일 높이 통로]     ██
  ██  시작점      ████████              ██
  ██              ████████    ┌─ 1타일 턱 (자연스럽게 점프)
  ██              ████████    │
  ██                          │
  ████████████████████████████████████████
```

**연출:**

| 시간 | 화면 | 플레이어 액션 | 학습 내용 |
|:-----|:-----|:-------------|:---------|
| 0초 | 검은 화면에서 페이드인. 에르다가 좁은 통로에 서 있다. 앞에 작은 빛 | 자연스럽게 →키를 누름 | 이동 |
| 5초 | 1타일 높이 턱이 앞에 | Z키 심볼 프롬프트 표시 (DEC-025) | 점프 |
| 10초 | 통로가 조금 넓어짐. 바닥에 부서진 파이프 장식 | 계속 이동 | 세계 인지 (메가스트럭처 내부) |
| 15초 | 통로 끝에 **밝은 빛**이 보임. 바람 소리가 들림 | 빛을 향해 이동 | 호기심 유도 |

**핵심:** 이 20초 동안 플레이어는 **←→ 이동 + Z 점프**만 배운다. 적 없음. 안전함. 좁은 통로가 "밖에 뭐가 있지?"라는 궁금증을 만든다.

---

### Act 2: 대공동 첫 공개 (20초~30초)

> **목적:** 수직 세계의 스케일을 한 번에 체감. "아래를 내려다보는 순간."

```
[씬 레이아웃 — 대공동 입구]

                        ← 에르다 도착 지점 (절벽 끝)
  ████████████████████▓▓
  ██  좁은 통로      ▓▓░░░░░░░░░░░░░░░░
  ████████████████████▓▓░░░░ 대  공  동 ░░░░
                      ▓▓░░░░ (The Shaft) ░░░░
                      ▓▓░░░░░░░░░░░░░░░░
                      ▓▓░░░░ ↓ 아래로   ░░░░
                      ▓▓░░░░  끝이 없다  ░░░░
                      ▓▓░░░░░░░░░░░░░░░░
                        ← 바닥이 보이지 않는다
```

**연출:**

| 시간 | 화면 | 음향 | 효과 |
|:-----|:-----|:-----|:-----|
| 20초 | 통로 끝 절벽에 도달. **카메라가 멈춘다** | 바람 소리가 커진다 | 플레이어 이동은 가능하지만 카메라가 고정 |
| 21초 | **카메라가 천천히 아래로 패닝** (1.5초간) | 깊은 반향음 (에코) | 플레이어 입력 일시 차단 (보여주기 우선) |
| 22초 | 패럴랙스 배경에 **대공동의 깊이**가 드러남. 레이어 3(원경)에 거대한 구조물 실루엣. 바닥이 보이지 않음 | 저음 드론 | 안개가 깊이에 따라 짙어짐 |
| 23초 | 카메라 패닝이 멈추고 **에르다 위치로 복귀** | -- | 조작 복원 |
| 25초 | 배경 원경에 **무언가 거대한 것이 아래에서 위로 천천히 올라오고 있다** | 매우 낮은 진동음 시작 | 아직 뭔지 모름. 실루엣만 |

**핵심:** 2초간의 카메라 하강 패닝이 "이 세계는 수직이다"를 각인한다. Made in Abyss의 첫 장면과 동일한 효과.

---

### Act 3: 빌더 통과 (30초~60초) — WOW MOMENT

> **목적:** 거대 빌더가 에르다 앞을 지나간다. 대사 없이 스케일과 경외감만으로.

```
[씬 레이아웃 — 빌더 통과]

    에르다 (절벽 끝에 서 있음)
        ↓
  ██████▓▓                              ██████
  ██████▓▓          ┌───────────┐       ██████
  ██████▓▓          │           │       ██████
  ██████▓▓          │  거대     │       ██████
  ██████▓▓          │  빌더     │       ██████
  ██████▓▓          │  (상승중) │       ██████
  ██████▓▓          │           │       ██████
  ██████▓▓          │     ◆     │ ← 공명점이 빛남
  ██████▓▓          │           │       ██████
  ██████▓▓          └───────────┘       ██████
  ██████▓▓                              ██████
         ← 대공동 좌벽          대공동 우벽 →
```

**연출 (핵심 — SotC 방식 "소리→진동→부분→전체"):**

| 시간 | 화면 | 음향 | 체감 |
|:-----|:-----|:-----|:-----|
| 30초 | 에르다가 절벽 끝에 서 있다. 아래에서 무언가 올라온다 | **저음 진동이 강해진다**. 금속 삐걱거림 | "뭔가 온다" |
| 33초 | 화면 하단에서 **거대한 금속 표면이 올라온다**. 처음에는 벽처럼 보임 | 기계 구동음 (느리고 무거운) | "벽이...올라오고 있다?" |
| 36초 | 표면에 **공명점(◆)이 청록색으로 빛나며** 지나간다 | 공명 주파수 (높은 톤, 짧게) | "빛나는 문양이 있다" |
| 39초 | **빌더의 중체(창문)가 지나간다.** 창문에서 내부의 빛이 새어나옴 | 내부 기계 소리가 잠깐 들림 | "안에 뭐가 있다" |
| 42초 | **빌더의 머리(Head)가 보인다.** 안테나/관측탑 구조. 전체 실루엣이 드러남 | 바람이 빌더 주변으로 흐른다 | "이건...기계? 생물?" |
| 45초 | 빌더가 에르다를 **지나쳐 위로 올라간다.** 에르다의 머리카락과 망토가 바람에 날린다 | 바람 소리 최고조 → 점점 사라짐 | **경외감 최고조** |
| 48초 | 빌더가 위로 사라진다. 대공동에 다시 고요. 에르다 혼자 | 고요. 바람만 | "...저걸 따라가야 하나?" |
| 50초 | **카메라가 살짝 아래를 가리킨다** — 절벽 아래에 착지 가능한 플랫폼이 보임 | -- | "아래로 내려가야 한다" |

**핵심 디테일:**

- 빌더 통과 시간은 **약 15초**. 이 15초 동안 플레이어는 **아무것도 할 수 없다** (입력 차단 아님 — 움직일 수 있지만 할 일이 없다). 자연스럽게 구경하게 됨
- 빌더는 **에르다보다 5-7 화면 크기.** 한 화면에 전체가 안 들어옴 → 부분만 보이다가 지나가면서 전체가 파악됨
- 빌더 표면에 **절차적 장식(녹, 파이프, 철근)이 보임** — "오래되었지만 아직 작동하는" 느낌
- 빌더의 공명점(◆)이 **청록색으로 빛남** — 나중에 공명정 렐릭을 얻으면 "아, 그때 본 그 빛나는 점!" 연결
- **에르다의 반응 없음.** 대사 0줄. 고개를 들어 올려다보는 것만 (시선 추적 애니메이션)

### Act 4: 착지 + 튜토리얼 시작 (60초~90초)

> **목적:** wow moment의 여운 속에서 본격 게임플레이로 자연스럽게 진입.

```
[씬 레이아웃 — 하강 시작]

  ██████▓▓                         ██████
  ██████▓▓  에르다 →  ←──── 절벽   ██████
  ██████▓▓            │            ██████
  ██████▓▓            │ 낙하      ██████
  ██████▓▓            ↓            ██████
  ██████▓▓      ┌───────────┐      ██████
  ██████▓▓      │ 착지 플랫폼│      ██████
  ██████▓▓      │  (T1 시작) │      ██████
  ██████▓▓      │           │      ██████
  ██████▓▓      │ 첫 적     │      ██████
  ██████▓▓      │ (Crawler) │      ██████
  ██████▓▓      └─────┬─────┘      ██████
  ██████▓▓            │            ██████
  ██████▓▓            ↓ 더 아래로   ██████
```

**연출:**

| 시간 | 화면 | 학습 내용 |
|:-----|:-----|:---------|
| 60초 | 절벽에서 아래로 낙하 → 안전한 플랫폼에 착지 | 낙하는 안전하다 (낙하 데미지 없음) |
| 65초 | 착지 지점에 **부서진 블레이드** (Broken Blade) | 첫 무기 획득 (자동 또는 C키) |
| 70초 | 앞에 **첫 적** (약한 Crawler) | C키 공격 학습 |
| 75초 | 적 처치 후 앞에 짧은 통로 → **세이브 포인트** 발견 | 세이브 시스템 학습 |
| 85초 | 세이브 포인트 너머로 T1 (천공의 정원) 입구가 보임 | 본격 탐험 시작 |

## 기술 사양

### 카메라 연출

```typescript
// Act 2: 대공동 첫 공개 — 카메라 하강 패닝
async showShaftReveal(): Promise<void> {
  this.player.inputLocked = true;

  // 1.5초간 카메라를 아래로 120px 패닝
  await this.camera.panTo(
    this.player.x,
    this.player.y + 120,  // 아래로
    1500,                  // 1.5초
    'easeInOut'
  );

  // 1초 대기 (심연을 바라보는 시간)
  await delay(1000);

  // 카메라 복귀
  await this.camera.panTo(
    this.player.x,
    this.player.y,
    800,
    'easeOut'
  );

  this.player.inputLocked = false;
}
```

### 빌더 통과 연출

```typescript
// Act 3: 빌더가 아래에서 위로 지나감
async showBuilderPassBy(): Promise<void> {
  // 빌더를 화면 아래 밖에 배치
  const builder = new IntroBuilder();
  builder.y = this.player.y + GAME_HEIGHT * 4;  // 화면 4개 아래
  this.worldContainer.addChild(builder);

  // 15초간 위로 이동 (빌더가 에르다를 지나침)
  const targetY = this.player.y - GAME_HEIGHT * 4;  // 화면 4개 위

  // 이동 트윈 (15초, 선형)
  await tween(builder, 'y', targetY, 15000, 'linear');

  // 빌더가 화면 밖으로 나가면 제거
  builder.destroy();
}
```

### 바람 효과

```typescript
// 빌더 통과 시 에르다에게 바람 영향
update(dt: number): void {
  if (this.introBuilder) {
    const dist = Math.abs(this.introBuilder.y - this.player.y);
    const windStrength = Math.max(0, 1 - dist / (GAME_HEIGHT * 2));

    // 에르다 스프라이트에 바람 효과
    this.player.sprite.skew.x = windStrength * 0.05;  // 살짝 기울어짐

    // 파티클: 바람에 날리는 먼지
    if (windStrength > 0.3) {
      this.spawnWindParticle(windStrength);
    }
  }
}
```

### 빌더 비주얼 (IntroBuilder)

```typescript
class IntroBuilder extends Container {
  constructor() {
    super();

    // 5-7 스크린 높이의 거대 구조물
    const HEIGHT = GAME_HEIGHT * 6;  // 6 화면 = 2160px @360
    const WIDTH = GAME_WIDTH * 0.4;  // 화면 폭의 40% = 256px @640

    // 본체 (어두운 금속 직사각형)
    const body = new Graphics();
    body.rect(0, 0, WIDTH, HEIGHT);
    body.fill(0x2a2a3e);
    body.stroke({ width: 2, color: 0x4a4a6a });
    this.addChild(body);

    // 공명점 (청록색 빛나는 점 3-5개)
    for (let i = 0; i < 4; i++) {
      const point = new Graphics();
      const py = HEIGHT * (0.2 + i * 0.2);
      point.circle(WIDTH * 0.5, py, 4);
      point.fill(0x00aacc);  // 청록 (빌더의 색)
      this.addChild(point);
      // 맥동 애니메이션 추가
    }

    // 창문 (내부 빛)
    const window = new Graphics();
    window.rect(WIDTH * 0.3, HEIGHT * 0.4, WIDTH * 0.4, HEIGHT * 0.15);
    window.fill(0x334455);
    window.stroke({ width: 1, color: 0x5a7a8c });
    this.addChild(window);

    // 절차적 장식 (녹, 파이프) — ProceduralDecorator 재사용
    // ...
  }
}
```

## 감정 곡선 다이어그램

```
감정
 ↑
 │                        ★ 빌더 전체 공개
 │                       ╱ ╲
 │                      ╱   ╲  빌더 통과 여운
 │            ◆ 대공동  ╱     ╲
 │           ╱  첫 공개╱       ╲
 │      ────╱                    ╲──── 착지 + 안정
 │     ╱                              ╲
 │  ──╱  좁은 통로                       ╲── 첫 전투
 │   (안전, 학습)                           ╲── 세이브
 │                                            ╲── 탐험 시작
 └──────────────────────────────────────────────→ 시간
  0초     20초     30초   45초   60초     90초
   Act 1    Act 2    Act 3         Act 4
```

**핵심:** Act 3의 빌더 통과가 감정 곡선의 **첫 번째 피크**. 이후 착지→전투→세이브로 안정화되면서 "방금 본 것은 뭐였지? 더 알아보고 싶다"라는 동기가 형성된다.

## 이후 연결 — "그 빌더를 다시 만난다"

| 시점 | 상황 | 감정 |
|:-----|:-----|:-----|
| **오프닝 (Act 3)** | 빌더가 지나간다. 올라다볼 수만 있다 | 경외 + "뭐지?" |
| **T1 중반** | 멀리서 빌더가 오가는 것이 배경에 보인다 | "아직 있구나" |
| **T2 초반** | 순회 빌더의 하부 플랫폼이 가까이 온다. **올라탈 수 있다** | "올라탈 수 있어?!" |
| **T2→T3** | 격벽 빌더가 길을 막고 있다. **안에 들어가야 한다** | "안에 들어가야 해?!" |
| **T2→T3 내부** | 빌더 내부 던전 클리어 → 코어 활성화 → **빌더가 움직인다** | "내가 깨웠다!" |

오프닝에서 **수동적으로 바라보기만 했던 존재**를, 게임 중반에 **올라타고, 안에 들어가고, 깨우는** 능동적 경험으로 전환한다. 이 전환이 ECHORIS의 성장 곡선과 일치한다.

## 대안 검토

### 대안 A: 빌더 위에서 시작 (탑승 상태로 오프닝)

```
에르다가 이동 중인 빌더 위에서 깨어남 → 빌더 표면 탐험 → 빌더가 정지 → 하차 → 튜토리얼
```

- **장점:** 즉시 빌더 체험. "이 게임은 이런 거다" 선언
- **단점:** 빌더의 크기를 체감할 수 없음 (위에 있으면 바닥이니까). 경외감 약화
- **판정:** **기각.** 빌더의 스케일은 "밖에서 올려다볼 때" 체감된다

### 대안 B: 빌더가 무너지는 것으로 시작

```
에르다가 빌더 잔해 위에 서 있음 → 빌더 잔해가 무너지며 낙하 → 착지 → 튜토리얼
```

- **장점:** 액션 즉시 시작. 긴장감
- **단점:** 빌더가 "무너지는 것"으로 첫인상이 고정 → "약한 존재" 이미지
- **판정:** **기각.** 빌더는 "아직 작동하는" 경외의 대상이어야 한다

### 대안 C: 빌더를 전혀 안 보여주고 나중에 서프라이즈 (채택 안)

- **단점:** 첫 30초에 wow moment가 없으면 이탈률 급증 (웹 게임 특성)
- **판정:** **기각.** 웹 게임은 첫 인상이 생명

### **채택안:** 본문의 4막 구조 (좁은 통로 → 대공동 → 빌더 통과 → 착지)
