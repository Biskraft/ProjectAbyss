## 1. 개요 (Overview)

인벤토리 UI는 플레이어가 보유한 장비 아이템(최대 20개)을 5열 4행 그리드로 표시하고, 방향키와 단축키를 통해 아이템을 선택 및 장착할 수 있는 패널이다. 화면 중앙에 반투명 오버레이와 함께 팝업되며, I키로 열고 닫는다. 아이템의 레어리티, 레벨, 아이템계 클리어 상태, 사이클 횟수가 시각적으로 표현된다.

---

## 2. 설계 의도 (Design Intent)

- **아이템계 진입의 전초:** 인벤토리는 단순한 장비 교체 화면이 아니라, 플레이어가 "어떤 아이템의 기억 속으로 들어갈지" 결정하는 출발점이다. 클리어 배지(초록 사각형)와 사이클 카운터는 플레이어가 각 아이템의 탐험 이력을 한눈에 파악하게 해준다.
- **레어리티 시각화:** Diablo 스타일의 레어리티 색상 코드를 채택하여, 아이템의 가치와 잠재력(이노센트 슬롯 수, 지층 수)을 색상만으로 즉각 판단할 수 있도록 한다.
- **키보드 전용 조작:** 마우스/터치 없이 방향키와 Z/X/C 기반 키보드만으로 완전한 인벤토리 조작을 지원한다. 웹 브라우저 게임이지만 콘솔 감각의 조작감을 유지한다.

## 3. 상세 규칙 (Detailed Rules)

### 3.1 패널 레이아웃

| 항목 | 값 |
| :--- | :--- |
| 슬롯 수 | 20 (5열 × 4행) |
| 슬롯 크기 | 20×20px |
| 슬롯 간격 | 2px |
| 패딩 | 8px |
| 패널 너비 (PANEL_W) | `5 × (20+2) + 2 + 8×2 = 128px` |
| 패널 높이 (PANEL_H) | `4 × (20+2) + 2 + 8×2 + 40 = 138px` (타이틀 + 정보 영역 포함) |
| 패널 위치 | 화면 중앙 정렬 (`floor((640-PANEL_W)/2)`, `floor((360-PANEL_H)/2)`) |
| 패널 배경색 | `0x1A1A2E` (alpha 0.95) |
| 패널 테두리색 | `0x4A4A6A` (width 1px) |
| 오버레이 배경 | `0x000000` (alpha 0.5) |
| 타이틀 텍스트 | "INVENTORY" — 8px, 흰색, 패널 내 (8, 4) |

### 3.2 슬롯 상태별 시각

| 상태 | 배경색 | 테두리 | 내부 아이콘 |
| :--- | :--- | :--- | :--- |
| 빈 슬롯 (기본) | `0x2A2A3E` | 없음 | 없음 |
| 선택됨 | `0x4A4A8A` | 없음 | 내부 아이템 표시 |
| 장착 중 | 해당 상태 배경색 유지 | 노란색 `0xFFFF00` (1px) | 내부 아이템 표시 |
| 아이템 있음 (기본) | `0x2A2A3E` | 없음 | 레어리티 색상 사각형 |

아이템이 있는 슬롯의 내부 렌더링:

```
1. 레어리티 색상 사각형: rect(3, 3, 14, 14) — 슬롯 내부 중앙
2. 레벨 인디케이터 (level > 0): rect(1, 1, 6, 6) — 검정 사각형, 좌상단
3. 클리어 배지 (worldProgress.cleared = true): rect(15, 15, 4, 4) — 초록 0x44FF44, 우하단
```

### 3.3 레어리티 색상 코드

레어리티 색상의 SSoT는 `Sheets/Content_Rarity.csv` → `rarityConfig.ts`이며, `RARITY_COLOR` 상수로 접근한다.

| 레어리티 | 색상 코드 | 색상명 |
| :--- | :--- | :--- |
| normal | `0xFFFFFF` | 흰색 |
| magic | `0x6969FF` | 파란색 |
| rare | `0xFFFF00` | 노란색 |
| legendary | `0xFF8000` | 주황색 |
| ancient | `0x00FF00` | 초록색 |

### 3.4 정보 텍스트 (infoText)

아이템 선택 시 패널 하단에 표시. 위치: 패널 내 `(PADDING, 16 + 4×(20+2) + 2 + 4)`

```
아이템 선택 시:
  줄 1: "{아이템명}{[E] 장착 중이면}  Lv{level}{C{cycle} 사이클 > 0이면} {CLR 클리어됐으면}"
  줄 2: "ATK:{finalAtk}  {RARITY 대문자}"
  줄 3: "X:Equip"

아이템 없음 / 비어있는 선택:
  "{items.length}/20 items"
```

예시:
```
Iron Sword [E] Lv3 C2 CLR
ATK:45  RARE
X:Equip
```

### 3.5 조작 체계

| 키 | 동작 |
| :--- | :--- |
| `I` | 인벤토리 열기/닫기 토글 |
| `← → ↑ ↓` | 슬롯 네비게이션 |
| `X` | 선택된 아이템 장착 |
| `ESC` / `I` | 닫기 |

네비게이션 경계 처리:
- 왼쪽 끝에서 `←`: 현재 인덱스 유지 (0 이하로 내려가지 않음)
- 오른쪽 끝에서 `→`: `min(count-1, selectedIndex+1)` (마지막 아이템 이후 이동 불가)
- 위쪽 이동: `max(0, selectedIndex - COLS)` (첫 행 이상으로 이동 불가)
- 아래쪽 이동: `min(count-1, selectedIndex + COLS)` (마지막 아이템 이후 이동 불가)

### 3.6 열림 동작

인벤토리가 열릴 때 아이템이 1개 이상 있으면 자동으로 첫 번째 아이템(인덱스 0)이 선택된다. 비어있으면 selectedIndex = -1.

### 3.7 Inventory 클래스 (게임 로직)

UI와 분리된 순수 로직 계층이다.

| 메서드 | 동작 |
| :--- | :--- |
| `add(item)` | 슬롯이 가득 차면 false 반환, 아니면 items 배열에 추가 후 true |
| `remove(uid)` | uid로 아이템 제거. 장착 중이면 equipped = null |
| `equip(uid)` | uid로 아이템 찾아 equipped로 지정. 존재하지 않으면 무시 |
| `unequip()` | equipped = null |
| `getWeaponAtk()` | equipped가 있으면 finalAtk 반환, 없으면 BARE_HAND_ATK 반환 |
| `getById(uid)` | uid로 아이템 조회 |

## 4. 엣지 케이스 (Edge Cases)

| 상황 | 처리 방식 |
| :--- | :--- |
| 인벤토리 가득 찬 상태에서 아이템 드롭 | `Inventory.isFull` 확인 후 add() 거부. 게임 씬에서 "인벤토리 가득" 처리 담당 |
| 아이템 0개인 상태에서 방향키 | count === 0이면 early return. 선택 변경 없음 |
| 선택 인덱스가 -1인 상태에서 X키 장착 | `inventory.items[-1]` = undefined. equipSelected()에서 item이 falsy이면 아무 동작 없음 |
| 같은 아이템을 두 번 장착 시도 | equipped가 이미 해당 uid이므로 덮어쓰기. 동작상 무해 |
| 장착 중인 아이템을 remove() | equipped = null로 자동 해제. 다음 refresh() 호출 시 테두리 사라짐 |
| rare 아이템이 장착 중일 때의 테두리 색상 충돌 | rare 레어리티 색 = `0xFFFF00`이고 장착 테두리도 `0xFFFF00`이므로 시각적으로 구분 불가. 장착 테두리 색상을 다른 색(예: `0xFFFFFF`)으로 변경 필요 |
| 20개 중 19번째 아이템 선택 후 ↓ 이동 | `min(count-1, 18 + 5) = min(19, 23) = 19`. 마지막 아이템으로 이동. 정상 동작 |
| 아이템이 5개 미만일 때 ↓ 이동 | 5번째 행 아이템이 없으므로 count-1 한계에서 멈춤 |

## 5. 검증 체크리스트 (Acceptance Criteria)

### 기능 검증

- [ ] I키를 누르면 인벤토리가 열리고 다시 누르면 닫힘
- [ ] 아이템이 있을 때 인벤토리가 열리면 첫 번째 슬롯이 자동 선택됨
- [ ] 방향키로 슬롯을 이동할 수 있으며 경계에서 더 이상 이동하지 않음
- [ ] X키로 선택된 아이템을 장착하면 해당 슬롯에 노란 테두리가 표시됨
- [ ] 장착된 아이템의 정보 텍스트에 `[E]` 태그가 표시됨
- [ ] 레어리티별 색상이 Rarity CSV와 일치함 (normal = 흰, magic = 파랑, rare = 노랑, legendary = 주황, ancient = 초록)
- [ ] 아이템계를 클리어한 아이템의 슬롯 우하단에 초록 사각형이 표시됨
- [ ] 레벨이 1 이상인 아이템의 슬롯 좌상단에 검정 사각형이 표시됨
- [ ] 사이클이 1 이상인 아이템의 정보 텍스트에 `C{cycle}` 태그가 표시됨
- [ ] 인벤토리가 가득 찬 상태에서 새 아이템을 얻으려 하면 추가되지 않음

### 경험 검증

- [ ] 레어리티 색상만 보고 강력한 아이템(legendary, ancient)을 즉시 식별할 수 있음
- [ ] 아이템계를 클리어한 아이템과 미클리어 아이템을 배지만으로 구분할 수 있음
- [ ] 인벤토리를 열고 닫는 전환이 즉각적이고 부드러움
