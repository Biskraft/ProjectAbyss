## 1. 개요 (Overview)

알림 시스템은 게임 이벤트를 플레이어에게 즉각 전달하는 네 가지 컴포넌트로 구성된다. 토스트(Toast)는 아이템 획득, 레벨업, 보스 클리어 같은 게임 이벤트 메시지를 전달한다. 데미지 넘버(DamageNumber)는 전투 중 타격값을 수치로 시각화하여 전투 피드백을 강화한다. 튜토리얼 힌트(TutorialHint)는 세션 내 최초 1회에 한해 새 기능을 안내한다. 조작 가이드(ControlsOverlay)는 항상 화면 우상단에 표시되어 조작법을 상기시킨다.

---

## 2. 설계 의도 (Design Intent)

- **사쿠라이 원칙 적용:** 데미지 넘버는 "데미지 수치가 시각적으로 팝해야 한다"는 사쿠라이 원칙을 따른다. 스케일 펀치(spawn 시 크게 등장 후 1.0으로 수렴)와 단계별 색상/크기 분류로 타격의 강도를 직관적으로 전달한다.
- **Flow 이론 적용:** 튜토리얼 힌트는 새 능력이 해금된 직후 맥락에 맞게 한 번만 표시하여, 플레이 흐름을 끊지 않으면서 학습을 지원한다 (Csikszentmihalyi의 스캐폴딩 원칙).
- **레이어 분리:** 4가지 컴포넌트는 각각 독립적인 Container를 유지하여 z-order와 생명주기를 독립적으로 관리한다.

## 3. 상세 규칙 (Detailed Rules)

### 3.1 토스트 알림 (ToastManager)

#### 일반 토스트 (show)

| 항목 | 값 |
| :--- | :--- |
| 표시 지속시간 | 2,000ms |
| 페이드 시작 시점 | 500ms 경과 후 (TOAST_DURATION - FADE_START = 2000 - 1500 = 500ms 남은 시점) |
| 페이드 공식 | `alpha = max(0, timer / (TOAST_DURATION - FADE_START))` |
| 텍스트 위치 | x = 320 (640px 캔버스 중앙), anchor.x = 0.5 |
| 첫 번째 토스트 y | 40 |
| 스택 간격 | 14px (다음 토스트는 이전 토스트 +14px 아래) |
| 부유 속도 | y -= dt × 0.01 (초당 약 10px 위로 이동) |
| 폰트 크기 | 8px |
| 기본 색상 | `0xFFFFFF` (흰색). 호출 시 color 파라미터로 변경 가능 |

페이드 시점 정확한 계산:
```
timer가 2000ms에서 시작
timer < (2000 - 1500) = 500ms 일 때 페이드 시작
alpha = max(0, timer / 500)
timer = 0이 되면 제거
```

#### 대형 토스트 (showBig)

| 항목 | 값 |
| :--- | :--- |
| 지속시간 | 기본 3,000ms (호출 시 durationMs 파라미터로 변경 가능) |
| 텍스트 위치 | x = 320, y = 180 (640×360 캔버스 정중앙). anchor = (0.5, 0.5) |
| 폰트 크기 | 16px |
| 기본 색상 | `0xFFD700` (금색) |
| 용도 | 보스 클리어, 레벨업, 아이템계 지층 완료 등 중요 이벤트 |

#### 스택 동작

여러 토스트가 동시에 표시될 때, 각 토스트는 독립적으로 타이머를 갖는다. 스택 y 위치는 생성 시점의 `toasts.length`를 기준으로 고정되며, 이후 다른 토스트가 제거되어도 y가 재정렬되지 않는다 (이미 존재하는 구현상 제약).

#### 씬 전환 시 정리

`clear()`를 호출하면 모든 활성 토스트가 즉시 제거된다. 씬 exit 시 반드시 호출해야 한다.

### 3.2 데미지 넘버 (DamageNumberManager)

#### 5단계 분류 체계

| 조건 | 색상 | 폰트 크기 | 스케일 펀치 |
| :--- | :--- | :--- | :--- |
| critical = true | `0xFF4444` (빨강) | 12px | 2.5배 |
| heavy = true | `0xFFFF44` (노랑) | 11px | 2.2배 |
| damage > 30 | `0xFF6644` (주황-빨강) | 10px | 1.8배 |
| damage > 15 | `0xFF8844` (주황) | 9px | 1.5배 |
| 그 외 | `0xFFFFFF` (흰색) | 8px | 1.3배 |

분류 우선순위: critical > heavy > damage > 30 > damage > 15 > 기본값

#### 물리 파라미터

| 항목 | 값 |
| :--- | :--- |
| 생존 시간 (LIFETIME) | 900ms |
| 초기 상승 속도 (vy) | -30 - -50 px/s (랜덤) |
| 중력 가속도 | +60 px/s² (감속 효과) |
| 최대 하강 속도 | +50 px/s (FLOAT_SPEED) |
| x 오프셋 | ±4px (랜덤) |
| 페이드 시작 | LIFETIME의 60% 경과 후 (540ms에서 페이드 시작) |
| 페이드 공식 | `alpha = max(0, timer / (LIFETIME × 0.4))` |

#### 스케일 펀치 애니메이션

| 항목 | 값 |
| :--- | :--- |
| 펀치 지속시간 (PUNCH_DURATION) | 150ms |
| 스케일 공식 | `scale = 1 + (baseScale - 1) × (1 - t)²` (탄성 이즈-아웃) |
| t 정의 | `t = 1 - (punchTimer / PUNCH_DURATION)` (0 → 1) |
| 펀치 종료 후 | scale = 1.0으로 고정 |

스케일 공식 예시 (critical, baseScale = 2.5):
```
t = 0.0 (시작): scale = 1 + 1.5 × 1.0 = 2.5  (최대 크기)
t = 0.5 (중간): scale = 1 + 1.5 × 0.25 = 1.375
t = 1.0 (종료): scale = 1 + 1.5 × 0.0 = 1.0  (정상 크기)
```

### 3.3 튜토리얼 힌트 (TutorialHint)

#### 표시 규칙

| 항목 | 값 |
| :--- | :--- |
| 표시 위치 (y) | 16px (BOX_Y) — 화면 상단 |
| 표시 위치 (x) | GAME_WIDTH / 2 = 320px (중앙 정렬) |
| 지속시간 | 4,000ms |
| 페이드 지속시간 | 500ms (마지막 500ms에서 페이드) |
| 동시 표시 | 1개만 (panel이 있으면 다음 tryShow() 무시) |
| 세션 내 중복 표시 | 동일 id는 절대 2회 표시하지 않음 (Set<string>으로 추적) |

#### 패널 디자인

| 항목 | 값 |
| :--- | :--- |
| 배경색 | `0x000000` (alpha 0.7) |
| 테두리색 | `0xFFFFFF` (alpha 0.3, width 1px) |
| 모서리 반경 | 3px |
| 패딩 | x: 12px, y: 6px |
| 폰트 크기 | 8px, 흰색 |

#### tryShow(id, text) 호출 규칙

```
1. shown.has(id)이면 → 즉시 반환 (이미 표시된 힌트)
2. panel이 존재하면 → 즉시 반환 (이미 표시 중인 힌트 있음)
3. shown.add(id) 등록
4. 패널 생성 및 timer = DISPLAY_DURATION(4000ms) 설정
```

X키 수동 해제 기능: `InputManager` 참조를 가지나, 현재 구현에서는 자동 타이머 만료만 처리한다. X키 해제는 미구현.

## 4. 엣지 케이스 (Edge Cases)

### 토스트

| 상황 | 처리 방식 |
| :--- | :--- |
| 토스트가 매우 많이 쌓이는 경우 (10개 이상) | y가 화면 하단을 넘어감. 최대 토스트 수 제한(예: 5개) 미구현 |
| showBig와 show가 동시에 표시되는 경우 | showBig는 화면 중앙(y=180), show는 상단(y=40+)에 배치되므로 겹칠 수 있음 |
| clear() 미호출 상태에서 씬 전환 | 이전 씬의 토스트가 새 씬에서 계속 표시됨 |
| 빈 메시지 문자열 전달 | 빈 BitmapText 생성. 패널은 표시되나 내용 없음 |

### 데미지 넘버

| 상황 | 처리 방식 |
| :--- | :--- |
| damage = 0 (0 데미지) | 흰색 "0" 표시. 기본 단계로 처리 |
| critical이 false이고 heavy가 false인데 damage > 30 | 주황-빨강(0xFF6644) 처리됨. 정상 동작 |
| 수백 개의 데미지 넘버가 동시에 생성되는 경우 | 900ms 후 제거. 피크 렌더링 부하 발생 가능. 최대 동시 표시 수 제한 미구현 |
| 스케일 펀치 도중 항목이 제거되는 경우 | timer <= 0이면 parent.removeChild 후 splice. 정상 처리 |

### 튜토리얼 힌트

| 상황 | 처리 방식 |
| :--- | :--- |
| 게임을 재시작해도 same ID의 힌트가 다시 표시됨 | shown Set은 세션 메모리에만 존재. 게임 재시작(페이지 새로고침) 시 초기화됨 — 의도된 동작 |
| text가 매우 길어 패널이 화면 너비를 초과하는 경우 | 패널 너비가 텍스트 크기에 따라 동적으로 계산됨. label.width + 24. 화면 밖으로 나갈 수 있음 |
| 동시에 tryShow를 연속 호출하는 경우 | 첫 번째 것만 표시. 나머지는 panel 존재 확인으로 거부됨. 큐잉 없음 |
| destroy() 후 tryShow() 호출 | panel이 null이므로 정상 진행. container가 제거된 상태이므로 addChild에서 오류 가능 |

## 5. 검증 체크리스트 (Acceptance Criteria)

### 기능 검증

- [ ] 일반 토스트가 표시 후 2초 내에 자동으로 사라짐
- [ ] 마지막 500ms에서 토스트가 페이드 아웃되며 위로 이동함
- [ ] 여러 토스트가 동시에 표시될 때 서로 겹치지 않고 세로로 쌓임
- [ ] showBig 토스트가 화면 정중앙에 금색 대형 텍스트로 표시됨
- [ ] 씬 전환 시 clear()를 호출하면 모든 토스트가 즉시 사라짐
- [ ] critical 데미지가 빨간 12px 텍스트로 크게 팝업됨
- [ ] heavy 데미지가 노란 11px 텍스트로 표시됨
- [ ] 기본 데미지가 흰 8px 텍스트로 표시됨
- [ ] 데미지 넘버가 900ms 후 완전히 사라짐
- [ ] 튜토리얼 힌트가 4초 후 자동으로 사라짐
- [ ] 동일 ID의 튜토리얼 힌트가 세션 내 두 번 이상 표시되지 않음
- [ ] 조작 가이드가 화면 우상단에 항상 표시됨

### 경험 검증

- [ ] 강한 타격(critical)과 약한 타격(기본)을 데미지 넘버 색상만 보고 즉시 구분할 수 있음
- [ ] 스케일 펀치가 타격의 임팩트를 강화함 (단조 타격감 강조)
- [ ] 튜토리얼 힌트가 플레이 흐름을 방해하지 않고 자연스럽게 사라짐
- [ ] 토스트 알림이 게임 화면의 핵심 시야(중앙 전투 영역)를 가리지 않음
