## 1. 개요 (Overview)

본 문서는 아이템계 내 보스 조우부터 귀환 결과 제시까지의 UI 흐름을 정의한다. 세 개의 독립 컴포넌트가 시퀀스를 구성한다.

**RR-01 보스 등급 표시**는 보스 전투 중 기존 HP 바 위에 Grade Label을 추가하여 "어떤 보스인가"를 즉각 전달한다.

**RR-02 지층 클리어 레벨업 팝업**은 보스를 처치하고 아이템이 레벨업하는 순간 즉각적인 보상 피드백을 제공한다. 이것이 지층 단위 만족감의 핵심이다.

**RR-03-RR-05 귀환 결과 화면**은 아이템계 전체 다이브의 결산이다. 플레이어가 투자한 시간과 리스크에 대한 보상을 수치로 시각화하여 "이번 다이브가 가치 있었다"는 확신을 심는다. 야리코미 루프에서 이 화면이 주는 만족감은 다음 다이브의 동기가 된다.

## 2. 설계 의도 (Design Intent)

### MDA Aesthetic 목표

| 컴포넌트 | 1차 Aesthetic | 2차 Aesthetic | 달성 방법 |
|:---|:---|:---|:---|
| RR-01 보스 등급 | Challenge | Sensation | 등급 색상이 "이 보스는 다르다"는 긴장감을 시각적으로 선언 |
| RR-02 레벨업 팝업 | Sensation | Achievement | 레벨 카운트업 + 스탯 수치화 = 두드림의 결과가 눈에 보임 |
| RR-03 귀환 결과 | Achievement | Discovery | 다이브 전후 비교 + 기억 단편 상태 변화 = 투자 가치 확인 |
| RR-04 사망 결과 | Submission | --- | 고통스러운 손실로 Mr. Gency Exit 학습 동기 생성 |

### SDT 검증

- **Autonomy:** 레벨업 팝업은 Z로 조기 닫기 가능. 결과 화면은 Z로 애니메이션 스킵 가능. 사망 결과 화면은 플레이어가 읽을 시간 충분히 보장.
- **Competence:** 귀환 결과의 before/after 수치 비교가 "내 선택이 이 성장을 만들었다"는 숙련 확인을 제공한다.
- **Relatedness:** 기억 단편 상태 변화("ATK Boost Lv.3 subdued!")가 아이템 내부 존재와의 관계 해소를 서사적으로 전달한다.

### 스파이크 검증

귀환 결과 화면은 스파이크("아이템의 기억이 던전이 된다")의 물질화다. "이 아이템 안에서 전투를 벌였고, 그 결과 이 아이템이 영구적으로 더 강해졌다"를 수치로 증명하는 화면이다. 스파이크를 희석하지 않으며, 오히려 다음 다이브의 동기를 강화한다.

## 3. 상세 규칙 (Detailed Rules)

### 3.1 보스 등급 표시 (Boss Grade Display) — RR-01

#### 3.1.1 개요

`UI_HUD_MasterPlan.md` §3.2에 정의된 보스 HP 바(200x6, x=220, y=338)를 기반으로 Grade Label과 보스명을 추가한다. 보스 HP 바 코드를 수정하지 않으며, 레이어를 상단에 추가하는 방식으로 구현한다.

#### 3.1.2 레이아웃 다이어그램

```
640x360 하단 중앙 영역 (y=318~350)

             [Grade Label]
              [Boss Name]
  [████████████████████░░░░░░░░░░░]
  x=220                         x=420
  |←────────────── 200px ───────────────→|
```

**좌표 및 크기 사양:**

| 요소 | x | y | 크기 | 정렬 |
|:---|:---|:---|:---|:---|
| Grade Label | 320 (화면 중앙) | 318 | fontSize 8 | 중앙 정렬 |
| Boss Name | 320 (화면 중앙) | 328 | fontSize 8 | 중앙 정렬 |
| HP Bar | 220 | 338 | 200x6 | 기존 §3.2 유지 |

Grade Label과 Boss Name 사이 간격: 10px (y 좌표 차이)
Boss Name과 HP Bar 사이 간격: 10px

#### 3.1.3 등급별 색상 및 레이블

| 보스 등급 | 인게임 레이블 | 색상 | 해시 |
|:---|:---|:---|:---|
| Item General | `ITEM GENERAL` | 흰색 | #FFFFFF |
| Item King | `ITEM KING` | 파란색 | #6969FF |
| Item God | `ITEM GOD` | 노란색 | #FFFF00 |
| Item Great God | `ITEM GREAT GOD` | 주황색 | #FF8000 |

보스 등급은 지층 번호와 레어리티에 따라 결정된다. 아이템 레어리티와 등급의 대응은 `System_ItemWorld_Core.md`의 보스 체계를 따른다.

Boss Name: fontSize 8, 색상 #FFFFFF (고정 흰색). 보스 이름은 "Memory Guardian" 또는 콘텐츠 정의된 고유명사.

Grade Label: fontSize 8, 색상은 위 표의 등급별 색상 적용.

#### 3.1.4 표시/소멸 규칙

- **표시 트리거:** 보스 엔티티가 활성화되고 HP > 0인 상태. 보스 방 진입 시 즉시 표시. 페이드인 없음 (즉각 표시).
- **소멸 트리거:** 보스 HP = 0이 되는 순간. 0.5초에 걸쳐 alpha 1.0 → 0.0으로 페이드아웃. HP 바 소멸과 동기화.
- **아이템계 내 일반 전투 중:** 보스 HUD 미표시. 보스 방 RoomType=Boss에서만 활성화.

### 3.2 지층 클리어 레벨업 팝업 (Stratum Clear Level-up Popup) — RR-02

#### 3.2.1 개요

보스 처치 후 아이템이 레벨업할 때 즉각 표시되는 오버레이 팝업이다. 지층 클리어의 물질적 보상(레벨+스탯 증가)을 순간적으로 확인시킨다. 자동 닫기 가능하며 [Z]로 조기 닫기도 가능하여 반복 플레이를 방해하지 않는다.

#### 3.2.2 레이아웃 다이어그램

```
640x360 화면 중앙 오버레이

┌──────────────────────────────────────────────┐
│ 반투명 검정 오버레이 alpha 0.5                │
│                                               │
│       ╔══[ STRATUM CLEARED! ]══╗             │
│       ║                        ║             │
│       ║  Iron Blade            ║  (레어리티 색상 텍스트)
│       ║  Lv.3 -> Lv.4          ║  (골드 #FFD700)
│       ║                        ║             │
│       ║  ATK:  45 -> 52  (+7)  ║  (초록 #22DD44)
│       ║  INT:   0 ->  0        ║  (회색 #666666, 변화 없음)
│       ║                        ║             │
│       ║  ATK Boost Lv.2        ║  (기억 단편 복종 시만 표시)
│       ║  Memory Shard Subdued!     ║  (초록 #22DD44)
│       ║                        ║             │
│       ║  [Z] Continue          ║             │
│       ╚════════════════════════╝             │
│                                               │
└──────────────────────────────────────────────┘

패널 크기: 240x100px
패널 위치: x=200, y=130 (640x360 기준 중앙, Math.floor((640-240)/2), Math.floor((360-100)/2))
```

#### 3.2.3 패널 사양

| 속성 | 값 |
|:---|:---|
| 패널 크기 | 240x100px |
| 패널 x, y | 200, 130 |
| 배경색 | #1A1A2E, alpha 0.95 |
| 보더 | #4A4A6A, 1px |
| 타이틀 텍스트 | "STRATUM CLEARED!", fontSize 8, #FFFFFF |
| 패딩 | 8px (상하좌우) |
| 오버레이 | 전체 화면 #000000, alpha 0.5 |

#### 3.2.4 콘텐츠 라인 사양

| 라인 | 내용 | 색상 | fontSize |
|:---|:---|:---|:---|
| 아이템명 | 아이템 이름 (예: "Iron Blade") | 레어리티 색상 (`RARITY_COLOR[rarity]`) | 8 |
| 레벨 변화 | "Lv.N -> Lv.M" | #FFD700 (골드) | 8 |
| 스탯 변화 | "ATK: X -> Y  (+Z)" | 증가 시 #22DD44 (초록), 변화 없음 #666666 | 7 |
| 기억 단편 복종 | "[기억 단편명] Lv.N  Memory Shard Subdued!" | #22DD44 (초록) | 7 |
| 닫기 안내 | "[Z] Continue" | #AAAAAA | 6 |

스탯 변화 라인은 레벨업으로 실제 변화한 스탯만 표시한다. 0 변화 라인은 표시하지 않는다. 단 ATK와 INT는 0이라도 항상 표시한다(게이트 달성 여부 추적용).

기억 단편 라인은 해당 지층 탐색 중 복종(Tamed)시킨 기억 단편가 1개 이상 있을 때만 표시한다. 2개 이상이면 "N Memory Shards Subdued!" 단일 라인으로 요약한다.

#### 3.2.5 트리거 및 닫기 규칙

- **표시 트리거:** 보스 처치 → 아이템 레벨업 처리 완료 직후. 보스 소멸 애니메이션(HP 바 페이드아웃 0.5초) 이후 표시.
- **자동 닫기:** 표시 후 3.0초 경과 시 자동 닫기.
- **수동 닫기:** [Z] 입력 시 즉시 닫기.
- **애니메이션 진행 중 [Z]:** 애니메이션 스킵 → 최종 상태로 즉시 전환. 3.0초 카운터는 최종 상태 전환 시점부터 시작.
- **닫기 후:** 다음 지층 전환 처리 실행. 최종 지층 클리어인 경우 §3.3 귀환 결과 화면으로 진행.
- **레벨업 없는 지층 클리어:** RR-02 표시 안 함. 단, 재사이클 시 레벨업이 없으면 "STRATUM CLEARED" 토스트(`UI_Notifications.md` 규격) 1초 표시.

#### 3.2.6 애니메이션 시퀀스

| 단계 | 내용 | 지속 시간 | 누적 시간 |
|:---:|:---|:---:|:---:|
| 1 | 패널 하단에서 슬라이드업 진입 (y+20 → y, ease-out) | 200ms | 200ms |
| 2 | 아이템명 + 타이틀 나타남 | 100ms | 300ms |
| 3 | 레벨 카운트업 (Lv.N이 M까지 1씩 증가, 각 스텝 150ms) | 150ms × (레벨 차이) | 300ms + α |
| 4 | 스탯 라인 순차 표시 (ATK, INT, HP ... 각 200ms 간격) | 200ms × 스탯 수 | α + β |
| 5 | 기억 단편 라인 표시 (있을 때) | 200ms | β + 200ms |
| 6 | "[Z] Continue" 나타남 | 100ms | 전체 완료 |

레벨 차이가 3 이상인 경우(다이브 중 여러 지층 클리어 불가 — 지층별 1레벨 고정이므로 레벨 차이는 항상 1): 레벨 카운트업 스텝은 항상 1회다.

### 3.3 귀환 결과 화면 (Return Result Screen) — RR-03

#### 3.3.1 개요

아이템계에서 월드로 귀환할 때 표시되는 전체 화면 결산 UI다. 정상 완료(모든 지층 클리어)와 Mr. Gency Exit(긴급 탈출) 두 경우 모두 이 화면을 사용한다. 야리코미 루프의 결산 지점으로, 플레이어가 "이번 다이브가 어땠는가"를 한눈에 파악한다.

이것은 ECHORIS에서 가장 중요한 단일 UI 화면이다. 이 화면의 만족도가 다음 다이브 결정을 좌우한다.

#### 3.3.2 레이아웃 다이어그램

| 아이템명 | 레어리티 색상 (`RARITY_COLOR[rarity]`) | 8 |
| 레어리티 배지 | 레어리티 색상 | 7 |
| 레벨 변화 라인 | #FFD700 (골드) | 8 |
| Cycle 라인 | #AAAAAA (회색) | 7 |

레벨 변화에서 "+N levels" 텍스트는 레벨 차이를 표시한다. 1레벨 증가 시 "Lv.1 -> Lv.2  (+1 level)", 복수 시 "+N levels".

Cycle 라인은 이번 다이브 중 재사이클이 발생한 경우에만 표시한다. 재사이클 미발생 시 라인 없음.

#### 3.3.5 STAT CHANGES 섹션

```
-- STAT CHANGES --
ATK:   30 -> 52  (+22)   [^]
INT:    0 ->  0  ( +0)
HP:     0 -> 10  (+10)   [^]
```

| 조건 | 색상 |
|:---|:---|
| 증가 (+N > 0) | #22DD44 (초록) + [^] 상승 아이콘 |
| 변화 없음 (+0) | #666666 (회색), [^] 없음 |
| 감소 (이론상 발생 안 함) | #FF4444 (빨강) + [v] 하락 아이콘 |

표시 스탯: ATK, INT, HP. 추후 스탯 추가 시 이 섹션에 순차 추가.

수치 정렬: "ATK:" 레이블 우측 정렬 고정 폭(3글자), 수치는 우측 정렬(최대 4자리 가정), 화살표(->) 고정 위치, 델타 (+N) 우측 정렬. fontSize 7.

#### 3.3.6 MEMORY_SHARDS 섹션

```
-- MEMORY_SHARDS --
[*] ATK Boost Lv.3   NEW! Subdued   ← 이번 다이브에서 새로 복종
[!] HP Boost Lv.2    Wild           ← Wild 상태로 이번 다이브 종료
[ ] DEF Boost Lv.1   Subdued        ← 이전 다이브에서 복종, 이번 변화 없음
```

| 아이콘 | 조건 | 색상 |
|:---|:---|:---|
| `[*]` | 이번 다이브에서 Tamed 전환 (NEW!) | #22DD44 (초록) |
| `[!]` | Wild 상태로 슬롯에 존재 | #FF4444 (빨강) |
| `[ ]` | 이미 Tamed, 이번 다이브 변화 없음 | #888888 (회색) |

"NEW! Subdued" 태그는 이번 다이브(진입~귀환 사이)에서 Tamed 전환된 기억 단편에만 붙는다.

기억 단편가 0개인 슬롯은 표시하지 않는다. 슬롯이 모두 비어 있으면 "No Memory Shards" 단일 회색 텍스트.

레어리티별 슬롯 수(Normal 2 ~ Ancient 8)에 따라 표시 줄 수가 달라진다. Ancient(8슬롯)의 경우 공간 초과가 우려되므로 최대 6줄 표시, 나머지는 "+N more" 요약 처리한다.

fontSize 7.

#### 3.3.7 STRATA PROGRESS 섹션

```
-- STRATA PROGRESS --
[V] Stratum 1   [V] Stratum 2
[ ] Stratum 3
```

| 기호 | 조건 | 색상 |
|:---|:---|:---|
| `[V]` | 이번 또는 이전 다이브에서 클리어 완료 | #22DD44 (초록) |
| `[>]` | 이번 다이브에서 도달했으나 클리어 미완 | #FFD700 (골드) |
| `[ ]` | 미도달 | #666666 (회색) |

지층 블록은 2개씩 한 줄에 배치한다. Legendary/Ancient(4지층+심연)의 경우:
- 1열: `[V] Stratum 1   [V] Stratum 2`
- 2열: `[V] Stratum 3   [ ] Stratum 4`
- Ancient 전용: `[ ] Abyss` (별도 줄)

fontSize 7.

#### 3.3.8 LOOT 섹션

```
-- LOOT --
Gold earned:   1,250
Items found:   3
```

- Gold earned: 이번 다이브 전체에서 획득한 골드 합계. 천 단위 쉼표 포맷.
- Items found: 이번 다이브에서 픽업한 아이템(소비/장비 포함) 개수.
- fontSize 7, 색상 #AAAAAA.

#### 3.3.9 표시 조건 및 트리거

- **정상 완료 트리거:** 최종 지층의 보스 처치 → RR-02 레벨업 팝업 닫기 → 다이브 연출(역방향) → 월드 착지 → 귀환 결과 화면 표시.
- **Mr. Gency Exit 트리거:** RR-05 확인 UI에서 YES 선택 → 다이브 연출(역방향) → 월드 착지 → 귀환 결과 화면 표시.
- **닫기:** [Z] 입력 시 화면 닫기 → 월드 정상 플레이 재개.
- **애니메이션 중 [Z]:** 현재 진행 중인 애니메이션 스텝을 모두 스킵하고 최종 상태 즉시 표시. 최종 상태에서 [Z]는 닫기.

#### 3.3.10 Mr. Gency Exit와 정상 완료의 차이

두 경우 모두 동일한 RR-03 화면을 사용하며, 타이틀만 달라진다.

| 귀환 방식 | 타이틀 | 비고 |
|:---|:---|:---|
| 모든 지층 클리어 | `DIVE COMPLETE` | 흰색 #FFFFFF |
| Mr. Gency Exit | `DIVE COMPLETE` (동일) | 흰색 #FFFFFF. 하단에 작은 회색 텍스트 "Emergency Exit used" 추가 |
| 사망 | `DIVE FAILED` | 빨간 #FF4444. §3.4 참조 |

"Emergency Exit used" 텍스트: fontSize 6, #888888, LOOT 섹션 아래 배치.

#### 3.3.11 애니메이션 시퀀스

이 시퀀스는 ECHORIS 야리코미 루프에서 보상 감각의 핵심이다. 각 스텝이 순차적으로 플레이되어 3초간 만족스러운 결산 리듬을 형성한다.

| 단계 | 내용 | 지속 시간 | 누적 시간 |
|:---:|:---|:---:|:---:|
| 1 | 검정 배경 페이드인 + 패널 보더 나타남 | 300ms | 300ms |
| 2 | 타이틀 "DIVE COMPLETE" 나타남 | 150ms | 450ms |
| 3 | ITEM 섹션: 아이템명 + 레어리티 배지 나타남 | 200ms | 650ms |
| 4 | ITEM 섹션: 레벨 카운트업 (Lv.1 → Lv.2 → ... → Lv.N, 각 300ms + 펀치 스케일 1.0→1.2→1.0 100ms) | 300ms × 레벨 차이 | 650ms + α |
| 5 | STAT CHANGES: ATK, INT, HP 수치 순차 카운트업 (각 200ms 간격 stagger) | 200ms × 스탯 수 | α + β |
| 6 | MEMORY_SHARDS: 기억 단편 라인 순차 표시 (각 200ms 간격) | 200ms × 기억 단편 수 | β + γ |
| 7 | STRATA PROGRESS: 체크마크 [V] 순차 채워짐 (각 150ms) | 150ms × 클리어 지층 수 | γ + δ |
| 8 | LOOT 섹션 나타남 | 200ms | δ + 200ms |
| 9 | "[Z] Continue" 나타남 + 입력 활성화 | 100ms | 전체 완료 |

**총 기준 시간:** 레벨 차이 1, 스탯 3, 기억 단편 2, 지층 2 기준 = 300 + 150 + 200 + 300 + 600 + 400 + 300 + 200 + 100 = 약 2.5초.

레벨 차이가 1인 경우(통상): 약 2.5초.
일반 케이스(기억 단편 없음, 지층 2): 약 2.0초.
최대 케이스(Ancient 8기억 단편, 4지층): 약 4.0초.

모든 경우에서 [Z]로 즉시 최종 상태 스킵 가능.

**펀치 스케일:** 레벨 카운트업의 각 스텝 전환 시 숫자가 scale 1.0 → 1.2 → 1.0으로 빠르게 bounce한다 (100ms). 이 미세한 물리감이 숫자가 단순 변경이 아닌 "타격처럼 올라갔다"는 감각을 준다.

### 3.4 사망 시 결과 화면 (Death Result Variant) — RR-04

#### 3.4.1 개요

아이템계에서 HP가 0이 되어 사망하면 진행 전체가 소멸하고 월드 세이브 포인트로 귀환한다. 이 화면은 손실을 명확히 전달하되, 동시에 "Mr. Gency Exit를 사용했으면 막을 수 있었다"는 학습 동기를 제공한다.

#### 3.4.2 레이아웃 다이어그램

```
640x360 전체 화면

┌──────────────────────────────────────────────────────────────────┐
│                    #000000 (순수 검정 배경)                       │
│                                                                   │
│     ╔════════════════[ DIVE FAILED ]════════════════╗            │
│     ║                                               ║            │
│     ║  ┌─ ITEM ───────────────────────────────┐    ║            │
│     ║  │  Iron Blade                 RARE      │    ║            │
│     ║  │  ~~Lv.1 -> Lv.4~~          Lv.1       │    ║  (취소선 빨강 → 최종값 흰색)
│     ║  └──────────────────────────────────────┘    ║            │
│     ║                                               ║            │
│     ║  -- STAT CHANGES (LOST) --------------------  ║            │
│     ║  ATK:  ~~30 -> 52~~    30  (no change)       ║  (취소선 빨강 → 최종값 흰색)
│     ║  INT:  ~~0  ->  0~~     0                    ║            │
│     ║  HP:   ~~0  -> 10~~     0                    ║            │
│     ║                                               ║            │
│     ║  -- MEMORY_SHARDS (LOST) -----------------------  ║            │
│     ║  [X] ATK Boost Lv.3   Subdued - LOST         ║  (빨강)
│     ║                                               ║            │
│     ║  All progress lost.                           ║  (빨강 #FF4444)
│     ║  Returning to save point.                     ║            │
│     ║                                               ║            │
│     ║  [Z] Continue                                 ║            │
│     ╚═══════════════════════════════════════════════╝            │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

#### 3.4.3 RR-03과의 차이점

| 요소 | RR-03 (성공) | RR-04 (사망) |
|:---|:---|:---|
| 타이틀 | "DIVE COMPLETE" (#FFFFFF) | "DIVE FAILED" (#FF4444) |
| 레벨 변화 | "Lv.1 -> Lv.4  (+3)" (#FFD700) | "~~Lv.1 -> Lv.4~~  Lv.1" (취소선 #FF4444 → 현재값 #FFFFFF) |
| 스탯 변화 | 증가 수치 (#22DD44) | 취소선 표시 + "no change" (#FF4444 취소선 → #666666 현재값) |
| 기억 단편 | 상태별 색상 | 복종 처리된 기억 단편는 "[X] ... LOST" (#FF4444) |
| 하단 메시지 | 없음 | "All progress lost. Returning to save point." (#FF4444) |
| STRATA PROGRESS | 클리어/미클리어 표시 | 섹션 전체 미표시 |
| LOOT | Gold/Items 표시 | 0으로 표시 ("Gold earned: 0"). 소지 골드는 마지막 세이브 시점으로 롤백 |

**취소선 구현 방법:** PixiJS BitmapText는 기본 취소선을 지원하지 않으므로, 텍스트 위에 1px 가로선을 오버레이로 렌더링한다. 빨간 텍스트(#FF4444)를 먼저 렌더, 그 중앙에 1px #FF4444 선 렌더.

#### 3.4.4 이 화면이 의도적으로 고통스러운 이유

플레이어가 Mr. Gency Exit를 사용하지 않고 사망했을 때, 이 화면은 "얼마나 많은 것을 잃었는가"를 명확히 보여준다. 이는 다음 다이브에서 긴급 탈출 아이템을 적극 사용하게 만드는 학습 동기다. 손실의 규모가 클수록 이 화면의 교훈은 강해진다.

#### 3.4.5 애니메이션 시퀀스

| 단계 | 내용 | 지속 시간 |
|:---:|:---|:---:|
| 1 | 검정 배경 페이드인 (빠른) | 200ms |
| 2 | 타이틀 "DIVE FAILED" 붉은 색으로 나타남 | 200ms |
| 3 | ITEM 섹션: 아이템명 → 레벨 취소선 라인 표시 | 300ms |
| 4 | STAT CHANGES: 취소선 수치들 순차 표시 | 200ms × 스탯 수 |
| 5 | MEMORY_SHARDS: LOST 라인 표시 (있을 때) | 200ms |
| 6 | 하단 메시지 나타남 | 200ms |
| 7 | "[Z] Continue" 나타남 + 입력 활성화 | 100ms |

**총 기준 시간:** 약 1.5초. 성공 화면(2.5초)보다 의도적으로 빠르다. 고통은 길게 늘이지 않는다. 단, 중간 스킵 없음([Z]는 최종 상태에서만 작동). 플레이어는 손실 전체를 봐야 한다.

### 3.5 Mr. Gency Exit 확인 UI — RR-05

#### 3.5.1 개요

Mr. Gency Exit 아이템 사용 시 표시되는 소형 확인 다이얼로그다. 이 아이템은 "지금까지의 진행 상황을 보존하고 탈출"하는 유일한 수단이므로, 오조작을 방지하는 확인 단계가 필요하다.

#### 3.5.2 레이아웃 다이어그램

```
640x360 화면 중앙

┌─────────────────────────────────────────────────────┐
│ 반투명 오버레이 #000000 alpha 0.65                   │
│                                                      │
│           ╔══════════════════════════╗              │
│           ║  Use Emergency Exit?     ║              │
│           ║  Progress will be saved. ║              │
│           ║                          ║              │
│           ║  [YES]           [NO]    ║              │
│           ╚══════════════════════════╝              │
│                                                      │
└─────────────────────────────────────────────────────┘

패널 크기: 200x80px
패널 위치: x=220, y=140
```

#### 3.5.3 패널 사양

| 속성 | 값 |
|:---|:---|
| 패널 크기 | 200x80px |
| 패널 위치 | x=220, y=140 (중앙 정렬) |
| 배경색 | #1A1A2E, alpha 0.95 |
| 보더 | #4A4A6A, 1px |
| 타이틀 텍스트 | "Use Emergency Exit?", fontSize 8, #FFFFFF |
| 부가 텍스트 | "Progress will be saved.", fontSize 6, #AAAAAA |
| YES 버튼 | 선택 시 배경 #22DD44, 비선택 시 #333355 |
| NO 버튼 | 선택 시 배경 #FF4444, 비선택 시 #333355 |
| 버튼 텍스트 | "YES" / "NO", fontSize 8, #FFFFFF |
| 버튼 크기 | 각 60x20px |

#### 3.5.4 입력 규칙

- **기본 선택:** NO (오조작 방지)
- **좌우 방향키:** YES ↔ NO 전환
- **Z 키:** 현재 선택 확정
- **ESC 키:** NO 선택과 동일 (다이얼로그 닫기, 아이템계 복귀)
- **YES 확정:** 다이브 역방향 연출 → 월드 귀환 → RR-03 귀환 결과 화면("Emergency Exit used" 표기 포함)
- **NO 확정:** 다이얼로그 닫기, 아이템계 복귀

선택 커서 이동 시 선택 강조(배경색 변경)는 즉각 적용. 확정 대기 상태에서 입력 없이 10초 경과 시 자동 NO 처리.

## 4. 엣지 케이스 (Edge Cases)

### 4.1 RR-01 보스 등급 표시

| 상황 | 처리 방식 |
|:---|:---|
| 보스 방에 2개 이상의 보스 엔티티 (이론상 없음) | 화면에 활성화된 첫 번째 보스 기준으로 Grade Label 표시. 두 번째 보스 처치 시 해당 Grade Label 업데이트 |
| 보스가 뒤로 달아나 Grade Label이 HP 바와 시각적으로 맞지 않는 경우 | HP 바와 Grade Label은 HUD 레이어에 고정 배치. 보스 스프라이트 위치에 종속되지 않음 |
| 보스 HP가 회복되는 경우 (특수 페이즈) | HP 바는 현재 HP를 항상 반영. Grade Label은 동일 보스이므로 유지 |
| 위치 텍스트(y=344)와 보스 HP 바(y=338) 겹침 | 보스전 중 위치 텍스트 alpha 0.0으로 설정. 보스 처치 후 복원 |

### 4.2 RR-02 레벨업 팝업

| 상황 | 처리 방식 |
|:---|:---|
| 보스 처치 후 레벨업이 없는 경우 (최대 레벨 도달 또는 재사이클 후) | RR-02 미표시. 토스트 "STRATUM CLEARED" 1초 표시 후 지층 전환 |
| 팝업 표시 중 ESC 입력 (아이템계 탈출 시도) | 팝업 즉시 닫기 → ESC 탈출 로직 처리. 레벨업 데이터는 이미 반영 완료 상태이므로 손실 없음 |
| 스탯 변화 수치가 음수인 경우 | 설계상 지층 클리어로 스탯이 감소하는 경우는 없음. 방어적 처리: 음수 발생 시 해당 라인 빨간 색으로 표시 + 데이터 로그 기록 |
| 기억 단편를 이번 지층에서 3개 이상 복종 | "N Memory Shards Subdued!" 단일 라인 요약 처리 |
| 팝업 표시 중 다른 방으로 이동 (기술적 버그) | 팝업 강제 닫기. 레벨업 데이터 손실 없음 (선처리) |

### 4.3 RR-03 귀환 결과 화면

| 상황 | 처리 방식 |
|:---|:---|
| 아이템명이 30자를 초과하는 경우 | 최대 24자 + "..." 절단. ITEM 섹션 가로 400px 기준 |
| 재사이클이 다이브 중 2회 이상 발생하는 경우 | "Cycle: 1 -> 3" (최초→최종 표기). 중간 사이클 생략 |
| Legendary/Ancient 기억 단편 8슬롯 전부 표시 불가 | 6줄 표시 후 "+N more memory shards" 회색 텍스트 |
| 레벨업 없이 귀환 (전 지층 이전에 Mr. Gency Exit) | ITEM 섹션에 "Lv.N (no change)" 표시. 스탯 섹션 "no change" 전부 회색 |
| 골드 획득량 0인 경우 | "Gold earned:   0" 회색 표시 |
| 아이템 획득 0인 경우 | "Items found:   0" 회색 표시 |
| 게임 창 비활성화(Alt+Tab) 중 애니메이션 | requestAnimationFrame 일시 정지. 복귀 시 중단 지점부터 재개 |
| [Z] 스킵 후 최종 상태에서 다시 [Z] 입력 | 화면 닫기 처리. 연속 두 번 [Z] 오입력 방지를 위해 최종 상태 전환 후 300ms 입력 대기시간 |

### 4.4 RR-04 사망 결과 화면

| 상황 | 처리 방식 |
|:---|:---|
| 사망 직전 Mr. Gency Exit 아이템 소지 중이었던 경우 | RR-04 화면 하단에 "You had an Emergency Exit. Use it next time." 메시지를 #FFAA00으로 추가 표시. 교습 강화 |
| 사망 직전 레벨업 팝업(RR-02)이 표시 중이었던 경우 | 이론상 불가(보스 처치 중 사망 없음). 방어적 처리: 팝업 강제 닫기 → 사망 처리 |
| 진입 직후 첫 방에서 즉사하여 스탯 변화가 전무한 경우 | RR-04 표시는 동일. 모든 수치가 "(no change)"로 채워진 취소선 없는 패널 |

### 4.5 RR-05 Mr. Gency Exit 확인

| 상황 | 처리 방식 |
|:---|:---|
| 확인 다이얼로그 표시 중 사망하는 경우 | 사망 처리 우선. 다이얼로그 강제 닫기 → RR-04 사망 결과 화면 |
| Mr. Gency Exit 아이템이 없는데 확인 UI가 열린 경우 | 데이터 레이어 오류. UI 열리기 전 아이템 소지 여부 검증. 소지 없으면 UI 미표시 |
| 확인 다이얼로그 표시 중 지층 전환이 발생하는 경우 (이론상 없음) | 다이얼로그 강제 닫기 |

## 5. 검증 체크리스트 (Acceptance Criteria)

### 기능 검증 — RR-01 보스 등급 표시

- [ ] 보스 방 진입 시 Grade Label이 HP 바 위에 즉각 표시된다
- [ ] Grade Label의 색상이 Item General=흰, Item King=파랑, Item God=노랑, Item Great God=주황으로 정확히 적용된다
- [ ] Boss Name이 Grade Label 아래 흰색으로 표시된다
- [ ] 보스 HP=0 시 Grade Label + Boss Name + HP 바가 0.5초 페이드아웃으로 동기화 소멸한다
- [ ] 일반 방(비보스 방)에서는 Grade Label이 표시되지 않는다
- [ ] 보스전 중 위치 텍스트가 숨겨지고, 보스 처치 후 복원된다

### 기능 검증 — RR-02 레벨업 팝업

- [ ] 보스 처치 후 레벨업 발생 시 팝업이 표시된다
- [ ] 레벨 카운트업 애니메이션이 각 스텝마다 펀치 스케일과 함께 재생된다
- [ ] 아이템명이 레어리티 색상으로 표시된다
- [ ] 스탯 증가 시 초록, 변화 없음 시 회색으로 표시된다
- [ ] 이번 지층에서 복종시킨 기억 단편가 있으면 기억 단편 라인이 표시된다
- [ ] [Z] 입력 시 애니메이션을 스킵하거나 팝업을 닫는다
- [ ] 3초 후 자동으로 닫힌다
- [ ] 레벨업이 없는 경우 팝업이 표시되지 않는다

### 기능 검증 — RR-03 귀환 결과 화면

- [ ] 정상 완료 후 월드 착지 시 결과 화면이 표시된다
- [ ] Mr. Gency Exit 후 결과 화면에 "Emergency Exit used" 텍스트가 표시된다
- [ ] ITEM 섹션이 올바른 아이템명, 레어리티 색상 보더, 레벨 변화를 표시한다
- [ ] STAT CHANGES가 다이브 전 스탯과 현재 스탯의 정확한 차이를 표시한다
- [ ] 증가 스탯은 초록, 변화 없음은 회색으로 표시된다
- [ ] MEMORY_SHARDS 섹션이 이번 다이브에서 복종한 기억 단편에 "NEW!" 태그를 표시한다
- [ ] Wild 기억 단편는 빨간 [!] 아이콘으로 표시된다
- [ ] STRATA PROGRESS에서 클리어 지층은 [V], 미클리어는 [ ]로 표시된다
- [ ] LOOT 섹션에 획득 골드와 아이템 수가 표시된다
- [ ] 애니메이션 시퀀스가 정해진 순서(타이틀→ITEM→레벨업→스탯→기억 단편→지층→LOOT→닫기)로 재생된다
- [ ] [Z] 입력 시 남은 애니메이션이 최종 상태로 즉시 전환된다
- [ ] 최종 상태에서 [Z] 입력 시 화면이 닫히고 월드 플레이가 재개된다

### 기능 검증 — RR-04 사망 결과 화면

- [ ] 사망 시 "DIVE FAILED" 빨간 타이틀이 표시된다
- [ ] 레벨 변화 라인이 취소선으로 표시된다
- [ ] 스탯이 취소선 + 현재값(변화 없음)으로 표시된다
- [ ] 이번 다이브에서 복종 처리된 기억 단편가 "LOST"로 표시된다
- [ ] "All progress lost. Returning to save point." 메시지가 빨간색으로 표시된다
- [ ] Mr. Gency Exit를 소지한 채 사망한 경우 교훈 메시지가 추가된다
- [ ] [Z]는 최종 상태(애니메이션 완료 후)에서만 닫기로 작동한다

### 기능 검증 — RR-05 Mr. Gency Exit 확인

- [ ] 긴급 탈출 아이템 사용 시 확인 다이얼로그가 표시된다
- [ ] 기본 선택이 NO이다
- [ ] 좌우 방향키로 YES/NO 전환, Z로 확정, ESC로 취소(NO)가 작동한다
- [ ] YES 선택 시 다이브 역방향 연출 후 RR-03 결과 화면으로 진행된다
- [ ] NO 선택 시 다이얼로그가 닫히고 아이템계 플레이가 재개된다
- [ ] 10초 입력 없을 시 자동 NO 처리된다

### 경험 검증

- [ ] 귀환 결과 화면(RR-03) 애니메이션을 끝까지 보았을 때 "이번 다이브가 가치 있었다"는 감각이 든다
- [ ] 레벨 카운트업의 펀치 스케일이 단조 타격감과 연결된다 (시청각 일치)
- [ ] 사망 결과 화면(RR-04)이 "다음에는 Mr. Gency Exit를 써야겠다"는 결심을 유도한다
- [ ] RR-05 확인 다이얼로그에서 YES를 누르는 것이 머뭇거림 없이 확신 있는 행동으로 느껴진다 (기본 선택이 NO인데도 YES를 누르는 의지 있는 행동)
- [ ] 귀환 결과 화면에서 기억 단편 "NEW! Subdued" 태그가 발견의 기쁨을 준다
- [ ] 10회 이상 연속 다이브 후에도 귀환 결과 화면이 답답하게 느껴지지 않는다 ([Z] 스킵 1회로 전체 스킵 가능)
