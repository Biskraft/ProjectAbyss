## 0. 설계 원칙

### 0.1. 적용 범위

- **월드 전용.** 미니맵은 월드(Overworld)에서만 표시된다
- **아이템계 없음.** 아이템계 진입 시 미니맵이 사라진다. 복귀 시 다시 나타남

### 0.2. 핵심 결정

| 결정 | 근거 |
|:---|:---|
| 월드만 미니맵 제공 | 월드는 핸드크래프트 구조로 재방문이 잦음. 미니맵이 내비게이션 안전망 역할 |
| 아이템계 미니맵 없음 | Spelunky 모델. 절차적 생성 공간에서 미니맵 없이 직접 탐험하는 긴장감 보존. 4x4 그리드(16방)는 암기 가능한 크기 |
| 진입 즉시 공개 (Blasphemous식) | 별도 조건(벤치/구매) 없이 방 진입만으로 맵 공개. 학습 비용 최소 |
| HUD 패널 16:9 | 게임 화면 비율(640x360)과 정합. 공간 낭비 최소 |
| 세이브 포인트 항상 표시 | 사망 페널티가 있는 게임에서 귀환 지점 인지는 심리적 안전망 (6개 게임 공통 패턴) |

## 1. HUD 미니맵 (World) — P1

### 1.1. 패널 사양

| 항목 | 값 |
|:---|:---|
| **위치** | 화면 우상단 |
| **크기** | 128 x 72 px (16:9 비율) |
| **배경** | 반투명 검정 (alpha 0.6) |
| **테두리** | 1px 회색 (#666666) |
| **투명도 (평상시)** | 70% 불투명 |
| **투명도 (전투 중)** | 40% 불투명 |
| **토글** | TAB 키로 표시/숨김 |
| **아이템계 진입 시** | 자동 숨김. 월드 복귀 시 자동 표시 |

### 1.2. 표시 범위

현재 방 + 인접 방 (상하좌우 3칸 이내). 월드의 방은 크기가 다양하므로, 현재 방을 중심에 놓고 주변 방을 상대 위치로 배치.

### 1.3. 방 상태와 표시

| 상태 | 조건 | 미니맵 표시 |
|:---|:---|:---|
| `UNDISCOVERED` | 진입 없음, Map Station 미활성 | 표시 안 함 |
| `OUTLINED` | Map Station 활성화로 외곽선 공개 (P2) | 층위색 어두운 외곽선 |
| `VISITED` | 한 번 이상 진입 | 층위 테마색으로 채움 |
| `CLEARED` | 방 내 모든 적 처치 | VISITED + 작은 체크 기호 |

**상태 전환:**
- `UNDISCOVERED` → `VISITED`: 방 진입 순간 즉시 (Blasphemous 모델)
- `UNDISCOVERED` → `OUTLINED`: Map Station 활성화 시 (P2)
- `VISITED` → `CLEARED`: 방 내 적 전멸 시
- 상태는 **영구 저장**. 사망/로그아웃 후에도 유지

### 1.4. 층위별 색상

| 층위 | 맵 색상 |
|:---|:---|
| 천공의 정원 (Tier 1) | #4A8A4A (연두) |
| 중앙 성채 (Tier 2) | #5A7A8C (회청) |
| 묘지/카타콤 (Tier 3) | #4A3A2A (어두운 갈색) |
| 지하 수로 (Tier 4) | #2A4A6C (남색) |
| 연구소 폐허 (Tier 5) | #6A4A8C (보라) |
| 빙결 보존소 (Tier 6) | #4AACCC (빙청) |
| 심연의 구 (Tier 7) | #8C2A2A (심홍) |

### 1.5. 현재 방 표시

현재 방은 **흰색 테두리 (2px)** + **깜박이는 플레이어 점 (3x3 px, 흰색)**.

## 2. 자동 마커 — P1

시스템이 자동으로 배치하는 아이콘. 해당 방 최초 진입/발견 시 표시.

| 마커 | 아이콘 | 색상 | 조건 |
|:---|:---|:---|:---|
| **세이브 포인트** | 별 ★ | 빨강 | 발견 즉시 |
| **모루 (아이템계 진입)** | 모루 ⚒ | 금색 | 발견 즉시 |
| **보스 방 (미처치)** | 해골 ☠ | 주황 | 진입 시 |
| **보스 방 (처치 완료)** | 해골에 X | 회색 | 보스 처치 시 변경 |
| **스탯 게이트** | 방패 | 파랑 | 접근 시 |
| **능력 게이트** | 열쇠 | 초록 | 접근 시 |
| **상점** | 동전 | 노랑 | 발견 즉시 |

## 3. 전체 맵 (Full Map) — P2

### 3.1. 접근

일시정지 메뉴 → 맵 탭. 또는 전용 키 (M).

### 3.2. 사양

| 항목 | 값 |
|:---|:---|
| **표시 범위** | 전체 세계 (7 층위 수직 배치) |
| **스크롤** | 상하 자유 스크롤 |
| **줌** | 2단계 (전체 조감 / 층위 상세) |
| **현재 위치** | 깜박이는 흰색 점 |
| **완료율** | 우하단 "탐험: XX%" |

### 3.3. Map Station

- 각 층위에 1개의 Map Station 방 존재
- 활성화 시 해당 층위의 모든 방이 `OUTLINED` 상태로 공개 (외곽선만, 내부 미공개)
- Map Station은 세이브 포인트 인접 배치 (안도 구간에서 준비하는 패턴)

### 3.4. 수동 마커

- 최대 10개 배치 가능
- 종류: 방문 필요 / 미수집 아이템 / 미해결 퍼즐 / 위험 구역
- 탐험한 방에만 배치 가능
- HUD 미니맵에도 표시

## 4. 구현 가이드

### 4.1. 데이터 구조

```typescript
interface RoomMapState {
  levelId: string;        // LDtk level identifier
  state: 'undiscovered' | 'outlined' | 'visited' | 'cleared';
  markers: string[];      // auto-marker IDs
  userMarkers: string[];  // player-placed markers
}
```

월드의 각 LDtk 레벨에 대해 RoomMapState를 SaveManager에 저장.

### 4.2. 렌더링 구조

```
MinimapContainer (128x72)
├── Background (반투명 검정)
├── RoomRects (인접 방들, 층위색으로 채움)
│   └── 현재 방: 흰색 테두리
├── MarkerIcons (자동 마커 아이콘)
├── PlayerDot (3x3 깜박이는 점)
└── TierLabel ("Tier 2")
```

### 4.3. 표시/숨김 조건

| 상황 | 미니맵 |
|:---|:---|
| 월드 탐험 중 | 표시 (70% 불투명) |
| 월드 전투 중 | 표시 (40% 불투명) |
| 아이템계 진입 | **숨김** |
| 일시정지 메뉴 | 숨김 (전체 맵 사용) |
| 대화/컷신 | 숨김 |

## 5. 스파이크 검증

> "아이템에 들어가면, 그 아이템의 기억이 던전이 된다"

| 요소 | 스파이크 강화 방식 |
|:---|:---|
| **아이템계 미니맵 없음** | "기억 속에서는 지도가 없다" = 기억의 미지성 강화. 탐험 긴장감 |
| **월드 미니맵 있음** | 월드는 "알려진 구조물". 기억과 대비되는 안전한 영역. 아이템계 진입 시 미니맵 사라짐 = 공간 차원 전환 체감 |
| **진입/복귀 시 표시 전환** | 아이템계 들어가면 미니맵 사라짐 → "여기는 다른 세계" 물리적 느낌 |
