## 1. 개요 (Overview)

세이브/로드 UI는 ECHORIS의 localStorage 기반 세이브 시스템을 플레이어에게 노출하는 세 가지 컴포넌트로 구성된다. 세이브 슬롯 선택 화면(14-1)은 메인 메뉴와 세이브 포인트에서 진입할 수 있으며, 3개의 독립 슬롯으로 다회차 플레이를 지원한다. 자동 저장 인디케이터(14-2)는 세이브가 발생할 때 HUD 우상단에 비침습적으로 표시된다. 종료 확인 다이얼로그(14-4)는 브라우저 탭 닫기(beforeunload)와 인게임 타이틀 복귀 시 저장되지 않은 진행이 사라짐을 경고한다. 세 컴포넌트 모두 키보드 전용 조작(Arrow/C/ESC)으로 완결되며, 마우스 입력을 요구하지 않는다.

## 2. 설계 의도 (Design Intent)

**세이브 행위는 탐험 재개의 의식이다.** ECHORIS의 세이브 포인트는 단순한 진행 백업이 아니라 "이 장비와 강화 상태로 다음 탐험을 이어간다"는 플레이어의 의지 표현이다. 슬롯 선택 화면은 이 의식을 가볍게 지원하되, 플레이 흐름을 끊지 않도록 설계한다.

- **스파이크 정렬:** 세이브 슬롯 정보에는 골드와 위치가 표시되어 "이 슬롯의 에르다가 어느 심연에 있는가"를 직관적으로 파악할 수 있다. 아이템 인벤토리나 이노센트 목록 같은 상세 정보는 슬롯 카드에 표시하지 않는다 — 그것은 아이템계 진입 시의 설렘으로 남겨둔다.
- **자동 저장 인디케이터 비침습 원칙 (Pattern C):** 자동 저장은 월드 탐험의 흐름 속에서 발생한다. 인디케이터는 패턴 C(Overlay) — 게임을 멈추지 않고, 입력을 요구하지 않으며, 화면 최소 공간만 점유한다. 플레이어의 주 시선(화면 중앙)과 멀리 떨어진 우상단에 1.5초간 표시된 후 사라진다.
- **종료 경고 기본값 NO:** 실수로 타이틀로 복귀하거나 탭을 닫는 것은 회복 불가능한 진행 손실이다. 기본 커서를 NO에 위치시켜 최악의 실수를 방지한다 (히스테리시스 원칙).
- **3슬롯 한정:** 웹 게임 특성상 세이브 수가 많을수록 용량 관리와 UI 복잡도가 증가한다. 3슬롯은 본인 + 가족/친구의 독립 플레이스루를 수용하면서 localStorage 부담을 최소화한다.

## 3. 상세 규칙 (Detailed Rules)

### 3.1 세이브 슬롯 선택 화면 (Save Slot Selection)

#### 3.1.1 진입 경로

| 진입 경로 | 트리거 | 동작 모드 |
| :--- | :--- | :--- |
| 메인 메뉴 "New Game" | 버튼 선택 + C | NEW 모드 |
| 메인 메뉴 "Continue" | 버튼 선택 + C | LOAD 모드 |
| 월드 세이브 포인트 | Pattern D (근접 + C) | SAVE 모드 |

**모드별 동작 차이:**

| 모드 | 빈 슬롯 선택 시 | 채워진 슬롯 선택 시 |
| :--- | :--- | :--- |
| NEW | 새 게임 시작 | 덮어쓰기 경고 (§3.5와 동일 패턴) 후 확인 시 새 게임 |
| LOAD | 비활성 (선택 불가, dim) | 해당 슬롯 데이터 로드 후 게임 씬 전환 |
| SAVE | 빈 슬롯에 현재 진행 저장 | 해당 슬롯에 현재 진행 저장 (덮어쓰기 경고 없음 — 현재 플레이스루와 동일 슬롯 저장이 기본) |

**SAVE 모드 상세:** 세이브 포인트에서 진입할 때, 현재 활성 슬롯 번호가 이미 결정되어 있으면 슬롯 선택 화면 없이 즉시 저장하고 자동 저장 인디케이터(§3.3)만 표시한다. 활성 슬롯이 없는 경우(최초 플레이)에만 슬롯 선택 화면을 표시한다.

#### 3.1.2 패널 레이아웃

|  +----------------------------+ |

| 패널 테두리색 | `#4A4A6A` (width 1px) |

#### 3.1.4 입력 규칙

| 키 | 동작 |
| :--- | :--- |
| Up/Down | 슬롯 포커스 이동 (비활성 슬롯은 건너뜀) |
| C | 선택 확인 — 모드에 따라 로드/저장/신규 시작 |

### 3.2 세이브 슬롯 정보 표시 (Slot Info Display)

슬롯 카드에 표시하는 각 필드의 데이터 소스와 포맷 규칙이다.

| 필드 | 소스 | 포맷 규칙 |
| :--- | :--- | :--- |
| 캐릭터명 | 고정값 `"Erda"` | 항상 `"Erda"` |
| 레벨 | `SaveData.player.level` (추가 예정 필드) | `"Lv.N"` — level이 없으면 `"Lv.?"` |
| 위치 (Tier + Area) | `SaveData.levelId` → 레벨 메타데이터 조회 | `"Tier N - [displayName]"` 형식. levelId가 없으면 `"Unknown Area"` |
| 플레이타임 | `SaveData.playtime` (밀리초) | `"HH:MM:SS"` 형식. `Math.floor(ms/3600000)`으로 시간 계산 |
| 골드 | `SaveData.gold` | 천 단위 쉼표 포맷. `toLocaleString()` 또는 수동 포맷 |
| 마지막 저장 일시 | `SaveData.savedAt` (타임스탬프, 추가 예정 필드) | `"YYYY-MM-DD  HH:MM"` 형식. Date 객체 변환 |
| 지도 완성도 % | Phase 2 이후 추가. 현재 슬롯 카드에 미표시 | `"Map: N%"` |

**level 및 savedAt 필드 추가 필요:** 현재 `System_Save_DataSchema.md`의 SaveData 스키마에 `player.level`과 `savedAt` 필드가 없다. 멀티 슬롯 구현 시 두 필드를 스키마 v4에 추가한다. 버전 마이그레이션 담당: `SaveManager.ts`.

**아이템계 저장 감지:** `SaveData.levelId`가 아이템계 레벨 식별자 패턴(`"itemworld_*"`)과 일치하는 경우 슬롯 카드 하단에 경고 텍스트 `"[!] Saved in Item World"`, fontSize 6, `#FF4444`를 표시한다. 이 상태는 정상 게임플레이에서 발생해서는 안 된다(세이브는 월드에서만 가능). 발생 시 구현 버그 신호로 취급한다.

### 3.3 자동 저장 인디케이터 (Auto-save Indicator)

#### 3.3.1 동작 조건

다음 이벤트에서 자동 저장이 발생하고 인디케이터가 표시된다:

- 세이브 포인트 Pattern D 상호작용 성공 (C키 확인 후 저장 완료 시)
- 아이템계 탈출 시 월드 귀환 자동 저장

인디케이터는 저장 성공 시에만 표시한다. 저장 실패(§3.6)는 별도 Toast로 처리한다.

#### 3.3.2 레이아웃 및 수치

```
  [640x360 화면 우상단 영역]
  ...미니맵(128x72, x=508, y=4)...

  x=612, y=80
  +----------+
  |[*] SAVING|   <- 인디케이터 컨테이너 (28x10px)
  +----------+
```

| 항목 | 값 |
| :--- | :--- |
| 위치 | `x=612, y=80` (미니맵 하단 4px 여백) |
| 아이콘 | 8×8px 회전 기어 스프라이트 또는 단순 원형 펄스 |
| 아이콘 색상 | `#00CED1` (teal) |
| 텍스트 | `"SAVING..."` |
| 텍스트 fontSize | 6 |
| 텍스트 색상 | `#FFFFFF` |
| 텍스트 위치 | 아이콘 우측 2px 간격 |
| 전체 컨테이너 | 28×10px |
| 총 표시 시간 | 1500ms |
| 페이드 인 | 0~500ms (alpha 0 → 1) |
| 홀드 | 500~1000ms (alpha 1) |
| 페이드 아웃 | 1000~1500ms (alpha 1 → 0) |
| z-order | HUD 레이어 최상위 (Toast와 동일 레이어) |
| 플레이어 입력 차단 | 없음 (Pattern C, Overlay) |

#### 3.3.3 중복 실행 처리

인디케이터 표시 중에 또 다른 저장이 발생하면 타이머를 리셋한다. 새 1500ms 카운트를 처음부터 재시작하고, alpha는 현재 값에서 이어지지 않고 1로 즉시 고정 후 재시작한다.

### 3.4 종료 확인 다이얼로그 (Exit Confirmation)

종료 확인은 두 계층으로 분리된다.

#### 3.4.1 계층 1: beforeunload 브라우저 경고

| 항목 | 규칙 |
| :--- | :--- |
| 트리거 | 브라우저 탭 닫기, 새로고침, 주소창 변경 |
| 구현 | JS `window.addEventListener('beforeunload', handler)` |
| 표시 내용 | 브라우저 기본 다이얼로그 ("Changes you made may not be saved.") |
| 커스텀 메시지 | 브라우저 보안 정책상 커스텀 불가 (Chrome 51+ 표준 메시지만 출력) |
| 등록 조건 | 게임 씬 진입 시 등록. 메인 메뉴 또는 타이틀로 귀환 완료 시 제거 (`removeEventListener`) |
| 저장 직후 처리 | 세이브 포인트 저장 성공 후 5초간 `beforeunload` 핸들러를 일시 해제하여 오탐 방지 (5초 후 재등록) |

#### 3.4.2 계층 2: 인게임 "QUIT TO TITLE" 확인 다이얼로그

포즈 메뉴의 "QUIT TO TITLE" 항목 선택 시 표시된다.

```
+--[ QUIT TO TITLE ]--+
|                      |
| Unsaved progress     |
| will be lost.        |
|                      |
|  < NO >   [ YES ]    |
|                      |
+----------------------+
```

| 항목 | 값 |
| :--- | :--- |
| 패널 크기 | 200×80px |
| 패널 위치 | 화면 정중앙 |
| 패널 배경색 | `#1A1A2E` (alpha 0.97) |
| 패널 테두리색 | `#FF6644` (경고색, 1px) |
| 타이틀 텍스트 | `"QUIT TO TITLE"`, fontSize 7, `#FF6644` |
| 본문 텍스트 | `"Unsaved progress will be lost."`, fontSize 6, `#CCCCCC`, 2줄 허용 |
| YES 버튼 텍스트 | `"YES"`, fontSize 7 |
| NO 버튼 텍스트 | `"NO"`, fontSize 7 |
| **기본 선택** | **NO** (안전 기본값) |
| NO 선택 시 스타일 | `#00CED1` 테두리 + `#FFFFFF` 텍스트 |
| YES 선택 시 스타일 | `#FF4444` 테두리 + `#FFFFFF` 텍스트 |
| 네비게이션 | Left/Right Arrow로 YES ↔ NO 전환 |
| 확인 | C키 |
| 취소 | ESC → NO와 동일 효과 (다이얼로그 닫기, 게임 복귀) |

**YES 선택 후 동작:** 게임 상태를 저장하지 않고 메인 메뉴 씬으로 전환한다. `beforeunload` 핸들러를 제거한다.

**NO / ESC 선택 후 동작:** 다이얼로그를 닫고 포즈 메뉴로 복귀한다.

---

### 3.5 세이브 삭제 확인 (Delete Confirmation)

슬롯 선택 화면(§3.1)에서 채워진 슬롯에 포커스한 상태로 X키를 누르면 2단계 삭제 확인이 시작된다.

#### 3.5.1 1단계: 삭제 의도 확인

```
+--[ DELETE SAVE ]----+
|                      |
| Delete Slot N?       |
| This cannot be       |
| undone.              |
|                      |
|  < NO >   [ YES ]    |
|                      |
+----------------------+
```

| 항목 | 값 |
| :--- | :--- |
| 패널 크기 | 200×80px |
| 패널 위치 | 화면 정중앙 |
| 패널 배경색 | `#1A1A2E` (alpha 0.97) |
| 패널 테두리색 | `#FF4444` (경고 강조, 1px) |
| 타이틀 텍스트 | `"DELETE SAVE"`, fontSize 7, `#FF4444` |
| 본문 텍스트 | `"Delete Slot N? This cannot be undone."`, fontSize 6, `#CCCCCC`, N=슬롯 번호 |
| 기본 선택 | NO |
| 네비게이션 | Left/Right Arrow |
| 확인 | C키 |
| 취소 | ESC 또는 NO 선택 후 C → 슬롯 선택 화면으로 복귀 |

#### 3.5.2 2단계: 최종 확인

1단계에서 YES를 선택하면 2단계 다이얼로그가 표시된다.

```
+--[ ARE YOU SURE? ]--+
|                      |
| Slot N data will be  |
| permanently deleted. |
|                      |
|  < NO >   [ YES ]    |
|                      |
+----------------------+
```

| 항목 | 값 |
| :--- | :--- |
| 패널 크기 | 200×80px (1단계와 동일) |
| 타이틀 텍스트 | `"ARE YOU SURE?"`, fontSize 7, `#FF4444` |
| 본문 텍스트 | `"Slot N data will be permanently deleted."`, fontSize 6, `#FF8888` |
| 기본 선택 | NO (2단계에서도 동일) |

2단계 YES 선택 시: `SaveManager.deleteSlot(N)`을 호출하고 슬롯 선택 화면을 새로고침한다. 삭제된 슬롯은 즉시 빈 슬롯(empty state)으로 전환된다.

**NEW 모드 덮어쓰기 경고:** 메인 메뉴 NEW 모드에서 채워진 슬롯을 선택하면 동일한 2단계 확인 다이얼로그를 사용한다. 타이틀 텍스트만 `"OVERWRITE SAVE"`로 교체한다.

### 3.6 저장소 용량 처리 (Storage Handling)

#### 3.6.1 용량 추정

| 항목 | 예상 크기 |
| :--- | :--- |
| 슬롯 당 기본 세이브 데이터 | ~3KB (인벤토리 20개 아이템 + worldProgress 포함) |
| 3슬롯 최대 | ~10KB |
| localStorage 일반 할당량 | ~5MB (브라우저별 상이) |
| 여유 비율 | ~500:1 (용량 문제 발생 가능성 매우 낮음) |

#### 3.6.2 QuotaExceededError 처리

`SaveManager.save(slotIndex, data)` 내부의 `try/catch`에서 `QuotaExceededError`를 포획 시:

1. 저장 자동 저장 인디케이터를 표시하지 않는다.
2. Toast 시스템(`UI_Notifications.md`)을 통해 다음 메시지를 표시한다:
   - 텍스트: `"Save failed: storage full"`
   - Toast 타입: 오류 (fontSize 7, `#FF4444`)
   - 지속 시간: 3000ms (일반 Toast보다 긴 2배 지속)
3. 저장 실패는 게임 크래시를 유발하지 않는다. 게임은 계속 실행된다.

#### 3.6.3 손상 데이터 감지

게임 시작 시 `SaveManager.load(slotIndex)` 호출에서 다음 중 하나가 발생하면 손상으로 판단한다:
- JSON 파싱 실패
- `version` 필드 불일치 (향후 버전 마이그레이션으로 전환 전까지 손상 취급)
- 필수 필드 누락 (`player`, `levelId`, `inventory` 중 하나라도 없음)

손상 감지 시 슬롯 카드에 다음을 표시한다:

```
+- SLOT N ------------------+
| [!] CORRUPTED DATA        |
| Last saved: YYYY-MM-DD    |
| Delete to create new save |
+----------------------------+
```

| 항목 | 값 |
| :--- | :--- |
| 헤더 텍스트 | `"[!] CORRUPTED DATA"`, fontSize 7, `#FF4444` |
| 서브 텍스트 | `"Delete to create new save"`, fontSize 6, `#FF8888` |
| 슬롯 테두리 | `#FF4444` (비선택), `#FF6644` 2px (선택됨) |
| 이 슬롯에서 LOAD 시도 | 차단 (비활성 처리, LOAD 모드에서 선택 불가) |
| 이 슬롯에서 X키 | 삭제 확인 다이얼로그 (§3.5) 진행 가능 |

## 4. 엣지 케이스 (Edge Cases)

| 상황 | 처리 방식 |
| :--- | :--- |
| 모든 슬롯이 비어 있는 상태에서 메인 메뉴 "Continue" 선택 | "Continue" 버튼 자체를 비활성 처리 (dim, 선택 불가). 슬롯 선택 화면 진입 없음. |
| LOAD 모드에서 비활성 슬롯에 커서 이동 시도 | 커서가 해당 슬롯을 건너뛰고 다음 유효 슬롯으로 이동. 모든 슬롯이 비어 있으면 커서 이동 없음. |
| SAVE 모드에서 동일 슬롯에 재저장 | 덮어쓰기 경고 없이 즉시 저장. 현재 플레이스루의 진행이 갱신된다. |
| 슬롯 선택 화면 열린 상태에서 포즈(ESC) | 슬롯 선택 화면이 닫히고 게임 씬으로 복귀. 포즈 메뉴 진입 없음 — ESC는 "이전 화면"을 의미. |
| 세이브 중 브라우저 탭이 닫힘 | 저장 도중 강제 종료. localStorage 쓰기는 동기(synchronous)이므로 `setItem` 완료 후 탭이 닫히면 데이터는 보존된다. `setItem` 호출 전에 닫히면 이전 슬롯 데이터가 유지된다. |
| Safari 시크릿 모드 | localStorage 접근 자체가 예외 발생. `SaveManager.isAvailable()` 검사 후 불가 시 메인 메뉴에 `"Saves disabled (private mode)"` Toast 1회 표시. 게임 진행은 가능하지만 세이브 포인트에서 저장이 발생하지 않는다. |
| 두 브라우저 탭에서 동일 세이브 슬롯 동시 사용 | 두 번째 탭이 첫 번째 탭의 저장을 덮어씀. 웹 게임 특성상 경고 불가. 문서화만 하고 별도 처리 없음. |
| `savedAt` 타임스탬프가 미래 시간인 경우 | 로컬 시스템 시간이 변경된 경우 발생 가능. 그대로 표시. 저장 시간 검증 없음. |
| 슬롯 삭제 2단계 확인 중 ESC | 2단계 다이얼로그 닫기 → 1단계 다이얼로그도 함께 닫기 → 슬롯 선택 화면으로 복귀. 삭제 미발생. |
| 자동 저장 인디케이터 표시 중 슬롯 선택 화면 열림 | 인디케이터를 즉시 숨긴다. 슬롯 선택 화면이 HUD 위에 올라오므로 시각적 충돌 방지. |
| `player.level` 필드가 없는 구버전 세이브 | 슬롯 카드에서 레벨 필드를 `"Lv.?"` 로 표시. 손상으로 취급하지 않음. |

## 5. 검증 체크리스트 (Acceptance Criteria)

### 기능 검증

**세이브 슬롯 선택 (14-1):**

- [ ] 메인 메뉴 "New Game"에서 슬롯 선택 화면이 열리며 3개 슬롯이 표시된다
- [ ] 메인 메뉴 "Continue"에서 슬롯 선택 화면이 열리며 빈 슬롯은 선택 불가 상태로 표시된다
- [ ] 세이브 포인트에서 C키를 누르면 활성 슬롯이 있을 때 슬롯 선택 화면 없이 즉시 저장된다
- [ ] 세이브 포인트에서 최초 플레이(활성 슬롯 없음) 시 슬롯 선택 화면이 표시된다
- [ ] Up/Down Arrow로 슬롯 간 포커스 이동이 작동한다
- [ ] LOAD 모드에서 Up/Down이 빈 슬롯을 건너뛴다
- [ ] 채워진 슬롯 선택 후 C키를 누르면 해당 슬롯 데이터로 게임이 로드된다
- [ ] NEW 모드에서 채워진 슬롯 선택 시 덮어쓰기 경고 2단계가 출력된다
- [ ] ESC 키가 슬롯 선택 화면을 닫고 이전 화면으로 복귀시킨다

**슬롯 정보 표시 (14-2 범주):**

- [ ] 채워진 슬롯에 캐릭터명, 레벨, 위치, 플레이타임, 골드, 마지막 저장 일시가 표시된다
- [ ] 플레이타임이 HH:MM:SS 포맷으로 정확히 변환된다 (예: 16335000ms → "04:32:15")
- [ ] 골드가 천 단위 쉼표 포맷으로 표시된다 (예: 12450 → "12,450")
- [ ] 빈 슬롯에 "- EMPTY -" 텍스트가 중앙 정렬로 표시된다
- [ ] 선택된 슬롯의 테두리가 `#00CED1`으로 변경된다

**자동 저장 인디케이터 (14-2):**

- [ ] 세이브 포인트 저장 성공 시 HUD 우상단에 인디케이터가 표시된다
- [ ] 아이템계 탈출 후 월드 귀환 저장 시 인디케이터가 표시된다
- [ ] 인디케이터가 정확히 1500ms 후 사라진다 (500ms 페이드인, 500ms 홀드, 500ms 페이드아웃)
- [ ] 인디케이터 표시 중 플레이어 이동/공격이 차단되지 않는다
- [ ] 저장 실패 시 인디케이터가 표시되지 않는다

**종료 확인 (14-4):**

- [ ] 게임 씬 진입 시 `beforeunload` 이벤트 핸들러가 등록된다
- [ ] 브라우저 탭 닫기 시도 시 브라우저 기본 경고 다이얼로그가 표시된다
- [ ] 메인 메뉴로 복귀 완료 후 `beforeunload` 핸들러가 제거된다
- [ ] 포즈 메뉴 "QUIT TO TITLE" 선택 시 확인 다이얼로그가 표시된다
- [ ] 다이얼로그 기본 커서가 "NO"에 위치한다
- [ ] Left/Right Arrow로 YES/NO 전환이 작동한다
- [ ] NO 선택(또는 ESC) 시 다이얼로그가 닫히고 게임이 재개된다
- [ ] YES 선택 시 저장 없이 메인 메뉴로 전환된다

**세이브 삭제 (14-3 범주):**

- [ ] 채워진 슬롯에서 X키를 누르면 1단계 삭제 확인 다이얼로그가 표시된다
- [ ] 1단계 YES → 2단계 다이얼로그가 표시된다
- [ ] 2단계 YES → 슬롯 데이터가 삭제되고 빈 슬롯으로 전환된다
- [ ] 어느 단계에서든 NO 또는 ESC → 삭제 미발생, 슬롯 선택 화면으로 복귀
- [ ] 빈 슬롯에서 X키를 눌러도 삭제 다이얼로그가 표시되지 않는다

**저장소 처리:**

- [ ] `QuotaExceededError` 발생 시 `"Save failed: storage full"` Toast가 3초간 표시된다
- [ ] 손상된 슬롯이 "[!] CORRUPTED DATA" 상태로 표시된다
- [ ] 손상된 슬롯은 LOAD 모드에서 선택 불가 처리된다
- [ ] 손상된 슬롯에서 X키로 삭제 확인 다이얼로그가 표시되고 삭제가 가능하다
