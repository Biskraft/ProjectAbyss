## 1. 개요 (Overview)

메뉴 시스템은 ECHORIS의 게임 플로우를 제어하는 최상위 UI 레이어다. 타이틀 화면에서 시작해 메인 메뉴, 인게임 일시정지, 사망 복귀, 오디오 설정까지 플레이어가 게임 세션을 시작·중단·재개하는 모든 진입점을 포함한다. 모든 메뉴는 키보드 전용(Arrow + Z + ESC)으로 조작되며, Pattern A (Modal) 인터랙션 패턴을 따른다. 인게임 텍스트와 레이블은 영문으로 표기한다.

---

## 2. 설계 의도 (Design Intent)

- **마찰 최소화:** 일시정지에서 게임 재개(CONTINUE)까지 키 입력 1회(ESC 또는 Z)로 도달한다. 불필요한 확인 단계를 삽입하지 않는다. 단, 저장 손실이 발생하는 QUIT 액션에는 확인 다이얼로그를 배치한다.
- **세계관 연속성:** 메뉴 화면도 ECHORIS의 시각 언어(메가스트럭처 실루엣, 청록+주황 팔레트, 픽셀아트)를 유지한다. 단, 게임플레이 공간과의 거리감을 확보하기 위해 배경 레이어를 어둡게 처리한다.
- **키보드 전용 일관성:** 모든 메뉴는 Arrow(탐색), Z(선택), ESC(뒤로)라는 단일 조작 체계를 사용한다. 마우스 입력은 지원하지 않는다(Phase 1).
- **상태 보존:** 일시정지 메뉴는 게임 루프를 완전히 멈추되, HUD와 게임 월드 화면을 오버레이 뒤에 그대로 유지하여 "언제든 돌아갈 수 있다"는 안정감을 제공한다.

## 3. 상세 규칙 (Detailed Rules)

### 3.1 타이틀 화면 (Title Screen)

#### 구성 요소

| 요소 | 위치 (x, y) | 크기 | 색상 / 스타일 |
|:-----|:-----------|:-----|:-------------|
| 패럴랙스 배경 (3레이어) | 전체 화면 | 640x360 | 청록 팔레트 (#0A1A1A ~ #003344) |
| "ECHORIS" 로고 | (320, 100) | 픽셀아트 스프라이트 | 흰색 기준, 미세 글로우 애니메이션 |
| "PRESS ANY KEY" 텍스트 | (320, 220) | fontSize 8 | 흰색 #FFFFFF, 알파 펄스 |
| 버전 텍스트 (v0.x.x) | (620, 354) | fontSize 5 | #444444, 우하단 정렬 |

#### 패럴랙스 배경 레이어

```
레이어 0 (가장 멀리) — 스크롤 속도 x0.1
  거대 구조물 실루엣 (The Shaft 수직 단면), 어두운 청록 (#0A1A1A)

레이어 1 (중간) — 스크롤 속도 x0.3
  중간 크기 빔/기둥 실루엣, 배경 대비 약간 밝은 (#1A2A2A)

레이어 2 (가까이) — 스크롤 속도 x0.6
  전경 구조물 파편, 주황 강조색 점 (#FF8000 알파 0.3)
```

> 패럴랙스는 마우스/키 입력 없이 자동으로 느리게 수평 이동한다 (속도: 0.05px/frame @ 60fps).

#### "PRESS ANY KEY" 펄스 애니메이션

- 알파값: 0.5에서 1.0 사이를 사인 곡선으로 진동
- 주기: 1.5초 (0.75초 페이드 인, 0.75초 페이드 아웃)
- 공식: `alpha = 0.5 + 0.5 * Math.sin(elapsed * (2 * Math.PI / 1.5))`

#### 로고 글로우 애니메이션

- 로고 스프라이트 주위에 청록색 (#00CED1) 외곽선 글로우를 렌더링
- 글로우 강도: 0.0~0.6을 3.0초 주기로 진동 (로고는 항상 불투명)
- PixiJS BlurFilter를 사용하거나, 스프라이트 1px 확장 복사본에 알파 적용

#### 진입 시퀀스

1. 타이틀 화면 진입 시: 0.5초 페이드 인 (검정 → 타이틀)
2. 임의 키 입력 감지 (ESC 포함): "PRESS ANY KEY" 텍스트가 사라지며 300ms 페이드 아웃
3. 동시에 300ms 페이드 인으로 메인 메뉴 표시
4. 타이틀 음악(BGM): Howler.js로 루프 재생 (Phase 1에서는 무음도 허용)

### 3.2 메인 메뉴 (Main Menu)

#### 구성 요소

```
640x360
┌─────────────────────────────────────────────────┐
│                                                 │
│     [타이틀 배경 -- 알파 0.7 어두운 오버레이]    │
│                                                 │
│         ┌──────────────────────────┐            │
│         │  (로고 축소 버전, 상단)   │            │
│         └──────────────────────────┘            │
│                                                 │
│         ┌──────────────────────────┐            │
│         │  > NEW GAME              │            │
│         │    CONTINUE              │            │
│         │    SETTINGS              │            │
│         │    CREDITS  (P3)         │            │
│         └──────────────────────────┘            │
│                                                 │
└─────────────────────────────────────────────────┘
```

#### 메뉴 패널 상세

| 속성 | 값 |
|:-----|:---|
| 패널 크기 | 180x100px |
| 패널 위치 | x=230, y=160 (중앙 정렬) |
| 패널 배경 | #0D0D1A, 알파 0.90 |
| 패널 테두리 | #2A2A4A, 1px |
| 항목 간격 | 16px (항목 상단 기준) |
| 항목 폰트 | fontSize 8 |
| 기본 텍스트 색상 | #FFFFFF |
| 선택 항목 색상 | #00CED1 |
| 커서 문자 | ">" (선택 항목 왼쪽 4px) |
| 배경 오버레이 | 타이틀 배경 위 #000000 알파 0.40 |

#### 메뉴 항목

| 항목 | 텍스트 | 조건 |
|:-----|:-------|:-----|
| 1 | NEW GAME | 항상 활성 |
| 2 | CONTINUE | localStorage에 세이브 데이터가 있을 때만 활성. 없으면 #444444로 표시, 선택 불가 |
| 3 | SETTINGS | 항상 활성. 설정 서브메뉴(3.5)로 진입 |
| 4 | CREDITS | P3 구현. 현재 표시하지 않음 |

#### 조작 규칙

| 입력 | 동작 |
|:-----|:-----|
| Arrow Up / Arrow Down | 커서 이동 (순환: 마지막 항목 다음 = 첫 항목) |
| Z | 선택한 항목 실행 |
| ESC | 메인 메뉴에서 타이틀 화면으로 복귀 (300ms 페이드) |

#### CONTINUE 세이브 데이터 확인

```
localStorage.getItem('echoris_save') !== null
```

세이브 데이터가 없으면 CONTINUE 항목의 텍스트 색상을 #444444로 설정하고, Z 키 입력 시 반응하지 않는다.

#### 메뉴 전환

- NEW GAME 선택: 300ms 페이드 아웃 후 게임 씬 로드 (새 세이브 데이터 생성)
- CONTINUE 선택: 300ms 페이드 아웃 후 기존 세이브 포인트에서 게임 씬 로드
- SETTINGS 선택: 200ms 페이드로 설정 메뉴(3.5) 진입. 부모 컨텍스트는 MAIN_MENU로 기록
- ESC: 200ms 페이드로 타이틀 화면 복귀

### 3.3 일시정지 메뉴 (Pause Menu)

> **P0 우선순위. 게임 플레이 가능 상태에서 반드시 작동해야 하는 최소 기능.**

#### 트리거

- ESC 키 입력 시 `isPaused` 플래그 토글
- 게임 루프: `isPaused === true`이면 `dt = 0` 처리 (또는 update 루프 스킵)
- HUD는 계속 렌더링 (게임 월드 프레임은 정지 상태로 유지)

#### 레이아웃

```
640x360

│  [HUD -- 뒤에 그대로 표시]                       │
│  [게임 월드 -- 마지막 프레임 정지 상태]           │
│                                                 │
│      ┌──────────────────────────────────┐       │
│      │           PAUSED                 │       │
│      │                                  │       │
│      │    > CONTINUE                    │       │
│      │      SETTINGS                    │       │

│                                                 │
│  [전체 화면 오버레이 #000000 알파 0.60]          │
```

#### 패널 상세

| 패널 배경 | #1A1A2E, 알파 0.95 |

```

  |                              |

      │         PAUSED           │  <- fontSize 10, y=132
      │ ─────────────────────── │  <- 구분선 y=146
      │                          │
y=154 │  >  CONTINUE             │  <- 기본 선택 항목

      │                          │
```

#### 조작 규칙

| 입력 | 동작 |
|:-----|:-----|
| Arrow Up / Arrow Down | 커서 이동 (3개 항목 순환) |
| Z | 선택한 항목 실행 |
| ESC | CONTINUE와 동일 (즉시 게임 재개) |

#### 항목 동작

| 항목 | 동작 |
|:-----|:-----|
| CONTINUE | 일시정지 해제, 200ms 페이드 아웃으로 게임 복귀 |
| SETTINGS | 200ms 페이드로 설정 메뉴(3.5) 진입. 부모 컨텍스트는 PAUSE로 기록 |
| QUIT TO TITLE | 확인 다이얼로그(아래) 표시 |

#### QUIT TO TITLE 확인 다이얼로그

```
x=240                       x=400
  |                            |
y=150 ┌────────────────────────┐
      │  Unsaved progress      │  <- fontSize 7, #FFCC44
      │  will be lost. Quit?   │
      │                        │
      │  > YES       NO        │  <- YES 기본 선택
      │                        │
y=200 └────────────────────────┘
```

| 속성 | 값 |
|:-----|:---|
| 다이얼로그 크기 | 160x60px |
| 위치 | x=240, y=150 (중앙 정렬) |
| 배경 | #1A1A2E, 알파 0.98 |
| 테두리 | #AA4444, 1px (경고 색상) |
| 경고 텍스트 | fontSize 7, #FFCC44 |
| YES / NO 폰트 | fontSize 8 |
| 기본 커서 위치 | NO (실수 방지. YES에서 Enter 누르는 것을 막기 위해 NO 기본 선택) |
| Z (YES 선택) | 300ms 페이드 아웃 후 타이틀 화면으로 이동, 게임 상태 초기화 |
| Z (NO 선택) 또는 ESC | 다이얼로그 닫고 일시정지 메뉴로 복귀 |

> **기본 커서를 NO로 설정하는 이유:** 플레이어가 급하게 ESC를 연타하다가 실수로 세이브 손실이 발생하는 것을 방지한다. "YES"를 의도적으로 선택하려면 Arrow 키를 눌러야 한다.

### 3.4 사망 화면 (Death Screen)

#### 트리거 조건

- `player.hp <= 0` 감지 즉시 사망 시퀀스 시작
- 아이템계와 월드 모두 동일한 시퀀스를 사용하되, 표시 메시지만 다름

#### 사망 시퀀스 타이밍

```
[HP = 0 감지]
     |
     +-- 1.5초 대기 (카메라는 사망 위치 고정. 사망 애니메이션 재생)
     |
     +-- 500ms 페이드 투 블랙 (#000000)
     |
     +-- 800ms 페이드 인으로 "YOU DIED" 텍스트 표시
     |
     +-- 패널 전체 표시 (통계 + 버튼)
     |
     +-- 플레이어 Z 입력 대기
     |
     +-- 300ms 페이드 투 블랙 후 세이브 포인트 씬 로드
```

#### 레이아웃

```
640x360
┌─────────────────────────────────────────────────┐

│       │           YOU DIED                │     │
│       │                                   │     │
```

| 버튼 키 레이블 "[Z]" | 7x7px 다크 박스 (#1a1a1a 알파 0.85, 테두리 #666666 1px, 흰색 텍스트 fontSize 5) |

#### 픽셀 좌표 기준 레이아웃 다이어그램

```
x=180                                     x=460
  |                                          |
y=100 ┌────────────────────────────────────┐
      │           YOU DIED                 │  <- #FF4444, fontSize 12, 중앙, y=112
      │ ─────────────────────────────────  │  <- 구분선 y=128
y=136 │  Time      :  00:12:34             │

```

#### 통계 항목 (P1 선택 구현)

| 항목 | 레이블 | 데이터 소스 |
|:-----|:-------|:-----------|
| 경과 시간 | Time | 세션 시작 시각 기준 누적 경과 초 |
| 처치 수 | Defeated | 세션 중 `enemiesDefeated` 카운터 |
| 골드 변동 | Gold | 마지막 세이브 시점 골드로 롤백. "1,250 -> 980 (-270)" 형식 |
| 획득 아이템 | Items | 세션 중 획득한 아이템 수 |

> **골드 롤백 규칙 (설계 결정):** 사망 시 소지 골드는 마지막 세이브 시점의 골드로 되돌린다. 세이브 이후 획득한 골드만 손실되고, 세이브 이전 골드는 보호된다. 장비 아이템은 유지된다. "세이브 포인트에서 저장해야 안전하다"는 학습을 자연스럽게 유도한다.

#### 아이템계 사망 추가 표시

아이템계 내부에서 사망했을 경우, 통계 영역 아래에 별도 경고 텍스트를 표시한다.

```
y=198 │  ! Item progress lost.             │  <- #FFCC44, fontSize 7
y=208 │    Return to World save point.     │  <- #AAAAAA, fontSize 7
```

- 아이템계 강화 진행도(해당 아이템계에서 얻은 기억 단편, 스탯 강화)는 모두 초기화된다.
- 아이템 자체(장비 아이템 오브젝트)는 유지된다. 강화 진행만 손실된다.

#### Z 키 입력 시 처리

1. 300ms 페이드 투 블랙
2. 게임 상태를 마지막 세이브 데이터로 복원 (HP 최대치, 골드를 세이브 시점 값으로 롤백, 사망 플래그 해제)
3. 마지막 세이브 포인트 위치로 씬 로드
4. 500ms 페이드 인으로 게임 월드 표시

## 4. 엣지 케이스 (Edge Cases)

| 케이스 | 상황 | 처리 방법 |
|:-------|:-----|:---------|
| 아이템계 내 ESC 키 | 아이템계 탈출 확인 프롬프트와 일시정지 메뉴 ESC가 충돌할 수 있음 | 아이템계 탈출 프롬프트(Pattern A Modal)가 활성 상태이면 일시정지 메뉴 ESC를 무시. 프롬프트가 닫힌 후에만 일시정지 진입 허용 |
| 일시정지 중 포커스 손실 | 웹 브라우저 탭 전환 시 ESC 키 이벤트가 누락될 수 있음 | `document.visibilitychange` 이벤트로 탭 비활성화 감지 시 자동으로 `isPaused = true` 설정 |
| 세이브 데이터 없이 CONTINUE | CONTINUE 항목이 활성화된 상태에서 저장 데이터가 삭제됨 | 메인 메뉴 진입 시마다 localStorage 재확인. CONTINUE 항목 색상 동적 업데이트 |
| 사망 중 ESC 입력 | 사망 시퀀스(1.5초 대기 + 페이드) 진행 중 ESC 누름 | 사망 시퀀스 중에는 모든 키 입력 무시. 사망 패널 표시 완료 후 Z 키만 허용 |
| 설정 슬라이더 0 입력 | BGM/SFX 볼륨을 0으로 설정 | 0은 유효한 값. Howler.js volume(0)으로 무음 처리. localStorage에 0 저장 |
| QUIT 확인 중 ESC | QUIT 확인 다이얼로그에서 ESC 입력 | NO 선택과 동일하게 처리 (다이얼로그 닫고 일시정지 메뉴 복귀) |
| 멀티플레이 중 사망 (Phase 3) | 코옵 파티원 사망 시 즉시 사망 화면 표시 여부 | Phase 3 설계 시 결정. 기본 방향: 파티원 전원 사망 시 사망 화면 표시. 1인 사망은 관전 모드 진입 |
| localStorage 쓰기 실패 | 프라이빗 브라우징 모드 등에서 localStorage 접근 불가 | try-catch로 감싸 실패 시 메모리에만 유지. 세션 종료 시 손실 허용 (에러 토스트 표시 불필요) |
| 해상도 변경 중 메뉴 열림 (P2) | 설정 메뉴에서 배율 변경 시 패널 위치 재계산 필요 | 배율 변경 시 메뉴 컨테이너 전체 rebuildLayout() 호출 |
| 사망 후 아이템계 재진입 | 아이템계에서 사망 후 복귀, 동일 아이템에 즉시 재진입 | 허용됨. 아이템계는 새 절차적 생성으로 초기화. 이전 진행도 없음 |

## 5. 검증 체크리스트 (Acceptance Criteria)

### 기능 검증 (Functional Criteria)

**일시정지 메뉴 (P0)**

- [ ] ESC 키 입력 시 게임 루프가 즉시 멈추고 오버레이가 표시된다
- [ ] 일시정지 상태에서 HUD와 게임 월드 마지막 프레임이 오버레이 뒤에 보인다
- [ ] CONTINUE 또는 ESC 재입력 시 게임이 정확히 멈춘 시점부터 재개된다
- [ ] SETTINGS 선택 시 설정 메뉴로 진입하고 BACK 시 일시정지 메뉴로 복귀된다
- [ ] QUIT TO TITLE 선택 시 확인 다이얼로그가 표시된다
- [ ] 확인 다이얼로그 기본 커서는 NO에 위치한다
- [ ] YES 선택 시 타이틀 화면으로 이동하고 게임 상태가 초기화된다
- [ ] NO 또는 ESC 입력 시 확인 다이얼로그가 닫히고 일시정지 메뉴로 복귀된다

**사망 화면 (P0)**

- [ ] HP = 0 판정 후 정확히 1.5초 뒤 페이드 투 블랙이 시작된다
- [ ] "YOU DIED" 텍스트가 800ms 페이드 인으로 표시된다
- [ ] 통계 항목(시간, 처치 수, 골드 손실, 획득 아이템)이 정확한 값을 표시한다
- [ ] 아이템계 사망 시 "Item progress lost" 경고가 추가로 표시된다
- [ ] Z 키 입력 시 마지막 세이브 포인트에서 게임이 재개된다
- [ ] 사망 후 복귀 시 HP가 최대치로 복원된다
- [ ] 사망 후 복귀 시 소지 골드가 마지막 세이브 시점 값으로 롤백된다

**타이틀 화면 (P1)**

- [ ] "PRESS ANY KEY" 텍스트가 1.5초 주기로 알파 펄스 애니메이션을 재생한다
- [ ] 임의 키 입력 후 300ms 페이드 전환으로 메인 메뉴가 표시된다
- [ ] 패럴랙스 배경이 자동으로 느리게 수평 이동한다

**메인 메뉴 (P1)**

- [ ] localStorage에 세이브 데이터가 없으면 CONTINUE가 #444444로 표시되고 Z 키에 반응하지 않는다
- [ ] localStorage에 세이브 데이터가 있으면 CONTINUE가 정상 색상으로 표시되고 선택 가능하다
- [ ] Arrow Up/Down으로 커서가 이동하며 항목 끝에서 처음으로 순환된다
- [ ] Z 키로 NEW GAME 선택 시 새 세이브 데이터를 생성하고 게임 씬을 로드한다
- [ ] ESC 키로 타이틀 화면으로 복귀한다

**설정 메뉴 — 오디오 탭 (P1)**

- [ ] BGM 슬라이더 조작 시 Howler.js 볼륨이 즉시 변경된다
- [ ] SFX 슬라이더 조작 시 SFX 그룹 볼륨이 즉시 변경된다
- [ ] 설정값이 localStorage에 저장된다
- [ ] 게임 재시작 후 저장된 볼륨 설정이 복원된다
- [ ] 볼륨 0 설정 시 소리가 완전히 소거된다
- [ ] Arrow Left/Right로 10 단위 조정이 정확히 이루어진다 (100 이상, 0 미만 초과 불가)

### 경험 검증 (Experiential Criteria)

다음 항목은 플레이테스트로 검증한다.

- [ ] **일시정지 해제 즉시 복귀감:** 일시정지를 해제했을 때 플레이어가 "어디서 멈췄는지" 즉각 파악할 수 있다. 전환 애니메이션이 너무 길어서 재진입 타이밍을 놓치는 일이 없다.
- [ ] **사망 화면 감정 설계:** "YOU DIED" 표시 이후 플레이어가 분노나 혼란 없이 재도전 의욕을 느낀다. 통계 수치가 지난 플레이의 성과를 인정하는 역할을 한다.
- [ ] **QUIT 확인 다이얼로그 실수 방지:** 플레이 세션 5회 중 실수로 QUIT TO TITLE까지 도달한 플레이어가 0명이어야 한다.
- [ ] **설정 볼륨 슬라이더 직관성:** 별도 설명 없이 Arrow 키로 슬라이더를 조작한다는 것을 즉각 파악한다.
- [ ] **타이틀 화면 분위기:** 타이틀 화면을 처음 본 플레이어가 "거대한 무언가의 내부"라는 세계관 분위기를 느낀다.
