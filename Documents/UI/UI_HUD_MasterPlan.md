## 3. HUD 마스터플랜 레이아웃 다이어그램

> **설계 원칙: 통합 HUD**
> 월드와 아이템계의 HUD를 분리하지 않는다. 동일한 레이아웃, 동일한 상수,
> 동일한 코드로 양쪽 공간을 처리한다. 공간별 차이는 HUD가 아니라 **콘텐츠**(맵 오버레이 유무,
> 문 마커 등)에서 발생한다.

### 3.1. 통합 HUD 레이아웃 (월드 + 아이템계 공용)

> **키 표시 원칙: "기능 옆에 키를 붙인다"**
> 별도 키 가이드 패널을 만들지 않는다. 대신 각 HUD 요소 자체에 해당 키 레이블을 붙인다.
> 미니맵 아래에 [M], 인벤토리 아이콘 옆에 [I], Flask 옆에 [R], 좌하단에 [Z][X][C].
> 이렇게 하면 모든 키가 자연스럽게 설명되고, 별도 패널이 화면을 가리지 않는다.
> 화살표 키는 표시 불필요 -- 이동은 자명하다.

```
640x360 (베이스 해상도)
┌─────────────────────────────────────────────────────────────────┐
│ 8px                                                     8px    │
│ ┌───────────────────────────────────────────────────────────┐  │
│ │                                                           │  │
│ │  ┌─ HP 영역 (좌상단) ──────────────────┐ ┌─ 우상단 ────┐ │  │
│ │  │  [ HP 바 100x8 ]                    │ │ ┌────────┐  │ │  │
│ │  │  ●●●[R] 48/100  (Flask+키+HP텍스트) │ │ │미니맵  │  │ │  │
│ │  └─────────────────────────────────────┘ │ │128x72  │  │ │  │
│ │                                          │ │        │  │ │  │
│ │                                          │ └────────┘  │ │  │
│ │                                          │  [M]  [I]   │ │  │
│ │                                          │  G: 12,345  │ │  │
│ │           [게임 플레이 영역]              └─────────────┘ │  │
│ │              ★ 캐릭터                                     │  │
│ │                                                           │  │
│ │                                                           │  │
│ │  위치 텍스트                                               │  │
│ │  "Tier 1 - Entrance Hall"                                  │  │
│ │                                                           │  │
│ │  [Z]Jump [X]Atk [C]Dash                                   │  │
│ │  (좌하단, 액션 키 3개)                                     │  │
│ └───────────────────────────────────────────────────────────┘  │
│ 8px                                                     8px    │
└─────────────────────────────────────────────────────────────────┘
```

**통합 HUD 사양:**

| 요소 | 위치 (x,y) | 크기 | 비고 |
|:---|:---|:---|:---|
| HP 바 | (8, 8) | **100x8** | 현재 80x6 → 확대. 화면 15.6% |
| HP 보더 | (7, 7) | 102x10 | 1px 보더 (#444444) |
| Flask 아이콘 | (8, 18) | **8px** 원 x 최대 8개, gap 2px | 현재 6px → 8px 확대 |
| **Flask 키 레이블** | (Flask 마지막 원 우측 +2, 18) | **[R]** 다크 박스 7x7 | Flask 아이콘 바로 옆 |
| HP 텍스트 | ([R] 우측 +3, 18) | fontSize 8 | "48/100" |
| 미니맵 | (504, 8) | 128x72 | 월드에서만. 아이템계에서 숨김 |
| **미니맵 키 레이블** | (504, 82) | **[M]** 다크 박스 7x7 | 미니맵 하단 좌측 |
| **인벤토리 키 레이블** | (514, 82) | **[I]** 다크 박스 7x7 | [M] 우측에 gap 3px |
| 골드 텍스트 | (632, 88) | fontSize 8, 우측 정렬 | [M][I] 아래 |
| **액션 키 바** | (8, 344) | [Z]Jump [X]Atk [C]Dash | 좌하단, 아래 상세 |
| 위치 텍스트 | (8, 334) | fontSize 8 | 액션 키 바 위 |
| Vignette | 전체 | 8px 두께 | 피격 시 빨간 테두리 페이드 |

#### 키 레이블 배치 상세

**A. Flask 옆 [R] 키 (좌상단, HP 영역 내)**

```
  ●●●[R] 48/100
  ~~~  ↑  ~~~~~
Flask  키  HP텍스트
```

- Flask 아이콘 나열이 끝나는 지점 바로 우측에 [R] 다크 박스 배치
- [R] 우측에 gap 3px → HP 텍스트
- Flask가 0개이면 [R] alpha 0.4 (회색)

**B. 미니맵 하단 [M] + [I] 키 (우상단)**

```
  ┌────────┐
  │미니맵   │
  │128x72  │
  └────────┘
   [M]  [I]     G: 12,345
```

- 미니맵(128x72, x=504, y=8) 바로 아래 (y=82)
- [M] x=504, [I] x=514 (gap 3px)
- 골드 텍스트는 [M][I] 아래 (y=88 → 현재 y=80에서 약간 내림)
- **아이템계에서:** 미니맵 숨김 + [M] 숨김. [I]만 우상단에 단독 표시

**C. 좌하단 액션 키 [Z][X][C] (좌하단)**

```
  [Z]Jump  [X]Atk  [C]Dash
```

- y=344 (현재 위치 텍스트 자리 → 위치 텍스트는 y=334로 올림)
- 각 액션: [키 박스 7x7] + 액션명(fontSize 5) + gap 6px → 다음 액션
- 대시 렐릭 미획득 시 [C]Dash alpha 0.4

| 요소 | x | y | 구성 |
|:---|:---|:---|:---|
| [Z]Jump | 8 | 344 | 키박스 7x7 + "Jump" fontSize 5 |
| [X]Atk | 38 | 344 | 키박스 7x7 + "Atk" fontSize 5 |
| [C]Dash | 62 | 344 | 키박스 7x7 + "Dash" fontSize 5 |

#### 키 레이블 공통 비주얼

| 속성 | 값 (640x360) | @1080p (3x) |
|:---|:---|:---|
| 키 박스 크기 | **7x7px** | 21x21px |
| 키 박스 배경 | #1a1a1a, alpha 0.85 | -- |
| 키 박스 보더 | #666666, 1px | -- |
| 키 박스 텍스트 | #FFFFFF, fontSize 5, 중앙 정렬 | 15px |
| 액션명 텍스트 | #AAAAAA, fontSize 5 | 15px |
| 대비 | 흰 on #1a1a1a = **15.4:1** | WCAG AAA 초과 |

#### 키 표시 총 목록 (전부 확인)

| 키 | 표시 위치 | 어떻게 설명되는가 |
|:---|:---|:---|
| **←→** | 미표시 | 이동은 자명. 환경 강제로 학습 |
| **↑↓** | 미표시 | 카메라 룩업/다운은 보조 기능. 자연 발견 |
| **Z** | 좌하단 `[Z]Jump` | 액션 키 바 |
| **X** | 좌하단 `[X]Atk` | 액션 키 바 |
| **C** | 좌하단 `[C]Dash` | 액션 키 바 |
| **R** | Flask 옆 `[R]` | HP 영역 내 컨텍스트 |
| **I** | 미니맵 하단 `[I]` | 미니맵 옆 컨텍스트 |
| **M** | 미니맵 하단 `[M]` | 미니맵 옆 컨텍스트 |
| **ESC** | 미표시 | 범용 관례. 게이머 본능 |

**= 8개 키 전부 설명 완료. 별도 키 가이드 패널 없음.**

**공간별 차이 (HUD 외부에서 처리):**

| 차이점 | 월드 | 아이템계 |
|:---|:---|:---|
| 미니맵 (우상단 128x72) | 상시 표시 | **없음** |
| [M] 월드맵 오버레이 | 작동 | **작동 안 함** (M키 무반응) |
| 내비게이션 보조 | 미니맵 + 월드맵 + 마커 | 문 마커(초록 발광)만 |
| 위치 텍스트 내용 | Tier명 + 구역명 | Stratum + 아이템명 + Lv |
| 데미지 숫자 | 없음 | 없음 |

> **아이템계 내비게이션 철학:** Spelunky와 동일하게 맵을 일절 제공하지 않는다.
> M키도 아이템계에서는 비활성. 플레이어는 문 마커(초록 발광)와 자신의 기억에만 의존한다.
> "다음 방에 뭐가 있을지 모르는" 긴장감이 아이템계 스파이크의 핵심 요소다.

### 3.3. 키 프롬프트 시스템 (Dead Cells 참조)

> **리서치 근거:** Dead Cells HUD 키 표시 + The End is Near 업데이트 3종 아이콘
> **설계 원칙:** 새로운 플레이어가 키를 즉시 인지할 수 있도록, 모든 인터랙션 지점에 키 아이콘을 표시한다.
> 현재 `ControlsOverlay.ts`(우상단 텍스트 목록)는 제거하고, 아래 3가지 방식으로 대체한다.

#### A. HUD 키 레이블 -- 무기/스킬 슬롯 내장 (상시 표시)

```
┌─ 하단 좌측 (x=8, y=340) ─────────────────────────┐
│                                                    │
│  ┌──────┐  ┌──────┐  ┌──────┐                     │
│  │ 무기 │  │ 스킬 │  │Flask │                     │
│  │ 아이콘│  │ 아이콘│  │ 아이콘│                     │
│  │      │  │      │  │      │                     │
│  │  [Z] │  │  [X] │  │  [C] │                     │
│  └──────┘  └──────┘  └──────┘                     │
│  16x16      16x16      16x16                      │
│  gap 2px                                           │
└────────────────────────────────────────────────────┘
```

**키 레이블 비주얼 사양 (640x360 기준):**

| 속성 | 값 | @1080p (3x) | 비고 |
|:---|:---|:---|:---|
| 형태 | 라운드 사각형 (1px radius) | 3px radius | Dead Cells 다크 박스 스타일 |
| 크기 | **7x7px** | 21x21px | 슬롯 아이콘(16px)의 약 44% |
| 배경색 | #1a1a1a (alpha 0.85) | -- | 어두운 반투명 |
| 보더 | #666666, 1px | 3px | 얇은 밝은 테두리 |
| 텍스트 | 흰색 (#FFFFFF), fontSize 5 | 15px | 대문자 1글자 ("Z","X","C") |
| 위치 | 슬롯 아이콘 하단-우측 코너 | -- | 아이콘에 살짝 겹침 (오프셋 -2, -2) |
| 대비 | 흰 on #1a1a1a = **15.4:1** | -- | WCAG AAA 초과 달성 |

**슬롯 배치 (향후 구현 시):**

| 슬롯 | 키 | 위치 |
|:---|:---|:---|
| 공격 (Attack) | Z | 좌하단 첫 번째 |
| 스킬/서브 (Sub) | X | 좌하단 두 번째 |
| 대시 (Dash) | C | 좌하단 세 번째 |
| 인벤토리 | I | 미니맵 근처 (월드) 또는 우하단 (아이템계) |

> 현재 Build 0에는 무기/스킬 슬롯 HUD가 없으므로, 슬롯 구현 시 키 레이블을 함께 넣는다.
> 그때까지는 ControlsOverlay.ts를 유지하되, 스타일을 다크 박스로 변경한다.

#### B. 컨텍스트 프롬프트 -- 상호작용 오브젝트 (거리 기반)

플레이어가 상호작용 가능한 오브젝트에 접근하면 **월드 스페이스**에 키 프롬프트 표시.

```
                    ╚═════════════╝
```

**사양:**

| 키 박스 크기 | **8x8px** | 24x24px |

| 텍스트 레이블 | fontSize 6, 흰색 | 18px |

| 배경 패널 | #000000 alpha 0.6, 자동 크기, padding 2px | -- |
| 위치 | 오브젝트 상단 중앙, 오브젝트에서 4px 위 | -- |

#### C. 튜토리얼 인라인 프롬프트 -- 환경 교습 텍스트
