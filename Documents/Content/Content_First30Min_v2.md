# 첫 30분 경험 흐름 — v2: 대공동 (The Shaft)

> **MVP 경험 플로우.** BLAME! 오마주 메가스트럭처. 과묵한 주인공.
> 대체 문서: `Content_First30Min_ExperienceFlow.md` (v1, 다화형 에르다)

## 0. 설계 공리

| 공리 | 규칙 |
|:--|:--|
| 에르다는 말하지 않는다 | 독백 없음. 생각 풍선 없음. 플레이어 캐릭터 대사 0줄. |
| 세계가 스스로 설명한다 | 건축, 조명, 적 배치, 아이템 배치 — 이것들이 곧 튜토리얼이다. |
| 폐소감 우선 | 모든 방은 좁게 시작한다. 넓은 공간은 획득하는 것이다. |
| 위기는 눈에 보여야 한다 | 플레이어는 메가스트럭처가 증식하고, 벽이 좁혀오고, 없던 복도가 생기는 것을 직접 목격해야 한다. |
| 아이템계 = 계시 | 첫 다이브가 플레이어가 "이 게임이 뭔지" 이해하는 순간이다. |

```
  ┌─────┐  ┌───────┤  [에르다의 대장간]─[시장]      │

     │         │                    │
     │    [격벽 2]             [대공동 전망]

     │         │                Tier 3이

  막다른 길 — 증식하는

```

**첫 30분 크리티컬 패스:**

```
대장간 → 봉인된 복도 → 대장간 (막힘)
  → 모루 타격 → 바닥 붕괴 → 아이템계
  → 보스 클리어 → 대장간 귀환
  → 증식 구역 발견 (없던 복도가 생김)
  → 격벽 2에서 STR 게이트 확인
  → 그림자 시장 도달
  = 30분 종료
```

## 2. 스크린별 상세 설계

### Act 1: 대장간 (0:00 ~ 3:00)

#### SCREEN 1 — 에르다의 대장간 (스폰)

어두운 방. 석벽. 중앙에 모루. 벽에 도구들. 램프 하나.
에르다가 모루 옆에 서 있다. 에코를 오른손에 들고 있다.

> **교습 패턴: A (공간 유도) + B (시각 차이화) + E (환경 서사)**

| 항목 | 내용 |
|:--|:--|
| 튜토리얼 | 없음. 플레이어가 방을 탐험하며 이동을 스스로 익힌다. |
| 아이템 | 모루 위에 검이 놓여 있다. 격벽 틈에서 꺼낸 것 (Magic 등급). |
| 상호작용 | 모루 근처에서 ↑ → 에르다가 검을 집어 살핀다. |
| 시각 단서 | 검이 희미하게 파란빛을 낸다 (Magic 레어리티). |
| 환경 스토리텔링 — 균열 지도 | 벽에 대공동 단면 지도가 핀으로 꽂혀 있다. 격벽 위치마다 마커가 있고, 여러 개에 X표(봉합 완료), 하나에 ?표(미조사). 의뢰서 따위는 없다. |
| 환경 스토리텔링 — 도구 | 선반에 격벽 조각 샘플, 균열 측정 도구, 에코 충전대. 에르다가 이 일을 오래 해왔다는 것을 보여줌. |
| 키리이 대응 | 키리이의 망막 분석 정보 = 에르다의 균열 지도. 둘 다 "찾고 있다"는 사실을 한 장면에 보여줌. |

**공간 레이아웃 / 조명 / 안전 구역:**
- 카메라 방향: 에르다 스폰 직후 오른쪽을 향해 있다. 모루가 화면 중앙~오른쪽에 배치.
- 조명 앵커: 모루 위 램프가 유일한 광원. 검의 파란빛 (#6969FF)이 램프보다 작지만 색 대비로 즉시 눈에 띈다.
- 안전 구역: 방 전체가 안전. 적 없음. 플레이어가 자유롭게 탐험하며 이동을 체화할 수 있다.
- 시선 유도: 방 좌측 벽(균열 지도)과 방 중앙(모루+검)이 두 개의 시각 앵커. 플레이어는 자연스럽게 둘 다 조사하게 된다.

대사 없음. 텍스트 프롬프트 없음. 플레이어가 움직이고, 검을 보고, 상호작용한다. 에르다가 누구인지, 왜 이 일을 하는지 설명하지 않는다. 행동만 있다.

---

#### SCREEN 2 — 봉인된 복도 (교습: 이동 + 점프)

대장간 문을 지나면: 오른쪽으로 뻗은 복도. 천장에 석재 파이프. 어두운 산업 조명.

> **교습 패턴: A (공간 유도) + B (시각 차이화)**

| 항목 | 내용 |
|:--|:--|
| 교습 | 이동 (좌/우), 점프 (바닥 틈새, 2타일 너비) |
| 적 | 없음. 순수 이동 학습 구간. (Celeste 스크린 1 원칙 — 한 스크린에 하나의 개념) |
| 건축 | BLAME! 스타일: 거대한 파이프, 리벳 박힌 금속 벽, 에르다를 왜소하게 만드는 스케일 |
| 복도 끝 | 봉인된 격벽 문. 가장자리에 균열이 보인다. |

**공간 레이아웃 / 조명 / 안전 구역:**
- 카메라 방향: 오른쪽. 직선 수평 복도. 진행 방향이 단 하나 — 패턴 A(공간 유도)의 교과서적 적용.
- 갭 배치: 2타일 너비의 바닥 틈새 2~3개. 점프하지 않으면 추락하도록 강제. 텍스트 없이 점프를 가르침.
- 조명: 천장 산업 조명이 복도를 따라 일정 간격으로 배치. 플레이어를 앞으로 이끈다.
- 봉인된 문 시각: 문 가장자리 균열이 주황 빛으로 희미하게 발광 (패턴 B) — "여기는 뭔가 있다"는 신호. 지금은 못 열지만 플레이어는 기억한다. 이것이 메트로베니아 훅.
- 안전 구역: 이 복도 전체. 실패(갭 추락)해도 낮은 위치로 떨어지며 피해 없음.

플레이어가 봉인된 문에 도달한다. 통과 불가. 대장간으로 돌아가야 한다.

**설계 의도:** 플레이어는 이제 막힌 길이 있다는 것을 안다. 이것이 메트로베니아 훅 — "나중에 다시 오면 열 수 있겠지."

---

#### SCREEN 3 — 대장간 귀환 + 첫 전투 (교습: 전투)

대장간으로 돌아오면 복도 입구 근처에 슬라임 2마리가 나타나 있다.

> **교습 패턴: D (죽음을 교습 도구로) + F (보상의 직접성)**

| 항목 | 내용 |
|:--|:--|
| 교습 | 공격 (Z키). 슬라임은 느리고 2히트에 처치. 첫 히트스탑 경험. |
| 감각 | 플레이어의 첫 타격이 3프레임 동안 화면을 멈춘다. "이거 좋은데." |
| 아이템 드롭 | 슬라임 하나가 Normal 검을 드롭. 위를 지나가면 획득. |

**공간 레이아웃 / 조명 / 안전 구역:**
- 슬라임 배치: 복도 입구(오른쪽)에서 등장, 에르다 방향으로 천천히 접근. 슬라임이 먼저 에르다에게 다가온다 — "Z키로 공격" 프롬프트 대신 슬라임의 이동이 공격 유발. 패턴 A의 적용.
- 실패 설계 (패턴 D): 슬라임 이동 속도 낮음(이동 속도 x0.4). 피격 시 넉백으로 밀려남. 처음 두세 번 맞아도 치명적이지 않다. Another World 원칙: 재도전 비용이 낮으면 죽음이 학습이 된다.
- 시각 빵부스러기: 슬라임 하나가 드롭한 Normal 검은 흰색 발광. 바닥에 놓인 상태에서도 눈에 띈다 (패턴 B). 에르다가 위를 지나가는 것만으로 자동 획득 — 아이템 획득 메커닉을 마찰 없이 교습.

슬라임 처치 후, 플레이어는 모루 위에 의뢰 검이 있는 대장간으로 돌아온다.

### Act 2: 다이브 (3:00 ~ 8:00)

#### SCREEN 4 — 에코 타격 (THE MOMENT)

플레이어가 의뢰 검이 있는 모루에 접근한다. ↑으로 상호작용.

> **교습 패턴: C (행동 시연) + F (보상의 직접성)**
> **Super Metroid 에테쿤 원칙 적용 — "게임이 먼저 시연하고, 플레이어가 나중에 복제한다"**

에르다가 검을 모루 위에 올린다. 에코를 든다. 내려친다.

**에코 타격 시퀀스 — 설계 의도:**
이 시퀀스는 플레이어가 입력하는 것이 아니라 에르다가 스스로 수행하는 컷씬 형태의 행동이다. 플레이어는 관찰자가 된다. 이것이 패턴 C(행동 시연)의 핵심 — "본 것을 나중에 따라한다." S11(두 번째 다이브)에서 플레이어는 같은 동작을 직접 선택하고 실행한다. 시연에서 주체적 실행으로의 전환이 이 두 스크린의 설계 축이다.

```yaml
echo_strike_sequence:
  step_1:
    action: "에코가 모루 위의 검을 타격"
    duration_frames: 1

  step_2:
    action: "히트스탑"
    duration_frames: 10
    effect: "화면 정지. 모든 사운드 컷. 고주파 공명음 시작."
    note: "이건 평범한 타격이 아니다. 뭔가 이상하다."

  step_3:
    action: "모루 균열 발광"
    effect: "모루 표면의 미세 균열이 빛나기 시작. 에코가 진동."
    duration_ms: 1500

  step_4:
    action: "바닥 붕괴"
    effect: "FloorCollapse.ts — 모루 아래 타일이 아래로 부서짐"
    camera: "화면 흔들림 (진폭: 대)"
    note: "에르다가 떨어진다. 검이 떨어진다. 모든 것이 떨어진다."

  step_5:
    action: "백색 섬광 → 씬 전환"
    effect: "완전 백색 → 0.5초 무음 → 아이템계 BGM 시작"
```

**대사 없음. 설명 없음.** 바닥이 무너진다. 에르다가 떨어진다. 플레이어는 이제 검 안에 있다.

#### SCREEN 5 — 아이템계 착지 (공간 도입)

에르다가 금속성 공간에 착지한다. 벽에 결정 무늬가 있다 — 검의 칼날과 같은 무늬.

> **교습 패턴: A (공간 유도) + B (시각 차이화)**

| 항목 | 내용 |
|:--|:--|
| 시각 | 벽이 파란빛으로 일렁인다 (Magic 레어리티 색상). 좁은 복도. |
| 플레이어 인식 | "검 안에 있는 거야...?" — 검과 일치하는 시각 디자인만으로 전달. |
| 첫 방 | 작다. 오른쪽으로 출구 하나. |

**공간 레이아웃 / 조명 / 안전 구역:**
- 착지 방 레이아웃: 고정 레이아웃 (절차적 생성 아님). Dead Cells 원칙 — 첫 방은 항상 동일하게 보장되어 일관된 최초 인상을 만든다.
- 시각 빵부스러기 (패턴 B): 착지 지점 벽의 결정 무늬가 Magic 검의 칼날 무늬와 동일한 패턴. 텍스트 없이 "이 공간이 검의 내부임"을 색과 패턴으로 전달. 파란빛 (#6969FF)이 대장간의 Magic 검 색상과 일치.
- 카메라 방향: 오른쪽 단방향. 출구가 오른쪽 하나만 존재 — 선택지 없이 전진.
- 안전 구역: 착지 방 자체는 적 없음. 플레이어가 공간을 인식하고 "여기가 어디지?"를 스스로 추론할 여유를 준다.

---

#### SCREEN 6 — 아이템계 전투 (3~4개 방)

4x4 그리드 지층. 절차적 생성 방에 적 배치.

> **교습 패턴: G (순차 레이어링) + F (보상의 직접성)**

| 항목 | 내용 |
|:--|:--|
| 적 | InnocentNPC (적대, 야생). 에르다에게 돌진하는 붉은 구체. |
| 전투 | 월드와 동일한 조작. 히트스탑, 넉백, 데미지 넘버. |
| 진행 | 방 클리어 → 아이템 EXP 축적 (HUD에 작은 카운터로 표시). |
| 방 수 | 보스 전 3~4개 방. 좁은 복도, 발판, 수직 통로. |

**공간 레이아웃 / 조명 / 안전 구역:**
- 새 레이어 하나만 추가 (패턴 G): 전투 메커닉(Z키)은 S3에서 이미 체화됨. 이 스크린이 추가하는 새 개념은 아이템 EXP 카운터 하나뿐. 기존에 아는 것 위에 새 정보 한 겹.
- 야생 이노센트 시각 차이화 (패턴 B): 붉은 구체 (#FF0000) + 빠른 돌진 패턴. 월드의 슬라임(중립 색상)과 대비되어 "이것은 다른 종류의 적"임을 색으로 전달.
- EXP 카운터 피드백 (패턴 F): 적 처치 시마다 HUD 소형 카운터가 즉시 수치 상승. 행동의 축적이 실시간 가시화. 플레이어는 숫자가 오르는 것을 보고 "계속 싸우면 뭔가 된다"는 것을 추론한다.
- 시각 빵부스러기: 방 문이 열릴 때 파란 파티클이 흩어진다. 다음 방의 방향을 파티클이 암시한다.

---

#### SCREEN 7 — 기억의 수문장 보스

아레나 방. 에르다 뒤로 문이 잠긴다.

> **교습 패턴: D (죽음을 교습 도구로) + F (보상의 직접성)**

| 항목 | 내용 |
|:--|:--|
| 보스 | 기억의 수문장 (아이템 장군). HP 80, ATK 12. |
| 패턴 | 패턴 1: 수평 돌진 (좌우 왕복). 패턴 2: 점프 내려찍기. |
| 텔레그래프 | 각 공격 전 0.5초 준비 동작. 2~3회 시도면 학습 가능. |
| 사망 | 보스가 파티클로 폭발. |

**사망 / 리스폰 설계 (패턴 D):**
- 보스에게 사망 시 보스 방 입구에서 3초 이내 리스폰. 이것은 협상 불가 설계 요구사항이다.
- 근거: Another World 원칙 — 재시작 속도가 1~3초 이내일 때 죽음이 "분노"가 아닌 "아, 그렇구나"가 된다. 리스폰이 느리면 패턴 D가 완전히 무너진다.
- 보스 방 직전에 체크포인트 노드 존재 (비가시적). 아이템계 진입 지점으로 돌려보내지 않음.

**보스 클리어 보상:**

```yaml
boss_clear:
  ui_display:
    text: "ATK +8"
    format: "화면 중앙 큰 텍스트. 금색. 화면 섬광."
    timing: "보스 사망 1초 후 표시"
  mechanical_effect:
    item_exp: 600  # Lv0 → Lv2 (ATK 15 → 23)
  feel: "숫자가 나타나는 그 순간이 THE moment. 말은 필요 없다."
```

**"ATK +8" 피드백 — 텍스트 없는 게임의 가장 중요한 UI 요소:**
이 순간은 패턴 F(보상의 직접성)의 집약점이다. 에르다 0발화 설계에서 플레이어에게 "아이템계 = 성장의 공간"임을 전달할 수 있는 유일한 수단이 이 숫자다. 세 채널 동시 활용:
1. 시각: 화면 중앙 금색 큰 텍스트 + 화면 전체 섬광
2. 청각: 레벨업 공명음 (에코 공명과 다른 음색)
3. 수치: HUD의 ATK 수치가 실시간 카운트업

이 세 채널이 동시에 활성화될 때 플레이어는 "강해졌다"는 것을 추론이 아닌 감각으로 받아들인다.

#### SCREEN 12 — 귀환 + ATK 게이트 발견

대장간 복귀. 더 강한 무기.

Screen 2의 봉인된 복도가 이제 증식 구역을 통해 도달 가능 — 증식하는 벽이 우연히 우회 경로를 열었다.

| 항목 | 내용 |
|:--|:--|
| 새 경로 | 증식 구역 → 옆 복도 → 격벽 2 부근으로 연결. |
| ATK 게이트 | 격벽 2 보임. 표시: "ATK 40". (주: 이 문서의 "STR 게이트" 표기는 ATK 게이트로 정정됨. System_World_StatGating.md 참조) |
| 플레이어 ATK | Rare Lv2 = 42 ATK면 간신히 통과. |

| 또는 | ATK 부족 시 → 에코가 튕겨남. 붉은 섬광. 더 강해져야 함. |

**"루프 인식" 순간 — 명시적 설계 노트:**
이 스크린은 PA 핵심 순환의 첫 번째 완전 체험 지점이다. 플레이어가 여기서 처음으로 다음 논리를 의식적으로 연결한다:
```
아이템계 진입 → 강해짐 (ATK +8 경험) → 막혔던 게이트가 열림 → 새 영역
```
이 연결이 일어나는 순간이 "루프 인식"이다. 설계 책임: 게이트 실패 피드백(에코 튕김 + 붉은 섬광)이 명확해야 한다. "왜 안 열리지?"가 아니라 "ATK가 부족하다, 강해져야 한다"는 결론이 피드백에서 즉시 읽혀야 한다.

게이트 성공 시 추가 피드백: 격벽이 부서지는 순간 하얀 파티클이 앞쪽(진행 방향)으로 흩어진다. 시각 빵부스러기가 다음 경로를 암시한다.

아래가 보인다. 끝없는 어둠. Tier 3, 4, 5... 보이지 않는 아래.
에르다의 균열 지도에 ?표가 남아있다. 아래쪽.

에코가 희미하게 진동한다. 아래에서 무언가가 올라오는 것 같은 — 공명.

에르다가 아래를 본다. 한 발짝 앞으로.

**Hyper Light Drifter 원칙 — 모션 스토리텔링:**
에르다는 한 마디도 하지 않는다. 하지만 에르다의 시선 방향(아래)과 신체 언어(한 발짝 앞으로)가 "저리 가야 한다"는 결의를 전달한다. 균열 지도의 ?표가 에르다의 손에서 잠깐 보인다. 방향이 일치한다. 텍스트 없이 목표가 전달된다.

에코 공명 시각: 에코에서 작은 파란 파티클이 아래 방향으로 흘러 내려간다. 방향 지시자이자 "에코가 무언가에 반응하고 있다"는 신호.

**페이드 아웃.** 텍스트 없음. 통계 화면 없음. 침묵과 어둠뿐. 그리고 타이틀: **PROJECT ABYSS**.

**30분 종료.** 플레이어는 이제:
- 핵심 루프를 경험했다 (탐험 → 다이브 → 강화 → 게이트 → 새 영역)
- 위기를 보았다 (증식하는 벽)
- 세이브 포인트에 도달했다 (그림자 시장)
- 계속할 동기가 있다: **아래에 더 있다. 에코가 반응한다. 내려가야 한다.**
- 의뢰를 받은 게 아니다. 에르다가 스스로 가는 것이다.
