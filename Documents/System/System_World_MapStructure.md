## 1. 개요 (Concept)

### 1-1. 설계 의도 (Design Intent)

Project Abyss의 월드는 **메트로베니아의 비선형 탐험**을 핵심으로 하는 단일 거대 성(Castle)이자 세계다. 플레이어는 중앙 성채에서 출발하여 6개 구역을 자유로운 순서로 탐험하고, 최종적으로 심연의 구에 도달한다.

이 구조는 세 가지 설계 목표를 동시에 달성해야 한다:

1. **메트로베니아 비선형성**: 플레이어가 탐험 순서를 결정하되, 어떤 순서를 택해도 반드시 깰 수 있는 구조 (Always Winnable)
2. **3-Space 솔로 격리**: 월드는 1인 플레이 전용 공간으로, 고독한 탐험감과 발견의 미지감을 보호한다
3. **순환 구조 연결**: 월드 탐험 → 아이템 획득 → 아이템계 진입 → 장비 강화 → 스탯 게이트 해금 → 새 구역 진입의 순환을 자연스럽게 유도한다

### 1-2. 설계 근거 (Reasoning)

| 결정 | 근거 |
| :--- | :--- |
| 허브 & 스포크 + 원형 순환 토폴로지 | 월하의 야상곡의 선형 구역 연결과 할로우 나이트의 극도의 비선형성 사이의 중간 지점. 충분한 자유도를 제공하면서도 방향성을 유지한다 |
| 7개 구역 (중앙 + 6개 주변) | 마법 수 7. 충분한 다양성 제공 (바이옴, 테마, 난이도)하면서 맵 복잡도 관리 가능 |
| 이중 게이트 (능력 + 스탯) | 순수 메트로베니아(능력 게이트만)는 아이템계로 진입할 동기가 약함. 스탯 게이트가 파밍 필요성을 자연스럽게 창출 |
| 절차적 마이크로 + 핸드크래프트 매크로 | 매크로 구조 고정은 각 구역의 정체성과 서사를 보호. 마이크로 절차적 변화는 재방문 가치를 제공 |

### 1-3. 3대 기둥 정렬 (Pillar Alignment)

| 기둥 | 월드 맵 구조에서의 구현 |
| :--- | :--- |
| **탐험가** | 능력 게이트가 세계 진행을 정의. 각 능력 획득 시 접근 가능한 구역이 기하급수적으로 확장된다. 플레이어는 지도를 채우며 탐험가가 된다 |
| **장인** | 스탯 게이트가 아이템계로의 자연스러운 통로. "저 벽을 부수려면 더 강해져야 한다"는 인식이 야리코미 동기를 생성한다 |
| **모험가** | 아이템계의 능력 게이트도 이 원칙을 따른다. 파티 역할 분담으로 솔로 불가능한 구역을 함께 극복한다 |

### 1-4. 저주받은 문제 (Cursed Problems)

| 문제 | 상충하는 요구 | 설계 해법 | 희생/제약 |
| :--- | :--- | :--- | :--- |
| 자유로운 탐험 vs 진행 보장 | 플레이어가 원하는 순서로 탐험해야 하는데, 모든 순서에서 깰 수 있어야 함 | Critical Path 알고리즘: 기본 능력만으로 게임을 클리어할 수 있는 경로를 설계. 능력/스탯 게이트는 Critical Path 위에 배치되지 않음 | 모든 선택지가 완전히 자유로운 것은 아님. 특정 구역은 다른 구역을 먼저 탐험해야 열릴 수 있음 |
| 파밍 효율 vs 탐험 동기 | 아이템계에서 무한 파밍하면 월드 탐험을 하지 않으려 함 | 스탯 게이트만 파밍으로 해결 가능. 능력 게이트는 월드에서만 획득. 양쪽 다 필요해야만 진행 가능 | 플레이어는 선호하지 않는 콘텐츠(탐험 또는 파밍)도 일부 강제됨 |
| 절차적 생성의 반복감 vs 탐험감 | 절차적 생성하면 방 구조가 매번 다르지만 정체성 상실 위험. 핸드크래프트만 하면 절차적 재방문 가치 손실 | 구역별 테마와 핸드크래프트된 핵심 건축 구조는 고정. Room 내부 Chunk 배치와 오브젝트 배치만 절차적 변화 | 일부 구역에서는 여전히 반복 구조감이 나타날 수 있음 |
| 맵 복잡도 관리 vs 콘텐츠 다양성 | 더 많은 구역 = 더 다양한 탐험, 하지만 구현 비용 증가 | 7개 구역으로 제한. 각 구역마다 고유한 바이옴, 테마, 난이도, 심연 침식 정도를 차별화 | 8개 이상 구역 추가 불가. 콘텐츠 다양성은 구역 내부 절차적 변화와 비경로 탐험으로 확보 |

## 2. 핵심 규칙 (Core Rules)

### 2-1. 월드 토폴로지 (World Topology)

```
                    [마법 연구소]
                          |
      [빙결 동굴] ─── [중앙 성채] ─── [천공의 탑]
                          |
                   [지하 수로]
                          |
                    [묘지/카타콤]
                          |
                    [심연의 구]
```

**원형 순환 (Circular Loop) 구조:**
- **중앙 성채 (Central Keep)**: 게임 시작점. 모든 6개 주변 구역과 연결된 허브.
- **인접 구역 (Tier 1)**: 중앙 성채와 직접 연결. 마법 연구소, 빙결 동굴, 천공의 탑, 지하 수로.
- **심연 구역 (Tier 2)**: 묘지/카타콤과 심연의 구. 여러 경로를 통해 도달 가능.

**경로 선택의 자유도:**
- 플레이어는 Tier 1 구역 4개를 자유로운 순서로 탐험할 수 있다.
- Tier 1 구역을 모두 탐험할 필요는 없지만, 특정 구역을 진입하려면 특정 능력/스탯 게이트를 통과해야 한다.
- 묘지/카타콤은 최소 2개 Tier 1 구역 클리어 후 접근 가능.
- 심연의 구는 4개 Tier 1 구역을 모두 클리어하거나, 특정 조건을 만족해야 접근 가능.

### 2-2. 구역 연결 규칙 (Zone Connectivity Rules)

| 구역 | 게이트 위치 | 진입 조건 | 출구 조건 |
| :--- | :--- | :--- | :--- |
| **중앙 성채** | Hub (항상 개방) | 게임 시작 | 자유 |
| **마법 연구소** | 중앙 성채 동쪽 | 기본 능력 (이단 점프 미필수) | 보스 처치 |
| **빙결 동굴** | 중앙 성채 서쪽 | 기본 능력 | 보스 처치 |
| **천공의 탑** | 중앙 성채 북동쪽 | 기본 능력 + STR 게이트 10 | 보스 처치 |
| **지하 수로** | 중앙 성채 남쪽 | 기본 능력 | 보스 처치 |
| **묘지/카타콤** | 지하 수로 남쪽 | 2개 Tier 1 구역 클리어 | 보스 처치 |
| **심연의 구** | 묘지 최심부 | 4개 Tier 1 구역 클리어 또는 최종 능력 획득 | 최종 보스 처치 |

### 2-3. Critical Path 정의 (Critical Path Definition)

**기본 능력만으로 도달 가능한 경로:**

```
중앙 성채 입구
  ↓ (기본 공격, 대시만으로 진행 가능)
마법 연구소 (또는 빙결 동굴) 진입
  ↓
보스 처치
  ↓
다음 구역으로 진행
  ↓
4개 Tier 1 구역 클리어
  ↓
묘지/카타콤 진입
  ↓
심연의 구 진입
  ↓
최종 보스 처치
```

**원칙:**
- Critical Path는 기본 능력(이단 점프, 벽 타기, 변신 없음)만으로 통과 가능해야 한다.
- 각 Critical Path 위의 Room은 Type 0 (막힌 방)으로 배치되지 않는다.
- 능력 게이트와 스탯 게이트는 Critical Path 위에 배치되지 않는다.
- 모든 시드에서 시작점에서 최종 보스까지 경로가 존재하는가를 검증한다 (System_World_ProcGen.md의 Phase 7 참조).

### 2-4. 능력 게이트 (Ability Gate) 배치 규칙

| 능력 | 획득 위치 | 새로 접근 가능한 구역/구간 | Gate Type |
| :--- | :--- | :--- | :--- |
| **이단 점프 (Double Jump)** | 마법 연구소 보스 처치 | 마법 연구소 상단 구간, 다른 구역 고도 높은 영역 | 구역 경계 Gate |
| **벽 타기 (Wall Climb)** | 빙결 동굴 보스 처치 | 빙결 동굴 좌측 절벽, 지하 수로 수직 통로 | 구역 경계 Gate |
| **안개 변신 (Fog Form)** | 묘지/카타콤 중반 보상 | 1타일 너비 틈새 진입 | 구역 내부 Gate |
| **수중 호흡 (Water Breathing)** | 지하 수로 중반 이노센트 | 수중 통로 진입 | 구역 내부 Gate |
| **역중력 (Gravity Invert)** | 천공의 탑 보스 처치 | 천공의 탑 상단부, 심연의 구 역중력 구간 | 구역 경계 Gate |
| **심연 저항 (Abyss Resistance)** | 최종 조건 | 심연의 구 심층 진입 | 최종 Gate |

**배치 원칙:**
- 능력 게이트는 **구역 경계에만** 배치된다. (구역 간 이동 차단)
- 구역 내부의 비경로 탐험 구간에서 보상으로 획득하는 능력도 있다.
- 같은 구역 내 두 번째 능력은 배치하지 않는다. (혼란 방지)
- 능력 게이트 위치는 모든 시드에서 동일하다. (매크로 구조 고정)

### 2-5. 스탯 게이트 (Stat Gate) 배치 규칙

| 스탯 | 임계값 구간 | 배치 위치 | 목적 |
| :--- | :--- | :--- | :--- |
| **STR (공격력)** | 10 ~ 20 ~ 30 | 각 구역 내부 고급 보상 경로 | 높은 방어도의 적/물체 파괴 |
| **INT (마력)** | 10 ~ 20 ~ 30 | 각 구역 내부, 특히 마법 연구소 | 마법 저항 적 처치, 마법 함정 해제 |
| **DEX (이동속도/명중도)** | 10 ~ 20 ~ 30 | 타이밍 기반 함정, 빠른 적 |  빠른 이동 필요 구간, 정밀 점프 |
| **VIT (방어력)** | 5 ~ 15 ~ 25 | 고난이도 적 밀집 구간 | 높은 피해 환경 생존 |
| **SPD (시간 기반)** | 5 ~ 10 ~ 15 | 시간 제한 구간, 천공의 탑 추락 구간 | 타이밍 기반 플랫포밍, 시간 제한 챌린지 |
| **LCK (아이템 발견)** | 10 ~ 20 ~ 30 | 숨겨진 경로 보상 포인트 | 숨겨진 아이템/이노센트 발견 |

**배치 원칙:**
- 스탯 게이트는 **구역 내부 비경로 탐험 구간에 배치**된다.
- Critical Path 위에 스탯 게이트를 배치하지 않는다. (진행 보장)
- 각 구역마다 3~5개의 스탯 게이트를 배치하여 선택적 도전을 제공한다.
- 스탯 게이트 임계값은 구역 난이도에 비례한다.

## 3. 수치 설계 (Numerical Design)

### 3-1. 구역 크기 정의 (Zone Size Definition)

| 구역 | Room Grid 크기 | 평균 Room 수 | 예상 탐험 시간 | 난이도 |
| :--- | :--- | :--- | :--- | :--- |
| **중앙 성채** | 4x4 | 12-16 | 10-15분 | 1 |
| **마법 연구소** | 4x3 | 10-14 | 20-25분 | 3 |
| **빙결 동굴** | 3x4 | 10-14 | 20-25분 | 3 |
| **천공의 탑** | 3x5 | 12-16 | 25-30분 | 4 |
| **지하 수로** | 5x3 | 12-16 | 20-25분 | 2 |
| **묘지/카타콤** | 4x4 | 12-16 | 25-30분 | 2-3 |
| **심연의 구** | 5x5 | 20-25 | 40-50분 | 5 |

**산출 근거:**
- 1 Room = 60x34 타일 (960x544 픽셀) = 평균 2-3분 탐험 시간.
- 구역 전체 탐험 시간 = Room 수 × 2-3분 + 보스 전투 시간(5-10분).
- 전체 월드 1회 클리어: 약 180-240분 (3-4시간).

### 3-2. 난이도 곡선 (Difficulty Curve)

```
난이도
  5 │                                        [심연]
    │
  4 │                    [천공]          ╱╲
    │                ╱╲         ╱─────────
  3 │          ╱────╲  ╱────╲╱
    │      ╱──────  ╱─────────
  2 │  ╱──[중앙]─────[지하][묘지]
    │╱
  1 │
    └─────────────────────────────────
    구역 진행
```

**권장 진행 순서 (가이드라인, 강제 아님):**
1. 중앙 성채 (난이도 1)
2. 지하 수로 (난이도 2)
3. 마법 연구소 또는 빙결 동굴 (난이도 3)
4. 나머지 난이도 3 구역
5. 천공의 탑 (난이도 4)
6. 묘지/카타콤 (난이도 2-3, 하지만 후반 도전 로케이션)
7. 심연의 구 (난이도 5)

**난이도 정의:**

| 난이도 | 적 밀도 | 적 체력 | 적 DPS | 보스 체력 | 스탯 게이트 임계값 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **1** | 낮음 (30%) | 낮음 | 낮음 | 낮음 | 임계값 없음 |
| **2** | 중간 (40%) | 중간 | 중간 | 중간 | STR/VIT 5~10 |
| **3** | 높음 (50%) | 높음 | 높음 | 높음 | STR/VIT 10~15 |
| **4** | 매우 높음 (60%) | 매우 높음 | 매우 높음 | 매우 높음 | STR/VIT 15~20 |
| **5** | 극도로 높음 (70%) | 극도로 높음 | 극도로 높음 | 극도로 높음 | STR/VIT 20~30 |

### 3-3. 스탯 게이트 임계값 (Stat Gate Thresholds)

| 스탯 | 구역 1-2 | 구역 3-4 | 구역 5-6 | 심연 |
| :--- | :--- | :--- | :--- | :--- |
| **STR** | 없음 ~ 5 | 10 | 15 | 20-25 |
| **INT** | 없음 ~ 3 | 8 | 15 | 20-25 |
| **DEX** | 없음 ~ 5 | 10 | 15 | 20 |
| **VIT** | 없음 ~ 3 | 8 | 12 | 18-25 |
| **SPD** | 없음 ~ 3 | 8 | 12 | 15 |
| **LCK** | 없음 ~ 5 | 10 | 15 | 20 |

**근거:**
- 각 스탯 임계값은 기본 수치(0) + 구역별 필수 강화 폭(5~30)으로 설정.
- 플레이어가 아이템계에서 얻을 수 있는 평균 장비 스탯 강화: 구역당 5~10.
- 따라서 플레이어는 각 구역 진입 전 1~2회 아이템계 진입 필요.

### 3-4. 보상 분포 (Reward Distribution)

| 보상 유형 | 중앙 | Tier 1 (각각) | 묘지 | 심연 |
| :--- | :--- | :--- | :--- | :--- |
| **장비 아이템** | 1 | 8-10 | 12-15 | 20-25 |
| **이노센트** | 0 | 2-3 | 5-8 | 10-15 |
| **소비 아이템** | 5 | 10-15 | 20-30 | 50+ |
| **희귀 아이템** | 0 | 1-2 | 3-5 | 5-10 |

**배치 원칙:**
- 보상은 난이도에 비례하여 증가.
- 비경로 탐험 보상 > Critical Path 보상.
- 보스 처치 보상이 가장 귀중 (Legendary 등급 가능).

## 5. 연동 (Integration)

### 5-1. 절차적 생성과의 연동 (System_World_ProcGen.md와의 관계)

| 측면 | 본 문서 (WMS) | 절차적 생성 문서 (WPG) | 역할 분담 |
| :--- | :--- | :--- | :--- |
| **구역 배치** | 구역 토폴로지 정의 (고정) | 구역 내부 Room Grid 생성 (절차적) | 토폴로지는 고정, 내부 구조는 변화 |
| **연결 포인트** | Portal 위치 정의 (고정) | Portal 정확한 좌표 생성 | 구역 경계점은 매크로 레벨에서 고정 |
| **게이트 배치** | 게이트 위치 원칙 정의 | 게이트 정확한 위치 검증 (Phase 7) | 본 문서가 규칙, WPG가 구현 |
| **바이옴 설정** | 구역별 테마 정의 | 바이옴별 Chunk 풀 선택 | 본 문서가 설계, WPG가 선택 |

### 5-2. 아이템계와의 연동 (Item World와의 순환)

**순환 메커니즘:**

```
[월드 구역 탐험]
     ↓
[장비 아이템 획득 (Normal ~ Legendary)]
     ↓
[아이템 선택 → 아이템계 진입]
     ↓
[아이템계 내 Strata 진행]
     ↓
[이노센트 정련 및 스탯 강화]
     ↓
[아이템계 클리어 또는 탈출]
     ↓
[월드로 복귀, 강화된 장비 장착]
     ↓
[월드에서 스탯 게이트 통과 가능]
     ↓
[새로운 구역 접근 또는 비경로 탐험 가능]
```

### 5-3. 3-Space 기반 검증 (Design_Architecture_3Space.md와의 정렬)

| 3-Space 원칙 | WMS에서의 구현 |
| :--- | :--- |
| 월드는 솔로 플레이 전용 | 월드는 1인 전용 공간. 마이크로 절차적 구조로 멀티플레이 불필요 |
| 탐험의 고독감 보호 | 다른 플레이어 흔적 불가시화. 비동기 고스트 메시지만 표시 가능 |
| 능력 게이트 우선 | 월드에서만 능력 획득. 아이템계 미사용 가능 |
| 파밍 순환 유도 | 스탯 게이트로 아이템계 진입 자연스럽게 유도 |

### 5-4. 메트로베니아 철학과의 정렬 (Design_Metroidvania_Philosophy.md와의 정렬)

| 메트로베니아 원칙 | WMS에서의 구현 |
| :--- | :--- |
| 비선형 탐험 | 4개 Tier 1 구역 진행 순서 자유. Critical Path로 Always Winnable 보장 |
| 능력 기반 게이팅 | 6개 능력 게이트로 세계 확장. 능력당 새 구역 또는 구간 개방 |
| 미지감과 발견 | 절차적 마이크로로 재방문 시 구조 변화. 숨겨진 보상 배치 |
| 백트래킹 가치 | 새 능력 획득 후 구 구역 재방문 시 새 경로 개방 |

### 5-5. 스탯 게이트 시스템과의 정렬

**이중 게이트의 상호작용:**

```
[능력 게이트]                [스탯 게이트]
    ↓                            ↓
월드에서만 획득          월드 + 아이템계 상호작용
    ↓                            ↓
새 구역/구간 개방          기존 구역 심화 탐험
    ↓                            ↓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    모두 함께 작동하여 순환 구조 형성
```

**양쪽 다 필요한 이유:**
- 능력만 있으면: 기동성 있지만 약함. 높은 방어도 적 처치 불가.
- 스탯만 있으면: 강하지만 갈 수 없는 곳이 많음. 탐험 동기 약함.
- 양쪽 모두: 강화되면서 동시에 세계가 확장. 성장의 전방위적 쾌감.

## 6. 검증 체크리스트 (Acceptance Criteria)

### 기능적 검증 (Functional Criteria)

- [ ] 모든 시드에서 중앙 성채 → 심연의 구까지 기본 능력만으로 도달 가능한가?
- [ ] 4개 Tier 1 구역을 모두 탐험하지 않아도 게임을 클리어할 수 있는가?
- [ ] 능력 게이트 배치가 각 능력의 획득 위치와 일치하는가?
- [ ] 스탯 게이트 임계값이 구역 난이도에 비례하는가?
- [ ] 모든 구역 간 포탈이 쌍방향으로 작동하는가?
- [ ] 절차적 생성 시 구역별 바이옴이 일관되게 유지되는가?
- [ ] 보스 배치가 Critical Path와 비경로를 구분하는가?

### 체감적 검증 (Experiential Criteria)

- [ ] 플레이어가 "저 벽을 부수려면 강해져야 한다"는 인식을 자연스럽게 느끼는가?
- [ ] 새 능력 획득 후 구 구역을 재방문할 때 "세계가 다시 태어난 느낌"이 드는가?
- [ ] 각 구역이 고유한 테마와 분위기를 유지하는가?
- [ ] 플레이어가 "다음은 어디로 갈까?"라고 자유롭게 선택할 수 있는가?
- [ ] 스탯 게이트 앞에서 막힐 때 게임을 포기하고 싶다는 생각이 드는가, 아니면 "아, 아이템계에 가야겠구나"라는 자연스러운 유도를 느끼는가?

### 난이도 곡선 검증

- [ ] 구역 진행에 따라 난이도가 부드럽게 증가하는가?
- [ ] 특정 구역에서 "벽"을 느껴 아이템계 진입을 하도록 유도되는가?
- [ ] 최종 보스가 적절한 도전 수준을 제시하는가?

### 콘텐츠 검증

- [ ] 숨겨진 보상이 충분한가? (각 구역당 3~5개)
- [ ] 보스 전투가 각각 고유한 패턴을 갖는가?
- [ ] 비경로 탐험의 보상이 경로 탐험의 보상과 차별화되는가?
