## 1. 시스템 개요

### 1.1. 설계 의도

> "모든 아이템의 기억 속에 등장하는 존재들은 악의가 아니라 맥락에서 비롯된다."

아이템 서사 문서마다 완전히 새로운 몬스터를 설계하면 콘텐츠 제작 비용이 폭발적으로 증가한다. 이 풀 시스템은 **테마 코드(T-코드)**를 기반으로 몬스터를 모듈화하여, 아이템 서사 작성자가 테마 조합만으로 아이템계의 몬스터 배치를 완성할 수 있도록 한다.

각 몬스터는 독립 전투 유닛이기 이전에 **서사 장치**다. 몬스터의 행동과 동기는 아이템의 기원(Origin)을 반영한다.

### 1.2. 테마 코드 체계

| 테마 코드 | 테마명 | 서사 기반 |
| :--- | :--- | :--- |

| T-FOREST | 숲 / 자연 | 아이템이 자연 환경 속에 있었던 기억 |
| T-SCHOLAR | 학자 / 서재 | 아이템이 지식 또는 연구의 도구였던 기억 |
| T-TRADE | 상인 / 시장 | 아이템이 거래되거나 소유가 이전된 기억 |

### 1.3. 지층별 테마 배정 규칙

| 지층 3 | 1~2개 | 아이템의 가장 오래된 기억 또는 핵심 선택 | T-PHANTOM 반드시 포함 권장 |

## 2. 테마별 몬스터 정의

몬스터 강도 기준:
- **약(弱):** 체력 낮음, 데미지 낮음, 도주/회피 패턴 위주. 첫 조우 시 죽지 않는다.
- **중(中):** 체력 보통, 데미지 보통, 명확한 공격 패턴 1~2가지.
- **강(强):** 지층 3 또는 레어리티 상위에서만 등장. 체력 높고 패턴 복잡.

### T-PHANTOM: 환영 / 선택 (지층 3 전용)

> 서사 의도: "선택하지 않은 삶"의 잔영. 이 테마는 지층 3에서만 등장하며, 아이템의 전 소유자가 선택한 길과 선택하지 않은 길이 공간으로 구현될 때 나타난다. 적대적 존재가 아니라 서사 장치다.

- 이 몬스터들은 처치하기 위해 존재하는 것이 아니라 서사를 완성하기 위해 존재한다.

- 처치 연출은 반드시 슬프거나 쓸쓸한 파티클 효과를 사용한다 (흰 연기, 꽃잎, 빛 입자 등).
- 처치 후 텍스트는 아이템 서사 문서의 "Fire" 요소와 연결된다.

## 6. 문서 활용 가이드

아이템 서사 문서 작성 시 이 풀을 활용하는 방법:

```
1. 아이템의 기원(Origin) 4요소를 정의한다.
2. 각 지층에 해당하는 테마 코드를 1~2개 선정한다.
3. 선정된 테마에서 4종의 몬스터를 선택한다 (약×2, 중×2 기본 구성).
4. 지층 3에는 T-PHANTOM 2종을 추가한다.
5. 몬스터의 "행동 서사"를 아이템의 기원과 연결하여 1줄 맥락을 추가한다.
6. 아이템 서사 문서에 테마 코드와 선택한 몬스터 ID를 명시한다.
```

**예시 — 할아버지의 부엌칼 (CNT-ITM-001):**
- 지층 1: T-HOME — 도망치는 재료(=도망치는 양파), 성난 주방도구(=성난 생선), 날뛰는 가축(=날뛰는 닭), 화난 양념 다발(=화난 마늘 다발)
- 지층 2: T-MILITARY — 기어오르는 침입자, 야간 침입자의 잔영, 불량배 무리
- 지층 3: T-PHANTOM — 젊은 자신의 환영, 성장한 자신의 환영 + T-MILITARY × T-HOME 혼합 1종
