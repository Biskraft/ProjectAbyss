## 1. 개요 (Concept)

### 1.1. 문제 정의 — 왜 템플릿 시스템이 필요한가?

`Content_Item_Narrative_GrandfatherKitchenKnife.md`는 Project Abyss 아이템 서사의 기준점이자 품질 목표다. 그러나 그 포맷을 그대로 100개 아이템에 적용하면 다음 문제가 발생한다.

| 문제 | 수치 | 영향 |
| :--- | :--- | :--- |
| 문서당 평균 라인 수 | ~500줄 | 100개 = 50,000줄 문서 관리 |
| 몬스터 명세 (고유) | 아이템당 10종 | 100개 = 1,000종 고유 몬스터 — 아트/기획 공수 불가 |
| 보스 명세 (고유) | 아이템당 3종 | 100개 = 300종 고유 보스 |
| 환경 오브젝트 (고유) | 아이템당 15+ | 100개 = 1,500+ 환경 오브젝트 |
| 40초 이벤트 | 아이템당 15개 | 100개 = 1,500개 고유 이벤트 명세 |

이 규모는 인디 팀 기준 현실적으로 불가능하다. 동시에 서사를 포기하거나 단순화하면 야리코미 동기("이 아이템의 이야기가 뭘까?")가 소멸한다.

**핵심 긴장:** 100개 아이템에 걸친 양(Scale)과 아이템당 서사 깊이(Depth)는 충돌한다.

### 1.2. 해결 전략 — 고유 콘텐츠 + 모듈 콘텐츠 분리

모든 콘텐츠를 두 가지로 분류한다.

**고유 콘텐츠 (Unique Content):** 아이템의 정체성과 감정적 울림을 만드는 것. 수작업 필수. 이것이 전체 품질의 90%를 결정한다.

**모듈 콘텐츠 (Modular Content):** 아이템의 테마와 레어리티에서 자동으로 유추되는 것. 공유 풀(Pool)에서 참조. 아이템마다 명세하지 않는다.

이 분리를 통해 **작성 공수의 80%를 절감하면서 서사 품질의 90%를 유지**하는 것이 목표다.

### 1.3. 목표

| 지표 | 기존 방식 | 목표 |
| :--- | :--- | :--- |
| 아이템당 문서 라인 수 | ~500줄 | ~150줄 (70% 절감) |
| 아이템당 작성 시간 | 기준 100% | 20% (80% 단축) |
| 서사 품질 | 기준 100% | 90% 이상 유지 |
| 병렬 생성 가능 여부 | 불가 (순차 의존) | 가능 (10개를 5개 에이전트 병렬) |
| 내러티브 일관성 | 수동 검증 | 체크리스트 자동화 |

## 2. 콘텐츠 분류: 고유 vs 모듈

### 2.1. 고유 콘텐츠 (per 아이템, 수작업 필수)

아래 항목들은 테마나 레어리티로 유추할 수 없다. 이것이 아이템의 "영혼"이다. 전량 수작업으로 작성한다.

| 항목 | 내용 | 예시 (할아버지의 부엌칼) |
| :--- | :--- | :--- |
| **아이템 프로필** | 이름, 기원 4요소(Creator/Purpose/History/Fate), 플레이버 텍스트 | "칼날에 기름때가 깊이 밴 칼. '충성'이라는 글자가 희미하게 새겨져 있다." |
| **서사 곡선** | 3막 구조 (감정 기조 + 핵심 질문), Fire & Ember 구조 | 지층1 따뜻함→지층2 의무→지층3 선택 |
| **정보 공개 타임라인** | 지층 진입 시점별 공개 정보/미공개 정보 | 지층1: 기름때·'충성' 글자 → 지층2: 경비병 신분 드러남 |
| **NPC 유령 대사** | 상태별 조건 대사 (첫 진입 / 재방문 / 지층별) | "이 복도를 수천 번은 걸었지. 부엌으로 가는 길이 더 길었어." |
| **이노센트 고유 대사** | 야생 상태 / 복종 상태 각 3줄 이상 | 맛의 정령: "이 칼에 내 레시피를 건드리지 마. 아직 안 됐어." |
| **Fire 모멘트** | 지층 3 클라이맥스 한 마디. 이 아이템 전체의 감정적 정점 | "사람을 베는 칼이 아니라 먹이는 칼을 쥐고 싶었다. 그게 전부야." |
| **보스 서사 오버라이드** | 지층별 보스 고유 처치 연출 텍스트, 드랍 오브젝트 이름 | 지층2 보스 드랍: "은퇴 축하 메달" |
| **환경 핵심 오브젝트** | 지층별 핵심 오브젝트 최대 3개 (나머지는 풀에서 자동 로드) | 지층3: 검과 방패(왼벽), 부엌칼(오른벽), 석양 창문 |

### 2.2. 모듈 콘텐츠 (공유 풀에서 참조)

아래 항목들은 테마 ID와 레어리티에서 자동으로 유추된다. 아이템 정의서에 명세하지 않고 공유 문서를 참조한다.

| 항목 | 참조 문서 | 결정 방식 |
| :--- | :--- | :--- |
| **몬스터 풀** | `System_ItemNarrative_MonsterPool.md` | 테마 ID별 기본 세트 + 레어리티별 등장 조건 |
| **보스 기본 템플릿** | `System_ItemNarrative_MonsterPool.md` (보스 섹션) | 테마 ID별 기본 보스 + 서사 오버라이드 3줄로 커스텀 |
| **환경 팔레트** | `System_ItemNarrative_EnvironmentPool.md` | 테마 ID별 배경/조명/지형 타입 자동 배정 |
| **40초 이벤트 풀** | `System_ItemNarrative_EnvironmentPool.md` (이벤트 섹션) | 테마별 20개 풀에서 지층당 5개 랜덤 선택 |
| **이노센트 기본 타입** | `System_Innocent_Core.md` §3 | 6개 기본 타입 중 선택. 이름/대사만 고유 |
| **배경 음악 태그** | `System_ItemNarrative_EnvironmentPool.md` (사운드 섹션) | 테마별 BGM 태그 자동 배정 |

> **주의:** 모듈 문서 3개(`MonsterPool`, `EnvironmentPool`, `InnocentTypes`)는 본 시스템 문서가 완성된 이후 별도 작성한다. 현재는 테마 정의와 인터페이스 명세만 확립한다.

## 3. 테마 시스템 (Theme System)

### 3.1. 테마 정의 — 10개 기본 테마

테마는 아이템계의 시각/음향/적 구성을 결정하는 핵심 모듈 단위다. 아이템의 기원(Origin)에서 자연스럽게 유추된다.

| 테마 ID | 이름 | 대표 환경 | 몬스터 세트 (키워드) | 보스 템플릿 | 적합 아이템 카테고리 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **T-HOME** | 가정/주방 | 부엌, 식당, 마당, 지하 창고 | 식재료 정령, 조리도구 반란군, 해충 | 식탐의 왕 | 부엌칼, 포크, 냄비, 바구니 |
| **T-MILITARY** | 군사/경비 | 성채 복도, 초소, 성벽, 감옥 | 유령 병사, 순찰 잔영, 방어 기계 | 잊힌 대장 | 검, 방패, 갑옷, 투구 |
| **T-FORGE** | 대장장이/공방 | 대장간, 용광로, 광산 갱도 | 금속 골렘, 불꽃 정령, 광석 수호자 | 부서진 거인 | 대검, 해머, 갑옷, 팔찌 |
| **T-FOREST** | 숲/자연 | 고목 뿌리, 개울, 동굴 입구, 수풀 | 야생 동물 잔영, 식물 수호자, 독충 | 숲의 수호자 | 활, 지팡이, 단검, 망토 |
| **T-SCHOLAR** | 학자/서재 | 도서관, 실험실, 관측탑, 문서 보관실 | 지식 수호 골렘, 잉크 정령, 책 파수꾼 | 지식의 잔영 | 지팡이, 책, 목걸이, 반지 |
| **T-TRADE** | 상인/시장 | 시장 골목, 창고, 항구 부두, 마차 | 도적 잔영, 상품 수호자, 계약 파괴자 | 파산한 상인왕 | 단검, 반지, 부적, 주머니 |
| **T-NOBLE** | 귀족/궁정 | 무도회장, 정원 미로, 왕궁 복도, 탑 | 허수아비 시종, 가면 무도인, 왕관 수호자 | 타락한 왕관 | 레이피어, 왕관, 망토, 장갑 |
| **T-UNDEAD** | 죽음/묘지 | 묘지, 카타콤, 납골당, 무너진 교회 | 해골 병사, 망자 방랑자, 봉인 수호자 | 잊힌 이름 | 낫, 해골 장식품, 랜턴, 두건 |
| **T-WAR** | 전쟁/전장 | 전장 폐허, 요새 잔해, 참호, 포대 | 전투 잔영, 기계 병기 파편, 징집 유령 | 전쟁의 기억 | 대검, 검, 방패, 활 |
| **T-SEA** | 바다/항해 | 갑판, 등대, 해저 유적, 어선 선창 | 해양 생물 변이체, 선원 유령, 심해 수호자 | 심해의 군주 | 작살, 닻 장식품, 나침반, 망원경 |

### 3.2. 테마 조합 규칙

아이템의 각 지층은 아이템의 역사적 깊이를 반영한다. 가장 최근 기억이 표면(지층 1)에, 가장 오래된 기억이 심층(지층 3+)에 위치한다.

| 지층 | 테마 배정 원칙 | 역할 |
| :--- | :--- | :--- |
| **지층 1** | 주요 테마 (아이템의 현재 정체성) | 플레이어가 처음 경험하는 아이템의 "얼굴" |
| **지층 2** | 보조 테마 (아이템의 이전 역할) | 아이템에 숨겨진 다른 면을 드러냄 |
| **지층 3** | 주요 테마 + 보조 테마 혼합 (Crossroads) | 두 정체성이 충돌하고 화해하는 공간 |
| **지층 4** (Legendary+) | 기원 테마 (아이템이 만들어진 당시) | 창조의 순간, 서사의 근원 |
| **심연** (Ancient) | 고유 테마 (세계관 핵심과 연결) | Ancient 아이템만 존재. 월드 서사와 직결 |

**예시 (할아버지의 부엌칼 — Magic, 3지층):**
- 지층 1: `T-HOME` (은퇴 후 요리사 인생)
- 지층 2: `T-MILITARY` (경비병 시절)
- 지층 3: `T-HOME + T-MILITARY` Crossroads (검을 내려놓고 칼을 잡은 선택의 순간)

**조합 불가 원칙:** 동일 테마를 지층 1과 지층 2에 중복 배정할 수 없다. 단, 지층 3 Crossroads에서 지층 1 테마 재등장은 허용한다 (감정적 귀환을 의도).

## 4. 아이템 서사 정의서 경량화 포맷

### 4.1. 경량화 템플릿 (목표: ~150줄)

아래가 아이템 하나를 정의하는 표준 포맷이다. 복사해서 사용하고, `[ ]` 안의 내용을 채운다.

```yaml
# ============================================================
# 아이템 서사 정의서 (Item Narrative Definition)
# 포맷 버전: SYS-INS-01 v1.0
# ============================================================

item_id: "[CNT-ITM-xxx]"
name: "[아이템 이름]"
rarity: "[Normal / Magic / Rare / Legendary / Ancient]"
category: "[Sword / Greatsword / Dagger / Bow / Staff / Shield / Armor / Accessory 등]"
narrative_archetype: "[SA-01 ~ SA-10 중 선택]"
canon_level: "[Established / Provisional / Under Review]"

# -----------------------------------------------------------
# 1. 기원 (Origin) — 수작업 필수
# -----------------------------------------------------------
origin:
  creator: "[누가, 어떤 목적으로 만들었는가]"
  purpose: "[어떤 용도로 사용되었는가 — 구체적 일상/사건 포함]"
  history: "[어느 시대(Era), 어떤 인물의 삶과 함께했는가]"
  fate: "[왜 지금 플레이어 손에 들어왔는가]"

flavor_text: >
  "[아이템 설명창 텍스트. 1~2문장. 설명하지 않고 보여준다.
   고유 콘텐츠의 이중성/모순/흔적을 암시.]"

# -----------------------------------------------------------
# 2. 테마 배정 (Theme Assignment) — 기원에서 유추
# -----------------------------------------------------------
themes:
  stratum_1:
    theme: "[T-HOME / T-MILITARY / T-FORGE 등]"
    variant: "[테마 내 세부 환경 키워드 — 예: kitchen, patrol_corridor]"
  stratum_2:
    theme: "[보조 테마 ID]"
    variant: "[세부 환경 키워드]"
  stratum_3:
    theme: "[주요 테마 ID + 보조 테마 ID — Crossroads]"
    variant: "crossroads"
  # stratum_4, abyss: Legendary / Ancient 아이템만 추가

# -----------------------------------------------------------
# 3. 서사 곡선 (Narrative Arc) — 수작업 필수
# -----------------------------------------------------------
narrative_arc:
  act1:
    stratum: 1
    emotion: "[감정 기조]"
    question: "[이 지층이 플레이어에게 던지는 핵심 질문]"
  act2:
    stratum: 2
    emotion: "[감정 기조]"
    question: "[이 지층이 플레이어에게 던지는 핵심 질문]"
  act3:
    stratum: 3
    emotion: "[감정 기조 — 앞 두 감정의 수렴 또는 충돌]"
    question: "[이 지층이 플레이어에게 던지는 핵심 질문]"
```

## 5. 생성 파이프라인 (Creation Pipeline)

### 5.1. 아이템 서사 7단계 워크플로우

단계별로 수작업(M)과 자동/참조(A)를 명확히 분리한다.

| 단계 | 작업 | 담당 | 소요 비중 |
| :--- | :--- | :--- | :--- |
| **1단계** | 서사 아키타입 선택 (SA-01~SA-10 중 1개) | M — world-builder | 5% |
| **2단계** | 기원 4요소 작성 (Creator / Purpose / History / Fate) + 플레이버 텍스트 | M — world-builder + writer | 30% |
| **3단계** | 테마 배정 (기원에서 유추, 지층별 주/보조/혼합) | M — world-builder | 5% |
| **4단계** | 서사 곡선 작성 (3막 감정 기조 + 핵심 질문 + Fire 모멘트) | M — world-builder + writer | 25% |
| **5단계** | NPC 유령 대사 작성 (상태 × 지층 조합) | M — writer | 20% |
| **6단계** | 이노센트 타입 선택 + 이름/대사 작성 | M — writer (타입은 A 참조) | 10% |
| **7단계** | 보스 오버라이드 + 환경 핵심 오브젝트 + 검증 체크리스트 실행 | M — world-builder (템플릿은 A 참조) | 5% |

**총 수작업 비중:** 전체의 약 20%가 창작적 의사결정. 나머지 80%는 템플릿 채우기와 모듈 참조.

### 5.2. 에이전트 병렬 생성 가능 범위

world-builder와 writer 에이전트가 공동 작업할 때의 병렬화 가능 구간을 정의한다.

| 구간 | 병렬 가능 여부 | 이유 |
| :--- | :--- | :--- |
| 1~3단계 (기원 + 테마) | **선행 필수** — 순차 실행 | 4~7단계의 입력값이 됨 |
| 4단계 (서사 곡선) | 1~3단계 완료 후 즉시 병렬 가능 | 다른 아이템과 독립 |
| 5~6단계 (대사 + 이노센트) | 4단계와 동시 병렬 가능 | 감정 기조가 확정되면 즉시 작성 가능 |
| 7단계 (검증) | 5~6단계 완료 후 | 전체 완성 후 실행 |

**10개 동시 생산 시나리오:** 5개 에이전트 쌍(world-builder + writer) × 각 2개 아이템 = 10개 병렬 생산. 테마 시스템과 모듈 풀이 확립된 이후 가능.

## 6. 서사 아키타입 카탈로그 (Narrative Archetype Catalogue)

아키타입은 아이템 서사의 감정 여정 패턴이다. 10개 중 1개를 선택하면 3막 구조의 감정 기조와 Fire 모멘트의 방향이 자동으로 설정된다.

| ID | 아키타입명 | 감정 여정 | 권장 레어리티 | Fire 모멘트 방향 | 예시 아이템 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **SA-01** | 기억 탐구 | 따뜻함 → 낯섦 → 이해 | Magic ~ Rare | 과거의 선택에 대한 조용한 긍정 | 할아버지의 부엌칼 |
| **SA-02** | 비극적 상실 | 평온 → 균열 → 수용 | Rare ~ Legendary | 돌아올 수 없음을 인정하는 한 마디 | 전사한 기사의 검 |
| **SA-03** | 복수와 해소 | 분노 → 행동 → 해방 (또는 공허) | Rare ~ Legendary | 복수가 완성되었을 때의 감정 — 기쁨이 아닌 것 | 배신자에게 바친 단검 |
| **SA-04** | 일상의 발견 | 평온 → 관찰 → 작은 놀라움 | Normal ~ Magic | 평범한 삶에서 발견한 아름다움의 조각 | 농부의 낫, 행상인의 지팡이 |
| **SA-05** | 금지된 지식 | 호기심 → 이해 → 공포 (또는 황홀) | Legendary ~ Ancient | 알았다면 좋았을까, 몰랐다면 좋았을까 | 금서 마도사의 지팡이 |
| **SA-06** | 장인의 자부심 | 집착 → 완성 → 의문 | Magic ~ Rare | 완벽한 것을 만든 장인이 느끼는 알 수 없는 허전함 | 대장장이의 마지막 망치 |
| **SA-07** | 약속과 배신 | 믿음 → 절망 → 용서 (또는 거부) | Rare ~ Legendary | 배신한 자와 배신당한 자, 둘 다 알고 있던 것 | 기사단의 서약 검 |
| **SA-08** | 자연의 순환 | 경외 → 일체감 → 평화 | Normal ~ Magic | 인간의 삶도 자연의 한 부분이었다는 깨달음 | 사냥꾼의 오래된 활 |
| **SA-09** | 권력과 타락 | 야망 → 절정 → 파멸 | Legendary ~ Ancient | 원했던 것을 얻은 순간이 끝의 시작이었음 | 왕의 홀, 찬탈자의 왕관 |
| **SA-10** | 희생과 구원 | 고통 → 결단 → 빛 | Ancient | 대가를 알면서도 선택한 자의 마지막 말 | 성녀의 방패, 영웅의 유물 |

**아키타입 선택 가이드:**
- 아이템 레어리티가 높을수록 감정의 진폭이 크고 복잡한 아키타입을 권장한다.
- Normal ~ Magic은 SA-01, SA-04, SA-06, SA-08처럼 일상적이고 따뜻한 것이 기본 톤이다.
- Legendary ~ Ancient는 SA-02, SA-05, SA-07, SA-09, SA-10처럼 역사적 무게를 가진 것이 적합하다.
- 하나의 아키타입이 한 게임 내에서 과도하게 반복되면 서사 피로가 온다. 각 아키타입별 사용 빈도를 추적한다.
