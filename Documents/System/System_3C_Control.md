## 1. 개요 (Concept)

### 1.1. 설계 철학 (Design Philosophy)

Project Abyss의 조작 체계는 다음 한 문장으로 정의한다:

> "한 손가락으로 배우고, 스킬 조합으로 깊어지는 전투"

웹 브라우저에서 구동되는 횡스크롤 메트로베니아 + 아이템계 온라인 액션 RPG로서, PC 키보드와 게임패드에서 직관적이고 접근성 높은 전투 경험을 제공하는 것이 최우선 설계 목표이다.

### 1.2. 방향 전환 — 정밀 액션에서 스킬 슬롯 기반으로 (Direction Shift)

#### 기존 방향 (폐기)

* 레퍼런스: 월하의 야상곡, 할로우 나이트, 데드셀
* 핵심 메커닉: 백스텝 i-frame, 가변 점프, 동시 입력 조합
* 문제: 정밀 타이밍 입력과 방향키 + 버튼 동시 입력 조합(백스텝=아래+회피, 슬라이딩=아래+점프)은 조작 복잡도가 높아 접근성이 낮음.

#### 변경 방향 (채택)

* 레퍼런스: 메이플스토리(스킬 슬롯 기반 전투), 던전파이터(스킬 키 기반 횡스크롤)
* 핵심 메커닉: 스킬 슬롯 4개 + 원터치 발동 + 자동 조준 + 쿨다운 관리
* 이점: 모든 행동이 단일 버튼 입력으로 완결. 방향키 조합 불필요. 접근성이 높으면서도 깊이 있는 전투.

### 1.3. 설계 근거 (Reasoning)

| 레퍼런스                  | 채택 요소                                          | 미채택 요소                    |
| :------------------------ | :------------------------------------------------- | :----------------------------- |
| 메이플스토리              | 스킬 슬롯 장착식, 허브에서 스킬 세팅 변경            | 키보드 전용 다수 단축키         |
| 브롤스타즈                | 가상 조이스틱 + 버튼의 직관적 레이아웃               | 듀얼 스틱 조준                  |
| 아키에이지 워             | ARPG 스킬 버튼 배치 참고                             | 자동 전투                       |
| 던전파이터                | 스킬 키 기반 횡스크롤 전투, 쿨다운 관리              | 커맨드 입력 (방향키 조합 스킬)  |

### 1.4. 3대 기둥 정렬 (Pillar Alignment)

| 기둥                           | 조작 체계와의 관계                                                  |
| :----------------------------- | :------------------------------------------------------------------ |
| 깊은 아이템계 (Deep Abyss)     | 스킬 조합이 층마다 달라져야 하므로, 허브에서 스킬 세팅 전략이 핵심    |
| 성장과 빌드 (Growth & Build)   | 스킬 해금/강화가 곧 조작 선택지 확장. 4슬롯 제한이 빌드 고민을 유발   |
| 아이템 경제 (Item Economy)     | 스킬 쿨다운과 자원 소모가 경제 순환에 직결                            |

### 1.5. 저주받은 문제 검증 (Cursed Problem Check)

* **조작이 쉬우면 전투가 단조로워지지 않는가?**: 스킬 4개의 조합 순서, 쿨다운 겹침 관리, 위치 선정(포지셔닝)이 전투 깊이를 제공한다. 개별 입력은 단순하지만, "언제 어떤 스킬을 쓸 것인가"라는 판단이 전투의 핵심이 된다.
* **4슬롯은 너무 적지 않은가?**: 던전파이터와 메이플스토리의 초기 레벨 경험을 참고하면, 기본 공격 + 4개 스킬이 횡스크롤 전투의 충분한 복잡도를 제공한다. 슬롯 부족은 "어떤 스킬을 가져갈 것인가"라는 전략적 선택으로 전환된다.
* **자동 조준이 스킬 표현을 단조롭게 만들지 않는가?**: 자동 조준은 타겟 선택만 자동화한다. 스킬 자체의 범위, 이펙트, 발동 타이밍, 이동 가능 여부는 스킬별로 차별화하여 체감 다양성을 확보한다.

### 1.6. 리스크와 보상 (Risk & Reward)

| 리스크                                  | 완화 방안                                                          |
| :-------------------------------------- | :----------------------------------------------------------------- |
| 조작 접근성 부족                        | 단일 버튼 원칙으로 모든 행동이 1버튼 완결. 키 리바인딩 지원                   |
| 자동 조준이 의도하지 않은 적을 타겟팅    | 사거리 + 부채꼴 각도(45도) 제한. 바라보는 방향 우선                   |
| 키보드와 게임패드 간 밸런스 격차        | 모든 행동이 단일 버튼. 양쪽 디바이스에서 동일한 조작 경험 제공        |
| 브라우저 키 이벤트 누락 (포커스 해제)    | `visibilitychange` + `blur` 이벤트 감지 후 전체 입력 초기화           |
| 네트워크 지연 시 스킬 발동 불일치        | 클라이언트 예측 + 서버 검증 이중 처리                                 |

### 2.4. 스킬 슬롯 시스템 (Skill Slot System)

4개의 액티브 스킬 슬롯을 통해 전투의 깊이를 확보한다.

* **슬롯 개수**: 4개 (SK1 ~ SK4)
* **장착 방식**: 허브(Hub)에서 보유 스킬 목록 중 4개를 선택하여 슬롯에 장착한다. 아이템계(Abyss) 및 월드(World) 진입 후에는 변경할 수 없다.
* **발동 방식**: 슬롯에 대응하는 버튼을 한 번 터치/클릭하면 즉시 발동한다. 방향키 조합, 차지, 길게 누르기 등의 추가 입력은 없다.
* **쿨다운**: 각 스킬은 개별 쿨다운을 가진다 (3초 ~ 15초, 스킬별 상이). 쿨다운 중에는 해당 버튼이 비활성 상태이며, 원형 게이지가 잔여 쿨다운을 표시한다.
* **자동 조준**: 스킬 발동 시, 사거리(`skill_auto_aim_range_tiles`) 이내이고 캐릭터가 바라보는 방향 기준 좌우 `skill_auto_aim_angle_deg`도 이내에 있는 적 중 가장 가까운 적을 자동으로 타겟팅한다. 사거리 내 적이 없으면 캐릭터가 바라보는 방향으로 발사한다.
* **스킬 사용 중 이동**: 스킬별로 3가지 중 하나로 설정한다.
  * `free`: 이동 가능 (즉발 스킬)
  * `slow`: 이동 속도 50% 감소 (시전 스킬)
  * `locked`: 이동 불가 (대형 스킬)

### 2.5. 기본 공격 (Basic Attack)

* 기본 공격 버튼을 연타하면 자동으로 3타 콤보가 연결된다.
* 타이밍 입력은 불필요하다. 버튼을 반복 입력하기만 하면 1타 -> 2타 -> 3타가 순차 실행된다.
* 각 타격 간 간격은 `basic_attack_interval_ms`(300ms)이다.
* 3타 이후에는 짧은 후딜(마지막 타격 간격의 2배)이 존재하며, 이후 다시 1타부터 시작한다.
* 사거리: `basic_attack_range_tiles`(2타일) 이내의 적에게 판정이 발생한다.

### 2.6. 점프 (Jump)

* 점프 버튼을 누르면 고정 높이(`jump_height_tiles` = 3타일)로 점프한다.
* 가변 높이 점프를 제거한다. 짧게 누르든 길게 누르든 동일한 높이로 점프한다.
* 이단 점프: 게임 진행 중 특정 능력을 해금하면 활성화된다. 공중에서 점프 버튼을 한 번 더 누르면 `double_jump_height_tiles`(2.5타일) 만큼 추가 상승한다.
* 하강 통과: 반투명 플랫폼 위에서 아래 방향 + 점프 동시 입력 시 플랫폼을 통과하여 하강한다. 이 조합은 유일한 방향키+버튼 조합이며, 전투 중 사용 빈도가 낮다.

### 2.7. 대시 (Dash) — 렐릭 해금 후 활성화

> **조건:** 대시는 기본 조작이 아니다. 게임 초반 월드에서 **대시 렐릭(Dash Relic)**을 획득한 이후에만 활성화된다. 렐릭 획득 전에는 KeyC / X(West) 버튼 입력이 무시된다.

* 대시 렐릭 획득 후, 렐릭 슬롯 버튼을 누르면 캐릭터가 바라보는 방향으로 고정 거리(`dash_distance` = 4타일)를 빠르게 이동한다.
* 기본 대시에 i-frame(무적)은 없다. 대시는 순수 이동 기동이다.
* 대시 후 `dash_cooldown_ms`(400ms) 쿨다운이 적용된다. 쿨다운 중에는 대시 버튼이 비활성이며, 잔여 쿨다운을 원형 게이지로 표시한다.
* 지상/공중 모두 사용 가능하다. 공중 대시는 착지 전 1회만 사용 가능하다.
* 3타 콤보 후딜(`combo_end_lag_ms`) 중 대시 입력 시 후딜을 즉시 캔슬한다. 이는 전투 리듬에서 "반격 기회"를 "회피 기회"로 전환하는 핵심 메커닉이다. (대시 렐릭 획득 후에만 가능)

## 3. 규칙 (Rules)

### 3.1. 입력 우선순위 (Input Priority)

동일 프레임에 복수 행동이 동시에 요청되면 다음 우선순위에 따라 하나만 실행한다:

```text
대시(Dash, 렐릭 해금 후) > 스킬(Skill) > 기본 공격(Basic Attack) > 점프(Jump) > 이동(Move)
```

* 우선순위가 높은 행동이 실행되면, 낮은 행동은 입력 버퍼에 저장된다.
* 이동은 다른 행동과 동시에 처리할 수 있다 (스킬의 이동 설정이 `free`인 경우).

### 3.2. 스킬 사용 중 이동 규칙 (Movement During Skill)

| 이동 설정   | 이동 속도    | 방향 전환 | 점프 |
| :---------- | :----------- | :-------: | :--: |
| free        | 100%         |     O     |   O  |
| slow        | 50%          |     O     |   X  |
| locked      | 0%           |     X     |   X  |

* `locked` 상태에서는 캐릭터가 스킬 시전 위치에 고정된다.
* 모든 설정에서 스킬 시전 중 다른 스킬 발동은 불가하다 (스킬 시전 시간이 종료되어야 다음 스킬 사용 가능).

### 3.3. 쿨다운 표시 규칙 (Cooldown Display)

* 각 스킬 버튼 위에 원형 게이지로 잔여 쿨다운을 표시한다.
* 쿨다운 진행 중: 버튼이 어둡게 표시되며, 시계 방향으로 원형 게이지가 차오른다.
* 쿨다운 완료: 버튼이 밝아지며 짧은 하이라이트 이펙트가 재생된다.

### 3.4. 자동 조준 규칙 (Auto-Aim Rules)

1. 캐릭터 위치를 기준으로 반경 `skill_auto_aim_range_tiles`(8타일) 이내의 적을 검색한다.
2. 바라보는 방향 기준 좌우 `skill_auto_aim_angle_deg`(45도) 이내에 있는 적만 후보로 선별한다.
3. 후보 중 가장 가까운 적을 타겟으로 선택한다.
4. 후보가 없으면 캐릭터가 바라보는 방향으로 스킬을 발사한다.
5. 기본 공격에는 자동 조준을 적용하지 않는다. 기본 공격은 항상 바라보는 방향 전방에 판정이 발생한다.

### 3.5. (삭제됨 — 가상 패드 커스터마이징은 모바일 미지원으로 삭제)

### 3.6. 화면 비율 및 레이아웃 (Screen Ratio & Layout)

* 기준 비율: 16:9.

### 3.7. 웹 브라우저 단축키 충돌 방지 (Browser Shortcut Conflict Prevention)

다음 키 조합은 게임 조작에 할당하지 않는다:

* `Ctrl + W` (탭 닫기)
* `Ctrl + R` / `F5` (새로고침)
* `Ctrl + T` (새 탭)
* `Alt + Tab` (창 전환)
* `Alt + F4` (창 닫기)
* `Ctrl + Shift + I` (개발자 도구)

게임에 할당된 키(Arrow Keys, KeyZ, KeyX, KeyC, KeyA~F 등)에 대해 `preventDefault()`를 호출하여 브라우저 기본 동작을 차단한다. `Ctrl`, `Alt` 조합 키에는 `preventDefault()`를 적용하지 않는다.

### 3.8. 게임패드 API 폴링 (Gamepad API Polling)

* `navigator.getGamepads()`를 매 프레임 `requestAnimationFrame` 콜백 내에서 폴링한다.
* 아날로그 스틱 데드존은 `gamepad_deadzone`(0.15) 값을 적용한다. 데드존 이내의 입력은 무시한다.
* 트리거(LT/RT)는 `gamepad_trigger_threshold`(0.3) 값을 초과해야 입력으로 인식한다.
* 동시에 2개 이상의 게임패드가 연결된 경우, 가장 먼저 연결된 패드(index 0)만 사용한다.

### 3.9. 키 리바인딩 (Key Rebinding — PC Only)

* PC(키보드/게임패드) 환경에서 키 리바인딩을 지원한다.
* 모든 게임 행동은 키 리바인딩이 가능하다.
* 하나의 행동에 1개의 키를 할당한다.
* 동일한 키를 2개 이상의 행동에 할당할 수 없다. 충돌 발생 시 기존 할당을 해제하고 경고 UI를 표시한다.
* 리바인딩 데이터는 `localStorage`에 JSON 형식으로 저장한다.
* "기본값으로 초기화" 버튼을 반드시 제공한다.
* `Escape`, `F11`은 리바인딩 대상에서 제외한다(시스템 예약 키).

## 5. 예외 처리 (Edge Cases)

### 5.1. (삭제됨 — 가상 패드 터치 영역은 모바일 미지원으로 삭제)

### 5.2. (삭제됨 — 화면 회전 대응은 모바일 미지원으로 삭제)

### 5.3. (삭제됨 — 멀티터치는 모바일 미지원으로 삭제)

### 5.4. 네트워크 지연 시 스킬 발동 처리 (Skill Activation Under Latency)

* 스킬 발동은 클라이언트 측에서 즉시 실행한다(Client-Side Prediction). 스킬 이펙트와 쿨다운 시작을 클라이언트에서 선 처리한다.
* 서버는 스킬 발동 요청을 검증한다(쿨다운 미완료, 자원 부족, 비정상 위치 등).
* 서버 검증 실패 시, 클라이언트의 예측 결과를 롤백한다. 쿨다운을 원복하고 스킬 이펙트를 취소한다.
* 서버 응답 지연이 `network_input_timeout_ms`(300ms)를 초과하면, 클라이언트 예측 결과를 유지하되 서버 보정 큐에 적재한다.
* 네트워크 단절이 `network_disconnect_threshold_ms`(5000ms)를 초과하면 "연결 끊김" UI를 표시하고 입력을 차단한다.

### 5.5. 포커스 해제 시 입력 초기화 (Input Reset on Focus Loss)

* `document.visibilitychange` 이벤트에서 `hidden` 상태를 감지하면 모든 키 상태를 릴리스 처리한다.
* `window.blur` 이벤트 발생 시에도 동일하게 전체 입력 상태를 초기화한다.
* 포커스 복귀 시 게임을 자동 일시정지 상태로 전환한다. 플레이어가 명시적으로 재개해야 게임이 진행된다.
* 포커스 해제 시 입력 버퍼도 함께 비운다.

### 5.6. 게임패드 핫플러그 (Gamepad Hot-Plug)

* **연결 시**: `gamepadconnected` 이벤트 → UI에 "게임패드 연결됨" 토스트 표시 → 입력 소스 자동 전환 → 화면 내 키 안내를 게임패드 아이콘으로 교체.
* **해제 시**: `gamepaddisconnected` 이벤트 → 게임 자동 일시정지 → UI에 "게임패드 연결 해제" 알림 표시 → 키보드+마우스 입력으로 복귀.
* 게임패드 해제 시점에 진행 중이던 행동(스킬 등)은 즉시 중단하고 캐릭터를 Idle 상태로 전환한다.

### 5.7. (삭제됨 — 모바일 가상 패드 최적화는 모바일 미지원으로 삭제)

### 5.8. 탭 비활성 상태에서의 타이머 (Timer in Inactive Tab)

* 브라우저는 비활성 탭에서 `requestAnimationFrame`을 중단하거나 `setTimeout`/`setInterval`을 1초 간격으로 제한한다.
* 탭 복귀 시 경과 시간(delta time)이 비정상적으로 큰 값을 가질 수 있으므로, delta time 상한을 `max_delta_time_ms`(100ms)로 제한한다.
* 쿨다운 타이머는 게임 내 경과 시간(game time)을 기준으로 동작하며, 실제 시계(wall clock)를 기준으로 하지 않는다.

## 검증 기준 (Verification Checklist)

* [ ] 모든 행동(이동, 점프, 공격, 스킬 1~4)이 단일 버튼 입력으로 실행되는가
* [ ] 키보드와 게임패드 모두로 아이템계(Abyss) 최심층까지 풀 플레이가 가능한가
* [ ] 스킬 4개 슬롯의 쿨다운 관리만으로 전투 깊이가 확보되는가 (단조롭지 않은가)
* [ ] 대시 버튼이 단일 입력으로 실행되며, 쿨다운 표시가 정확한가
* [ ] 자동 조준이 사거리 + 부채꼴 각도 내 최근접 적을 정확히 타겟팅하는가
* [ ] 자동 조준 후보가 없을 때 바라보는 방향으로 발사하는가
* [ ] 기본 공격 3타 자동 콤보가 연타만으로 정상 연결되는가
* [ ] 고정 높이 점프가 모든 플랫폼 구간에서 도달 가능한 레벨 디자인과 일치하는가
* [ ] PC 키보드, 게임패드 2종 입력이 동일한 게임 경험을 제공하는가
* [ ] 네트워크 지연 300ms 이상에서도 클라이언트 예측이 자연스럽게 동작하는가
* [ ] 서버 검증 실패 시 스킬 쿨다운이 올바르게 롤백되는가
* [ ] 포커스 해제 시 모든 키 상태가 초기화되고 복귀 시 일시정지 상태인가
* [ ] 게임패드 연결/해제 시 자동 전환 및 일시정지가 동작하는가
* [ ] 브라우저 단축키(Ctrl+W, F5 등)가 게임 입력으로 소비되지 않는가
* [ ] 키 리바인딩이 저장 및 복원되는가
