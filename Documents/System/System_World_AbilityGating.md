## 1. 개요 (Overview)

### 1-1. 정의

능력 게이트는 특정 능력 획득 이후에만 진입 가능한 탐험 구간이다. 플레이어는 "보이지만 갈 수 없는 곳"을 경험하고, 능력 획득 후 재방문하며 세계의 확장을 체감한다. 이는 메트로베니아 탐험의 핵심 루프를 형성한다.

### 1-2. 설계 의도

| 차원 | 의도 |
| :--- | :--- |
| **감정** (MDA 미학) | 발견의 기쁨(Discovery), 성취감(Accomplishment), 능력 성장의 직접 체감 |
| **심리** (SDT) | 자율성: 비선형 탐험 경로 선택 / 역량: 능력 획득으로 새 구간 도달 / 관계성: 세계와의 상호작용 |
| **기둥 정렬** | 탐험가 판타지: 능력을 하나씩 얻으며 세계를 뚫어나간다 |

## 2. 핵심 규칙 (Detailed Rules)

### 2-1. 6대 능력 해금 테이블

| 능력 | 획득 방식 | 획득 층위 | 획득 트리거 | 게이트 표현 (Design_Level_Standards §4-1 참조) |
| :--- | :--- | :--- | :--- | :--- |
| **대시** | 렐릭 발견 | 중앙 성채 (Tier 2) | 첫 아이템계 클리어 후 | 대시로만 건너는 수평 간격 (3-4 타일 갭) |
| **이단 점프** | 보스 처치 후 렐릭 | 지하 수로 | 수로 보스 처치 | 5-6 타일 수직 간격 |
| **벽 타기** | 보스 처치 후 렐릭 | 묘지/카타콤 | 카타콤 보스 처치 | 연속 수직 벽면 (발판 없음) + 텍스처 단서 |
| **안개 변신** | 비밀 방/렐릭 | 연구소 폐허 | 숨겨진 아이템 발견 또는 보스 처치 | 1타일 높이 격자/좁은 틈새 |
| **수중 호흡** | 보스 처치 후 렐릭 | 빙결 보존소 | 빙결 보존소 보스 처치 | 산소 게이지 초과 수중 구간 |
| **역중력** | Ancient 아이템 심층 | 심연의 구 | 아이템계 최심층 진입 | 천장에 배치된 통로/아이템 |

> **대시 변경 이력:** 기본 조작에서 렐릭 해금으로 전환 (DEC-006, 2026-04-07). 첫 아이템계 클리어 보상으로 획득. 이전 구간 재탐험 시 대시 전용 지름길 해금.

### 2-2. 능력 해금 순서 (Critical Path)

```
0단계: 중앙 성채 (기본 이동만 — 대시 없음)
  ↓ 첫 아이템계 클리어
1단계: 대시 획득 → 이전 구간 지름길 해금
  ↓ (자유 탐험 가능)
2단계: 지하 수로 → 이단 점프 획득
  ↓ (이전 층위 재탐험 + 새 층위 진입 가능)
3단계: 묘지/카타콤 + 연구소 폐허 (순서 자유) → 벽 타기 + 안개 변신 획득
  ↓
4단계: 빙결 보존소 → 수중 호흡 획득
  ↓
5단계: 천공의 정원 (모든 능력 활용)
  ↓
6단계: 심연의 구 → 역중력 (Ancient 아이템 진입 시)
```

**핵심 원칙:**
- 각 능력은 하나의 층위 보스와 연결된다 (예외: 안개 변신은 비밀 방 또는 보스).
- 동일 층위 내 능력은 최대 1개만 해금된다 (과잉 차단 방지).
- 비선형이지만 Critical Path는 명확하다: 1단계 없이는 다음 단계 불가.

### 2-3. 게이트 배치 규칙 (§4.1 참조 + Project Abyss 추가 규칙)

| 규칙 | 설명 | 검증 방법 |
| :--- | :--- | :--- |
| **보상 필수** | 게이트 너머는 반드시 보상(아이템, 로어, 숏컷)이 존재 | 빈 방이 없는지 확인 |
| **티저 배치** | 게이트 직전에서 게이트 너머를 볼 수 있게 설계 | 같은 화면 내 보상 가시성 확인 |
| **최대 3개** | 한 층위에 동일 능력 게이트는 최대 3개까지 | 층위별 게이트 개수 집계 |
| **Critical Path만** | 능력 게이트는 Critical Path에만 배치. 스탯 게이트와는 구분 | 경로 분류 명확화 |
| **포사이트닝** | 게이트 너머를 보여주되 명확히 도달 불가 상태로 설계 | 게이트 물리 파라미터와 캐릭터 능력 대비 |

### 2-4. 능력 간 시너지 매트릭스

| 조합 | 시너지 효과 | 활용 예시 |
| :--- | :--- | :--- |
| **이단 점프 + 벽 타기** | 수직 공간 극대 활용 (공중 기동 + 벽 점프) | 높고 좁은 탑 형태 구간 |
| **이단 점프 + 안개 변신** | 높이 도달 후 좁은 틈새 진입 | 높은 선반 위의 1타일 틈새 |
| **벽 타기 + 안개 변신** | 벽 슬라이드 중 안개 변신으로 답답한 벽 우회 | 연속 벽면 중 1타일 격자 |
| **수중 호흡 + 모든 능력** | 수심 깊은 곳의 능력 게이트 진입 | 물속 높이 도달, 벽 타기 |
| **역중력 + 이중 점프** | 천장 탐험 후 낙하 안전성 보장 | 역중력 구간 → 기본 구간 복귀 |

## 3. 수치 설계 (Formulas & Numeric Specifications)

### 3-1. 게이트 수직 간격 (Design_Level_Standards 참조)

**이단 점프 게이트:**
- 최소 간격: 5 타일 (기본 점프 4 + 이중 점프 2.5 = 6.5 가능)
- 최대 간격: 6.5 타일 (이중 점프 여유 0)
- 배치: 상승 선택지에서만 (내려가는 길에서는 게이트 아님)

**벽 타기 게이트:**
- 수직 벽면: 발판 없이 순수 벽 슬라이드만으로 상승
- 최소 높이: 3 타일 (기본 점프 높이 이상)
- 특징: 양쪽 벽면 모두 발판 없어야 함 (단측벽 X)

**안개 변신 게이트:**
- 틈새 높이: 정확히 1 타일 (2타일 이상이면 기본 점프로 통과)
- 폭: 1~2 타일
- 시각 단서: 안개/연기 이펙트, 틈새에서 빛 새어남

**수중 호흡 게이트:**
- 산소 게이지 초과: 기본 산소량(예: 30초) 초과 수중 구간
- 배치: 수심 깊은 곳의 상자, 통로 (호흡 기회 없는 연속 구간)

**역중력 게이트:**
- 천장 높이: 기본 5타일 이상 하강 가능한 높이
- 배치: 아이템 최심층(Level 4) 이상에만 배치

### 3-2. 층위별 능력 게이트 배치 맵

| 층위 | 이단 점프 | 벽 타기 | 안개 변신 | 수중 호흡 | 역중력 | 합계 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: |
| 중앙 성채 | — | — | — | — | — | 0 |
| 지하 수로 | 1(획득처) | 0 | 0 | 0 | 0 | 1 |
| 묘지/카타콤 | 1 | 1(획득처) | 0 | 0 | 0 | 2 |
| 연구소 폐허 | 2 | 1 | 1(획득처) | 0 | 0 | 4 |
| 빙결 보존소 | 1 | 1 | 1 | 1(획득처) | 0 | 4 |
| 천공의 정원 | 2 | 2 | 2 | 1 | 0 | 7 |
| 심연의 구 | 1 | 1 | 1 | 1 | 1(획득처) | 5 |

## 4. 흐름 (Flow)

### 4-1. 능력 획득 프로세스

```
보스 방 진입 → 보스 전투 → 보스 처치
    ↓
능력 렐릭 생성 (또는 비밀 방에서 발견)
    ↓
플레이어가 렐릭 획득 (자동 장착 또는 수동 장착)
    ↓
화면 팝업: "[능력명] 획득! [설명 + 활용 팁]"
    ↓
즉시 해당 능력 사용 가능 (층위 내 테스트 공간에서 사용)
    ↓
귀환 경로: 숏컷 또는 기존 루트
    ↓
능력을 활용한 새 층위 탐험 시작
```

### 4-2. 재방문 흐름 (이전 층위 → 새 게이트 발견)

```
1. 플레이어가 이전 층위 재방문
2. 새로 획득한 능력으로 진입 가능한 게이트 발견
3. "아, 저 벽도 뛰어넘을 수 있겠네!" → 자연스러운 탐험 유도
4. 게이트 진입 → 보상 발견 → 로어 텍스트 또는 숏컷 획득
5. 해당 층위 완성도 증가 (맵 API 기반 추적)
```

## 5. 연동 (Dependencies)

### 5-1. 입출력 계약

| 시스템 | 방향 | 데이터 | 설명 |
| :--- | :--- | :--- | :--- |
| **Character (System_3C_Character)** | 입력 | `currentAbilities: Ability[]` | 플레이어가 소유한 능력 목록 |
| **Level Design (Design_Level_Standards)** | 참조 | 게이트 수치 (타일, 높이, 물리) | 게이트 배치 기준값 |
| **Map Structure** | 출력 | `gateUnlocked: Gate[]` | 해금된 게이트 목록 |
| **Progression (World System)** | 출력 | `areaCompletion%` | 층위별 탐험 완료도 |
| **Item World** | 입력 | `isAncientItemActive` | 역중력 능력 활성 조건 |

### 5-2. 시스템 상호작용

**탐험 순환 (The Exploration Loop):**
```
플레이어 이동
  ↓
게이트 감지 (collision check: 지형 높이 vs 능력 파라미터)
  ↓
진입 불가? → 화면 아래 "요구 능력" 표시
  ↓
진입 가능? → 영역 진입 허용
  ↓
보상 발견 → 로어/스탯 증가
```

**시퀀스 브레이킹 정책:**
- 게이트 우회는 불가능하도록 설계 (물리적으로 임계값을 넘을 수 없음)
- 예외: 극도로 숙련된 플레이어의 버그 활용은 허용하되, 보상 진행은 막지 않음
- Critical Path 우회는 금지하되, 층위 내 선택적 경로 우회는 권장 (메트로베니아 정신)

### 5-3. 에지 케이스

| 상황 | 처리 | 근거 |
| :--- | :--- | :--- |
| 플레이어가 능력 없이 렐릭 상자 앞에서 맴돎 | 반복 표시: "요구 능력: [능력명]" | 플레이어 학습 보조 |
| 능력 획득 직후 게이트 자동 진입 | 선택지 제공: 지금 진입 또는 나중에 | 플레이어 자율성 존중 |
| 역중력 능력이 없는데 천장 통로 접근 | 일반 지형처럼 무시됨 (진입 불가) | 역중력은 최후 능력 |
| 멀티플레이: 파티 멤버 중 일부만 능력 소유 | 능력 보유자만 게이트 진입 가능 (1~4인) | 파티 구성의 전략성 부여 |

## 6. 검증 기준 (Acceptance Criteria)

### 기능적 검증

- [ ] 5대 능력이 각각 고유한 게이트를 생성하고 명확히 구분되는가
- [ ] Critical Path의 능력 해금 순서가 선형이며, 다른 순서로는 진행 불가한가
- [ ] 각 게이트의 물리 파라미터(높이, 폭, 틈새)가 Design_Level_Standards의 기준값과 일치하는가
- [ ] 게이트 너머 모든 영역에 보상이 배치되어 있는가 (빈 방 없음)
- [ ] 층위당 동일 능력 게이트가 3개를 초과하지 않는가

### 경험적 검증

- [ ] 플레이어가 게이트 직전에서 게이트 너머를 "보여야겠다"고 느끼는가 (포사이트닝)
- [ ] 능력 획득 후 즉시 활용할 수 있는 게이트가 가까이 있는가 (보상 기대감)
- [ ] 이전 층위 재방문 시 새로운 게이트 발견이 자연스럽게 유도되는가
- [ ] 능력 간 시너지 조합이 플레이어의 창의성을 자극하는가
