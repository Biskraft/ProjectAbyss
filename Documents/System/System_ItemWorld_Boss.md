## 1. 개요 (Concept)

### 1.1. 설계 의도 (Intent)

> **"보스는 아이템의 기억을 지키는 수문장이다. 처치하는 것이 아니라, 기억의 주인이 쌓은 삶의 무게를 넘어서는 것이다."**

아이템계 보스 시스템은 두 가지 설계 문제를 동시에 해결해야 한다.

첫째, **반복해도 재미있어야 한다.** 아이템계 보스는 수십, 수백 번 반복 도전하는 야리코미의 핵심 목표물이다. 월드 보스처럼 "1회성 클라이맥스"로 설계하면 반복에서 재미가 소진된다. 가중 랜덤 패턴 풀, 플레이어 성장과 연동된 도전 곡선, 짧고 명확한 전투 사이클이 반복 재미의 세 축이다.

둘째, **4등급 계층이 체감되어야 한다.** 아이템 장군(1지층)은 1~2분의 관문 보스이고, 아이템 대신(4지층+)은 5~10분의 극한 도전이다. 이 차이는 단순히 HP와 공격력 숫자의 차이가 아니라, 패턴 복잡도, 텔레그래프 압박, 아레나 크기, 페이즈 수의 구조적 차이여야 한다. 플레이어는 같은 "보스"를 치는 것이 아니라 다른 등급의 도전을 경험해야 한다.

내러티브 측면에서, 모든 보스는 그 아이템을 사용했던 자의 기억이 결정화된 존재다. 검의 아이템계 보스는 검사의 기억을, 지팡이의 아이템계 보스는 마법사의 기억을 체현한다. 에르다가 보스를 처치할 때마다 그 기억이 해방된다. "이 기억의 주인은 누구였을까"라는 질문이 야리코미의 감정적 동기가 된다.

### 1.2. 설계 근거 (Reasoning)

| 설계 결정 | 채택 이유 | 기각된 대안 |
| :--- | :--- | :--- |
| **4단계 등급 계층** | 각 지층이 독립적인 난이도 단위를 갖도록 함. 등급별로 구조적으로 다른 경험(패턴 수, 페이즈, 아레나) 제공 | 모든 보스 동일 구조 + 수치만 증가 — "스폰지 보스" 비판, 반복 재미 소멸 |
| **가중 랜덤 패턴 풀** | 패턴 학습 가능성 유지 + 매 도전의 신선함. Cuphead의 RNG 기반 패턴 조합 방식 차용 | 순수 고정 루틴 — 메타 공략 후 루틴 외우기 게임으로 전락 |
| **4레이어 모듈형 구성** | 본체 × 공격 모듈 × 수식어 × 아레나 조합으로 소수 에셋으로 최대 다양성 생성. 구현 비용 대비 콘텐츠 밀도 최대화 | 보스 완전 수동 제작 — 야리코미 반복 구조에서 충분한 다양성 확보 불가 |
| **HP 임계점 기반 페이즈 전환** | 예측 가능한 전환점이 학습 가능성 보장. 추가 패턴 소개의 자연스러운 계기 | 시간 기반 전환 — 리듬감 맞추기 어렵고 "느린 플레이어" 페널티 |
| **텔레그래프 등급별 배율** | 장군은 여유 있고 대신은 촉박함. 난이도를 숫자가 아닌 "읽을 시간"으로 표현 | 모든 등급 동일 텔레그래프 — 고등급 보스가 단순히 "더 빠른 쉬운 보스"가 됨 |
| **아레나 크기 등급 비례** | 대형 보스의 공간감과 이동 전략이 아레나 크기에 의존. 장군은 작고 아늑한 방, 대신은 넓고 위압적인 공간 | 모든 보스 동일 아레나 — 대형 보스의 시각적 압도감과 전략적 이동 공간 소멸 |
| **무기 유형 연동 외형** | 아이템계가 "내 무기의 기억"이라는 내러티브 일관성 확보. 검 아이템계와 지팡이 아이템계가 다른 보스를 보여줌 | 모든 아이템 동일 보스 외형 — 내러티브 몰입 붕괴, 아이템 수집 동기 약화 |

## 2. 메커닉 (Mechanics)

### 2.1. 4단계 보스 계층 (Four-Tier Boss Hierarchy)

아이템계의 모든 보스는 "기억의 문(Memory Gate)"이라는 공통 내러티브 프레임을 공유하되, 4가지 등급으로 구분된다. 각 등급은 독립적인 구조적 특성을 가지며 수치 조정만으로 차별화하지 않는다.

#### 등급별 핵심 프로파일

| 등급 | 한국어 | 영어 | 출현 지층 | 내러티브 정체 |
| :--- | :--- | :--- | :--- | :--- |
| Tier 1 | 아이템 장군 | Item General | 1지층 말단 | 기억의 수문장 (Memory Gatekeeper) — 가장 오래된 첫 기억의 문지기. 아이템을 처음 사용했던 자의 습관과 반사가 결정화됨 |
| Tier 2 | 아이템 왕 | Item King | 2지층 말단 | 기억의 군주 (Memory Sovereign) — 숙련의 기억. 사용자가 무기를 완전히 자신의 것으로 만들었던 시절의 정수 |
| Tier 3 | 아이템 신 | Item God | 3지층 말단 | 기억의 신 (Memory Deity) — 절정의 기억. 아이템을 사용한 인생에서 가장 강렬했던 순간들이 집결 |
| Tier 4 | 아이템 대신 | Item Overlord | 4지층 + 심연 | 기억의 핵 (Memory Core) — 기억의 본질. 아이템에 깃든 모든 기억의 원형이자 핵심. 이를 넘어서는 것은 아이템의 완전한 해방 |

### 2.2. 4레이어 모듈형 보스 구성 (Four-Layer Modular Boss Composition)

모든 아이템계 보스는 4개의 독립 레이어를 조합하여 생성된다. 동일한 등급이라도 아이템의 무기 유형, 레어리티, 생성 시드에 따라 다른 조합이 나타나 반복 플레이의 신선함을 제공한다.

보스 인스턴스 = 본체 템플릿 × 공격 모듈 세트 × 수식어 × 아레나 변형

| 검류 (Sword) | 갑옷 입은 기사 실루엣 | 쌍검 무사 | 기억의 검성 | 심연의 검 화신 |
| 대검류 (Greatsword) | 무거운 석상 | 지진의 거인 | 붕괴의 신 | 세계를 부수는 자 |

### 2.3. 보스 AI 구조: FSM + BT 하이브리드 (AI Architecture)

보스 AI는 상태 기계(Finite State Machine)와 행동 트리(Behavior Tree)를 계층적으로 결합한다.

- **FSM (외부 계층):** 전투 단계(Phase)와 상태(Idle/Engage/Attack/Stagger/Dead)를 관리
- **BT (내부 계층):** 각 상태 내에서 패턴을 선택하고 실행

```
[Phase FSM]
  ├─ IDLE         → 보스 방 미진입, 대기 애니메이션
  ├─ ENGAGE       → 플레이어 감지, 진입 연출, BT 활성화
  ├─ ATTACK       → [Pattern BT] 실행 중
  ├─ STAGGER      → 취약 피격, 일시 경직
  ├─ PHASE_CHANGE → HP 임계점 도달, 전환 연출
  └─ DEAD         → 처치 연출, 보상 지급, 다음 지층 문 개방
```

```
[Pattern BT] — ATTACK 상태에서 실행
  Root (Selector)
  ├─ 페이즈 전환 체크 (Condition: HP <= threshold?)
  │   └─ → PHASE_CHANGE 전환
  ├─ 쿨다운 체크 (Condition: 마지막 패턴 종료 후 delay 경과?)
  │   └─ 미경과 시: MOVE_IDLE (배회/포지셔닝)
  └─ 패턴 선택 (WeightedRandom: 활성 패턴 풀에서 선택)
      ├─ Pattern_A (가중치 W_A)
      ├─ Pattern_B (가중치 W_B)
      ├─ ...
      └─ Pattern_N (가중치 W_N)
```

#### 패턴 선택 가중치 조정 규칙

- 동일 패턴을 2회 연속 선택하지 않도록 직전 실행 패턴의 가중치를 ×0.1로 임시 감소
- 플레이어가 공중에 있을 때 대공 패턴(지상 투사체, 에어 플런지)의 가중치를 ×1.5로 증가
- 플레이어와 거리 `dist < 200px`일 때 근접 패턴 가중치 ×1.5, 원거리 패턴 가중치 ×0.5
- 분노 상태(Enraged)에서 모든 패턴의 쿨다운 delay를 ×0.6으로 감소

#### 상태 전환 조건 표

| 현재 상태 | 전환 조건 | 다음 상태 |
| :--- | :--- | :--- |
| IDLE | 플레이어가 감지 반경 내 진입 | ENGAGE |
| ENGAGE | 진입 연출 완료 | ATTACK |
| ATTACK | HP ≤ 페이즈 임계점 | PHASE_CHANGE |
| ATTACK | 취약 부위 피격 | STAGGER |
| ATTACK | HP = 0 | DEAD |
| STAGGER | 경직 시간 경과 (`staggerDuration`ms) | ATTACK |
| PHASE_CHANGE | 전환 연출 완료 | ATTACK |

### 2.5. 페이즈 전환 시스템 (Phase Transition System)

모든 페이즈 전환은 HP 임계점 기반으로 트리거된다. 페이즈 수는 등급에 따라 다르다.

#### 등급별 페이즈 구조

| 등급 | 페이즈 수 | P1 전환점 | P2 전환점 | P3 전환점 | 분노 상태 |
| :--- | :---: | :---: | :---: | :---: | :--- |
| 장군 | 1~2 | 없음 or 50% | 없음 | 없음 | 없음 |
| 왕 | 2~3 | 60% | 30% or 없음 | 없음 | P2 진입 시 5초 |
| 신 | 3 | 75% | 50% | 25% | P3 진입 시 10초 |
| 대신 | 4 | 75% | 50% | 25% | P4 전체 (HP 0까지) |

#### 페이즈 전환 시퀀스 (0.8초)

모든 페이즈 전환은 동일한 기본 시퀀스를 따른다.

| 단계 | 시간 | 내용 |
| :--- | :--- | :--- |
| 1. 취약 진입 | 0.0~0.2초 | HP 임계점 도달. 보스 일시 무적. 모든 진행 중 패턴 즉시 취소 |
| 2. 전환 연출 | 0.2~0.6초 | 등급별 전환 이펙트 재생. BGM 전환 (또는 변주). 보스 외형 부분 변화 |
| 3. 새 패턴 소개 | 0.6~0.8초 | 무적 해제. 새 페이즈 첫 패턴의 텔레그래프를 표준보다 1.5배 느리게 시작 (학습 기회 제공) |

> **HP 회복 금지 원칙:** 페이즈 전환 시 보스 HP를 회복시키지 않는다. HP 회복이 있는 전환은 플레이어를 실망시키고 전투를 지루하게 만든다. 유일한 예외는 수식어 MOD-03(재생)으로, 이는 전환이 아닌 지속 회복이다.

#### 페이즈별 패턴 풀 변화

| 페이즈 전환 | 패턴 변화 규칙 |
| :--- | :--- |
| P1 → P2 | 장판/구역거부 패턴 1개 추가 활성. 기존 근접 패턴 속도 ×1.1 |
| P2 → P3 | 투사체 패턴 1개 추가 활성. 동시 패턴 수 +1. 안전 윈도우 ×0.85 감소 |
| P3 → P4 (대신 전용) | 특수 패턴 SPC-05(무기 기억 해방) 활성. 분노 상태 영구 진입. 모든 패턴 속도 ×1.2 |

### 2.6. 텔레그래프 시스템 (Telegraph System)

모든 보스 공격은 반드시 다중 감각 텔레그래프를 제공한다. 텔레그래프 없는 공격은 설계 오류로 간주한다.

#### 5채널 텔레그래프 구성

| 채널 | 방법 | 표준 타이밍 | Project Abyss 구현 |
| :--- | :--- | :--- | :--- |
| 시각 (Visual) | 보스 예비 동작 애니메이션 | 300~600ms 전 | 공격 유형별 고유 선딜 모션 |
| 색상 (Color) | 공격 유형별 색상 | 200~400ms 전 | 빨간색=위험, 주황=추적 |
| 음향 (Audio) | SFX + 보이스 | 100~300ms 전 | 공격 분류별 고유 차지 SFX |
| 포지셔닝 (Position) | 공격 전 이동 패턴 | 500ms~1초 전 | 돌진 전 후퇴, 폭발 전 거리 확보 |
| UI 표시 | 화면 경고 | 즉시~200ms 전 | 화면 외 공격은 화살표, 장판은 바닥 표시 |

#### 등급별 텔레그래프 배율

텔레그래프 기준 시간을 배율로 조정한다. 기준 시간은 패턴 분류별로 정의된 표준값이다.

```yaml
telegraph_multiplier:
  tier_1_general:  1.2   # 표준보다 20% 더 긴 예고. 초보자 진입 용이
  tier_2_king:     1.0   # 표준값
  tier_3_god:      0.85  # 표준보다 15% 짧음. 반응 속도 요구
  tier_4_overlord: 0.70  # 표준보다 30% 짧음. 패턴 숙달 전제

# 분노 상태(Enraged)에서 추가 배율 적용
enraged_additional_multiplier: 0.85

# 최소 텔레그래프 보장값 (ms). 이 미만으로 내려가지 않음
min_telegraph_ms:
  simple_pattern: 240   # 단순 패턴 (점프 1가지 선택)
  complex_pattern: 340  # 복합 패턴 (회피 선택)
  aoe_pattern: 500      # 광역 공격 (완전 이동 필요)
  instakill_pattern: 800 # 즉사급 공격 (위험 구역 탈출)
```

### 2.8. 멀티플레이 보스 스케일링 (Multiplayer Scaling)

아이템계 보스는 솔로와 파티 모두에서 적절한 도전을 제공해야 한다.

#### HP 스케일링

SYS-IW-01 §2.5의 멀티플레이 규칙을 보스에 동일 적용한다. **SSoT는 SYS-IW-01이며 이 문서는 참조만 한다.**

```
Boss_HP(N명) = Base_HP × hp_scale[N]

hp_scale (SYS-IW-01 §2.5 기준):
  1인: ×1.0
  2인: ×1.5
  3인: ×2.0
  4인: ×2.5
```

#### 패턴 스케일링

HP 증가와 별도로 인원 수에 따라 패턴 조합이 변화한다.

| 인원 | 패턴 변화 |
| :---: | :--- |
| 1인 | 기본 패턴 세트. 단일 타겟 패턴 비율 높음 |
| 2인 | 기본 패턴 유지. 어그로 전환 시스템 활성화 |
| 3~4인 | AOE 패턴 1개 추가 활성. 공격 속도 ×(1 + (N-1)×0.1). 협동 기믹 패턴 활성화 |

> 어그로 시스템 상세: `Documents/System/System_ItemWorld_Core.md` (SYS-IW-01) 섹션 5.2 참조

#### 파티 협동 기믹 패턴

2인 이상에서 활성화되는 특수 패턴이다. 협동 기믹은 "강제"가 아닌 "보상 강화" 원칙으로 설계한다. 솔로로도 클리어 가능하지만 파티에서 기믹 성공 시 더 효율적이다.

## 3. 규칙 (Rules)

### 3.1. 보스 진입 규칙

- 보스 방 입장 시 방 입구 문이 잠긴다 (보스 처치 또는 사망 탈출 전까지 개방 불가)
- 보스 방 입장 즉시 HP/MP 오브가 8개 스폰된다 (진입 준비 완화)
- 보스 방 입장 시 BGM이 보스 전투 테마로 전환된다
- 왕 이상 등급 보스 방 직전에 세이브 포인트가 배치된다

### 3.2. 패턴 공정성 보장 규칙

모든 아이템계 보스에 반드시 적용해야 하는 공정성 원칙이다.

1. **클리어 불가 조합 금지:** 동시 활성 패턴들이 "회피 불가능한 조합"을 만들어서는 안 된다. 패턴 풀 조합 설계 시 모든 가능한 동시 조합이 클리어 가능한지 검증한다.
2. **안전 윈도우 최소 보장:** 패턴과 패턴 사이에 반드시 최소 `min_safe_window_ms` 이상의 안전 구간이 있어야 한다. 분노 상태에서도 0.5초 이하로 내려가지 않는다.
3. **텔레그래프 의무화:** 모든 공격은 `min_telegraph_ms` 이상의 텔레그래프를 반드시 제공한다. 텔레그래프 없는 공격이 검출되면 구현 오류로 처리한다.
4. **최소 접근 공간 보장:** 장판/구역거부 패턴이 아레나 전체를 덮어서는 안 된다. AOE-04(전체 폭발) 패턴에서는 반드시 안전 구역 최소 2개를 보장한다.
5. **반격 창 의무화:** 모든 페이즈에서 공격 회복 동작(Recovery) 이후 `min_attack_window_ms` 이상의 반격 창이 존재해야 한다.

### 3.3. 보스 사망 및 보상 지급 규칙

1. HP가 0이 되면 DEAD 상태로 전환된다
2. DEAD 전환 시 0.5초 마지막 타격 히트스탑 (최대 피드백) 적용
3. 마지막 타격 이펙트 재생 (등급별 차별화된 소멸 연출, 1.5~3초)
4. 소멸 연출 완료 후 보상 아이템 스폰
5. 보스 방 입구 문 개방
6. 지층 클리어 선택 UI 표시: "탈출 (귀환)" vs "계속 (다음 지층)"

### 3.4. 스태거 규칙

스태거(Stagger)는 특정 취약 부위 피격 시 발생한다.

| 스태거 조건 | 지속 시간 (`staggerDuration`) | 무적 여부 |
| :--- | :--- | :--- |
| 취약 부위 피격 (협동 기믹 등) | 5~10초 (기믹 의존) | 없음 |
| 페이즈 전환 | 0.8초 (전환 시퀀스 동안) | 있음 |

스태거 중 보스는 모든 패턴 실행이 중단되고 공격을 받는다. 스태거 종료 후 ATTACK 상태로 복귀하며 새 패턴 선택부터 시작한다.

### 3.5. 내러티브 연동 규칙

보스의 외형과 행동은 아이템의 무기 유형과 과거 사용자의 기억을 반영한다.

1. **무기 유형 → 본체 외형:** 레이어 1(본체 템플릿)은 무기 카테고리와 등급의 교차점으로 결정된다 (섹션 2.2 본체 템플릿 표 참조)
2. **원소 속성 → 패턴 시각화:** 무기에 원소 속성이 있으면 투사체와 장판 패턴이 해당 원소 색상으로 표현된다
3. **처치 후 기억 단편:** 대신(Tier 4) 처치 시 0.5~1초의 기억 단편 이미지가 화면에 플래시된다 (Phase 2 구현)
4. **에르다의 독백:** 지층 진입 시 에르다의 짧은 독백이 재생된다. "이 기억의 주인은 무엇을 위해 이 검을 들었을까." 대신 처치 후에는 감정적으로 더 깊은 독백이 재생된다

## 5. 예외 처리 (Edge Cases)

### 5.1. 클리어 불가 패턴 조합 발생 시

**상황:** 가중 랜덤 패턴 선택이 회피 불가능한 동시 조합을 만든 경우

**처리:** 패턴 BT의 패턴 선택 단계에서 "클리어 가능성 검증"을 실행한다.

```
checksum 함수: isComboFeasible(activePatterns[], arenaLayout)
  → 모든 활성 패턴의 위험 구역 합집합이 아레나 전체를 덮으면 false
  → false 반환 시 가장 최근 선택된 패턴을 재선택 (최대 3회 재시도)
  → 3회 재시도 후에도 false이면 강제로 안전 패턴(MEL-01, PRJ-01) 중 하나 선택
```

### 5.2. 네트워크 단절 (멀티플레이 중 보스 전투)

**상황:** 파티원 1인 이상이 보스 전투 중 접속 끊김

**처리:**
- 접속 끊긴 파티원은 "사망" 처리 (15초 부활 대기 없이 즉시)
- 보스 HP는 현재 값 유지. 단, 남은 인원에 맞게 HP 스케일링을 재계산하지 않는다 (재계산은 악용 가능)
- 남은 파티원이 0명이면 보스 전투 자동 종료, 현재 지층 재시작
- 접속 복귀 파티원은 현재 지층 재진입 가능 (보스 방 직전 체크포인트부터)

### 5.3. 보스 방 진입 중 캐릭터 사망

**상황:** 보스 방 입구 문이 잠기기 전 적 투사체에 맞아 진입 직후 사망

**처리:**
- DEAD 상태 전환 즉시 문 개방 잠금 해제
- 사망 탈출 처리 (SYS-IW-01 사망 패널티 적용)
- 보스는 IDLE 상태로 복귀

### 5.4. 수식어 MOD-04(분열) 사용 시 분신 처리

**상황:** 페이즈 전환 시 소환된 분신이 보스보다 높은 어그로를 유지하는 경우

**처리:**
- 분신은 독립적인 어그로 시스템을 갖지 않는다. 분신의 모든 공격은 실체 보스 패턴 풀과 완전히 독립된 고정 패턴(MEL-01 반복)만 사용
- 분신의 HP는 실체 보스 HP의 10%. 파괴 시 소멸 이펙트만 재생 (보상 없음)
- 분신은 페이즈 당 최대 1회 소환. 페이즈 내에서 분신 재소환 없음

### 5.5. 보스 HP가 페이즈 임계점과 정확히 일치하는 데미지로 처치된 경우

**상황:** HP가 26%인 보스에게 정확히 26%의 데미지를 줘서 즉사하는 경우 (P3 → P4 전환이 아닌 즉사)

**처리:**
- P3 → P4 페이즈 전환이 발생하지 않는다. HP 0 도달 판정이 페이즈 임계점 판정보다 우선한다
- 즉시 DEAD 상태로 전환
- 페이즈 전환 보상(없음)과 처치 보상 중 처치 보상만 지급

### 5.7. 보스 전투 중 탈출 아이템(Mr. Gency's Exit 계열) 사용

**상황:** 보스 전투 중 즉시 귀환 아이템 사용

**처리:**
- 보스 방 문 잠금 해제
- 귀환 처리 (SYS-IW-01 탈출 아이템 귀환 규칙 적용)
- 보스 진행 상태 없음. 다음 진입 시 해당 지층 처음부터 재시작

### 5.8. 동일 등급 보스 연속 도전 시 패턴 가중치 조정

**상황:** 동일 아이템의 동일 지층을 재방문하여 같은 보스와 재전투

**처리:**
- 이전 도전에서 3회 이상 반복 등장한 패턴의 가중치를 ×0.7로 임시 감소 (세션 내 한정)
- 세션 종료(허브 귀환) 시 가중치 원복
- 이 조정은 학습된 플레이어가 더 다양한 패턴을 경험하게 하는 "숙달 보상 신선함 시스템"이다

### 8.2. 경험 검증 (Experiential Criteria)

다음은 플레이테스트를 통해 검증해야 하는 경험 기준이다.
