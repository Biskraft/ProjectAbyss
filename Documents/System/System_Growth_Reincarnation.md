## 1. 개요 (Concept)

### 1.1. 설계 의도 (Intent)

Project Abyss의 아이템 전생 시스템을 다음 한 문장으로 정의한다:

> "전생은 리셋이 아니라 재탄생이다. 장비를 녹여 다시 벼림으로써, 아이템은 기억을 더 깊이 담을 수 있는 그릇이 된다."

에르다 벤-나흐트의 직업 정체성인 "대장장이의 전생" 개념을 게임 메커닉에 직결한다. 망치 에코로 장비를 두드려 아이템계에 진입하는 것과 마찬가지로, 전생은 장비를 다시 용광로에 녹여 더 강한 형태로 재주조하는 행위다.

- 전생은 아이템 레벨과 이노센트 배치를 리셋하지만, 적성치(Aptitude) 누적과 이노센트 슬롯 확장을 영구적으로 부여한다.
- 전생 후 아이템은 이전 사이클보다 빠르게 성장한다. 이것이 자발적 리셋의 핵심 합리성이다.
- 전생 횟수가 누적될수록 아이템은 서사적으로 진화하며, 스크린에서도 시각적 변화가 나타난다.

### 1.2. 설계 근거 (Reasoning)

| 결정 | 근거 | 참고 문헌 |
| :--- | :--- | :--- |

## 2. 메커닉 (Mechanics)

### 2.1. 전생의 정의와 위치

전생(Reincarnation)은 아이템 장비 하나를 대상으로 실행되는 단일 행위이다. 플레이어는 전생 대상 아이템을 선택하여 다음 위치에서 전생을 실행할 수 있다.

| 실행 위치 | 조건 | 설명 |
| :--- | :--- | :--- |
| Hub 대장간 | 허브 진입 후 대장간 NPC 상호작용 | 에르다가 에코로 장비를 제단 위에 올려 두드리는 풀 연출 재생 |
| 필드 균열 제단 | 월드 구역 내 제단 발견 후 상호작용 | 간략 연출. 허브 대장간보다 소재 비용 10% 추가 |
| 기억의 방랑자 제단 | 방랑자 임시 제단 생성 후 상호작용 | 단발 사용 가능. 보상 배율 +5% 보너스 |

### 2.2. 전생 조건

아이템이 전생 가능 상태가 되려면 다음 조건을 모두 충족해야 한다.

1. 해당 아이템의 아이템계 모든 지층을 최소 1회 이상 클리어한 상태
   - 예시: Rare 아이템 → 3개 지층 전부 클리어 기록 존재
2. 전생 소재 보유량이 요건 이상
   - 소재 수치는 `Sheets/Content_System_Reincarnation.csv` 확정 후 기입
3. 아이템이 현재 아이템계에 진입 중이 아닌 상태 (플레이어가 해당 아이템 내부에 있지 않음)

조건 미충족 시, 전생 UI에서 미충족 항목이 시각적으로 표시되고 실행 버튼이 비활성화된다.

### 2.3. 전생 시 리셋 항목

전생 실행 시 다음 항목이 초기화된다.

| 리셋 항목 | 초기화 내용 | 보존 여부 |
| :--- | :--- | :--- |
| Item Level | 0으로 초기화 | 보존 안 됨 |
| 이노센트 복종 상태 | 복종(Tamed) → 야생(Wild)으로 환원 | 보존 안 됨 (단, 반복 야생 적용 — §4.2 참조) |
| 아이템계 지층 클리어 기록 | 초기화 (재도전 필요) | 보존 안 됨 |
| 지층 배율 스탯 보너스 | 초기화 | 보존 안 됨 |

### 2.4. 전생 시 보존 및 강화 항목

전생 실행 시 다음 항목은 보존되거나 영구적으로 강화된다.

| 보존/강화 항목 | 내용 | 누적 방식 |
| :--- | :--- | :--- |
| 적성치(Aptitude) | 전생 횟수에 따라 스탯 기반 배율 영구 상승 | 횟수별 단계 누적 (§3.1) |
| 이노센트 슬롯 추가 | 최초 1회 전생 시 슬롯 +1 | 레어리티 상한 내에서 추가 |
| 아이템 설명 텍스트 | 전생 기록이 설명 하단에 누적 | 전생 횟수별 텍스트 진화 (§2.6) |
| 아이템 외형 이펙트 | 전생 횟수에 따라 외형 글로우 등급 상승 | 횟수별 단계 적용 (§2.6) |
| 아이템 기본 이름 | 변경 없음 (단, 전생 기록 부제 추가) | - |

### 2.5. 전생 실행 플로우

```
[플레이어] 전생 대상 아이템 선택
           ↓
[시스템] 전생 조건 점검
  ├─ 지층 클리어 완료? ─→ NO: UI에 미완료 지층 표시, 실행 불가
  └─ 소재 보유 충족? ─→ NO: 소재 부족 표시, 실행 불가
           ↓ 조건 충족
[UI] 전생 미리보기 표시
  - "잃는 것": 현재 Item Level, 이노센트 복종 수, 지층 보너스
  - "얻는 것": 적성치 +N%, 이노센트 슬롯 변화, 전생 기록 번호
           ↓ 플레이어 확인 선택
[시스템] 소재 차감
           ↓
[연출] 전생 애니메이션 재생 (Hub 대장간 풀 연출 / 필드 제단 간략 연출)
           ↓
[시스템] 리셋 항목 초기화
           ↓
[시스템] 적성치 보너스 영구 적용
           ↓
[시스템] 이노센트 슬롯 갱신 (1회 전생인 경우 +1)
           ↓
[시스템] 아이템 설명 텍스트 업데이트
           ↓
[UI] 전생 완료 화면 표시 (전생 횟수, 적성치 현황, 아이템 외형 미리보기)
           ↓
[플레이어] 아이템계 재진입 또는 다른 활동으로 복귀
```

### 2.6. 전생 횟수별 서사 및 외형 변화

전생 횟수에 따라 아이템의 시각/서사 표현이 단계적으로 진화한다.

| 전생 횟수 | 아이템 부제 예시 (검 기준) | 외형 이펙트 | 해금 콘텐츠 |
| :--- | :--- | :--- | :--- |
| 0회 (초기) | "낡은 검" | 없음 | - |
| 1회 | "단련된 검" | 희미한 흰색 글로우 | 이노센트 슬롯 +1 |
| 3회 | "전승된 검" | 청색 맥동 글로우 | 전생 아이템계 전용 지층 해금, 특수 이노센트 합성 가능 |
| 5회 | "기억을 새긴 검" | 황금색 윤곽선 | Ancient 레어리티 승급 자격 |
| 10회 | "기억의 결정체" | 초록 심층 글로우 (Ancient 드랍 연출 수준) | 스탯 상한 해제 (야리코미 극한), 전생 기념 칭호 획득 |

아이템 설명 텍스트 하단에 전생 기록이 누적된다. 예시:

```
[낡은 검]
오래된 단조 기술로 만든 검. 무게 중심이 약간 앞쪽에 있다.

── 기억 기록 ──
1번째 벼림: 지층을 모두 돌았다. 어떤 검이 되려는 것인지 이제 보이기 시작했다.
3번째 벼림: 두 번의 불 속에서도 형태는 남았다. 이 검은 아직 더 담을 수 있다.
```

## 3. 수치 설계 (Numerical Design)

### 3.1. 적성치(Aptitude) 누적 공식

적성치는 전생 횟수에 따라 단계별로 누적되는 영구 스탯 배율이다. 모든 FinalStat 계산에서 기본 적성치 100%에 누적 보너스가 더해진다.

공식:
```
FinalStat(전생 포함) = FinalStat(기본) × (1.0 + AptitudeBonus)
AptitudeBonus = Σ(전생 단계별 추가율)
```

| 전생 횟수 | 해당 전생의 추가율 | 누적 AptitudeBonus | 비고 |
| :---: | :---: | :---: | :--- |
| 1회 | +5% | +5% | 1회 전생 이노센트 슬롯 +1 동시 적용 |
| 2회 | +5% | +10% | - |
| 3회 | +5% | +15% | 전생 전용 지층 해금 |
| 4회 | +5% | +20% | - |
| 5회 | +5% | +25% | Ancient 승급 자격 해금 |
| 6회 | +3% | +28% | 수확 체감 시작 |
| 7회 | +3% | +31% | - |
| 8회 | +3% | +34% | - |
| 9회 | +2% | +36% | - |
| 10회 | +2% | +38% | 스탯 상한 해제 |
| 11회 이상 | +1%/회 | 누적 | 소프트 캡 유지 |

> **설계 원칙:** 1~5회 전생은 빠른 보상으로 자발적 리셋 동기를 형성한다. 6회 이후에는 수확 체감으로 무한 스택 인플레이션을 억제한다. 10회 이후는 극한 야리코미 플레이어 전용 영역이다.

예시 계산:
```
3회 전생 Rare 검, Item Level 15, FinalSTR(기본) = 200
AptitudeBonus = 15%
FinalSTR(전생 포함) = 200 × 1.15 = 230
```

### 3.2. 전생 소재 요건 (초안)

수치는 `Sheets/Content_System_Reincarnation.csv` 확정 전 설계 가이드라인이다.

| 레어리티 | 소재 1 (전생석) | 소재 2 (기억 결정) | 소재 3 (심연 먼지) | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| Normal | 5개 | 0개 | 0개 | 입문 전생 |
| Magic | 8개 | 2개 | 0개 | - |
| Rare | 12개 | 5개 | 0개 | - |
| Legendary | 20개 | 10개 | 3개 | - |
| Ancient | 30개 | 20개 | 10개 | 심연 지층 클리어 필수 |

소재 획득 원천:
- 전생석: 아이템계 지층 클리어 확정 드랍 (지층 수 × 1~2개)
- 기억 결정: 이노센트 보스 처치 확정 드랍
- 심연 먼지: 아이템계 심연 지층 전용 드랍

> **설계 원칙:** 정상적인 아이템계 플레이 과정에서 자급자족 가능한 수량으로 설정한다. 별도 소재 전용 파밍 없이 전생 1회에 필요한 소재가 해당 아이템의 지층 클리어 과정 중 자연스럽게 수집되어야 한다.

### 3.3. 이노센트 반복 야생 보정

전생 후 이노센트가 야생으로 복귀할 때, 이전 전생 횟수만큼 복종 시도 성공률이 상승한다.

공식:
```
복종 성공률(반복 야생) = 기본 복종 성공률 × (1 + 0.1 × 반복 횟수)
반복 횟수 = 해당 이노센트가 이 아이템에서 복종된 총 횟수
```

예시:
```
기본 복종 성공률 50%, 반복 횟수 2회
복종 성공률(반복 야생) = 50% × (1 + 0.2) = 60%
```

상한: 최대 85%. 이노센트 포획의 긴장감을 유지하기 위해 100%로 올라가지 않는다.

### 3.4. 전생 후 성장 속도 비교

전생이 실제로 더 빠른 성장을 보장함을 수치로 보여주는 예시.

| 단계 | 전생 0회 (초기) | 전생 1회 (적성치 +5%) | 차이 |
| :--- | :--- | :--- | :--- |
| Item Level 5 도달 | 기준 시간 T | T × 0.95 (5% 빠름) | -5% 시간 |
| Item Level 10 도달 | 기준 시간 T2 | T2 × 0.93 | -7% 시간 (보스 추가 보너스 복합) |
| 이노센트 3마리 복종 | 기준 시간 T3 | T3 × 0.90 | -10% 시간 (반복 야생 보정) |

> **참고:** 이 수치는 플레이테스트 검증 대상이다. 목표는 전생 후 "체감 가능한 속도 차이"가 나타나되, 전생 미수행 플레이어가 열세를 느끼지 않는 수준이다.

## 4. 흐름 (Flow)

### 4.1. 전생 루프의 장기 사이클

```
[아이템 획득 (월드 드랍)]
           ↓
[아이템계 진입 — 지층 클리어 + 이노센트 수집]
           ↓
[아이템 강화 한계 도달]
  - Item Level 상한
  - 이노센트 슬롯 포화
  - 지층 스탯 보너스 적용 완료
           ↓
[전생 결정 (플레이어 자유 선택)]
           ↓
[전생 실행 — 소재 소비 + 리셋 + 적성치 영구 획득]
           ↓
[아이템계 재진입 — 이전보다 빠른 사이클]
           ↓
[다음 전생 목표 인식 또는 Ancient 승급 자격 확인]
           ↓
(루프 반복)
```

### 4.2. 반복 야생 이노센트 처리

전생 후 이노센트가 야생으로 환원될 때, 시스템은 해당 이노센트의 이전 복종 기록을 내부적으로 유지한다.

```
[전생 실행]
           ↓
[이노센트 복종 상태 → 야생 환원]
           ↓
[내부 반복 횟수 카운터 +1]
           ↓
[아이템계 재진입 시 야생 이노센트 재등장]
           ↓
[복종 시도 — 반복 야생 보정 성공률 적용]
           ↓
[복종 성공 시 이전보다 낮은 저항으로 복종]
```

### 4.3. 장기 목표 계층과 전생의 위치

| 목표 계층 | 전생 관련 단기/중기/장기 목표 | 달성 보상 |
| :--- | :--- | :--- |
| 단기 (현재 세션) | 이번 아이템계 사이클에서 전생 소재 N개 수집 | 소재 직접 획득 |
| 중기 (수 세션) | 전생 조건 충족 (전 지층 클리어 완료) | 전생 실행 자격 |
| 장기 (수 주) | 전생 3회 달성 — 전생 전용 지층 해금 | 새 콘텐츠 접근 |
| 야리코미 극한 | 전생 5회 — Ancient 승급 자격, 10회 — 스탯 상한 해제 | 야리코미 최정점 상징 |

## 5. 연동 (Integration)

### 5.1. System_Growth_LevelExp.md와의 연동

- 캐릭터 레벨(Lv 1~60)은 전생의 영향을 받지 않는다. 아이템 전생은 장비 레이어에서만 작동한다.
- Item Level이 리셋되면 `ItemExpToNext` 계산 기준도 초기화된다. (SYS-LVL-02 §2.3 참조)
- 적성치 보너스는 `FinalStat` 계산 시 별도 곱셈 레이어로 추가된다.

```
FinalStat(최종) = (BaseStat + EquipStat + InnocentBonus) × (1.0 + AptitudeBonus)
```

### 5.2. System_Equipment_Rarity.md와의 연동

- 이노센트 슬롯 추가(1회 전생 시 +1)는 레어리티별 상한을 초과할 수 없다.
  - 예시: Normal 아이템(슬롯 상한 2)에 전생 슬롯 추가를 실행하더라도 상한이 2이면 적용 불가.
  - 예외: Magic 이상 아이템은 레어리티 상한 이내에서 +1 적용.
- 전생 5회 달성 시 Ancient 승급 자격은, 해당 레어리티 아이템이 Legendary인 경우에만 유효하다. Normal/Magic/Rare는 레어리티 승급 조건이 별도 존재한다. (System_Equipment_Rarity.md 참조)

### 5.3. System_ItemNarrative_Template.md와의 연동

- 전생 실행 시 아이템 설명 텍스트에 전생 기록 한 줄이 추가된다.
- 텍스트 생성 규칙은 `System_ItemNarrative_Template.md` §전생 기록 섹션을 따른다.
- 전생 횟수별 서사 부제("낡은 검" → "기억의 결정체")는 해당 문서에서 정의한 템플릿을 아이템 유형별로 치환하여 생성한다.

### 5.4. Design_Progression_Reward.md와의 연동

- 이 문서의 전생 메커닉 전체는 `Design_Progression_Reward.md` §7의 "전생 철학"을 시스템으로 구현한 것이다. 철학 정의와 재구축 근거는 D-11이 SSoT이다.
- 수치 인플레이션 관리는 §6.3 "Low Power Gap + High Rarity Scaling" 원칙을 따른다. 전생 적성치 누적은 이 원칙 내에서 작동한다.

### 5.5. Design_Yarikomi_Philosophy.md와의 연동

- 전생은 야리코미의 "리셋 함수"이다. 지층 재도전을 유발하는 핵심 메커닉이다.
- 세션 리듬의 "한 지층만 더" 심리가 전생 후 다시 1지층부터 시작함으로써 재점화된다.
- 탈출 판단 메커닉(긴급 탈출 소모품)은 전생 소재 획득 효율에도 영향을 줄 수 있으므로 Phase 3 조정 대상이다.

### 5.6. 데이터 시트 연동

| 시트 파일 | 내용 | 상태 |
| :--- | :--- | :--- |
| `Sheets/Content_System_Reincarnation.csv` | 소재 요건 수치, 적성치 단계별 수치, 이노센트 반복 야생 성공률 | 추후 생성 |

## 6. 튜닝 가이드 (Tuning Guide)

### 6.1. 적성치 누적 속도 조정

파라미터: `AptitudeGainPerReincarnation[n]` (전생 횟수 n별 추가율 배열)

| 시나리오 | 조정 방향 | 기대 효과 | 부작용 |
| :--- | :--- | :--- | :--- |
| 전생 동기 부족 (플레이어가 전생 안 함) | 1~3회 추가율 +2~3% 상향 | 초기 전생의 수치 이득 강조 | 인플레이션 가속 위험 |
| 전생 남용 (너무 자주 전생함) | 6회 이후 체감 강화 (-1%씩 추가 감소) | 적성치 소프트 캡 효과 강화 | 야리코미 극한 플레이어 동기 약화 |
| 전생 후 성장 속도 차이 안 느껴짐 | 반복 야생 보정 상한을 85% → 90%로 상향 | 재복종 체감 속도 개선 | 이노센트 수집 긴장감 약화 |

### 6.2. 전생 소재 수급 속도 조정

파라미터: `ReincarnationMaterialDrop[레어리티][지층]` (지층별 드랍 수량)

현재 설계 목표: 해당 레어리티 아이템의 전 지층을 한 번 클리어하면 전생 소재의 70~80%가 수집되도록 한다. 나머지 20~30%는 이전 파밍 세션에서 자연 누적된 잉여 소재로 충당.

| 조정 항목 | 느린 경우 | 빠른 경우 |
| :--- | :--- | :--- |
| 전생석 드랍 수 | 지층당 +1개 | 지층당 -1개 |
| 기억 결정 드랍 | 보스 처치 시 +1 추가 보장 | 이노센트 보스만 드랍 유지 |

### 6.3. 전생 연출 시간 조정

| 실행 위치 | 현재 연출 시간 | 조정 범위 | 비고 |
| :--- | :--- | :--- | :--- |
| Hub 대장간 (풀 연출) | 5~7초 | 3~10초 | 처음 전생 시에는 스킵 불가, 이후 스킵 가능 |
| 필드 균열 제단 (간략) | 2~3초 | 1.5~4초 | 항상 스킵 가능 |

> **설계 원칙:** 첫 번째 전생 연출은 스킵 불가로 설정하여 "이벤트"의 감각을 부여한다. 반복 전생에서는 플레이어 효율을 위해 스킵을 허용한다.

## 7. 예외 처리 (Edge Cases)

### 7.1. 아이템계 진입 중 전생 시도

아이템계 내부에 있는 동안 해당 아이템은 전생 대상으로 선택 불가능하다. 전생 UI에서 해당 아이템은 "현재 진행 중" 상태로 표시되고 선택이 잠긴다.

### 7.2. 전생 직후 접속 끊김

전생 처리는 서버에서 원자적으로 실행된다. 소재 차감 → 리셋 → 적성치 적용의 세 단계 중 어느 단계에서 연결이 끊겨도, 복구 시 마지막 성공한 단계 이후 상태로 복원한다.

복구 우선순위: 소재 미차감 상태에서 끊긴 경우 소재 환급 처리. 적성치 미적용 상태에서 끊긴 경우 재연결 시 자동 적용.

### 7.3. 이노센트 슬롯 초과 후 전생

전생 전에 이미 슬롯 +1이 적용된 아이템(예: Magic 아이템으로 전생 1회 완료 → 슬롯 4)이 재전생 시, 두 번째 전생의 슬롯 추가는 레어리티 상한 초과로 인해 무효화된다. 플레이어에게 "슬롯 추가 없음" 안내 팝업을 표시한다. 적성치 누적 보너스는 정상 적용된다.

### 7.4. Ancient 아이템의 전생

Ancient 아이템은 현재 아이템계 최상위 레어리티이므로 레어리티 승급 보너스가 없다. Ancient 전생 시:
- 적성치 누적 정상 적용
- 이노센트 슬롯 추가 적용 (슬롯 상한 8슬롯 내에서)
- "아이템 대신(Item Overlord)" 보스 처치로만 슬롯 추가 가능한 특수 슬롯은 전생 후에도 보존됨

### 7.5. 레어리티 승급 직후 전생

레어리티 승급(Normal → Magic 등)이 이루어진 직후, 전생 가능 여부는 새 레어리티 기준의 전 지층 클리어를 요구한다. 이전 레어리티의 지층 클리어 기록은 승급 시 초기화된다. 이것은 레어리티 승급과 전생이 독립적인 마일스톤으로 작동하도록 하기 위함이다.

### 7.6. 전생 소재 수량이 모자란 경우

소재 부족 상태에서 전생을 시도하면 실행 버튼이 잠기고 부족한 소재 항목이 강조 표시된다. 강조 표시 항목에 "어디서 수집하는가" 링크 정보(해당 소재 드랍 지층 정보)를 추가로 표시한다.

### 7.7. 멀티플레이 중 파티원 아이템 전생

파티 플레이 중에는 파티원의 아이템을 전생할 수 없다. 전생은 항상 자신의 아이템에만 적용된다. 파티 공유 아이템 기능은 현재 설계에 존재하지 않는다.
