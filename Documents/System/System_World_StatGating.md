## 1. 개요 (Concept)

### 1.1. 설계 의도 (Intent)

> "스탯 게이트는 탐험에 벽을 만들고, 그 벽을 넘기 위해 플레이어를 아이템계로 자연스럽게 유도한다. 강해지고 돌아올 때의 쾌감이 야리코미의 원동력이다."

스탯 게이트(Stat Gate)는 장비 ATK 또는 INT 수치가 임계값 이상일 때만 통과 가능한 비-크리티컬 경로의 장벽이다. 능력 게이트(이단 점프 등)가 "새 능력을 얻으면 간다"는 순간 쾌감을 제공한다면, 스탯 게이트는 "스탯이 더 높아야 간다"는 압박을 통해 **장비 강화와 아이템계 진입을 자연스럽게 동기부여**한다. 스탯 게이트는 ATK(물리 장벽) 또는 INT(마법 봉인) 수치로 판정한다. 장비 강화(아이템계)로 ATK/INT를 올려 게이트를 해금한다.

### 1.2. 설계 근거 (Reasoning)

| 결정                            | 근거                                                                                       |
| :------------------------------ | :----------------------------------------------------------------------------------------- |
| ATK/INT 이중 스탯 게이트        | ATK(물리 장벽)와 INT(마법 봉인) 두 축으로 빌드 다양성 확보. 물리 빌드와 원소 빌드 모두에 탐험 목표 부여 |
| 비-크리티컬 경로에만 배치       | Critical Path의 자유도와 접근성 보호. 선택적 보상 구간이므로 임계값을 높게 설정 가능      |
| "현재 + IW 1~2회 클리어"로 설정 | 최소 2주기 루프를 강제. 한 번의 아이템계 클리어만으로는 모든 게이트를 뚫지 못해 재진입 동기 유지 |
| 숫자 명시 (모호함 금지)         | "파괴 가능한 벽" 추상 표현 X, "ATK 150 필요" 또는 "INT 80 필요" 구체적 명시. 플레이어가 목표를 정확히 인지 |
| 층위 난이도와 연결              | 난이도 1~5 층위에 맞춰 게이트 임계값을 스케일. 층위 내 위치도 스탯 게이트 난이도 곡선 준용 |

### 1.3. 3대 기둥 정렬 (Pillar Alignment)

| 기둥                  | 스탯 게이팅에서의 구현                                                                 |
| :-------------------- | :------------------------------------------------------------------------------------ |
| **메트로베니아 탐험**  | ATK/INT 임계값이 월드의 비선형 경로를 개방. "저 벽을 부수려면 ATK를, 저 봉인을 풀려면 INT를 올려야 한다" = 탐험 목표 창출 |
| **아이템계 야리코미**  | 스탯 게이트 발견 → 막힘 → 아이템계 진입 → 장비 ATK/INT 강화 → 재방문 → 해금의 자연스러운 유도 |
| **온라인 멀티플레이**  | 파티원 전체의 장비 강화 수준이 진행 가능 범위를 결정. 높은 ATK/INT 게이트는 파티 전체의 성장을 요구 |

### 1.4. 저주받은 문제 검증 (Cursed Problem Check)

| 문제                                                   | 설계의 선택                                                                  |
| :----------------------------------------------------- | :--------------------------------------------------------------------------- |
| 스탯 게이트가 너무 높으면 진입 기피                    | 현재 장비 + IW 1~2회로 도달 원칙 준수. 단회 클리어 불가는 X                |
| 스탯 게이트가 너무 낮으면 자동으로 뚫려 의미 상실     | 기본 능력만으로는 불가능한 높이. 최소 1회 아이템계 진입 필수                |
| ATK/INT 이중 게이트로 충분한 다양성이 있는가           | ATK(물리)와 INT(마법) 두 축이 빌드 다양성을 보장. 물리 빌드는 ATK 게이트 우선, 원소 빌드는 INT 게이트 우선. 능력 게이트와 조합으로 추가 다양성 확보 |
| 게이트 조건이 불명확하면 답답함                       | 숫자로 정확히 명시. "ATK 120 필요" — 스탯 패널에서 현황과 비교 가능        |
| 비-크리티컬이라는 개념이 혼동되지 않는가               | 매뉴얼과 UI에서 명확히 구분. "선택적 보상 경로"로 안내                      |

### 1.5. 위험과 보상 (Risk & Reward)

| 전략                  | 위험 (Risk)                          | 보상 (Reward)                                |
| :-------------------- | :----------------------------------- | :------------------------------------------- |
| ATK 극대화 빌드       | HP/INT 부족, 마법 봉인 해금 불가    | 높은 ATK 게이트 조기 해금 + 빠른 물리 진행   |
| INT 극대화 빌드       | HP/ATK 부족, 물리 장벽 해금 불가    | 높은 INT 게이트 조기 해금 + 강력한 원소 데미지 |
| 밸런스 빌드 (ATK+INT) | 양쪽 게이트 해금이 느림             | 전투 안정성 확보 + 양쪽 게이트 점진적 해금   |
| 아이템계 집중 파밍     | 월드 탐험 진행이 늦어짐             | 고레어리티 장비로 ATK/INT 대폭 상승 후 다수 게이트 한꺼번에 해금 |

## 2. 핵심 규칙 (Core Rules)

### 2.1. ATK/INT 스탯 게이트 정의

스탯 게이트는 ATK 또는 INT 수치로 판정한다. 장비 강화(아이템계)로 ATK/INT를 올려 게이트를 해금한다.

| 스탯 ID | 스탯명    | 게이트 표현               | 환경 예시               | 담당 시스템                |
| :------ | :-------- | :----------------------- | :---------------------- | :------------------------ |
| ATK     | 공격력    | 물리 장벽 (파괴 가능한 벽/바리케이드) | 금이 간 벽, 쇠사슬, 중문, 바위 | System_Combat_Damage.md    |
| INT     | 지력      | 마법 봉인 (룬 장벽/마법 잠금) | 룬 문양 장벽, 마법 봉인 문, 아케인 잠금장치 | System_Combat_Damage.md    |

> **설계 변경 이력:** 기존 6대 스탯 게이트(STR/INT/DEX/VIT/SPD/LCK)는 ATK/INT 이중 게이트로 재설계되었습니다. ATK는 물리 장벽(파괴), INT는 마법 봉인(해제)을 담당합니다.

### 2.1.1. 능력 게이트 (Ability Gate) — 렐릭 해금

스탯 게이트와 별도로, 특정 **렐릭(Relic)**을 획득해야 통과할 수 있는 능력 게이트가 존재한다. 렐릭은 월드 보스 처치 또는 특정 탐험 지점에서 획득한다.

| 순서 | 렐릭 | 능력 | 게이트 표현 | 획득 시기 (예상) |
| :--- | :--- | :--- | :---------- | :-------------- |
| 1 | 대시 렐릭 | 대시 (Dash) — i-frame 150ms, 쿨다운 2초 | 좁은 틈 + 위험 구간 (타이밍 통과 불가 → 대시 관통 가능) | Tier 1 후반 ~ Tier 2 초반 |
| 2 | 벽타기 렐릭 | 벽 타기 (Wall Climb) | 수직 벽면 (점프로 도달 불가한 높이) | Tier 2 ~ 3 |
| 3 | 이단점프 렐릭 | 이단 점프 (Double Jump) | 넓은 간격 (단일 점프 사거리 초과) | Tier 3 |
| 4 | 안개변신 렐릭 | 안개 변신 (Mist Form) | 격자/철창 (실체 통과 불가) | Tier 4 |
| 5 | 수중호흡 렐릭 | 수중 호흡 (Aqua Breath) | 침수 구간 (장시간 잠수 불가) | Tier 4 ~ 5 |
| 6 | 역중력 렐릭 | 역중력 (Reverse Gravity) | 천장 경로 (일반 이동 불가) | Tier 5 ~ 6 |

#### 능력 게이트 설계 원칙

- **렐릭은 C키(렐릭 슬롯)에 배정되지 않는다.** 대시만 C키에 배정되며, 나머지 렐릭은 자동 발동(벽 접촉 시 벽타기, 공중 점프 시 이단점프 등) 또는 상황 발동이다.
- **렐릭 획득 전 게이트는 "보이지만 통과 불가"** — 포사이트닝(Foresightning). 플레이어는 "나중에 올 수 있겠구나"를 인지한다.
- **렐릭 획득 후 재방문은 항상 보상** — 이전에 통과 못 한 곳에 아이템/숏컷/비밀이 있다.
- **대시는 가장 먼저 획득하는 렐릭** — 전투 깊이(후딜 캔슬, i-frame 회피)와 탐험 기동성을 동시에 확장하는 핵심 능력이다.

> **스탯 게이트 vs 능력 게이트:** 스탯 게이트는 아이템계(야리코미)로 유도하고, 능력 게이트는 월드 탐험(메트로베니아)으로 유도한다. 이 이중 구조가 PA의 순환 루프를 만든다.

### 2.2. 게이트 배치 원칙

#### 위치 기준

- **크리티컬 경로 (Critical Path):** 스탯 게이트 배치 금지. 기본 스탯만으로 모든 주요 층위를 통과할 수 있어야 한다.
- **비-크리티컬 경로 (Non-Critical Path):** 선택적 보상 구간, 숨겨진 보상실, 제한된 자원 구간 등에만 배치.
- **층위 내 위치:** 층위 난이도 곡선과 일치. 입구는 낮은 임계값, 보스 근처나 깊숙이는 높은 임계값.

#### 밀도 기준

| 층위 난이도 | ATK 게이트 | INT 게이트 | 능력 게이트 | 평균 간격  | 비고                                  |
| :--------- | :--------- | :--------- | :--------- | :--------- | :------------------------------------ |
| 1 (중앙성채) | 0-1개      | 0개        | 대시 (Tier 1 후반) | N/A (선택지) | 튜토리얼 층위. 대시 렐릭이 첫 능력 게이트 |
| 2 (지하 수로) | 1-2개      | 0-1개      | 벽타기      | 2-3개 Room  | 스탯 게이트 첫 도입 + 벽타기 게이트 |
| 3 (연구소 폐허) | 1-2개      | 1-2개      | 이단점프    | 2-3개 Room  | ATK+INT 혼합 + 이단점프 게이트     |
| 4 (천공의 정원) | 1-2개      | 1-2개      | 안개변신, 수중호흡 | 2-3개 Room  | 높은 ATK/INT + 복합 능력 게이트    |
| 5 (심연의 구) | 2-3개      | 1-2개      | 역중력      | 1-2개 Room  | 극한 임계값 + 역중력                |

#### 전체 게이트 유형 배분 비율 (R7 리서치 적용)

게임 전체 스탯 게이트의 유형 배분은 아래 비율을 원칙으로 한다. 이 비율이 ATK 빌드와 INT 빌드의 탐험 목표를 균등하게 분배하면서, 초반 ATK 우위 → 중반 교차 → 후반 INT 우위라는 파워 커브와 일치하도록 설계된다.

| 게이트 유형              | 전체 배분 비율 | 주 배치 시기       | 설계 의도                                                              |
| :----------------------- | :------------: | :----------------- | :--------------------------------------------------------------------- |
| ATK 게이트 (물리 장벽)   | **55%**        | 초반~중반 층위 집중 | 초반 ATK 빌드가 더 자연스럽게 진행하도록. 신규 플레이어 적응 지원      |
| INT 게이트 (마법 봉인)   | **35%**        | 중반~후반 층위 집중 | 중반 원소 해금 이후 INT 빌드의 탐험 목표를 지속 제공                  |
| 복합 게이트 (ATK+INT 동시) | **10%**      | 선택 경로 최심부   | 하이브리드 빌드에만 이점이 되는 희귀 경로. Ancient 등급 보상 영역 배치 |

> **배치 원칙 상세:**
> - ATK 게이트는 층위 1~3에 비율이 높다. 층위 4~5로 갈수록 INT 게이트 비중이 증가한다.
> - INT 게이트는 층위 2 첫 등장(지하 수로) 이후 점진적으로 증가하며, 층위 4~5에서 ATK 게이트와 거의 동등해진다.
> - 복합 게이트는 층위당 0~1개로 제한. 아이템계 深 지층 클리어 보상 영역 또는 Ancient 전용 경로에만 배치.
> - 이 비율은 플레이테스트 이후 조정 가능하나, 복합 게이트가 10% 초과하면 하이브리드 빌드가 독점 경로를 갖게 되어 전문화 압력이 희석된다.

### 2.3. 게이트 UX — "발견 → 블로킹 → 명시"

#### Phase 1: 발견 (Discovery)

플레이어가 게이트 앞에 도달했을 때:
- 게이트의 시각적 단서가 명확히 보인다 (금이 간 벽 / 룬 문양 / 움직이는 톱니).
- 화면 범위 내에서 게이트 너머 보상이 보여야 한다 (Teaser).

#### Phase 2: 블로킹 (Blocking)

플레이어가 통과 시도할 때:
- 캐릭터가 게이트에 막히며 거부 모션 (밀려남, 불이 튕기는 등).
- HUD에 현재 스탯과 필요 스탯을 비교하는 UI 팝업: **"ATK 120 필요. 현재: ATK 85"** 또는 **"INT 80 필요. 현재: INT 45"**
- 게이트 오브젝트 위에 숫자 플래그: **"ATK 120"** 또는 **"INT 80"**

#### Phase 3: 명시 (Clarification)

게이트 옆에 항상 텍스트 표시:
- ATK 게이트: "이 벽은 강한 공격력(ATK)으로만 부술 수 있다." / "필요: ATK 120, 보유: ATK 85"
- INT 게이트: "이 봉인은 강한 마력(INT)으로만 해제할 수 있다." / "필요: INT 80, 보유: INT 45"
- 방법: "장비를 강화하거나 이노센트(Gladiator/Tutor)를 수집하면 스탯이 올라간다."

### 2.4. 해금 연출 (Phase 2 예정)

플레이어가 충분한 스탯으로 재방문했을 때:
- 게이트 파괴 시 시각/음성 피드백 (폭발, 금이 부서짐, "우르르" 소리).
- 배경 변화 (빛이 새어나옴, 봉인 해제 이펙트).
- 조건 만족 시 자동 진입 또는 진입 버튼 하이라이트.

## 3. 수치 설계 (Numerical Design)

### 3.1. 임계값 설정 원칙

**기본 공식:**
```
Gate_Threshold = (Base_Stat + Equip_Stat_MVP) + (EquipStat_IW_1clearCycle * 1.5)
```

단계적으로:
1. **Base_Stat**: 캐릭터 기본 스탯 (Lv 1, 기본 장비)
2. **Equip_Stat_MVP**: MVP 구간 월드에서 획득 가능한 장비 스탯
3. **EquipStat_IW_1clearCycle**: 월드 장비 + 아이템계 1회 클리어 후 강화된 장비 스탯

**목표:** 현재 장비만으로는 불가능하지만, 아이템계 1~2회 클리어 후 재방문하면 해금 가능.

### 3.2. 층위×스탯 게이트 임계값 매트릭스 (예시)

| 층위 | 층위 난이도 | ATK 임계값 | INT 임계값 | 비고                              |
| :--- | :--------: | :--------: | :--------: | :------------------------------- |
| 중앙 성채 | 1 | - | - | 게이트 없음 (튜토리얼) |
| 지하 수로 | 2 | 50~75 | 30~50 | 낮은 게이트 (첫 도입) |
| 묘지/카타콤 | 2 | 60~80 | 40~60 | 낮은~중간 게이트 |
| 연구소 폐허 | 3 | 90~110 | 60~80 | 중간 게이트 (INT 게이트 다수) |
| 빙결 보존소 | 3 | 100~120 | 70~90 | 중간~높은 게이트 |
| 천공의 정원 | 4 | 130~150 | 90~110 | 높은 게이트 |
| 심연의 구 | 5 | 170~200 | 120~150 | 극한 게이트 |

**주의:** 위 수치는 **예시**이며, 실제 수치는 다음 단계에서 플레이테스트 후 조정된다.

### 3.3. 아이템계 보상과 임계값 연동

| 아이템계 클리어 횟수 | 획득 장비 ATK 범위 | 예상 도달 가능 ATK | 비고 |
| :-: | :--- | :--- | :--- |
| 0회 (기본) | Base + Equip | ATK 40~65 | 게이트 차단 |
| 1회 클리어 | +20~30% ATK 증가 | ATK 55~85 | 낮은 게이트 해금 |
| 2회 클리어 | +40~50% ATK 증가 | ATK 75~120 | 중간 게이트 해금 |
| 3회 이상 | +60%+ ATK 증가 | ATK 120+ | 높은 게이트 해금 |

## 4. 흐름 설계 (Flow)

### 4.1. 스탯 게이트 발견→해금 사이클

```
[월드 탐험]
  └─> 비-크리티컬 경로 발견
       └─> 스탯 게이트 앞 도달
            ├─ [발견] 시각적 단서 명시 + 필요 ATK 표시 (HUD)
            ├─ [블로킹] 현재 ATK로 통과 시도 → 실패
            └─ [판단] "ATK를 올려야 한다" 인식
                 └─> [아이템계 진입 동기 발생]

[아이템계 진입]
  └─> 해당 층위의 장비 아이템 진입
       └─> 지층 클리어 (1~2회)
            └─> 장비 강화 + ATK 증가
                 └─> [월드로 복귀]

[월드 재방문]
  └─> 동일 게이트 재도전
       ├─ [성공] ATK 충분 → 게이트 해금 → 보상 확보
       └─ [부분 성공] ATK 아직 부족 → 추가 아이템계 클리어 필요
            └─> 이중 순환 강화
```

### 4.2. 플레이어 의사결정 포인트

| 상황 | 플레이어 선택 | 시스템 반응 |
| :--- | :--- | :--- |
| 낮은 게이트 (ATK 60) 앞에서 현재 ATK 70 | "통과 시도" | 성공 → 보상 확보 |
| 중간 게이트 (ATK 100) 앞에서 현재 ATK 70 | "통과 시도" | 실패 → 아이템계 권유 |
| " | "아이템계 진입" | 장비 ATK 강화 후 복귀 |
| 높은 게이트 (ATK 150) 앞에서 현재 ATK 110 | "통과 시도" | 실패 → 더 많은 아이템계 필요 인식 |

## 5. 연동 (Integration)

### 5.1. System_Growth_Stats.md와의 연동

- **ATK 합산 공식**: `Final_ATK = Base_ATK + Equipment_ATK + Innocent_ATK_Bonus`
- **INT 합산 공식**: `Final_INT = Base_INT + Equipment_INT + Innocent_INT_Bonus`
- **게이트 판정**: ATK 게이트: `Final_ATK ≥ Gate_Threshold ? Success : Blocked` / INT 게이트: `Final_INT ≥ Gate_Threshold ? Success : Blocked`
- **임계값 설정 기준**: 장비 레어리티와 아이템계 강화 수준에 따른 ATK/INT 성장 곡선과 일치

### 5.2. System_ItemWorld_Core.md와의 연동

- **클리어 보상**: 보스 처치 시 장비 ATK 영구 증가 (IWC-06-A)
- **지층 수 레어리티 비례**: 고레어리티 장비 = 더 많은 지층 = 더 많은 보상
- **순환 진입**: 더 높은 레어리티 아이템의 아이템계일수록 더 높은 ATK 증가 → 높은 임계값 게이트 해금

### 5.3. System_Equipment_Rarity.md와의 연동

- **레어리티 배율**: `EquipATK = BaseEquipATK * Rarity_Multiplier`
- **고레어리티 장비**: x2.2 (Legendary) ~ x3.0 (Ancient) 배율
- **게이트 임계값 충족**: 같은 gear level에서 고레어리티 장비는 낮은 아이템계 진입으로 ATK 임계값 달성 가능

### 5.4. Design_Metroidvania_Philosophy.md와의 연동

- **§4.1 이중 게이트**: 능력 게이트와 스탯 게이트의 공존
- **§4.2 포사이트닝/티징**: 게이트 너머 보상이 화면 내에서 보이게 배치
- **백트래킹의 쾌감**: 스탯 게이트 해금 후 재방문 시 숨겨진 경로 개방

### 5.5. Design_Level_Standards.md와의 연동

- **§4-2 스탯 게이트 설계**: 시각적 단서, 환경 표현, 이펙트
- **게이트 너머 보상 배치**: 반드시 보상 존재, 빈 방 금지
- **배치 체크리스트**: 스탯 게이트가 비-크리티컬 경로에만 있는지 확인

### 5.6. System_World_ProcGen.md와의 연동

- **층위 난이도와 게이트 임계값 연동**: 난이도 1~5 층위에 맞춘 스탯 게이트 분포
- **Room Template 생성**: 스탯 게이트 오브젝트 배치 데이터 포함
- **절차적 생성 보상**: 게이트 클리어 보상(아이템, 금화, 이노센트)도 Room Template에 연동

## 6. 예외 처리 (Edge Cases)

| 상황 | 처리 |
|:-----|:-----|
| 이노센트 해제/제거로 스탯 하락 → 이미 해금한 게이트 역전 | 게이트 해금은 1회 판정(일회성)이다. 한 번 해금된 게이트는 이후 스탯이 하락해도 다시 잠기지 않는다. 해금 상태는 서버에 영구 기록 |
| 장비 교체로 ATK/INT 하락 시 미해금 게이트 접근 | 장비 교체 후 스탯이 임계값 미만이면 게이트는 잠긴 상태를 유지한다. 교체 직전 HUD에 경고("이 장비로 교체하면 ATK가 120 → 85로 하락합니다")를 표시 |
| 멀티플레이에서 파티원 스탯 불일치 | 스탯 게이트는 각 플레이어 개별 판정이다. 파티 리더가 해금해도 다른 파티원의 Final_ATK/INT가 부족하면 통과 불가. 부족 파티원에게 "ATK 부족" UI를 표시하고 대기 |
| 버프/디버프로 일시적 스탯 변동 | 스탯 게이트 판정은 버프/디버프를 제외한 기본 스탯(Base + Equipment + Innocent)만 사용한다. 임시 버프로 게이트를 우회하는 것을 방지 |
| 게이트 임계값이 현재 레어리티 한계 초과 | 게이트 앞에 "권장 장비 레어리티: Rare 이상" 힌트를 추가 표시한다. 해당 층위에서 획득 가능한 최고 레어리티 장비 + IW 2회 클리어로 반드시 도달 가능하도록 임계값 검증 |

- [ ] 모든 ATK/INT 게이트가 비-크리티컬 경로에 배치되었는가?
- [ ] Critical Path가 어떤 ATK/INT 게이트도 요구하지 않는가?
- [ ] 각 게이트 ATK/INT 임계값이 "현재 + IW 1~2회" 원칙에 따라 설정되었는가?
- [ ] 각 게이트 앞에 필요 ATK 또는 INT가 숫자로 명시되어 있는가?
- [ ] 게이트 너머에 보상(아이템, 금화, 경험치, 비밀)이 존재하는가?
- [ ] 게이트 너머 보상이 게이트 직전에서 시각적으로 보이는가?
- [ ] 층위 난이도가 ATK 게이트 임계값과 일치하는가?
- [ ] 게이트 시각적 단서(파괴 가능한 벽/이펙트)가 일관되는가?
- [ ] 같은 층위 내 여러 ATK 게이트가 너무 높지 않아 플레이어를 좌절시키지 않는가?
- [ ] 아이템계 클리어 보상과 월드 ATK 게이트 임계값이 선형으로 진행하는가?

## 용어 정리 (Terminology)

| 용어 | 정의 |
| :--- | :--- |
| **스탯 게이트 (Stat Gate)** | ATK 또는 INT가 임계값 이상일 때만 통과 가능한 환경 장벽 |
| **임계값 (Threshold)** | ATK 또는 INT가 도달해야 하는 최소 수치 (예: ATK 100, INT 80) |
| **비-크리티컬 경로** | 플레이어가 선택적으로 탐험할 수 있는 경로, 주요 진행에 영향 없음 |
| **크리티컬 경로** | 게임을 완료하기 위해 반드시 통과해야 하는 경로 |
| **아이템계 클리어 사이클** | 월드 → 아이템계 진입 → 지층 클리어 → 강화 → 월드 복귀의 1회 반복 |
| **Final_ATK** | 캐릭터의 최종 ATK 수치 (Base_ATK + Equipment_ATK + Innocent_ATK_Bonus 합산) |
| **Final_INT** | 캐릭터의 최종 INT 수치 (Base_INT + Equipment_INT + Innocent_INT_Bonus 합산) |
