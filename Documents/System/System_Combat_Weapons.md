## 1. 개요 (Concept)

### 1.1. 설계 의도 (Intent)

ECHORIS 무기 시스템은 다음 한 문장으로 정의한다:

> "무기 하나를 고르는 순간, 그 플레이어의 전투 언어가 결정된다"

무기는 ATK 수치 컨테이너가 아니다. 카테고리가 기본 공격의 사거리, 타격 속도, 커버 범위를 결정하며, 각 무기의 시그니처 메커닉이 전투의 리듬과 포지셔닝 전략을 차별화한다. Blade를 든 플레이어는 3타 피니셔의 강력한 일격을 위해 콤보를 완성하고, Shiv를 든 플레이어는 적의 뒤를 노려 배후 강타를 터뜨리며, Railbow를 든 플레이어는 대시 직후 카이팅 샷으로 안전 거리에서 딜을 쌓는다.

**7종 무기 체계 설계 철학:** 근접 5종(Blade/Cleaver/Shiv/Harpoon/Chain)은 사거리×속도 5분할 구도를 형성하고, 원거리 2종(Railbow/Emitter)은 물리(ATK)와 파동(INT) 축으로 분화한다. Emitter는 유일한 INT 스케일링 무기로, INT 스탯의 존재 이유를 담당한다.

야리코미 기둥과의 정렬: 각 무기는 독자적인 아이템계 공간을 보유한다. 무기 레어리티 상승 → ATK/INT 증가 → 스탯 게이트 해금 → 탐험 범위 확장이라는 순환의 직접 동력이다. 플레이어가 어떤 무기를 선택하느냐는 야리코미의 방향성을 결정하는 첫 번째 판단이다.

## 2. 핵심 규칙 (Core Rules)

### 2.1. 무기 공통 규칙

1. 플레이어는 무기 슬롯 1개를 보유하며 동시에 1개 무기만 장착 가능하다.
2. 무기 슬롯이 비어 있으면 기본 공격 버튼 입력은 무시된다. (맨주먹 공격 없음)
3. In-Combat 상태에서 무기 교체 시도 시 시스템이 거부하고 UI 잠금 아이콘을 표시한다.
4. Out-of-Combat 전환 후 `out_of_combat_delay_s`(기본값 3.0초) 경과 시 교체 잠금 해제.
5. 레어리티가 동일한 경우 무기 카테고리 간 기본 ATK는 동일하다. 카테고리 간 성능 차이는 히트박스, 콤보 속도, 시그니처 메커닉으로만 발생한다.

### 2.2. 콤보 진행 공통 규칙

콤보 흐름의 상세 규칙은 `System_Combat_Action.md` 2.2절(자동 콤보 시스템)을 따른다. 본 문서에서는 무기별 타수와 파라미터만 정의한다.

1. 공격 버튼 연타 시 자동으로 콤보가 진행된다. 방향 입력은 불필요하다.
2. 콤보 마지막 타격 이후 `combo_end_lag_ms` 프레임 동안 후딜이 발생한다.
3. 후딜 중 공격 버튼 입력 시 콤보가 1타부터 재시작된다.
4. 공중 상태에서 지상 콤보 입력은 공중 전방 공격으로 전환된다. (하방 입력 + 공격 버튼 = 공중 하방 공격)

### 2.3. 시그니처 메커닉 공통 규칙

1. 시그니처 메커닉은 모두 패시브다. 별도 버튼 입력 없이 조건 충족 시 자동 발동한다.
2. 시그니처 메커닉의 데미지 배율은 ATK 기본값에 곱해진다. 크리티컬 배율과는 별도로 계산된다.
3. 각 시그니처의 시각적 큐(히트 스파크 색상 변경, 아이콘 점등 등)는 `System_Combat_HitFeedback.md`의 CMB-07-O, CMB-07-P 규칙을 따른다.

## 4. Blade 시그니처: 3타 강화 (Heavy Finisher) 상세 규칙

### 4.1. 3타 강화 메커닉 개요

> "콤보를 끝까지 이어가면, 마지막 일격이 결정적이다"

Blade의 시그니처는 **3타 피니셔의 고데미지**다. 1타(×1.0)와 2타(×1.1)는 가볍지만, 3타(×1.5)가 1-2타 합산(×2.1)보다 약간 낮아도 단일 타격으로 가장 강하다. 콤보를 끝까지 완료하는 것 자체가 보상이다. 별도 게이지, 스택, 조건이 없다. **탭 세 번으로 강한 일격** — Blade는 가장 단순하고 직관적인 무기다.

**데미지 배율:**

| 타격 | 배율 | 누적 합 | 비고 |
| :--- | :--- | :--- | :--- |
| 1타 | ×1.0 | 1.0 | 가벼운 수평 베기 |
| 2타 | ×1.1 | 2.1 | 대각 베기 |
| **3타** | **×1.5** | **3.6** | **대회전 피니셔 — 콤보 총 데미지의 42%가 3타에 집중** |

**3타 피니셔 특성:**
- 히트박스 38×24px (전 타격 중 최대)
- 히트스탑 4프레임 (1-2타의 2배)
- 넉백 80px (1-2타의 2배)
- 인챈트 원소 적용 시 3타에서만 상태이상 확률 2배

Blade는 **기준 무기(baseline)**다. 다른 6종 무기는 Blade와의 차이로 자신의 정체성을 정의한다. 따라서 Blade의 시그니처는 "특별한 조건"이 없어야 한다.

### 9.3. 밸런스 기준

- [ ] Blade DPS가 Shiv 시그니처 풀 활용(배후 강타 100%) DPS와 ±10% 이내로 유지된다
- [ ] Emitter가 원소 최적 조합에서 근접 무기보다 15% 이상 높은 DPS를 기록하지 않는다
- [ ] Harpoon이 일렬 3체 적 기준 Blade 대비 실효 DPS 1.25배를 초과하지 않는다
- [ ] Chain 끝단 강타가 적중률 50% 기준 Blade DPS의 1.1배를 초과하지 않는다
- [ ] 아이템계 16룸 표준 구성에서 어떤 무기도 클리어 불가(실질적 막힘) 상황을 발생시키지 않는다

## 10. 예외 처리 (Edge Cases)

| 상황 | 처리 |
|:-----|:-----|
| 무기 교체 중 피격 | Out-of-Combat 전환 후 `out_of_combat_delay_s` 대기 중 피격 시 교체 UI가 자동 닫히고 In-Combat 상태로 복귀한다. 교체 잠금 타이머가 리셋된다 |
| 콤보 도중 타겟 사망으로 3타 헛스윙 | 3타 히트박스가 적중하지 않아도 후딜(`combo_end_lag_ms`)은 정상 발생한다. 시그니처 게이지/배율은 적중 판정 시에만 적용되므로 헛스윙 시 발동하지 않는다 |
| ATK 0 무기 장착 시 | ATK 0 무기는 데미지 공식에서 최종 데미지가 0 이하로 계산된다. 최소 보장 데미지 1을 적용한다 (`max(1, calculated_damage)`). 히트 피드백은 라이트 등급으로 정상 출력 |
| 시그니처 발동 조건과 히트스탑 충돌 | 히트스탑 중 콤보 윈도우 타이머가 정지하므로 시그니처 판정 시간이 실질적으로 연장된다. 히트스탑 해제 후 시그니처 상태를 재평가한다 |
| 공중 하방 공격 후 즉시 착지 | 하방 공격 바운스가 발생하지 않는 높이(지면 직상 2px 이내)에서는 바운스를 생략하고 착지 상태로 즉시 전환한다. 후딜은 지상 착지 프레임으로 대체 |
| Harpoon 관통 도중 벽 충돌 | 관통 히트박스는 지형을 무시하지 않음. 벽 충돌 시 즉시 소멸. 통과한 적 수에 따른 감쇠는 누적 유지 |
| Chain 끝단 판정 경계 케이스 | 사거리 끝단 12px 판정은 적 hitbox 중심 기준. 적이 이동 중이면 히트 프레임 시점의 위치로 판정 |
