# Wiki Note 2026-04-28

## Item World / Beginner Grace

- Beginner assistance is handled as a real buff, not as scattered enemy or boss exceptions.
- The active buff id is `beginner_grace`.
- The buff adds configurable attack and defense values to the player.
- The buff is removed after the first Item World boss is defeated.
- In `?debug=1`, the HUD debug text shows the active buff state.

## Debug HUD Toggle

- `Shift+I` toggles the global debug info state.
- Previous issue: `Game.ts` consumed the `INVENTORY` input before `ItemWorldScene` could handle `Shift+I`, so the Item World debug HUD toggle did not work.
- Current rule: HUD debug text follows the global `Debug.infoVisible` state.
- `?debug=1` shows debug HUD text by default.
- `Shift+I` hides or shows the debug HUD text after that.

## Item World First-Entry / Onboarding

- The first normal Item World keeps the short 2x2 tutorial layout until the first boss is defeated.
- `clearedRooms` must not end the first-entry override by itself.
- If the player exits with ESC and re-enters before defeating the first boss, the tutorial layout should still apply.
- On first boss clear, store the `BOSS_KILLED` event so onboarding does not repeat on later entries.

## Parallax Tiling Attempt

- Tried changing large-map parallax scrolling to a `TilingSprite.tilePosition` based implementation for maps such as `Shaft_01`.
- The attempt was reverted by request.
- Current parallax behavior remains the previous implementation:
  - create a large world-space `TilingSprite`
  - move each parallax layer container based on camera position

## Verification

- After reverting the parallax attempt, `npm run build` passed.
- Remaining build warnings are the existing resource/chunk warnings.
